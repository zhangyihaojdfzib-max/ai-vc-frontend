Vercel Sandbox 现在会在会话之间自动保存和恢复文件系统状态。持久化功能默认开启，这意味着无需手动管理快照或跟踪状态。

每个 Sandbox 都有一个持久且可自定义的名称，作为项目中的唯一标识。你可以通过名称创建、检索或恢复 Sandbox。Vercel 会自动启动和关闭会话，而不会中断你的工作流程。

## 创建持久化 Sandbox

当你调用 `Sandbox.create()` 时，持久化功能默认启用：

```
1import { Sandbox } from "@vercel/sandbox";2
34const sandbox = await Sandbox.create({ name: "my-sandbox" });5
6await sandbox.runCommand("npm", ["install"]);7
8await sandbox.stop();
```

默认创建持久化 Sandbox

每个自动快照都会消耗快照存储空间，该费用与计算费用分开计费。对于临时工作负载，可选择关闭持久化以降低存储成本：

```
1import { Sandbox } from "@vercel/sandbox";2
3const sandbox = await Sandbox.create({ persistent: false });4
56await sandbox.update({ persistent: false });
```

创建临时 Sandbox

若要通过 CLI 关闭持久化，请在 `sandbox create` 命令后添加 `--non-persistent` 参数。非持久化 Sandbox 会在会话结束时丢弃其文件系统。

## 恢复持久化 Sandbox

恢复是自动进行的。对已停止的 Sandbox 调用任何方法（如 `runCommand()` 或 `writeFiles()`），都会从最近的快照启动一个新会话。

```
1import { Sandbox } from "@vercel/sandbox";2
3const resumedSandbox = await Sandbox.get({ name: "my-sandbox" });4
56await resumedSandbox.runCommand("npm", ["test"]);
```

自动恢复已停止的 Sandbox

## 其他改进

- `Sandbox.fork()`：从现有 Sandbox 创建新 Sandbox
- `Sandbox.getOrCreate()`：用于长期运行 Sandbox 的幂等检索或创建操作
- `Sandbox.delete()`：永久删除 Sandbox
- 更丰富的 `sandbox.stop()`：返回快照元数据以及活跃 CPU 和网络传输总量
- 生命周期钩子：`onCreate` 和 `onResume` 钩子，适用于 `create`、`get` 和 `getOrCreate`
- 标签：为 Sandbox 分配自定义属性，用于多租户跟踪

## 开始使用

升级到最新版本，即可默认创建持久化 Sandbox：

- `pnpm install @vercel/sandbox@latest` # SDK
- `pnpm install -g sandbox@latest`    # CLI

在文档中了解更多关于持久化 Sandbox 的信息。
