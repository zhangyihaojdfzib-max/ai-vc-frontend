Vercel Workflow 现在无需任何代码更改即可对所有用户数据进行端到端加密。工作流输入、步骤参数、返回值、Hook 负载和流数据在写入事件日志之前会自动加密。

这使得跨边界传递敏感数据（如 API 密钥、Token 或用户凭证）变得安全。事件日志仅存储密文，而您的步骤函数则与之前完全一样地工作。

您的工作流和步骤函数与之前完全一样地工作；所有流经事件日志的数据都会自动加密。

每个 Vercel 部署都会获得一个唯一的加密密钥。密钥派生和加密堆栈的工作原理如下：

- 每个工作流运行通过 HKDF-SHA256 派生自己的密钥
- 使用 AES-256-GCM 加密数据，以确保机密性和完整性
- 加密字段在仪表板中显示为锁定的占位符，直到解密

您可以通过两种方法访问加密数据：

Web 仪表板：在运行详情面板中点击“解密”按钮。解密完全通过 Web Crypto API 在浏览器中进行，因此可观测性服务器永远不会看到您的明文数据。CLI：在 `inspect` 命令中添加 `--decrypt` 标志。

```
1npx workflow inspect run <run-id> --decrypt --withData​
```

解密遵循与项目环境变量相同的权限模型，这意味着如果您缺乏查看环境变量的权限，则无法访问工作流数据。每次解密请求都会记录在您的 Vercel 审计日志中，为您的团队提供对访问事件的完全可见性。

虽然端到端加密已内置于 Vercel 平台中，但自定义的 `World` 实现可以选择使用此功能。您可以提供自己的 `getEncryptionKeyForRun()` 方法，核心运行时会自动使用该方法。更多信息请参阅 Workflow DevKit 文档。
