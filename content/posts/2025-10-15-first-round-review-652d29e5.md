在创业圈，我们经常听到这样的故事：创始人从小梦想创办一家公司——任何公司都行，其宏大的商业抱负源于童年卖柠檬水的小摊。但这个故事并非如此。

Amjad Masad 的创始人之路始于一个专注的愿景：让每个人都能拥有创造软件的力量。经营一家公司更像是实现这一目标的手段。“我一直对创业之路充满向往，但在创造技术时，我始终以我的世界观为指引，”Masad 说道。

他的出发点是这样一种前提：编写代码应该像在浏览器中打开一个新标签页一样简单。在2024年，当 LLM 已经能够为你编写代码的时代，这或许看起来并不那么具有突破性。但尽管像 Google Docs 这样的协作式、原生网络工具在21世纪末期就已出现，在网页浏览器中进行实时协作编程却出人意料地长时间未能普及。

在2008年，当 Masad 还是一名计算机科学专业的学生时，这当然是不可能的。如果开发者想要在一个项目上合作，他们通常需要访问同一台物理机器。这使得与不在你身边的人合作开发项目几乎不可能——更不用说与地球另一端的人一起工作了。

这个痛点催生了一个开源项目，后来成为了 Replit，Masad 常将其称为“一个成功的疯狂科学实验”。他和几位大学朋友构建了这个项目，当时名为 Repl.it，作为一种初步方式，让你在需要时随时获得你常用编程语言的 REPL（计算机编程术语，代表读取-求值-打印-循环）。

这个“科学实验”本应保持原样——安全地藏在一个隐喻性的实验室里，基本上可以当作其他开发者的沙盒来使用。事实上，大学毕业后，他将其搁置一旁，转而追求软件工程师和管理者的职业生涯，先后在 Codecademy 和 Facebook 任职。

然而，Repl.it 持续增长，并吸引了一大批狂热的开发者粉丝，他们都沉迷于能够在网页浏览器的编码环境中协作。很快，产品市场契合的早期信号变得如此明显，以至于 Masad 再也无法忽视它们。最初只是大学课堂的一种变通方法，最终变成了一个价值10亿美元的副业项目。

自作为初创公司推出以来的八年里，Replit 飞速发展。如今，它拥有3000万用户，并已筹集超过2亿美元的资金。这种开拓精神也一直在延续。公司在2022年发布其 AI 编码辅助工具 Ghostwriter 后，增长再次大幅飙升，用户数量翻了一番。就在上个月，它推出了 Replit Agent，一个更先进的 AI 驱动工具，可以被视为“结对程序员”，负责配置开发环境、安装依赖项和执行代码。

在这次独家专访中，Masad 回顾了这一切，回忆了他作为大学生时看到的客户对 Replit 喜爱的早期迹象，是什么最终说服他以创始人的身份重新拾起这个想法，以及他押注的那些逆向策略如何加速了 Replit 的增长。

Replit 的故事提醒我们，不仅每家公司，而且每位创始人的道路，都略有不同。那么，让我们把时间拨回到2008年，回到 Replit 旅程的起点。

## 创建早期原型

作为约旦苏玛雅公主大学的一名大学生，Masad 满怀想法地来到校园。但现实很快扼杀了这些宏伟的计划。

“作为计算机科学专业的学生，我们上编程课时首先面临的就是设置开发环境这个最初的苦差事。这不是一次性的事情，而是持续不断的，”Masad 说道。即使清除了最初的那个障碍，前方的道路依然充满坎坷。一旦环境设置好，几乎不可能与任何人进行异步协作。

“如果你和任何一位工程师交谈，他们都会知道这种挣扎，”他说。“我写了一些代码，在我的机器上能运行，但在我的项目伙伴或教授的机器上却不行。这使得一起编码变得极其困难。”

那是在2008年，像 Google Docs 这样的单页应用刚刚兴起，越来越多的科技公司开始将产品迁移到云端。Masad 认为这意味着一个基于网络的协作式代码编辑器即将问世。“但当我环顾四周时，却一无所获。没有一个平台能让你运行代码、共享程序并与他人协作。”

于是他决定自己动手。带着年轻开发者的天真和雄心，他开始构建他设想中的工具。“我对自己说，‘这能有多难？’”Masad 笑着说。“结果发现，原型很容易。我创建了一个文本框，你可以在里面输入 JavaScript，点击一个按钮，它就会执行代码。还有一种方法可以保存和分享它，我的朋友们都很喜欢。”

受到积极反响的鼓舞，Masad 全身心投入到这种新方法中，将各种语言编译成 JavaScript——最终扩展了他的项目，使其包含更多语言。（他甚至尝试用 JavaScript 编写一个 Python 解释器，这个挑战耗费了他12个月的时间，最终他承认失败。）但一次偶然的发现改变了一切。“有一天，我偶然发现了 Mozilla 一个研究小组的项目，”Masad 说。“他们正在探索将不同语言编译成 JavaScript 的想法，将 JavaScript 视为字节码解释器。这激发了我的想象力。”

令他高兴的是，这奏效了。他开发了一个原型，首次在浏览器中支持了 Python、Ruby 和其他几种语言，并将其开源。这个早期项目，他称之为 Repl.it，为后来成为公司 Replit 的集成开发环境平台奠定了基础。但这里需要明确一点——在 Masad 看来，在那个时候，Repl.it 仍然只是一个副业项目。

## 另辟蹊径

在业余时间开发 Repl.it 的同时，Masad 也在为科技领域的职业生涯（他希望有一天能因此来到美国）奠定基础。毕业后，他在雅虎位于约旦的办事处找到了一份软件工程师的工作。但他继续在晚上和周末开发 Repl.it，以跟上项目日益增长的需求，并与他的联合创始人（其中一位，Haya Odeh，最终成为了他的妻子）一起工作。

尽管没有一位联合创始人全职投入这个项目，但到2011年，它在开发者中，尤其是学生开发者中，越来越受欢迎——这是一个强烈的信号，表明无论 Masad 是否推动，它已经在走向产品市场契合的道路上。“那一年，它最终在 Hacker News 和 GitHub 上爆火，”他说。“虽然作为演示应用的 Replit 表现不错，但真正起飞的是底层引擎，叫做 JS REPL。

这种增长激增恰逢硅谷大规模开放在线课程运动的兴起。“在线学习的想法正在激发人们的想象力，”Masad 说。不久之后，几家美国科技公司，如教育科技公司 Udacity，开始联系 Masad 和他的联合创始人。但 Masad 遇到的其中两家公司尤其加速了他的职业生涯：

### Codecademy

Codecademy 是该领域的一家知名企业，当时刚从 Y Combinator 毕业，开始大量使用 Masad 为 Replit 编写的软件包。2011 年，在 JS REPL 成为开发者们在 HackerNews 上点赞追捧的热门工具后不久，Codecademy 的创始人联系了 Masad 寻求帮助，最初以每小时 15 美元的价格为他提供了一份外包工作。这很快演变成了一个机会，让 Masad 以创始工程师的身份加入该公司。

面对在资源有限的约旦创办自己的初创公司，还是加入 Codecademy 并移居美国的选择，Masad 选择了后者。“第二个选择对我来说更令人兴奋，”他解释道，“它让我得以实现长久以来的梦想——来到美国。”

### Facebook

在 Codecademy 工作两年后，Masad 于 2013 年加入了 Facebook。他被马克·扎克伯格关于 Internet.org 的愿景所吸引，这是一项旨在提供普遍互联网接入的倡议。记住时间点很关键。2013 年，构建移动产品的热潮正如火如荼，塑造这一未来的前景深深吸引了 Masad。

“世界正处于一场巨大变革之中，很快每个人都将通过手机和平板电脑访问计算机，”他说，“看到互联网接入民主化的进展令人兴奋。”

在 Facebook，Masad 最初从事 Android 应用开发，但很快又回到了基础设施项目。他成为了 React Native 团队的早期成员之一，该团队致力于将 Web 技术引入移动端。Masad 负责编写 React Native 的大部分 JavaScript 基础设施，并最终受命组建公司第一个 JavaScript 基础设施团队。

“我们想要现代化 Facebook 的 JavaScript 基础设施。所以我组建了一个团队，这个团队最终也负责构建 Jest、Babel 和 Yarn。我们基本上在 Facebook 构建了现代化的 JavaScript 开发工具链，”他说。

## 重返 Replit

在某个阶段，当 Masad 全身心投入在 Codecademy 和 Facebook 的工作时，Replit 逐渐淡出了他的视野。但在 2016 年，仍在 Facebook 任职的 Masad 开始思考下一步的打算。他迄今为止的职业决策都围绕着三个核心理念：

- 1. 普遍连接、计算机素养和计算机访问是未来趋势。
- 2. 世界各地将会涌现出更多、更多样化的开发者。
- 3. 软件开发可以做得更好。

“这些理念主导了我的职业发展轨迹，我被符合这一框架的项目所吸引，”Masad 说。在这些相同原则的指引下，他意识到是时候回到最初的起点了。

巧合的是，他并非唯一一个仍在思考这个项目的人。“尽管此时的 Replit 已被废弃，部分功能也已损坏，但开发者们仍然不断回来使用这个产品，因为市场上没有其他替代品，”Masad 说，“有一些雄心勃勃的项目，比如被 AWS 收购的 Cloud Nine，但没有一个是 Web 原生的。”

这是一个信号，表明 Masad 正处于一个商业产品初步实现产品市场契合的边缘。当他退后一步思考时，他意识到全力投入这个项目能够实现他的全部三个核心理念，而这正是他时隔多年后重返最初项目所需的推动力。那是一个激动人心的时刻；Masad 基本上是从一张白纸开始，手头只有他大学时期留下的现有演示页面。

但如果 Replit 有朝一日要成为一家价值数十亿美元的公司，Masad 就必须放手一搏。以下是 Masad 在重新拾起 Replit 后早期做出的两个冒险举动：

### 转向云端

将 Replit 从基于浏览器的执行转向基于云的项目，对 Masad 来说是一个急转弯。毕竟，这是最初让那个开源项目取得巨大成功的切入点。但是，当以全新的眼光审视这个项目，更重要的是，考虑到商业化时，Masad 开始越来越将浏览器视为一条“死胡同”。

“在非洲的 Chromebook 上下载几十兆字节的 JavaScript 根本行不通，”他说，“我的想法是，随着普遍连接和互联网带宽的改善，云端才是未来的方向。这在当时是有争议的，因为涌现出了一批做基于浏览器执行的竞争对手。”

通过从浏览器转向，Masad 开始着手为云端重写项目。“Docker 刚刚问世，使得基于云端的沙箱化变得更加容易。我开始添加越来越多的语言，项目也再次起飞了。”

### 全力投入 Web 原生

Masad 想要做出的另一个改变，是基于他看到竞争对手们回避的方向，即让 Replit 成为 Web 原生软件。当时所有其他流行的 IDE 都是桌面应用，但 Masad 的愿景是在浏览器中无缝部署代码，并配以出色的 Web 原生用户界面。这意味着要从头开始重建 Replit。

“构建 Web 原生软件的一个具体事实是，URL 非常重要，”他说，“Figma 就是一个例子。你可以将你的 URL 分享给某人，他们就能进入与你相同的环境，产品默认就变成了多人协作模式。当时没有哪个在线 IDE 能做到这一点，所以这就是我们试图复制的。”

但追求这条道路并非易事。“构建 Web 原生软件意味着我们面临着更大的挑战。变现会更难，专业开发者在很长一段时间内无法在工作中使用它，而且我们可能很难获得融资，”他说。

### 寻找融资并实现核心产品变现

Masad 对融资的担忧是有道理的。Replit 在 2016 年首次申请 Y Combinator 时被拒绝了。尽管用户增长势头良好，但他们仍难以筹集到种子轮前的资金。

最终，Vanta 的创始人 Christina Cacioppo 为他们开出了第一张支票，随后 Masad 在 Codecademy 时期结识的 Roy Bahat 也为他们的项目投资了 60 万美元。

在这笔资金的注入以及更稳固的产品方向带来的新决心推动下，Masad 和他的联合创始团队开始为教育市场构建商业产品，尽管有些勉强。

“当时大约有 10 万人在使用 Replit，”Masad 回忆道，“他们中既有通过老师找到它的学生，也有自己发现它的自学成才的青少年。许多 Web 原生的青少年甚至不知道文件和文件夹的概念，所以他们会在谷歌上搜索基于 Web 的协作编码环境，然后找到我们。我们通过人们分享 REPL 链接建立了庞大的网络足迹，这种病毒式增长为我们带来了显著的吸引力，”他说。

鉴于已经拥有的坚实基础，将学生和教育工作者作为初始客户画像自然是合理的，但 Masad 有所保留。“我一直把教育市场看作我们的垫脚石。我不喜欢从中变现，因为我希望 Replit 是人人可及的。”

通过专注于这个更狭窄的理想客户画像，Replit 首次开始有了收入，而且根据 Masad 的说法，金额“相当可观”。但即使有现金流入，团队扩张仍然困难。“那对我们来说是一段艰难的时期，”Masad 说，“我们看到了巨大的增长，这很棒，但为了支撑这种增长，我不得不随时待命。”

Masad 又向 YC 提交了两次申请，一次在 2016 年，另一次在次年。

他不断被拒绝——事实上，他们甚至从未进入面试环节。“我们就是无法打入硅谷圈子的核心，”他说。一次又一次，Masad 和 Odeh 被告知 Replit 不是一个可持续的业务。

所以你可以想象 Masad 在 2017 年冬天时的惊讶，当时担任 YC 总裁的 Sam Altman 在 Twitter 上联系了他。“他告诉我，Paul Graham 在 Hacker News 上发现了我们，而且事实上，在 YC 起飞之前，他就想构建一个类似 Replit 的东西，”Masad 说。在接下来的几个月里，Graham 和 Masad 通过电子邮件保持联系，交流想法和沟通。Graham 鼓励 Masad 再次申请，他勉强答应了。

然而，时机再紧张不过了；申请截止日期就在第二天早上。由于之前三次填写表格已经精疲力竭，Masad 没有时间（也没有耐心）上传视频演示。“所以我上传了 Rick Astley 的《Never Going to Give You Up》音乐视频，”他说。这差点让他们失去申请资格。但总体而言，评审 Replit 申请的人对其创意和联合创始团队的激情印象深刻，Replit 被 YC W18 批次录取。

## 培育增长

Replit 的种子轮资金只能支撑他们走这么远。到 2018 年，Masad 和他的联合创始人开始寻求再次融资。刚从 Y Combinator 毕业，Masad 就在他的融资演示文稿中阐述了他对 Replit 的长期愿景（大致受 Elon Musk 的“总体规划”启发）。它被分解为三个步骤，甚至在大语言模型成为主流之前就预示了 AI 对编码的影响前景：

- 步骤 1：创建一个协作式云端开发环境
- 步骤 2：训练机器学习模型，使编码更简单、更易用（基于此网络及其将收集的数据）。
- 步骤 3：成为一个平台，使“在互联网上制作软件这一令人难以置信的财富创造引擎”的访问民主化。

步骤一通过 Replit 的原始原型以及产品在学生和教育工作者中找到的初步产品市场契合得以解决。为了实现计划中的下一步，Masad 知道需要换挡并专注于更大的市场。这意味着要接触专业开发者。

“直到今天，我们业务面临的最大问题是：‘Replit 能否成为开发者的日常工具？’我在这里将开发者定义为那些赚钱并从事经济生产活动的专业人士，”Masad 说。在这种情况下，开发者是指那些为他们在 Replit 上制作的软件向他人收费的人，或者在 Replit 上构建初创公司的用户。“这些信号对我来说很突出，如果我们想构建一个不仅仅是玩具的东西，这些就是我们应该深入探索的领域。”

这个用户画像成为了公司的北极星。

正是在这个时候，他开始更批判性地思考如何将 Replit 发展成一项颠覆性技术，他引用哈佛商学院教授 Clayton Christensen 的《创新者的窘境》作为这些想法的关键素材。“颠覆性技术始于市场的底层。当它通过某种趋势变得更强大或借助顺风时，它正乘着产品或供应链的改进之势，最终将占据主导地位，”Masad 说。

“经典的例子是 x86 微计算机 PC 架构如何杀死大型机并成为计算的标准方式。这就是我采用的视角，”他说。“我试图弄清楚我们在这条能力曲线上处于什么位置，以及我们是否达到了人们觉得使用 Replit 充满力量而非受限的地步。”

鉴于这种新思维，Masad 在 Replit 旅程的这个阶段做出了三个关键决定。回顾过去，这些决定最终使 Replit 从一个原地打转的种子轮初创公司变成了一家呈现曲棍球棒式增长的公司：

### 1. 广泛布局，下大赌注

当时，公司们都在押注哪种语言会是未来——大多数都全力投入 JavaScript。“我记得在 2018 年，一个巨头出现了，”Masad 说。“编程界的传奇人物 Joel Spolsky 推出了一个名为 Glitch 的项目，”Masad 说。“Glitch 的种子轮融资是 3000 万美元。他们立下标杆，宣称 JavaScript 是未来的语言。”

而 Replit 则反其道而行之。“一旦我看到了这一点，我决定押注成为一个通用平台，”他说。“我认为我们不应该从事押注特定语言的业务，相反，我们应该尽可能多地支持各种语言。”

随着 Python 开始快速增长，在 CS101 课程中超越 Java 和 Lisp，并在 Web 开发和数据科学领域越来越受欢迎，这一直觉被证明是 Replit 增长的秘密杠杆。到 2020 年疫情爆发时，Replit 的用户数量和参与度是其最接近竞争对手的 10 到 100 倍。

Replit 平台上的开发者已超过 2000 万。尤其自豪的是，我们正在拓展“开发者”的定义：不仅仅是工程师在使用 Replit，还有营销人员、独立创业者、科学家、学生——以及更多！https://t.co/3fvl0SIbuCpic.twitter.com/8EDTHJAspt

事后看来，一切似乎都更美好。但创始人深知，偏离“坚持狭窄范围”的传统产品建议而做出转型决定并不容易。

“有 Peter Thiel 的《从零到一》的思考方式，我认为这很出色。但它也是对 Steve Jobs 如何进行产品管理和设计的误解，”Masad 说。“人们认为苹果软件非常简单。但他们忘记了苹果为了产生那些看似非常简单的像素而构建了整个技术栈。简单并不意味着你不应该拥有一个强大的平台。”

他用 Replit 的一个客户字节跳动来进一步阐述这一点。

“可能会让人惊讶的是，字节跳动是 Replit 的客户。他们将其用于他们构建的一个名为 LarkBase 和 LarkSuite 的平台，”他说。“你可以把它描述成 G Suite 或 Workday，或者一百种其他的美国软件。但有趣的是，现在你看到 TikTok 增加了购物、TikTok Live 以及所有其他功能。中国软件往往增长更快，也更有雄心。这是对硅谷狭窄构建模式的一个挑战。也许这里有一个历史教训。在平台转型的早期阶段，你可以做得很窄，并且可以找到很多产品市场契合，但随着你的产品开始成熟，比如 Web 或云或其他什么，你实际上必须变得更有雄心。”

### 2. 早期采用 AI

Masad 押注通用平台的赌注开始产生巨大回报。到 2019 年底，他们的用户群激增至 600 万用户，增长了 122%，几乎完全依靠口碑增长。Masad 还能够在 2020 年初以比以往融资尝试轻松得多的方式完成 A 轮融资。直到此时，Replit 仍然是一个由三位联合创始人（Masad、Odeh 和 Masad 的兄弟 Faris）组成的紧密团队。他们将资金投入到扩大团队规模上。

但 Masad 在此期间也投资 AI 的关键（且具有预见性的）决定真正巩固了 Replit 的领先地位。鉴于他对 Replit 的长期愿景，当 GPT-1 于 2018 年首次推出时，Masad 就一直密切关注着它。

“直到GPT-2出现，我们才开始真正编写代码并进行实验，”Masad说。“但理解机器学习技术栈非常困难，训练过程也仍然难以触及，所以尽管我们有一些令人兴奋的实验成果，却从未发布过任何产品。”

这一切在2021年随着GPT-3的发布而彻底改变。Replit与OpenAI合作推出了Explain Code功能，为开发者提供逐步的自然语言代码解释。“这是GPT最早投入生产环境的应用程序之一，”Masad表示。

这催生了Ghostwriter的诞生，即Replit的Copilot版本（现在简称为Replit AI）。但他们遇到了一个巨大障碍：延迟。“当时GPT-3速度慢且成本高昂。但我设想中的编程环境应该是AI原生的，而非附加功能。”带着这一愿景，Replit团队决定从零开始构建。

### 3. 与企业合作

随着Replit在核心REPL产品开发上持续取得进展，并利用AI工具开展内部研发项目，其最大的增长杠杆或许来自与大型企业客户的合作。

他们的首个突破出现在2022年6月，当时Replit发布了基于Salesforce CodeGen模型的首款AI代码补全产品。“他们训练的模型效果不错，但速度很慢，”他说。“我们在内部做了大量工作，重写了代码库的许多部分以提升速度。发布时引起了很大反响，因为这是我们首个基于开源技术的产品发布。”

自那以后，Replit持续为用户推出新的生成式AI功能。同年10月，Replit将AI代码补全工具Ghostwriter集成到核心平台，向所有用户开放。同时，通过与Google Cloud集成，Replit达成了首个大型企业合作。这两步举措帮助Masad朝着他长期目标——让互联网软件开发民主化——又迈进了一步（这是他六年前向投资者阐述的长期愿景中的第三步）。在集成Ghostwriter的同时，Replit也是首批开源代码专用LLM的公司之一，该模型名为“replit-code-v1.5-3b”。

随后增长迅猛，Replit的用户基数从2021年12月的约1000万翻倍，到2023年2月达到2000万。

## 展望未来

Replit的AI能力持续进化。2023年10月，Replit将Ghostwriter重新作为名为Replit AI的生成式AI工具套件的一部分推出。最近，它发布了初代代码编写工具的更高级版本——Replit Agent，这是一个仅需几句话就能创建和部署应用程序的AI系统，无需编写一行代码。用户只需一个文本提示词，就能从零开始创建和设计可运行的应用程序。
