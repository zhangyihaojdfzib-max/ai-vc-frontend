使用 Node.js 和 Python 运行时的 Vercel Functions 现在支持 Pro 和 Enterprise 团队最长 30 分钟的执行时长，是此前 800 秒限制的两倍以上。对其他运行时的支持即将推出。

将更长时间运行的 Functions 用于需要更多时间完成的工作，包括：

- 长时间的 LLM 推理和工具调用
- 持续流式输出数分钟的 AI 响应
- 文档和媒体处理
- OCR 和提取
- 网页抓取和浏览器自动化
- 复杂工作流步骤或队列处理器

Fluid Compute 使长时间运行的工作保持成本效益。仅在代码执行时按活跃 CPU 计费，当 Function 等待 I/O（如 AI 模型调用、数据库查询和第三方 API）时暂停计费。

设置 `maxDuration` 以选择启用。对于 Next.js App Router，在路由文件中进行配置：

```
export const maxDuration = 1800; 
export async function POST(request: Request) {  return Response.json({ ok: true });}
```

对于其他运行时和框架，在 `vercel.json` 中为特定函数路径配置 `maxDuration`：

```
{  "$schema": "https://openapi.vercel.sh/vercel.json",  "functions": {    "api/long-task.py": {      "maxDuration": 1800    }  }}
```

超过 800 秒的时长处于测试阶段，需要 Fluid Compute。在文档中了解有关为 Vercel Functions 配置最大时长的更多信息。
