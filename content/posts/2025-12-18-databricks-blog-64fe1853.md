数据协作是现代AI创新的基石，尤其是在组织与外部合作伙伴协作以解锁新洞察时。然而，在促进协作的同时保护敏感数据，数据隐私和知识产权保护仍然是主要挑战。

为了弥合这一差距，各行各业的客户正在使用 Databricks Clean Rooms（数据洁净室）对敏感数据进行共享分析，并实现隐私优先的协作。

我们整理了以下关于洁净室的10个最常见问题。这些问题涵盖了洁净室是什么、它们如何保护数据和知识产权、如何在多云和跨平台环境中工作，以及如何开始使用。让我们开始吧。

**1. 什么是数据洁净室？**
数据洁净室是一个安全的环境，您和您的合作伙伴可以在其中共同处理敏感数据以提取有用的洞察，而无需共享底层敏感的原始数据。

在 Databricks 中，您可以创建一个洁净室，添加想要使用的资产，并在一个隔离、安全且受管控的环境中仅运行已批准的笔记本。

**2. 洁净室何时有用？**
当多方需要分析敏感数据但又不共享其原始数据时，洁净室非常有用。这通常是由于隐私法规、合同或知识产权保护的要求。

它们被广泛应用于许多行业，包括广告、医疗保健、金融、政府、交通和数据货币化。

一些例子包括：
*   **广告和营销**：在不暴露PII（个人身份信息）的情况下进行身份解析、活动策划和效果衡量、零售媒体的数据货币化以及品牌协作。
*   **金融服务**：银行、保险公司和信用卡公司合并数据以改善运营、进行欺诈检测和分析。

洁净室在允许协作和数据丰富的同时，保护客户数据。

**4. 洁净室与 Delta Sharing 有何不同？**
可以这样理解：当一方需要在其自身环境中对数据拥有只读访问权限，并且查看底层记录是可接受的情况下，Delta Sharing 是合适的选择。

当数据必须保持私密时，洁净室为多方分析增加了一个安全、受控的空间。合作伙伴可以联接数据资产，运行共同批准的代码，并且只返回各方都同意的输出结果。这在您必须满足严格的隐私保证或支持受监管的工作流程时非常有用。实际上，洁净室中共享的数据在幕后仍然使用 Delta Sharing 协议。

例如，零售商可能使用 Delta Sharing 授予供应商对销售表的只读访问权限，以便供应商了解产品销售情况。而当这对合作伙伴需要联接双方更丰富、更敏感的数据（如客户特征或详细库存），运行已批准的笔记本，并且只共享聚合输出（如需求预测或高风险商品清单）时，他们就会使用洁净室。

**5. 洁净室如何保护我的数据和知识产权？**
洁净室的设计确保您的合作伙伴永远看不到您的原始数据或知识产权。您的数据保留在您自己的 Unity Catalog 中，您仅通过 Delta Sharing 在洁净室中共享特定资产，而这些共享由已批准的笔记本控制。

为了在洁净室中强制执行这些保护措施：
*   **数据不移动**：您的原始数据保留在您自己的云租户和区域中。
*   **仅共享元数据**：只有表/视图的元数据（模式、位置）通过 Delta Sharing 共享。
*   **代码控制**：所有分析都通过已批准的笔记本运行，这些笔记本定义了允许的操作。
*   **仅输出可见**：合作伙伴只能看到笔记本运行产生的输出（例如聚合结果、模型预测），而不是彼此的原始数据行。

**6. 洁净室是否支持多云协作？**
是的。洁净室专为多云和跨区域协作而设计，只要每个参与者都有一个启用了 Unity Catalog 的工作空间，并且在其元存储上启用了 Delta Sharing。这意味着，在 Azure 上使用 Databricks 的组织可以与在 AWS 或 GCP 上的合作伙伴在洁净室中协作。

**7. 我能否与使用 Snowflake 或 BigQuery 的合作伙伴协作？**
是的，完全可以。Lakehouse Federation（湖仓联邦）将 Snowflake、BigQuery 和传统数据仓库等外部系统作为外部目录暴露在 Unity Catalog 中。一旦外部表在 UC 中可用，您就可以像共享任何其他表或视图一样，在洁净室中共享它们。

其高级工作原理如下：您使用 Lakehouse Federation 创建连接和外部目录，将外部数据源暴露在 Unity Catalog 中，而无需将所有数据复制到 Databricks。一旦这些外部表在 Unity Catalog 中可用，您就可以像任何其他 Unity Catalog 管理的表或视图一样，将它们共享到洁净室中。

**8. 洁净室内部如何运作？**
在洁净室内，您几乎通过笔记本来完成所有事情。您添加一个包含所需分析代码的 SQL 或 Python 笔记本，您的合作伙伴审查并批准该笔记本，然后它就可以运行了。

*   **简单案例**：您可能有一个 SQL 笔记本，用于计算零售商购买记录与媒体合作伙伴曝光记录之间重叠的哈希 ID 数量，然后输出触达率、频率和转化率。
*   **更高级的案例**：您使用 Python 笔记本联接双方的特征，在合并数据上训练或评分模型，并将预测结果写入输出表。已批准的运行者可以看到输出，但没有人能看到另一方的原始记录。

**9. 洁净室中可以有多少参与者？**
在一个 Databricks 洁净室中，最多可以有10个组织（您加上9个合作伙伴）在一个安全环境中协同工作，即使你们位于不同的云或数据平台上。每个团队将其数据保留在自己的 Unity Catalog 中，并且只共享他们想在洁净室中使用的特定表、视图或文件。

一旦所有人加入，每一方都可以提议 SQL 或 Python 笔记本，这些笔记本在运行前需要获得批准，因此各方都对逻辑感到放心。

**10. 如何开始使用洁净室？**
以下是一个简单的入门方法：
1.  **创建洁净室**：在 Databricks 工作空间中，导航到“数据”>“洁净室”，然后单击“创建洁净室”。
2.  **邀请合作伙伴**：通过电子邮件邀请您的合作伙伴加入洁净室。
3.  **共享资产**：将您想要使用的表、视图或笔记本添加到洁净室。
4.  **协作分析**：创建或上传笔记本，提交供合作伙伴批准，然后运行分析以获取共享洞察。
