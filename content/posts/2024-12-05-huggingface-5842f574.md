# 欢迎 PaliGemma 2 – 谷歌全新视觉语言模型

我们很高兴欢迎谷歌全新视觉语言模型 PaliGemma 2，这是 PaliGemma 的新一代版本。与其前身一样，PaliGemma 2 在视觉部分使用相同的强大 SigLIP，但在文本解码器部分升级为最新的 Gemma 2。

PaliGemma 2 推出了新的预训练（pt）模型，参数规模分别为 3B、10B 和 28B。所有这些模型都支持多种输入分辨率：224x224、448x448 和 896x896。这些组合为不同用例提供了极大的灵活性，从业者可以在质量/效率方面选择所需的平衡。相比之下，之前的 PaliGemma 仅提供 3B 版本。

预训练模型设计用于轻松微调到下游任务。第一代 PaliGemma 被社区广泛用于多种用途。随着新增变体带来的更高灵活性，结合更好的预训练质量，我们迫不及待想看到这次社区能做出什么成果。

例如，谷歌还在 DOCCI 数据集上发布了一些微调变体，展示了强大且稳健的、长篇、细致入微且详细的描述能力。微调后的 DOCCI 模型提供 3B 和 10B 版本，支持 448x448 的输入分辨率。

此次发布包括所有开源模型仓库、Transformers 集成、微调脚本，以及我们自己在 VQAv2 数据集上为视觉问答微调的模型演示。

PaliGemma 2 是谷歌于 5 月发布的 PaliGemma 视觉语言模型的新一代版本。PaliGemma 2 将强大的 SigLIP 图像编码器与 Gemma 2 语言模型连接起来。

![PaliGemma2 架构](/images/posts/5cef8c86f5ac.png)

新模型基于 Gemma 2 2B、9B 和 27B 语言模型，分别对应 3B、10B 和 28B 的 PaliGemma 2 变体，其命名考虑了（紧凑型）图像编码器的额外参数。如上所述，它们支持三种不同分辨率，为微调到下游任务提供了极大的灵活性。

## 如何使用 Transformers

您可以使用 🤗 transformers 中的 PaliGemmaForConditionalGeneration 和 AutoProcessor API 对 PaliGemma 2 模型进行推理。请确保安装 transformers 4.47 或更高版本：

```bash
pip install --upgrade transformers

```

之后，您可以按如下方式运行推理。像往常一样，请确保遵循您所用任务训练模型时使用的提示词格式：

```python
from transformers import AutoProcessor, PaliGemmaForConditionalGeneration
from PIL import Image
import requests

model_id = "google/paligemma2-10b-ft-docci-448"
model = PaliGemmaForConditionalGeneration.from_pretrained(model_id)
model = model.to("cuda")
processor = AutoProcessor.from_pretrained(model_id)

prompt = "<image>caption en"
image_file = "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/cats.png"
raw_image = Image.open(requests.get(image_file, stream=True).raw).convert("RGB")

inputs = processor(prompt, raw_image, return_tensors="pt").to("cuda")
output = model.generate(**inputs, max_new_tokens=200)

input_len = inputs["input_ids"].shape[-1]
print(processor.decode(output[0][input_len:], skip_special_tokens=True))


```

您也可以使用 `transformers` 的 `bitsandbytes` 集成功能来加载量化后的模型。以下示例使用了 4 位 `nf4` 量化：

```python
from transformers import BitsAndBytesConfig

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16
)
model = PaligemmaForConditionalGeneration.from_pretrained(
    model_id,
    quantization_config=bnb_config,
    device_map={"":0}
)

```

我们通过在 `textvqa` 数据集上评估一个经过微调的 3B 检查点（使用 224x224 输入图像），快速测试了量化带来的性能下降。以下是在验证集 5,000 条数据上得到的结果：

- bfloat16，无量化：准确率 60.04%。
- 8 位量化：准确率 59.78%。
- 4 位量化（使用上述代码片段中的配置）：准确率 58.72%。

这些数字非常令人鼓舞！当然，量化对于较大的检查点最有价值，我们建议您始终在您将使用的领域和任务上测量结果。

如果您之前微调过 PaliGemma，那么微调 PaliGemma 2 的 API 是相同的，您可以直接使用现有代码。我们提供了微调脚本和笔记本，供您微调模型、冻结部分模型，或应用 LoRA 或 QLoRA 等内存高效微调技术。
