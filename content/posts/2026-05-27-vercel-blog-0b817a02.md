Vercel CLI 现已附带一个可选的实验性原生二进制版本，启动速度更快、安全性更高，且无需依赖 Node.js 运行时。

这些二进制文件经过代码签名，您的操作系统可验证其来自 Vercel 且未被篡改。此外，在 macOS 上，凭据会存储在与该二进制文件绑定的系统钥匙串中，因此其他进程未经明确许可无法访问这些凭据。

您可以通过安装实验性包来选择使用：

```
pnpm i -g @vercel/vc-native -f
```

必须使用 `-f` 标志，因为 `@vercel/vc-native` 会安装与标准 CLI 相同的全局 `vercel` 和 `vc` 二进制名称。安装后，`vercel` 和 `vc` 将在 macOS、Linux 和 Windows 的 x64 及 arm64 架构上运行原生二进制文件。

欢迎在 GitHub 上告诉我们您的想法。
