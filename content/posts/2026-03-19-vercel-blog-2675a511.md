大多数知识型 Agent（智能体）的构建方式都如出一辙。你选择一个向量数据库，然后构建一个分块管道。你选择一个嵌入模型，然后调整检索参数。

几周后，你的 Agent（智能体）错误地回答了一个问题，而你却不知道它检索了哪个数据块，也不知道为什么那个数据块得分最高。

我们内部以及为在 Vercel 上构建 Agent（智能体）的团队中反复看到这种模式。嵌入技术栈适用于语义相似性搜索，但当需要从结构化数据中获取特定值时，它就力不从心了。这种失败模式是静默的：Agent（智能体）自信地返回了错误的数据块，而你无法追溯从问题到答案的路径。

这就是为什么我们尝试了不同的方法。我们用文件系统替换了向量管道，并给 Agent（智能体）提供了 `bash`。我们的销售电话摘要 Agent（智能体）每次调用的成本从约 1.00 美元降到了约 0.25 美元，而且输出质量也得到了提升。Agent（智能体）只是在做它已经知道如何做的事情：读取文件、运行 `grep` 和浏览目录。

## 链接到标题使用 Vercel Sandbox 进行基于文件的搜索

无需向量数据库。无需分块管道。无需嵌入模型。

你的 Agent（智能体）在隔离的 **Vercel Sandbox** 中使用 `grep`、`find` 和 `cat`。

其工作原理如下：

1.  你通过管理界面添加数据源，它们被存储在 Postgres 中
2.  内容通过 **Vercel Workflow** 同步到快照仓库
3.  当 Agent（智能体）需要搜索时，一个 Vercel Sandbox 会加载该**快照**
4.  Agent（智能体）的 `bash` 和 `bash_batch` 工具执行文件系统命令
5.  Agent（智能体）返回答案，并可附带引用

结果具有确定性、可解释性且速度快。当 Agent（智能体）给出错误答案时，你打开追踪记录就能看到：它执行了 `grep -r "pricing" docs/`，读取了 `docs/plans/enterprise.md`，并提取了错误的部分。你修复文件或调整 Agent（智能体）的搜索策略即可。整个调试周期只需几分钟。

相比之下，向量方法则不然。如果 Agent（智能体）返回了一个糟糕的数据块，你必须确定它检索了哪个数据块，然后弄清楚为什么它的得分是 0.82 而正确数据块的得分是 0.79。问题可能出在分块边界、嵌入模型或相似度阈值上。使用文件系统搜索，无需猜测它为什么选择了那个数据块，也无需在黑暗中调整检索分数。你是在调试一个问题，而不是一个管道。

| 嵌入方法 | 文件系统 |
| :--- | :--- |
| 黑盒评分 | 透明命令 |
| 难以调试 | 检查实际文件 |
| 需要调优 | 开箱即用 |

LLM（大语言模型）已经理解文件系统。它们在大量代码上进行了训练：浏览目录、在文件中进行 Grep 搜索、管理跨复杂代码库的状态。如果 Agent（智能体）在代码的文件系统操作方面表现出色，那么它们在处理任何事物时都会表现出色。这就是 `filesystem` 和 `bash` 方法背后的洞见。

你不是在教模型一项新技能；你是在使用它最擅长的技能。无需维护嵌入管道或扩展向量数据库。添加数据源，同步，然后搜索。

## 链接到标题Chat SDK：一个 Agent（智能体），覆盖所有平台

你的 Agent（智能体）拥有一个知识库、一个代码库和一个唯一的事实来源。然而，你的工程师分散在 Slack 上，你的社区遍布在 Discord 中，你的错误报告深埋在 GitHub 里。一个能理解这三者的单一 Agent（智能体）应运而生。

Chat SDK 将你的知识 Agent（智能体）连接到你的用户所在的每个平台。导入你需要的适配器，将每个适配器指向同一个 Agent（智能体）管道，你的 Agent（智能体）就能在任何 Chat SDK 支持的平台上运行。

Chat SDK 知识 Agent（智能体）示例

每个适配器处理平台特定的问题（例如，身份验证、事件格式、消息传递），而 Agent（智能体）本身保持不变。无论平台如何，只要机器人被提及，就会触发 `onNewMention`。Agent（智能体）接收消息文本，通过相同的基于文件系统的管道流式传输响应，并回复到该线程。

```

6const bot = new Chat({7  userName: "knowledge-agent",8  adapters: {9    slack: createSlackAdapter(),10    discord: createDiscordAdapter(),11  },12  state: createRedisState(),13});14
15bot.onNewMention(async (thread, message) => {16  await thread.subscribe();17  const result = await agent.stream({ prompt: message.text });18  await thread.post(result);19});
```

该模板开箱即用，提供了 GitHub 和 Discord 适配器，并且 Chat SDK 已经支持 Slack、Microsoft Teams、Google Chat 等。请参阅 `adapter` 目录，获取官方和社区适配器的完整列表。
