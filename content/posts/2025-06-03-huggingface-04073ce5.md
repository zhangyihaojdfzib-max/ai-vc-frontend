# SmolVLA：基于Lerobot社区数据训练的高效视觉-语言-动作模型

## 🧭 内容提要

今天我们推出 **SmolVLA**，一个紧凑（4.5亿参数）、开源的机器人视觉-语言-动作模型，可在消费级硬件上运行。

- 仅在兼容许可、开源社区共享的 `lerobot` 标签数据集上进行预训练。
- SmolVLA-450M 在仿真（LIBERO, Meta-World）和真实世界任务（SO100, SO101）上，性能优于更大的 VLA 模型以及如 ACT 等强基线模型。
- 支持**异步推理**，响应速度**提升30%**，任务吞吐量**提高2倍**。

## 引言

过去几年，Transformer 推动了人工智能领域的显著进步，从具备类人推理能力的语言模型到能同时理解图像和文本的多模态系统。然而，在现实世界的机器人技术中，进展要缓慢得多。机器人仍然难以在不同的物体、环境和任务中进行泛化。这种有限的进展源于**高质量、多样化数据的缺乏**，以及**缺乏能在物理世界中像人类一样推理和行动的模型**。

为了应对这些挑战，该领域最近转向了**视觉-语言-动作模型**，其目标是在单一架构内统一感知、语言理解和动作预测。VLA 通常以原始视觉观察和自然语言指令作为输入，并输出相应的机器人动作。尽管前景广阔，但 VLA 最近的许多进展仍被锁定在基于大规模私有数据集训练的专有模型背后，通常需要昂贵的硬件设置和大量的工程资源。
因此，更广泛的机器人研究社区在复现和构建这些模型方面面临着重大障碍。

SmolVLA 通过提供一个开源、紧凑且高效的 VLA 模型来弥补这一差距，该模型可以仅使用公开可用的数据集在**消费级硬件**上进行训练。通过不仅发布模型权重，还使用非常经济实惠的开源硬件，SmolVLA 旨在普及对视觉-语言-动作模型的访问，并加速面向通用机器人 Agent 的研究。

## 认识 SmolVLA！

**SmolVLA-450M** 是我们开源、紧凑但功能强大的 VLA 模型。它的特点是：

- 足够小，可以在 CPU 上运行，在单个消费级 GPU 上训练，甚至可以在 MacBook 上运行！
- 在公开、社区共享的机器人数据上训练
- 随附完整的训练和推理方案发布
- 可以在非常经济实惠的硬件（SO-100, SO-101, LeKiwi 等）上进行测试和部署

受大语言模型训练范式的启发，SmolVLA 经历了在通用操作数据上的预训练阶段，然后是针对特定任务的后训练。在架构上，它结合了 Transformer 和**流匹配解码器**，并通过以下设计选择针对速度和低延迟推理进行了优化：

- 跳过视觉模型一半的层，以实现更快的推理和更小的体积
- 交错自注意力块和交叉注意力块
- 使用更少的视觉 Token
- 利用更小的预训练 VLM

尽管使用了不到 3 万个训练片段——比其他 VLA 模型少一个数量级——SmolVLA 在仿真和现实世界中都**匹配甚至超越了**更大模型的性能。

为了让实时机器人技术更易于使用，我们引入了**异步推理栈**。这项技术将机器人执行动作的方式与它们理解所见所闻的方式分离开来。由于这种分离，机器人可以在快速变化的环境中更迅速地做出反应。

图 2. SmolVLA 接收来自多个摄像头的 RGB 图像序列、机器人当前的感知运动状态以及自然语言指令作为输入。VLM 将这些编码成上下文特征，这些特征作为条件输入给动作专家，以生成连续的动作序列。

## 🚀 如何使用 SmolVLA？

SmolVLA 设计得易于使用和集成——无论您是在自己的数据上进行微调，还是将其插入现有的机器人技术栈。

### 安装

首先，安装所需的依赖项：

```python
git clone https://github.com/huggingface/lerobot.git
cd lerobot
pip install -e ".[smolvla]"

```

### 微调预训练模型

使用我们的预训练 4.5 亿参数模型 `smolvla_base`，配合 lerobot 训练框架：

```python
python lerobot/scripts/train.py \
  --policy.path=lerobot/smolvla_base \
  --dataset.repo_id=lerobot/svla_so100_stacking \
  --batch_size=64 \
  --steps=20000  

```

### 从零开始训练

如果您希望从架构（预训练的 VLM + 动作专家）开始构建，而不是使用预训练的检查点：

```python
python lerobot/scripts/train.py \
  --policy.type=smolvla \
  --dataset.repo_id=lerobot/svla_so100_stacking \
  --batch_size=64 \
  --steps=200000

```

您也可以直接加载 `SmolVLAPolicy`：

```python
from lerobot.common.policies.smolvla.modeling_smolvla import SmolVLAPolicy
policy = SmolVLAPolicy.from_pretrained("lerobot/smolvla_base")

```

## 方法

SmolVLA 不仅是一个轻量级但功能强大的模型，也是一种训练和评估通用机器人策略的方法。在本节中，我们将介绍 SmolVLA 背后的**模型架构**以及用于评估的**异步推理**设置，后者已被证明更具适应性且能实现更快的恢复。

SmolVLA 由两个核心组件组成：一个处理多模态输入的**视觉-语言模型**，以及一个输出机器人控制命令的**动作专家**。下面，我们将分享 SmolVLA 架构主要组件和异步推理的细节。更多细节可以在我们的**技术报告**中找到。

### 主要架构

#### 视觉-语言模型

我们使用 **SmolVLM2** 作为我们的 VLM 骨干网络。它针对多图像输入进行了优化，由 SigLIP 视觉编码器和 **SmolLM2** 语言解码器组成。

- **图像 Token** 通过视觉编码器提取。
- **语言指令** 被 Token 化并直接馈送到解码器。
- **感知运动状态** 通过一个线性层投影成单个 Token，以与语言模型的 Token 维度对齐。

解码器层处理拼接后的图像、语言和状态 Token。生成的特征随后传递给动作专家。

#### 动作专家：流匹配 Transformer

SmolVLA的动作专家是一个紧凑的Transformer（约1亿参数），它根据VLM的输出生成动作块，即未来的机器人动作序列。它使用流匹配目标进行训练，该目标教会模型将带噪声的样本引导回真实数据。相比之下，虽然离散动作表示（例如，通过Token化）功能强大，但它们通常需要自回归解码，这在推理时速度慢且效率低下。而流匹配允许对连续动作进行直接、非自回归的预测，从而实现高精度的实时控制。

更直观地说，在训练过程中，我们向机器人的真实动作序列添加随机噪声，并要求模型预测将其带回正确轨迹的“校正向量”。这在动作空间上形成了一个平滑的向量场，帮助模型学习准确且稳定的控制策略。

我们使用具有交错注意力块的Transformer架构来实现这一点（见图2），并将其隐藏层大小减少到VLM的75%，以保持模型轻量级便于部署。

### 为效率和鲁棒性所做的设计选择

虽然将视觉语言模型与动作预测模块结合是近期VLA系统（如Pi0、GR00T、Diffusion Policy）的常见设计模式，但我们发现了几种能显著增强鲁棒性和性能的架构选择。在SmolVLA中，我们应用了三种关键技术：减少视觉Token数量、跳过VLM的上层，以及在动作专家中交错使用交叉注意力和自注意力层。

#### 视觉Token缩减

高分辨率图像能改善感知，但会显著减慢推理速度。为了取得平衡，SmolVLA在训练和推理期间将每帧的视觉Token数量限制为64个。例如，一张512×512的图像使用PixelShuffle作为一种高效的混洗技术，被压缩为仅64个Token，而不是1024个。虽然底层的视觉语言模型最初是使用图像平铺进行预训练以获得更广泛的覆盖范围，但SmolVLA在运行时仅使用全局图像，以保持推理轻量且快速。

#### 通过层跳跃实现更快推理

我们使用VLM中间层的特征，而不是总是依赖VLM的最后一层——后者可能计算成本高且有时并非最优。先前的研究表明，早期层通常能为下游任务提供更好的表示。
在SmolVLA中，动作专家在训练期间仅关注VLM特征直到一个可配置的层N，该层数设置为总层数的一半。这使VLM和动作专家的计算成本减半，显著加快了推理速度，同时性能损失最小。

#### 交错的交叉注意力和自注意力

在动作专家内部，注意力层在以下两者之间交替：

*   交叉注意力：动作Token关注VLM的特征
*   自注意力：动作Token相互关注（因果地——仅关注过去的Token）

我们发现这种交错设计比使用完整的注意力块更轻量且更有效。仅依赖CA或仅依赖SA的模型往往会牺牲平滑性或基础性。

在SmolVLA中，CA确保动作在感知和指令上有良好的条件约束，而SA则提高了时间平滑性——这对于现实世界的控制尤其关键，因为抖动的预测可能导致不安全或不稳定的行为。

## 异步推理

![异步推理](/images/posts/65dd69410b6e.png)

图3. 异步推理。异步推理栈的示意图。注意，策略可以在远程服务器上运行，可能使用GPU。

现代视觉运动策略输出动作块——要执行的动作序列。管理它们有两种方式：

*   同步：机器人执行一个块，然后在计算下一个块时暂停。简单，但会导致延迟，在此期间机器人无法对新输入做出反应。
*   异步：在执行当前块的同时，机器人已经将最新的观测发送到策略服务器（可能托管在GPU上）以计算下一个块。这避免了空闲时间并提高了反应性。

我们的异步栈将动作执行与块预测解耦，从而实现了更高的适应性，并且在运行时完全没有执行延迟。它依赖于以下关键机制：

1.  提前触发：当队列长度低于阈值（例如70%）时，我们向策略服务器发送一个观测，请求新的动作块。
2.  解耦的线程：控制循环持续执行 → 推理并行发生（非阻塞）。
3.  块融合：来自连续块的重叠动作通过简单的合并规则进行拼接，以避免抖动。

我们对发布异步推理感到非常兴奋，因为它保证了在不改变模型的情况下获得更高的适应性和改进的性能。简而言之，异步推理通过重叠执行和远程预测来保持机器人的响应能力。

## 社区数据集

虽然视觉和语言模型在LAION、ImageNet和Common Crawl等网络规模的数据集上蓬勃发展，但机器人学缺乏类似的资源。没有“机器人互联网”。相反，数据分散在不同的机器人类型、传感器、控制方案和格式中——形成了孤立的“数据孤岛”。在我们之前的文章中，我们探讨了如何通过开放、协作的努力来解决这种碎片化问题。正如ImageNet通过提供庞大、多样的基准测试推动了计算机视觉的突破，我们相信社区驱动的机器人数据集可以为通用机器人策略发挥同样的基础性作用。

SmolVLA是我们迈向这一愿景的第一步：它是在精心策划的、公开可用的、社区贡献的数据集混合上进行预训练的，这些数据集旨在反映现实世界的变化。我们不仅优化数据集大小，更注重多样性：一系列行为、相机视角和具身形式，以促进迁移和泛化。

SmolVLA中使用的所有训练数据都来自LeRobot社区数据集，这些机器人数据集在Hugging Face Hub上以`lerobot`标签共享。这些数据集收集自从实验室到客厅的各种环境，代表了一种开放、去中心化的努力，以扩展现实世界的机器人数据。

### 改进任务标注

社区数据集的一个常见问题是任务描述嘈杂或缺失。许多片段缺乏标注，或者包含模糊的标签，如“task desc”或“Move”、“Pick”。为了提高质量并标准化跨数据集的文本输入，我们使用Qwen2.5-VL-3B-Instruct来生成简洁的、面向动作的描述。

给定样本帧和原始标签，我们提示模型在30个字符以内重写指令，以动作动词开头（例如，“Pick”、“Place”、“Open”）。

使用的提示词是：

以下为当前任务描述：{current_task}。请生成一句非常简短、清晰且完整的句子，描述机械臂执行的动作（最多30个字符）。不要包含不必要的词语。
保持简洁。
以下是一些示例：拾取立方体并将其放入盒子中、打开抽屉等。
请直接以动作动词开头，例如“拾取”、“放置”、“打开”等。
类似于所提供的示例，机械臂完成的主要动作是什么？

### 标准化相机视角

另一个挑战是相机命名不一致。一些数据集使用了清晰的名称，如 `top` 或 `wrist.right`，而另一些则使用了含义模糊的标签，如 `images.laptop`，其含义各不相同。
为了解决这个问题，我们手动检查了数据集，并将每个相机视角映射到一个标准化方案中：`OBS_IMAGE_1`：俯视图 `OBS_IMAGE_2`：腕部视角 `OBS_IMAGE_3+`：附加视角

我们进一步分离了社区数据集预训练和多任务微调的贡献。如果没有在 LeRobot 社区数据集上进行预训练，SmolVLA 在 SO100 上的初始成功率是 `51.7%`。在社区收集的数据上进行预训练后，性能跃升至 `78.3%`，绝对提升了 `+26.6%`。多任务微调进一步提升了性能，即使在低数据情况下也显示出强大的任务迁移能力。
