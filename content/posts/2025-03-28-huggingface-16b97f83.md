# 🚀 在 Intel Gaudi 上使用 TGI 加速 LLM 推理

我们很高兴地宣布，已将 Intel Gaudi 硬件的原生支持直接集成到 Text Generation Inference (TGI) 中——这是我们面向大语言模型 (LLM) 的生产级服务解决方案。此次集成将 Intel 专用 AI 加速器的强大能力引入我们的高性能推理栈，为开源 AI 社区提供更多部署选择 🎉

## ✨ 新特性

我们已在 PR#3091 中将 Gaudi 支持完全集成到 TGI 的主代码库中。此前，我们在 `tgi-gaudi` 中维护了一个独立的 Gaudi 设备分支。这对用户来说操作繁琐，且导致我们无法在发布时支持最新的 TGI 功能。现在，借助新的 TGI 多后端架构，我们直接在 TGI 上支持 Gaudi——无需再折腾自定义仓库 🙌

该集成支持 Intel 全系列 Gaudi 硬件：

- Gaudi1 💻：可在 AWS EC2 DL1 实例上使用
- Gaudi2 💻💻：可在 Intel Tiber AI Cloud 和 Denvr Dataworks 上使用
- Gaudi3 💻💻💻：可在 Intel Tiber AI Cloud、IBM Cloud 以及 Dell、HP 和 Supermicro 等 OEM 厂商处使用

您还可以在 Intel Gaudi 产品页面上找到更多关于 Gaudi 硬件的信息

## 🌟 为何重要

TGI 的 Gaudi 后端提供了几个关键优势：

- 硬件多样性 🔄：除传统 GPU 外，为生产环境部署 LLM 提供更多选择
- 成本效益 💰：Gaudi 硬件在特定工作负载下通常提供极具竞争力的性价比
- 生产就绪 ⚙️：TGI 的所有稳健特性（动态批处理、流式响应等）现可在 Gaudi 上使用
- 模型支持 🤖：在 Gaudi 硬件上运行 Llama 3.1、Mixtral、Mistral 等流行模型
- 高级特性 🔥：支持多卡推理（分片）、视觉语言模型和 FP8 精度

## 🚦 在 Gaudi 上开始使用 TGI

在 Gaudi 上运行 TGI 最简单的方法是使用我们的官方 Docker 镜像。您需要在 Gaudi 硬件机器上运行该镜像。以下是一个入门的基本示例：

```bash
model=meta-llama/Meta-Llama-3.1-8B-Instruct 
volume=$PWD/data 
hf_token=YOUR_HF_ACCESS_TOKEN

docker run --runtime=habana --cap-add=sys_nice --ipc=host \
 -p 8080:80 \
 -v $volume:/data \
 -e HF_TOKEN=$hf_token \
 -e HABANA_VISIBLE_DEVICES=all \
 ghcr.io/huggingface/text-generation-inference:3.2.1-gaudi \
 --model-id $model 

```

服务器运行后，您可以发送推理请求：

```bash
curl 127.0.0.1:8080/generate
 -X POST
 -d '{"inputs":"What is Deep Learning?","parameters":{"max_new_tokens":32}}'
 -H 'Content-Type: application/json'

```

有关在 Gaudi 上使用 TGI 的全面文档，包括操作指南和高级配置，请参阅新的专用 Gaudi 后端文档。
