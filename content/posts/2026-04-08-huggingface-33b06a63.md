# ALTK‑Evolve：AI Agent（智能体）的持续在职学习

## 内容提要

- 大多数 AI Agent（智能体）只会重读执行记录，而非学习原则，因此它们会重复犯错，且无法将经验教训迁移到新情境中。
- ALTK‑Evolve 能将原始的 Agent（智能体）执行轨迹转化为可复用的指导原则。
- 在基准测试中，该方法显著提升了可靠性，尤其是在困难任务（在 AppWorld 上提升 Δ 14.2%）和多步骤任务上，且不会增加上下文负担。

## “永恒的实习生”问题

想象一位才华横溢的帮厨，他熟记了所有菜谱，但每天早上都会忘记你的厨房布局。他不记得你的烤箱温度偏高，也不记得老顾客喜欢多放盐；他会严格按照食谱卡操作，但当你没有柠檬时就会手足无措。这就是大多数 AI Agent（智能体）的现状：它们擅长遵循提示词，却不善于积累关于你所在环境的智慧。将昨天的日志重新喂给提示词，只是让它们重读历史；这并不能帮助它们从中归纳出通用原则。

一个新手需要为“油醋汁”和“橙汁鸭”准备不同的食谱。而一位主厨则学会了“酸味平衡脂肪”的原则，并将其应用到各处。同样，可靠的 Agent（智能体）应该从经验中提炼原则，并将其应用于新任务，而不仅仅是旧任务的近似复制。这个长期记忆子系统正是为此而生：它将交互轨迹转化为候选指导原则，进行质量筛选，并仅在行动时刻注入相关的指导。Agent（智能体）需要的是原则，而非记录。

最近的一项MIT研究发现，95%的试点失败是因为 Agent（智能体）无法适应和在工作中学习。ALTK-Evolve 利用长期情景记忆来解决这一学习差距，帮助 Agent（智能体）更好地进行推理。

## 解决方案：ALTK‑Evolve 的长期记忆

Evolve 是一个用于 AI Agent（智能体）的记忆系统，可以帮助 Agent（智能体）随时间推移而改进，从先前执行生成的指导原则中学习并加以利用。

在操作上，该系统作为一个持续循环运行：

1.  **向下流动（观察与提取）**：在交互层（例如 Langfuse 或其他基于 OpenTelemetry 的可观测性工具）中捕获完整的 Agent（智能体）轨迹（用户话语、思考、工具调用、结果）。可插拔的提取器从轨迹中挖掘结构化模式，并将其持久化为候选实体。
2.  **向上流动（精炼与检索）**：一个后台的整合与评分作业会合并重复项、修剪弱规则、并提升已验证的策略，从而演化出一个高质量的实体库，例如指导原则、策略和标准操作程序。检索过程仅通过交互层拉取相关条目，并将其注入回应用层的上下文中。

![architecture (1)](/images/posts/3d5a3c355cf9.png)

这种方法行之有效，主要有以下几个关键原因：

-   **教授判断力**：将一次性事件转化为可跨任务迁移的通用策略。
-   **控制噪声**：评分机制使记忆保持精简有用，而非一个不断增长的杂物抽屉。
-   **渐进式披露**：检索是即时进行的，不会将所有内容都塞入上下文。

我们在AppWorld上评估了该框架，其中 Agent（智能体）通过 API 完成现实的多步骤任务，平均跨 1.8 个应用调用 9.5 个 API，困难案例需要更复杂的控制流。一个 ReAct Agent（智能体）接收任务指令加上先前运行（训练/开发集）中生成的前 5 条检索到的指导原则，并在未见过的分区（测试-常规集）上进行测试。我们报告了场景目标完成率，这是一个严格的、要求在所有变体上都成功的连贯性指标。

以下是评估中的一些关键结论：

-   **泛化能力**：Agent（智能体）在未见过的测试-常规任务上有所改进，证明它正在学习原则，而非死记硬背“食谱”。
-   **复杂性扩展**：任务越困难，Agent（智能体）从简洁的学习到的指导原则中获益越多，在更困难的任务上提升最大。困难任务的成功率相对提高了 74%，在这些任务中，指导原则对于驾驭复杂的控制流非常有用。

我们通过 MCP 将 ALTK‑Evolve 直接集成到CUGA中，创建了一个紧密、低开销的学习循环。在每次运行前，会调用 `get_guidelines` MCP 工具来呈现特定任务的指导，减少试错。运行结束后，CUGA 通过 `save_trajectory` 发送回结构化的执行轨迹，以便 Evolve 可以从实际发生的情况中学习并改进未来的指导。其结果是形成了一个随时间推移而不断改进，同时保持透明、可组合且易于采用的集成。
