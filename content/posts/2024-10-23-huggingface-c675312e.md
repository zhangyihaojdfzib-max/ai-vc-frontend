# 介绍 SynthID Text

你是否觉得难以区分文本是由人类撰写还是由 AI 生成的？能够识别 AI 生成的内容对于促进对信息的信任、帮助解决错误归属和错误信息等问题至关重要。今天，Google DeepMind 和 Hugging Face 激动地宣布，在即将发布的 Transformers v4.46.0 中推出 SynthID Text。该技术允许你使用一个 logits 处理器为生成任务中的 AI 生成文本添加水印，并通过一个分类器检测这些水印。

请查阅发表在《Nature》上的 SynthID Text 论文，了解该算法的完整技术细节，并访问 Google 的 Responsible GenAI Toolkit，了解如何在你的产品中应用 SynthID Text。

## 工作原理

SynthID Text 的主要目标是以一种方式将水印编码到 AI 生成的文本中，帮助你确定文本是否来自你的 LLM，同时不影响底层 LLM 的工作方式或对生成质量产生负面影响。Google DeepMind 开发了一种水印技术，该技术使用一个称为 g-function 的伪随机函数来增强任何 LLM 的生成过程，使得水印对人类不可察觉，但对训练过的模型可见。该技术已实现为一个生成工具，兼容任何 LLM，无需修改即可使用 `model.generate()` API，同时提供了一个端到端示例，展示如何训练检测器识别带水印的文本。请查阅研究论文，其中包含关于 SynthID Text 算法的更完整细节。

## 配置水印

水印通过一个数据类进行配置，该数据类参数化 g-function 及其在 tournament 采样过程中的应用方式。你使用的每个模型都应拥有自己的水印配置，并且应安全且私密地存储，否则你的水印可能被他人复制。

在每个水印配置中，你必须定义两个参数：

- `keys` 参数是一个整数列表，用于计算模型词汇表中各 token 的 g-function 分数。建议使用 20 到 30 个唯一且随机生成的数字，以平衡可检测性与生成质量。
- `ngram_len` 参数用于平衡鲁棒性和可检测性；值越大，水印越容易被检测到，但代价是更容易因变化而失效。一个良好的默认值是 5，但至少需要为 2。

`keys` 参数是一个整数列表，用于计算模型词汇表中各 token 的 g-function 分数。建议使用 20 到 30 个唯一且随机生成的数字，以平衡可检测性与生成质量。

`ngram_len` 参数用于平衡鲁棒性和可检测性；值越大，水印越容易被检测到，但代价是更容易因变化而失效。一个良好的默认值是 5，但至少需要为 2。

你可以根据性能需求进一步配置水印。有关更多信息，请参阅 `SynthIDTextWatermarkingConfig` 类。

研究论文中包含关于特定配置值如何影响水印性能的额外分析。

## 应用水印

应用水印是对现有生成调用的一个简单修改。定义好配置后，将一个 `SynthIDTextWatermarkingConfig` 对象作为 `watermarking_config=` 参数传递给 `model.generate()`，所有生成的文本都将携带水印。请查看 SynthID Text Space，获取水印应用的交互式示例，并看看你是否能分辨出来。

```py
from transformers import (
    AutoModelForCausalLM,
    AutoTokenizer,
    SynthIDTextWatermarkingConfig,
)


tokenizer = AutoTokenizer.from_pretrained('repo/id')
model = AutoModelForCausalLM.from_pretrained('repo/id')


watermarking_config = SynthIDTextWatermarkingConfig(
    keys=[654, 400, 836, 123, 340, 443, 597, 160, 57, ...],
    ngram_len=5,
)


tokenized_prompts = tokenizer(["your prompts here"])
output_sequences = model.generate(
    **tokenized_prompts,
    watermarking_config=watermarking_config,
    do_sample=True,
)
watermarked_text = tokenizer.batch_decode(output_sequences)

```

## 检测水印

水印设计为可被训练过的分类器检测到，但对人类不可察觉。你与模型一起使用的每个水印配置都需要训练一个检测器来识别该水印。

基本的检测器训练流程如下：

- 确定一个水印配置。
- 收集一个检测器训练集，分为带水印和不带水印，以及训练集和测试集，建议至少包含 10k 个样本。
- 使用你的模型生成不带水印的输出。
- 使用你的模型生成带水印的输出。
- 训练你的水印检测分类器。
- 将你的模型与水印配置及关联的检测器投入生产。

Transformers 中提供了一个贝叶斯检测器类，以及一个端到端示例，展示如何使用特定水印配置训练检测器识别带水印的文本。使用相同 tokenizer 的模型也可以共享水印配置和检测器，从而共享一个共同的水印，前提是检测器的训练集包含所有共享该水印的模型的样本。

这个训练好的检测器可以上传到私有 HF Hub，以便在你的组织内访问。Google 的 Responsible GenAI Toolkit 提供了更多关于如何将 SynthID Text 投入生产的信息。

## 局限性

SynthID Text 水印对某些变换具有鲁棒性，例如裁剪文本片段、修改几个单词或轻度改写，但该方法确实存在局限性。

- 水印应用对事实性回答效果较差，因为在不降低准确性的情况下增强生成的机会较少。
- 当 AI 生成的文本被彻底重写或翻译成另一种语言时，检测器的置信度分数可能会大幅降低。

SynthID Text 并非旨在直接阻止有动机的对手造成伤害。然而，它可以使 AI 生成的内容更难被用于恶意目的，并且可以与其他方法结合使用，以更好地覆盖不同类型的内容和平台。
