# 推出 GPT‑5.4

专为专业工作设计

今天，我们在 ChatGPT（作为 GPT‑5.4 Thinking）、API 和 Codex 中发布 GPT‑5.4。这是我们为专业工作打造的最强大、最高效的边界模型。同时，我们还在 ChatGPT 和 API 中推出 GPT‑5.4 Pro，以满足用户在复杂任务中对极致性能的需求。

GPT‑5.4 将我们近期在推理、编码和 Agent（智能体）工作流方面的最佳进展整合到一个统一的边界模型中。它融合了 GPT‑5.3‑Codex 行业领先的编码能力，同时改进了模型在工具、软件环境以及涉及电子表格、演示文稿和文档的专业任务中的工作方式。最终，该模型能够准确、有效且高效地完成复杂的实际工作——以更少的来回交互交付您所需的结果。

在 ChatGPT 中，GPT‑5.4 Thinking 现在可以预先提供其思考计划，因此您可以在其工作过程中**中途调整方向**，并在无需额外轮次的情况下，获得更符合您需求的最终输出。GPT‑5.4 Thinking 还改进了**深度网络研究**，特别是针对高度具体的查询，同时**更好地维持上下文**以处理需要更长时间思考的问题。这些改进共同意味着更高质量的答案，这些答案能更快生成并始终与当前任务相关。

在 Codex 和 API 中，GPT‑5.4 是我们发布的首个具备原生、最先进**计算机使用能力**的通用模型，使 Agent（智能体）能够操作计算机并在不同应用程序间执行复杂工作流。它支持高达 **100 万 Token 的上下文窗口**，使 Agent（智能体）能够跨长时间跨度规划、执行和验证任务。GPT‑5.4 还通过**工具搜索**改进了模型在大型工具和连接器生态系统中的工作方式，帮助 Agent（智能体）更高效地找到并使用合适的工具，同时不牺牲智能水平。最后，GPT‑5.4 是我们迄今为止**Token 效率最高的推理模型**，与 GPT‑5.2 相比，在解决问题时使用的 Token 显著减少——这意味着更低的 Token 消耗和更快的速度。

结合在通用推理、编码和专业知识工作方面的进步，GPT‑5.4 在 ChatGPT、API 和 Codex 中实现了更可靠的 Agent（智能体）、更快的开发者工作流以及更高质量的输出。

## 知识工作

在 GPT‑5.2 通用推理能力的基础上，GPT‑5.4 在专业人士关心的实际任务上提供了更加一致和精炼的结果。

在 **GDPval**（测试 Agent（智能体）在 44 个职业中生成明确知识工作的能力）上，GPT‑5.4 达到了新的最先进水平，在 **83.0%** 的比较中匹配或超越行业专业人士，而 GPT‑5.2 为 70.9%。

在 GDPval 中，模型尝试跨越美国 GDP 贡献最大的前 9 个行业中 44 个职业的明确知识工作。任务要求生成实际工作产品，例如销售演示文稿、会计电子表格、急诊安排表、制造图表或短视频。GPT‑5.4 的推理努力设置为 xhigh，GPT‑5.2 设置为 heavy（ChatGPT 中略低的级别）。

我们特别关注改进 GPT‑5.4 创建和编辑电子表格、演示文稿和文档的能力。在一个模拟初级投资银行分析师可能做的电子表格建模任务的内部基准测试中，GPT‑5.4 的平均得分为 **87.3%**，而 GPT‑5.2 为 68.4%。在一组演示文稿评估提示词中，人类评估员在 **68.0%** 的情况下更偏好 GPT‑5.4 生成的演示文稿，而非 GPT‑5.2 的，原因是其更强的美学效果、更大的视觉多样性以及更有效地使用图像生成。

您可以在 ChatGPT 中使用 GPT‑5.4 Thinking 或 Pro 尝试这些功能。如果您是企业客户，我们建议使用我们新发布的 **ChatGPT for Excel 插件**（在新窗口中打开），该插件也于今天推出。我们还更新了 Codex 和 API 中可用的**电子表格**（在新窗口中打开）和**演示文稿技能**（在新窗口中打开）。

为了使 GPT‑5.4 在实际工作中表现更好，我们继续在减少幻觉和错误方面取得进展。GPT‑5.4 是我们迄今为止最符合事实的模型：在一组用户标记了事实错误的去标识化提示词中，相对于 GPT‑5.2，GPT‑5.4 的单个声明错误可能性降低 33%，其完整响应包含任何错误的可能性降低 18%。

## 计算机使用与视觉

GPT‑5.4 是我们首款具备原生**计算机使用能力**的通用模型，标志着开发者和 Agent（智能体）领域的重大进步。对于构建能在网站和软件系统中完成实际任务的 Agent（智能体）的开发者来说，它是目前可用的最佳模型。

我们设计了 GPT‑5.4，使其在广泛的计算机使用工作负载中表现出色。它擅长编写代码以通过 Playwright 等库操作计算机，以及根据截图发出鼠标和键盘命令。其行为可通过开发者消息进行引导，这意味着开发者可以调整行为以适应特定用例。开发者甚至可以通过指定自定义确认策略，为模型配置不同风险容忍度的安全行为。

该模型的性能和灵活性体现在不同环境下测试计算机使用的基准测试中。在 **OSWorld-Verified**（衡量模型通过截图和键盘/鼠标操作导航桌面环境的能力）上，GPT‑5.4 达到了最先进的 **75.0%** 成功率，远超 GPT‑5.2 的 47.3%，并超越了人类表现（72.4%）。¹

在 **WebArena-Verified**（测试浏览器使用）上，GPT‑5.4 在使用 DOM 和截图驱动交互时达到了领先的 **67.3%** 成功率，而 GPT‑5.2 为 65.4%。在 **Online-Mind2Web**（同样测试浏览器使用）上，GPT‑5.4 仅使用基于截图的观察就达到了 **92.8%** 的成功率，优于 ChatGPT Atlas 的 Agent 模式（成功率为 70.9%）。

工具让步是指助手让步以等待工具响应。如果并行调用 3 个工具，然后再并行调用 3 个工具，则让步次数为 2。工具让步比工具调用更能反映延迟，因为它们体现了并行化的优势。

GPT‑5.4 解释浏览器界面的截图，并通过基于坐标的点击与 UI 元素交互，以发送电子邮件和安排日历事件。视频未加速。

GPT‑5.4 改进的计算机使用能力建立在其改进的通用视觉感知能力之上。在 **MMMU-Pro**（模型视觉理解和推理测试）上，GPT‑5.4 在不使用工具的情况下达到了 **81.2%** 的成功率，优于 GPT‑5.2 的 79.5%。改进的视觉感知也转化为更好的文档解析能力。在 **OmniDocBench** 上，GPT‑5.4 在不使用推理努力的情况下，平均误差（通过模型预测与真实值之间的归一化编辑距离衡量）为 **0.109**，优于 GPT‑5.2 的 0.140。

MMMUPro 运行时的推理努力设置为 xhigh。OmniDocBench 运行时的推理努力设置为 none，以反映低成本、低延迟的性能。

我们也在改进对高密度、高分辨率图像的视觉理解能力，这类图像对完整保真度有较高要求。从 GPT‑5.4 开始，我们引入了一种新的**原始图像输入细节**⁠（在新窗口中打开）级别，支持最高 10.24M 总像素或 6000 像素最大边长（取两者中较低值）的全保真感知；**高**图像输入细节级别现在支持最高 2.56M 总像素或 2048 像素最大边长。在面向 API 用户的早期测试中，我们观察到，使用 **原始** 或 **高** 细节时，模型在定位能力、图像理解和点击准确率方面均有显著提升。

在 API 中，开发者可以通过更新后的 **计算机工具** 使用这些功能。请参阅我们的 **更新文档**⁠（在新窗口中打开）了解推荐的最佳实践。

## 编程

GPT‑5.4 结合了 GPT‑5.3‑Codex 的编程优势与领先的知识工作和计算机使用能力，这些能力在需要模型使用工具、迭代并在较少人工干预下推进工作的较长任务中最为关键。在 SWE-Bench Pro 上，它的表现与 GPT‑5.3‑Codex 持平或更优，同时推理延迟更低。

我们通过观察模型的生产行为并离线模拟来估算延迟。延迟估算考虑了工具调用时长（代码执行时间）、采样 Token 和输入 Token。实际延迟可能有显著差异，并且取决于我们模拟中未包含的许多因素。推理力度从“无”到“极高”进行了扫描。

在 Codex 中开启 **/fast** 模式后，GPT‑5.4 的 Token 生成速度最高可提升 1.5 倍。这是同一个模型，同样的智能，只是速度更快。这意味着用户可以在编程、迭代和调试过程中保持流畅状态。开发者也可以通过 API 使用 **优先级处理**⁠（在新窗口中打开）以同样的高速访问 GPT‑5.4。

在评估和内部测试中，我们发现 GPT‑5.4 在复杂的前端任务中表现出色，其结果在美观性和功能性上均显著优于我们之前发布的所有模型。

为了展示模型在计算机使用和编程能力上的协同提升，我们还发布了一项实验性的 Codex 技能，名为“Playwright（交互式）”⁠（在新窗口中打开）。这使得 Codex 能够对 Web 和 Electron 应用进行可视化调试；它甚至可以在构建应用的同时，对正在构建的应用进行测试。

使用 GPT‑5.4 根据一个简单的提示词制作的游乐园模拟游戏，利用 Playwright Interactive 进行浏览器游戏测试，并利用图像生成立体像素资源集。该模拟包括基于瓦片的路径放置、游乐设施和景观建造、游客寻路、排队和游乐设施循环，而金钱、游客数量、满意度、清洁度和评分等公园指标会根据布局表现和游客反应而升降。Playwright 被用于自动化浏览器游戏测试，通过构建和扩展公园、放置和移除路径及景点、检查摄像头导航，并验证游客、队列、游乐设施状态和 UI 指标在几轮游戏过程中是否正确更新。

提示词：使用 $playwright-interactive 和 $imagegen。创建一个交互式的立体像素游乐园模拟游戏，我可以在浏览器中构建和导航。使用 imagegen 建立整体视觉愿景并生成游戏资源，包括游乐设施、路径、地形、树木、水域、小吃摊、装饰物、建筑、图标和 UI 插图。世界应感觉连贯、精致且视觉丰富，具有从立体视角看效果良好的高级美术风格。让我能够放置和移除路径、添加景点、布置景观，并在监控游客活动、游乐设施状态和公园发展的同时，在公园内平滑移动。包括可信的游客移动、简单的公园管理系统（如金钱、清洁度、排队和满意度），并使体验感觉有趣、清晰且完整，而不是一个粗糙的原型。优先考虑魅力、可读性和强烈的游戏感，而非真实感。

在游戏测试时，务必通过几轮游戏来构建和扩展公园，验证放置和导航是否顺畅，确认游客对公园布局和景点的反应，并确保视觉效果、UI 和交互感觉稳定且连贯。

## 工具使用

通过 GPT‑5.4，我们显著改进了模型与外部工具协作的方式。Agent（智能体）现在可以在更大的工具生态系统中运行，更可靠地选择正确的工具，并以更低的成本和延迟完成多步骤工作流。

### 工具搜索

在 API 中，GPT‑5.4 引入了 **工具搜索**⁠（在新窗口中打开），这使得模型在拥有大量工具时也能高效工作。

以前，当模型获得工具时，所有工具定义都会预先包含在提示词中。对于拥有众多工具的系统，这可能为每个请求增加数千甚至数万个 Token，从而增加成本、减慢响应速度，并用模型可能永远不会使用的信息挤占上下文窗口。

通过工具搜索，GPT‑5.4 接收的是一个轻量级的可用工具列表以及一个工具搜索能力。当模型需要使用某个工具时，它可以查找该工具的定义，并在那一刻将其附加到对话中。

这种方法显著减少了需要大量工具的工作流所需的 Token 数量，并保留了缓存，使请求更快、更便宜。它还使 Agent（智能体）能够可靠地与更大的工具生态系统协作。对于可能包含数万个 Token 工具定义的 MCP 服务器，效率提升可能非常显著。

为了展示效率提升，我们使用 Scale 的 **MCP Atlas**⁠（在新窗口中打开）基准测试评估了 250 个任务，所有 36 个 MCP 服务器以两种模式启用：（1）将每个 MCP 函数直接暴露在模型上下文中，以及（2）将所有 MCP 服务器置于工具搜索之后。工具搜索配置将总 Token 使用量减少了 47%，同时达到了相同的准确率。

示例 Token 数量来自对 MCP-Atlas 公共数据集中 250 个任务的平均值。

### Agent（智能体）式工具调用

GPT‑5.4 还改进了 **工具调用**，使其在推理过程中决定何时以及如何使用工具时更加准确和高效，尤其是在 API 中。与 GPT‑5.2 相比，它在 Toolathlon（一个测试 AI Agent（智能体）使用真实世界工具和 API 完成多步骤任务能力的基准测试）上，以更少的轮次实现了更高的准确率。例如，一个 Agent（智能体）需要阅读电子邮件、提取作业附件、上传、评分并将结果记录在电子表格中。

对于偏好推理力度为“无”的延迟敏感型用例，GPT‑5.4 在其前代基础上进一步改进。

在 **τ2-bench**⁠（在新窗口中打开）中，模型必须使用工具来完成一项客户服务任务，其中可能有一个模拟用户能够与世界状态进行通信和采取行动。推理力度设置为“无”。

### 改进的网络搜索

GPT‑5.4 在 Agent（智能体）式网络搜索方面表现更佳。在 BrowseComp（一项衡量 AI Agent（智能体）持续浏览网页以查找难以定位信息能力的指标）上，GPT‑5.4 相比 GPT‑5.2 提升了 **17 个百分点**，而 GPT‑5.4 Pro 以 89.3% 的成绩创下了新的最优水平。

在实践中，这意味着 GPT‑5.4 Thinking 在回答需要从网络上多个来源汇总信息的问题时能力更强。它可以更持久地进行多轮搜索以识别最相关的来源，尤其是对于“大海捞针”式的问题，并将这些信息综合成一个清晰、论证充分的答案。

在BrowseComp中，我们使用了一个搜索阻止列表，排除了包含基准答案的评估网站，以防止污染并确保性能的公平衡量。GPT‑5.4的测量时间晚于GPT‑5.2，因此分数反映了模型、我们的搜索系统以及互联网状态的变化。GPT‑5.4在测试时使用了更长的、更新后的阻止列表。模型使用ChatGPT搜索工具，该工具与API搜索可能存在细微差异。

## 可操控性

类似于Codex在开始工作时概述其方法的方式，GPT‑5.4 Thinking在ChatGPT中现在会为更长、更复杂的查询添加一个前言来概述其工作。您还可以在回答过程中添加指令或调整其方向。这使得引导模型达到您想要的确切结果变得更加容易，无需重新开始或进行多次额外轮次。此功能现已可在chatgpt.com⁠（在新窗口中打开）和Android应用上使用，即将在iOS应用上推出。

该模型还可以在更困难的任务上思考更长时间，同时保持对对话中先前步骤更强的感知能力。这使其能够处理更长的工作流程和更复杂的提示词，同时在整个过程中保持回答的连贯性和相关性。

## 安全性

在过去的几个月里，我们在准备部署GPT‑5.4的同时，持续改进了随GPT‑5.3‑Codex引入的安全措施。与GPT‑5.3‑Codex类似，根据我们的准备框架，我们将GPT‑5.4视为高网络安全能力模型，并按照系统卡⁠（在新窗口中打开）中记录的方式，部署了相应的保护措施。这些措施包括扩展的网络安全堆栈，包括监控系统、可信访问控制，以及对零数据保留（ZDR）表面上的高风险请求进行异步阻止，同时持续投资于更广泛的安全生态系统。

由于网络安全能力本质上是双重用途的，我们在部署时保持谨慎态度，同时继续调整我们的策略和分类器。对于ZDR表面上的某些客户，请求级阻止仍是我们网络风险缓解堆栈的一部分；由于分类器仍在改进中，随着我们不断完善这些安全措施，可能会出现一些误报。这些更新旨在改善安全措施的实际运作方式，包括减少不必要的拒绝和过度谨慎的回答，同时保持对滥用的强大防护。
