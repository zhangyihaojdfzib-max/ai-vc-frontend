## 摘要

一个影响 Next.js 图像优化功能的漏洞已得到修复。该漏洞影响版本低于 v15.4.5 和 v14.2.31 的情况，涉及缓存投毒问题，导致来自 API 路由的敏感图像响应被缓存，并随后提供给未经授权的用户。

Vercel 部署从未受此漏洞影响。

## 影响

当 API 路由用于返回基于请求头（例如 `Cookie`、`Authorization`）而变化的图像内容，并且这些图像通过 Next.js 图像优化功能处理时，优化后的图像可能会被缓存，而缓存键中并未包含这些请求头。这可能导致：

*   **未经授权地泄露**用户特定或受保护的图像内容
*   通过 CDN 或内部缓存**跨用户泄露**条件性内容

此问题的发生无需用户交互，也不需要提升的权限，仅需要一个先前的授权请求来填充缓存。

## 解决方案

该问题已通过确保请求头不会被转发到代理到图像端点的请求中得到解决。这确保了图像端点不能被用于提供需要授权数据的图像，因此也就不会被缓存。

修复版本：
*   Next.js v15.4.5
*   Next.js v14.2.31
