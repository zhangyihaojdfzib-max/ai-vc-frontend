# 保护非人类身份：自动化吊销、OAuth 与范围化权限

2026-04-14

- Justin Hutchings
- Adam Bouhmad
- Rebecca Varley

![](/images/posts/f5c4f265f4ab.png)

Agent（智能体）能让你以前所未有的速度构建软件，但保护你的环境和所编写的代码——既要防范错误也要抵御恶意行为——需要付出真正的努力。开放网络应用安全项目（OWASP）详细列举了智能体 AI 系统中存在的诸多风险，包括凭证泄露、用户冒充和权限提升的风险。这些风险可能导致你的环境遭受极端损害，包括服务拒绝、数据丢失或数据泄露——这可能造成难以估量的财务和声誉损失。

这是一个身份问题。在现代开发中，“身份”不仅仅是人——还包括代表你行事的 Agent（智能体）、脚本和第三方工具。要保护这些非人类身份，你需要管理它们的整个生命周期：确保其凭证（Token）不被泄露，查看哪些应用程序通过 OAuth 拥有访问权限，并使用细粒度的 RBAC 来缩小其权限范围。

今天，我们推出更新以满足这些需求：可扫描的 Token 以保护你的凭证，OAuth 可见性以管理你的主体，以及资源范围化的 RBAC 以微调你的策略。

## 理解身份：主体、凭证与策略

在自主 Agent（智能体）时代保护互联网安全，我们必须重新思考如何处理身份。无论请求是来自人类开发者还是 AI Agent（智能体），与 API 的每次交互都依赖于三个核心支柱：

- **主体（旅行者）**：这就是身份本身——“谁”。可能是你通过 OAuth 登录，也可能是一个后台 Agent（智能体）使用 API Token 来部署代码。
- **凭证（护照）**：这是该身份的证明。在这个世界里，你的 API Token 就是你的护照。如果它被盗或泄露，任何人都可以“冒充”你的身份。
- **策略（签证）**：这定义了该身份被允许做什么。仅仅因为你持有有效护照，并不意味着你拥有进入每个国家的签证。策略确保即使是经过验证的身份，也只能访问其需要的特定资源。

当这三个支柱没有被统一管理时，安全性就会崩溃。你可能有一个使用被盗凭证的有效主体，或者一个拥有过于宽泛策略的合法身份。

## 泄露 Token 检测

Agent（智能体）和其他第三方应用程序使用 API Token 来访问 Cloudflare API。我们看到人们泄露其密钥最简单的方式之一，就是意外地将它们推送到公共 GitHub 仓库。GitGuardian 报告称，去年有超过 2800 万个密钥被发布到公共 GitHub 仓库，并且 AI 导致泄露发生的速度比以前快了 5 倍。

如果说 API Token 是数字护照，那么将其泄露到公共仓库就像把你的护照丢在公园长椅上。任何发现它的人都可以冒充该身份，直到该凭证被注销。我们与 GitHub 的合作就像是为这些凭证建立了一个全球“失物招领处”。在你意识到护照丢失之前，我们已经识别了该凭证，通过校验和验证了其真实性，并使其失效以防止滥用。

我们正在与几家领先的凭证扫描工具合作，以帮助主动发现你泄露的 Token 并在其被恶意使用之前将其吊销。我们知道，在你、你的员工或你的某个 Agent（智能体）犯错并将密钥推送到不该出现的地方之前，这不是“是否”会发生的问题，而是“何时”会发生的问题。

#### GitHub

我们已与 GitHub 合作，并参与了他们的 Secret Scanning 计划，以在公共和私有仓库中查找你的 Token。如果我们收到通知称某个 Token 已泄露到公共仓库，我们将自动吊销该 Token 以防止其被恶意使用。对于私有仓库，GitHub 将通知你任何泄露的 Cloudflare Token，你可以清理这些 Token。

##### 工作原理

我们已与 GitHub 共享了新的 Token 格式（见下文！），他们现在会在每次提交时扫描这些格式。如果他们发现看起来像泄露的 Cloudflare Token 的内容，他们会验证该 Token 是否真实（使用校验和），向我们发送 Webhook 以吊销它，然后我们通过电子邮件通知你，以便你可以在 Dashboard 设置中生成一个新的 Token。

这意味着我们会在漏洞被发现时立即堵上。当你意识到自己犯错时，我们已经修复了它。

我们希望这是一种你不需要使用的功能，但我们的合作伙伴正在密切关注泄露情况，以帮助你保持安全。

#### Cloudflare One

Cloudflare One 客户也受到保护，免受这些泄露的影响。通过配置“凭证和密钥”DLP 配置文件，组织可以在凭证可能传播的任何地方激活防护：

- **网络流量（Cloudflare Gateway）**：将这些条目应用到策略中，以检测和阻止在你的网络中移动的 Cloudflare API Token。文件上传、出站请求或下载中的 Token 会在到达目的地之前被阻止。
- **出站电子邮件（Cloudflare Email Security）**：Microsoft 365 客户可以将同样的防护扩展到 Outlook。DLP Assist 插件在邮件发送前扫描消息，在 Token 被发送到外部之前将其捕获。
- **静态数据（Cloudflare CASB）**：Cloudflare 的云访问安全代理应用相同的配置文件来扫描连接的 SaaS 应用程序中的文件，捕获保存在或共享在 Google Drive、OneDrive、Dropbox 和其他集成服务中的 Token。

不过，最新颖的暴露途径是 AI 流量。Cloudflare AI Gateway 集成了相同的 DLP 配置文件，可以实时扫描和阻止传入的提示词和传出的 AI 模型响应。

#### 其他凭证扫描器

凭证扫描要发挥作用，唯一的方法是在你所在的地方与你对接，因此我们正在与多个开源和商业凭证扫描器合作，以确保无论你使用哪种密钥扫描器，都能得到保护。

到目前为止，Cloudflare 的 API Token 看起来相当通用，因此凭证扫描器很难以高置信度识别它们。这些自动化安全工具扫描你的代码仓库，寻找暴露的凭证，如 API 密钥、Token 或密码。“cf”前缀使 Cloudflare Token 能够以更高的置信度被立即识别，校验和使工具能够轻松地静态验证它们。你现有的 Token 将继续有效，但你生成的每个新 Token 都将使用可扫描的格式，以便能够以高置信度轻松检测到。

## 细粒度资源级权限

如果说令牌是护照，那么资源范围的权限就是其中的签证。拥有有效的护照可以让您通过前门，但它不应该让您进入大楼的每个房间。通过将范围缩小到特定资源——例如单个负载均衡器池或特定的网关策略——您可以确保即使身份已验证，它也仅拥有前往严格必要之处的"签证"。

去年，我们宣布了对Cloudflare基于角色的访问控制（RBAC）系统中资源范围权限的支持，适用于我们的多款零信任产品。这使您能够为用户和Agent（智能体）设置大小合适的权限，以最小化安全风险。我们已经将此功能扩展到多个新的资源级权限。现在支持资源范围的对象包括：

*   Access Applications（访问应用程序）
*   Access Identity Providers（访问身份提供商）
*   Access Policies（访问策略）
*   Access Service Tokens（访问服务令牌）
*   Access Targets（访问目标）

我们还彻底改进了API令牌创建体验，使客户能够直接从Cloudflare仪表板更轻松地配置和管理账户API令牌。

当您向Cloudflare账户添加成员或创建API令牌时，通常会为该主体分配一个策略。权限策略是赋予主体执行操作权限的凭证，无论是管理Cloudflare One Access Applications（访问应用程序）还是DNS记录。没有策略，主体可以进行身份验证，但无权在账户内执行任何操作。

策略由三个组成部分构成：主体、角色和范围。主体是您授予访问权限的对象，无论是人类用户、非人类身份（NHI）（如API令牌），还是日益增多的代表用户行事的Agent（智能体）。角色定义了允许他们执行的操作。范围决定了这些权限的适用范围，历史上，这被限制在整个账户或单个区域。

## 保护您的账户

这些更新提供了构建真正最小权限架构所需的细粒度基础模块。通过改进我们管理权限和凭证的方式，开发人员和企业可以对其用户、应用程序、Agent（智能体）和访问Cloudflare的脚本的安全性态势更有信心。最小权限并非新概念，对于企业而言，它也从来不是可选项。无论是人类管理员管理区域，还是Agent（智能体）以编程方式部署Worker，期望都是一样的：他们只应被授权执行被赋予的任务，别无其他。
