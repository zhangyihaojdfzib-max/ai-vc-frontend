# 深度线性网络中的非线性计算

![深度线性网络中的非线性计算](/images/posts/aa0f686c2fb8.webp)

我们已经证明，使用浮点运算实现的深度线性网络实际上并非线性，并且能够执行非线性计算。我们采用进化策略来寻找利用这一特性的线性网络参数，从而解决非平凡问题。

神经网络由线性层堆叠后接tanh或ReLU等非线性激活函数构成。如果没有非线性激活函数，连续的线性层在理论上将等价于单个线性层。因此，浮点运算具有足够非线性以产生可训练的深度网络这一事实令人惊讶。

## 背景

计算机使用的数字并非完美的数学对象，而是采用有限比特数的近似表示。浮点数是计算机常用的数学对象表示方式。每个浮点数由小数部分和指数部分组合表示。在IEEE的float32标准中，23位用于小数部分，8位用于指数部分，1位用于符号位。

![单精度浮点格式](/images/posts/62d10c9a4cc7.png)

图片来源：维基百科（在新窗口打开）

由于这些约定以及所使用的二进制格式，最小的正规非零数（二进制）为`1.0..0 x 2^-126`，我们将其称为`min`。然而，下一个可表示数为`1.0..01 x 2^-126`，可写作`min + 0.0..01 x 2^-126`。显然，第二个数与`min`之间的差距比0与`min`之间的差距小2^20倍。在float32中，当数值小于最小可表示数时，它们会被映射为零。由于这种"下溢"现象，在零附近所有涉及浮点数的运算都变得非线性。

这些限制的一个例外是非规格化数（在新窗口打开），该特性可在某些计算硬件上禁用。虽然GPU和cuBLAS默认启用非规格化数，但TensorFlow在构建所有原语时均禁用非规格化数（设置`ftz=true`标志）。这意味着TensorFlow中任何非矩阵乘运算之后都隐含着非线性（前提是计算规模接近1e-38）。

因此，尽管通常"数学"数字与其正常浮点表示之间的差异很小，但在零附近存在巨大差距，近似误差可能非常显著。

![Image1](/images/posts/a5c70dfaea94.png)

这可能导致一些奇怪的效果，使得熟悉的数学规则不再适用。例如，`(a+b)×c`变得不等于`a×c+b×c`。

例如，设置`a=0.4×min`，`b=0.5×min`，`c=1/min`。

那么：`(a+b)×c = (0.4×min + 0.5×min) × 1/min = (0+0) × 1/min = 0`。然而：`(a×c)+(b×c) = 0.4×min/min + 0.5×min×1/min = 0.9`。

在另一个例子中，设置`a=2.5×min`，`b=-1.6×min`，`c=1×min`。

那么：`(a+b)+c = (0) + 1×min = min`。然而：`(b+c)+a = (0×min) + 2.5×min = 2.5×min`。

在这个最小尺度上，基本的加法运算已经变得非线性！

## 利用进化策略挖掘非线性

我们想知道这种固有的非线性是否可以被用作计算非线性，这将使深度线性网络能够执行非线性计算。挑战在于现代微分库无法感知这些最小尺度上的非线性。因此，通过反向传播训练神经网络来利用这些非线性将非常困难甚至不可能。

我们可以使用进化策略（在新窗口打开）来估计梯度，而无需依赖符号微分。通过进化策略，我们确实能够利用float32在零附近的行为作为计算非线性。在MNIST上训练时，通过反向传播训练的深度线性网络达到94%的训练准确率和92%的测试准确率。相比之下，相同的线性网络在使用进化策略训练并确保激活值足够小以处于float32非线性范围时，可以达到>99%的训练准确率和96.7%的测试准确率。这种训练性能的提升源于进化策略利用了float32表示中的非线性。这些强大的非线性使得任何层都能生成新颖的特征，这些特征是低层特征的非线性组合。以下是网络结构：
