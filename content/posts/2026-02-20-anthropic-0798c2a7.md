# 将前沿网络安全能力赋予防御者

Claude Code Security 是网页版 Claude Code 内置的一项新功能，现以有限研究预览形式推出。它能扫描代码库中的安全漏洞，并针对性地提出软件补丁建议供人工审核，帮助团队发现并修复传统方法常会遗漏的安全问题。安全团队面临一个普遍挑战：软件漏洞过多，而处理人员不足。现有分析工具虽能提供帮助，但作用有限，因为它们通常只寻找已知模式。要发现攻击者常利用的那些微妙且依赖上下文的漏洞，需要熟练的人类研究人员，而他们正面临着不断积压的工作量。

人工智能正在开始改变这种局面。我们最近证明，Claude 能够检测新颖的高危漏洞。但帮助防御者发现和修复漏洞的能力，同样可能被攻击者用来利用这些漏洞。

Claude Code Security 旨在将这种能力完全交到防御者手中，保护代码免受这类新型人工智能攻击。我们正以有限研究预览的形式向企业版和团队版客户发布，并为开源代码库维护者提供快速访问通道，以便我们共同完善其功能，确保其负责任地部署。

## Claude Code Security 的工作原理

静态分析——一种广泛部署的自动化安全测试形式——通常是基于规则的，这意味着它将代码与已知漏洞模式进行匹配。这能捕获常见问题，如暴露的密码或过时的加密，但常常遗漏更复杂的漏洞，例如业务逻辑缺陷或访问控制失效。

Claude Code Security 并非扫描已知模式，而是像人类安全研究人员一样阅读和理解您的代码：理解组件如何交互，追踪数据如何在应用程序中流动，并捕获基于规则的工具会遗漏的复杂漏洞。

每个发现结果在到达分析师之前，都会经过多阶段验证流程。Claude 会重新检查每个结果，试图证明或反驳自己的发现，并过滤掉误报。发现结果还会被分配严重性评级，以便团队可以优先关注最重要的修复。

经过验证的发现结果会显示在 Claude Code Security 仪表板中，团队可以在那里进行审查、检查建议的补丁并批准修复。由于这些问题通常涉及仅从源代码难以评估的细微差别，Claude 还会为每个发现提供置信度评级。未经人工批准，任何更改都不会应用：Claude Code Security 负责识别问题并提出解决方案，但最终决定权始终在开发者手中。

## 使用 Claude 进行网络安全防护

Claude Code Security 建立在超过一年对 Claude 网络安全能力的研究基础上。我们的 Frontier Red Team 一直在系统性地对这些能力进行压力测试：让 Claude 参加竞争性的夺旗赛，与太平洋西北国家实验室合作试验使用 AI 防御关键基础设施，并完善 Claude 在代码中发现和修补真实漏洞的能力。

因此，Claude 的网络防御能力得到了显著提升。使用本月初发布的 Claude Opus 4.6，我们的团队在生产环境的开源代码库中发现了超过 500 个漏洞——这些漏洞尽管经过多年的专家审查，却已潜伏数十年未被发现。我们目前正与维护者一起进行分类处理和负责任地披露，并计划与开源社区扩大我们的安全工作。

我们也使用 Claude 来审查我们自己的代码，并发现它在保护 Anthropic 系统安全方面极为有效。我们构建 Claude Code Security，就是为了让这些相同的防御能力更广泛地可用。并且由于它建立在 Claude Code 之上，团队可以在他们已经使用的工具中审查发现结果并迭代修复方案。

## 未来之路

这是网络安全的关键时刻。鉴于模型在发现长期隐藏的漏洞和安全问题方面已变得如此有效，我们预计在不久的将来，全球很大一部分代码将由 AI 进行扫描。

攻击者将使用 AI 比以往更快地发现可利用的弱点。但行动迅速的防御者也能发现这些相同的弱点，修补它们，从而降低遭受攻击的风险。Claude Code Security 是我们朝着实现更安全的代码库和全行业更高安全基线目标迈出的一步。
