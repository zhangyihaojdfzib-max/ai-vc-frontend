# 你本可以设计出最先进的位置编码

**高尔定律**  
一个能够正常运行的复杂系统，无一例外都是从某个能够正常运行的简单系统演化而来的。  
——约翰·高尔

本文将引导你逐步发现Transformer模型中最先进的位置编码方法。我们将通过迭代改进位置编码方式，最终得到最新版**Llama 3.2**及大多数现代Transformer中使用的**旋转位置编码（RoPE）**。本文力求限制所需的数学知识，但期望读者具备基础的线性代数、三角学知识以及对自注意力机制的理解。

## 问题陈述

**观其伴，知其意。**  
——约翰·鲁伯特·弗斯

与所有问题一样，我们首先要准确理解想要达成的目标。Transformer中的自注意力机制用于理解序列中Token之间的关系。自注意力是一种**集合**运算，这意味着它具有**置换等变性**。如果我们不向自注意力机制注入位置信息，许多重要的关系将**无法被确定**。

这一点通过示例最能说明。

## 示例说明

考虑以下句子，其中同一个词出现在不同位置：

**那只狗追了另一只狗**

直观上，两个"狗"指代不同的实体。让我们看看如果先将它们分词，映射到**Llama 3.2 1B**的真实Token嵌入，再通过`torch.nn.MultiheadAttention`会发生什么。

```python
import torch
import torch.nn as nn
from transformers import AutoTokenizer, AutoModel

model_id = "meta-llama/Llama-3.2-1B"
tok = AutoTokenizer.from_pretrained(model_id)
model = AutoModel.from_pretrained(model_id)

text = "The dog chased another dog"
tokens = tok(text, return_tensors="pt")["input_ids"]
embeddings = model.embed_tokens(tokens)
hdim = embeddings.shape[-1]

W_q = nn.Linear(hdim, hdim, bias=False)
W_k = nn.Linear(hdim, hdim, bias=False)
W_v = nn.Linear(hdim, hdim, bias=False)
mha = nn.MultiheadAttention(embed_dim=hdim, num_heads=4, batch_first=True)

with torch.no_grad():
    for param in mha.parameters():
        nn.init.normal_(param, std=0.1) 

output, _ = mha(W_q(embeddings), W_k(embeddings), W_v(embeddings))

dog1_out = output[0, 2]
dog2_out = output[0, 5]
print(f"Dog output identical?: {torch.allclose(dog1_out, dog2_out, atol=1e-6)}") 
```

如我们所见，在没有位置信息的情况下，（多头）自注意力操作对**不同位置上的相同Token**的输出是**完全相同的**，尽管这些Token显然代表不同的实体。让我们开始设计一种用位置信息增强自注意力机制的方法，使其能够根据位置编码来确定词之间的关系。

理想的位置编码方案应该具备哪些特性？

## 理想特性

让我们尝试定义一些理想特性，使优化过程尽可能简单。

#### 特性1 - 每个位置具有唯一编码（跨序列）

每个位置都需要一个唯一的编码，且该编码不随序列长度变化——位置5的Token，无论当前序列长度是10还是10,000，其编码都应相同。

#### 特性2 - 两个编码位置之间存在线性关系

位置之间的关系应在数学上保持简单。如果我们知道位置 \( p \) 的编码，那么应该能够直接计算出位置 \( p+k \) 的编码，从而使模型更容易学习位置模式。

试想我们在数轴上表示数字的方式：很容易理解5距离3有2步，或者10距离15有5步。我们的编码中也应存在这种直观的关系。

#### 特性3 - 能够泛化到比训练时更长的序列

为了提高模型在现实世界中的实用性，它们需要能够泛化到训练分布之外。因此，我们的编码方案需要足够灵活，能够处理意料之外的输入长度，同时不违反其他理想特性。

#### 特性4 - 由模型可学习的确定性过程生成

理想情况下，我们的位置编码应来自一个确定性过程。这应使模型能够高效地学习我们编码方案背后的机制。

#### 特性5 - 可扩展到多个维度

随着多模态模型成为常态，我们的位置编码方案能够自然地从一维扩展到 \( n \) 维至关重要。这将使模型能够处理图像或脑部扫描等数据，它们分别是二维和四维的。

现在我们知道了理想特性（以下简称 \( Pr_n \)），让我们开始设计并迭代我们的编码方案。

## 整数位置编码

首先想到的方法可能是简单地将Token位置的整数值（范围从0到 \( L \)，其中 \( L \) 是当前序列的长度）加到Token嵌入的每个分量上。

在上面的动画中，我们根据索引为Token "chased" 创建了位置编码向量，并将其加到Token嵌入上。这里的嵌入值是Llama 3.2 1B真实值的一个子集。我们可以观察到它们集中在0附近。这是可取的，可以避免训练过程中的**梯度消失或爆炸**，因此我们希望在整个模型中保持这一特性。

很明显，我们当前这种朴素的方法会引发问题。位置值的数量级远远超过了输入的实际值。这意味着信噪比非常低，模型很难将语义信息与位置信息分离开来。

基于这一新认识，一个自然的后续思路可能是用 \( \frac{1}{N} \) 对位置值进行归一化。这会将值限制在0和1之间，但会引入另一个问题。如果我们选择 \( N \) 为当前序列的长度，那么对于不同长度的序列，位置值将完全不同，违反了 \( Pr_1 \)。

有没有更好的方法来确保我们的数值在0和1之间？
如果我们对此深思熟虑一番，可能会想到从十进制数切换到二进制数。

## 二进制位置编码

我们不必将（可能归一化的）整数位置加到嵌入的每个分量上，而是可以将其转换为二进制表示，并将我们的值**拉 伸**以匹配嵌入维度，如下所示。

我们将目标位置（252）转换为其二进制表示（11111100），并将每个比特位加到Token嵌入的对应分量上。最低有效位（LSB）将在每个后续Token的0和1之间循环，而最高有效位（MSB）将每 \( 2^{n-1} \) 个Token循环一次，其中 \( n \) 是比特位数。你可以在下面的动画中看到不同索引的位置编码向量[1]。

我们解决了值范围问题，现在拥有了跨不同序列长度保持一致的唯一编码。如果我们绘制Token嵌入的低维版本，并可视化在不同值下添加二进制位置向量的结果，会发生什么呢？

我们可以看到结果非常"跳跃"（正如我们对二进制的离散性质所预期的那样）。优化过程喜欢平滑、连续且可预测的变化。我们是否知道任何具有类似值范围且平滑连续的函数？

如果我们稍加留意，可能会注意到 \( \sin \) 和 \( \cos \) 都符合要求！

## 正弦位置编码

以上动画展示了当我们以逐渐增大的波长交替从正弦和余弦函数中抽取每个分量时的位置嵌入。如果将其与之前的动画进行比较，你会发现惊人的相似性！

我们现在已经得到了正弦位置嵌入；它最初在《Attention is all you need》论文中被定义。让我们来看一下公式：

PE(pos,2i) = sin(pos / 10000^(2i/d))
PE(pos,2i+1) = cos(pos / 10000^(2i/d))

其中 pos 是 Token 的位置索引，i 是位置编码向量中的分量索引，d 是模型维度。10,000 是基础波长（以下称为 θ），我们根据分量索引对其进行拉伸或压缩。我鼓励你代入一些实际数值来感受这种几何级数的变化。

这个公式中有几个部分乍看之下令人困惑。作者为什么选择 10,000？为什么我们要对偶数位置和奇数位置分别使用正弦和余弦函数？

似乎使用 10,000 作为基础波长是通过实验确定的[2]。解读同时使用正弦和余弦函数的原因则更为复杂，但对于我们逐步理解的方法至关重要。这里的关键在于我们希望两个编码位置 Pr₂ 之间存在线性关系。要理解如何通过同时使用正弦和余弦函数来产生这种线性关系，我们需要深入探讨一些三角学知识。

考虑一系列正弦和余弦对，每一对都与一个频率 ωᵢ 相关联。我们的目标是找到一个线性变换矩阵 M，能够将这些正弦函数平移一个固定的偏移量 k：

M · [sin(ωᵢp), cos(ωᵢp)]ᵀ = [sin(ωᵢ(p + k)), cos(ωᵢ(p + k))]ᵀ

频率 ωᵢ 遵循一个随维度索引 i 递减的几何级数，定义为：

ωᵢ = 1 / 10000^(2i/d)

为了找到这个变换矩阵，我们可以将其表示为一个通用的 2×2 矩阵，包含未知系数 u₁、v₁、u₂ 和 v₂：

[u₁ v₁; u₂ v₂] · [sin(ωᵢp), cos(ωᵢp)]ᵀ = [sin(ωᵢ(p + k)), cos(ωᵢ(p + k))]ᵀ

通过对右侧应用三角加法公式，我们可以将其展开为：

[u₁ v₁; u₂ v₂] · [sin(ωᵢp), cos(ωᵢp)]ᵀ = [sin(ωᵢp)cos(ωᵢk) + cos(ωᵢp)sin(ωᵢk), cos(ωᵢp)cos(ωᵢk) - sin(ωᵢp)sin(ωᵢk)]ᵀ

通过匹配系数，这个展开式给出了一个包含两个方程的方程组：

u₁sin(ωᵢp) + v₁cos(ωᵢp) = cos(ωᵢk)sin(ωᵢp) + sin(ωᵢk)cos(ωᵢp)
u₂sin(ωᵢp) + v₂cos(ωᵢp) = -sin(ωᵢk)sin(ωᵢp) + cos(ωᵢk)cos(ωᵢp)

通过比较等式两边 sin(ωᵢp) 和 cos(ωᵢp) 的项，我们可以解出未知系数：

u₁ = cos(ωᵢk), v₁ = sin(ωᵢk)
u₂ = -sin(ωᵢk), v₂ = cos(ωᵢk)

这些解给出了我们最终的变换矩阵 Mₖ：

Mₖ = [cos(ωᵢk) sin(ωᵢk); -sin(ωᵢk) cos(ωᵢk)]

如果你之前做过任何游戏编程，你可能会注意到我们推导出的结果出奇地熟悉。没错，这就是旋转矩阵！[3]

因此，Noam Shazeer 在《Attention is all you need》中设计的编码方案早在 2017 年就已经将相对位置编码为旋转！尽管旋转已经摆在台面上，但从正弦位置编码到 RoPE 仍然花了 4 年时间……

## 绝对位置编码 vs 相对位置编码

既然我们已经知道旋转在这里很重要，让我们回到我们的示例，并尝试为下一步迭代发现一些直觉。

0      1      2      3      4
The dog chased another dog

-2    -1      0      1      2
The dog chased another dog

上面我们可以看到 Token 的绝对位置，以及从 "chased" 到每个其他 Token 的相对位置。使用正弦位置编码，我们生成了一个表示绝对位置的独立向量，并通过一些三角技巧，我们能够编码相对位置。

当我们试图理解这些句子时，"这个" 词是这篇博客文章中的第 2157 个词，这重要吗？还是我们更关心它与其周围词语的关系？一个词的绝对位置很少对意义产生影响——重要的是词与词之间如何相互关联。

## 上下文中的位置编码

从这一点开始，关键是要在自注意力的上下文中考虑位置编码。重申一下，自注意力机制使模型能够权衡输入序列中不同元素的重要性，并动态调整它们对输出的影响。

Attn(Q, K, V) = softmax(QKᵀ / √dₖ)V

在我们之前的所有迭代中，我们生成了一个独立的位置编码向量，并在进行 Q、K 和 V 投影之前将其添加到 Token 嵌入中。通过将位置信息直接添加到 Token 嵌入中，我们实际上在用位置信息污染语义信息。我们应该尝试在不改变范数的情况下编码信息。转向乘法是关键。

使用字典的类比，当在字典（键）中查找一个词（查询）时，附近的词应该比远处的词具有更大的影响。一个 Token 对另一个 Token 的影响由 QKᵀ 点积决定——所以这正是我们应该集中进行位置编码的地方！

a⃗ · b⃗ = |a⃗| |b⃗| cos θ

上述点积的几何解释为我们提供了深刻的洞见。我们可以仅通过增大或减小两个向量之间的夹角来调节它们的点积结果。此外，旋转向量对其范数（编码了Token的语义信息）完全没有影响。

因此，现在我们知道了应该关注的重点，并从另一个角度理解了为什么旋转可能是编码位置信息的合理"通道"。接下来，让我们将所有内容整合起来！

## 旋转位置编码

旋转位置编码（RoPE）由RoFormer论文定义（Jianlin Su在其博客中独立设计了该方案）。虽然直接看最终结果可能觉得有些玄妙，但通过结合自注意力（更具体地说是点积）来思考正弦编码，我们就能理解这一切是如何运作的。

与正弦编码类似，我们将向量 **q** 或 **k**（而非投影前的 **x**）分解为二维对/块。我们不再通过添加从频率递减的正弦函数中提取的向量来直接编码绝对位置，而是直接通过将每对向量与旋转矩阵相乘来编码相对位置。

设 **q** 或 **k** 为位置 *p* 处的输入向量。我们创建一个块对角矩阵，其中 **Mᵢ** 是该分量对所需旋转对应的旋转矩阵：

R(**q**, *p*) = ( **M₁** **M₂** ⋱ **M_(d/2)** ) ( q₁ q₂ ⋮ q_d )

与正弦编码非常相似，**Mᵢ** 简单地定义为：

**Mᵢ** = [ cos(ωᵢ *p*)  sin(ωᵢ *p*) ; -sin(ωᵢ *p*)  cos(ωᵢ *p*) ]

在实践中，我们不会使用矩阵乘法来计算RoPE，因为对于这种稀疏矩阵来说计算效率太低。相反，我们可以利用计算中的规律模式，直接对元素对独立应用旋转：

R_{Θ,p}^d q = ( q₁ q₂ q₃ q₄ ⋮ q_{d-1} q_d ) ⊗ ( cos pθ₁ cos pθ₁ cos pθ₂ cos pθ₂ ⋮ cos pθ_{d/2} cos pθ_{d/2} ) + ( -q₂ q₁ -q₄ q₃ ⋮ -q_d q_{d-1} ) ⊗ ( sin pθ₁ sin pθ₁ sin pθ₂ sin pθ₂ ⋮ sin pθ_{d/2} sin pθ_{d/2} )

仅此而已！通过在计算点积之前巧妙地将旋转应用于 **q** 和 **k** 的二维块，并将加法改为乘法，我们可以在评估中获得显著的性能提升[4][^4][4]。

## 将RoPE扩展到n维

我们已经探讨了RoPE在1D情况下的应用，至此我希望你已经对这个Transformer中确实不太直观的组件有了直观的理解。最后，让我们探讨如何将其扩展到更高维度，例如图像。

一个自然的初步直觉可能是直接使用图像中的 [x; y] 坐标对。这看起来可能很直观，毕竟我们之前几乎是任意地将分量配对。然而，这将是一个错误！

在1D情况下，我们通过对输入向量中的值对进行旋转来编码相对位置 *m - n*。对于2D数据，我们需要独立地编码水平和垂直的相对位置，例如 *m - n* 和 *i - j*。RoPE的巧妙之处在于它如何处理多个维度。它不是试图在单个旋转中编码所有位置信息，而是将同一维度内的分量配对并进行旋转，否则我们会混淆 *x* 和 *y* 的偏移信息。通过独立处理每个维度，我们保持了空间的自然结构。这可以推广到任意数量的维度！

RoPE是位置编码的最终形态吗？DeepMind最近的这篇论文深入分析了RoPE，并指出了一些根本性问题。简而言之：RoPE并非完美解决方案，模型主要关注低频部分，但该论文表明，移除（而非旋转）最低频率可以提升Gemma 2B的性能！
