2018年6月25日

# OpenAI Five

我们的五神经网络团队OpenAI Five已开始在Dota 2中击败业余人类战队。

![一群人坐在大屏幕前，屏幕上显示Dota 2界面](/images/posts/1e2f73cf700f.jpg)

我们的五神经网络团队OpenAI Five已开始在Dota 2（在新窗口打开）中击败业余人类战队。虽然今天我们带有限制条件（在新窗口打开）进行比赛，但我们的目标是在8月的国际邀请赛（在新窗口打开）中击败一支顶级职业战队，仅受限于有限的英雄池。我们可能不会成功：Dota 2是世界上最受欢迎且最复杂（在新窗口打开）的电子竞技游戏之一，拥有富有创造力和积极进取的职业选手，他们全年训练（在新窗口打开）以争夺Dota每年4000万美元奖金池（在新窗口打开）的一部分（这是所有电子竞技游戏中最大的奖金池）。

OpenAI Five每天通过自我对弈进行相当于180年时长的比赛，并通过自博弈学习。它使用近端策略优化的扩展版本进行训练，运行在256块GPU和128,000个CPU核心上——这是我们去年为玩该游戏更简单的单人变体（在新窗口打开）所构建系统的更大规模版本。通过为每个英雄使用独立的LSTM（在新窗口打开）且不使用人类数据，它学会了可识别的策略。这表明强化学习（在新窗口打开）可以在足够大且可实现规模下产生长期规划——无需根本性突破，这与我们项目启动时的预期相反。

为了衡量我们的进展，我们将在8月5日举办一场对阵顶级选手的比赛。在Twitch上关注（在新窗口打开）我们以观看直播，或申请（在新窗口打开）现场观赛邀请！

## 问题

人工智能的一个里程碑是在像星际争霸（在新窗口打开）或Dota这样的复杂视频游戏中超越人类能力。相比国际象棋（在新窗口打开）或围棋（在新窗口打开）等先前的人工智能里程碑，复杂视频游戏开始捕捉现实世界的混乱性和连续性。我们希望解决复杂视频游戏的系统将具有高度通用性，并应用于游戏之外。

Dota 2是一款实时策略游戏，由两支各五名玩家的队伍进行比赛，每名玩家控制一个名为“英雄”的角色。玩Dota的人工智能必须掌握以下内容：

- **长时间跨度。** Dota游戏以每秒30帧的速度运行，平均持续45分钟，每局游戏产生80,000个时间步。大多数动作（如命令英雄移动到（在新窗口打开）某个位置）单独影响较小，但某些个别动作如使用回城卷轴（在新窗口打开）可能从战略上影响游戏；某些策略（在新窗口打开）可能贯穿整局游戏。OpenAI Five每四帧观察一次，产生20,000步。国际象棋（在新窗口打开）通常在40步内结束，围棋（在新窗口打开）在150步内结束，且几乎每一步都具有战略意义。
- **部分可观测状态。** 单位和建筑只能看到周围区域。地图其余部分被迷雾覆盖，隐藏敌人及其策略。高水平游戏需要基于不完整数据进行推断，并模拟对手可能的意图。国际象棋和围棋都是完全信息游戏。
- **高维、连续动作空间。** 在Dota中，每个英雄可以执行数十种动作，许多动作以另一个单位或地面位置为目标。我们将空间离散化为每个英雄170,000种可能的动作（并非每个时间步都有效，例如在冷却（在新窗口打开）中使用技能）；不计连续部分，每个时间步平均约有1,000个有效动作。国际象棋的平均动作数（在新窗口打开）为35；围棋为250。
- **高维、连续观察空间。** Dota在一个大型连续地图（在新窗口打开）上进行，包含十个英雄、数十个建筑、数十个NPC（在新窗口打开）单位，以及符文、树木、守卫等大量游戏特性。我们的模型通过Valve的Bot API（在新窗口打开）将Dota游戏状态观察为20,000个（主要为浮点数）数字，代表人类允许访问的所有信息。国际象棋棋盘自然表示为约70个枚举值（8x8棋盘，6种棋子类型及少量历史（在新窗口打开）信息（在新窗口打开））；围棋棋盘表示为约400个枚举值（19x19棋盘，2种棋子类型加劫（在新窗口打开））。

Dota的规则也非常复杂——该游戏已持续开发十多年，游戏逻辑由数十万行代码实现。这些逻辑每个时间步需要毫秒级执行，而国际象棋或围棋引擎只需纳秒级。游戏大约每两周更新一次，不断改变环境语义。

## 我们的方法

我们的系统使用大规模版本的近端策略优化进行学习。OpenAI Five和我们早期的1v1机器人（在新窗口打开）完全通过自博弈学习。它们从随机参数开始，不使用搜索（在新窗口打开）或从人类回放中引导。

强化学习研究人员（包括我们自己）普遍认为（在新窗口打开），长时间跨度需要根本性的新进展，例如分层（在新窗口打开）强化学习（在新窗口打开）。我们的结果表明，我们一直低估了当今算法的能力——至少在足够规模运行并以合理方式探索（在新窗口打开）时是这样。

我们的智能体被训练为最大化未来奖励的指数衰减和，权重由称为γ的指数衰减因子决定。在OpenAI Five的最新训练运行中，我们将γ从0.998（未来奖励半衰期为46秒）退火至0.9997（未来奖励半衰期为5分钟）。作为对比，PPO（在新窗口打开）论文中最长跨度半衰期为0.5秒，Rainbow（在新窗口打开）论文中最长跨度半衰期为4.4秒，Observe and Look Further（在新窗口打开）论文中使用的半衰期为46秒。

虽然当前版本的OpenAI Five在补刀（在新窗口打开）方面较弱（观察我们的测试比赛，职业Dota解说Blitz（在新窗口打开）估计其水平约为Dota玩家中位数），但其目标优先级（在新窗口打开）与常见职业策略相符。获取长期奖励（如战略地图控制）通常需要牺牲短期奖励（如通过打钱（在新窗口打开）获得的金币），因为集结攻击防御塔需要时间。这一观察结果强化了我们的信念：该系统确实在优化长期跨度。

## 模型结构

OpenAI Five的每个网络（在新窗口打开）包含一个单层、1024单元的LSTM（在新窗口打开），它观察当前游戏状态（从Valve的Bot API（在新窗口打开）提取）并通过多个可能的动作头发出动作。每个动作头具有语义含义，例如延迟此动作的时间步数、选择哪个动作、该动作在单位周围网格中的X或Y坐标等。动作头独立计算。

OpenAI Five使用的观察空间和动作空间的交互演示。OpenAI Five将世界视为20,000个数字的列表，并通过发出8个枚举值的列表来执行动作。选择不同的动作和目标以了解OpenAI Five如何编码每个动作，以及它如何观察世界。图像显示人类视角下的场景。

OpenAI Five 能够对与其观察到的状态相关的缺失信息做出反应。例如，直到最近，OpenAI Five 的观测数据中并不包含弹片区域（即敌方单位受到投射物攻击的区域），而人类玩家可以在屏幕上看到这些区域。然而，我们观察到 OpenAI Five 学会了走出活跃的弹片区域（尽管并非主动避免进入），因为它能看到自己的生命值在下降。

## 探索

在拥有能够处理长时域任务的学习算法的情况下，我们仍然需要对环境进行探索。即使有我们的限制条件，游戏中仍有数百种物品、数十种建筑、法术和单位类型，以及大量需要学习的游戏机制——其中许多能产生强大的组合效果。高效探索这个组合爆炸式的空间并非易事。

OpenAI Five 通过自我对弈（从随机权重开始）进行学习，这为探索环境提供了自然的课程。为了避免“策略崩溃”，该 Agent 在 80% 的游戏中与自己对抗，其余 20% 则与过去的自己对抗。在最初的游戏中，英雄们在地图上漫无目的地行走。经过数小时的训练，诸如分路、补刀或中路争夺等概念开始出现。几天后，它们开始持续采用基本的人类策略：尝试从对手那里偷取赏金神符，走到己方一塔附近补刀，并在地图上轮转英雄以获得分路优势。随着进一步训练，它们逐渐精通了诸如五人推进等高级策略。

2017 年 3 月，我们的第一个 Agent 击败了机器人，但在面对人类时却感到困惑。为了强制在策略空间中进行探索，在训练期间（且仅在训练期间），我们对单位的属性（生命值、速度、初始等级等）进行了随机化处理，随后它开始击败人类。后来，当一名测试玩家持续击败我们的 1v1 机器人时，我们增加了训练的随机化程度，该测试玩家便开始输掉比赛。（我们的机器人团队同时将类似的随机化技术应用于实体机器人，以实现从模拟环境到现实世界的迁移。）

OpenAI Five 使用了我们为 1v1 机器人编写的随机化方案。它还使用了一种新的“分路分配”方案。在每局训练游戏开始时，我们会随机将每个英雄“分配”到某几条分路，并对其偏离这些分路的行为进行惩罚，直到游戏中一个随机选定的时间点。

良好的奖励机制也有助于探索。我们的奖励主要由人类用来判断自己在游戏中表现如何的指标组成：净资产、击杀数、死亡数、助攻数、补刀数等。我们通过减去对方队伍的平均奖励来对每个 Agent 的奖励进行后处理，以防止 Agent 发现正和博弈的情况。

我们硬编码了物品和技能出装方案（最初是为我们的脚本基线编写的），并随机选择使用哪种方案。信使管理也从脚本基线中引入。

## 协作

OpenAI Five 的英雄神经网络之间没有显式的通信渠道。团队协作由一个我们称之为“团队精神”的超参数控制。“团队精神”的取值范围为 0 到 1，它决定了 OpenAI Five 的每个英雄在多大程度上应关注其个人奖励函数，而非团队奖励函数的平均值。我们在训练过程中将其值从 0 退火到 1。

## Rapid

我们的系统实现为一个名为 Rapid 的通用强化学习训练系统，可应用于任何 Gym 环境。我们已在 OpenAI 使用 Rapid 解决了其他问题，包括竞争性自我对弈。

![Rapid 架构](/images/posts/2adcd8cc9544.png)

该训练系统分为 Rollout Worker 和 Optimizer 节点。Rollout Worker 运行游戏副本和一个收集经验的 Agent；Optimizer 节点则在一组 GPU 上执行同步梯度下降。Rollout Worker 通过 Redis 将它们的经验同步给 Optimizer。每个实验还包含用于评估训练后 Agent 与参考 Agent 对比的 Worker，以及诸如 TensorBoard、Sentry 和 Grafana 等监控软件。

在同步梯度下降过程中，每个 GPU 在其批次数据上计算梯度，然后对梯度进行全局平均。我们最初使用 MPI 的 allreduce 进行平均，但现在使用我们自己的 NCCL2 封装器，该封装器可并行化 GPU 计算和网络数据传输。在不同数量的 GPU 上同步 58MB 数据（OpenAI Five 参数的大小）的延迟如右图所示。该延迟足够低，以至于可以被与其并行运行的 GPU 计算所掩盖。

我们为 Rapid 实现了 Kubernetes、Azure 和 GCP 后端。

## 比赛

迄今为止，OpenAI Five 已（在我们的限制条件下）与以下每支队伍进行了比赛：

1.  最佳 OpenAI 员工队：2.5k MMR（第 46 百分位）
2.  观看 OpenAI 员工比赛的最佳观众玩家（包括解说了第一场 OpenAI 员工比赛的 Blitz）：4–6k MMR（第 90-99 百分位），尽管他们从未作为团队一起比赛过。
3.  Valve 员工队：2.5–4k MMR（第 46-90 百分位）
4.  业余队伍：4.2k MMR（第 93 百分位），作为团队进行训练。
5.  半职业队伍：5.5k MMR（第 99 百分位），作为团队进行训练。

4 月 23 日版本的 OpenAI Five 首次击败了我们的脚本基线。5 月 15 日版本的 OpenAI Five 与队伍 1 实力相当，赢了一局，输了一局。6 月 6 日版本的 OpenAI Five 在与队伍 1-3 的所有比赛中都取得了决定性胜利。我们与队伍 4 和 5 进行了非正式的训练赛，预计会惨败，但 OpenAI Five 在与这两支队伍的前三场比赛中赢下了两场。

![两名戴着耳机的竞技玩家正在互相交谈，而现场观众在他们面前观看](/images/posts/c35bdbdb64e6.jpg)

我们观察到 OpenAI Five：

-   反复牺牲自己的安全路（夜魇的上路；天辉的下路），以换取对敌方安全路的控制，迫使战斗发生在对手更难防守的一侧。这种策略在过去几年中才出现在职业赛场上，现在被认为是主流战术。Blitz 评论说，他是在玩了八年之后，当 Team Liquid 告诉他时，才学到这一点的。
-   比对手更快地从游戏前期过渡到中期。它通过以下方式做到这一点：(1) 当玩家在分路上过度深入时，成功设置游走（玩家在地图上移动以伏击敌方英雄——见动画），以及 (2) 在对手组织起反击之前，集结队伍推掉防御塔。
-   在几个方面偏离了当前的打法风格，例如让辅助英雄（通常不会优先获得资源）获得大量早期经验和金钱。OpenAI Five 的这种优先级分配使其伤害峰值更早到来，并能更有效地扩大优势，赢得团战并利用对手的失误确保快速获胜。

OpenAI 最佳玩家与我们的机器人队伍比赛后颁发的奖杯。一个奖杯给人类，一个奖杯给机器人（由我们团队的 Susan Zhang 代表领取！）

## 与人类的差异

OpenAI Five 能够获取与人类相同的信息，但它能立即看到人类需要手动检查的数据，如位置、生命值和物品栏。我们的方法从根本上并不依赖于观察状态，但仅从游戏中渲染像素则需要数千个 GPU。

OpenAI Five 平均每分钟执行约150-170次操作（由于每4帧观察一次，理论最大值为450次）。帧完美时机虽然对熟练玩家来说可能实现，但对OpenAI Five而言轻而易举。OpenAI Five的平均反应时间为80毫秒，比人类更快。

这些差异在1v1中最为显著（我们的机器人反应时间为67毫秒），但随着我们看到人类从机器人身上学习并适应，竞技环境相对公平。去年TI之后数月内，数十位职业选手使用我们的1v1机器人进行训练。据Blitz称，1v1机器人改变了人们对1v1的思考方式（机器人采用了快节奏打法，现在所有人都已适应这种节奏）。

## 意外发现

- **二元奖励也能带来良好表现。** 我们的1v1模型使用了塑形奖励，包括补刀、击杀等奖励。我们进行了一项实验，仅根据输赢给予奖励，结果训练速度慢了一个数量级，并在中途出现一定程度的平台期，与我们通常看到的平滑学习曲线形成对比。该实验在4500个核心和16块K80 GPU上运行，训练水平达到半职业选手水平（70 TrueSkill），而非我们最佳1v1机器人的90 TrueSkill。
- **卡兵可以从零开始学习。** 在1v1中，我们使用传统强化学习配合"卡兵"奖励学会了卡兵。我们一位团队成员在度假期间（向现在的妻子求婚！）让一个2v2模型继续训练，想看看延长训练时间能提升多少性能。令他惊讶的是，模型在没有特殊指导或奖励的情况下学会了卡兵。
- **我们仍在修复漏洞。** 图表展示了击败业余玩家的代码训练过程，与修复了若干漏洞的版本对比——例如训练中罕见的崩溃，或导致达到25级时获得巨大负奖励的漏洞。事实证明，在隐藏严重漏洞的情况下，仍有可能击败优秀人类玩家！
