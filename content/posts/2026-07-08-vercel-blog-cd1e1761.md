现在您可以直接通过 CLI 更新项目的框架预设和构建设置，无需打开仪表盘。

在已关联的项目中运行 `vercel project update`，或直接传入项目名称：

```
1▲ vercel project update my-project --framework nextjs --build-command "pnpm build"2✓ 已更新         项目设置3  项目           my-project4  框架           Vite (vite) → Next.js (nextjs)5  构建命令       npm run build → pnpm build
```

设置也可以恢复为自动检测，而非显式覆盖：

```
1vercel project update my-project --auto-detect build-command,output-directory
```

如需自动化或进一步检查，可将结果以 JSON 格式返回：

```
1vercel project update my-project --framework vite --format=json
```

JSON 输出包含是否有任何更改、已更改的设置列表、项目 ID 和名称，以及请求的设置。输出内容写入标准输出，不混入其他输出，因此可安全地进行管道传输和解析。

Agent（智能体）可利用此功能端到端修复配置错误的项目：在诊断构建失败后，Agent（智能体）可修正框架预设、构建命令或输出目录并重新部署，无需访问仪表盘。无效的框架标识符和设置会在 API 调用前被拒绝并给出建议。

请使用 `npm i -g vercel@latest` 将 Vercel CLI 更新至 v54.21.1 或更高版本以开始使用。更多信息请参阅 Vercel 项目文档。
