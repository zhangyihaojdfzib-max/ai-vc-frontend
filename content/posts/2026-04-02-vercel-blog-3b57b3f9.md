现在，Go API 后端可以在 Vercel 上实现零配置部署。

```
1package main2
3import (4	"fmt"5	"net/http"6	"os"7)8
9func main() {10	port := os.Getenv("PORT")11	if port == "" {12		port = "3000"13	}14
15	http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {16		fmt.Fprintf(w, "Hello World")17	})18
19	addr := ":" + port20	fmt.Printf("Listening on %s\n", addr)21	http.ListenAndServe(addr, nil)22}
```

Vercel 现已将 Go 服务器识别为一流后端，并自动调配正确的资源、配置您的应用程序，无需在 `vercel.json` 文件中进行重定向或遵循 `/api` 文件夹的惯例。

Vercel 上的后端默认使用 **Fluid compute** 和 **Active CPU pricing**。您的 Go API 会随流量自动扩展，并且您只需为活跃的 CPU 时间付费，无需为闲置容量付费。
