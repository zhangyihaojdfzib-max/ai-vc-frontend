HTTP 请求成本很低。Vercel 收费约每百万次2美元，每次调用不到1美分。但向前沿模型上的Agent（智能体）发送一条提示词就可能花费2美元，这使得AI的成本高出数百万倍，而推理窃取成为攻击者能运营的利润率最高的业务之一。我们已在自家API上目睹过此类攻击。

如果你的AI端点暴露在互联网上，被滥用的风险很高，且很容易导致数万美元甚至更高的账单。

保护这些端点需要对每一次AI请求进行验证，而不是基于会话或注册。仅靠速率限制和身份验证墙是不够的，因为每次会话只运行一次的检查，其成本会被数千次被盗调用分摊。

在Vercel，我们通过BotID深度分析来拦截每一次AI请求，你也可以用几行代码在自己的端点上实现同样的防护。

## 链接到标题什么是推理窃取

推理窃取是指未经授权使用他人付费的AI推理，无论是用于免费消费还是下游转售。运营方为每次AI调用付费；攻击者不支付推理费用，然后以折扣价转售Token。这已超出速率限制滥用的范畴，实际上是在市场上转售被盗资源。

### 链接到标题哪些AI端点面临风险？

任何面向互联网、能让调用者对LLM提示词进行有意义的控制的端点都是目标。端点越通用，每次被盗调用的收益就越高。

AI游乐场（如AI SDK Playground）是最危险的形式，因为调用者对提示词、模型以及通常的参数拥有最大控制权。被盗调用能干净地接入任何标准客户端。

当系统提示词在服务端固定时，支持机器人和文档助手的暴露风险较低，但攻击者已学会如何以足够低的成本绕过系统提示词来操控模型，从而使转售变得可行。

转售价值取决于被盗调用能多容易地接入与提供商兼容的客户端。

被盗推理的回报足够高，以至于攻击者会采购数千个住宅代理IP，并以击败你防御所需的任何规模注册一次性账户。速率限制被IP地址池稀释，而真实账户能通过身份验证。

适配器还充当攻击者下游用户的会话边界。他们向适配器认证，而不是你的端点。当调用到达你的API时，它已经越过了你计划防御的边界。检查必须在适配器代理的调用上运行，而不是在其背后的会话上。

## 链接到标题如何防御推理窃取

保护AI端点免受推理窃取需要对每次请求进行验证。我们在Vercel使用带有深度分析的BotID，在AI请求到达之前，在路由处理器内部调用它。

### 链接到标题验证必须在每次AI请求上运行

如果我们的防御是在会话开始时而不是每次请求时运行，攻击者只需支付一次绕过成本，就能获得数十万次被盗调用。任何每次会话运行的检查都会将攻击者的绕过成本分摊到后续的每一次推理调用中。每次请求的防御迫使该比例降至1:1，即使推理价格很高，在每次调用上击败检查也不值得付出成本。

这就是成本不对称对防御方有利的地方。推理是攻击者每次调用窃取的最昂贵资源，而验证是每次调用最便宜的保护成本之一。

### 链接到标题使用BotID深度分析实现请求验证

传统的图像验证码已无法抵御现代攻击者，因为使推理值得窃取的同一AI模型可以轻松绕过它们。

我们在AI端点上部署Vercel BotID，拦截每一次请求。BotID是一种由Kasada提供支持的带有深度分析的隐形验证码，它使用客户端机器学习来区分人类和机器人，而无需可见的挑战，因此可以在每次请求上运行，而不仅仅在会话开始时。

BotID深度分析在流量激增的头几分钟内检测并阻止了超过一万次机器人请求。在24小时内，端点上的请求量恢复到正常水平。

在服务端，`checkBotId()` 在路由处理器内部运行，并为当前正在处理的请求返回一个分类结果。

```
12import { checkBotId } from 'botid/server';3import { NextRequest, NextResponse } from 'next/server';4
5export async function POST(request: NextRequest) {6  const verification = await checkBotId();7  if (verification.isBot) {8    return NextResponse.json({ error: 'Access denied' }, { status: 403 });9  }10  11}
```

该路由也必须在客户端声明。否则，`checkBotId()` 会失败，因为BotID不会将挑战标头附加到请求上：

```
12import { initBotId } from 'botid/client/core';3
4initBotId({5  protect: [{ path: '/api/ai-chat', method: 'POST' }],6});
```

请参阅BotID文档了解 `next.config.ts` 包装器和完整设置。
