# Databricks Genie 为 Atlassian Rovo 提供对话式洞察力

## Atlassian 正利用 Databricks AI/BI Genie 对话 API 驱动其全新的 Databricks Query Runner Agent（智能体），让每位员工无需 SQL 即可做出明智的数据驱动决策。

发布日期：2026年1月14日

-   Atlassian 全新的 Databricks Query Runner Agent（智能体）由 Databricks AI/BI Genie 对话 API 驱动，使用户能够通过自然语言在 Atlassian 的 Rovo 平台上获取按需洞察。
-   该集成赋能从营销到产品等非技术团队，使其能够直接在 Atlassian 工作流程中做出数据驱动决策。
-   双方合作将通过新的 Rovo Search Connector for Databricks 和 Atlassian Rovo MCP 连接器进一步扩展，使企业数据更易访问。

## 驱动团队协作的未来

对数百万用户而言，Atlassian 是工作发生的地方。Jira 和 Confluence 等工具是现代协作的支柱，用于跟踪复杂项目并存储关键的组织知识。在 Databricks，我们同样相信数据和 AI 是未来协作的基础。

挑战始终在于弥合数据与日常工作之间的鸿沟。Atlassian 和 Databricks 携手合作以消除这一鸿沟，为使用 Rovo 的任何人带来 AI 驱动的对话式分析。用户无需费力编写 SQL 或等待专家，只需用自然语言提问，即可从其 Databricks 环境中获得真实、可追溯的洞察，从而实现更快速、更智能、更民主的决策。

## 通过 Databricks Query Runner + Genie 简化分析

为实现这一愿景，Atlassian 构建了 **Databricks Query Runner** Agent（智能体），由 **Databricks AI/BI Genie API** 驱动，从而在 **Atlassian Rovo** 内实现实时、自助式分析。

**AI/BI Genie** 是内置的“引擎”，使团队中的任何人（从产品经理到营销人员和支持负责人）都能通过简单的英语提问来查询结构化的 Databricks 数据。这将数据访问从专业专家转移至整个组织，消除了信息孤岛和延迟。

此集成提供了强大的新功能：

-   **保持工作流顺畅**：在 Rovo 内交付洞察，使用户保持专注，无需切换标签页或工具。他们可以在编写 Confluence 页面或更新 Jira 工单时，直接在 Rovo 中发起聊天。用户现在可以通过 Rovo 上的 Databricks Query Runner Agent（智能体）直接访问数据洞察。
-   **赋能每个人**：使数据驱动决策不仅对分析师，而且对整个企业都变得可及。
-   **企业级安全性**：构建于 Atlassian 可信的云平台之上；数据保持安全、合规且受管控。

Rovo 与 Atlassian 由 Genie 驱动的 Agent（智能体）相结合，为整个业务释放价值：

-   产品经理可以提问：“过去90天内，哪些客户群体对功能X的参与度最高？”以指导路线图优先级排序。
-   营销负责人可以提问：“我们最近一次营销活动的投资回报率和转化率提升情况如何？”并让数据直接填充到他们的Confluence回顾文档中。
-   支持负责人可以提问：“本季度工单数量或解决时间是否与更高的客户流失率相关？”以实时分析流失风险。
-   财务经理可以提问：“在扩展功能Y之后，云基础设施成本趋势如何？”以获取用于财务规划的成本明细。

## 携手共建协作分析的未来

Atlassian的Databricks Query Runner的发布仅仅是一个开始。基于与Atlassian Analytics建立的数据共享合作伙伴关系以及Delta Sharing协议，这些新的AI功能正在重塑协作工作的未来。

我们也很高兴能与Atlassian合作，在今年晚些时候推出面向Databricks的新Rovo搜索连接器，这将使用户能够直接在Rovo内部搜索和查找Databricks表和仪表板。与此同时，Databricks正致力于将Atlassian Rovo MCP服务器引入Databricks Marketplace，进一步拓宽企业客户对强大分析功能的访问。

这种合作伙伴关系正在通过消除技术壁垒，使组织中的每个人都能访问安全的实时数据，从而重塑协作工作的未来。
