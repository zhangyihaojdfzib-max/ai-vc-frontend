# 停止被动应对安全漏洞，通过用户风险评分主动预防

2026-03-04

- Nevins Bartolomeo
- Noelle Kagan
- Ann Ming Samborski

![](/images/posts/ea8d6d78a307.png)

大多数安全团队每天都在进行一场高风险的游戏。用户的凭证被钓鱼窃取，或者他们不小心下载了恶意文件，突然间你就进入了事件响应模式。

我们构建了我们的 SASE 平台——Cloudflare One，以打破这种循环。通过在您的应用程序和互联网流量前部署 Access 和 Gateway，我们为您提供了决定谁可以进入以及他们可以去哪里的工具。

今天，我们让这些决策变得更智能。您现在可以将**用户风险评分**直接纳入您的**零信任网络访问（ZTNA）** 策略中。您不再仅仅检查"这个用户是谁？"和"他们的设备是否健康？"，现在还可以问"这个用户最近的行为如何？"并实时调整他们的访问权限。

### 第一步：从"是什么"到"怎么样"

多年来，传统的企业访问权限是二元的。您要么拥有正确的登录凭证和正确的证书，要么没有。但身份是动态的。一个合法用户如果其账户被盗用，或者开始表现出"内部威胁"行为——例如不可能的地理位置移动、多次登录失败尝试，或通过移动敏感数据触发数据丢失防护规则——就可能变成风险。

Cloudflare One 现在会根据这些行为持续为您组织中的每个用户计算风险评分。

用户及其风险评分示例列表

一旦您的团队开始使用 Cloudflare One，您就可以在仪表板中导航到 **团队与资源 > 用户 > 风险评分** 部分。在这里，您可以定义哪些行为对您很重要。例如，您可能决定不可能的地理位置移动具有"高"风险级别，而使用需要更新的设备则为"中"风险。

Cloudflare 的风险引擎持续评估来自整个 SASE 平台的遥测数据。对于内部信号，引擎会监控来自 **Cloudflare Access**（例如，成功/失败的登录、地理位置上下文）和 **Cloudflare Gateway**（例如，恶意软件命中、风险浏览类别或 DLP 中的敏感数据触发）的日志。

对于第三方信号，我们已经与 CrowdStrike 和 **SentinelOne** 等合作伙伴建立了服务到服务的集成。这些集成允许 Cloudflare 接收外部遥测数据，例如 **CrowdStrike 的设备安全状况属性**，并将其映射到用户的配置文件中。

计算逻辑被设计为确定性的：

1.  **选择**：管理员选择为其组织启用哪些特定的"风险行为"（不可能的地理位置移动、违反 DLP 规则等）。
2.  **聚合**：引擎识别与用户相关的所有风险事件。
3.  **评分**：用户的风险评分由该期间内触发的任何**已启用**行为的最高风险级别（低、中或高）决定。
4.  **重置**：如果管理员调查并清除了某个事件，他们可以手动重置用户的评分，这会保留历史记录，但根据未来收集的风险数据重置其访问权限。

### 第二步：轻松应用自适应访问

知道用户有风险是第一步。自动对此采取措施是第二步。

过去，如果安全分析师发现可疑用户，他们必须手动撤销会话或将用户移入其**身份提供商（IdP）** 中的"受限"组。这需要时间——而攻击者正利用这段时间进行横向移动。

现在，您可以构建**自适应访问**策略。当您创建或编辑 Access 策略时，会发现一个新的选择器：**用户风险评分**。

Access 策略中新的用户风险评分选择器示例。

这使您可以创建全局或特定于应用程序的规则，例如："如果用户的风险评分为高，则他们无法访问财务门户"，或者"如果用户的风险评分为中，则他们必须使用物理安全密钥登录。" 此类规则确保在应用额外安全层的同时，企业运营不会中断。

### 第三步：形成闭环

该系统最棒的部分在于它是动态的。如果用户的评分在调查员审查并清除后下降，他们的访问权限将根据您的策略自动恢复。目前，基于风险的访问可以在活动会话中因风险评分升高而撤销访问权限。未来，我们将探索将其扩展到在活动会话中因风险评分变化而强制执行升级的多因素认证（MFA）。

我们还确保此功能与您已使用的工具协同工作。如果您使用 Okta，Cloudflare 可以**将这些风险信号共享回 Okta**，确保在网络中被标记的用户在您的**单点登录（SSO）** 入口处也受到限制。此集成使用**共享信号框架**，该框架支持跨平台共享风险信号。

### 更快行动，保持安全

我们构建 Cloudflare One 是为了让安全团队不再充当"拒绝部门"，而开始成为"可以，且安全"的部门。将用户风险评分纳入您的 Access 策略是这一旅程的下一步。它将您的安全从登录时的静态快照，转变为与您的网络架构持续进行的动态对话。

如果您已经是 Cloudflare 客户，今天就可以在仪表板中开始探索这些风险信号。如果您仍在与遗留 VPN 或手动安全审查作斗争，我们很乐意帮助您实现转变。
