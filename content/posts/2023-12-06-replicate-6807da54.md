过去我曾将AI模型描述为“魔法盒子”，它们接收输入，进行转换，然后给我们输出。我们可以在不必深入理解其工作原理的情况下使用这些魔法盒子。

我们的代码将需要三个魔法盒子：

- 一个能够通过电脑摄像头“看见”并描述我们所见的视觉模型。
- 一个撰写脚本（在我的例子中，是大卫·爱登堡风格）的语言模型。
- 一个将文字作为输入并输出语音音频文件的文本转语音模型。

## 一个能“看见”的模型

![Pasted image 20231201120324.png](/images/posts/ebeb28b756a9.webp)

我们的第一步是找到一个能“看见”的视觉模型。我们熟悉的许多模型，如ChatGPT或Stable Diffusion，都以文本作为输入。我们向模型发送文本，然后得到文本、图像或视频作为返回。但对于我们的人生旁白，我们需要一个能以图像作为输入，并能回答关于这些图像问题的模型。

我们的视觉模型有两个输入：一张图像和一个文本提示词。然后模型返回一个文本响应。

我们有几个选择，但请注意，我是有偏好的。

### Llava 13B

Llava 13B是一个开源的视觉模型。它价格便宜，并且对于我们的目的来说足够快。这是我推荐的选择。Llava在A40 GPU上运行，每秒钟运行成本为0.000725美元。

### GPT-4-Vision

这是我在演示视频中使用的模型。它比Llava更聪明，但运行速度稍慢，成本也更高。

GPT-4-Vision有点复杂。它按图像分辨率加上每Token收费。一张250px乘140px的图像成本为0.00255美元，每1K输出Token收费0.03美元。你还需要通过他们的候补名单才能尝试。

## 一个撰写脚本的模型

![a model that writes a script](/images/posts/7e9a7ff86291.webp)

接下来，我们需要一个模型来撰写要被旁白的脚本。这个魔法盒的输出就是大卫·爱登堡在我视频中说的话。

这是一个如何使用Mistral 7B来做到这一点的例子。我们的提示词是：

```
这是一张图像的描述。你能假装你是{narrator}，并像在自然纪录片中一样来旁白它吗？请简短一些。
如果我做了任何稍微有趣的事情，请大做文章！

描述：
图像中的人正将一个红色杯子举到嘴边，
张大嘴巴，假装要咬或喝杯子里的东西。
```

这里的描述是前一步中我们视觉模型的输出。我们的响应看起来像这样：

```
在这个引人入胜的观察中，我们看到了人类行为的一个非凡展示。
图像中的人手持一个红色杯子，准备咬或喝。
当他们张大嘴巴时，我们可以看到他们牙齿和下颌的复杂运作，
这是人体惊人适应能力的证明。
这是一个值得一看的景象，提醒着我们构成我们自身的复杂性。
```

预测ID

我建议将返回的最大Token数限制在128左右。我们希望响应时间尽可能快。上面的输出花了2.7秒，所以目前我们的总响应时间大约是5-6秒。

注意，我们也可以使用GPT-4-Vision来返回我们的响应。实际上，我们甚至可以跳过一个步骤——我们不需要先请求对图像的描述，然后再将其翻译成大卫·爱登堡风格的脚本。我们可以在一个预测中完成这两件事。GPT-4-Vision足够聪明，可以同时完成。

这是我使用的系统提示词：

```
你是{narrator}。像自然纪录片一样旁白这张人类的图片。
让它带点讽刺和幽默。不要重复自己。简短一些。
如果我做了任何稍微有趣的事情，请大做文章！
```

GPT-4-Vision API在3.5-8秒内返回响应（但通常更接近3秒）。

## 一个会说话的模型

最后，我们希望我们的模型能说话。并且要有风格地说话！我们不想要机器人般的声音。我们想要一些热情。

![Pasted image 20231201142650.png](/images/posts/117ee41390c3.webp)

这里有很多选择。你能获得的最高质量输出是通过ElevenLabs的语音克隆功能。一个更便宜、开源的语音克隆替代品是XTTS-v2。两者都允许你上传文本让说话者说，以及音频让说话者模仿其声音。然后你会得到一个听起来像你说话者音频的输出。

如果你使用Eleven，请使用他们的Turbo v2模型——它有400毫秒的延迟。查看我这里的play_audio()函数。
