现在，当您的机器人在 Linear 或 GitHub 评论中被提及时，您可以读取父级问题或拉取请求。`message.subject` 解析为该父级对象，包含标题、状态、URL 以及完整的类型化负载。

```
bot.onNewMention(async (thread, message) => {  const subject = await message.subject;
  if (subject) {    await thread.post(      `这是关于：${subject.title} (${subject.status})\n${subject.url}`    );  }});
```

使用父级问题的标题、状态和 URL 回复提及

`message.subject` 按消息缓存，因此重复访问仅调用一次 API。在 Slack 和其他聊天平台上，由于没有父级资源，它会解析为 `null`。

## 链接到标题直接访问平台 SDK

GitHub、Linear 和 Slack 适配器现在暴露了其底层平台 SDK。您可以通过直接调用提供商 API 来扩展您的机器人。

```
const { octokit } = bot.getAdapter("github");await octokit.rest.issues.addLabels({ owner: "vercel", repo: "chat", issue_number: 42, labels: ["triaged"] });
const { linearClient } = bot.getAdapter("linear");await linearClient.createIssue({ teamId: "TEAM_ID", title: "调查不稳定的测试" });
const { webClient } = bot.getAdapter("slack");await webClient.pins.add({ channel: "C123ABC", timestamp: "1234567890.123456" });
```

添加 GitHub 标签、创建 Linear 问题或固定 Slack 消息

之前的 `.client` 获取器在适配器上保留为 `@deprecated` 别名。

阅读文档开始使用，或探索我们的模板。
