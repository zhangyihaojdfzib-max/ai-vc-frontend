# Cloudflare 自豪地加入英国政府网络弹性承诺计划

![](/images/posts/64fe04e339e8.png)

今天，英国政府推出了网络弹性承诺计划：一个自愿性框架，邀请各组织承诺落实基础网络安全治理、董事会层面的问责制，以及覆盖整个供应链的全面网络安全措施。Cloudflare 自豪地加入该承诺计划的创始签署方行列，并继续与科学、创新与技术部（DSIT）、国家网络安全中心及其他机构长期合作，共同为英国打造更安全、面向未来的数字经济。

该承诺计划的三大核心支柱——安全民主化、领导层问责制和彻底透明化——自 Cloudflare 成立之初便深植于我们的基因。我们并未将这一框架视为需要达成的新承诺，而是将其视为英国政府对 Cloudflare 十多年来所倡导的安全理念与原则的认可与肯定。我们很高兴看到行业其他参与者正朝着这个方向迈进。

这项承诺计划是重要的一步，而它出台之际正值网络风险高企之时。2026 年第一季度，Cloudflare 的全球网络平均每天拦截 2340 亿次网络威胁。近期，我们缓解了一次峰值达到 31.4 Tbps 的超大规模 DDoS 攻击。2025 年底，Cloudflare 数据显示，英国已上升为全球第六大 DDoS 攻击目标地区，攻击者越来越多地针对金融服务、航空和地方政府基础设施中的应用层服务。这一趋势与英国网络安全漏洞调查的更广泛数据相符，该调查显示，43% 的受访英国企业和 28% 的慈善机构报告在过去一年中遭受过网络事件。

与此同时，前沿 AI 模型正在迅速改变安全格局，降低了攻击者的准入门槛，并实现了更自动化的漏洞扫描和更具欺骗性的钓鱼活动。Cloudflare 早已为这一转变做好准备。我们最近发布的前沿网络模型防御架构体现了同样的原则：安全必须像企业面临的威胁一样快速演进。从基于机器学习的攻击评分到零信任访问控制，该架构的每一层都已向 Cloudflare 客户开放。

在此背景下，该承诺计划实现了一个关键目标：它认识到集体防御至关重要。它要求组织将网络弹性作为领导层优先事项，实施适当的控制措施以提升威胁感知能力，并帮助确保供应链达到有意义的安全基线。大多数漏洞仍然利用了众所周知的薄弱环节，如未修补的系统、薄弱的访问控制或糟糕的供应商监督。鼓励更多组织通过加强治理、监控和实施来弥补这些漏洞，是一个必要的起点。

Cloudflare 完全支持英国政府提升各类规模企业及组织内部网络安全治理水平的使命。每一个提升自身安全基线的组织，都在为其他人创造一个更安全的互联网。Cloudflare 的使命是帮助构建更好的互联网，我们始终相信，网络安全和弹性只有在普及时才能发挥最大效用。一个更具弹性的互联网，才是一个更好的互联网。

## 为什么弹性至关重要

网络弹性日益被视为一项核心业务需求。客户期望服务始终可用、响应迅速且值得信赖。即使在运营环境因攻击增加、服务中断、滥用或复杂性提升而变得更加困难的情况下，这一点依然成立。

归根结底，弹性不仅仅关乎出问题后的恢复。它关乎设计能够主动追踪威胁信号、无缝吸收干扰并适应改进的安全系统和运营模式。从这个意义上说，安全与弹性密不可分。安全控制措施正是实现弹性的基石。

## Cloudflare 如何通过安全措施帮助增强弹性

凭借我们网络的规模，我们可以在威胁到达核心系统之前，将防护推向边缘，从而帮助组织构建弹性。我们通过几个核心架构原则来思考网络弹性：

### 安全是默认配置，而非产品层级

Cloudflare 相信基线安全防护应惠及所有人，并且自成立以来一直践行这一原则。我们是首家向所有用户提供流量加密所需的 SSL 证书的公司。我们通过"伽利略计划"和"雅典娜计划"等影响力项目保护弱势群体的声音。我们不断突破互联网密码学的边界，包括在我们的网络上部署后量子密码学。我们的免费计划包含不限流量的 DDoS 防护，无论攻击的规模、持续时间或数量如何，同时还提供全球内容分发网络（CDN）和 DNSSEC 的访问权限。这些能力历来需要昂贵的硬件和专业安全团队。但是，只有当中小企业、地方当局、公共服务机构和初创公司能够负担得起参与成本时，该承诺计划提升组织弹性、提高英国经济整体网络安全底线的目标才能实现。我们的模式直接支持了这一目标。

### 网络即传感器

由于 Cloudflare 直接与全球超过 13,000 个网络对等互联，我们能够在攻击模式出现时即刻发现。在一个网络部分收集到的威胁情报，可以在几秒钟内转化为对所有其他部分的防护。在缓解新加坡客户攻击时检测到的威胁，可以立即成为保护谢菲尔德客户的规则。同样的可见性也有助于改进我们跨 Cloudflare 网络和安全服务检测、评分和响应攻击的方式。大规模的可视性为 Cloudflare 的客户和网络带来了大规模的弹性。

### Cloudflare 是零号客户

我们的客户受益于与我们保护自身系统完全相同的、业界领先的安全产品和基础设施。Cloudflare 员工使用 Cloudflare Access 和 Gateway 访问内部应用程序，对内部系统的每个请求都需要基于硬密钥的多因素认证、状态检查和加密验证的身份令牌。我们首先在自己身上测试每一层安全措施，并利用内部经验为自己和网络构建更好的安全解决方案。通过将安全融入业务的每一个层面，Cloudflare 展示了自下而上的承诺，这正是该承诺计划的核心所在。

### 透明度与响应

最后，弹性要求在出现问题时保持诚实和透明，并承诺为未来加强系统。当安全事件或零日漏洞出现时，我们会在 Cloudflare 博客上发布深入的技术事后分析报告。我们分享入侵指标和架构回顾，以便更广泛的安全社区能够从我们的遥测数据中学习。但透明度只是第一步。我们将每一次事件都视为让我们的网络更具弹性的使命。在去年秋天一次重大服务中断后，我们的"橙色代码"行动动员工程团队为弹性而重建。他们设计了"小范围故障"的系统，并构建了新工具来强制执行更安全的配置变更并自动化最佳实践，确保同样的故障不会再次发生。

## Cloudflare 如何落实网络弹性承诺计划

如上所述，当今的自愿承诺要求公司和组织在董事会责任与治理、供应链安全以及英国《网络 essentials》认证计划下的技术要求方面遵守特定标准。作为一家全球网络安全和网络韧性提供商，我们运营着先进的内部网络安全治理模式。

### 董事会责任与治理

鉴于网络安全和韧性是 Cloudflare 全球业务的核心，我们自豪地成为在董事会层面制定和倡导加强网络安全实践的领导者。

我们的董事会将网络风险监督视为核心职责。Cloudflare 的董事会至少每季度从首席安全官处接收网络安全简报，包括直接威胁简报。此外，董事会审计委员会每季度接收企业风险管理简报，其中特别关注网络风险以及公司定期审查和缓解网络威胁与风险的流程。

我们感谢 DSIT 提供的工具包和资源，可用于对标、强化和支持整个英国经济中董事会的持续治理工作。

### 供应链安全与网络 essentials（CE）

Cloudflare 遵循严格的国际安全合规认证。我们要求供应链满足全面的国际标准，这些标准纳入并扩展了网络 essentials 的核心要求。Cloudflare 在全球范围内管理供应商风险，优先采用涵盖并超越网络 essentials 计划基本技术控制的全面国际安全框架。

更具体地说，Cloudflare 要求关键供应商遵守严格的、国际认可的安全合规认证和报告——主要是 ISO 27001 和 SOC 2 Type II。这些框架明确要求实施防火墙、安全配置、用户访问控制、恶意软件防护和补丁管理（网络 essentials 的五大核心支柱）。

Cloudflare 将继续使用基于风险的方法评估供应商。我们赞赏 DSIT 扩大网络 essentials 供应商检查工具的使用范围，Cloudflare 可在英国境内采用该工具进行本地化供应链验证。对于英国网络 essentials 并非原生或实用认证的全球供应商，Cloudflare 将接受等效的国际认证（如 ISO 27001）作为稳健安全态势的充分验证。这些实践有助于确保 Cloudflare 的关键供应链经过严格的安全审查，达到网络 essentials 旨在实现的风险降低效果。

## 展望未来

网络韧性并非一次性的承诺——而是一种持续实践，旨在构建能够安全失效、快速恢复并学习改进的系统。对于英国各地的组织而言，这意味着将网络安全作为业务关键优先事项，获得领导层支持，让团队了解所面临的威胁，并对供应链进行风险管理。该承诺为每个组织设定了应努力达到的基准。

Cloudflare 的平台建立在这样的信念之上：安全性和韧性应具有普遍性，既可供最小的开发者使用，也可供最大的企业使用。我们自豪地与 DSIT 及本承诺的其他签署方站在一起，并期待通过持续的合作与创新，提升英国及全球的网络韧性。
