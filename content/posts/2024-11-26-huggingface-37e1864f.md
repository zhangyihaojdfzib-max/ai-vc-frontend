# 重构 Hugging Face 的上传与下载

作为 Hugging Face Xet 团队改进 Hugging Face Hub 存储后端工作的一部分，我们分析了 24 小时内的 Hugging Face 上传请求，以更好地理解访问模式。在 2024 年 10 月 11 日，我们观察到：

- 来自 88 个国家的上传请求
- 820 万次上传请求
- 130.8 TB 的数据传输量

下图直观展示了这一活动，各国按每小时上传字节数进行着色。

目前，上传数据存储在 us-east-1 的 S3 存储桶中，并使用 S3 Transfer Acceleration 进行优化。下载数据则通过 AWS Cloudfront 作为 CDN 进行缓存和分发。Cloudfront 的 400 多个便捷边缘节点提供了全球覆盖和低延迟数据传输。然而，与大多数 CDN 一样，它针对网页内容进行了优化，文件大小限制为 50GB。

虽然这一大小限制对于典型的互联网文件传输来说是合理的，但模型和数据集仓库中文件体积的不断增长带来了挑战。例如，meta-llama/Meta-Llama-3-70B 的权重总计 131GB，被分割成 30 个文件，以满足 Hub 建议将权重分块为 20GB 段的要求。此外，要为上传和下载启用高级去重或压缩技术，需要重新构想我们处理文件传输的方式。

## 自定义上传与下载协议

为了将 Hugging Face 基础设施推向当前极限之外，我们正在重新设计 Hub 的上传和下载架构。我们计划引入一个内容寻址存储（CAS）作为内容分发的第一站。这使我们能够实现一个基于"简单读取、智能写入"指导理念的自定义协议。与将文件视为不透明数据块的 Git LFS 不同，我们的方法在字节级别分析文件，从而发现提升模型和数据集仓库中大型文件传输速度的机会。

读取路径优先考虑简单性和速度，以确保高吞吐量和低延迟。对文件的请求被路由到 CAS 服务器，该服务器提供重构信息。数据本身仍由 us-east-1 的 S3 存储桶支持，AWS CloudFront 继续作为下载的 CDN。

写入路径则更为复杂，以优化上传速度并提供额外的安全保障。与读取类似，上传请求被路由到 CAS 服务器，但我们不是在文件级别进行查询，而是对数据块进行操作。当找到匹配项时，CAS 服务器指示客户端（例如 huggingface_hub）仅传输必要的（新）数据块。这些数据块在上传到 S3 之前由 CAS 进行验证。

通过在字节级别管理文件，我们可以针对不同文件格式调整优化方案。例如，我们已探索了提高 Parquet 文件的可去重性，目前正在研究压缩张量文件（如 Safetensors），这有望将上传速度提升 10-25%。随着新格式的出现，我们处于独特的位置，能够开发进一步增强 Hub 开发体验的改进措施。

该协议还为企业客户和高级用户带来了显著改进。引入文件传输控制平面提供了额外保障，确保恶意或无效数据无法上传。在运营层面，上传不再是一个黑箱。增强的遥测功能提供了审计追踪和详细日志记录，使 Hub 基础设施团队能够快速高效地识别和解决问题。

## 面向全球访问的设计

为了支持这一自定义协议，我们需要确定 CAS 服务的最佳地理分布。最初考虑了 AWS Lambda@Edge，因其广泛的全球覆盖有助于最小化往返时间。然而，它对 Cloudfront 触发器的依赖使其与我们更新后的上传路径不兼容。相反，我们选择在 AWS 34 个区域中的少数几个部署 CAS 节点。

通过仔细分析我们 24 小时的 S3 PUT 请求窗口，我们识别出了揭示 Hub 数据上传分布的全球流量模式。正如预期，大部分活动来自北美和欧洲，全天都有持续的高容量上传。数据还突显了亚洲强大且不断增长的影响力。通过聚焦这些核心区域，我们可以部署 CAS 存在点，以平衡存储和网络资源，同时最小化延迟。

虽然 AWS 提供 34 个区域，但我们的目标是在保持高用户体验的同时控制基础设施成本。在此快照涵盖的 88 个国家中，上方的帕累托图显示，前 7 个国家占上传字节数的 80%，而前 20 个国家贡献了总上传量和请求量的 95%。

美国是上传流量的主要来源，因此需要在该区域设立 PoP。在欧洲，大部分活动集中在中部和西部国家（如卢森堡、英国和德国），尽管在非洲（特别是阿尔及利亚、埃及和南非）也有一些额外活动。亚洲的上传流量主要由新加坡、香港、日本和韩国驱动。

如果我们使用简单的启发式方法分配流量，可以将 CAS 覆盖范围划分为三个主要区域：

- us-east-1：服务南北美洲
- eu-west-3：服务欧洲、中东和非洲
- ap-southeast-1：服务亚洲和大洋洲

这最终非常有效。美国和欧洲占上传字节数的 78.4%，而亚洲占 21.6%。

这种区域划分使得我们的三个 CAS PoP 负载均衡良好，ap-southeast-1 具有额外的增长容量，us-east-1 和 eu-west-3 可根据需要灵活扩展。

根据预期流量，我们计划按如下方式分配资源：

- us-east-1：4 个节点
- eu-west-3：4 个节点
- ap-southeast-1：2 个节点

## 验证与审查

尽管我们增加了部分用户的第一跳距离，但对 Hub 整体带宽的影响将是有限的。我们的预估显示，虽然所有上传的累积带宽将从 48.5 Mbps 降至 42.5 Mbps（减少 12%），但性能损失将被其他系统优化所抵消。

我们目前正致力于在 2024 年底前将基础设施投入生产，首先在 us-east-1 部署一个 CAS。然后，我们将开始将内部仓库复制到新的存储系统以基准测试传输性能，之后将 CAS 复制到上述其他 PoP 进行更多基准测试。基于这些结果，我们将继续优化我们的方法，以确保明年存储后端全面就位时一切运行顺畅。
