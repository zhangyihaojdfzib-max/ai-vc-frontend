## 什么是合成数据？

合成数据是模仿真实世界数据的人工生成数据。对于图像生成模型而言，合成数据指的是由模型创建的图像，而非真实的照片或人类创作的艺术品。与仅使用真实世界图像相比，使用合成数据有助于创建更多样化和全面的训练数据集。

## 技巧一：从单张图像生成训练数据

`consistent-character` 模型是由多产且独特的 @fofr 开发的图像生成器。它以一张人物图像作为输入，并生成该人物在各种姿势、风格和表情下的多张图像。使用 consistent-character 是快速启动Flux微调的好方法，尤其是在您没有大量训练图像的情况下。

以下是一个使用 Replicate JavaScript 客户端从单张图像输入生成一批训练图像的快速示例：

```
import Replicate from "replicate";

const replicate = new Replicate();
const model = "fofr/consistent-character:9c77a3c2f884193fcee4d89645f02a0b9def9434f9e03cb98460456b831c8772";
const input = {
  prompt: "A closeup headshot photo of a young woman in a grey sweater",
  subject: "https://replicate.delivery/pbxt/L0gy7uyLE5UP0uz12cndDdSOIgw5R3rV5N6G2pbt7kEK9dCr/0_3.webp",
  output_format: "webp",
  output_quality: 80,
  randomise_poses: true,
  number_of_outputs: 5,
  number_of_images_per_pose: 1
}
const output = await replicate.run(model, { input });

console.log(output);
```

## 技巧二：使用微调模型的输出作为训练数据

有时，当您微调Flux时，训练出的模型并不能始终如一地产生您期望质量的图像。也许每十张输出中只有一张符合您的期望。幸运的是，您可以选取这些好的输出，并将其用作训练数据来训练模型的改进版本。

该过程如下：

1.  仅用少量图像创建一个新的微调模型。
2.  使用API运行您的模型以生成大批量图像。
3.  仔细筛选生成的图像并选择好的那些。
4.  使用这些输出作为训练数据运行新的训练任务。

为了简化从模型生成大量图像并将其下载到本地机器的过程，您可以使用像 `aimg` 这样的工具。运行 `aimg` 只需要一个 Replicate API 令牌和最新版本的 Node.js。

以下是一个命令，每次使用完全相同的提示词生成50张图像：

```
# 在 https://replicate.com/account/api-tokens 创建新令牌
export REPLICATE_API_TOKEN=r8_...

# 生成50张图像
npx aimg "a photo of ZIKI the man" --model=zeke/ziki-flux --count 50
```

您还可以使用 `--subject` 标志在输出中获得更多变化，该标志将为每张图像自动生成唯一的提示词：

```
# 生成50张图像，每张都有唯一的提示词
npx aimg --subject "ZIKI the man" --model=zeke/ziki-flux --count 50
```

一旦您收集到一批喜欢的图像，将它们打包成zip文件：

```
zip training-data.zip *.webp
```

然后通过Web或API启动新的训练任务。

注意：所有Replicate模型（包括微调模型）都是版本化的，因此您可以在开始第二次训练任务时使用现有模型作为目标模型，训练过程将自动创建模型的新版本。您的第一个版本将保持不变，您仍然可以访问它并使用它生成图像。

## 技巧三：组合LoRA以多样化训练数据

您可能会发现，您微调的Flux模型仅输出单一风格的图像，例如写实照片。您可能试图生成看起来像绘画或插图的图像，但无论您在提示词中加入多少与绘画相关的关键词，它都只输出照片级写实图像。

Replicate上Flux微调的一个鲜为人知的功能是，您可以在单个输出图像中组合多个LoRA风格。LoRA代表“低秩适应”。这里我不会深入探讨LoRA工作的技术细节，但重要的是要知道，在微调图像模型的上下文中，它已成为“训练权重”的行业术语。当您提到“一个LoRA”时，您指的是添加到基础Flux模型中以构成“微调模型”的一组特定训练权重。

组合LoRA是生成独特图像的一种非常有趣的方式，但您也可以将其作为一种技术来多样化您的训练数据，从而创建您自己微调Flux模型的更好版本。

从高层次来看，该过程如下：

1.  使用您可用的任何训练数据创建一个微调模型。
2.  探索社区的LoRA微调模型，并挑选一些您喜欢的。
3.  使用您的微调模型生成图像，并将其与您选择的其他LoRA组合。
4.  仔细筛选输出并选择符合您期望的图像。
5.  使用这些输出作为训练数据运行新的训练任务。

要找到可与您的模型组合的LoRA，请查看Replicate上的Flux微调模型和Hugging Face上的Replicate LoRA微调模型。

要使用Replicate API生成组合了LoRA的图像，请设置 `extra_lora` 和 `extra_lora_scale` 输入参数，并确保在提示词中使用两个模型的触发词。

以下是一个使用 Replicate JavaScript 客户端生成组合LoRA图像的示例：

```
import Replicate from "replicate";

const replicate = new Replicate();
const model = "zeke/ziki-flux:dadc276a9062240e68f110ca06521752f334777a94f031feb0ae78ae3edca58e";
const input = {
  prompt: "ZIKI the man, illustrated MSMRB style",
  lora_scale: 1,
  extra_lora: "jakedahn/flux-midsummer-blues",
  extra_lora_scale: 1.22,
  num_outputs: 4,
  aspect_ratio: "1:1",
  guidance_scale: 3.5,
  output_quality: 80,
  prompt_strength: 0.8,
}
const output = await replicate.run(model, { input });

console.log(output);
```

组合LoRA时需要记住的关键点是：

1. 请务必在提示词中使用两个模型的**触发词**，以确保LoRA风格正确应用。
2. **extra_lora** 参数应设置为你想与模型结合的LoRA名称。你可以使用模型的简称，如 `jakedahn/flux-midsummer-blues`，或权重文件的完整URL。
3. **extra_lora_scale** 参数应设置在-1到2之间。数值越高，额外LoRA风格的效果越明显。
4. 通过在0.9到1.1之间调整多个LoRA的强度值来尝试平衡它们。
