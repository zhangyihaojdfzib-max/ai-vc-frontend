# 推出 OpenAI o3 和 o4-mini

2025年6月10日更新：OpenAI o3‑pro 现已在 ChatGPT 中面向 Pro 用户提供，同时也在我们的 API 中可用。与 OpenAI o1‑pro 类似，o3‑pro 是我们最智能模型 OpenAI o3 的一个版本，旨在进行更长时间的思考并提供最可靠的回答。完整详情请参阅我们的发布说明⁠（在新窗口中打开）。

今天，我们发布了 OpenAI o3 和 o4-mini，这是我们 o 系列模型的最新版本，这些模型经过训练，在回答之前会进行更长时间的思考。它们是迄今为止我们发布的最智能的模型，代表着 ChatGPT 能力的一次阶梯式提升，惠及从好奇用户到高级研究人员的每一个人。首次，我们的推理模型能够以 Agent（智能体）方式使用并组合 ChatGPT 中的每一个工具——这包括搜索网络、使用 Python 分析上传的文件和其他数据、对视觉输入进行深度推理，甚至生成图像。关键的是，这些模型经过训练，能够推理何时以及如何使用工具，以正确的输出格式生成详细且深思熟虑的回答，通常在不到一分钟的时间内解决更复杂的问题。这使得它们能够更有效地处理多层面的问题，向着一个更能代表你独立执行任务的 Agent（智能体）化 ChatGPT 迈出了一步。最先进的推理能力与完整工具访问权限的结合，在学术基准测试和实际任务中带来了显著更强的性能，为智能性和实用性树立了新的标准。

## 有哪些变化

OpenAI o3 是我们最强大的推理模型，在编程、数学、科学、视觉感知等领域都突破了前沿。它在 Codeforces、SWE-bench（无需构建自定义模型专用脚手架）和 MMMU 等基准测试中创下了新的 SOTA（最先进水平）。它非常适合需要多层面分析且答案可能并非显而易见的复杂查询。它在分析图像、图表和图形等视觉任务中表现尤为出色。在外部专家的评估中，o3 在困难的实际任务中比 OpenAI o1 减少了 20% 的重大错误——尤其是在编程、商业/咨询和创意构思等领域表现出色。早期测试者强调其作为思维伙伴的分析严谨性，并着重指出其生成和批判性评估新颖假设的能力——尤其是在生物学、数学和工程学背景下。

OpenAI o4-mini 是一个较小的模型，针对快速、经济高效的推理进行了优化——就其规模和成本而言，它实现了卓越的性能，特别是在数学、编程和视觉任务方面。它是 AIME 2024 和 2025 年基准测试中表现最佳的模型。尽管使用计算机工具会显著降低 AIME 考试的难度，但我们也注意到，o4-mini 在允许使用 Python 解释器的情况下，在 AIME 2025 上达到了 99.5% 的 pass@1（100% 的 consensus@8）。虽然这些结果不应与未使用工具的模型性能进行比较，但它们很好地说明了 o4-mini 如何有效地利用可用工具；o3 在 AIME 2025 上通过使用工具也显示出类似的改进（98.4% pass@1，100% consensus@8）。

在专家评估中，o4-mini 在非 STEM 任务以及数据科学等领域也优于其前身 o3-mini。得益于其效率，o4-mini 支持比 o3 高得多的使用限制，使其成为处理需要推理的高容量、高吞吐量问题的强大选择。外部专家评估人员对这两个模型的评价是，由于智能性的提升以及网络来源的引入，它们在指令遵循方面表现更好，并且能生成比前代更有用、更可验证的回答。与我们之前迭代的推理模型相比，这两个模型应该会感觉更自然、更具对话性，尤其是当它们引用记忆和过往对话来使回答更具个性化和相关性时。

在 OpenAI o3 的开发过程中，我们观察到大规模强化学习表现出与 GPT 系列预训练中相同的“更多计算 = 更好性能”趋势。通过重新追溯扩展路径——这次是在强化学习中——我们在训练计算和推理时间推理方面都推进了一个数量级，但仍然看到了清晰的性能提升，验证了模型允许思考的时间越长，其性能就会持续提升。在与 OpenAI o1 相同的延迟和成本下，o3 在 ChatGPT 中提供了更高的性能——并且我们已经验证，如果让它思考更长时间，其性能会持续攀升。

我们还通过强化学习训练了两个模型使用工具——不仅教会它们如何使用工具，还教会它们推理何时使用工具。它们根据期望结果部署工具的能力使它们在开放式情境中——尤其是涉及视觉推理和多步骤工作流的情境中——更具能力。正如早期测试者所报告的那样，这一改进既体现在学术基准测试中，也体现在实际任务中。

## 迈向 Agent（智能体）工具使用

OpenAI o3 和 o4-mini 可以完全访问 ChatGPT 中的工具，以及通过 API 中的函数调用访问您自己的自定义工具。这些模型经过训练，能够推理如何解决问题，选择何时以及如何使用工具，以正确的输出格式快速生成详细且深思熟虑的回答——通常在一分钟之内。

例如，用户可能会问：“今年夏天加州的能源使用量与去年相比会如何？”模型可以搜索网络获取公共事业数据，编写 Python 代码构建预测，生成图表或图像，并解释预测背后的关键因素，将多个工具调用串联起来。推理使模型能够根据遇到的信息做出反应和调整。例如，它们可以在搜索提供商的帮助下多次搜索网络，查看结果，并在需要更多信息时尝试新的搜索。

这种灵活、策略性的方法使模型能够处理那些需要访问超出模型内置知识的最新信息、扩展推理、综合以及跨模态输出生成的任务。
