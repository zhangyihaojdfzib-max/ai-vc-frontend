你的Agent（智能体）现在可以通过Vercel MCP和CLI检查Agent Runs（智能体运行记录），eve是开源Agent（智能体）框架。

当部署到Vercel时，eve的追踪数据会自动被采集，并以Agent Runs（智能体运行记录）的形式呈现。新的Vercel MCP工具和Vercel CLI命令让你能够查找包含运行记录的项目、列出最近的运行记录，以及获取完整的追踪数据，包括推理过程、工具调用和Token用量。

Vercel MCP工具：

- list_agent_run_projects：查找团队中有Agent Runs（智能体运行记录）活动的项目。
- list_agent_runs：列出某个项目最近的运行记录。
- get_agent_run：检查元数据、生命周期事件、用量以及子Agent（智能体）数据。
- get_agent_run_trace：获取某次运行记录的追踪数据，包括轮次、消息、推理过程、工具调用、Token用量以及工具输入/输出。

Vercel CLI命令：

- vercel agent-runs projects
- vercel agent-runs list
- vercel agent-runs inspect <runId>
- vercel agent-runs trace <runId>

vercel agent-runs projects

vercel agent-runs list

vercel agent-runs inspect <runId>

vercel agent-runs trace <runId>

每个CLI子命令都支持--json参数以输出机器可读的格式，并且追踪数据在通过管道传输时会渲染为markdown格式，因此没有MCP访问权限的编码Agent（智能体）可以直接调用CLI来调试自身的运行记录。

向你的编码Agent（智能体）提问，例如“显示我项目最新的生产环境Agent Runs（智能体运行记录）”或“根据最近的运行记录更新技能”。

通过安装Vercel MCP开始使用：

```
npx add-mcp https://mcp.vercel.com
```

或者升级到最新版本的Vercel CLI：

```
npm i -g vercel@latest
```
