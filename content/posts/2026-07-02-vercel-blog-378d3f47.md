Vercel AI Gateway 现已支持路由规则。

路由规则是一种防火墙风格的规则，用于控制你的团队可以使用哪些模型，在网关层面应用，而非在你的应用程序代码中。

当某个模型下线或退役时，你通常需要发布代码变更来迁移。使用路由规则，你只需推送一条规则，所有请求便会立即重新路由。规则分为两种类型：

功能

用途

重写

使用另一个模型来处理某个模型的请求

在模型不可用时保持流量畅通、从已退役模型迁移、统一使用某个模型，或将昂贵的模型路由到更便宜的模型

拒绝

阻止对某个模型的请求

让你的团队远离你尚未批准的模型

规则适用于使用你团队 AI Gateway 凭证发出的每一个请求。你可以通过 Vercel CLI 进行管理。

### 复制链接到标题 重写

创建一条重写规则，指定源模型和目标模型。Gateway 会透明地替换为目标模型，因此你的应用程序仍会继续请求源模型：

```
vercel ai-gateway rules add --type rewrite \  --source anthropic/claude-opus-4.8 \  --destination anthropic/claude-haiku-4.5
```

### 复制链接到标题 拒绝

创建一条拒绝规则来阻止某个模型。对该模型的请求将返回 403：

```
vercel ai-gateway rules add --type deny --source openai/gpt-5.5
```

规则仅改变处理请求的模型。你已配置的其他所有内容仍适用于目标模型，包括：

- 请求级别：BYOK、模型回退、排序、唯一过滤器以及提供商选项。
- 团队级别：零数据保留和提供商白名单。

请求级别：BYOK、模型回退、排序、唯一过滤器以及提供商选项。

团队级别：零数据保留和提供商白名单。

路由规则目前处于测试阶段。如需了解更多信息，请阅读路由规则文档。
