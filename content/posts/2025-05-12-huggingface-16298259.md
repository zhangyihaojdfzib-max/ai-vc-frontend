# 视觉语言模型（更优、更快、更强）

## 动机

视觉语言模型（VLMs）已成为当下的热门话题。在之前的一篇博客文章（发布于2024年4月）中，我们详细讨论了VLMs。其中很大一部分内容是关于LLaVA的，这是第一个成功且易于复现的开源视觉语言模型，同时还分享了如何发现、评估和微调开源模型的技巧。

## 新模型趋势

在本节中，我们将探讨新型的VLMs。其中一些是全新的，另一些则是先前研究的改进版本。

## 任意到任意模型

顾名思义，任意到任意模型是指能够接受任何模态输入并输出任何模态（图像、文本、音频）的模型。它们通过模态对齐来实现这一点，即一种模态的输入可以转换为另一种模态（例如，“狗”这个词可以与狗的图片或这个词的发音相关联）。

这些模型拥有多个编码器（每种模态一个），然后将嵌入向量融合在一起，创建一个共享的表示空间。解码器（单个或多个）使用共享的潜在空间作为输入，并解码成所选模态。构建任意到任意模型的最早尝试是Meta的Chameleon，它可以接受图像和文本输入，并输出图像和文本。Meta没有在该模型中发布图像生成功能，因此Alpha-VLLM发布了Lumina-mGPT，它在Chameleon的基础上构建了图像生成功能。

最新且能力最强的任意到任意模型，Qwen 2.5 Omni（下图）是理解任意到任意模型架构的一个很好的例子。

![Qwen-Omni](/images/posts/2336fc1bde4f.png)

Qwen2.5-Omni采用了一种新颖的“思考者-讲述者”架构，其中“思考者”处理文本生成，“讲述者”以流式方式生成自然语音响应。MiniCPM-o 2.6是一个80亿参数的多模态模型，能够理解和生成视觉、语音和语言模态的内容。由深度求索AI推出的Janus-Pro-7B是一个统一的多模态模型，在跨模态理解和生成方面表现出色。它采用了解耦的视觉编码架构，将理解和生成过程分离。

我们预计未来几年此类模型的数量将会增加。一个众所周知的直觉是，多模态学习是我们能够更好地学习深度表示的唯一途径。我们在此集合中整理了一些任意到任意模型和演示。

### 推理模型

推理模型是能够解决复杂问题的模型。我们首先在大语言模型中看到了它们，现在视觉语言模型中也出现了。直到2025年，只有一个开源的多模态推理模型，即Qwen的QVQ-72B-preview。这是一个由阿里巴巴Qwen团队开发的实验性模型，附带了许多免责声明。

今年出现了另一个参与者，Moonshot AI团队的Kimi-VL-A3B-Thinking。它由MoonViT（SigLIP-so-400M）作为图像编码器和一个专家混合（MoE）解码器组成，总参数量为160亿，但激活参数仅为28亿。该模型是Kimi-VL基础视觉语言模型经过长思维链微调和进一步对齐（强化学习）的版本。您可以在此处尝试该模型。

作者还发布了一个指令微调版本，名为Kimi-VL-A3B-Instruct。

该模型可以处理长视频、PDF、截图等多种输入。它同样具备智能体能力。

![kimi-vl](/images/posts/c3fe249935b1.png)

### 小而强大的模型

社区过去通过参数数量来扩展智能，后来则通过高质量的合成数据。在某个阶段之后，基准测试趋于饱和，扩展模型的收益开始递减。社区转而通过各种方法（如知识蒸馏）来缩小大型模型。这是有道理的，因为它降低了计算成本，简化了部署，并解锁了本地执行等用例，增强了数据隐私。

当我们说小型视觉语言模型时，通常指的是参数量小于20亿、可以在消费级GPU上运行的模型。SmolVLM是小型视觉语言模型的一个很好的示例模型系列。作者没有选择缩小大型模型，而是直接尝试将模型压缩到极小的参数量，如2.56亿、5亿和22亿。例如，SmolVLM2试图在这些尺寸下解决视频理解问题，并发现5亿参数是一个很好的折衷方案。在Hugging Face，我们构建了一个iPhone应用程序HuggingSnap，以证明这些模型尺寸可以在消费级设备上实现视频理解。

另一个引人注目的模型是Google DeepMind的gemma3-4b-it。它特别令人兴奋，因为它是拥有128k Token上下文窗口的最小多模态模型之一，并支持140多种语言。该模型属于Gemma 3模型家族，其最大模型在当时在Chatbot Arena排行榜上名列第一。随后，最大的模型被蒸馏成了一个10亿参数的变体。

最后，虽然不是最小的，但Qwen2.5-VL-3B-Instruct值得注意。该模型可以执行各种任务，从定位（目标检测和指向）到文档理解，再到智能体任务；上下文长度高达32k Token。

您可以通过MLX和Llama.cpp集成来使用小型模型。对于MLX，假设您已安装，可以使用以下一行命令开始使用SmolVLM-500M-Instruct：

```bash
python3 -m mlx_vlm.generate --model HuggingfaceTB/SmolVLM-500M-Instruct --max-tokens 400 --temp 0.0 --image https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/vlm_example.jpg --prompt "What is in this image?" 

```

您可以使用以下一行命令，通过CLI在llama.cpp中使用GGUF格式的gemma-3-4b-it模型：

```bash
llama-mtmd-cli -hf ggml-org/gemma-3-4b-it-GGUF  

```

您也可以按如下方式启动该模型的服务。

```bash
llama-server -hf ggml-org/gemma-3-4b-it-GGUF  

```

我们想特别提及moondream2和Florence-2，因为它们是最早尝试构建最小视觉语言模型的先驱。在这篇博客中，我们主要关注较新的模型（主要是2024年4月之后发布的模型）。

### 专家混合作为解码器

专家混合（MoEs）模型通过动态选择并仅激活最相关的子模型（称为“专家”）来处理给定的输入数据片段，为密集架构提供了一种替代方案。这种选择性激活（由路由器完成）机制已被证明有潜力在利用更少计算资源的同时，显著提升模型性能和运行效率。

混合专家模型（MoE）在推理速度上优于参数规模相近的密集模型，因为它只选择性激活网络中较小的部分。它们在训练过程中也收敛得更快。但凡事皆有代价，由于整个模型都需加载到GPU上（即使实际只使用其中一小部分），MoE需要更高的内存成本。

在广泛采用的Transformer架构中，MoE层最常见的集成方式是替换每个Transformer块中的标准前馈网络（FFN）层。密集模型在推理时需调用整个模型，而规模相似的MoE模型则选择性激活部分专家。这有助于提升计算利用率并加速推理。

采用混合专家解码器的视觉语言模型似乎表现出更强的性能。例如，目前最先进的开放推理模型Kimi-VL就采用了混合专家解码器。混合专家模型在多个方向展现出潜力：MoE-LLaVA专注于效率与减少幻觉，DeepSeek-VL2则具备广泛的多模态能力。最新版Llama（Llama 4）也是具备视觉能力的MoE模型。将MoE作为解码器是一个前景广阔的研究方向，我们预计此类模型将会增多。

若想深入理解MoE，我们推荐阅读这篇精彩文章。

### 视觉-语言-动作模型

视觉语言模型（VLM）甚至正在机器人领域崭露头角！在那里，它们被称为视觉-语言-动作模型（VLA）。但请注意，这些本质上仍是VLM，只是稍作扩展。VLA接收图像和文本指令，并返回指示机器人直接执行动作的文本。VLA通过添加动作和状态Token来扩展视觉语言模型，使其能够与物理环境交互并实施控制。这些额外Token代表系统的内部状态（对环境感知）、动作（根据指令执行的操作）以及时间相关信息（如任务步骤顺序）。这些Token会附加在视觉语言输入之后，用于生成动作或策略。

VLA通常在基础VLM之上进行微调。有些人进一步扩展这一定义，将VLA定义为任何与现实或数字世界进行视觉交互的模型。按此定义，VLA可实现UI导航或应用于智能体工作流。但许多人认为这些应用仍属于VLM范畴。

VLA的优秀范例包括π0和π0-FAST——这是Physical Intelligence发布的首个机器人基础模型，已移植至Hugging Face的LeRobot库。这些模型在7个机器人平台和68项独特任务上进行了训练，在叠衣服、收拾餐桌、商品装袋、箱子组装、物体抓取等复杂现实任务中展现出强大的零样本和微调性能。

GR00T N1是英伟达面向通用人形机器人的开源VLA基础模型。它通过结合智能推理与实时运动控制的系统，理解图像和语言并将其转化为动作，例如移动手臂或遵循指令。GR00T N1同样基于LeRobot数据集格式构建，该开放标准旨在简化机器人演示数据的共享与训练。

## 专项能力

### 视觉语言模型的目标检测、分割与计数

如先前章节所见，VLM能够实现传统计算机视觉任务的泛化。模型现在可以接收图像和多样化提示（如开放式文本），并输出带有定位Token的结构化文本（用于检测、分割等任务）。

去年，PaliGemma是首个尝试解决此类任务的模型。该模型接收图像和文本输入，其中文本包含目标物体描述及任务前缀。文本提示类似“分割条纹猫”或“检测屋顶上的鸟”。

对于检测任务，模型以Token形式输出边界框坐标。而对于分割任务，模型则输出检测Token和分割Token。这些分割Token并非所有分割像素坐标，而是通过变分自编码器解码为有效分割掩码的码本索引（如下图所示）。

![PaliGemma3](/images/posts/6d5b57e034fd.png)

PaliGemma之后出现了许多执行定位任务的模型。去年末，升级版PaliGemma 2问世，具备相同能力且性能更优。随后出现的还有Allen AI的Molmo模型，它可以通过点标注实例并统计物体数量。

![molmo](/images/posts/01ac60304673.png)

Qwen2.5-VL同样能检测、指向并计数物体，且将UI元素也视为物体！

![Qwen2.5VL](/images/posts/f33cdc78f2.png)

### 多模态安全模型

生产环境中的视觉语言模型需要对输入和输出进行过滤，以防止越狱和有害输出，确保合规性。有害内容涵盖从暴力到色情等多种类型。多模态安全模型由此应运而生：它们在VLM前后端进行部署，以过滤输入输出内容。这类模型类似于LLM安全模型，但增加了图像输入能力。

2025年初，谷歌推出了首个开源多模态安全模型ShieldGemma 2。它基于纯文本安全模型ShieldGemma构建，可接收图像和内容策略，并判断图像是否符合给定策略的安全性要求。策略指图像不适宜的具体标准。ShieldGemma 2也可用于过滤图像生成模型的输出。

Meta发布的Llama Guard 4是密集型的多模态多语言安全模型。它通过安全性微调从Llama 4 Scout（多模态混合专家模型）进行密集剪枝得到。

![Llama Guard 4](/images/posts/559fa0c6d086.png)

该模型可用于纯文本和多模态推理，还能接收视觉语言模型输出或完整对话记录，在发送给用户前进行内容过滤。

### 多模态RAG：检索器与重排序器

现在让我们看看检索增强生成（RAG）在多模态领域的发展。针对通常以PDF格式存储的复杂文档，RAG处理包含三个步骤：

1. 将文档完全解析为文本
2. 将纯文本和查询传递给检索器和重排序器，获取最相关文档
3. 将相关上下文和查询传递给LLM

传统PDF解析器包含多个组件以保留文档结构和视觉元素，例如布局、表格、图像、图表等，最终全部渲染为Markdown格式。但这种配置可能难以维护。

![Traditional Parsing](/images/posts/d34c33860c18.png)

随着视觉语言模型的兴起，这一问题得到解决：现在已有多模态检索器和重排序器。

![Multimodal RAG](/images/posts/901cb9d4f828.png)

多模态检索器以PDF文档堆和查询作为输入，返回最相关的页码及其置信度分数。分数代表页面包含查询答案的可能性，或查询与页面的相关程度。这绕过了脆弱的解析步骤。

最相关的页面随后与查询一起输入视觉语言模型，由VLM生成答案。

目前主要有两种多模态检索器架构：

1. 文档截图嵌入（DSE, MCDSE）
2. 类ColBERT模型（ColPali, ColQwen2, ColSmolVLM）

DSE模型包含文本编码器和图像编码器，每个查询返回单个向量。返回分数基于嵌入向量点积的softmax计算。每个文本段落对应单个向量。

![DSE](/images/posts/7a70fbec50ef.png)

类似ColBERT的模型，例如ColPali，也属于双编码器模型，但有一个特别之处：ColPali使用视觉语言模型作为图像编码器，并使用大语言模型作为文本编码器。这些模型本质上并非编码器，但会输出嵌入向量，这些向量随后被传递给一个“MaxSim”模块。与DSE不同，其输出是多个向量，每个Token对应一个。在MaxSim中，会计算每个文本Token嵌入与每个图像块嵌入之间的相似度，这种方法能更好地捕捉细微差别。因此，类似ColBERT的模型成本效益较低，但性能更好。

下方您可以看到ColPali的索引延迟。由于它只是一个单一模型，维护起来也更容易。

![ColPali](/images/posts/c94764b8a7bf.png)

在Hugging Face Hub上，您可以在“Visual Document Retrieval”任务下找到这些模型。

该任务最流行的基准测试是ViDoRe，它包含英语和法语文档，文档类型涵盖财务报告、科学图表到行政文件。ViDoRe的每个示例都包含作为图像的文档、一个查询和可能的答案。与查询匹配的文档有助于进行对比预训练，因此ViDoRe的训练集被用于训练新模型。

## 多模态智能体

视觉语言模型解锁了许多智能体工作流，从与文档对话到计算机操作。这里我们将重点介绍后者，因为它需要更高级的智能体能力。最近，出现了许多能够理解和操作UI的视觉语言模型。最新的是字节跳动发布的UI-TARS-1.5，它在浏览器、计算机和手机操作方面表现出色。它还能进行带推理的游戏玩法，并在开放世界游戏中操作。今年另一个有影响力的发布是MAGMA-8B，它是一个用于UI导航和与现实世界物理交互的基础模型。此外，Qwen2.5-VL（特别是其32B变体，因为它进一步在智能体任务上进行了训练）和Kimi-VL推理模型在GUI智能体任务中表现良好。

2025年初，我们推出了smolagents，一个新的轻量级智能体库，它实现了ReAct框架。不久之后，我们为该库实现了视觉语言支持。这种集成体现在两个用例上：

*   在运行开始时，一次性提供图像。这对于需要工具使用的文档AI很有用。
*   动态检索图像。这对于诸如使用VLM智能体进行GUI控制等情况很有用，智能体会反复截取屏幕截图。

该库为用户提供了构建块，以便他们利用图像理解能力构建自己的智能体工作流。我们提供了不同的脚本和单行CLI命令，让用户可以轻松上手。

对于第一种情况，假设我们希望一个智能体描述文档（这本身智能体属性不强，但适用于最小化用例）。您可以像下面这样初始化CodeAgent（一个能自己写代码的智能体！）：

```python
agent = CodeAgent(tools=[], model=model) 
agent.run("Describe these documents:", images=[document_1, document_2, document_3])

```

对于后一种需要智能体获取屏幕截图的用例，我们可以定义一个回调函数，在每个`ActionStep`结束时执行。对于您自己需要动态获取图像的使用场景，可以按需修改回调函数。为简洁起见，我们在此不详细定义。您可以选择在本博客文章末尾阅读相关博客文章和脚本本身。现在，让我们看看如何使用回调和浏览器控制步骤来初始化智能体。

## 视频语言模型

如今大多数视觉语言模型都能处理视频，因为视频可以表示为一序列帧。然而，视频理解很棘手，因为帧之间存在时间关系且帧数量巨大，因此需要使用不同的技术来选择一组有代表性的视频帧。

![Video LMs](/images/posts/c8f2c5540d11.png)

自去年以来，社区已经权衡了解决这个问题的不同方法和技巧。

一个很好的例子是Meta的LongVU模型。它通过将视频帧传递给DINOv2进行下采样，以挑选最相似的帧并将其移除，然后模型根据文本查询进一步筛选最相关的帧，其中文本和帧都被投影到同一空间并计算相似度。Qwen2.5VL能够处理长上下文，并适应动态FPS速率，因为该模型是用不同帧率的视频训练的。通过扩展的多模态RoPE，它理解帧的绝对时间位置，能够处理不同的速率，并且仍然理解现实生活中事件发生的速度。另一个模型是Gemma 3，它可以接受在文本提示中与时间戳交错的视频帧，例如“Frame 00.00: <image>..”，并且在视频理解任务上表现非常出色。

![MRoPE](/images/posts/99b6276bb382.png)

## 视觉语言模型的新对齐技术

偏好优化是语言模型的一种替代微调方法，也可以扩展到视觉语言模型。这种方法不依赖固定标签，而是侧重于基于偏好来比较和排序候选响应。`trl`库支持直接偏好优化，包括用于VLM。

下面是一个示例，展示了用于VLM微调的DPO偏好数据集的结构。每个条目由一个图像+问题对以及两个相应的答案组成：一个被选中的答案和一个被拒绝的答案。VLM被微调以生成与首选答案一致的响应。

![DPO](/images/posts/2d2dd9eaa936.png)

此过程的一个示例数据集是RLAIF-V，它包含超过83000个根据上述结构格式化的标注样本。正如DPOTrainer所期望的那样，每个条目包括一个图像列表（通常是一个）、一个提示词、一个选中的答案和一个被拒绝的答案。有一个RLAIF-V格式化数据集，已经相应地进行了格式化。下面是一个单个样本的示例：

```json
{'images': [<PIL.JpegImagePlugin.JpegImageFile image mode=L size=980x812 at 0x154505570>],
 'prompt': [ { "content": [ { "text": null, "type": "image" }, { "text": "这名捕手应该使用什么？", "type": "text" } ], "role": "user" } ],
 'rejected': [ { "content": [ { "text": "这名捕手，根据其编号识别...", "type": "text" } ], "role": "assistant" } ],
 'chosen': [ { "content": [ { "text": "图片中的捕手应该使用棒球手套...", "type": "text" } ], "role": "assistant" } ]}
```

## 新的基准测试

过去一年，基准测试也发生了显著演变。在我们之前的博客中，我们将 MMMU 和 MMBench 描述为评估视觉语言模型的两个新兴基准。随着该领域的快速发展，模型在这些基准上的表现已趋于饱和，我们需要更好的评估工具。为此，我们需要在通用基准之上，能够评估特定能力的工具。

![MMT-Bench](/images/posts/3fa92ce73323.png)

现在，我们重点介绍两个突出的通用基准：MMT-Bench 和 MMMU-Pro。

### MMT-Bench

MMT-Bench 旨在评估视觉语言模型在需要专业知识、精确视觉识别、定位、推理和规划的各种多模态任务上的表现。该基准包含来自不同多模态场景的 31325 道多选视觉问题，涵盖图像、文本、视频和点云模态。它包含 32 个不同的元任务和 162 个子任务，覆盖了包括 OCR、视觉识别或视觉语言检索在内的多种任务。

### MMMU-Pro

MMMU-Pro 是原始 MMMU 基准的改进版本。它也评估先进 AI 模型在多种模态下的真实理解能力。它比 MMMU 更复杂，例如，它有一个纯视觉输入设置，并将候选选项数量从 4 个增加到 10 个。该基准还融入了真实世界模拟，其纯视觉问题来源于模拟显示器内捕获的屏幕截图或照片，具有不同的背景、字体样式和大小，以模拟真实世界条件。
