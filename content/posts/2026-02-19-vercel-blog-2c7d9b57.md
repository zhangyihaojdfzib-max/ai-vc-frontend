Vercel Blob 现已支持为合同、发票和内部报告等敏感文件提供私有存储。私有存储要求所有操作都需经过身份验证，防止通过公开 URL 暴露文件。

公共存储允许对媒体资源进行公开读取，而私有存储则需要身份验证。

您可以通过 **Storage 仪表板** 或使用 CLI 创建一个私有存储：

CLI 命令

```
1vercel blob create-store [name] --access private
```

在已关联的 Vercel 项目内创建时，CLI 会提示您连接该存储，并自动添加 `BLOB_READ_WRITE_TOKEN` 环境变量。SDK 将使用此变量在您的部署中对操作进行身份验证。

SDK 安装

```
1pnpm add @vercel/blob@2.3
```

要上传文件，请使用 `put` 或 `upload` 并指定 `access: 'private'` 选项。

上传示例

```
1import { put } from '@vercel/blob';2
3export async function POST(request: Request) {4  5  6  const filename = request.nextUrl.searchParams.get('filename');7  const blob = await put(filename, request.body, {8    access: 'private',9  });10
11  return Response.json(blob);12}
```

要下载文件，请使用 `get` 方法来流式传输文件。

检索示例

```
1import { get } from '@vercel/blob';2
3export async function GET(req: Request) {4  5  6  const filename = request.nextUrl.searchParams.get('filename');7  const { stream, blob } = await get(filename, {8    access: "private",9  });10
11  return new Response(stream, {12    headers: {13      "Content-Type": blob.contentType,14    },15  });16}
```

私有存储目前处于测试阶段，适用于所有套餐，遵循标准的 Vercel Blob 定价。
