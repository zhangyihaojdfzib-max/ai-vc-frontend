## Cohere 模型

Cohere 和 Cohere Labs 将其在特定业务应用中表现出色的大量模型引入推理提供商。让我们详细探索其中一些。

### CohereLabs/c4ai-command-a-03-2025🔗

针对需要快速、安全且高质量 AI 的高要求企业进行了优化。其 256k 上下文长度（是大多数领先模型的 2 倍）可以处理更长的企业文档。其他关键特性包括 Cohere 的先进检索增强生成（RAG）及可验证引用、Agent（智能体）工具使用、企业级安全性以及强大的多语言性能（支持 23 种语言）。

### CohereLabs/aya-expanse-32b🔗

专注于最先进的多语言支持，应用了多语言预训练的最新研究成果。支持阿拉伯语、中文（简体和繁体）、捷克语、荷兰语、英语、法语、德语、希腊语、希伯来语、印地语、印尼语、意大利语、日语、韩语、波斯语、波兰语、葡萄牙语、罗马尼亚语、俄语、西班牙语、土耳其语、乌克兰语和越南语，上下文长度为 128K。

### CohereLabs/c4ai-command-r7b-12-2024🔗

非常适合低成本或低延迟用例，在其同类开放权重模型中，跨真实世界任务带来了最先进的性能。该模型提供 128k 的上下文长度。它提供了多语言支持、引用验证的检索增强生成（RAG）、推理、工具使用和 Agent（智能体）行为的强大组合。同样支持 23 种语言。

### CohereLabs/aya-vision-32b🔗

320 亿参数模型，具有针对各种视觉语言用例优化的先进能力，包括 OCR、字幕生成、视觉推理、摘要、问答、代码等。它将多模态能力扩展到 23 种语言，覆盖全球超过一半的人口。

#### 从 Python，使用 huggingface_hub

以下示例展示了如何使用 Cohere 作为推理提供商来使用 Command A。您可以使用 Hugging Face Token 通过 Hugging Face 进行自动路由，或者如果您有的话，也可以使用您自己的 Cohere API 密钥。

安装 huggingface_hub v0.30.0 或更高版本：

```sh
pip install -U "huggingface_hub>=0.30.0"

```

通过定义 provider 参数，使用 huggingface_hub Python 库调用 Cohere 端点。

```py
from huggingface_hub import InferenceClient

client = InferenceClient(
    provider="cohere",
    api_key="xxxxxxxxxxxxxxxxxxxxxxxx",
)

messages = [
        {
            "role": "user",
            "content": "如何制作极辣的蛋黄酱？"
        }
]

completion = client.chat.completions.create(
    model="CohereLabs/c4ai-command-r7b-12-2024",
    messages=messages,
    temperature=0.7,
    max_tokens=512,
)

print(completion.choices[0].message)

```

```py
image_path = "img.jpg"
```


## 使用 Cohere 模型的工具调用

Cohere 的模型为推理提供商带来了最先进的 Agent（智能体）工具使用能力，让我们详细探讨这一点。Hugging Face Hub 客户端和 OpenAI 客户端都通过推理提供商与工具兼容，因此可以扩展上述示例。

首先，我们需要定义模型要使用的工具。下面我们定义了 get_flight_info，它使用两个地点调用 API 获取最新的航班信息。此工具定义将由模型的聊天模板表示。我们也可以在模型卡（🎉 开源）中探索它。

```py
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_flight_info",
            "description": "获取两个城市或机场之间的航班信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "loc_origin": {
                        "type": "string",
                        "description": "出发机场，例如 MIA",
                    },
                    "loc_destination": {
                        "type": "string",
                        "description": "目的机场，例如 NYC",
                    },
                },
                "required": ["loc_origin", "loc_destination"],
            },
        },
    }
]

```

接下来，我们需要将消息传递给推理客户端，以便模型在相关情况下使用工具。在下面的示例中，为了清晰起见，我们将助手的工具调用定义在 `tool_calls` 中。

```py

messages = [
    {"role": "developer", "content": "今天是4月30日"},
    {
        "role": "user",
        "content": "从迈阿密到西雅图的下一班航班是什么时候？",
    },
    {
        "role": "assistant",
        "tool_calls": [
            {
                "function": {
                    "arguments": '{ "loc_destination": "Seattle", "loc_origin": "Miami" }',
                    "name": "get_flight_info",
                },
                "id": "get_flight_info0",
                "type": "function",
            }
        ],
    },
    {
        "role": "tool",
        "name": "get_flight_info",
        "tool_call_id": "get_flight_info0",
        "content": "迈阿密到西雅图，5月1日上午10点。",
    },
]

```

最后，将工具和消息传递给 create 方法。

```py
from huggingface_hub import InferenceClient

client = InferenceClient(
    provider="cohere",
    api_key="xxxxxxxxxxxxxxxxxxxxxxxx",
)

completion = client.chat.completions.create(
    model="CohereLabs/c4ai-command-r7b-12-2024",
    messages=messages,
    tools=tools,
    temperature=0.7,
    max_tokens=512,
)

print(completion.choices[0].message)

```
