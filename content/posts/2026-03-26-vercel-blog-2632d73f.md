使用最新测试版包创建的 Vercel Sandboxes 现在将在项目中拥有唯一且可自定义的名称，取代了之前基于 ID 的标识方式。命名使沙盒更易于查找和引用：

```
1import { Sandbox } from '@vercel/sandbox';2
34const sandbox = await Sandbox.create({ name: 'user-a-workspace' });5await sandbox.runCommand('npm', ['install']);6await sandbox.stop();7
89const sandbox = await Sandbox.get({ name: 'user-a-workspace' });10await sandbox.runCommand('npm', ['run', 'dev']);
```

测试版 CLI 增加了配置管理和会话检查功能：

```
1  2  sandbox create --name user-alice3
4  5  sandbox run --name user-alice -- npm test6
7  8  sandbox sessions list user-alice
```

命名沙盒是识别自动持久化的机制，它使您的会话在创建和恢复时更容易被识别。

立即安装测试版包以尝试命名沙盒：对于 SDK，运行 `pnpm install @vercel/sandbox@beta`；对于 CLI，运行 `pnpm install -g sandbox@beta`。
