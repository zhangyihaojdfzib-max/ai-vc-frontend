Node.js 18（长期支持将于2025年4月30日结束）和Vercel旧版构建镜像将于2025年9月1日弃用。如果您在该日期仍使用旧版构建镜像，新构建将显示错误。

旧版构建镜像与最新构建镜像之间有哪些变化？

- Node.js的最低版本现为20.x
- Python工具链版本现为3.12
- Ruby工具链版本现为3.3.x

Node.js的最低版本现为20.x

Python工具链版本现为3.12

Ruby工具链版本现为3.3.x

如何判断我是否仍在使用旧版构建镜像？

- 在构建与部署设置中使用Node.js 18.x的项目使用旧版构建镜像
- 在package.json中使用overrides的项目使用旧版构建镜像

在构建与部署设置中使用Node.js 18.x的项目使用旧版构建镜像

在package.json中使用overrides的项目使用旧版构建镜像

我现有的部署会受影响吗？

现有部署不会受到影响。但是，下次部署时需要更新Node.js版本。

如何查看我的项目是否受影响？

您可以通过运行以下命令查看哪些项目受此弃用影响：

```
1npm i -g vercel@latest2vercel project ls --update-required
```

查看需要更新的项目。

如何升级？

要通过仪表板升级，请访问您项目的构建与部署设置并升级版本。

要通过代码升级，请在package.json中使用engines字段：

```
1{2  "engines": { 3    "node": "22.x"4  }5}
```

该日期与之前宣布的Vercel平台上Node.js 18弃用日期一致。了解更多关于构建镜像差异的信息。
