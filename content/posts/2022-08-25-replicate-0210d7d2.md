自从那些四耳无脸兔子出现以来，我们已经取得了长足的进步。本周早些时候，我们有幸获得了来自 The Great A100 Cluster 的权重。那么，让我们好好利用它们吧。

我们将使用 Replicate 来运行模型，这样你就不需要 GPU，并且我们会将其部署到 Fly.io 上，这样即使你的电脑关机，它也能一直运行。

### 编写机器人代码

创建一个名为 bot.py 的新文件，并粘贴以下代码：

```
from discord import Intents
from discord.ext import commands
from dotenv import load_dotenv
import os
import replicate

load_dotenv()

intents = Intents.default()
intents.message_content = True

bot = commands.Bot(
    command_prefix="!",
    description="在 Replicate 上运行模型！",
    intents=intents,
)


@bot.command()
async def dream(ctx, *, prompt):
    """使用 stable-diffusion 模型根据文本提示词生成图像"""
    msg = await ctx.send(f"“{prompt}”\n> 生成中...")

    model = replicate.models.get("stability-ai/stable-diffusion")
    version = model.versions.get("27b93a2413e7f36cd83da926f3656280b2931564ff050bf9575f1fdf9bcd7478")
    image = version.predict(prompt=prompt)[0]

    await msg.edit(content=f"“{prompt}”\n{image}")


bot.run(os.environ["DISCORD_TOKEN"])
```

这个文件定义了你的机器人命令（!dream）及其工作方式。以后当你想要修改机器人时，你会回到这个文件。

## 使用机器人

在你的 Discord 服务器的某个频道中输入 !help，检查你的机器人是否在运行。你的机器人应该会回复一个它可以运行的命令列表，其中包括 !dream。


## 使用 Fly.io 部署你的机器人

在本地运行机器人很方便，尤其是在你积极开发它的时候。然而，缺点是它只能在你在线时运行。如果你正在为一个服务器构建机器人，你可能希望即使你不在线，它也能保持在线。

有很多方法可以部署这样的应用。最简单的方法包括 Vercel、Fly.io 或 Heroku 等服务。在本教程中，你将使用 Fly 将机器人部署到云端。

首先，请参阅 Fly 的“速通”指南来安装 flyctl 命令行工具并创建一个 Fly 账户。

然后创建一个名为 Dockerfile 的新文件，并粘贴以下代码：

```
FROM python:3.10
WORKDIR /code
COPY requirements.txt /code/
RUN pip install -r requirements.txt
COPY . /code
CMD python bot.py
```

然后创建一个新的 Fly 应用：

```
flyctl launch
```

该命令将生成一个名为 fly.toml 的新文件，但它是为运行 Web 应用而设计的，因此你需要做一些修改。删除 [[services]] 块及其下面的所有内容。你修改后的文件应该看起来像这样：

```
# fly.toml file generated for your-fly-app on 2022-08-25T17:37:31+01:00
app = "your-fly-app"
kill_signal = "SIGINT"
kill_timeout = 5
processes = []

[env]

[experimental]
  allowed_public_ports = []
  auto_rollback = true
```

然后使用本地 .env 文件中的密钥配置你的 Fly 应用：

```
flyctl secrets set REPLICATE_API_TOKEN=... DISCORD_TOKEN=...
```

就是这样！你的机器人现在正在云端运行。

如果你在服务器中运行机器人，你需要做的一件事是让使用它的人了解模型许可协议。你可能想把它添加到服务器规则、频道主题或类似的地方。
