# 宣布推出变现网关：通过 x402 对 Cloudflare 背后的任何资源收费

今天，我们宣布推出 Cloudflare 变现网关，这是一个引擎，将使 Cloudflare 客户能够对受 Cloudflare 保护的任何资产收费：网页、数据集、API 或 MCP 工具。

它将提供一个统一控制平面来管理跨应用程序的支付策略和访问控制，同时通过在边缘处理支付验证和执行来保护您的源站免受高支付量的影响。上线时，支付将通过 x402（我们与超过 25 位行业领导者通过 x402 基金会共同构建的开放协议）以稳定币结算。

### 网络商业模式的演变

30 年来，网络一直运行在一个简单的经济交易上：用内容换取人类注意力。这种注意力通过广告、订阅和电子商务变现。这个交易为我们所知的互联网提供了资金。

但随着 Agent（智能体）成为主要的互联网用户，这种模式正在瓦解。Agent（智能体）不会看广告，也不需要为它想访问的所有工具保持月度订阅。它读取一个页面或消费一次数据源，获取所需内容后便离开。在整个网络上，AI 爬虫每为其送回一个访客，就会请求内容数百到数万次。

这一现实需要一种新模式：为一切按使用量定价。如果注意力和电子商务正从网站转向 AI 工具和 AI 编写的软件，那么 Agent（智能体）应该为其所需的输入付费——训练数据、推理内容、开发者工具和 API 使用。软件的自然支付单位是请求、Token 或结果，而不是席位或月份。以下是一些可能的例子：

- 每次网络搜索几美分，按调用计费
- 上传端点收取 0.001 美元基础费加每 MB 0.01 美元
- 每次解决的支持升级收费 0.99 美元，仅在成功完成时支付

这与当答案引擎使用创作者内容时向其付费的转变相同——每当内容或资源被使用时进行公平的价值交换，通过为此目的构建的中立轨道定价。人们通常设想 Agent（智能体）购买高价资产如网络域名，但 Agent（智能体）支付的大部分内容位于任何结账流程的上游，且定价远低于此。

互联网的一部分已经以这种方式运作。云和 API 多年来一直按调用和按小时销售，但仅面向已知买家：用户注册后获得 API 密钥，并产生按使用量计费的账单。内容大多跳过支付，转而依靠广告。这些商业模式从未能够为未经验证的买家处理低于美分的交易，因为支付轨道成本太高且结算时间太长。低于某个价格时，收款成本超过了收款本身的价值。

历史上，按使用量计费难以实施。企业需要有效成为支付公司，运行自己的会计系统以稳健且可审计的方式跟踪内部使用情况。跟踪这种使用需要对后端系统进行重大改造。许多人转而选择按席位定价，因为它更简单且通常更有利可图。

Agent（智能体）颠覆了这一动态。单个 Agent（智能体）可以全天候完成整个团队的工作，使得一次性固定费用与实际消费脱节。同时，Agent（智能体）可以无摩擦地进行数千次微支付，而要求人为批准每次支付将带来难以承受的负担。按使用量定价正是 Agent（智能体）的生存空间，也是基于稳定币的微支付大放异彩之处。这是因为稳定币（如 Open USD 和 USDC）允许买家在互联网上转移极小金额，产生可忽略的费用并在不到一秒内结算。这在今天的其他支付轨道上是不可行的。

这就是我们可以提供帮助的地方。Cloudflare 多年来一直在为我们自己的计费系统和客户的 analytics（分析）构建按使用量计费系统。凭借我们作为买卖双方之间代理层的地位，我们可以极大简化基于网络的资产的按使用量计费实施。如下所示，在 Cloudflare 支持按使用量计费的情况下，支付证据可以移入请求本身，支付验证和请求路径合并。

这对您的好处是：计量、支付交换和结算从您的源站移出。留在您这里的是重要的东西——您的规则、您的价格和您的收入。您无需为买家注册或搭建计费系统。您只需编写一条规则，Agent（智能体）买家就会为其使用付费。

### x402 回顾

去年在内容独立日，我们让网站所有者一键控制哪些 AI 爬虫可以访问他们的内容，并通过按爬取付费让他们向爬虫收费。变现网关是下一步：您不仅可以向爬虫收取内容费用，还可以向任何调用者收取任何资源的费用，从 API 到数据到 MCP 工具调用，而您无需自行构建支付机制。

x402 是一个开放协议，使得通过 HTTP 支付成为可能，其名称来源于它最终启用的 402 状态码。x402 交换很简单：客户端请求一个受支付保护的资源。服务器不提供该资源，而是响应 402 Payment Required 和一个包含价格、接受的资产和支付地点的小型负载。客户端支付并重复请求，附上支付证明。验证者进行验证，服务器返回资源。这一切都在普通的 HTTP 请求和响应中完成，无需重定向到结账页面，也无需调用单独的支付 API。结算点对点进行，因此买家发送给卖家的任何资金都直接存入卖家的钱包。我们正在设计变现网关以保持低支付开销，目标是亚秒级支付结算。

x402 支付流程：AI Agent（智能体） ↔ API 服务器 ↔ 区块链，来源：GitHub 上的 x402 Readme

两个特性使 x402 非常适合机器支付。支付金额可以很小，低至美分的一小部分，因为该协议几乎不增加开销。而且买家无需在卖家处拥有账户，因为支付本身就是凭证。x402 与轨道无关，但它天然适合稳定币，稳定币可以在不到一秒内结算美分的一小部分，且零退款。

### 变现网关的功能

变现网关将提供一个灵活的支付规则 API，使您能够精确表达何时希望调用者为访问您的数字资源付费。

以下是其工作原理。Token、API、MCP 工具调用和数据已经流经该路径。您将能够尽可能精确地决定哪些流量需要付费。您将能够通过编写表达式来执行您的决定，这些表达式类似于您已经为其他 Cloudflare 规则编写的表达式，通过一个简单、专用的产品 API。变现网关将随 Cloudflare 覆盖 330 多个城市的全球网络扩展，这意味着 x402 握手将在靠近您的买家的位置发生。这将减少请求延迟并保护您的源站。

以下是计划中的一些功能示例：

- 针对特定REST动词收费：对特定路由的调用要求付费，例如对/api/premium/*的每次GET或POST请求收取0.01美元。
- 可变定价：根据任务复杂程度收取不同费用，例如图像生成可能根据使用的算力收取最高2美元的费用。
- 仅对未认证调用者收费：拦截来自源站的HTTP 401“未授权”响应，改为返回带有定价和付款说明的402“要求付款”响应。

当请求匹配时，变现网关将在放行前验证付款。您可以在仪表盘中设置这些规则，或通过Cloudflare API和Terraform以代码形式管理它们，因此付费端点只是基础设施配置的另一部分。

变现网关最初将允许用户要求买家以稳定币支付服务和资源费用。卖家可以将积累的稳定币用于自身交易，或将稳定币兑换为等值法定货币存入银行账户。使用变现网关提供了一种扩大产品可寻址市场的方式。通过该网关，Agent（智能体）可以请求您的资源，获知价格，完成支付，并获取响应。无需注册、无需API密钥、无需事先建立关系。您将自行决定需要了解买家的多少信息，并可以灵活地要求Agent（智能体）通过Web Bot Auth进行认证，并对其已有账户应用基于使用量的定价。

### 我们对未来的展望

变现网关将把请求转化为支付，为Cloudflare客户带来新的收入机会，但其前景远不止于此。

Agent（智能体）是一种代表用户自主行动的软件，而Agent（智能体）正开始独立行动。很快它们将携带钱包，在无人干预的情况下自行购买所需：数据集、API调用、工具、计算资源块。其中一些资源将是免费的，另一些则需要通过验证的Agent（智能体）身份来证明其身份及所代表的实体。许多资源将同时需要身份验证和支付，而Cloudflare是少数能够在单个请求内完成所有这些环节的平台——在源站看到调用之前，验证Agent（智能体）、应用规则并检查支付。Agent（智能体）将成为互联网上的主要买家，而请求则成为交易。

如今，互联网上流动着大量价值未被变现或变现不足，并非因为无人愿意付费，而是因为从未存在过收费工具。每一次有用的API调用、每一个答案、每一次Agent（智能体）的工具调用都具有价值，而如今几乎没有任何一项获得付费。这就是我们面前的机会，也是变现网关将解锁的潜力。

这正是我们努力构建的目标：一个以Agent（智能体）为先的互联网，内置互联网级别的结算能力。让创造有价值事物的人，能够自动从使用该事物的软件那里获得报酬。让最小的新API也能以与互联网上最大公司相同的条件触达相同的买家，让独立创作者能够从使用其作品的大语言模型那里获得报酬。这就是互联网的下一个商业模式，而我们正在为此构建基础设施。
