### 什么是 GUI Agent？

**工作中的 GUI Agent - 由 OSWorld 提供**

简而言之，AI Agent 是在虚拟世界中行动的机器人。（更全面的定义请见此处）

具体来说，“GUI Agent” 是存在于图形用户界面中的 Agent。可以理解为“一个能在我的桌面或手机上执行点击和导航操作的 Agent”，类似于 Claude Computer Use。

本质上，这意味着驱动 Agent 的 AI 模型将收到一个任务（例如“填写此 Excel 列的其余部分”）以及 GUI 的屏幕截图。利用这些信息，它将决定在系统上采取行动：`click(x=130, y=540)` 来打开网页浏览器，`type("Value for XYZ in 2025")`，`scroll(down=2)` 来进一步阅读……要查看 GUI Agent 的实际运行情况，您可以尝试我们由 **Qwen2.5-VL-72B** 驱动的 **Open Computer Agent**。

一个好的 GUI Agent 能够像我们一样操作计算机，从而解锁所有计算机任务：滚动浏览谷歌地图、编辑文件、在线购买商品。这涉及多种难以评估的能力。

### 隆重介绍 ScreenSuite 🥳

相关文献，例如 Xu 等人 (2025) 或 Qin 等人 (2025)，通常将 GUI Agent 的能力分为几类：

1.  **感知**：正确感知屏幕上显示的信息
2.  **定位**：理解元素的位置——这对于点击正确的位置至关重要
3.  **单步操作**：通过一次操作正确解决指令
4.  **多步 Agent**：在 GUI 环境中通过多个操作解决更高级别的目标。

因此，我们的第一个贡献是**收集并统一了一套全面的、包含 13 个基准测试的套件**，涵盖了所有这些 GUI Agent 能力。

如果您查看上面列出的最后一类，评估多步 Agent 能力尤其具有挑战性，因为它需要虚拟机来运行 Agent 的环境，无论是 Windows、Android 还是 Ubuntu……为了解决这个问题，我们既支持 **E2B 桌面** 远程沙箱，又从零开始创建了一个新选项，可以轻松在 Docker 中启动 Ubuntu 或 Android 虚拟机！

**实现细节**

我们精心设计了基准测试套件，注重模块化和一致性，确保任务和环境之间的高度统一。在需要时，特别是对于在线基准测试，我们利用 **smolagents** 作为框架层来简化 Agent 的执行和编排。

为了支持可重复性和易用性，我们构建了定制的 Docker 化容器，允许本地部署完整的 **Ubuntu 桌面** 或 **Android** 环境。

与许多依赖可访问性树或其他元数据以及视觉输入的现有 GUI 基准测试不同，我们的堆栈**有意地仅使用视觉输入**。虽然这可能导致在某些既定排行榜上得分不同，但我们认为这创造了一个更真实、更具挑战性的设置，能更好地反映人类感知和与图形界面交互的方式。

– 所有 Agent 框架（Android World、OSWorld、GAIAWeb、Mind2Web）都使用 **smolagents** 并且**仅依赖视觉**，没有任何可访问性树或 DOM 的辅助（与其他来源报告的评估设置形成对比）。
– **Mind2Web (Multimodal)** 最初使用基于**元素名称的多项选择**（基于可访问性树和屏幕截图），但后来被调整为**仅使用视觉**在边界框内进行**点击精度**评估，这显著增加了任务难度。

### 在 ScreenSuite 上对领先的 VLM 进行排名 📊

我们已经在基准测试中评估了领先的 VLM

-   **Qwen-2.5-VL 系列模型**，从 3B 到 72B。这些模型以其出色的定位能力而闻名，换句话说，它们知道图像中任何元素的坐标，这使它们非常适合需要精确点击的 GUI Agent。
-   **UI-Tars-1.5-7B**，字节跳动的全能选手。
-   **Holo1-7B**，H 公司的最新模型，在其规模上展现出极其出色的定位能力。
-   **GPT-4o**

我们的得分与各种来源报告的得分基本一致！需要注意的是，我们仅基于视觉进行评估，这导致了一些差异，请参见上面的实现细节。

💡 请注意，ScreenSuite 并非旨在精确复现业界发布的基准测试：我们基于**视觉**评估模型的 **GUI Agent 能力**。因此，在像 Mind2Web 这样的基准测试中，其他基准测试会为 Agent 提供信息丰富的上下文视图（如 DOM 或可访问性树），而我们的评估设置要困难得多，因此 ScreenSuite 的得分与其他来源不匹配。
