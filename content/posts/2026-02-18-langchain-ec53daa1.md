今天，我们正在扩展 LangSmith Agent Builder 的功能。这是一次围绕一个简单理念构建的重大更新：与 Agent（智能体）协作，应如同与队友协作一般。

我们基于这一理念重建了 Agent Builder。现在，有一个始终可用的 Agent（“聊天”），您可以用它来运行临时任务或创建专门的 Agent。我们还增加了 Agent 使用上传文件的能力，并明确了您的 Agent 可以使用哪些工具。

免费试用 Agent Builder。

### 以下是最新功能

- **一个拥有您所有工具的 Agent**：一个中心的“聊天”Agent 现在可以访问您工作空间中的每一个工具，因此您可以提问和执行操作，而无需先创建一个专门的 Agent。
- **将任何聊天转化为 Agent**：通过对话方式处理任务，然后一键将其转化为可重复使用的 Agent。
- **上传文件**：直接将 CSV、图像和文本文件添加到聊天中，用于数据分析、图像处理，或为您的 Agent 提供参考资料。
- **集中管理工具**：在统一的工具注册表中查看、验证和添加工具，并配有用于管理的管理员控制。

## 一个拥有您所有工具的 Agent

Agent Builder 聊天可以访问您连接到工作空间的每一个工具，例如 Slack、Gmail、Linear、Pylon 以及您通过远程 MCP 服务器连接的任何其他工具。这意味着您可以提出诸如“我有哪些未解决的 Linear 工单？”或“总结 #support 频道中今天的请求”这样的问题，而无需先设置一个专门的 Agent。

输入一个问题或请求，然后 Agent Builder 会制定计划、调用正确的工具，并逐步完成。如果某些操作需要您的批准，例如发送消息或创建工单，它会在采取行动前征求您的意见。

Agent Builder 还可以根据您的需求串联使用多个工具。如果您正在处理电子邮件，可以要求它快速总结最近 10 条消息。然后要求它为任何会议请求创建日历事件。Agent Builder 会根据需要调用新的工具。

您可以同时运行多个聊天。在一个对话中启动一项任务，打开一个新线程，处理其他事情。每个聊天都是独立运行的，因此您可以随时返回任何一个聊天查看进度、批准操作或提出后续问题。

## 将任何聊天转化为 Agent

这个中心的聊天 Agent 提供了一种强大的新方式来创建 Agent。您可以从一个问题开始，例如“创建一份本周来自 Pylon 的支持工单摘要，并为我的团队起草一份 Slack 更新。” Agent Builder 会处理该任务，您可以通过反馈来完善输出。

在任何时候，您都可以选择“将此对话转化为可重复使用的 Agent”，从而为该任务创建一个 Agent。

就这样。您就拥有了一个可以为您持续处理事务的循环 Agent。无需进行提示词工程，也无需处理 if/then 逻辑。您与 Agent Builder 的对话就是设置过程。并且每次对话都会被保存以供将来参考，因此您将来总是可以回到某个任务并将其转化为 Agent，即使您最初没有想到这样做。

我们发现这种模式特别适用于：

- **研究**：“从网上查找关于我前 10 大客户的最新消息，并在 Slack 中向我发送摘要。”
- **写作**：“审阅我们公司博客上的最近 20 篇帖子，并制定一份语音和语调指南作为 Google 文档。”
- **摘要**：“提取本周的 Linear 问题，分析趋势，并起草一份摘要。”
- **沟通**：“对于 HubSpot 中的每个新潜在客户，研究其公司并起草一封个性化的外联邮件。”

对于您创建的任何 Agent，您可以手动触发它们、为它们设置计划，或者让它们响应外部事件，例如 Slack 消息或电子邮件。并且因为它们始于与您的真实对话，所以它们已经知道好的输出应该是什么样子。

## 向任何对话上传文件

在此之前，Agent Builder 只能处理来自您已连接工具的数据以及您在聊天中编写的内容。现在，您可以直接上传文件，例如 CSV、截图或风格指南，您的 Agent 可以基于这些文件进行操作。上传文件可实现新的功能，例如：

**即时分析数据**。上传一个 CSV 文件，例如上个季度的销售数据，并要求 Agent Builder 查找趋势并在 Slack 中向您的团队发送报告。

**处理图像**。上传一张截图或照片，并要求 Agent Builder 使用它，例如将白板笔记的照片转换为 Google 文档。

**导入文档和参考资料**。上传一份写作风格指南或现有的提示词，让您的 Agent 有一个良好的开端。这在构建新 Agent 时特别有用。

## 集中管理您的工具

我们还让您能更轻松地在一个地方查看和管理所有工具。

从您的工作空间设置中，您可以查看所有已连接的工具、验证新工具以及添加远程 MCP 服务器。如果某个工具需要重新验证，您会立即看到。当您连接一个新工具时，它会立即对所有 Agent 可用。

只有工作空间管理员可以添加新工具，因此您的团队可以控制 Agent 可以访问哪些工具。
