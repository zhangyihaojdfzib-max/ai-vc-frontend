您现在可以使用 Vercel Connect 来管理 Chat SDK 机器人的凭证，无需自行存储或轮换 Token 或签名密钥。

新的 @vercel/connect/chat 子路径为 Slack、GitHub 和 Linear 提供了适配器辅助函数。每个辅助函数接收一个连接器 UID，并返回一个配置对象，您可以将其展开到匹配的适配器工厂中。

```
1import { createSlackAdapter } from "@chat-adapter/slack";2import { connectSlackAdapter } from "@vercel/connect/chat";3
4export const adapter = createSlackAdapter({5  ...connectSlackAdapter("slack/acme-slack"),6});
```

使用 Vercel Connect SDK 中的 connectSlackAdapter 配置您的 Slack 适配器

出站机器人调用使用由 getToken 支持的函数形式 Token 字段，因此每个 API 请求都会获得一个由 Vercel Connect 为您轮换的、新鲜的短期 Token。

入站触发器使用 webhookVerifier 来验证 Connect 附加到转发事件上的 OIDC Token，从而消除了对 signingSecret 的需求。

阅读文档以使用 Chat SDK 设置您的第一个连接器，或按照知识库指南使用 Vercel Connect 构建您自己的 Slack 机器人。
