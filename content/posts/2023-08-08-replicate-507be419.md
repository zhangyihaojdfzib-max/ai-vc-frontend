## 什么是微调？

微调是一个过程，它利用预训练模型并用更多数据进行训练，从而创建一个更适合特定任务的新模型。您可以对像 SDXL 这样的图像生成模型进行微调，使用您自己的图像来创建一个新版本的模型，使其更擅长生成特定人物、物体或风格的图像。

## 准备您的训练图像

训练 API 需要一个包含您训练图像的 zip 文件。少量图像（5-6 张）就足以对 SDXL 进行针对单个人物的微调，但如果您的训练主题更复杂或图像差异很大，您可能需要更多图像。

请参考 SDXL 仓库中的示例数据集以获得灵感。

在准备训练图像时，请牢记以下指南：

- 图像可以是您自己、您的宠物、您最喜欢的毛绒玩具或任何独特的物体。
- 图像应仅包含主题本身，没有背景噪音或其他物体。
- 未经他人同意，请勿使用他人的图像。
- 图像可以是 JPEG 或 PNG 格式。
- 尺寸和大小无关紧要。
- 文件名无关紧要。

将您的图像放入一个文件夹并压缩成 zip 文件。zip 文件的目录结构无关紧要：

```
zip -r data.zip data
```

## 添加您的 Replicate API Token

在开始训练任务之前，您需要从 replicate.com/account 获取您的 Replicate API Token。在您的 shell 中，将该 Token 存储在一个名为 REPLICATE_API_TOKEN 的环境变量中。

```
export REPLICATE_API_TOKEN=r8_...
```

## 上传您的训练数据

将您的训练数据 zip 文件上传到互联网上某个可公开访问的位置，例如 S3 存储桶或 GitHub Pages 站点。

## 创建一个模型

您还需要在 Replicate 上创建一个模型，该模型将成为训练后 SDXL 版本的目标位置。前往 replicate.com/create 创建模型。在下面的示例中，我们将其命名为 my-name/my-model。

您可以将模型设为公共或私有。如果您的模型是私有的，只有您能运行它。如果您的模型是公共的，任何人都可以运行它，但只有您能更新它。

## 开始训练

现在您已经收集了训练数据并创建了模型，是时候使用 Replicate 的 API 开始训练过程了。

本指南使用 Python，但如果您想使用其他语言，可以使用客户端库或直接调用 HTTP API。

如果您尚未配置 Python 环境，可以使用 Google Colab 上的托管 Jupyter Notebook 来启动训练过程：

首先安装 Replicate Python 包：

```
pip install replicate
```

然后创建一个训练任务：

```
import replicate

training = replicate.trainings.create(
    version="stability-ai/sdxl:39ed52f2a78e934b3ba6e2a89f5b1c712de7dfea535525255b1aa35c5565e08b",
    input={
        "input_images": "https://my-domain/my-input-images.zip",
    },
    destination="my-name/my-model"
)
print(training)
```

input_images 输入参数是必需的，但您也可以设置其他输入。有关完整输入列表，请参阅 SDXL README 中的训练输入。请注意，默认情况下我们将使用 LoRA 进行训练，如果您想使用 Dreambooth，可以将 is_loRA 设置为 false。如果您只想执行文本反转，可以将 lora_lr 设置为 0。

## 使用 Dreambooth + LoRA 对人脸数据集进行微调

如果您对人脸进行微调，默认的训练参数效果会很好，但您也可以使用 use_face_detection_instead 设置。这将自动使用人脸分割，使训练仅聚焦于图像中的人脸。

```
import replicate

training = replicate.trainings.create(
    version="stability-ai/sdxl:39ed52f2a78e934b3ba6e2a89f5b1c712de7dfea535525255b1aa35c5565e08b",
    input={
        "input_images": "https://my-domain/face-images.zip",
        "use_face_detection_instead": True,
    },
    destination="my-name/my-model"
)
```

## 对风格进行微调

要获得最佳风格效果，您需要：

- 提高 LoRA 学习率，这可以防止训练过于关注细节。尝试不同的值，如 1e-4、2e-4。我们的芭比微调使用了 4e-4。
- 使用不同的 caption_prefix 来指代一种风格

```
import replicate

training = replicate.trainings.create(
    version="stability-ai/sdxl:39ed52f2a78e934b3ba6e2a89f5b1c712de7dfea535525255b1aa35c5565e08b",
    input={
        "input_images": "https://my-domain/style-images.zip",
        "lora_lr": 2e-4,
        "caption_prefix": 'In the style of TOK,',
    },
    destination="my-name/my-model"
)
```

## 微调的工作原理

在微调开始之前，输入图像会经过多个模型的预处理：

- SwinIR 将输入图像放大到更高分辨率。
- BLIP 为每个输入图像生成文本描述。
- CLIPSeg 移除图像中对训练无趣或无帮助的区域。

完整的训练参数列表可在 SDXL 模型 README 中找到。

## 进阶：将微调后的模型与 Diffusers 结合使用

如果您直接使用 diffusers 库来构建自定义管道，您可以使用在 Replicate 上训练的模型的权重。

training 对象的 .output 字段包含指向训练后版本的指针以及训练权重的 URL：

```
print(training.output)
# {
#   'version': 'cloneofsimo/sdxl_mixes:...',
#   'weights': 'https://pbxt.replicate.delivery/.../trained_model.tar'
# }
```

下载 weights 字段中的 .tar 文件并解压。现在您可以使用 diffusers 包加载权重。

```
import torch
from diffusers import DiffusionPipeline
```

```python
from safetensors import safe_open
from dataset_and_utils import TokenEmbeddingsHandler
from diffusers.models import AutoencoderKL

pipe = DiffusionPipeline.from_pretrained(
        "stabilityai/stable-diffusion-xl-base-1.0",
        torch_dtype=torch.float16,
        variant="fp16",
).to("cuda")

with safe_open("training_out/unet.safetensors", framework="pt", device="cuda") as f:
    for key in f.keys():
       tensors[key] = f.get_tensor(key)

pipe.unet.load_state_dict(tensors, strict=False) # 应耗时 < 2 秒

text_encoders = [pipe.text_encoder, pipe.text_encoder_2]
tokenizers = [pipe.tokenizer, pipe.tokenizer_2]

embhandler = TokenEmbeddingsHandler(text_encoders, tokenizers)
embhandler.load_embeddings("training_out/embeddings.pti")
```

根据 `special_params.json` 中的提示词对模型进行提示，生成输出结果。

pipe(prompt="A photo of <s0><s1>").images[0].save("monster.png")

## 进阶：用自定义提示词替换训练时生成的提示词

对于大多数用户而言，BLIP 为训练生成的描述文字效果已经足够好。但您也可以通过向训练用的输入图像 zip 文件中添加 `caption.csv` 文件来提供自定义描述文字。每张输入图像都需要有对应的描述文字。关于格式的具体要求，请参考示例 csv 文件。
