Chat SDK 现在通过新的 `vendor-official` 适配器支持 Novu。

一个处理器集即可让你的 Agent（智能体）接入 Slack、Microsoft Teams、WhatsApp、Telegram 和电子邮件。Novu 负责处理凭证、身份和投递，将 OAuth 和 Token 保留在你的应用之外，并将每个渠道映射到同一个用户。你的 Agent（智能体）始终知道自己在与谁对话。

你的 Agent（智能体）还可以发送主动通知，并在同一个循环中处理回复，无论客户使用哪个渠道。

```
1import { Chat } from "chat";
2import { createMemoryState } from "@chat-adapter/state-memory";
3import { createNovuAdapter } from "@novu/chat-sdk-adapter";
4
5const novu = createNovuAdapter();
6
7const chat = new Chat({
8  userName: "support",
9  adapters: { novu },
10  state: createMemoryState(),
11});
12
13chat.onNewMention(async (thread, message) => {
14  await thread.post(`Hi! You said: ${message.text}`);
15});
16
17chat.onSubscribedMessage(async (thread, message) => {
18  await thread.post(`echo: ${message.text}`);
19});
20
21await chat.initialize();
```

配置适配器并将其连接到 Novu 系统

一条 CLI 命令即可将真实渠道连接到你的 Agent（智能体）：

```
npx novu connect --runtime chat-sdk
```

它会让你登录，创建一个桥接 Agent（智能体），设置你选择的渠道（例如 Slack），并将凭证添加到你的项目中。

阅读 Novu 文档以开始使用。
