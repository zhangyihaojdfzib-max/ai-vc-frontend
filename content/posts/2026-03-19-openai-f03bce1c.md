我们与首席执行官兼联合创始人 Chaz Englander 探讨了金融机构如何演变，以及近期的AI进展如何自动化并简化其运营。

#### 您第一次有意义的AI接触是什么？它如何影响了 Model ML 的创立？

在我们出售上一家公司后，我和我的兄弟意识到我们并不喜欢投资，但却痴迷于通过 GPT 驱动的函数调用来实现投资流程的自动化。

我们是一个六人的家族办公室，但借助这些由 GPT‑3.5 驱动的LLM（大语言模型），感觉我们拥有了一个60人团队的杠杆效应。

我们为自己构建了 Model ML 的原型，当时并未计划将其商业化。但一旦我们看到通过自动化研究工作流所带来的洞察力提升和效率增益，我们就知道我们发现了有价值的东西。

#### 您观察到金融服务公司内部正在发生哪些变化？

历史上需要数天、数周甚至数月才能完成的任务，其中一些现在可以在几分钟或几小时内完成。例如，准备季度收益总结过去需要数小时。现在，Agent（智能体）处理整个流程：它们提取数据、格式化幻灯片，并将 PowerPoint 发布到 SharePoint，全程无需人工干预。我认为这将是我们在今年看到的最大转变……你早上来上班时，你的工作已经在那里了。

这迫使人们重新思考人类在何处创造价值，以及公司需要如何重新规划团队在当下和未来产生影响力的领域。

我们看到公司正在将人员转移到更高价值、基于判断的角色中。至少在我们看来，与我们合作的公司的领导者，正是那些以AI原生方式重新思考整个组织架构的人。这极其困难，这就是为什么我们早期会充当顾问，帮助他们确定AI在当下最适用的领域，并为他们未来12个月我们认为最具影响力的领域做好准备。

我们看到金融机构内部的人员现在影响力更大了，而不是更小了。随着自动化承担了繁重的工作，人们可以专注于关系和战略思考。那些获胜的公司将是那些重新思考其整个运营结构以利用这一转变的公司。

#### 与通用AI工具相比，Model ML 如何脱颖而出？新的模型能力如何使您的客户受益？

在金融领域，准确性、合规性和工作流适配性不是可选项——它们是入场券。通用工具的不足恰恰在于这种特定性。Model ML 从第一天起就在两个关键层面为金融服务专门构建。

首先，在Agent（智能体）层，我们构建并微调了专门用于解析和交互金融专业人士日常使用的各类数据的系统，包括结构化和非结构化数据，涵盖 Sharepoint 等工具，以及 Capital IQ、FactSet 和 Crunchbase 等常见数据集，这些数据集可能包含数百个表和20TB的数据。12个月前，在这些数据集之上构建一个Agent（智能体）几乎是不可能的。这些不仅仅是回答问题的模型；它们具有上下文感知能力，理解数据模式，能编写代码，并能从TB级的复杂数据中检索信息。

其次，是应用层：用户与Agent（智能体）交互的界面，专为金融领域设计。它为公司提供工具来构建Agent（智能体），以自动化端到端工作流程，并实现以前无法完成的分析。在使用案例方面，我们每天看到数十个新用例，目前已有数千个用例，其中许多是客户在注册后即可开箱即用的。

我们看到每次新模型发布都带来了显著的阶段性变化，我们已将这些变化转化为客户的即时利益。推理和编码能力等方面的进步，使我们产品的某些领域达到了极高的水平。最近随着 OpenAI o3‑pro、o3、o4‑mini 和 GPT‑4.1 模型的发布，这些新模型在推理、多模态能力、指令遵循和工具集成方面带来了巨大改进。凭借更大的上下文窗口和更先进的推理能力，我们现在能够解锁端到端工作流程。现在，用户可以将数据收集、分析和演示文稿创建任务串联起来，完全自主地生成格式完整的输出。

#### 展望未来12个月，您认为变化最大的是什么？

我认为未来最深刻的转变是端到端工作流自动化的兴起，届时你的系统将充当控制塔，监督着一支数字工作者大军。随着这些Agent（智能体）在你的整个数字世界中承担更复杂、多步骤的任务，甚至用户界面和我们与硬件交互的方式也将开始改变。这可能超出了未来12个月的范围，但这是我们前进的方向。

接下来将出现的是你可以在我们产品中构建的真正自主的Agent（智能体）。我们的Agent（智能体）能够执行复杂的工作流，从你的CRM、电子邮件、文件、外部数据供应商、会议记录等处收集、分析和呈现数据。这些Agent（智能体）不会只是等待指令；它们将预判需要完成的任务，无论是周期性的（每日、每周、每月、每季度、每年），还是由现实世界事件触发的——就像你在会议后或回复电子邮件时会要求团队成员做的那样。

真正的转变在于，这些工作流将端到端地自动运行，并在你所有系统之间进行深度推理和编排。输出成果可能像一份100页的 PowerPoint 那样庞大，完全由机器构建——更快、更一致，并且7x24小时可用。

这就是未来：自主的数字团队运行着驱动你业务的工作流——更好、更快、始终在线。

#### 在AI发展如此迅速的情况下，您如何保持团队的敏捷性？

我们相信，AI原生公司在结构上会有所不同。层级更少，周期更快，反馈循环更紧密。我们采用了扁平化结构。Arnie [我的联合创始人] 和我每人都有两位数的直接下属。这听起来可能很夸张，但AI使其变得可管理。所有一对一会议都由AI辅助。笔记、待办事项、背景信息，一切都得到了简化。这让我们行动更快，并更贴近产品。我们认为现代公司就是这样运行的：更像控制塔，而不是孤立的层级结构。

保持敏捷的一部分在于押注生态系统和基础模型会变得更好。真正的关键——也许这是创始人思维和工程组织的一部分——就是不要对你的代码感情用事。我们过去什么都自己构建：Agent（智能体）抽象、服务连接器，所有一切。现在，如果 OpenAI 或开源社区发布了更好的东西——比如 OpenAI 的 Agent SDK 或 MCP 连接器——我们只需将其接入并删除我们的代码。

我们并非试图通过维护基础设施来取胜；我们试图通过为客户创造价值成果来取胜。
