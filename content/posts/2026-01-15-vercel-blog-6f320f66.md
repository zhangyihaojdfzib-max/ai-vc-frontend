AWS 数据库现已登陆 Vercel 应用市场和 v0。

从今天起，开发者可以直接通过 Vercel 仪表板，以引导式设置流程和自动项目配置，快速部署 Aurora PostgreSQL、Amazon DynamoDB 和 Aurora DSQL。v0 在构建项目时，还能创建并连接 AWS 账户和数据库。

AWS 数据库支撑着全球许多大型应用程序的运行。此次发布将 AWS 数据基础设施融入 Vercel 开发者体验，让您无需手动设置 IAM、配置凭证或在不同控制台间切换，即可搭建生产级数据库。新的 AWS 团队可**通过免费入门计划和 100 美元赠金开始使用**。

## 在几秒钟内开始使用 v0

在构建时告诉 v0 使用 AWS 数据库，它就会自动部署数据库、创建或链接您的 AWS 账户，并配置连接。您只需几分钟就能从提示词开始，构建出带有生产级数据库的可用应用。

要在 v0 上开始，可以尝试使用以下提示词来查看实际效果：

> 为我构建一个内部员工仪表板，用于跟踪承包商工时，并使用 AWS 数据库

这对两个平台来说都是一个里程碑。现在，更多构建者可以使用 AWS 基础设施，包括设计全栈应用原型的设计师、验证想法的创始人，以及希望无需学习 IAM 策略即可获得生产级数据库的开发者。您只需描述要构建的内容，v0 就会处理 AWS 设置。

## 通过应用市场连接 AWS 数据库

对于现有项目或希望对部署有更多控制权的团队，您可以直接从 Vercel 应用市场添加 AWS 数据库。根据您是否拥有现有 AWS 账户，有两种路径可选。

### 新的 AWS 团队：创建 AWS 账户

如果您还没有 AWS 账户或偏好简化设置，可以直接从 Vercel 仪表板一键部署数据库。选择 **Aurora PostgreSQL**、**Amazon DynamoDB** 或 **Aurora DSQL**，选择区域和套餐，Vercel 将处理其余事宜。

此流程非常适合希望获得完全托管体验而无需接触 AWS 控制台的团队。

1.  导航至 **Vercel 应用市场 → AWS**
2.  选择 **创建新的**（Vercel 托管账户）
3.  选择 **Aurora PostgreSQL**、**Aurora DSQL** 或 **Amazon DynamoDB**
4.  选择您的 AWS 区域和套餐
5.  将资源连接到您的项目。Vercel 会自动注入环境变量和配置

### 现有 AWS 团队：链接您的 AWS 账户

如果您的团队已经管理着 AWS 基础设施，您可以使用 **AWS IAM 临时委托权限** 连接您自己的账户。这使您可以在 Vercel 上创建 AWS 资源并将其连接到您的项目，同时保留现有的 AWS 设置。

要使用现有 AWS 账户安装数据库：

2.  选择 **链接现有 AWS 账户**
3.  向 Vercel 授予临时权限并点击 **允许访问**
4.  您将被重定向回 Vercel
5.  选择 **Aurora PostgreSQL**、**Aurora DSQL** 或 **Amazon DynamoDB**
6.  选择您的 AWS 区域和套餐，然后创建数据库
7.  将资源连接到您的项目。Vercel 会自动注入环境变量和配置

## 当前 AWS 集成可实现的功能

无论您使用 v0 还是应用市场，该集成都能让您：

*   在任何受支持的 AWS 区域部署 **Aurora PostgreSQL**、**Aurora DSQL** 或 **Amazon DynamoDB**
*   将连接凭证作为环境变量注入到您的 Vercel 项目中
*   从 Vercel 管理资源生命周期，包括销毁
*   需要深度配置时，在 AWS 控制台中打开资源
*   无需手动设置即可使用 AWS 原生的扩展性和持久性功能

您的数据库会像其他资源一样显示在“存储”选项卡中。这为您提供了跨提供商的应用数据层的统一视图。
