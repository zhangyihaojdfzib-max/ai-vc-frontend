# 推出 Claude 4

今天，我们推出下一代 Claude 模型：**Claude Opus 4** 和 **Claude Sonnet 4**，为编码、高级推理和 AI Agent（智能体）树立了新标准。

Claude Opus 4 是全球最佳的编码模型，在复杂、长期运行的任务和 Agent（智能体）工作流中表现出持续稳定的性能。Claude Sonnet 4 是对 Claude Sonnet 3.7 的重大升级，在提供卓越编码和推理能力的同时，能更精确地响应您的指令。

除了模型，我们还宣布：

- **支持工具使用的扩展思考（测试版）**：两款模型在扩展思考过程中都能使用工具——例如**网络搜索**——这使得 Claude 能够在推理和使用工具之间交替进行，从而改进回答。
- **新的模型能力**：两款模型都能并行使用工具，更精确地遵循指令，并且——当开发者授予其本地文件访问权限时——展现出显著提升的记忆能力，能够提取并保存关键事实以保持连续性，并随时间积累隐性知识。
- **Claude Code 现已全面开放**：在研究预览期间获得广泛积极反馈后，我们正在扩展开发者与 Claude 协作的方式。Claude Code 现在支持通过 GitHub Actions 处理后台任务，并与 VS Code 和 JetBrains 实现原生集成，直接在您的文件中显示编辑内容，实现无缝结对编程。
- **新的 API 能力**：我们将在 API 上发布**四项新能力**，使开发者能够构建更强大的 AI Agent（智能体）：代码执行工具、MCP 连接器、Files API，以及可将提示词缓存长达一小时的能力。

Claude Opus 4 和 Sonnet 4 是混合模型，提供两种模式：近即时响应和用于深度推理的扩展思考。Pro、Max、Team 和 Enterprise 版 Claude 计划均包含这两款模型和扩展思考功能，其中 Sonnet 4 也向免费用户开放。两款模型均可在我们的 API、Amazon Bedrock 和 Google Cloud 的 Vertex AI 上使用。定价与之前的 Opus 和 Sonnet 模型保持一致：Opus 4 为每百万 Token 15/75 美元（输入/输出），Sonnet 4 为每百万 Token 3/15 美元。

## Claude 4

Claude Opus 4 是我们迄今为止最强大的模型，也是全球最佳的编码模型，在 SWE-bench（72.5%）和 Terminal-bench（43.2%）上领先。它在需要集中精力和数千个步骤的长期运行任务上提供持续稳定的性能，能够连续工作数小时——显著优于所有 Sonnet 模型，并极大地扩展了 AI Agent（智能体）所能完成的任务范围。

Claude Opus 4 擅长编码和解决复杂问题，为前沿的 Agent（智能体）产品提供动力。**Cursor** 称其为编码领域的尖端技术，在复杂代码库理解方面实现了飞跃。**Replit** 报告称其在跨多个文件的复杂修改方面精度更高，取得了显著进步。**Block** 称其是首个在其 Agent（智能体）**codename goose** 中，在编辑和调试过程中提升代码质量，同时保持完全性能和可靠性的模型。**Rakuten** 通过一个要求苛刻的开源重构任务验证了其能力，该任务独立运行了 7 小时且性能稳定。**Cognition** 指出 Opus 4 擅长解决其他模型无法应对的复杂挑战，成功处理了先前模型遗漏的关键操作。

Claude Sonnet 4 在 Sonnet 3.7 行业领先能力的基础上实现了显著改进，在编码方面表现出色，在 SWE-bench 上达到了 72.7% 的尖端水平。该模型在性能和效率之间取得了平衡，适用于内部和外部用例，并增强了可操控性，以便更好地控制实现方式。虽然在大多数领域不及 Opus 4，但它提供了能力和实用性的最佳组合。

**GitHub** 表示 Claude Sonnet 4 在 Agent（智能体）场景中表现出色，将作为 GitHub Copilot 中新编码 Agent（智能体）的驱动模型引入。**Manus** 强调了其在遵循复杂指令、清晰推理和美观输出方面的改进。**iGent** 报告称 Sonnet 4 擅长自主进行多功能应用开发，并且在问题解决和代码库导航方面有实质性改进——将导航错误率从 20% 降低到接近零。**Sourcegraph** 表示该模型有望成为软件开发领域的重大飞跃——能够更长时间地保持正轨、更深入地理解问题并提供更优雅的代码质量。**Augment Code** 报告了更高的成功率、更精准的代码编辑以及更细致地处理复杂任务的能力，使其成为其首选主模型。

## 模型改进

除了支持工具使用的扩展思考、并行工具执行和记忆改进外，我们还显著减少了模型使用捷径或漏洞来完成任务的行为。在特别容易受到捷径和漏洞影响的 Agent（智能体）任务上，这两款模型出现此类行为的可能性比 Sonnet 3.7 低 65%。

Claude Opus 4 在记忆能力方面也显著优于所有先前模型。当开发者构建为 Claude 提供本地文件访问权限的应用程序时，Opus 4 擅长创建和维护“记忆文件”来存储关键信息。这解锁了更好的长期任务感知、连贯性和在 Agent（智能体）任务上的表现——例如 Opus 4 在玩《宝可梦》时创建了一个“导航指南”。

最后，我们为 Claude 4 模型引入了思考摘要功能，使用一个较小的模型来压缩冗长的思考过程。这种摘要功能大约只在 5% 的情况下需要——大多数思考过程足够短，可以完整显示。需要原始思维链进行高级提示词工程的用户可以**联系销售**了解我们的新开发者模式，以保留完全访问权限。

#### 性能基准报告

Claude Opus 4 和 Sonnet 4 是混合推理模型。本博文中报告的基准测试显示了使用或不使用扩展思考时获得的最高分数。我们在下面为每个结果注明了是否使用了扩展思考：

- **无扩展思维**：SWE-bench Verified、Terminal-bench
- **扩展思维（最高64K Token）**：
    - TAU-bench（未报告无扩展思维的结果）
    - GPQA Diamond（无扩展思维：Opus 4得分74.9%，Sonnet 4得分70.0%）
    - MMMLU（无扩展思维：Opus 4得分87.4%，Sonnet 4得分85.4%）
    - MMMU（无扩展思维：Opus 4得分73.7%，Sonnet 4得分72.6%）
    - AIME（无扩展思维：Opus 4得分33.9%，Sonnet 4得分33.1%）

#### TAU-bench 方法

得分是通过在航空和零售Agent策略的提示词附录中添加指令实现的，该指令引导Claude在使用扩展思维结合工具时更好地发挥其推理能力。在多轮任务轨迹中，我们鼓励模型在解决问题时写下其思考过程，这有别于我们通常的思维模式，以最大限度地发挥其推理能力。为了适应Claude因进行更多思考而产生的额外步骤，最大步骤数（按模型完成次数计算）从30步增加到了100步（大多数任务轨迹在30步内完成，仅有一个轨迹超过了50步）。

#### SWE-bench 方法

对于Claude 4系列模型，我们继续使用相同的简单框架，该框架仅为模型配备我们先前发布中描述的两个工具——一个bash工具和一个通过字符串替换操作的文件编辑工具。我们不再包含Claude 3.7 Sonnet使用的第三个“规划工具”。在所有Claude 4模型上，我们报告的是全部500个问题的得分。OpenAI模型的得分报告基于477个问题的子集。

对于我们的“高计算量”数据，我们采用了额外的复杂性和并行测试时计算，具体如下：

- 我们采样多个并行尝试。
- 我们丢弃那些破坏了仓库中可见回归测试的补丁，类似于Agentless（Xia等人，2024）采用的拒绝采样方法；注意，未使用任何隐藏的测试信息。
- 然后，我们使用一个内部评分模型从剩余的尝试中选择最佳候选方案。

这导致Opus 4和Sonnet 4的得分分别为79.4%和80.2%。
