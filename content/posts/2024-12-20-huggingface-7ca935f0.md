# 使用 Big Bench Audio 评估音频推理能力

原生语音到语音模型的出现为提升语音 Agent（智能体）能力并简化语音工作流程提供了令人兴奋的机遇。然而，评估这种简化是否以模型性能为代价或引入其他权衡因素至关重要。

为支持这一分析，Artificial Analysis 发布了 **Big Bench Audio**，这是一个用于评估音频语言模型推理能力的新评估数据集。该数据集将 **Big Bench Hard** 中的问题（因其对高级推理的严格测试而被选中）改编到音频领域。

本文介绍了 Big Bench Audio 数据集以及 GPT-4o 和 Gemini 1.5 系列模型的初步基准测试结果。我们的分析从多种模态考察了这些模型：原生语音到语音、语音到文本、文本到语音和文本到文本。我们在下方以及 Artificial Analysis 网站新增的语音到语音页面上展示了结果摘要。我们的初步结果显示存在显著的"语音推理差距"：GPT-4o 在纯文本版本数据集上达到 92% 的准确率，但其语音到语音性能降至 66%。

## Big Bench Audio 数据集

**Big Bench Audio** 包含从 Big Bench Hard 四个类别中选出的 **1,000 个音频问题**，每个类别因其适合音频评估而被选中：

- **形式谬误**：评估给定陈述的逻辑推理
- **导航**：判断导航步骤是否返回起点
- **物体计数**：统计集合中的特定物品数量
- **谎言之网**：评估以自然语言表达的布尔逻辑

每个类别贡献 250 个问题，形成一个平衡的数据集，避免过度依赖视觉元素或口头表达时可能产生歧义的文本。

数据集中的每个问题结构如下：

```json
{
  "category": "formal_fallacies",
  "official_answer": "invalid",
  "file_name": "data/question_0.mp3",
  "id": 0
}
```

音频文件使用来自 Artificial Analysis Speech Arena 排名靠前的文本到语音模型的 **23 种合成语音** 生成。每次音频生成均通过 Levenshtein 距离与转录文本进行严格验证，边缘案例则经过人工审核。要了解更多关于数据集创建方式的信息，请查看数据集卡片。

3. **文本到语音**：提供问题的文本版本，模型生成包含答案的输出音频文件。

### 评估方法

为确保所有配置下评估的一致性和可扩展性，我们开发了一个使用 LLM 评估器的自动化评估系统。其工作原理如下：

1. 对于音频响应，我们首先使用 OpenAI 的 Whisper API 将其转录为文本
2. 对于文本响应，我们直接将其用作"候选答案"
3. LLM 评估器接收：候选答案、官方答案、原始问题（作为上下文）

- 候选答案
- 官方答案
- 原始问题（作为上下文）

LLM 评估器接收候选答案、官方答案和原始问题作为上下文，并被提示将候选答案标记为 **正确** 或 **错误**。我们使用 Anthropic 的 Claude 3.5 Sonnet（2024 年 10 月版）作为 Artificial Analysis 上列出的 Big Bench Audio 分数的 LLM 评估器。

LLM 评估器提示词：

```plaintext
评估以下候选答案是正确还是错误。
候选答案正确必须与官方答案一致。
如果候选答案自相矛盾，评估首次提出的答案。
如果候选答案提供了最终答案和计算过程，仅评估最终答案。
如果候选答案包含无关信息，仅评估相关信息。
如果候选答案包含数值，拼写形式（如 7 或 seven）均可接受。
如果候选答案涉及人名拼写错误（如 Leda 或 Lida，Autry 或 Audrie）也可接受。

问题，仅供参考：开始问题 {question} \n\n结束问题

官方答案：{official_answer}

开始评估候选答案

{candidate_answer}

结束评估候选答案

仅回复 CORRECT 或 INCORRECT。
```

### 音频推理差距

我们的分析揭示了文本推理与音频推理之间存在显著的性能差距。GPT-4o（2024年8月版）在数据集的文本到文本版本中达到了92%的准确率，而其语音到语音版本（GPT-4o实时预览版2024年10月）得分仅为66%。文本到语音配置的中间性能为74%，这表明语音输入和语音输出共同导致了这一性能差距。

### 当前语音到语音流水线在推理方面优于原生音频

传统的流水线方法（使用Whisper进行转录、GPT-4o（2024年8月版）进行推理、TTS-1进行语音生成）与纯文本处理相比，性能下降极小。这表明，在推理准确性至关重要的应用中，流水线方法目前提供了性能与音频能力的最佳平衡。

我们预计这一差距可能会随时间缩小，并将继续使用Big Bench Audio测试新的语音到语音模型。敬请期待即将推出的Google Gemini 2.0 Flash语音到语音模式更新！
