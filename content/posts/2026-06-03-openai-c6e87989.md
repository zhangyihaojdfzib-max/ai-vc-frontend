# Wasmer 如何利用 Codex 构建边缘端 Node.js 运行时

团队借助 Codex 与 GPT‑5.5 承接更具雄心的项目，并将完成速度提升 10 至 20 倍。

10 至 20 倍

开发速度提升

Wasmer 的工程师今年取得了一项突破：他们找到了在 WebAssembly 沙箱中运行 Node.js 工作负载的方法，使开发者无需 Docker 即可运行 JavaScript 应用、MCP 和 Agent（智能体）。如果没有 Codex，这项工作需要一年时间，但借助 Codex，仅用了两周。如今，他们成为首家在边缘层提供完整 Node.js 支持的云主机。

## 以焕然一新的雄心进行构建

Wasmer 是一家年轻的公司，团队规模小，目标却很大：打造一个能够跨本地和全球环境扩展、且不受平台限制的边缘计算平台。

“我们承接的项目极具挑战性，如果没有 AI，完成它们需要耗费大量时间，”创始人兼 CEO Syrus Akbary Nieto 表示，“现在，事情变得简单多了，速度也快多了。”

这种简洁与速度直接转化成了一个开创性项目：Edge.js⁠（在新窗口打开），这是一个能够在 AI 和边缘计算场景下运行 Node.js 工作负载的 JavaScript 运行时。

这是团队长久以来想要开展的项目，但他们此前并不总是拥有足够的资源。“这里的每个人技术能力都非常强，但我们就是没有时间投入到这些项目上。而现在，我们可以推出以前不可能实现的新产品，”Nieto 解释道。

借助 Codex，Wasmer 的工程师能够成倍放大他们的努力。“我们的开发速度至少提升了 10 到 20 倍，”Nieto 说。

## 跨语言与代码层级的推理能力

与许多团队一样，Wasmer 的工程师最初对 AI 持怀疑态度。“一开始我们并不太信任 AI 的输出，”Nieto 解释道。但当团队开始尝试后，结果超出了他们的预期。“在过去一年，尤其是最近几个月，我们一直在使用 Codex，效果真的非常非常好。”

随着 Codex 推理能力的提升，Wasmer 工程师需要手把手指导的时间越来越少。“我们实际上正在逐渐脱离 IDE 本身，不再频繁地直接触碰代码。我们只是引导它朝着我们想要的方向前进，”Nieto 说。

团队从项目伊始到最终完成，全程使用 Codex，从构建最初的架构模块，到打磨最终产品。在整个过程中，Codex 帮助团队发现 Bug 并定位其根本原因。

“有些 Bug 我们根本没想到会出现，而一旦我们开始发现它们，Codex 就直接进入了调试状态，”Nieto 说，“令我们印象深刻的是，它从调试到找到根本原因并确定解决方案的速度之快。”

Nieto 解释说，通常情况下，开发者需要具备特殊专业知识才能发现这些 Bug。但 Codex，“能够熟练运用控制台日志来追踪调用，以及像 LLD 这样的底层调试器，后者可以访问汇编级别的内容。Codex 能够深入到非常底层的层面，看清代码之下发生的事情。”

## 将不可能变为现实

Nieto 解释说，在 Edge.js 之前，在边缘端运行 JavaScript 运行时是不可能的。而在 Codex 出现之前，Wasmer 团队也不可能在如此短的时间内承接如此雄心勃勃的项目。

“Codex 让一家小公司实现了以前只有大公司才能做到的事情。没有它，这个项目根本不可能完成，”Nieto 说。

随着雄心被激发，Wasmer 团队正着眼于更宏大的项目。“现在，我们手中掌握着以前不可能实现的技术。我们需要去解决更具挑战性的问题，”Nieto 说。
