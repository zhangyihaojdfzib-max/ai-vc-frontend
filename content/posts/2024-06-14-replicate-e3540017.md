# 部署自定义版本的 Stable Diffusion 3

-   zeke

当 Stable Diffusion 的第一个版本于 2022 年 8 月问世时，围绕它爆发了一场创新热潮。在其发布后的几天内，开源社区就创建了许多引人注目的衍生模型，例如生成平铺图像的**材质扩散模型**、提供 AI 画笔的**修复模型**，以及能在提示词之间进行插值的**动画模型**。

快进近两年，**Stable Diffusion 3 (SD3)** 于本周刚刚发布。人们对这个新模型的能力充满期待，我们预计会看到大量与 Stable Diffusion 早期阶段类似的开源创新。

在这篇文章中，我们将向您展示如何在 Replicate 上运行您自己的自定义版本 Stable Diffusion 3。

## 先决条件

要遵循本指南，您需要：

*   **Git**：您需要 Git 来克隆 `cog-stable-diffusion-3` 仓库。
*   **Docker**：您将使用 Cog 命令行工具来构建和推送您的模型。Cog 使用 Docker 为您的模型创建一个容器。在运行 Cog 之前，您需要**安装并启动 Docker**。您可以在终端中输入 `docker info` 来确认 Docker 正在运行。
*   **一个 Replicate 账户**。

## 步骤 0：在 Replicate 上创建您的模型

首先，在 Replicate 上为您的模型创建一个页面。访问 `replicate.com/create`，您将看到一个创建模型的表单。

在您的个人账户下创建模型，或者如果您想与团队共享模型访问权限，可以在**组织**下创建。从屏幕左上角选择用户名或组织。

为您的模型选择一个名称，将硬件设置为“GPU A40 Small”，然后点击**创建**。

您的模型默认是私有的，但您可以随时将其公开，即使在创建之后也可以。

## 步骤 1：安装 Cog

接下来，您将使用 **Cog** 来打包 SD3 并将其推送到 Replicate。Cog 是一个开源工具，可以轻松地将机器学习模型放入 Docker 容器中。

运行以下命令安装 Cog：

```
sudo curl -o /usr/local/bin/cog -L https://github.com/replicate/cog/releases/latest/download/cog_`uname -s`_`uname -m`
sudo chmod +x /usr/local/bin/cog
```

## 步骤 2：克隆 SD3 仓库

接下来，使用 Git 克隆 `cog-stable-diffusion-3` 仓库：

```
git clone https://github.com/replicate/cog-stable-diffusion-3
cd cog-stable-diffusion-3
```

此仓库包含一个使用流行的 **Diffusers** Python 库实现的 SD3。

## 步骤 3：推送到 Replicate

现在是时候将您的模型推送到 Replicate 了：

```
cog login
cog push r8.im/<owner-name>/<model-name>
```

所有者名称和模型名称必须与您在上面创建模型时设置的值匹配。

注意：您也可以在 `cog.yaml` 文件中设置 `image` 属性。这允许您在不指定镜像的情况下运行 `cog push`，并且对于阅读您模型源代码的人来说，也能让您在 Replicate 上的模型页面更容易被发现。

## 步骤 4：迭代

现在，您应该已经在 Replicate 上运行着您自己的自定义版本 SD3 了！您可以在 Replicate 网站上或通过 API 运行它。

现在是时候开始对模型进行实验，根据您的需求进行定制了。以下是一些想法：

*   添加对**修复**的支持。
*   添加一个**超分辨率模型**，将生成的图像调整到更高的分辨率。
*   更新**预测接口**，以允许向底层模型输入更多参数。

祝您探索愉快！
