- 人工标注显示，在三分制评分中，评判器与专家的差异最高达到 0.68 MAE。MemAlign 是 MLflow 中的一个开源对齐框架，仅使用约 50 个标注样本就缩小了这一差距。

最近发布的 Genie Code 是 Databricks 专为数据工作打造的自主 AI 伙伴。它取代了 Databricks Assistant，同时整合了多个 Agent（智能体），并提供了新的集成点和能力。Genie Code 与 Unity Catalog 深度集成，这意味着它能够理解你的表、列、血缘关系、指标视图和业务定义（语义）。这种上下文感知能力使 Genie Code 对数据从业者来说比通用聊天机器人有用得多。

当 Genie Code 为传统机器学习任务（例如“构建一个客户流失预测模型”）生成笔记本时，我们期望它能产出一个可用于生产的工作流，包括安装合适的 Python 库、数据探索与预处理、模型训练、调优、注册与部署，以及性能评估。我们还期望每一步都真正基于数据：例如，Genie Code 应理解二分类问题中的类别不平衡会导致截然不同的工作流和成功指标。

为了确保 Genie Code 始终遵循 Databricks 原生最佳实践，并避免例如跳过交叉验证、未能发现数据泄露或不正确的数据插补等问题，我们需要一种严谨的方法来回答一个问题：我们如何知道生成的代码是否真的足够好？生成的笔记本在很大程度上取决于客户试图解决的问题，而不同客户之间差异巨大，因此这是一个非常不简单的问题。

在本文中，我们将介绍如何为 Genie Code 的传统机器学习能力构建一个评估流水线，以及如何使用 MemAlign（MLflow 中一个新的开源对齐框架）来缩小我们发现的大语言模型评判器与人类专家之间的巨大差距。改进后的评判器帮助我们识别并修复了 Genie Code 在机器学习指导方面的缺陷，否则这些缺陷可能会被我们忽略。

## 构建评估框架

一个稳健的评估框架需要满足以下需求：

- **爬山优化**：量化提示词、工具、技能和架构变化对输出的影响。
- **防止回归**：确保改进“模型训练”不会意外地降低“数据探索”的质量。
- **基准测试**：衡量不同基础模型（大语言模型后端）对笔记本质量的影响。
- **持续集成**：监控底层 Agent（智能体）循环的变化如何最终影响机器学习任务。

评估传统机器学习笔记本是最复杂的评估任务之一，因为它涉及代码质量、最佳机器学习实践以及基于数据的适应性调整。为了处理评估机器学习笔记本这样广泛且杂乱的任务，我们采用“大语言模型作为评判器”的方法——一个由人类教导、了解优秀笔记本具体样子的“专家”大语言模型。我们创建了九个评判器，它们被提示从大多数机器学习工作流中出现的九个维度来评估机器学习笔记本：

- **3（好）**：笔记本在该维度达到高标准。它展示了最佳实践，涵盖了预期范围，并恰当地处理了边缘情况。

## 评估评估系统

通过使用大语言模型评判器（而非人类）来评估 Genie Code 的产出，我们实际上是用一个难题替换了另一个难题：开箱即用的评判器在手上的任务中并不熟练，并且与人类评分不一致。我们的问题陈述是使大语言模型评判器的评分与人类评估者的评分对齐。

用于大语言模型评判器评估的测试集包含 50 个由 Genie Code 生成的笔记本（“测试用例”），人类专家对每个适用维度进行了评分，并提供了简短的理由作为我们的真实标准。在两个分数之间的灰色地带，评分者被允许表达自己的判断，但评分标准的设计使得这种情况很少发生。

人机对齐的衡量指标是每个维度上评分的平均绝对误差（MAE）。结果好坏参半，某些维度显示出强对齐（4 个维度的 MAE <= 0.10），而其他维度则显示出显著差异：

- **模型训练**：MAE 为 0.680
- **模型使用**：MAE 为 0.562
- **数据插补**：MAE 为 0.474
- **数据探索**：MAE 为 0.407

这种差距的存在是因为人类和大语言模型对同一评分标准的理解方式不同。人类评分者能够发现一个微妙缺陷的插补策略，或一个“能运行”但逻辑不合理的训练循环，而大语言模型评判器往往会错过这些技术细节。我们还发现评判器存在经典的积极性偏差——它过于“礼貌”，这妨碍了获得客观结果。

很明显，在相同的评分标准下，大语言模型评判器和人类不会产生相同的结果——这是一种不对齐。这正是 MemAlign 设计用来解决的问题。

## 使用 MemAlign 进行对齐

MemAlign 是 MLflow 中的一个框架，给定极少的人类自然语言反馈，它可以在人类评分者和大语言模型评判器之间执行对齐。这是通过从阅读人类反馈中形成的两种“记忆”来实现的：

- **语义记忆**存储广义的指导原则——从反馈中提炼出的、广泛适用的规则。
- **情景记忆**存储具体的例子——评判器出错的情况，作为未来决策的锚点保留下来。

在推理时，MemAlign 通过提取所有语义指导原则并检索与当前输入最相关的情景示例来构建工作上下文。评判器将所有内容以及原始评分标准加载到其上下文中，并利用积累的知识为所有未来的笔记本给出更准确的评分。

使 MemAlign 脱颖而出的关键特性是仅使用少量示例就能实现高性能。这是因为 MemAlign 有效地从自然语言反馈中丰富的学习信号中提炼知识，并将其整合到双记忆系统中。

### K折交叉验证

遵循机器学习训练-测试范式，我们在50个测试用例（笔记本）上应用了K折交叉验证（K=4），从而避免了数据泄露以及单独标注测试集的需求。对于每一折，我们执行以下操作：

### 自助法置信区间

为了在无额外标注数据的情况下计算置信区间，我们从原始50个样本中通过有放回抽样生成了100个自助样本。通过重复此过程10,000次并追踪人类评分与机器评分之间的平均绝对误差（MAE），我们计算了人机对齐的置信区间，并以95%置信区间定义统计显著的变化。

## 结果

![评判者与人类对齐](/images/posts/78d5f57446a3.png)

9个维度中有3个维度显示出统计显著的改进：

- 模型训练：MAE降低0.500（从0.680降至0.180），降幅达74%
- 模型使用：MAE降低0.438（从0.562降至0.125），降幅达78%
- 数据插补：MAE降低0.421（从0.474降至0.053），降幅达89%

这3个维度属于最初严重失调的4个维度。初始对齐较弱表明LLM与人类对共享评分标准的理解存在根本性差异，而MemAlign注入的记忆似乎提供了足够的上下文，使双方“达成共识”。

- 指标评估和MLflow日志记录原本已良好对齐（原始MAE < 0.10），其退化不具有统计显著性（实验噪声）
- 数据探索出现轻微退化（-0.130），但考虑到其置信区间[-0.33, +0.09]，不具有统计显著性。该维度表现出最高的评分者间方差，这种噪声阻碍了MemAlign的改进（甚至可能产生了负面影响）。

### 仅语义记忆实验

MemAlign的双记忆结构让我们质疑两者是否都对评判者对齐有实际贡献。特别是，情景记忆旨在通过提供一组最相似的已标注笔记本作为参考点（利用最近邻搜索）来帮助评判者。但如果检索到的笔记本（最近邻）实际上与当前笔记本并不相似——只是“最不相似”呢？将这些信息加载到评判者的上下文中可能反而会混淆问题，而非提供帮助。我们评分的领域（机器学习笔记本）非常广泛，最初我们假设50个笔记本的集合根本不足以提供足够密集的记忆供评判者回忆。

没有情景记忆的情况下，结果显著恶化：

- 模型训练仍有改进（+0.420），但增益小于完整MemAlign的+0.500，且对齐后的MAE为0.260，而完整版为0.180。
- 模型使用完全失去统计显著性——改进幅度从+0.438降至+0.294，且置信区间现在跨越零值。
- 数据插补从89%的错误降低变为零改进——对齐后的MAE与原始MAE持平（0.455）。
- MLflow日志记录和指标评估实际上显著退化。没有情景示例作为评判者的锚点，仅靠提炼后的指南给原本已校准良好的维度引入了噪声，将MLflow日志记录的MAE从0.062推高至0.396。

![完整记忆对齐 vs 仅语义记忆](/images/posts/0a40d20f7a0c.png)

这与我们的预期完全相反。我们最初假设稀疏的标注集最终会混淆评判者，但几乎所有维度在没有情景记忆的情况下都变得更差。唯一的例外是数据探索，去掉情景示例可能反而有所帮助——没有标注者存在分歧的具体笔记本，评判者仅依赖提炼后的指南，从而获得噪声更少的信号。

结论：即使输入数据庞大且杂乱，情景记忆仍能显著提升评判者的性能。语义记忆和情景记忆都是MemAlign功能不可或缺的组成部分。

## 结论：缩小专家差距

判断一个编码Agent是否在履行职责已经足够困难，而评估一个自主AI合作伙伴在构建和执行传统机器学习工作流方面的表现则属于另一层级的复杂性。由于AI产品的快速迭代，根本没有足够的时间让专家监控Agent的“持续集成”。唯一可行的可扩展解决方案是LLM评判者——但我们仍然需要一个人类评审团来监督LLM评判者。

通过应用MemAlign，我们在最关键的维度上将评判者错误降低了74–89%。但正如任何机器学习/LLM工作一样，结果的质量取决于输入信息的质量，因此请确保标注工作足够专业。

要点总结：

- 衡量你的测量系统：一个充满噪声的系统不利于评估，只有当我们投入时间和资源去实际验证和改进评判者时，我们才能信任自己的评估系统。
- 仅靠评分标准是不够的：人类理解指令的方式与LLM理解指令的方式之间存在细微差异。这些差异需要被考虑，而像MemAlign这样的对齐工具是弥合差距的有效方法。
- 标注质量 > 数量：当人类标注者之间存在分歧时（正如我们在数据探索维度退化中看到的那样），对齐过程就没有连贯的信号可以学习。

MemAlign随MLflow一起提供，并且仅需约50个标注示例即可为我们工作。如果你的LLM评判者与专家不匹配，花一个下午试试这个工具是值得的。
