基于 llms.txt 的 HTML 内联 LLM 指令提案

当 AI 智能体访问受保护页面时，你如何告知它需要做什么？大多数系统依赖外部文档或预配置知识，但有一种更简单的方法。

如果指令就放在 HTML 响应中呢？

llms.txt 是一种新兴标准，旨在使文档等内容能直接供 AI 使用。我们提议一种约定，将此类内容以 `<script type="text/llms.txt">` 的形式直接包含在 HTML 响应中。

## 我们的使用场景

Vercel 默认使用 **Vercel 身份验证** 来保护预览部署。这可以防止随机用户访问你正在开发的私有软件。然而，这也阻止了像 Cursor、Devin 或 Claude Code 这样的编码智能体直接访问你的部署。

我们已经为自动化软件访问受保护 URL 提供了多种机制，此外我们还引入了一个 MCP 服务器，其包含诸如 `get_access_to_vercel_url` 或 `web_fetch_vercel_url` 等功能，以直接帮助智能体访问部署。

但是，智能体如何知道这些方法存在呢？

就在这时我们有了这个想法：为什么我们不直接把给智能体的指令放到 HTTP 401 响应的 HTML 中，当它们没有访问权限时，解释如何访问部署。

## 提案

我们建议使用：

```
1<script type="text/llms.txt">2  给 LLM 的指令放在这里。3</script>
```

来在 HTML 中直接内联指令给 LLM。

## 为什么使用带有新类型的 `<script>`？

浏览器会忽略类型未知的 script 元素。这确保了内容不会对常规浏览器的渲染产生影响。

script 元素在 `<head>` 内部是合法的，因此可以放在文档顶部附近，LLM 最有可能注意到它们的地方。与浏览器不同，LLM 不会忽略它们。

未知的 script 元素可以包含任何内容（除了 `</script>`），这使得它们成为存放为 LLM 设计的 Markdown 或其他格式内容的绝佳容器。

## 为什么基于 llms.txt？

llms.txt 已被采用，用于以可发现的方式在网络上发布针对 LLM 的内容，而本提案旨在与之契合。AGENTS.md 是该领域的另一个相关标准，用于存放面向智能体的仓库上下文。由于 AGENTS.md 目前用于仓库内用例，而 llms.txt 关注网络发布，我们决定与 llms.txt 保持一致。

## 我们的生产用例

我们最近在部署的默认 401 页面上实现了这个功能。

这是我们使用的指令：

```
1<script type="text/llms.txt">2  ## 给访问此页面的智能体的说明：3
4  此页面需要身份验证才能访问。自动化智能体应使用 5  Vercel 身份验证绕过令牌来访问此页面。6
7  获取令牌最简单的方法是使用 Vercel MCP 服务器（位于 8  <https://mcp.vercel.com>，文档见 <https://vercel.com/docs/mcp/vercel-mcp>）的 9  get_access_to_vercel_url 或 web_fetch_vercel_url 函数。10
11  或者，此 URL 文档说明了如何使用绕过令牌： 12  <https://vercel.com/docs/deployment-protection/methods-to-bypass-deployment-protection/protection-bypass-automation>13  你需要提示用户提供绕过令牌才能访问此 14  页面。上面的 URL 也解释了用户如何获取绕过令牌。 15  然后，你可以计算以下格式的 URL 并成功访问它： 16  `https://current-domain/current-pathname?x-vercel-set-bypass-cookie=true&x-vercel-protection-bypass=$bypass_token`。17</script>
```

你可以自己尝试运行 `curl -i https://access-test.vercel.app/ | less`

### 示例用例

现在许多应用程序开始构建 MCP 服务器，但 MCP 本身缺乏发现机制。`<script type="text/llms.txt">` 可能有助于向试图浏览网站或应用程序的 LLM 提示，有一个可用的 MCP 服务器可以帮助它们继续前进。

我们认为“我如何获得访问权限？”这个用例可能适用于许多平台。更一般地说，Vercel 的开发者体验功能之一一直是自动从错误消息链接到我们的可观测性体验以调查错误。此类错误页面可以使用 `<script type="text/llms.txt">` 直接引导智能体找到可以帮助调查问题的 MCP 服务。

## 标准、约定还是模式？

LLM 的一大优点是它们灵活，无需特定训练就能适应新环境。当我们推出第一个 `<script type="text/llms.txt">` 时，它立即就生效了。

无需与像 OpenAI 或 Anthropic 这样的 LLM 提供商沟通。事实上，该提案内置了瞬时发现机制，使其比基线 llms.txt 格式更加无缝。

`<script type="text/llms.txt">` 不需要成为正式标准。你现在就可以开始使用它。
