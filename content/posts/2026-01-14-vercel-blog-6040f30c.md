今天我们发布了一套全新的 AI Elements 组件，旨在与 AI SDK 的 `Transcription`（转录）和 `Speech`（语音）功能协同工作，帮助您构建由自然语言驱动的下一代语音 Agent（智能体）、转录服务和应用程序。

## Persona（角色形象）

`Persona` 组件展示了一个动态响应的 AI 视觉形象，可根据不同的对话状态做出反应。它基于 Rive WebGL2 构建，为各种 AI 交互状态（包括空闲、聆听、思考、说话和休眠）提供了流畅、高性能的动画。该组件支持多种视觉变体，以适应不同的设计美学。

```
1npx ai-elements@latest add persona
```

## Speech Input（语音输入）

`SpeechInput` 组件为您的应用程序提供了一个易于使用的语音输入捕获界面。它在支持的浏览器（Chrome、Edge）中使用 Web Speech API 进行实时转录，对于不支持 Web Speech API 的浏览器（Firefox、Safari），则回退到使用 MediaRecorder 配合外部转录服务。

```
1npx ai-elements@latest add speech-input
```

## Transcription（转录）

`Transcription` 组件提供了一个灵活的渲染属性接口，用于显示带有同步播放功能的音频转录文本。它会根据播放时间自动高亮当前片段，并支持点击跳转功能，实现交互式导航。

```
1npx ai-elements@latest add transcription
```

## Audio Player（音频播放器）

`AudioPlayer` 组件在 media-chrome 基础上构建，提供了一个灵活且可定制的音频播放界面。它具有可组合的架构，允许您构建带有自定义控件、元数据显示以及与 AI 生成音频内容无缝集成的音频体验。

```
1npx ai-elements@latest add audio-player
```

## Microphone Selector（麦克风选择器）

`MicSelector` 组件提供了一个灵活且可组合的界面，用于选择麦克风输入设备。它基于 shadcn/ui 的 Command 和 Popover 组件构建，具备自动设备检测、权限处理、动态设备列表更新以及智能设备名称解析等功能。

```
1npx ai-elements@latest add mic-selector
```

## Voice Selector（语音选择器）

`VoiceSelector` 组件提供了一个灵活且可组合的界面，用于选择 AI 语音。它基于 shadcn/ui 的 Dialog 和 Command 组件构建，具有可搜索的语音列表，支持元数据显示（性别、口音、年龄）、分组和可自定义的布局。该组件包含一个上下文提供者，允许从任何嵌套组件访问语音选择状态。

```
1npx ai-elements@latest add voice-selector
```
