全新的 @vercel/related-projects 包可帮助跨不同 Vercel 项目同步部署信息，确保您的应用始终引用最新的预览或生产部署 URL，无需手动更新或修改环境变量。

此前，开发者必须手动输入部署 URL、管理连接字符串或使用环境变量来保持项目间的有效通信。现在，这些数据在构建和运行时均可自动获取并更新。

例如，一个包含以下内容的单体仓库：

- 一个从 API 获取数据的前端 Next.js 项目
- 一个提供数据的后端 Express.js API 项目

一个从 API 获取数据的前端 Next.js 项目

一个提供数据的后端 Express.js API 项目

相关项目现在可以确保前端的每个预览部署自动引用后端的对应预览部署，从而在测试跨两个项目的变更时避免使用硬编码值。

相关项目通过 Vercel 项目 ID 进行关联。您可以在 Vercel 控制台的"项目设置"页面中找到您的项目 ID。
