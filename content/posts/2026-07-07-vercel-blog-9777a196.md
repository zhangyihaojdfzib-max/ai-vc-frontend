GitHub Tools 现在通过新的 `@github-tools/sdk/eve` 子路径提供一套工具集。`agent/tools/` 中的单个文件即可注册所有 GitHub 工具，或使用诸如 `maintainer` 的预设配置，从而用九行代码构建一个完整的 GitHub Agent（智能体）。

```
You are a GitHub code review assistant.
import { defineAgent } from "eve";
export default defineAgent({  model: "anthropic/claude-sonnet-5",});
import { createGithubTools } from "@github-tools/sdk/eve";
export default createGithubTools({  preset: "maintainer",});
```

使用 GitHub Tools 的 'maintainer' 预设配置你的 GitHub Agent（智能体）

- 默认安全：每个写入工具（如 `mergePullRequest`）都需要审批，除非你选择退出。使用 `always`、`once` 或基于输入的条件谓词来控制单个工具；暂停操作在重启和部署后仍会保留。
- 预设配置：`code-review`、`issue-triage`、`repo-explorer`、`ci-ops` 和 `maintainer` 可单独或合并使用，以限定工具集范围。
- 精简读取：高容量读取工具（如 `listPullRequestFiles` 和 `getCommit`）会精简模型看到的内容，而通道仍能获取完整数据。
