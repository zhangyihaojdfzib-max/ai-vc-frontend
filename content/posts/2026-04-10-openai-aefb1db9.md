# 推出 GPT‑5.4 mini 与 nano

专为编码和子智能体优化的快速高效模型

今天我们发布 GPT‑5.4 mini 和 nano，这是我们迄今为止能力最强的小型模型。它们将 GPT‑5.4 的诸多优势带到了为高吞吐量工作负载设计的、更快、更高效的模型中。

GPT‑5.4 mini 在编码、推理、多模态理解和工具使用方面相比 GPT‑5 mini 有显著提升，同时运行速度提高了两倍以上。在包括 SWE-Bench Pro 和 OSWorld-Verified 在内的多项评估中，它也接近了更大的 GPT‑5.4 模型的性能。

GPT‑5.4 nano 是 GPT‑5.4 系列中最小、最经济的版本，适用于速度和成本最为关键的任务。它也是 GPT‑5 nano 的一次重大升级。我们推荐将其用于处理较简单支持任务的分类、数据提取、排序和编码子智能体。

这些模型专为那些延迟直接影响产品体验的工作负载类型而构建：需要响应迅速的编码助手、快速完成支持任务的子智能体、捕获和解释屏幕截图的计算机使用系统，以及能够实时对图像进行推理的多模态应用。在这些场景中，最好的模型往往不是最大的那个，而是能够快速响应、可靠使用工具，并且在复杂的专业任务上仍能表现出色的那个。

## 编码

GPT‑5.4 mini 和 nano 在受益于快速迭代的编码工作流程中尤其有效。这些模型能够以低延迟处理针对性编辑、代码库导航、前端生成和调试循环，使其非常适合需要以更快速度和更低成本完成的编码任务。

在基准测试中，GPT‑5.4 mini 在相似延迟下持续优于 GPT‑5‑mini，并且在运行速度快得多的同时接近 GPT‑5.4 级别的通过率，为编码工作流程提供了最强的性能与延迟权衡之一。

我们通过观察模型的生产行为并离线模拟来估算延迟。延迟估算考虑了工具调用时长（代码执行时间）、采样的 Token 和输入 Token。实际延迟可能会有很大差异，并取决于我们模拟中未捕获的许多因素。同样，成本是基于撰写本文时这些模型的 API 定价估算的。成本未来可能会发生变化。推理强度从低到 xhigh 进行了扫描。

#### 子智能体

GPT‑5.4 mini 也非常适合结合不同规模模型的系统。例如，在 Codex 中，像 GPT‑5.4 这样的大型模型可以处理规划、协调和最终判断，同时委托 GPT‑5.4 mini 子智能体并行处理范围更窄的子任务——例如搜索代码库、审查大文件或处理支持文档。了解子智能体在 Codex 中如何工作的详情，请参阅文档⁠(在新窗口中打开)。

随着小型模型变得更快、能力更强，这种模式变得更加有用。开发者无需为所有事情都使用一个模型，而是可以构建这样的系统：由大型模型决定做什么，而小型模型则大规模地快速执行。对于这种工作流程风格，GPT‑5.4 mini 是我们迄今为止最强大的 mini 模型。

## 计算机使用

GPT‑5.4 mini 在多模态任务上也很强大，尤其是与计算机使用相关的任务。该模型能够快速解读密集用户界面的屏幕截图，以完成计算机使用任务。在 OSWorld-Verified 上，GPT‑5.4 mini 接近 GPT‑5.4 的性能，同时大幅优于 GPT‑5 mini。

## 可用性与定价

GPT‑5.4 mini 现已通过 API、Codex 和 ChatGPT 提供。

在 API 中，GPT‑5.4 mini 支持文本和图像输入、工具使用、函数调用、网络搜索、文件搜索、计算机使用和技能。它拥有 400k 的上下文窗口，输入 Token 每 100 万个成本为 0.75 美元，输出 Token 每 100 万个成本为 4.50 美元。

在 Codex 中，GPT‑5.4 mini 可在 Codex 应用、CLI、IDE 扩展和网页端使用。它仅消耗 GPT‑5.4 配额的 30%，让开发者能够以大约三分之一成本在 Codex 中快速处理较简单的编码任务。Codex 还可以委托给 GPT‑5.4 mini 子智能体，以便推理密集度较低的工作在更便宜的模型上运行。

在 ChatGPT 中，GPT‑5.4 mini 通过 + 菜单中的“思考”功能向免费版和 Go 用户提供。对于所有其他用户，GPT‑5.4 mini 作为 GPT‑5.4 思考功能的速率限制后备方案提供。

GPT‑5.4 nano 仅在 API 中提供，输入 Token 每 100 万个成本为 0.20 美元，输出 Token 每 100 万个成本为 1.25 美元。
