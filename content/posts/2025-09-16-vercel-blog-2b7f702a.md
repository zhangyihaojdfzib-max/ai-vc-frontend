## 摘要

Shai-Halud 供应链攻击活动已升级。最初影响约 18 个核心 npm 包（如 chalk、debug、ansi-styles 等）的 Qix 入侵事件，现已扩散：

*   通过 Tinycolor “蠕虫” 载体攻击了超过 40 个额外软件包。
*   CrowdStrike /crowdstrike-publisher 命名空间也遭到入侵，发布了多个木马化版本。
*   DuckDB 维护者账户 (duckdb_admin) 发布了恶意版本，其使用的钱包窃取恶意软件与 Qix 事件中使用的相同。该 DuckDB 子集未影响任何 Vercel 客户。

## 对 Vercel 客户的影响

*   我们识别出**一小部分（10个）** Vercel 客户项目，其构建过程（直接或间接）依赖于受入侵的软件包版本。
*   受影响的客户已收到通知，并获得了项目级别的指导。
*   在 DuckDB 事件中，没有 Vercel 客户的构建受到影响。

## 我们采取的行动

| 行动 | 状态 |
| :--- | :--- |
| 将 Tinycolor、CrowdStrike、Qix 和 DuckDB 受影响软件包中已知的受入侵版本加入阻止列表 | ✅ 已完成 |
| 清除使用这些版本的 Vercel 项目的构建缓存 | ✅ 对受影响项目已完成 |
| 协调使用干净的依赖项/锁定安全版本进行安全重建 | ✅ 进行中 / 对受影响项目已完成 |
| 针对符合 Shai-Halud 攻击特征的新软件包发布，提升平台告警和检测阈值 | ✅ 已提升监控级别并生效 |

## 我们正在关注和进行的工作

*   与 npm、开源维护者及生态系统安全合作伙伴密切合作，追踪 Shai-Halud 攻击的进一步扩散。
*   增强我们的供应链防御，**确保 Vercel 上的部署默认保持安全**：对生命周期/postinstall 脚本、锁文件卫生状况和注册表验证实施更严格的政策。
*   加强内部 CI/CD 控制和开发者工具，以尽早发现可疑的软件包行为。

## 给 Vercel 用户的建议

*   对于使用 pnpm 的团队，可考虑启用 pnpm 10.16 中引入的新 `minimumReleaseAge` 设置，以延迟依赖项更新（例如，24 小时）。这有助于降低因发布后不久被发现并移除的受入侵版本所带来的风险。
*   审计您的依赖项（直接和间接），检查是否包含来自这些受影响命名空间的软件包。
*   使用锁定的安全版本和干净的锁文件（`pnpm ci`）进行重建。
*   轮换任何可能曾在存在受入侵依赖项的环境中使用过的 npm / GitHub / CI/CD 令牌。
*   检查 GitHub 仓库是否存在未经授权的工作流或意外的 `.github/workflows` 添加项。
*   强制执行最小权限原则（尤其是在自动化工作流中），并限制生命周期脚本的权限。
