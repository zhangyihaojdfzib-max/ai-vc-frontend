## 从文本生成图像  

使用几行 Python 代码，你可以通过文本以编程方式生成图像。  

Replicate 允许我们通过 `replicate.models.get` 使用 `f"{username}/{model_name}"` 查找模型。  

对于笔记本中的示例，我们将使用 `"afiaka87/glid-3-xl"`，这是一个生成逼真图像的优秀模型。为了有趣，让我们生成一张牛油果灯泡的图像！  

```
model = replicate.models.get("afiaka87/glid-3-xl")
version = model.versions.get("d74db2a276065cf0d42fe9e2917219112ddf8c698f5d9acbe1cc353b58097dab")
```

Replicate 上的模型通过 `.predict` 方法运行。让我们快速查看一下 `.predict` 的命名/关键字参数。  

每个模型的命名/关键字参数会有所不同。`glid-3-xl` 需要一个输入 `prompt`——你想要可视化的场景描述。  

我们还将 `seed` 设置为 0。设置手动种子会促使模型对一组输入返回完全相同的输出。否则，种子会被随机选择。输出可能仍有细微差异，但管理种子通常是个好主意，Replicate 上的许多模型都支持它。  

```
prediction_generator = version.predict(prompt="an image of a fresh avocado in the form of a lightbulb", seed=0)
```

调用 `.predict` 只是初始化模型，但不会将其排队在 Replicate 上运行。要运行模型，只需迭代生成器。  

```
generated_image_batches = list(prediction_generator)
final_image_batch = generated_image_batches[-1] # ["https://...",]
print(final_image_batch)
```

因为我们只对模型返回的最终完成输出感兴趣，所以我们可以将生成器转换为列表并获取最后一个（-1）元素。  

最终批次是一个 URL 列表，其大小由 `batch_size`（默认为 1）决定。

## 增强图像  

API 开启了许多可能性，比如将一个模型的输出作为另一个模型的输入。一个常见的例子是超分辨率，其中一个模型生成的图像被输入到超分辨率模型中以放大它。  

我们将使用 `"raoumer/srrescgan"` 来放大我们的牛油果灯泡图像，但 Replicate 上还有许多超分辨率模型 供你探索。  

```
generation_to_enhance = Path(final_image_batch[0]) # 默认列表中只有一个 URL。
upscaling_model_api = replicate.models.get("raoumer/srrescgan")
high_res_outputs = upscaling_model_api.predict(image=generation_to_enhance)
```

## 创建图像的变体  

一些文生图模型允许你传入一个称为初始图像的现有图像。这会生成图像的不同变体，并受到指定提示词的一定影响。  

我们将使用 `"laion-ai/ongo"`，这是 `glid-3-xl` 在 WikiArt 上微调的一个版本。  

你需要一张图像来创建变体。我们将使用 `ongo` 来变化这张农舍图像：  

![作为初始图像的农舍](/images/posts/99914ab80f43.webp)  

Replicate 的图像输入可以是 URL 或本地路径。  

```
init_image = Path("/assets/blog/exploring-text-to-image-models/farmhouse.jpeg")
```

调整各种设置以改善模型性能可能很有价值，或者你也可以直接移除可选参数，使用模型作者设置的默认值。  

- `init_image`：一个 `pathlib.Path` 类型的“初始图像”，用于与生成结果混合，使模型在指定提示词之外还受到提供图像的影响。也可以是 URL（转换为 Path）。  
- `guidance_scale`：决定生成结果受文本引导的程度。  
- `batch_size`：1 到 12 的整数。要生成的变体数量。较小的批次大小比大的批次大小快得多。  
- `steps`：30 到 250 的整数。模型运行的离散时间步数。当使用 `init_image` 时，实际时间步数为 `steps * init_skip_fraction`（默认减半）。增加步数会提高准确性，但会牺牲性能。  
- `init_skip_fraction`：0.0 到 1.0 的小数。默认为 `0.5`。图像对生成结果的影响程度。0.0 几乎不使用，1.0 仅编码图像而不受模型影响。  

如有疑问，你可以直接移除某个参数，它将使用默认值。  

```
model = replicate.models.get("laion-ai/ongo")
version = model.versions.get("1b3cd15121ec450baa71bbbdacddef9217519f12ca12ccfef36eeaa20ad89b9d")
ongo_variation_generator = version.predict(
    prompt="professional painting of a red lakehouse in the style of monet",
    guidance_scale=10.0, # 1.0 - 100.0
    total_steps=250, # 30-250
    init_skip_fraction=0.35, # 0.0 - 1.0
    batch_size=3, # 1 - 12
    init_image=Path("/assets/blog/exploring-text-to-image-models/farmhouse.jpeg"),
    seed=0, #
)
```

回想一下，`.predict` 只是返回一个生成器。要开始预测，你必须先枚举它以获取最终的批次输出 URL。我们只对最后一个批次感兴趣。  

```
ongo_variations_final_batch = list(ongo_variation_generator)[-1]
```

使用批次大小为 3 意味着我们应该得到 3 个图像 URL。  

```
number_of_variations = len(ongo_variations_final_batch) # 应等于批次大小
print(ongo_variations_final_batch) # 应是一个 URL 列表
```

我们推理的输出是一系列美丽的湖畔别墅，风格源自原始农舍图像！
