采用Agentic（智能体化）AI系统的组织正在日益增多。这些系统能提升生产力，将人们从重复性任务中解放出来。随着这些系统不断成熟并投入生产环境，企业将因其自主运行能力以及代表我们做出更优决策的能力而获得巨大价值。

然而，随着Agentic AI的普及，新的挑战也随之出现。其一是熟练人才的短缺。构建和管理有效的智能体系统需要深入的技术专长，而对经验丰富的AI工程师的需求正在快速增长。另一个挑战是，专家们越来越难以跟上该领域技术、框架和工具的持续演进步伐。

Kasal正是为应对这些挑战而创建的。它是一个由智能体驱动的平台，允许不同技能水平的用户通过直观的可视化界面来设计、开发和部署有效的Agentic AI系统。非专家可以利用Kasal直观的用户界面，构建满足其特定需求的复杂Agentic AI系统。专家则可以使用Kasal快速启动项目，并随后将其智能体导出为代码，以便进行更深度的定制和开发。

Kasal的目标是在企业环境中，为专家和非专家普及Agentic AI。

## 什么是 Kasal

Kasal 是一个优先采用用户界面的框架，用于设计、运行和观察单智能体及多智能体工作流。您无需手动编写复杂的编排代码，只需在可视化画布上拖放智能体，或通过对话助手简单描述您的需求。Kasal 将自动为您构建工作流。随后，您可以连接工具、运行智能体并实时观察其行为。在后台，Kasal 使用 CrewAI（一个用于创建和编排 AI 智能体的开源 Python 框架），但将其封装在一个对 Databricks 友好的应用层中，该层负责管理身份验证、部署和监控。这意味着您可视化设计的相同流程可以轻松投入生产环境。生成的流程也可以导出为代码，允许 AI 工程师根据需要进一步优化或扩展它。

## Kasal 的重要性

Kasal 带来了三项核心能力：由智能体驱动的可视化工作流设计器、与 Databricks 的深度集成，以及一个包含 MCP 服务器、Genie、自定义 API 和数据连接器的可扩展工具包。

*   **可视化编排**：使智能体、任务交接和分支逻辑清晰可见、易于理解，便于与非开发人员评审以及后续审计。此外，Kasal 内嵌的智能体能够理解用户在定义想要构建的智能体类型时的意图，并提出符合行业最佳实践的优化设计方案。
*   **Databricks 原生**：Kasal 可以作为 Databricks App 安装或通过 Marketplace 访问，允许用户继承工作区的身份验证和治理策略，而无需自行管理。此外，Kasal 生成的流程原生利用 Databricks 功能，使其从一开始就具备企业级和生产就绪的特性。这些功能包括用于追踪和跟踪的 MLflow、用于记忆的 Vector Search、用于服务的 Databricks Apps、用于事务日志记录的 Lakebase、代表用户身份验证等等。
*   **可扩展性**：Kasal 为 MCP 服务器、Genie 空间、Agent Bricks 以及自定义 API 和数据连接器提供一流的支持。它还可以将流程导出为 Notebook，提供对其内部逻辑的完全透明性，并允许 AI 工程师以充分的灵活性，在初始版本基础上扩展和完善解决方案。

Kasal 的实时可观测性为多智能体 AI 工作流提供双层监控。通过 Kasal 前端，业务用户可以查看执行时间线，跟踪工作流状态、智能体交互和任务进度。同时，MLflow 追踪集成允许 AI 工程师调试模型性能、LLM 调用和智能体行为。当部署在 Databricks Apps 上时，Kasal 使用 Databricks OBO 身份验证进行用户隔离，并使用生产就绪的 SQLite 或 Lakebase 持久化存储，以实现透明的智能体操作。

## 开始使用 Kasal

典型的用户旅程始于向 Kasal 描述您想要构建的智能体规格。例如，您可以提问：“创建一个计划，为我们的销售代表生成一份针对不同客户量身定制的、用于销售我们不同产品的推介演示文稿。”然后，Kasal 将利用其提示词和大语言模型生成一个结构化的计划，通常是分层级的。

在此示例中，如果计划处于**顺序模式**，智能体将按照设定的顺序一个接一个地运行。然而，如果计划处于**分层模式**，它将包含一个管理智能体和若干子智能体，每个子智能体负责特定任务：例如，一个负责检索和分析客户数据，另一个负责检索产品数据，一个负责结合两者来构思推介的故事线，还有一个负责生成反映详细信息和故事线的演示文稿。

然后，您可以执行该计划，为特定客户生成产品演示文稿。如果您希望修改工作流，例如尝试不同的模型或工具，可以通过 Kasal 用户界面轻松完成。

如果您认为在 Kasal 中创建的计划对他人有价值，可以将其注册到 Kasal 的目录中，以便在未来的会话中复用和调用。如果您希望在 Kasal 之外将该计划工业化，可以导出其代码并围绕其构建生产流水线。您拥有完全的灵活性，可以将该计划扩展并集成到您更广泛的解决方案架构中。

## 您可以用 Kasal 构建什么

我们已经看到用户使用 Kasal 构建了各种各样的智能体和多智能体 AI 系统。以下是一些示例：

*   **数据分析流水线**：查询、分析和可视化数据的智能体。
*   **内容生成系统**：用于研究、写作和内容创作的协作智能体。
*   **业务流程自动化**：能够适应和做出决策的智能工作流。
*   **研究与开发**：收集、综合和呈现见解的智能体。

2.  **从源代码部署**：克隆 Databricks Labs GitHub [仓库](repository)并运行提供的部署[脚本](script)。如果您希望自定义或扩展 Kasal 以满足特定需求，此选项是理想选择。
