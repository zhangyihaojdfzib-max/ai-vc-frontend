我们很高兴地宣布Genie Code，这是Databricks Genie家族的最新成员。在过去的六个月里，Agent（智能体）驱动的编码工具已经从根本上改变了软件工程；Genie Code为数据团队带来了同样的变革。Genie Code能够自主执行复杂任务，例如构建流水线、调试故障、交付仪表板以及维护生产系统。

## Agent（智能体）驱动数据工作的兴起

Agent（智能体）驱动的编码工具已经改变了软件工程，使开发人员超越了自动补全，走向了Agent（智能体）驱动的开发。现在，工程师只需一个提示词，就能在几秒钟内搭建功能框架、重构代码并部署原型。这一转变是由LLM（大语言模型）的进步以及能够理解现代软件代码库复杂上下文的Agent（智能体）系统所推动的。

市场上的大多数Agent（智能体）都将代码视为最终产品。然而，对于数据团队来说，代码仅仅是操作和理解底层数据的工具。这正是为什么以软件为中心的Agent（智能体）在处理数据工作时常常遇到困难。在数据生态系统中，上下文不仅存在于脚本中，还存在于使用模式、血缘关系和业务语义中。

获取这种上下文至关重要，因为风险很高。仪表板驱动着业务决策，流水线支撑着生产系统，机器学习模型影响着现实世界的结果。对于数据团队来说，Agent（智能体）提供的速度和杠杆作用必须与绝对的准确性、可重复性和治理相结合。

## Genie Code支持数据工作的完整生命周期

### 训练和评估机器学习模型

Genie Code充当嵌入您工作流程中的专职机器学习工程师。要求它"训练一个预测@sales_table中销售额的预测模型"，它将推理整个流水线：

- 识别和分析特征
- 正确划分训练、验证和测试数据集
- 训练多种模型类型并进行比较，运行超参数扫描以训练最佳模型。
- 根据AUC、F1、RMSE和R²等指标评估结果
- 生成特征重要性图、混淆矩阵和ROC曲线
- 在MLflow中跟踪实验
- 根据模型诊断结果推荐改进方案

一旦部署在Databricks模型服务上，Genie Code会持续参与：它可以检查端点健康状况、分析追踪记录并推荐优化方案。您可以在下面的"从代码到生产：使用Genie Code进行可观测性"部分阅读更多相关内容。

### 定制化与可扩展性

Genie Code 是一个灵活的平台，旨在根据您团队的特定标准和外部技术栈进行定制。扩展其能力主要有三种方式：

1.  **通过模型上下文协议（MCP）连接外部工具**：Genie Code 支持模型上下文协议（MCP），这是一个开放标准，使其能够安全地与您的外部工具、API 和文档交互。这实现了可延伸至 Databricks 工作空间之外的自主工作流。

    例如，当您被分配一项 Jira 任务来训练新的 ML 模型时，Genie Code 可以自动从中收集上下文信息，执行任务，并将结果更新到任务单中。

    ![Genie Code 支持 MCP](/images/posts/fc59892f6479.png)

    通过 MCP 将 Genie 连接到您的内部 Confluence、Google Drive、GitHub 或 Notion，使其在故障排除时能够参考您团队特定的操作手册和数据字典。

2.  **Agent 技能**：定义领域特定能力，以教导 Genie Code 如何一致地执行复杂任务。无论是您公司处理 PII 掩码的特定方式，还是用于数据验证的自定义框架，技能都能确保 AI 每次都能遵循组织的最佳实践。技能遵循开放的 Agent Skills 格式。

3.  **记忆**：Genie Code 越用越智能。通过持久化记忆，该 Agent 会根据您过去的交互自动更新其内部指令。它会学习您的编码偏好，记住您最常使用的数据集，并在不同会话间保留上下文。

## 从代码到生产：Genie Code 的可观测性

编写代码只是第一步，维护代码才是真正的挑战。Genie Code 充当可观测性 Agent，确保您的数据和 AI 工作流保持健康。虽然成千上万的客户使用 Databricks 来提供复杂的 AI 应用程序，但在生产环境中调试这些模型通常是生命周期中最耗时的部分。

Genie Code 现已直接与 Databricks Model Serving 和 MLflow 3.0 集成，以自动化此过程。您无需再手动搜索日志和追踪信息，而是可以使用 Genie 进行：

- **端点健康检查**：通过一个提示词即可获取涵盖计算、请求处理和服务器日志的完整状态报告。

![使用 Genie Code 执行端点健康检查](/images/posts/c1838131b782.png)

- **Agent 质量分析**：实时发现复杂 Agent 追踪中的细微问题，例如幻觉、错误的工具调用以及用户受挫模式。

- **生产故障排除**：当发生事件时，Genie 会交叉引用服务器日志和指标，自动化完成第一轮诊断，缩短问题解决时间。
- **端点优化**：根据 Databricks 最佳实践，获取关于预置并发、硬件配置和自动扩缩容的建议。

### 保障工作负载健康的后台 Agent

Genie Code 设计为在后台运行，即使您关闭笔记本电脑，您的数据也能保持健康。您可以并行部署多个 Agent 来处理那些通常消耗数据工程师一周时间的运维工作。这些后台 Agent 超越了被动支持，通过处理诸如响应作业失败和管理例行升级等重复性任务，转向主动维护。当管道中断时，Agent 会识别根本原因，并在安全沙箱环境中验证修复方案后，才建议进行修复。

例如，如果生产管道因模式不匹配（例如某列从 `INT (150)` 更改为 `STRING ("150 USD")`）而失败，Genie Code 将精确定位故障并自动修复损坏的管道。

后台 Agent 即将推出。

## 基于 Unity Catalog：集成的安全与治理

Genie Code 直接构建在 Unity Catalog 之上。这种集成确保了该 Agent 遵循与 Databricks 平台其他部分相同的安全和治理规则。

当 Genie Code 搜索数据时，它只会显示用户有权访问的资产。当它构建管道时，会遵守现有的数据血缘和访问控制。

- **原生修订历史记录**：每次编辑都通过 Databricks 版本控制系统进行跟踪。您可以完全放心地回滚跨 Notebook、查询、文件和 Lakeflow 管道的更改。
- **内置防护措施**：Genie Code 被设计为在执行可能修改底层表的代码之前，主动请求确认。
- **访问控制强制执行**：Genie Code 绝不会暴露用户无权查看的数据资产。
- **全面的审计日志记录**：您的组织可以通过现有的审计基础设施，完全了解 Genie Code 的使用情况。
