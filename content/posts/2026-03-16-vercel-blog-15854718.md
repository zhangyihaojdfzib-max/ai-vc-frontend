Streamdown 是一个 React 组件库，它让渲染流式 Markdown 内容变得简单而美观。专为 AI 驱动的应用程序构建，它处理了当 Markdown 被 Token 化并实时流式传输时出现的独特挑战。

v2.5 版本增加了内联 KaTeX 支持、交错式流式动画，并修复了代码块、CSV 导出相关问题，以及更好的 Tailwind v3 兼容性。

**流式解析器改进**

新的 `inlineKatex` 选项会在流式传输过程中自动补全 `$formula` 为 `$formula$`，避免与货币符号产生歧义，该选项默认值为 `false`。同时，当流式传输产生不完整的闭合 `$` 时，块级 KaTeX 的补全问题也已修复。

另外，单词字符之间的单个 `~`（例如 `20~25°C`）现在会被转义，以防止错误的删除线渲染。这通过一个新的、默认启用的 `singleTilde` 选项来控制。

**交错式流式动画**

流式单词和字符动画现在会按顺序级联显示，而不是同时全部动画化。时序可通过新的 `stagger` 选项进行配置（默认为 40 毫秒）。设置 `stagger: 0` 可恢复之前的行为。

**代码块**

自定义渲染器现在可以通过一个新的可选 `meta` 属性接收来自代码围栏的原始元字符串，而 `lineNumbers` 属性允许您禁用行号。

长行现在会水平滚动而不是被截断，已完成的代码块在接收到新的流式内容时不再重新渲染，未知或截断的语言标识符会回退到纯文本高亮而不是抛出错误。

**错误修复**

`save()` 现在会为 `text/csv` 内容添加 UTF-8 BOM，以便 Windows 上的 Excel 能正确检测编码。仅限 Tailwind v4：`*:last:/*:first:` 语法已被替换为等效的任意变体，修复了 Tailwind CSS v3 中的插入符号渲染问题。

阅读[文档](https://streamdown.dev)以开始使用。
