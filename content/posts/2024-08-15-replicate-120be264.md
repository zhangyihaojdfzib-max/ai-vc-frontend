## 如何微调 FLUX.1

您可以在 Replicate 上通过上传一些图像来微调 FLUX.1，可以通过网页或 API 进行。

如果您不熟悉 Replicate，我们让通过 API 运行 AI 变得简单。您无需寻找高性能 GPU，无需处理环境和容器，也无需担心扩展问题。您只需编写普通代码，使用普通 API，并且只为实际使用付费。

### 准备训练数据

要开始微调，您需要一组代表您想教给模型的概念的图像。这些图像应足够多样化，以涵盖概念的不同方面。例如，如果您要针对特定角色进行微调，请包含不同场景、姿势和光照下的图像。

以下是一些指导原则：

- 使用 12-20 张图像以获得最佳效果
- 尽可能使用大尺寸图像
- 使用 JPEG 或 PNG 格式
- （可选）为每张图像创建一个同名的 .txt 文件，其中包含描述文字

准备好图像（以及可选的描述文字）后，将它们压缩成一个单独的 zip 文件。

### 在网页上创建训练

要在网页上启动训练过程，请导航至 Replicate 上的 Ostris 的 FLUX.1 [dev] 训练器。

首先，选择一个模型作为目标，或者在模型选择器字段中输入名称来创建一个新模型。

接下来，将包含训练数据的 zip 文件作为 `input_images` 上传，然后设置训练参数。

`trigger_word` 指的是您正在训练的对象、风格或概念。选择一个不是真实单词的字符串，例如 `TOK`，或者与训练内容相关的词，例如 `CYBRPNK`。您指定的触发词将在训练期间与所有图像关联。然后，当您运行微调后的模型时，可以在提示词中包含该触发词来激活您的概念。

对于 `steps`，1000 是一个不错的起点。

将 `learning_rate`、`batch_size` 和 `resolution` 保持为默认值。除非您想提供自己的描述文字，否则请保持 `autocaptioning` 为启用状态。

如果您想在 Hugging Face 上保存模型，请输入您的 Hugging Face 令牌并设置仓库 ID。

填写完表单后，点击“创建训练”以开始微调过程。

### 通过 API 创建训练

或者，您可以通过 API 在自己的代码中创建训练。

确保您的环境中设置了 `REPLICATE_API_TOKEN`。您可以在账户设置中找到它。

```
export REPLICATE_API_TOKEN=r8_***************************
```

创建一个新模型，它将作为您微调后权重的目标位置。训练过程完成后，您训练好的模型将存放在这里。

```
import replicate

model = replicate.models.create(
    owner="yourusername",
    name="flux-your-model-name",
    visibility="public",  # 或者如果您愿意，可以设为 "private"
    hardware="gpu-t4",  # 对于微调模型，Replicate 会覆盖此设置
    description="A fine-tuned FLUX.1 model"
)

print(f"Model created: {model.name}")
print(f"Model URL: https://replicate.com/{model.owner}/{model.name}")
```

现在您有了模型，通过创建一个新的训练运行来启动训练过程。您需要提供输入图像、步数以及任何其他所需的参数。

```
# 现在使用此模型作为您训练的目标
training = replicate.trainings.create(
    version="ostris/flux-dev-lora-trainer:4ffd32160efd92e956d39c5338a9b8fbafca58e03f791f6d8011f3e20e8ea6fa",
    input={
        "input_images": open("/path/to/your/local/training-images.zip", "rb"),
        "steps": 1000,
        "hf_token": "YOUR_HUGGING_FACE_TOKEN",  # 可选
        "hf_repo_id": "YOUR_HUGGING_FACE_REPO_ID",  # 可选
    },
    destination=f"{model.owner}/{model.name}"
)

print(f"Training started: {training.status}")
print(f"Training URL: https://replicate.com/p/{training.id}")
```

请注意，此时您为模型选择哪种硬件并不重要，因为我们为所有 FLUX.1 微调都路由到 H100。这么多步的训练通常需要 20-30 分钟，费用低于 2 美元。

### 使用您训练好的模型

训练完成后，您可以直接在 Replicate 上使用训练好的模型，就像使用任何其他模型一样。

您可以在网页上运行它：

1.  转到您在 Replicate 上的模型页面（例如，`https://replicate.com/yourusername/flux-your-model-name`）。
2.  在提示词输入中，包含您的触发词（例如“bad 70s food”）以激活您微调后的概念。
3.  根据需要调整其他输入。
4.  点击“运行”以生成图像。

或者，通过 API 运行。例如，使用 Python 客户端：

```
import replicate

output = replicate.run(
    "yourusername/flux-your-model-name:version_id",
    input={
        "prompt": "A portrait photo of a space station, bad 70s food",
        "num_inference_steps": 28,
        "guidance_scale": 7.5,
        "model": "dev",
    }
)

print(f"Generated image URL: {output}")
```

将 `yourusername/flux-your-model-name:version_id` 替换为您实际的模型详细信息。

您可以在模型页面的“API”选项卡上找到有关使用 API 运行它的更多信息。

### 分享您的模型

如果您希望其他人能够发现和使用您新微调的模型，您需要将其设为公开。

如果您使用基于网页的训练表单创建了新模型，默认情况下它是私有的。

要将您的模型设为公开，请转到模型设置页面并将可见性设置为“公开”。

一旦您的模型公开，您就可以通过发送模型页面的 URL 与他人分享，并且它将出现在网站的“探索”部分以及 Flux 微调模型合集中。

### 使用 FLUX.1 [schnell] 进行更快的生成

您可以将您的 FLUX.1 [dev] LoRA 与较小的 FLUX.1 [schnell] 模型一起使用，以更快、更便宜地生成图像。只需在生成时将 `model` 参数从 `dev` 更改为 `schnell`，并将 `num_inference_steps` 降低到较小的值，例如 4。

请注意，输出仍将遵循 FLUX.1 [dev] 的非商业许可。

### 许可和商业用途

如果您在 Replicate 上使用 FLUX.1 模型及其微调版本生成图像，则可以将这些图像用于商业用途。

如果您从 Replicate 下载模型权重并在自己的计算机上生成图像，则不能将这些图像用于商业用途。

### 定价

训练过程运行的时间将按秒计费。FLUX 模型的训练在 Nvidia H100 GPU 硬件上运行，每秒成本为 $0.001528。对于一个典型的 20 分钟训练（通常使用约 20 张训练图像和 1000 步），预计费用约为 1.85 美元。

模型训练完成后，您可以像使用其他 Replicate 模型一样通过 API 运行它，并且仅按生成图像所需的时间计费。与其他定制模型不同，您无需为私有模型的闲置时间付费。
