### 起源故事

这个起源故事值得重述，因为它塑造了我们看待这个项目的方式。

舒丁是我共事过的最有才华的 Web 工程师之一。他掌握的关于 React 和浏览器的知识，是大多数人永远无法发现的。去年，他花了一个周末的时间，把所有东西都写了下来。就像一部 Web 圣经。我们想弄清楚如何将其发布出去。我们考虑过写一篇博客文章或文档，或许下一代模型最终会学习这些内容——但我们要等到 Claude Sonnet 8 或 GPT-9 才能看到结果。另一方面，MCP 服务器对于本质上是一系列 Markdown 文档的集合来说，感觉又太重了。

技能似乎是提供按需知识的最快方式。在编写安装 React 最佳实践的说明时，我最终复制粘贴了相同的安装说明，以便将技能安装到 Cursor、Claude Code、Codex 以及其他 10 多个编码智能体中，只是安装目录略有不同。

因此，我构建了一个 CLI，可以一次性将其安装到所有主要的编码智能体中。这就成了 `npx skills`。我们添加了遥测功能，以便在技能被安装时展示新技能，这些数据后来成为了 skills.sh 上排行榜的动力来源。整个项目从构思到在 Vercel 上投入生产只用了几天时间。Vercel 的 CTO Malte Ubl 对此描述得非常贴切：它是智能体上下文的包管理器。

现在，我们正在追踪 69,000 个技能，并且不仅让它们易于发现，还让它们易于安装，只需简单的命令，例如：

```
npx skills add vercel-labs/agent-skills --skill vercel-react-best-practices
```

### 我们需要解决的安全问题

增长会带来攻击面，快速增长则使其更快出现。

技能一经流行，质量差异便随之而来。来自 Socket 的 Ryan 向我们展示了一个具体例子：一个在 Markdown 层面看起来完全干净的技能，却包含了一个在安装时打开远程 shell 的 Python 文件。如果不检查目录中的每个文件，你永远无法发现这个问题。

这就是为什么我们宣布与 Gen、Socket 和 Snyk 建立安全合作伙伴关系，对所有现有技能以及每一个新提交的技能进行审计。

- Socket 正在进行跨生态系统的静态分析，并结合基于 LLM 的降噪处理，根据其基准测试报告，精确率达到 95%，召回率达到 98%，F1 分数达到 97%。
- Gen 正在构建一个名为 Sage 的实时智能体信任层，监控进出你智能体的每一个连接，让它们能够自由运行，而无需担心数据泄露或提示词注入的风险。
- Snyk 正将其在软件包安全方面的经验带入技能领域。

我们正在构建一个审计排行榜，为每个技能提供评估和建议。目标不是封锁一切，而是让你能够自信地快速前进。我们一直在寻找新的安全合作伙伴，他们能为审计技能带来独特的视角，并为技能提供更多的信任信号。

### 演示环节给我们的启示

周二晚上，八位合作伙伴展示了演示，有几个主题反复出现。

**技能填补了训练数据截止日期的空白。** Ben Davis 进行了一项对照实验来证明这一点。

他尝试用四种不同的方式让编码智能体实现 Svelte 远程函数（一个相对较新的 API）：无上下文、带有文档的技能文件、指向 MCP 的技能，以及项目中的代码示例。

所有带有上下文的方法都奏效了。

无上下文的运行（他不得不使用一个精简版模型来防止其推断解决方案）产生了完全错误的输出。当你给模型提供正确的模式时，它们足够聪明，能够正确使用这些模式。没有上下文，它们就会退回到过时的训练数据。

**内容比媒介更重要。** Ben 实验的一个有趣启示并非技能是唯一途径。关键在于获取正确的上下文，而如果你还没有一个基线，技能是最快的起点。现有的代码示例、内联文档和 MCP 提示都有效。

技能只是向任何人分发该上下文的最简单方式。

**智能体现在可以驱动整个技术栈。** 来自 Expo 的 Evan Bacon 展示了完全由 Claude Code 使用 Expo 技能驱动的原生 iOS 功能升级。

新的 SwiftUI 组件、手势驱动的过渡效果和标签栏更新都是自动应用的。他们还在一个开发中的技能中使用了 LLDB 集成，让智能体能够读取原生 iOS 视图层次结构，并自动修复 notoriously 难以处理的键盘处理错误。

他们的生产应用 Expo Go 现在会在每次崩溃发生时自动修复。对于任何花费时间与 Xcode 搏斗过的人来说，这是一个重要的声明。

**技能正在成为基础设施。** Nick Khami 展示了 Mintlify 为他们托管的每个文档站点（包括 Claude Code 自己的文档、Coinbase、Perplexity 和 Lovable）自动生成一个技能。

访问这些站点的流量中，现在有 50% 来自编码智能体，而一年前这一比例是 10%。技能不再是文档团队编写的东西；它是拥有结构良好文档的副产品。Sentry 的 David Cramer 构建了 Warden，这是一个通过 GitHub Actions 在拉取请求上将技能作为代码检查器运行的框架，将智能体视为一个静态分析层。

### 我们的发展方向

Vercel 的 CEO Guillermo Rauch 在周二晚上说了一句让我反复思考的话：智能体会犯错。

它们有时会告诉你你绝对正确，然后却去做错误的事情。在 AI 时代，交付质量不仅仅意味着庆祝你烧掉了多少 Token。它意味着提高这些 Token 实际产出的标准。

技能是解决这个问题的一个答案。它们是我们影响智能体创造内容、使其跟上框架变化、并通过为它们提供直达正确答案的路径（而不是让它们四处摸索）来提高其 Token 效率的方式。

两百万次安装是真实的信号。安全合作伙伴关系使其成为团队可以依赖的东西。而演示表明，最有趣的技能工作并不在 CLI 层面。而是在那些现在将技能视为大规模分发知识的一等公民的智能体和工具中。
