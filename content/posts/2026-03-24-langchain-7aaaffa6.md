Moda 是一款面向非设计师的 AI 原生设计平台，用户包括营销人员、创始人、销售人员和小企业主等。即使没有设计背景，他们也能借助 Moda 制作专业水准的演示文稿、社交媒体帖子、宣传册和 PDF 文件。可以将其理解为 Canva 或 Figma，但它配备了一个类似 Cursor 风格的 AI 侧边栏，能够直接在完全可编辑的 2D 矢量画布上构建和迭代设计。

![Moda chat](/images/posts/3efc6076c019.png)

Moda 的核心是一个基于 Deep Agents 构建的多智能体系统，LangSmith 提供了可观测性层，使团队能够快速迭代并充满信心地交付产品。

## 挑战：让 AI 擅长视觉设计

AI 代码生成效果良好，部分原因在于 HTML 和 CSS 本身已有如 Flexbox 和网格这样的布局抽象。开发者描述的是关系和相对大小，而非像素坐标。

视觉设计则没有对等的抽象。最接近的标准是 PowerPoint 的 XML 规范，这是一个已有 40 年历史的格式，充斥着冗长、绝对的 XY 坐标，而众所周知，LLM（大语言模型）极不擅长对此类信息进行推理。使用 XML 的工具生成的设计，看起来和其他所有 AI 生成的演示文稿如出一辙。

Moda 需要一个能够生成真正美观设计的系统，以及一个能够以生产级质量处理复杂的、多轮次的、基于视觉的任务的智能体架构。

## 智能体框架：基于 Deep Agents 构建

Moda 的 AI 系统由三个智能体组成：

1.  **设计智能体**：Cursor 风格侧边栏背后的主要智能体，负责处理画布上所有实时的设计创建和迭代工作。
2.  **研究智能体**：从外部来源（例如公司网站）获取结构化内容，并将其存储到 Moda 内每个用户的文件系统中。
3.  **品牌工具包智能体**：从网站、上传的品牌手册或现有的幻灯片中提取品牌资产（颜色、字体、标志、品牌声音），从而让每个设计从一开始就符合品牌调性。

**研究智能体和品牌工具包智能体**都运行在 Deep Agents 上。这是团队最新的智能体，他们对此投入了大量资源。**设计智能体**运行在一个定制的 LangGraph 循环上——这是在 Deep Agents 之前构建的旧实现——团队正在积极评估是否也将其迁移过来。

这三个智能体共享相同的整体架构：一个轻量级的分类步骤、一个主智能体循环、动态上下文加载以及在 LangSmith 中的完整追踪。

## 上下文工程：至关重要的细节

要让一个设计智能体产出真正优秀、视觉连贯且品牌准确（而不仅仅是技术正确）的输出，需要进行大量有意识的上下文工程。

以下是 Moda 的发现。

### 使用自定义 DSL 而非原始场景图

构建设计智能体最困难的部分之一，是找到一种能让 LLM 有效推理的视觉布局表示方法。原始画布状态冗长、坐标密集且 Token 消耗巨大——这与模型思考结构和布局的方式并不契合。

Moda 开发了一个上下文表示层，为智能体提供了一个更清晰、更紧凑的画布视图，从而降低了 Token 成本并提高了输出质量。具体细节是专有的，但其通用原则与让 LLM 在网页开发中表现出色的原理相同：为模型提供可以推理的布局抽象，而非原始的数字坐标。

"LLM 不擅长数学。PowerPoint 的 XML 规范包含一堆 XY 坐标——这是数据的一种良好表示，但对于 LLM 来说，这不是描述它希望元素放置位置的理想方式。" — Ravi Parikh, Moda.app 联合创始人

Deep Agents 和 LangSmith 在此至关重要。团队广泛使用 LangSmith 追踪来评估不同的上下文表示如何影响智能体行为，迭代决定包含哪些信息、如何组织信息，以及在哪里设置缓存断点能对成本和延迟产生最大影响。

### 分类 → 技能 → 主循环

每个请求在到达主智能体之前，都会经过一个轻量级的分类节点（使用快速且便宜的 Haiku 模型）。分类节点会确定输出格式（幻灯片、PDF、LinkedIn 轮播图、标志等），并预加载相关的**技能**。这些技能是包含设计最佳实践、格式指南和特定任务创意说明的 Markdown 文档。

技能以人类消息的形式注入，并在系统提示词之后和技能块之后设置了**提示词缓存断点**。这使得系统提示词保持固定并始终被缓存，同时允许每个请求动态注入上下文。

如果主智能体在循环中判定需要，它也可以动态拉取额外的技能。分类步骤只是预先加载高置信度的技能，以避免不必要的额外轮次。

### 动态工具加载

设计智能体在上下文中始终运行着 12-15 个核心工具。另外还有大约 30 个工具可通过需求动态加载，智能体在识别到特殊需求（例如解析上传的 PowerPoint 文件）时，可以调用一个名为 `RequestToolActivation` 的工具来激活它们。

每个额外工具在提示词前缀中会增加 50-300 个 Token 的成本，并且加载新工具会破坏提示词缓存。但计算结果是合理的：绝大多数请求不需要额外的工具，因此保持上下文精简总体上是有利的。

"如果只看数据，大多数请求不需要激活任何额外工具，而且只在上下文中保留 12 到 15 个工具的感觉真的很好。" — Ravi Parikh

### 根据画布大小扩展上下文

并非每个请求都需要完全了解整个项目。对于较小的画布，智能体可以处理当前状态的完整视图。对于较大的项目——例如一个 20 页的幻灯片——Moda 会动态管理智能体接收的上下文量，为其提供更高级别的摘要，并允许其根据需要通过工具拉取细节。

这使得 Token 使用量受到控制，同时又不牺牲智能体在复杂的多页面项目中做出明智设计决策的能力。LangSmith 按节点进行的成本追踪使得在上下文丰富度和效率之间找到合适的平衡点变得简单直接。

## 用户体验：设计的 "Cursor 时刻"

Moda 最深思熟虑的产品选择之一是其交互模型。与 AI 生成静态输出然后交付给用户的"生成-替换"流程不同，Moda 的 AI 直接在完全可编辑的 2D 矢量画布上工作。智能体创建的每个元素都可以立即被用户选择、移动、调整大小和修改样式。

这将用户与 AI 的关系从"接受或拒绝"转变为真正的协作。AI 生成一个坚实的起点，用户对其进行完善。双方都不必承担所有工作。

Cursor 风格的侧边栏强化了这一点：它始终存在，始终能感知画布上的内容，并且专为迭代式的来回交互而非一次性生成而设计。特别是对于非设计师来说，这消除了面对空白画布的恐惧，同时让他们能够掌控最终结果。

## 使用 LangSmith 实现可观测性

由于所有三个智能体都通过 LangSmith 进行追踪，Ravi 对每次执行都有完全的可见性。他在积极开发时总是保持其开启。

关键工作流程包括：

*   **提示词与工具迭代**：进行更改，运行查询，立即调出追踪记录，准确查看智能体做了什么以及为什么这样做。
*   **成本追踪**：按节点细分 Token 成本，使昂贵的步骤一目了然。
*   **缓存命中分析**：考虑到动态技能和工具加载，这一点尤为重要；能快速显示缓存有效和失效的位置。
*   **错误诊断**：在工具调用失败和意外的模型行为演变为面向用户的问题之前，将其暴露出来。

"如果我在迭代提示词，如果我在迭代工具集，我会进行更改，运行查询，然后在 LangSmith 中调出那个追踪记录，看看发生了什么……它让我们行动得更快。" — Ravi Parikh

Moda目前尚未进行正式评估，但这已在规划路线图中。目前，LangSmith追踪记录作为主要的反馈循环，用于捕捉回归问题并验证改进效果。
