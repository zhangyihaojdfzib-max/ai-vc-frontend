Vercel的AI网关现已支持**备用模型**功能，用于在主模型失效或不可用时自动切换。除了防范供应商级别的故障外，模型备用机制还能处理模型间的错误与能力不匹配问题（例如多模态、工具调用等）。

备用模型将按指定顺序逐一尝试，直至请求成功或所有选项耗尽。任何错误（如上下文限制、不支持的输入或供应商服务中断）都可能触发备用切换。请求将根据最终成功响应的模型进行计费。

以下示例展示了主模型不支持多模态能力时，自动回退至具备该能力模型的情况。使用时，请在`providerOptions`中的`models`参数内指定备用模型列表：

```
1import { streamText } from 'ai';2
3const result = streamText({4  model: 'openai/gpt-oss-120b', 5  prompt: 'Parse the attached PDF for tables and graphs, \6  and return the highest performing categories this year',7  providerOptions: {8    gateway: {9      models: [10        'google/gemini-2.5-pro',11        'anthropic/claude-sonnet-4.5',12        'meta/llama-3.1-8b'13      ], 14    },15  },16})
```

若需在模型路由基础上同时预设供应商路由，可在`providerOptions`中同时指定`models`和供应商列表（通过`order`或`only`参数）：

```
1import { streamText } from 'ai';2
3const result = streamText({4  model: 'openai/gpt-5-nano', 5  prompt: 'Parse the attached PDF for tables and graphs, \6  and return the highest performing categories this year',7  providerOptions: {8    gateway: {9      order: ['vertex', 'cerebras'], 10      models: [11        'google/gemini-2.5-flash',12        'openai/gpt-oss-120b'13      ], 14    },15  },16})
```

AI网关还内置了**可观测性**功能，支持**自带密钥**，并提供**OpenAI兼容的API**接口。
