# 使用 Agent Builder 模板即时部署智能体

隆重推出 Agent Builder 模板库：这是一个为常见任务准备的、可即时部署的智能体集合，并配备了您已在使用的工具。

LangSmith Agent Builder 允许任何人通过一个简单的提示词来构建智能体。要求它为您构建一个市场研究智能体，它会通过提出相关问题来创建您所需的内容。

但有时您希望从一个现成的方案开始。今天我们推出 **Agent Builder 模板库**，并扩展了我们的工具集成，以帮助您更快地从想法过渡到可运行的智能体。

Agent Builder 模板是为常见工作预先构建的智能体，已连接好工具并包含了智能体指令。它们已准备就绪，可以部署，并且完全可定制。您可以更新智能体的指令、添加工具并设置审批要求。

与传统的工作流自动化不同，您无需规划每一步，也无需花费数小时调试更改。只需像对待队友一样给您的智能体反馈，它就会学习。

我们与最了解其领域的公司共同构建了这些模板，包括 Tavily、PagerDuty、Exa、Box 和 Arcade，并且我们会定期添加新模板。[探索模板库](https://smith.langchain.com/agent-builder/templates)

今天就试试这些智能体模板：

-   日历简报（Google Calendar）：每天早晨查看您的日历，并向您发送包含会议参与者研究信息的摘要。
-   邮件助手（Gmail）：对您的电子邮件进行分类，并起草回复供您审批。
-   事件响应器（PagerDuty）：分析警报，交叉参考您的运行手册，并推荐行动方案。
-   文件接收审核（Box）：审核提交的文件，并准备摘要供您审批。
-   人才寻源（Exa）：根据您的职位描述搜索 LinkedIn，并发送推荐的候选人资料。
-   竞争对手研究（Tavily）：进行深入的竞争研究，并提供简洁的报告。
-   社交媒体监控（X + Slack）：监控 X，并将包含最新消息的每日摘要发送到 Slack。

### 为您的智能体选择最佳模型

无论您是从提示词还是模板开始，不同的工作可能需要不同的模型。成本、延迟和推理要求会因您的智能体是在夜间总结邮件还是实时回答问题而有所不同。这就是为什么 Agent Builder 不会将您锁定在单一模型上。它支持 OpenAI、Anthropic 和 Google Gemini 模型，以及任何遵循 OpenAI 或 Anthropic 规范的自定义或开源模型。

为了展示可能性，Baseten 使用他们的 GLM 4.7 模型构建了一个智能体，该模型能够快速响应以实现实时用户交互。将您首选的模型提供商连接到 Agent Builder，然后就可以开始构建了。
