# 聚焦智能体

过去九个月，人工智能领域涌现了几波明显的创新浪潮，每一波都拓展了我们对可能性的认知。去年夏天，Stable Diffusion 带来了图像生成的时刻。随后在十一月，ChatGPT 将我们的集体注意力转向了 LLM（大语言模型），而随着 GPT-4 及众多其他模型的发布，这股热潮只增不减。LangChain 普及了 Agent（智能体）的概念，如今我们看到这一进程的下一步——自主智能体——正在兴起。AutoGPT 在 4 月 21 日 GitHub 星标数突破了 10 万（超过了 Go、Kubernetes 或 Node.js），此时距离其发布还不到一个月。BabyAGI 也获得了 1.2 万星标，并激发了一系列项目。

自主智能体可以被视为由语言模型驱动的机器人，它们能够分解复杂问题并迭代解决，代表用户采取行动。我们可以用一个简单的例子来说明仅使用 LLM、使用智能体以及使用自主智能体分别能实现什么。仅使用 LLM，我们可以查找某个城市的最佳餐厅。使用智能体，我们可以让它查找评分最高且有可用餐位的餐厅，并预订两人位。使用自主智能体，我们可以要求它找到一家最适合我的日程安排和偏好的餐厅，然后为我和我最好的朋友预订。自主智能体通过将任务分解为子任务，并在每一步之间利用记忆来指导其行动，从而完成此类任务。

不同的人对此有不同的看法：自主智能体要么将被证明是一次持久的范式转变——预示着通用人工智能（AGI）可能的前景——要么只是众多迭代方法中的又一个瞬间。

不难理解为何自主智能体会捕获我们的集体想象力。它们让我们很容易就能想象，在 LLM 之上构建轻量级的小型应用能实现什么。我们自己也曾尝试启动智能体——探索了从电子邮件摘要（从抓取谷歌趋势到综合与总结）到复杂的旅行行程规划等各种应用。

尽管这些例子可能引人入胜，但自主智能体在目前状态下仍有许多不足之处。在性能、用户控制和输出质量方面都有很大的改进空间。目前仍处于早期阶段，智能体要想实现大规模应用，至少需要克服三个重大障碍：

1.  **逻辑推理 ≠ 良好执行**：原则上，GPT-4 能够进行思维链推理并将任务分解为多步骤流程。但在实践中，智能体常常难以独立执行其子任务。它们难以判断何时应该“退一步思考”，导致陷入循环执行同一任务的困境，或者可能“幻觉”出一个步骤并因此卡住，因为缺乏外部反馈。
2.  **计算成本**：这类应用的架构依赖于递归循环，这可能导致对 LLM 的多次重复调用。目前，使用 OpenAI 的 API 等工具，每次调用的成本相对较低（但可能会遇到 API 限制！）。然而，对于自建模型而言，成本等式可能大不相同。
3.  **学习能力**：由于自主智能体是临时启动且后续不再复用，它们无法从提示词或先前的尝试中学习，也很少从错误中吸取教训。不过，帮助智能体持久化运行的服务即将出现，这应该会使管理它们变得更加容易。

如果我们能解决其中一些挑战，就可以设想一个“智能体间交互”的未来。可以为常见任务创建专门的智能体。与其为每项任务都启动一个新智能体，不如将某些步骤“外包”，依赖预训练的智能体来完成按输出付费的任务，并将这些输出作为下一步的输入。换句话说，你的人工智能可以雇佣或外包给另一个人工智能。不同的智能体可以覆盖“核心”任务，而新的工具层可能作为“粘合剂”出现，将整个过程整合在一起。

要实现这一目标需要什么？当前的实现依赖于 GPT-4 API，由上下文窗口有限的短期智能体使用。为了充分发挥其潜力，下一代智能体需要具备以下特性：

*   **计算感知**：将最小化资源使用作为目标函数。
*   **数据感知**：为任务寻找并连接到正确的模型或数据源。
*   **智能体感知**：寻找、复用并与智能体生态系统进行通信。
*   **安全感知**：检查输出和沙箱化代码是第一步，还需要更严格的控制来防止滥用。
*   **用户感知**：从用户行为和偏好中学习，以优化性能。

我们认为自主智能体可能成为人工智能应用领域中一个有趣的部分，而这项技术才刚刚开始成熟。智能体的演进将会是什么样子？**告诉我们**你正在构建什么！
