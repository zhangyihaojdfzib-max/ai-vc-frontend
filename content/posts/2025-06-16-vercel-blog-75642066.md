# Roboto Studio 如何简化基础设施、改善开发者体验，并为 Tray.ai 整合数千个页面

#### 99.86%

构建时间缩短

#### 超过100万

月页面浏览量

Tray.ai 是一个可组合的 AI 集成与自动化平台，企业可用其大规模构建智能、安全的 AI Agent（智能体）。

为了现代化其营销网站，他们与 Roboto Studio 合作，从原有遗留解决方案和过时的 Next.js 版本迁移。目标：简化架构，整合分散的代码仓库，将内容和表单管理统一到一个系统中。

迁移到 Vercel 后，构建时间从一整天缩短到仅两分钟。

## 从脆弱的流水线到快速部署

迁移前，Tray.ai 的基础设施受限于遗留决策。较旧版本的 Next.js 搭配 Serverless、250MB 的包构建，以及僵化的开发-测试-生产流水线，使开发速度极其缓慢。预览变更需要完整部署。表单逻辑位于独立的代码仓库中。构建频繁失败，即使成功也需要一整天时间。

Roboto Studio 使用 Next.js 从头重建了网站。他们将所有内容整合到单个 Turborepo 单体仓库中，采用自定义页面解析和深度嵌套的动态路由，以处理超过50万个页面的复杂性。

在 Vercel 上，团队现在对每个分支推送使用基于 Git 的自动部署。开发者可以通过模拟生产环境的预览部署查看变更，从而在合并前发现并修复问题。

## 统一的基础设施，更少的未知因素

新的网站架构采用混合方法，结合静态站点生成（SSG）、服务端渲染（SSR）和客户端渲染（CSR），每种方式在合适的场景下使用。Roboto 还使用 Vercel Functions 重建了 Tray 的表单引擎，并集成 reCAPTCHA 以增强保护。

借助 Vercel Analytics 和 Speed Insights，团队现在可以了解真实用户如何与网站交互。Vercel Dashboard 中的性能洞察成为核心指标。

此外，可靠性大幅提升。团队现在能立即捕获构建错误，无需再层层排查或追踪失败的部署。过去需要额外步骤和时间的问题，现在发生时即可发现，使修复更快、更直接。

## 边缘的弹性

当团队通过 Firewall Observability 发现针对动态 SSR 路由的 DDoS 攻击时，他们迅速将最易受攻击的页面重构为静态缓存版本。问题在无需任何基础设施干预的情况下得到解决。这种灵活性内置于架构中，由 Vercel 的 Edge Network 提供支持。

他们现在每月服务超过100万页面浏览量，且架构设计具备可扩展性。

## 成果

Tray 的迁移不仅仅是修复构建失败，更是解锁更好的工作方式。借助 Vercel 上的现代 Next.js 架构，他们现在部署更快，通过预览部署更快验证想法，并利用实时可见性调试问题。

将多个代码仓库整合为单一代码库降低了复杂性，使表单、内容和共享逻辑的管理更加便捷。结合分析、性能洞察和边缘缓存，新栈为 Tray 提供了更快、更可靠的基础，专为迭代、扩展和速度而构建。
