## Claude Code 现已支持直接发布博客文章

这篇文章是在 Claude Code 会话中撰写，并直接以草稿形式发布到 feld.com 的。没有复制粘贴，也没有通过邮件发送给自己。我只是在 iterm2 中输入了 `/blog-feld` 命令，文章就出现在了我的博客上。

整个设置过程大约花了十分钟。

我让 Claude 研究如何连接到 feld.com（托管在 WordPress.com 上）以实现直接发布。它研究了三种方法：WordPress.com REST API、官方的 WordPress MCP 连接器，以及 WordPress 插件 MCP Adapter。WordPress MCP 连接器是只读的（因此对发布无用）。MCP Adapter 仅适用于自托管的 WordPress（不适用于 WordPress.com）。这样就只剩下使用 OAuth 的 REST API 了。

Claude 编写了一个名为 `/blog-feld` 的命令来处理整个工作流程：查看我在当前对话中讨论的内容，将其整理成一篇博文，向我展示摘要，与我进行交互式编辑，然后将其作为草稿推送到 feld.com。

它从不直接发布——我仍然会在点击发布按钮前，在 WordPress 编辑器中审核所有内容。

对于身份验证，WordPress.com 要求使用 OAuth。通常，我设置这个的经验是繁琐的。但这次，Claude 只是一步一步地告诉我该怎么做。

– 我在 developer.wordpress.com 注册了一个应用（获取客户端 ID 和密钥）。
– Claude 设置了授权码流程。
– 我访问了一个 URL，点击“批准”，然后浏览器重定向到 localhost，并在 URL 中附带了一个授权码。
– 页面本身没有加载，但授权码就显示在地址栏里。
– 我截取了页面截图，将其粘贴到 iterm2 中，然后 Claude 用它换取了访问令牌。

完成。

为了验证是否成功，Claude 从 API 拉取了我的最后三篇博文：“Tech I’m Obsessed With”、“Blurry Transitions” 和 “Interview With Guy Kawasaki”。

这是一件小事，但正是这种小事能改变行为模式。每天当我使用 Claude Code 工作时，我都会想到很多类似的事情。我不再等待别人来实现它，或者付费使用第三方服务，而是直接在 Claude Code 中创建它，并使其成为我工作环境的永久组成部分。

反正我最近也越来越多地在 Claude Code 会话中写作——构思想法、编辑和迭代。之前的障碍总是在最后一步：复制文本，打开 WordPress，粘贴进去，格式化，修复被破坏的格式。现在，这一步消失了。

在对话中思考与为博客写作，变成了一回事。
