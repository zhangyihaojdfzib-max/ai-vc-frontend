# 推出 Codex 应用

通过全新的 macOS 版 Codex 应用，拓展开发者所能实现的功能。

2026年3月4日更新：Codex 应用现已支持 Windows 平台。

今天，我们正式推出 macOS 版 Codex 应用——一个强大的全新界面，旨在轻松同时管理多个 Agent（智能体）、并行运行任务，并与 Agent（智能体）在长时间运行的任务上进行协作。

我们也很高兴让更多人了解 Codex 现在所能实现的功能。在限定时间内，我们将 Codex 纳入 ChatGPT Free 和 Go 套餐，并将 Plus、Pro、Business、Enterprise 和 Edu 套餐的速率限制提升一倍。这些更高的限制适用于您使用 Codex 的所有场景——无论是在应用中、通过 CLI、在 IDE 中，还是在云端。

Codex 应用改变了软件的构建方式以及能够参与构建的人群——从与单个编码 Agent（智能体）配对进行针对性编辑，到监督协调的 Agent（智能体）团队完成设计、构建、发布和维护软件的整个生命周期。

## Codex 应用：Agent（智能体）的指挥中心

自 2025 年 4 月推出 Codex 以来，开发者与 Agent（智能体）协作的方式已发生根本性改变。模型现在能够端到端地处理复杂、长时间运行的任务，开发者正在跨项目编排多个 Agent（智能体）：委派工作、并行运行任务，并信任 Agent（智能体）承担可能持续数小时、数天或数周的大型项目。核心挑战已从 Agent（智能体）能做什么，转变为人们如何大规模地指导、监督和协作——现有的 IDE 和基于终端的工具并非为支持这种工作方式而构建。

这种新的构建方式结合新的模型能力，需要一种不同的工具，这正是我们推出 Codex 桌面应用（一个 Agent（智能体）指挥中心）的原因。

### 并行与多个 Agent（智能体）协作

Codex 应用为与 Agent（智能体）进行多任务处理提供了一个专注的空间。Agent（智能体）在按项目组织的独立线程中运行，因此您可以在不同任务之间无缝切换而不会丢失上下文。该应用允许您在线程中审查 Agent（智能体）的更改，对差异发表评论，甚至在编辑器中打开以进行手动修改。

它还内置了对工作树的支持，因此多个 Agent（智能体）可以在同一个仓库上工作而不会产生冲突。每个 Agent（智能体）在代码的隔离副本上工作，允许您探索不同的路径，而无需追踪它们如何影响您的代码库。当 Agent（智能体）工作时，您可以在本地检出更改，或者让它继续推进，而不影响您本地的 git 状态。

该应用会从 Codex CLI 和 IDE 扩展中获取您的会话历史和配置，因此您可以立即开始将其用于现有项目。

### 通过技能超越代码生成

技能将指令、资源和脚本捆绑在一起，以便 Codex 能够可靠地连接到工具、运行工作流，并根据您团队的偏好完成任务。Codex 应用包含一个专门的界面来创建和管理技能。您可以明确要求 Codex 使用特定技能，或者让它根据手头的任务自动使用。

### 通过自动化委托重复性工作

借助 Codex 应用，您还可以设置自动化，让 Codex 按照自动计划在后台工作。自动化将指令与可选技能相结合，按您定义的计划运行。自动化完成后，结果会进入审核队列，以便您在有需要时重新介入并继续工作。

### 契合您工作方式的个性

开发者在与 Agent（智能体）协作时有着不同的偏好。有些人想要一个直接、以执行为重点的伙伴；另一些人则更喜欢更具沟通性、更互动的交互。Codex 现在允许开发者在两种个性之间进行选择——一种简洁务实，另一种更具对话性和同理心——而能力保持不变，以契合您最喜欢的方式。只需在应用、CLI 和 IDE 扩展中使用 /personality 命令即可。

## 默认安全，设计可配置

我们在整个 Codex Agent（智能体）堆栈中集成了安全设计。Codex 应用使用原生、开源⁠（在新窗口中打开）且可配置的系统级沙箱，与 Codex CLI 相同。默认情况下，Codex Agent（智能体）仅限于编辑其工作所在文件夹或分支中的文件，并使用缓存的网络搜索，然后在需要运行需要更高权限（如网络访问）的命令时请求许可。您可以为您的项目或团队配置规则⁠（在新窗口中打开），允许某些命令自动以更高权限运行。

## 未来展望

企业和开发者越来越依赖 Codex 进行端到端开发。自 12 月中旬 GPT‑5.2‑Codex 发布以来，Codex 的整体使用量翻了一番，而在过去一个月中，已有超过一百万开发者使用过 Codex。我们将继续扩展开发者使用 Codex 的场景和方式，包括在 Windows 上提供该应用、推动模型能力的前沿发展，以及推出更快的推理。

在应用内，我们将根据实际反馈不断优化多 Agent（智能体）工作流程，使管理并行工作以及在 Agent（智能体）之间切换而不丢失上下文变得更加容易。我们还在构建支持云端触发的自动化功能，以便 Codex 可以持续在后台运行——而不仅仅是在您的电脑开机时。

Codex 建立在一个简单的理念之上：一切由代码控制。一个 Agent（智能体）在推理和生成代码方面越出色，它在所有形式的技术和知识工作中的能力就越强。然而，当前的一个关键挑战是前沿模型的能力与人们在实际中轻松使用它们之间的差距。Codex 旨在弥合这一差距，使人们更容易指导、监督和应用我们模型的全部智能来完成实际工作。我们专注于让 Codex 成为最佳的编程 Agent（智能体），这也为其成为能够处理超出编写代码范围的广泛知识工作任务的强大 Agent（智能体）奠定了基础。我们期待看到您用 Codex 构建的成果！
