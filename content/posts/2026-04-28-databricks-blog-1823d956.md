- Genie Code 是一个专为数据领域构建的自主 AI 伙伴  
- 数据工程师可以直接在 Lakeflow 中使用 Genie Code，从在 Pipeline Editor 中构建管道到在 Lakeflow Jobs 中编排工作流  
- Genie Code 支持完整的数据工程生命周期——从开发和编排到监控和调试——全部在一个 Agent 体验中完成  

借助 Genie Code，数据工程师可以使用自然语言生成生产就绪的数据管道，通过作业进行编排，并调试故障。过去需要数周才能完成的任务——查找数据、构建转换、拼接作业以及修复故障——现在可以在数小时内完成，同时保持与治理和运营标准的一致性。

下面，我们将介绍这在实际中如何运作：发现数据、构建管道、编排作业以及调试故障，全部通过一次对话完成。

## 使用自然语言构建并编排完整、生产就绪的管道和作业

Genie Code 现在可以在一个线程中带你从探索阶段直达计划好的管道和作业，帮助你端到端地编写和运营它们。

它加速了 Lakeflow Spark Declarative Pipelines 的开发，并简化了通过 Lakeflow Jobs 编排和运行管道及笔记本的方式。Genie Code 理解你的管道和作业上下文，能够访问代码、配置和运行结果。

Genie Code 在数据工程生命周期的关键阶段提供帮助：

- **搜索数据资产，而不仅仅是代码**：Genie Code 使用流行度、血缘关系、代码示例和 Unity Catalog 元数据来识别与你的任务最相关的数据集。例如，你可以让 Genie Code 解释表之间的关系，或追踪数据如何流经管道。在 SiriusXM，团队使用 Genie Code 更快地理解表关系。
- **构建和修改管道**：首先用自然语言描述你想要的管道，例如基于奖章架构构建的欺诈检测管道。Genie Code 会生成一个包含 Bronze、Silver 和 Gold 层的 Spark Declarative Pipeline，包括源、转换、数据质量期望和输出。之后，你可以要求更改、审查建议的差异，并运行和测试管道。
- **定义和编排作业**：无需手动定义和维护编排逻辑。你描述所需的作业，包括任务、依赖关系和调度计划。Genie Code 会为你配置它，然后帮助用自然语言修改、调试和修复编排问题。
- **扩展和演进现有工作流**：随着需求变化，Genie Code 帮助你用新的数据集和转换更新管道和作业。它理解你管道的当前结构和结果，并可以通过编写用于变更数据捕获的 AutoCDC 流程、配置 Auto Loader、应用数据质量期望以及遵循奖章架构来扩展它们。
- **采用声明式自动化捆绑包（DABs）的最佳实践**：Genie Code 可以直接在你现有的 DABs 项目中工作：添加资源、更新配置、验证捆绑包并部署到目标环境。因此，你可以为数据项目采用软件工程最佳实践，如源代码控制、测试和 CI/CD，而无需手动编写 YAML。
- **在不降低标准的情况下更快工作**：这些能力减少了手动工作，同时保持工作流与企业需求一致。管道通过 Unity Catalog 保持治理，并遵循性能和数据的既定模式，而作业则继承一致的调度、重试和依赖关系配置。数据工程师保持控制，但花在重复性工作上的时间更少。

## 监控、诊断和调试管道与作业

- **理解和改进管道行为**：Genie Code 可以检查数据集和管道输出，帮助你端到端地理解管道。例如，它可以总结转换、追踪数据如何流入下游表，并突出显示行数或模式中的意外变化。
- **调试和诊断作业与管道故障**：当管道或作业失败时，Genie Code 帮助你处理问题。它分析错误，在相关文件中提出更新建议，并在应用任何更改之前向你显示差异。你可以审查每个更新并决定下一步操作。这可以将漫长的手动调试周期转变为更快的引导式迭代。
- **扩展和定制 Genie Code**：Genie Code 不限于内置功能。团队可以通过自定义指令、Agent 技能以及通过 MCP 服务器集成外部系统来扩展它，使 Genie Code 能够处理领域特定逻辑、内部工具和自定义工作流。这确保了 Genie Code 适应你的环境和领域知识。
