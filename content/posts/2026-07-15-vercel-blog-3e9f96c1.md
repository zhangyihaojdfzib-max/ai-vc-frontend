GitHub Tools 现已通过新的 `@github-tools/sdk/connect` 子路径，为 Vercel Connect 提供了一流支持。你的 Agent（智能体）不再需要存储长期有效的个人访问令牌，而是通过连接器在运行时生成短期、有作用域限定的 GitHub 令牌。无需存储、轮换或泄露任何密钥。

将 GitHub 连接器关联到你的项目，并选择一个预设：

```
1import { connectGithubTools } from "@github-tools/sdk/connect";2import { generateText } from "ai";3
4const tools = connectGithubTools("github/my-connector", {5  preset: "code-review",6});7
8const { text } = await generateText({9  model: "anthropic/claude-sonnet-5",10  tools,11  prompt: "Summarize open PRs on vercel-labs/github-tools.",12});
```

使用自定义 Agent（智能体）审查 GitHub PR

- **预设作用域**：诸如 `code-review`、`issue-triage` 和 `maintainer` 等预设会自动映射到 Connect 作用域，因此令牌仅携带工具集所需的权限。
- **与 eve 配合使用**：从 `@github-tools/sdk/connect/eve` 导入，并在 `agent/tools/` 中放置一个文件，即可注册完整的工具集并启用 Connect 支持的身份验证。
- **多租户就绪**：每次调用时可覆盖 `installationId`、`repositories` 或 `scopes`，以针对特定安装或限制对单个仓库的访问。
- **Vercel 上零配置**：部署时会自动使用 OIDC 令牌进行身份验证。对于本地开发，运行 `vercel link` 和 `vercel env pull`。
