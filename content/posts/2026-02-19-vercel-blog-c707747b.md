Vercel 现已支持通过 API 和 CLI 以编程方式访问账单用量与成本数据。新的 `/billing/charges` 端点以 **FOCUS v1.3** 开放标准格式返回数据，使团队无需自定义转换逻辑即可将成本数据导入 FinOps 工具。

该端点支持 1 天粒度，最大日期范围为一年。响应以换行符分隔的 JSON（JSONL）格式流式传输，以高效处理大型数据集。

SDK 用法

```
1import { Vercel } from "@vercel/sdk";2
3const vercel = new Vercel({4  bearerToken: "<YOUR_BEARER_TOKEN_HERE>",5});6

15  for await (const event of result) {16    17    console.log(event);18  }19}20run();
```

curl 用法

```
1curl -N --request GET \2  --url 'https://api.vercel.com/v1/billing/charges?teamId=<team>' \3  --header 'Authorization: Bearer <token>' \4  --header 'Accept-Encoding: gzip' \5  --compressed
```

CLI 用法

如需快速查看，`vercel usage` 命令可显示当前周期或自定义日期范围内的账单用量。这包括每项服务的信用使用情况和成本。

查看当前计费周期的用量

```
1vercel usage
```

查看自定义日期范围的用量

```
1vercel usage --from 2025-01-01 --to 2025-01-31
```

Vantage 也已发布原生集成，可将 Vercel 团队连接到 Vantage 账户。这会自动同步用量和成本数据以及其他工具，简化成本可观测性。详情请阅读 **Vantage 公告博客**。

了解更多信息，请查阅 **API 文档** 和 **CLI 参考**。
