## 链接到标题连接模型

Agent（智能体）并非运行在单一模型上。每个任务都有不同的成本、延迟和能力权衡，正确的选择取决于Agent（智能体）正在做什么。它需要一个接口来访问任何模型，一种在它们之间进行路由的方式，以及一种将结果流式传输回用户的方法。

AI SDK为Agent（智能体）提供了一个接口来调用任何模型，而AI Gateway则通过单个端点路由到数百个模型。

### 链接到标题AI SDK

每个实验室都通过自己的API暴露模型调用。流式传输、工具调用、结构化输出以及请求的格式各不相同，因此您支持的每个供应商都需要额外构建和维护一个集成。

AI SDK是用于构建AI应用、Agent（智能体）和框架的单一接口。它与平台、框架和模型无关，允许您生成文本、图像、语音、视频等。

```

3const { text } = await generateText({4  model: 'anthropic/claude-sonnet-4.6',5  prompt: 'Summarize the latest deploys.',6});7
8console.log(text);
```

通过更改一个字符串来切换模型，而不是更改您的代码。

### 链接到标题AI Gateway

Token现在是一种生产依赖，就像带宽之于网络一样，而Agent（智能体）根据任务使用不同的模型。跨实验室的集成意味着需要管理来自不同供应商的独立密钥和计费，这些供应商往往价格昂贵、有速率限制且不断变化。

AI Gateway是Token的CDN，在我们运行了十多年的全球网络上路由它们。它通过单个端点路由每个调用，在供应商宕机时进行故障转移，并跟踪所有供应商的成本和使用情况。您支付供应商的价格，无任何加价，并且可以使用您自己的密钥。

SERHANT.使用单个密钥运行三个模型，将市场分析发送给Claude，营销文案发送给GPT，图像生成发送给Gemini。

## 链接到标题执行复杂工作流

Agent（智能体）按顺序处理任务，有时需要几分钟或几小时，这些任务通常要求它们在安全环境中执行代码和其他操作。

Workflow SDK使Agent（智能体）运行具有持久性，而Vercel Sandbox为Agent（智能体）提供其独立的虚拟机。

### 链接到标题Workflow SDK

当Agent（智能体）工作流深处的某个步骤失败且无法恢复时，整个作业将重新开始，重新运行您已付费的每个模型调用。解决持久性问题意味着您需要自行构建和维护重试机制、状态持久化和编排。

Workflow SDK对每个作业的每个步骤进行检查点保存，保持状态，重试失败的步骤，并在需要等待人工、慢速API或Webhook时暂停。运行从最后一个成功的步骤恢复，而不是从头开始。

FLORA将其创意Agent（智能体）构建在Workflow SDK上，单个创意会话会扩展到五十多个图像模型。每个步骤都会持久化并在失败时重试，因此长时间运行永远不会丢失其状态。

### 链接到标题Vercel Sandbox

Agent（智能体）读取文件、运行命令和编写代码。这种自由是它们能力强大的原因。但如果没有约束，这也是一种风险。代码未经审查，命令可能出错，一个错误的步骤可能会触及它永远不应该触及的东西。

Vercel Sandbox为每个Agent（智能体）提供其自己的微虚拟机，一个完整的Linux计算机，包含文件系统、Docker支持及其自己的内核，与宿主机和其他所有沙箱隔离。凭据仅在Agent（智能体）的代码调用服务时才被注入，因此它可以使用所需内容而无需看到原始Token。


在一次性、隔离的微虚拟机中运行Agent（智能体）未经审查的代码。

沙箱为您和您的Agent（智能体）提供了与Vercel数十亿预览部署和每日六百万次构建相同的底层原语。

一个只与模型对话的Agent（智能体）能做的不多。要使其有用，它必须能够访问数据和外部系统，并与使用它的人进行通信。这两种连接都必须安全。

Vercel Connect为Agent（智能体）提供作用域限定、短期有效的数据和系统访问权限。Chat SDK将Agent（智能体）部署到用户已经在使用的应用中。

打开拉取请求、更新记录、查询数据仓库。要求Agent（智能体）执行这些工作意味着要授予它访问您所用平台的权限，而目前这通常意味着一个长期有效的Token，其权限范围足以涵盖Agent（智能体）可能执行的任何操作。它永不过期，并且没有人能说出是哪个用户授权了Agent（智能体）执行的哪个操作。

### 链接到标题Chat SDK

人们不会只在一个标签页中工作。他们在Slack、GitHub、Linear、WhatsApp和Discord之间切换，并期望您的Agent（智能体）出现在每个平台中。自行将其部署到这些平台意味着需要为每个平台进行不同的API集成、身份验证流程和消息格式。
