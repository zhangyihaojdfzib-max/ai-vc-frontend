## 概览

- Vega 让用户能够证明政府签发证件中的事实——年龄、身份、职业状态——而无需透露证件本身。证件始终留在设备上。
- 零知识证明在普通客户端设备上生成时间低于 100 毫秒，无需可信设置，使隐私保护的身份验证能够大规模实用。
- 折叠复用证明意味着重复出示——向不同服务或通过 AI Agent——在首次证明后跳过大部分高成本工作。
- Vega 面向移动驾驶执照和欧盟数字身份钱包等真实世界格式，采用 Rust 构建，并将于近期开源。

AI 正在改变人们与数字世界的交互方式，从 AI 驱动的助手到代表用户行动的自主 Agent。随着这些能力的增长，强数字身份的价值也在提升：用户需要可靠的方式来建立信任，无论是证明自己是人类，还是向 AI 中介服务出示凭证。政府签发的证件仍然是信任的最强基础，但当今的验证方法往往要求用户交出证件。随着 AI Agent 开始代表人类行动并与去中心化系统交互，对快速、保护隐私的凭证证明方式的需求只会增长。

这些需求已在政策层面显现。各国政府正在迅速推进数字身份规范化。欧盟数字身份（EUDI）框架旨在让所有欧盟公民都能使用数字钱包，而欧盟年龄验证蓝图和英国《在线安全法案》等举措则强制要求基于政府 ID 的年龄验证方法。应用提供商面临两难：要么使用基于 AI 的年龄估算等精度较低的方法，要么通过要求上传 ID 来牺牲用户隐私。

证件被上传、处理，有时被存储，最终（希望如此）被删除。但高调的数据泄露事件反复暴露了用户为常规验证而分享的政府 ID。这些并非极端案例，而是要求用户分享最敏感文件以证明单一信息这一系统的可预见后果。

这正是我们通过 Vega 想要回答的问题：能否在不透露证件本身的情况下，实现对证件中信息的实用化证明？

## Vega 之路：从理念到实践

零知识证明（ZKP）是实现这一目标的密码学工具。理念很简单：它允许用户证明一个声明，例如"我已年满 21 岁"，而无需透露任何其他信息。在实践中，这意味着用户可以用驾驶执照证明年龄，而验证方——无论是网站、应用还是由 AI Agent 中介的服务——都无需看到执照。证明直接作用于已签发的证件，因此签发方无需做任何更改。

这并非新概念。挑战始终在于实用性。先前的系统要么需要可信设置（且每次逻辑变更时需重复设置），要么为了避开可信设置而牺牲性能，往往在此过程中生成庞大的证明。对于实际应用，证明需要生成速度快、体积小以便快速传输，并且效率足够高以在移动设备上运行。

我们花了数年时间致力于实用化解决方案。隐私保护身份一直是贯穿始终的激励性应用，Vega 的证明系统借鉴了该方向研究的多个构建模块：

- Spartan 展示了如何高效证明 R1CS（一种用于通用证明系统表达语句的标准方式），具有简洁证明且无需可信设置。
- Nova 引入了折叠方案，让证明者能将多个计算实例压缩为一个。
- HyperNova 表明 Nova 的折叠也为零知识提供了关键构建模块：将真实实例与随机实例折叠可隐藏底层秘密数据，这项技术被称为"NovaBlindFold"。
- NeutronNova 提供了同时处理一批实例的最高效折叠方案。

Vega 将这些构建模块整合为一个统一的证明系统。关键设计目标是简洁性。Spartan、Nova 和 NeutronNova 以直接方式组合，电路由少量标准组件构成，没有复杂的多域构造，也无需可信设置。在此简洁基础上，Vega 增加了跨同一证件的多个证明复用工作的能力，以及一种以最小开销实现零知识的新方法。最终成果是一个易于审计、易于扩展至新证件格式且易于部署的系统。

### 性能

Vega 从典型的移动驾驶执照生成年龄的零知识证明（约 2 KB）仅需 92 毫秒（在普通客户端设备上）。生成的证明大小为 108 KB，可在 23 毫秒内完成验证。无需可信设置。证明密钥为 464 KB，可轻松存入任何手机。对于较小的证件，证明时间降至 62 毫秒，证明大小为 83 KB，验证时间为 17 毫秒。在实际使用中，用户点击按钮出示证件，92 毫秒后证明即完成。服务方仅获知所请求的事实；证件从未离开手机。

### 哈希问题及其折叠解决方案

证件证明需要完成两项高成本操作：用 SHA-256 哈希证件字节，以及验证签发方的数字签名。签名验证通常是瓶颈，但 Vega 通过在与签名算术原生兼容的域中工作来避免这一成本。因此，哈希成为主要成本。SHA-256 的工作原理是每次对 64 字节块应用相同的压缩函数。简单的电路会直接展开所有迭代，因此其规模随证件长度增长。对于典型的移动驾驶执照，这需要 30 个压缩块，全部包含在单个电路中。

我们采用了不同方法。我们不展开整个哈希，而是定义一个小型"步骤"电路来证明单个 SHA-256 压缩步骤，并为每个块实例化一次。由于这些步骤实例结构相同，我们可以使用 NeutronNova 的折叠方案将它们合并为单个实例。证明者需要将 30 个步骤实例折叠为一个，但这一折叠成本适中。Spartan 随后只需证明一个步骤大小的电路，以及一个处理其余检查（包括签名验证和年龄谓词）的独立"核心"电路，而非包含 30 个展开块的庞大电路。证明密钥只需描述一个步骤和一个核心，因此无论证件长度如何，密钥大小始终保持较小。

这里有一个微妙的隐私问题需要解决。凭证的长度各不相同，如果电路大小随凭证变化，就会泄露信息。为防止这种情况，所有步骤电路共享一个包含中间摘要的承诺表。核心电路使用私有索引选择相应的摘要。如果证明者选择了错误的条目，发行者的签名检查就会失败。

### 低成本实现零知识

证明系统需要具备零知识性：验证者除了被证明的主张之外，不应学到任何其他信息。现有的实现方法通常工程复杂，且会给证明者带来显著的开销。我们找到了一种更简单的方法。

一个标准的首要步骤是使用隐藏性密码学承诺来承诺证明者发送的每一条消息，这样验证者看到的是承诺而非实际值。更难的问题在于证明这些隐藏值本应能通过验证者的检查。我们将这些检查表示为一个小的约束系统，仅需几百个约束，因为验证者只执行对数级别的操作。然后，我们通过 Nova 的折叠方案将这个约束系统与一个随机实例进行折叠。这一步隐藏了底层数据，因此零知识开销仅与这个小型约束系统相关，而非完整的秘密数据。

### 一次证明，多次出示

向一个网站出示凭证的用户，很可能还会向另一个、再另一个网站出示。在一个 AI Agent 代表用户处理大量此类交互的世界里，同一份凭证可能每天需要被出示数十次。凭证本身在这些出示之间不会改变。改变的是会话随机数（来自验证者的新鲜随机值），以及可能的日期或谓词阈值。

Vega 利用这种结构，将证明者的秘密数据分为三部分。共享数据（SHA-256 表）和预承诺部分，如发行者签名和字段位置，在凭证首次加载时计算并承诺一次。在线部分，如设备签名和今天的日期，则在每次出示时重新承诺。在每次证明之前，预计算的承诺会使用新的随机数进行刷新，这比重算它们成本更低，并确保关于同一凭证的两次证明无法被关联。

### 避免解析器

Vega 效率的另一个重要方面来自它处理凭证格式的方式。移动驾驶执照采用简明二进制对象表示（CBOR）编码，而构建一个完整的 CBOR 解析器电路既复杂又昂贵。但我们意识到我们实际上并不需要解析器。凭证字节由受信任的发行者签名，因此我们知道其格式正确。我们只需要深入其中抓取特定字段。

我们将凭证视为一个可字节寻址的查找表。证明者说：“设备公钥从第 847 字节开始”，并提供这些字节。电路检查三件事：这些字节确实与经过认证的凭证匹配；字段开头出现正确的 CBOR 前缀，以防止证明者声称错误的字段；地址是连续的，以防止证明者拼接来自无关位置的字节。这用一个简单的查找操作取代了整个解析器。

同样的查找思路也用于上述的长度隐藏哈希：电路构建一个包含所有中间 SHA-256 摘要的表，并在真实消息结束的位置选择正确的那个。

### 设备绑定

零知识凭证证明只有与持有凭证的人绑定才有用。没有设备绑定，获得泄露凭证的人可以为任何会话生成有效证明。在 AI Agent 的世界里，这一点更为重要：如果一个 Agent 可以代表用户出示证明，我们需要密码学上的保证，证明该证明确实来自用户的设备，而非攻击者或未经授权的 Agent。

Vega 通过要求持有者的设备使用设备私钥（该私钥绑定到手机的安全元件）对一个新的会话随机数进行签名来解决这个问题。电路通过查找从凭证中提取设备公钥，并验证设备对会话随机数哈希的签名。由于设备私钥永远不会离开安全硬件，仅拥有已签名的凭证不足以生成有效的证明。

## 未来方向

Vega 使用 Rust 实现，并将很快开源。驱动 Vega 的证明系统已作为开源的 spartan2（在新标签页中打开）项目在 GitHub 上提供。与 Darya Kaviani 合作的论文将在即将于旧金山举行的 IEEE 安全与隐私研讨会上发表。

虽然我们以移动驾驶执照作为一个具体且适时的应用案例，特别是考虑到像欧盟数字身份钱包这样的新兴框架，但我们的证明系统和电路技术是通用的。它们适用于任何具有稳定字节编码和数字签名的凭证格式。

我们看到了几个方向，其中相同的原语将变得越来越重要。

**代表人类携带身份的 Agent。** 随着自主 AI Agent 开始代表人类行动——无论是预订旅行、与服务交互还是签订协议——这些 Agent 将需要证明其所代表人类的相关事实。例如，“我的委托人是成年人”或“我的委托人是一名持证医生”。Agent 应该能够在无需持有底层凭证的情况下携带这些证明。在人类设备上生成、并通过设备绑定与 Agent 会话关联的零知识证明，让 Agent 能够在不持有秘密的情况下出示身份信号。

**将链下身份桥接到链上系统。** 去中心化系统越来越需要现实世界的身份信号，例如 KYC 合规、合格投资者身份和司法管辖区检查。目前，这通过将文件上传到中心化中介机构来处理，该中介机构随后发布一个链上证明。用户会两次失去隐私：一次是向中介机构，另一次是在链上，因为该证明可能在多次交互中被关联。基于链下凭证的零知识证明可以直接桥接这一点：用户证明其政府签发凭证中的某个事实，链上验证者仅接收该证明。没有中介机构看到凭证，并且重新随机化确保了重复证明无法被关联。

随着数字身份授权的扩展以及 AI 重塑人类和 Agent 建立信任的方式，对隐私保护凭证验证的需求只会增长。我们将 Vega 视为更广泛转变中的一步：从一个证明关于自己的事实需要放弃身份的世界，转向一个密码学让你能够保留身份的世界。
