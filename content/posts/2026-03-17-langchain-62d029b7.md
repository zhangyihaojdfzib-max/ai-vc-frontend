今天，我们正式推出 LangSmith 沙盒的私有预览版：这是一个安全、可扩展的运行非受信代码的环境。

当 Agent（智能体）能够执行代码时，它们会变得有用得多。它们可以分析数据、调用 API 并从头开始构建应用程序。但是，让 LLM（大语言模型）运行任意代码而不与您的基础设施隔离是有风险的。沙盒为您提供了临时的、锁定的环境，Agent 可以在其中安全地运行代码，同时您可以控制它们能访问什么以及能消耗多少资源。

使用 LangSmith 沙盒，您只需一行代码即可通过 LangSmith SDK 启动一个沙盒。添加您的 API 密钥，引入 SDK，就可以开始了。

我们一直在内部使用沙盒来支持像 Open SWE 这样的项目，现在我们将它们开放出来，以便您可以使用相同的基础构件进行构建。

## 为什么需要沙盒？

像 Cursor、Claude Code 和 OpenClaw 这样的编码 Agent 展示了赋予 Agent 编写和运行代码能力是多么有用。但如果没有隔离，Agent 可能会在您的本地环境中执行**破坏性或恶意操作**。

传统的容器设计用于运行已知的、经过审查的应用程序代码。Agent 生成的代码则不同：它是不可信且不可预测的。Web 服务器处理一组已知的操作。而 Agent 可能会尝试任何事情，包括恶意命令。

自行构建安全的代码执行环境通常意味着启动容器、锁定网络访问、将输出传回给您的 Agent，并在完成后拆除一切。然后您还需要处理资源限制，因为如果不加约束，运行代码的 Agent 会迅速消耗 CPU、内存和磁盘。随着越来越多的 Agent 成为编码 Agent，这个问题会变得更加复杂。

一些需要此类环境的工作负载示例：

- 一个**编码助手**，在响应前运行并验证自己的输出
- 一个**CI 风格的 Agent**，在打开 PR 之前克隆仓库、安装依赖项并运行测试套件（如 Open SWE）
- 一个**数据分析 Agent**，针对数据集执行 Python 脚本并返回结果

## LangSmith 平台的一部分

LangSmith 沙盒使用与 LangSmith 其他部分相同的 SDK 和基础设施。如果您已经在使用 Python 或 JavaScript 客户端进行追踪或部署，那么无需添加任何新东西就可以启动沙盒。

沙盒还与 LangSmith 部署直接集成，因此您可以将沙盒附加到 Agent 线程。它们与 LangChain 的 Deep Agents 开源框架以及 Open SWE 都有原生集成。

## 发布内容

### 运行时配置

- **使用您自己的 Docker 镜像。** 使用我们的默认镜像或指向您自己的私有仓库。用您所需的精确文件系统和工具启动每个沙盒。
- **沙盒模板。** 一次性定义镜像、CPU 和内存配置，然后在每次启动沙盒时重复使用。与 BYOD 镜像结合，实现完全自定义的环境。
- **共享访问：** 允许多个 Agent 访问同一个沙盒，这样您就不需要在隔离环境之间传输工件。
- **池化和自动扩缩。** 预先配置一个热沙盒池，这样 Agent 就不必等待冷启动。随着需求增加，额外的沙盒会自动启动。

### 执行

- **长时间运行的会话：** 耗时数分钟或数小时的 Agent 任务不会超时。沙盒支持通过 WebSocket 进行持久命令，并实时流式输出，让您可以在运行时看到正在发生的情况。
- **跨交互的持久状态。** 您的 Agent 可以在多个线程中使用同一个沙盒，而不会丢失上下文。文件、已安装的包和环境状态在多次运行之间得以保留。
- **隧道。** 将沙盒端口暴露给您的本地机器，以便您可以在部署之前预览 Agent 的输出。

### SDK 和集成

- **框架无关：** 可以将 LangSmith 沙盒与 LangChain OSS、其他框架一起使用，或者完全不使用任何框架。
- **Python 和 JavaScript SDK：** 通过 LangSmith SDK 提供两种语言的一流客户端。
- **Deep Agents 集成：** 以最少的配置将沙盒直接插入到 Agent 工作流中。

### 安全性

- **身份验证代理：** 沙盒通过身份验证代理访问外部服务，因此密钥永远不会接触运行时环境。凭证完全不会进入沙盒。
- **微虚拟机隔离：** 每个沙盒都在硬件虚拟化的微虚拟机中运行，而不仅仅是 Linux 命名空间。沙盒之间实现内核级隔离。

## 未来计划

我们正在积极开发沙盒，其功能将超越今天发布的内容。我们正在积极探索的一些领域包括：

- **共享卷：** 赋予 Agent 跨沙盒共享状态的能力。Agent 1 写入卷，Agent 2 从其停止的地方继续。
- **二进制授权：** 控制哪些二进制文件可以在沙盒内运行。Agent 容易出现意外行为，例如安装软件包、导出凭证或在非预期任务上消耗计算资源。二进制授权允许您像在受管理的企业笔记本电脑上一样限制执行，限制可以运行哪些程序、可以访问哪些域以及允许进行哪些网络调用。
- **完整的执行追踪：** 目前，沙盒调用会与您的 Agent 运行一起被追踪。我们正在努力追踪虚拟机内发生的所有事情，包括每个进程和网络调用。这同时也是一个审计日志，为您提供沙盒在何时做了何事的完整记录。
