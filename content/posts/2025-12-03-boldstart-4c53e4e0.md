# Amazon Bedrock AgentCore 迎来关键治理更新

当亚马逊在7月宣布 Bedrock AgentCore 时，这感觉意义重大。这是一个主要的云平台增加了一种基于组件的方式来构建 Agent（智能体），其中包含了许多开发者需要、以便在企业环境中安心构建应用的部分。这包括运行时、浏览器、可观测性、身份、内存管理组件，甚至一个代码解释器。

然而，缺失的是在 Agent 部署后对其进行治理的方法，这肯定是企业客户所要求的。亚马逊在本周的 AWS re:Invent 上宣布了两个新的 AgentCore 组件，解决了这个问题：策略和评估。前者让公司能够精确定义 Agent 被允许访问哪些应用程序、API、MCP 服务器等，以及在什么条件下访问。后者让开发者能够持续检查 Agent 是否按设计意图运行，即使在发生模型更新等重大变更时也是如此。

在一篇宣布新更新的博客文章中，该公司认识到，客户在大型组织内部署 Agent 时需要信心，这不仅要求能够观察它们，还需要为其运行设置防护栏，以确保它们无法访问敏感信息，就像某些人类员工也不被允许访问一样。

"开发团队必须在赋予 Agent 自主权和确保其在可接受的边界内运行、并达到面向客户和员工所需的质量之间取得平衡。"正如 AWS 首席执行官 Matt Garman 在本周的主题演讲中所说，为 Agent 的行为以及它们能够访问的工具设定清晰的边界至关重要。

## 绝对关键的组件

去年夏天该产品发布时，云顾问 David Linthicum 并未留下深刻印象，他告诉 FastForward，这看起来像是"一系列现有的 AWS 工具，加上一些新的抽象概念打包而成"，他至今仍坚持这一观点。

本周，他补充说，治理组件绝对是必不可少的，并对第一个版本没有包含这些组件感到惊讶。"我不确定没有治理功能，他们是如何推出这样的产品的，但他们确实这么做了，"他说。"显然，市场指出了缺失的部分，他们现在正在添加。"

但 Constellation Research 的分析师 Holger Mueller 表示，这似乎正是你期望从一个 Agent 框架 V2 版本中看到的那种更新。"首先你必须构建 Agent，然后你必须治理它们，" Mueller 说。"与所有治理框架一样，它们需要经过实战检验，我们只有在明年春天才能知道它们到底有多好。"

虽然这些策略和评估组件至关重要，但它们目前尚未广泛可用。它们是在本周以预览版形式宣布的。该公司指出，自推出以来，AgentCore SDK 已被下载 200 万次。客户包括 PGA Tour 和 Workday。
