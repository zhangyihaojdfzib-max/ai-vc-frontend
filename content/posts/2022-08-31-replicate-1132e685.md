# 在 M1 Mac 的 GPU 上运行 Stable Diffusion  

- bfirsh  

Stable Diffusion 是开源的，因此任何人都可以运行和修改它。这正是过去一周涌现出大量创作的原因。  

你可以在 Replicate 的云端运行 Stable Diffusion，但也可以本地运行。除了生成预测结果，你还可以对其进行破解、修改并构建新功能。要让它在 M1 Mac 的 GPU 上运行有些繁琐，因此我们编写了这份指南来教你如何操作。  

这一切要归功于所有为 GitHub 上这个 stable-diffusion 分支做出贡献的人，以及在这个 GitHub 讨论串中解决问题的人。我们只是他们出色工作的传递者。  

我们在先前工作的基础上做了一件事：使用 pip 而非 Conda 来安装依赖项。这样设置起来更简单，而且由于使用了二进制轮子，无需编译任何内容。  

## 前提条件  

- 配备 M1 或 M2 芯片的 Mac。  
- 16GB 或更大内存。8GB 内存也能运行，但速度极慢。  
- macOS 12.3 或更高版本。  

## 设置 Python  

你需要 Python 3.10 来运行 Stable Diffusion。运行 `python -V` 查看已安装的 Python 版本：  

$ python3 -V                                                                                       !11338  
Python 3.10.6  

如果版本是 3.10 或更高（如此处所示），那就没问题了！直接跳到下一步。  

否则，你需要安装 Python 3.10。最简单的方法是使用 Homebrew。首先，如果尚未安装 Homebrew，请先安装它。  

然后，安装最新版本的 Python：  

brew update  
brew install python  

现在运行 `python3 -V`，你应该会看到 3.10 或更高版本。你可能需要重新打开控制台才能生效。  

## 克隆仓库并安装依赖项  

运行以下命令来克隆 Stable Diffusion 的分支：  

git clone -b apple-silicon-mps-support https://github.com/bfirsh/stable-diffusion.git  
cd stable-diffusion  
mkdir -p models/ldm/stable-diffusion-v1/  

然后，设置一个虚拟环境来安装依赖项：  

python3 -m pip install virtualenv  
python3 -m virtualenv venv  

激活虚拟环境：  

source venv/bin/activate  

（每次你想运行 Stable Diffusion 时，都需要再次运行此命令。）  

接着，安装依赖项：  

pip install -r requirements.txt  

如果你看到类似 `Failed building wheel for onnx` 的错误，可能需要安装这些包：  

brew install cmake protobuf rust  

## 下载权重文件  

前往 Hugging Face 仓库，阅读并理解许可协议，然后点击“Access repository”。  

在该页面下载 `sd-v1-4.ckpt`（约 4 GB），并将其保存为你之前创建的目录中的 `models/ldm/stable-diffusion-v1/model.ckpt`。  

## 运行它！  

现在，你可以运行 Stable Diffusion 了：  

python scripts/txt2img.py --prompt "a red juicy apple floating in outer space, like a planet" --n_samples 1 --n_iter 1 --plms  

你的输出在 `outputs/txt2img-samples/` 目录下。就是这样。
