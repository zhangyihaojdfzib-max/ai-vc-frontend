## 寻找市场契合点的初创公司

Sensay 最初并非一个员工离职知识留存平台。

公司的原始使命极具人文关怀：构建能够捕捉个人知识、声音和形象的“数字副本”，以便家庭在认知衰退发生前保存记忆。团队曾与养老院和护理机构合作，专注于为希望留下个人印记的阿尔茨海默症和痴呆症患者提供服务。

但产品现实很快显现。为这一人群构建和分发软件非常困难，尤其是在早期版本仅为文本形式时。市场虽然存在，但产品并不契合。

对于首席技术官兼联合创始人 Marco Bettiolo 而言，教训很明确：Sensay 需要一个技术栈，能够让他们快速交付、测试和调整方向，而不会积累那种拖慢初创公司步伐的基础设施负担。

他曾近距离目睹过这种负担。在之前的两家公司，Marco 曾帮助构建了旨在提升工程团队效率的基础设施：内部部署系统、预览环境、回滚机制、监控和告警系统。这些系统虽然有效，但需要多年的开发、专门的人员配置和持续的维护。

“我曾负责在两家不同的公司内部构建基础设施，两次都花费了数百万美元，” Marco 回忆道。

对于试图寻找产品市场契合点的初创公司而言，平台工作就变成了一个不产生收入的“第二产品”。Marco 深知他不想再构建那样的基础设施了。

## 三次转型，一个代码库

构建数字副本让 Sensay 团队获得了一个更广泛的洞见：他们实际上是在构建一个 AI 知识引擎。核心目标是让 LLM 的响应基于特定的源材料，而非依赖通用的训练数据。如果你要重现某人的生活经历——例如，一位参加过二战、说着一种现已消失的方言的祖父——AI 需要反映这个人的真实知识和声音，而不是臆造的泛泛之谈。

这一认识催生了他们的第二次迭代：一个 API 平台，让团队能够使用自己的知识库构建基于事实的 Agent。他们构建了一个聊天小部件，获得了客户，并发现要在竞争激烈的领域销售 API，需要他们当时尚不具备的技术销售专长。

客户不断询问一个特定的垂直领域问题：员工离职导致的知识流失。于是，他们构建了一个离职知识留存平台，用于捕捉离职员工的机构知识，使其在企业内部保持可访问性。

当获得在 Web Summit 上发布的机会时，团队将所有工作压缩到了一个紧张的时间表内。“在 Web Summit 之前的六周里，我们从零开始开发 MVP，到在活动上实现生产环境发布，” Marco 说。

六周。从概念到在大型会议上的实时产品。“如果我必须从构建自己的基础设施开始，我们不可能在六周内发布任何东西，六个月也不行。”

## 实现速度的关键

对 Sensay 而言，Vercel 的优势不在于任何单一功能，而在于消除了那些让产品交付感觉有风险的工作。

**预览部署**意味着每个分支都会获得一个可立即测试和分享的有效 URL，无需合并，也无需预发布分支的开销。“当我们需要快速推进时，能够在 PR 上独立测试功能是不可思议的，” Marco 解释道。“这让你可以控制影响范围。”

**功能开关**成为了一种开发原语。团队不再维护那些必然导致合并冲突的长期分支，而是让所有人都在同一个主干代码库上工作。未完成的功能只需隐藏在开关后面，直到准备就绪。使用 **Vercel Toolbar**，开发者和团队成员可以安全地开启或关闭功能。

**即时回滚**降低了部署的心理成本。域名配置和环境变量使得上线过程可重复且快速。**防火墙**和机器人防护等安全功能消除了对额外服务的需求。**使用情况仪表板**提供了资源消耗的可视性。“从第一天起你就能获得生产力，” Marco 说。“你只需推送到分支，产品就上线了。”

其结果是形成了一个既能快速推进又不失控制的工作流。“如果没有仪表板，要理解我为何以及如何消耗你们的资源将会是一团糟。”

Sensay 采用混合架构，将无服务器前后端工作负载运行在 Vercel 上，而有状态工作负载则运行在其他地方。Next.js 驱动前端，他们的 API 作为一个独立的 Vercel 项目使用 Hono 运行，并且他们使用 **VercelAI SDK** 来简化和维护 AI 功能及模型访问的构建。

“所有无服务器工作负载都在 Vercel 上，” Marco 解释道。“所以我们仍然没有任何 DevOps 人员。”

## 价值百万美元的问题

当被问及与内部构建基础设施相比节省了多少成本时，Marco 直接给出了数字。

他并非夸大其词。在他之前的公司，他领导着一个 20 人的团队，专门负责构建内部部署工具。账单高达数百万美元。该系统虽然有效，但花费了数年时间才达到 Vercel 在第一天就提供的生产力水平。

对于初创公司而言，这几百万美元（更不用说节省的时间）是能否达到产品市场契合点，还是在构建基础设施时耗尽资金的关键区别。“立即使用一个结构良好的平台上线是显而易见的选择，” Marco 说。

## 未来展望：让知识保持活力

Sensay 近期的重点是分发：将离职知识留存产品交到试点合作伙伴手中，并根据真实工作流程进行迭代。通过产品使用情况和客户对话，产品方向正变得更加清晰：让离职员工的语音和文本访谈过程顺畅无阻。

但更大的机会在于知识被捕捉之后。“一旦你捕捉了知识，人们会问，‘好的，但接下来我想让它尽可能有用，’” Marco 指出。团队希望的是保持更新的活知识库，而不是会过时的静态快照。

Sensay 的故事从根本上讲是关于达到产品市场契合点的速度。

“谁知道如果我们决定内部构建一切，我们现在会在哪里，” Marco 反思道。“肯定不是今天的样子。”
