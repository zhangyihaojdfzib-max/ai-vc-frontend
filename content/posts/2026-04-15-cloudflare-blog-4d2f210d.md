隆重介绍 Agent Lee。

Agent Lee 是一个内置于仪表盘的 AI 助手，它理解你的 Cloudflare 账户。

它能帮助你进行故障排除——这在今天是一项繁琐的手动工作。如果你的 Worker 在 UTC 时间 02:00 开始返回 503 错误，要找到根本原因——无论是 R2 存储桶、配置错误的路由，还是隐藏的速率限制——你都需要打开半打标签页，并希望自己能识别出模式。大多数开发者并没有一个熟知整个平台的队友在凌晨两点站在他们身后提供帮助。但 Agent Lee 可以。

而且它不仅仅在凌晨两点为你排查故障。Agent Lee 还会当场为你解决问题。

Agent Lee 已在活跃的测试版中运行，期间它每天服务超过 18,000 名用户，每天执行近 25 万次工具调用。虽然我们对它当前的能力和在生产环境中的成功充满信心，但这仍是一个我们持续开发的系统。由于它仍处于测试阶段，在我们优化其性能的过程中，你可能会遇到意想不到的限制或边缘情况。我们鼓励你使用下面的反馈表，帮助我们每天让它变得更好。

## Agent Lee 能做什么

Agent Lee 直接内置于仪表盘，并理解你账户中的资源。它知道你的 Workers、你的区域、你的 DNS 配置、你的错误率。那些如今分散在六个标签页和两个浏览器窗口中的知识，现在将汇聚于一处，并且你可以与之对话。

使用自然语言，你可以用它来：

*   **回答关于你账户的问题：** "显示我的 Worker 上排名前 5 的错误信息。"
*   **调试问题：** "我无法通过 www 前缀访问我的网站。"
*   **应用更改：** "为我的域名启用 Access。"
*   **部署资源：** "为我的照片创建一个新的 R2 存储桶，并将其连接到我的 Worker。"

你无需在不同产品间切换，只需描述你想做什么，Agent Lee 就会通过指导和可视化帮助你达成目标。它检索上下文，使用正确的工具，并根据你提出的问题类型创建动态可视化。询问过去 24 小时你的错误率情况，它会内联渲染一个图表，数据来自你的实际流量，而不是将你发送到一个单独的分析页面。

Agent Lee 不是在回答常见问题——它是在大规模地、针对真实账户进行实际工作。如今，Agent Lee 每天服务约 18,000 名用户，每天在 DNS、Workers、SSL/TLS、R2、域名注册、缓存、Cloudflare Tunnel、API Shield 等产品上执行约 25 万次工具调用。

## 我们如何构建它

### Codemode

Agent Lee 没有直接将 MCP 工具定义呈现给模型，而是使用 **Codemode** 将工具转换为 TypeScript API，并要求模型编写调用它的代码。

这样做效果更好，原因有几个。LLM（大语言模型）已经见过大量的真实世界 TypeScript 代码，但很少见到工具调用示例，因此它们在处理代码时更准确。对于多步骤任务，模型还可以在单个脚本中将调用链接在一起，并仅返回最终结果，从而最终跳过往返通信。

生成的代码被发送到上游的 Cloudflare MCP 服务器进行沙箱执行，但会经过一个充当凭证代理的 Durable Object。在任何调用发出之前，DO 会通过检查方法和主体来将生成的代码分类为读取或写入操作。读取操作直接代理。写入操作会被阻止，直到你通过确认门明确批准。API 密钥永远不会出现在生成的代码中——它们保存在 DO 内部，并在进行上游调用时在服务器端注入。安全边界不仅仅是一个会被丢弃的沙箱；它是一个权限架构，从结构上防止未经你批准的写入操作发生。

### MCP 权限系统

Agent Lee 连接到 Cloudflare 自己的 MCP 服务器，该服务器公开了两个工具：一个用于查询 API 端点的搜索工具，以及一个用于编写执行 API 请求代码的执行工具。这是 Agent Lee 读取你账户（并在你批准时写入）的界面。

写入操作需要通过一个确认系统，该系统在任何代码执行之前都会显示批准步骤。Agent Lee 无法跳过此步骤。权限模型是执行层，你看到的确认提示不仅仅是用户体验上的礼貌。它是门禁。

## 构建于你可用的相同技术栈之上

Agent Lee 构建所依赖的每一个基础组件都可供我们所有客户使用：**Agents SDK**、**Workers AI**、**Durable Objects**，以及任何 Cloudflare 开发者都可用的相同 MCP 基础设施。我们没有构建你无法使用的内部工具——相反，我们使用你同样可以访问的 Cloudflare 乐高积木来构建它。

在我们自己的基础组件上构建 Agent Lee 不仅仅是一个设计原则。这是找出什么有效、什么无效的最快方法。我们在生产环境中、与真实用户、针对真实账户构建了它。这意味着我们遇到的每一个限制，都是我们可以在平台中修复的限制。每一个有效的模式，我们都可以让下一个基于它构建的团队更容易地使用。

这些不是观点。它们是每天来自 18,000 名用户的 25 万次工具调用告诉我们的。

## 生成式 UI

与平台交互应该感觉像是在与专家协作。对话应该超越简单的文本。借助 Agent Lee，随着对话的进行，平台会动态生成 UI 组件以及文本响应，以提供更丰富、更具可操作性的体验。

例如，如果你询问本月的网站流量趋势，你得到的将不仅仅是一段数字。Agent Lee 将渲染一个交互式折线图，让你一目了然地可视化活动的高峰和低谷。

为了给你充分的创意控制权，每次对话都伴随着一个自适应网格。在这里，你可以在网格上点击并拖动，为新 UI 块开辟空间，然后只需描述你想看到的内容，让智能体处理繁重的工作。

目前，我们支持多样化的可视化块库，包括动态表格、交互式图表、架构图等。通过将自然语言的灵活性与结构化 UI 的清晰性相结合，Agent Lee 将你的聊天记录转变为一个动态的仪表盘。

## 衡量质量与安全性

一个能在你账户上执行操作的智能体必须是可靠且安全的。确认机制允许智能体系统在执行过程中主动向用户或其他系统征求信息、偏好或批准。当 Agent Lee 需要代表用户执行非读取操作时，我们通过要求在用户界面中进行明确的批准操作来使用确认机制。这些防护措施使 Agent Lee 能够真正成为你在安全管理资源时的合作伙伴。

除了安全性，我们还持续衡量质量。

- 用于衡量对话成功率和信息准确性的评估体系。
- 来自用户交互的反馈信号（点赞/点踩）。
- 工具调用执行成功率和幻觉评分器。
- 按产品细分的对话表现分析。

这些系统帮助我们持续改进Agent Lee（智能体李），同时确保用户保持控制权。

## 我们的未来愿景

仪表板中的Agent Lee仅仅是个开始。

更宏大的愿景是让Agent Lee成为整个Cloudflare平台的交互界面——无论身处何处。今天的仪表板，接下来的命令行界面，以及您移动中的手机。您使用的界面不应成为限制。您应该能够描述需求并让它完成，无论您身在何方。

在此基础上，Agent Lee将变得主动。它不再被动等待询问，而是关注对您重要的事物：您的Workers、流量、错误阈值，并在需要关注时主动联系。一个仅会响应的智能体是有用的，但一个能率先发现问题的智能体则截然不同。

这一切的基石是上下文。Agent Lee已经了解您的账户配置。随着时间的推移，它将了解更多：您之前的询问、您所在的页面、您上周的调试内容。这种累积的上下文能让平台感觉不像工具，而更像协作者。

我们尚未完全实现。今天的Agent Lee是第一步，它已在生产环境中运行，并大规模处理实际工作。其架构正是为实现后续愿景而构建。
