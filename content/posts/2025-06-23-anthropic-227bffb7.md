# Claude 3.5 Sonnet 登陆 GitHub Copilot

从今天起，全新的 **Claude 3.5 Sonnet** 开始在 GitHub Copilot 上推出，使开发者能够直接在 Visual Studio Code 和 GitHub.com 中选择 Claude 3.5 Sonnet 进行编码。此次集成将 Claude 的编码能力带给了 GitHub 超过 1 亿开发者的社区。

升级后的 Claude 3.5 Sonnet 在 **SWE-bench Verified** 基准测试中超越了所有公开可用的模型，该基准用于衡量模型解决现实世界 GitHub 问题的能力。它还在 **HumanEval** 基准测试中获得了同类最高分（93.7%），该测试评估模型根据自然语言描述编写正确 Python 函数的能力。

Claude 3.5 Sonnet 将在未来几周内以公开预览版的形式提供给所有 GitHub Copilot Chat 用户和组织。

## 使用场景

借助对您整个代码库上下文的了解，您可以在 GitHub Copilot 上使用 Claude 3.5 Sonnet 来：

*   **编写生产就绪的代码：** 要求 GitHub Copilot Chat 将自然语言描述转换为符合您项目模式和惯例的高质量代码。
*   **通过行内聊天调试：** 高亮显示有问题的代码以获取即时分析和修复，或要求 GitHub Copilot Chat 解释错误信息并提出解决方案。
*   **根据实现创建测试：** 选择任何函数以自动生成与您的测试框架匹配并覆盖边缘情况的全面测试套件。
*   **通过上下文解释理解代码：** 在 VS Code 中将鼠标悬停在函数上或高亮显示代码块，以获取关于代码功能及其在项目中作用的解释。

## 开始使用

GitHub 从今天开始以公开预览版的形式发布 Claude 3.5 Sonnet。Claude 3.5 Sonnet 通过 Amazon Bedrock 在 GitHub Copilot 上运行，利用 Bedrock 的跨区域推理功能进一步增强可靠性。

所有开发者和组织都应在未来几周内获得 Claude 3.5 Sonnet 的访问权限。请查阅 GitHub 的[博客](https://github.blog)和[文档](https://docs.github.com)以了解更多信息。
