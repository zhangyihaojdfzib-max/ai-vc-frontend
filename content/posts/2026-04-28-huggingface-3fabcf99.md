# 介绍 NVIDIA Nemotron 3 Nano Omni：面向文档、音频和视频 Agent 的长上下文多模态智能

- NVIDIA Nemotron 3 Nano Omni 是一款全新的全模态理解模型，专为真实世界文档分析、多图像推理、自动语音识别、长音频-视频理解、Agent 化计算机使用和通用推理而构建。
- 它将 Nemotron 多模态产品线从强大的视觉-语言系统扩展为更广泛的文本+图像+视频+音频模型。
- Nemotron 3 Nano Omni 在 MMlongbench-Doc、OCRBenchV2 等复杂文档智能排行榜上实现了同类最佳精度，同时在 WorldSense 和 DailyOmni 等视频和音频排行榜上也处于领先地位。它在 VoiceBench 上实现了音频理解的最高精度，并在 MediaPerf 上被评为最具成本效益的开源视频理解模型。
- 在底层，它结合了 Nemotron 3 混合 Mamba-Transformer 混合专家骨干网络、C-RADIOv4-H 视觉编码器和 Parakeet-TDT-0.6B-v2 音频编码器。
- 该架构旨在保留精细的视觉细节，增加原生音频理解能力，并扩展到非常长的多模态上下文，用于密集图像、文档、视频和混合模态推理。
- 训练方案采用分阶段多模态对齐和上下文扩展，随后进行偏好优化和多模态强化学习。
- 在多模态用例中，Nemotron 3 Nano Omni 的吞吐量比同类产品高出 9 倍，单流推理速度高出 2.9 倍。
- 在 HuggingFace 下载 BF16、FP8 和 NVFP4 检查点。
- 有关模型架构、训练方案、数据管道和基准测试的更多信息，请阅读完整的 Nemotron 3 Nano Omni 报告。

## 基准测试亮点

在 Nemotron Nano V2 VL 的基础上，Nemotron 3 Nano Omni 实现了显著的视觉提升，并增加了全新的音频和视频+音频能力——同时在许多领域领先于另一个开源权重全模态模型 Qwen3-Omni。

## Nemotron 3 Nano Omni 的设计目标

从宏观层面来看，Nemotron 3 Nano Omni 针对五类工作负载：

### 1. 真实世界文档分析

这不仅关乎 OCR。该模型定位于处理冗长、杂乱、高价值的文档，其理解依赖于布局、表格、图形、公式、章节结构和跨页引用。例如合同、技术论文、报告、手册、多页表格或合规文件包。该模型可处理 100 页以上的文档。

### 2. 自动语音识别

Nemotron 3 Nano Omni 包含强大的语音理解能力，可在各种音频条件下实现高质量转录。它能够处理包含不同说话者、口音和背景噪音的长篇音频。这些能力可以集成到更广泛的工作流程中，使口语内容能够被转录、分析，并与其他模态结合用于摘要、问答和跨模态推理等任务。

### 3. 长音频-视频理解

许多企业和开发者工作流程依赖于混合的音频和视觉证据：带旁白的屏幕录制、培训视频、带幻灯片的会议、教程、产品演示、客户支持记录以及长篇视频档案。Nemotron 3 Nano Omni 旨在联合推理这些输入。

### 4. Agent 化计算机使用

Nemotron 3 Nano Omni 模型专门针对 Agent 化计算机使用进行了训练，使其能够协助完成图形用户界面环境中的任务。其能力包括解释屏幕截图、监控用户界面状态、基于屏幕视觉内容进行推理，以及帮助进行动作选择或工作流自动化。

### 5. 通用多模态推理

该模型的设计超越了感知层面。它擅长需要跨长上下文窗口、多模态以及结构化或半结构化证据综合信息的推理密集型任务。它可以执行多步推理、进行计算，并连接来自文本、图像、表格和其他输入的信息，以得出连贯且有充分依据的答案。

## 模型架构与关键创新

Nemotron 3 Nano Omni 采用统一的编码器-投影仪-解码器设计。语言骨干网络是 Nemotron 3 Nano 30B-A3B，搭配 C-RADIOv4-H 视觉编码器和 Parakeet-TDT-0.6B-v2 音频编码器。特定模态的编码器通过轻量级投影仪连接到 LLM 骨干网络。

### 用于长多模态上下文的混合 Mamba-Transformer-MoE 骨干网络

模型骨干网络交错排列三个关键组件：23 个 Mamba 选择性状态空间层，用于高效的长上下文处理；23 个 MoE 层，包含 128 个专家、top-6 路由和一个共享专家，用于条件容量；以及 6 个分组查询注意力层，以保持强大的全局交互和表达能力。

Nemotron 3 Nano Omni 将状态空间模型、注意力和 MoE 统一在一个设计中，在保持强大推理性能的同时，对于长多模态上下文仍然实用。

### 用于密集文档、图表和屏幕的动态分辨率

在视觉方面，Nemotron 3 Nano Omni 用原生宽高比的动态分辨率处理取代了 v2 模型中使用的分块策略。每个图像可以使用可变数量的 16 x 16 块来表示，每个图像最少 1,024 个视觉块，最多 13,312 个视觉块。对于方形图像，这分别相当于 512 x 512 和 1840 x 1840。

这种灵活性对于处理高分辨率、复杂的视觉输入至关重要，例如 OCR 密集型文档、财务表格、幻灯片、研究图表、屏幕截图和 GUI 布局——尤其是在需要同时理解精细细节和整体结构的情况下。

### 用于视频的 Conv3D 时间压缩

对于视频，Nemotron 3 Nano Omni 使用专用的 Conv3D 管状嵌入路径。它不是独立嵌入每一帧，而是在 ViT 之前将每对连续帧融合成一个单一的"管状"，从而将语言模型需要关注的视觉 Token 数量减半。这使得我们可以在相同的 Token 预算下将帧数加倍，或者在相同的帧数下将 Token 数量减半。

### EVS——高效视频采样

EVS 是一项重要功能，在推理时启用，它会在视觉编码器之后丢弃冗余的视频 Token。这减少了延迟并提高了吞吐量，同时保持了准确性。视频的第一帧被完整保留，然后对于每一后续帧，EVS 保留视频正在变化的"动态"Token，并丢弃与前一帧相比没有变化的"静态"Token。我们将其与 Conv3D 结合使用以实现卓越的压缩：Conv3D 将成对帧的 Token 融合为一个，然后 EVS 修剪冗余的静态信息。

### 原生音频输入，而不仅仅是文本转录

音频方面由 Parakeet-TDT-0.6B-v2 驱动，通过其自己的 2 层 MLP 投影仪连接到骨干网络。音频以 16 kHz 采样，模型使用长达 1,200 秒（20 分钟）的输入进行训练，而 LLM 最大上下文长度支持 5 小时以上。

这代表了从传统VLM管线的转变，通过在共享的多模态序列中实现原生音频处理，使得音频、视觉和文本Token能够被联合建模。这对于诸如带旁白的屏幕录制、语音会改变视觉含义的视频问答、长篇教学或会议内容，以及需要基于时间轴进行多模态推理的任务等场景至关重要。

### 轻量级模态投影仪与统一的Token交错排列

每个编码器通过一个轻量级的2层MLP投影仪与LLM连接，该投影仪将编码器特征映射到共享的嵌入空间。投影完成后，视觉、音频和文本Token被交错排列并联合处理。

这种设计保持了整个系统的模块化，同时仍然能够在主干网络内部实现真正的跨模态推理。

## 训练数据、基础设施与系统故事

SFT阶段在NVIDIA H100上训练，根据阶段不同，规模从32个节点扩展到128个节点。技术栈使用了Megatron-LM、Transformer Engine和Megatron Energon，并采用了张量并行、专家并行、序列并行、针对长上下文阶段的上下文并行、在线序列打包以及选择性激活重计算。

SFT后的强化学习使用NeMo-RL和NeMo Gym，后端为Megatron。RL基础设施采用了基于Ray的分布式设置，横跨B200和H100集群，并进行了多模态去重，因此重复的推演不会成倍增加图像、视频和音频的内存占用。

我们开源了大部分训练代码。

### 使用RL塑造可靠的多模态行为

我们在Nemotron 3 Nano Omni中引入了多环境文本和全模态训练。我们的文本RL训练阶段在Nemo-Gym的多种环境中进行，评估模型执行一系列动作的能力，例如工具调用、编写代码以及满足可验证标准的多部分规划。

全模态RL训练模型在一个统一框架内跨图像、视频、音频和文本进行推理，涵盖从单模态到完全多模态场景的任务。一个多样化的验证器套件评估多种格式的输出，如选择题、数学题、GUI定位和ASR，同时有意包含无法回答的案例，以教导模型在证据不足时放弃回答而非产生幻觉。

### 数据与数据管线

Nemotron 3 Nano Omni在一个增强数据集上训练，该数据集强调跨多种模态的高质量推理。我们显著扩展了任务覆盖范围，并为公开数据集有限的复杂推理场景引入了合成数据。为支持这一点，我们构建了针对特定任务的多阶段管线，用于可扩展的合成数据生成。

例如，我们使用NeMo Data Designer从一个大型真实世界PDF语料库中生成了大约1140万个合成问答对（约450亿个Token）。该数据集用于在后训练阶段增强长上下文文档推理，并在MMLongBench-Doc上将整体准确率提升了2.19倍。

我们在Data Designer开发者说明中详细介绍了完整的管线演进过程，包括失败分析和关键经验教训。该说明还包含了九个可运行的管线配方，可作为构建您自己的文档理解数据集的起点。
