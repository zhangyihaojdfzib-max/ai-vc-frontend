OpenAI 的 GPT-5.4 Mini 和 GPT-5.4 Nano 现已登陆 Vercel AI Gateway。这两款模型在各自规模级别中，于编码和计算机使用方面均提供了顶尖性能，专为子 Agent（智能体）工作流打造，即多个较小模型协同处理大型任务的各个部分。

这些模型还支持详细程度和推理级别参数，让您可以控制响应的细节以及模型在回答前的推理深度。

**GPT-5.4 Mini**

GPT-5.4 Mini 在代码生成、工具编排和多步骤浏览器交互方面，比之前的同级别 Mini 模型表现更可靠。对于需要在能力与成本之间取得平衡的 Agent（智能体）任务，它是一个强大的默认选择。要使用此模型，请在 AI SDK 中将模型设置为 `openai/gpt-5.4-mini`。

```
1import { streamText } from 'ai';2
3const result = streamText({4  model: 'openai/gpt-5.4-mini',5  prompt:6    `Scaffold a new Next.js API route that connects to our7     Postgres database, validates the incoming webhook payload,8     and writes the event to the audit_logs table.`,9});
```

**GPT-5.4 Nano**

GPT-5.4 Nano 在评估中表现接近 GPT-5.4 Mini，但价格更低。该模型非常适合高吞吐量的用例，例如子 Agent（智能体）工作流，其成本会随着并行调用次数的增加而增加。要使用此模型，请在 AI SDK 中将模型设置为 `openai/gpt-5.4-nano`。

```
1import { streamText } from 'ai';2
3const result = streamText({4  model: 'openai/gpt-5.4-nano',5  prompt:6    `Check each file in the PR diff for unused imports,7     flag any that can be removed, and return the results8     as a JSON array with file path and line number.`,9});
```

AI Gateway 提供了一个统一的 API 来调用模型、跟踪使用情况和成本，并配置重试、故障转移和性能优化，以实现高于供应商的正常运行时间。它包含内置的可观测性、自带密钥支持以及具有自动重试功能的智能供应商路由。

了解更多关于 [AI Gateway](https://vercel.com/docs/ai/gateway) 的信息，查看 [AI Gateway 模型排行榜](https://vercel.com/ai/gateway/leaderboard) 或在我们的 [模型游乐场](https://sdk.vercel.ai/playground) 中试用。
