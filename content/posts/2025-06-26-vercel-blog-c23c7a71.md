## 通过 AI Gateway 简化模型访问

`AI SDK` 通过统一交互模式，帮助开发者集成模型而无需担心特定供应商的 API，从而简化了 AI 开发。现在，我们通过 `AI Gateway` 更进一步。

AI Gateway 为您提供一个单一的端点来访问跨供应商的广泛 AI 模型，具有更好的正常运行时间、更快的响应速度且无锁定。只需一行代码即可切换供应商，智能路由请求，并设置故障转移以提高正常运行时间。

开发者可以使用来自 OpenAI、Anthropic、Google、xAI 等供应商的模型，享受基于使用量的按供应商标价计费、自带密钥支持、改进的可观测性（包括按模型的使用量和延迟指标）以及简化的身份验证。

随着新模型的推出，AI Gateway 帮助您的应用保持灵活性，无需重写代码或更改基础设施。但访问模型只是问题的一半。高效运行它们需要不同的计算方式。

## 通过 Fluid 和 Active CPU 定价实现更智能的计算

传统的无服务器平台并非为高效处理 AI 推理、Agent（智能体）和 MCP 服务器等 I/O 密集型工作负载而设计。它们需要即时扩展，但通常在操作之间处于闲置状态。

`Fluid 计算` 摆脱了一对一的无服务器模型。Fluid 不是为每次调用启动单独的实例，而是智能地跨调用编排计算。多个并发请求可以共享相同的基础资源。通过函数内并发等优化，团队看到了高达 85% 的成本节约。

现在，我们在 Fluid 的基础上引入了一种新的定价模型：`Active CPU`。

*   `Active CPU` 对您的代码实际执行的时间计费
*   `预置内存` 覆盖您的函数等待的时间，按 1/11 的费率计费
*   `调用次数` 按每次函数调用计数（就像在传统的无服务器中一样）

如果您的 AI 调用需要 30 秒响应，但只使用了 300 毫秒的计算时间，您只需为那 300 毫秒付费。其余部分通过 Fluid 和内存重用高效覆盖。

这是 AI 时代的现代计算定价。细粒度、基于使用量，并为推理优化。是您后端和 API 代码的理想基础设施。

## 使用 Vercel Sandbox 运行不受信任的代码

随着 AI Agent（智能体）生成可执行代码，您需要一种方法来安全地运行这些并非您编写的代码，同时不损害您的基础设施。这就是我们推出 `Vercel Sandbox` 的原因。

Sandbox 是一个用于不受信任代码的隔离、临时的执行环境。它支持 Node.js 和 Python，可扩展到数百个并发环境，并允许开发者在安全容器中流式传输日志、安装依赖项和控制运行时行为。

这些环境是隔离的微型虚拟机，支持长达 45 分钟的执行时间。Sandbox 是一个独立的 SDK，可在任何地方工作，包括非 Vercel 平台，并使用 Fluid 的 Active CPU 定价。

我们在开发 `v0` 时为自己构建了它，现在它可供您使用。


## 通过 Rolling Releases 实现更安全的部署

速度不仅仅是代码或计算性能。它关乎充满信心地交付。

`Rolling Releases` 允许将新部署安全、增量地推送给一部分用户，具备内置监控、发布控制，且无需自定义路由。如果像首字节时间这样的指标下降或错误激增，发布可以暂停或完全中止。

您可以为每个项目配置发布阶段，并决定每个阶段的推进方式。更新通过我们的快速传播管道在 300 毫秒内全局传播。每次发布都包含实时监控，以比较版本之间的错误率和 Speed Insights，以及通过 REST API、CLI、仪表板或 Terraform 提供的灵活控制。

## 通过 BotID 实现无形的机器人检测

我们构建了**Bot Management**套件，旨在防御各类自动化滥用行为，从嘈杂的爬虫到隐秘的凭据填充攻击。但现代复杂的机器人程序已不再像传统机器人——它们能执行JavaScript、破解验证码，并像真实用户一样操作浏览器。仅靠检查请求头或实施速率限制等传统防御手段有时已力不从心。

为此，我们与**Kasada**合作推出了**Vercel BotID**以扩展防护能力。BotID是一种隐形验证码技术，专门保护结账、登录、注册、API等关键路径，以及会触发高成本后端调用的操作（例如基于LLM的端点）。

BotID会注入轻量级混淆代码，这些代码在每次加载时动态变化，能有效抵御重放攻击、篡改和静态分析。它在后台执行深度信号分析，且无需用户交互操作——从此告别点击交通信号灯式的验证。

```
1import { checkBotId } from "botid/server";2
3export async function POST(req: Request) {4  const { isBot } = await checkBotId();5
6  if (isBot) {7    return new Response("Access Denied", { status: 403 });8  }9
10  const result = await expensiveOrCriticalOperation();11
12  return new Response("Success!");13}
```

配置过程简单直观，无需配置文件或复杂调优：安装软件包、设置重写规则、挂载客户端并在服务端验证请求即可。

BotID流量将显示在防火墙仪表盘中，支持按判定结果、用户代理、国家和IP地址进行筛选。

随着应用复杂性增长，除了防范攻击，获取应用的可观测性也至关重要。我们的首要目标是让BotID惠及所有开发者——即使您未托管在Vercel上。

## 通过Vercel Queues将任务转移至后台处理

**Vercel Queues**让您能将任务转移至后台执行，确保耗时较长的作业可靠完成。这意味着用户无需在请求过程中等待缓慢操作结束，同时您的应用能以更可靠的方式处理重试与故障。

```
1import { send, receive } from "@vercel/queue";2
3await send("topic", { message: "Hello World!" });4
5await receive("topic", "consumer", (m) => {6  console.log(m.message); 7});
```

该功能适用于AI视频处理、邮件发送或更新外部服务等后台作业。它采用仅追加日志存储消息，确保任务持久化且永不丢失。
