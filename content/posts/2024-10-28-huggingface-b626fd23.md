# 专家支持案例研究：利用LLM-as-a-Judge增强RAG应用

这是一篇由Digital Green撰写的客座博客文章。Digital Green正在参与一项由CGIAR领导的合作项目，为小农户提供农业支持。

全球约有5亿小农户：他们在全球粮食安全中发挥着关键作用。及时获取准确信息对于这些农民做出明智决策和提高产量至关重要。

"农业推广服务"为农民提供农业技术建议，并为他们提供必要的投入和服务以支持农业生产。仅在印度就有30万名农业推广人员，他们提供关于改良农业实践的必要信息，并帮助小农户进行决策。

尽管推广人员数量可观，但仍不足以应对所有需求：他们与农民的互动比例通常为1:1000。通过合作和技术手段触达农业推广人员和农民仍然是关键。

由此诞生了GAIA项目，这是由CGIAR发起的一项合作计划。该项目通过专家支持计划，由Hugging Face担任导师，Digital Green作为项目合作伙伴。

GAIA有一个宏伟的目标：将GARDIAN门户中精心维护的多年农业研究成果以研究论文的形式交到农民手中。全球范围内有近46,000篇研究论文和报告，涵盖了数十年来不同作物的农业知识。

Digital Green立即看到了基于检索增强生成（RAG）技术，在经批准的、经过整理的信息上开发智能聊天机器人的潜力。因此，他们决定开发Farmer.chat，这是一个利用大语言模型（LLM）能力，为农民和一线推广人员提供个性化和可靠农业建议的聊天机器人。

为如此多样的语言、地域、作物和用例创建这样一个聊天机器人是一个巨大的挑战：所传播的信息必须与当地农场的具体情况相关，使用农民能够理解的语言和语气，并且准确（基于可信来源），以便农民据此采取行动。为了评估系统性能，CGIAR团队和HF专家合作建立了一套强大的评估体系，即LLM-as-a-judge系统。

让我们来看看他们是如何应对这一挑战的！

## 系统架构

![系统架构](/images/posts/788aa60de3d3.png)

整个系统使用多个组件，以便提供基于多种工具和外部知识的聊天机器人答案。它包含几个关键要素：

- **知识库**：
  - **预处理**：第一步是在Scio维护的API帮助下，将PDF文档导入Farmer.chat流程。在知识库中，主题被自动分类到相关地理区域，并进行语义分组。
  - **语义分块**：带有元数据的组织化文件被处理，将语义相似的句子分组到文本块中。该函数目前使用small-text嵌入进行余弦相似度计算。
  - **转换为向量数据库格式**：每个文本块使用嵌入模型转换为向量表示，并将向量表示存储在QdrantDB中。

- **RAG流程**：它确保所传递的信息基于内容而非外部信息。它由两部分组成：
  - **信息检索**：在知识库中搜索与用户查询匹配的相关信息。这涉及调用知识库构建器中创建的向量数据库API，以获取必要的文本块。
  - **生成**：利用文本块中检索到的信息和用户查询，生成器调用LLM，生成满足用户需求的人性化响应。

- **面向用户的Agent（智能体）**：
  - 规划Agent（智能体）底层使用GPT-4o。其任务是：
    - 理解用户意图
    - 根据用户意图和工具描述，决定还需要哪些信息
    - 向用户询问这些信息，直到需求明确
    - 一旦需求明确，调用执行Agent（智能体）
    - 从执行Agent（智能体）获取响应并生成回复
  - 该Agent（智能体）运行基于ReAct的提示词，以逐步方式思考，调用相应工具并分析响应。然后它可以利用其工具进行回答：目前，Agent（智能体）使用以下工具集：
    - 继续对话
    - RAG问答端点
    - 视频检索端点
    - 天气端点
    - 作物表

现在这个系统有许多活动部件，每个部件对性能的某些方面都有根本性影响。因此，我们需要仔细进行性能评估。

在过去一年中，Farmer.chat的使用量已增长到服务超过2万名农民，处理超过34万次查询。我们如何在这个规模上评估系统的性能？

在每周的头脑风暴会议中，Hugging Face提到了LLM作为评判者，并提供了他们的笔记本LLM as a Judge的链接。这个资源被详细讨论，随后形成的实践帮助引导了Farmer.chat的开发。

## LLM-as-Judge的力量

Farmer.Chat采用复杂的检索增强生成（RAG）流程，向农民提供基于知识库的准确且相关的信息。
RAG流程使用LLM从庞大的知识库中检索信息，然后生成简洁且信息丰富的回复。

但我们如何衡量这个流程的有效性？

这里的困难在于，没有确定的指标可以用来评价答案的质量、简洁性、精确性……

这就是 **LLM-as-a-judge（LLM 作为评判者）** 技术的用武之地。其理念很简单：让一个 LLM 根据任意指标对输出进行评分。  
其巨大优势在于，指标可以是任何内容：LLM-as-a-Judge 具有极强的通用性。

例如，你可以用它来评估提示词的清晰度，如下所示：

```
你将收到一个关于农业的用户输入，你的任务是根据多个方面对其进行评分。
请逐步思考，并根据以下三个标准对用户输入进行评分，每个标准给出一个分数：
1) 意图和请求清晰。
2) 主题明确。
3) 目标实体及其属性（例如“抗病性”或“高产”）明确。
请使用 1 到 3 的整数等级进行评分，1 为最差，3 为最好。

为每个标准打分后，取平均值并四舍五入到最接近的整数，作为最终分数。

示例：
用户输入：“说说巴蒂安咖啡品种的好处”
标准 1：得分 3，因为意图明确（了解巴蒂安咖啡品种）且请求清晰（总结好处）。
标准 2：得分 3，主题明确（咖啡品种）。
标准 3：得分 2，因为实体明确（巴蒂安品种），但属性不明确。

```

正如我们之前引用的[这篇文章](https://example.com)所述，使用 LLM-as-a-judge 的关键在于清晰定义任务、标准以及整数评分等级。

Farmer.Chat 背后的研究团队利用 LLM 的能力来评估几个关键指标：

- **提示词清晰度**：该指标评估用户清晰表达问题的能力。LLM 经过训练，可以评估用户意图的清晰度、主题的具体性以及实体-属性的识别，从而深入了解用户有效沟通需求的能力。
- **问题类型**：该指标根据认知复杂度将用户问题分为不同类别。LLM 分析用户查询，并将其归入六个类别之一，例如“记忆”、“理解”、“应用”、“分析”、“评估”和“创造”，帮助我们理解用户交互的认知需求。
- **已回答查询**：该指标追踪聊天机器人回答问题的百分比，深入了解知识库的广度以及平台处理广泛查询的能力。
- **RAG 准确性**：该指标评估 RAG 管道检索信息的忠实度和相关性。LLM 充当评判者，将检索到的信息与用户查询进行比较，并评估回答是否准确和相关。

它使我们能够超越简单地衡量聊天机器人能回答多少问题或响应速度有多快。  
相反，我们可以更深入地探究回答的质量，并以更细致的方式理解用户体验。

对于 RAG 准确性，我们使用 LLM-as-a-judge 以二元等级进行评估：0 或 1。  
但任务的分解方式形成了一个成熟的流程，最终得出一个分数。我们在大约 360 个问题上与人工评估员进行了测试：发现 LLM 的回答实际上表现出色，与人工评估具有高度相关性！

以下是受 RAGAS 库启发的提示词。

```
你是一个自然语言推理引擎。你将收到一组事实陈述和上下文。你需要分析每个陈述在给定上下文中是否事实正确。你可以给出“是”（1）和“否”（0）作为判定分数。

请遵循以下规则：
如果陈述可以从上下文中推导出来，则给出分数 1。
如果没有陈述且没有上下文，则给出分数 1。
如果陈述不能从上下文中推导出来，则给出分数 0。
如果没有上下文但有陈述，则给出分数 0。

#### 输入：

上下文：{context}

陈述：{statements}

```

上面的上下文变量是用于生成回答的输入文本块，而陈述是由另一个 LLM 调用生成的原子事实陈述。

这是非常重要的一步，因为它实现了规模化评估，这在处理大量文档和查询时至关重要。  
LLM-as-a-judge 的核心是提供指标，这些指标如同指南针，指引我们为 AI 管道选择各种可用方案。

## 结果：为 RAG 对 LLM 进行基准测试

我们创建了一个包含超过 700 个用户查询的样本数据集，这些查询在不同价值链（作物）和日期（月份）中随机分布。虽然本次升级本身有 11 个不同的版本，使用 RAG 准确性和已回答百分比进行评估，但同样的方法也用于衡量领先 LLM 的性能，且在每个 LLM 调用中不改变提示词。在此实验中，我们选择了 OpenAI 的 GPT-4-Turbo、Gemini-1.5（Pro 和 Flash 版本）以及 Llama-3-70B-Instruct。

我们发现，在这四个模型中，事实正确回答的最高水平（“忠实度”列）由 Gemini-1.5-Pro 获得，Gemini-1.5-Flash 和 GPT-4-turbo 紧随其后。

我们发现，仅基于忠实度而言，Gemini-1.5-Pro 优于其他模型。但如果同时考虑模型接受回答问题的百分比，Llama-3-70B 和 Gemini-1.5-Flash 表现更好。

最终，我们选择了 Gemini-1.5-Flash，因为它在未回答问题比例低和忠实度极高之间取得了优越的平衡。

## 结论

通过利用 LLM 作为评判者，我们能够更深入地理解用户行为以及 AI 驱动工具在农业背景下的有效性。这种数据驱动的方法对于以下方面至关重要：

- **改善用户体验**：通过识别用户在表达需求方面存在困难或 RAG 管道性能未达预期的领域，我们可以改进平台的设计和功能。
- **优化知识库**：对未回答查询的分析有助于我们识别知识库中的空白，并优先安排内容开发。
- **选择合适的 LLM**：通过根据关键指标对不同 LLM 进行基准测试，我们可以就哪些模型最适合特定任务和场景做出明智的决策。

LLM 作为评判者来评估 AI 系统性能的能力是一项颠覆性的创新。它使我们能够以更客观、更数据驱动的方式衡量这些系统的影响，最终为农业开发出更稳健、更有效、更用户友好的 AI 工具。

在过去一年多的时间里，我们不断迭代我们的产品。在这短短的时间内，我们实现了：
