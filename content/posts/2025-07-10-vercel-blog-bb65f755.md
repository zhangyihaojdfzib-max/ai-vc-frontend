## 统一、自主驱动的平台

AI 云之所以强大，其原理与前端云取得成功的原因相同：基础设施应源于代码，而非手动配置。**框架定义的基础设施** 能自动将您的应用逻辑转化为运行的云服务。随着我们看到智能体和 AI 生成的代码比以往任何时候都多，这一点变得尤为重要。

借助 AI 云，您（或您的智能体）可以在不接触底层基础设施的情况下构建 AI 应用。

以 **AI SDK** 为例，它通过标准化众多供应商的代码，让您轻松使用 LLM，并允许您在不更改代码的情况下切换模型。AI 推理通过通用化许多供应商特定的流程而变得简单。

当 AI SDK 部署在 Vercel 应用中时，调用会被路由到相应的供应商。它们也可以通过 **AI 网关**，这是一个全局的、与供应商无关的层，用于管理 API 密钥、供应商账户，并通过重试、回退和性能优化来提高可用性。

```
1import { streamText, StreamingTextResponse, tool } from 'ai';2import { z } from 'zod';3
4export async function POST(req: Request) {5  const { prompt } = await req.json();6

27  return new StreamingTextResponse(result);28}
```

一个使用 AI SDK 和 AI 网关的示例 AI API 端点。其结构类似于传统端点，通过一个简单的包来接受前端的提示词并流式返回响应。

在此示例中，AI SDK 定义了交互，而 AI 网关处理执行。两者共同降低了构建和扩展能够真正推理和推断意图的 AI 功能所需的开销。

这些 AI 调用通常作为简单的函数运行，必须能够即时扩展。但与典型工作负载不同，LLM 交互常常涉及等待时间和较长的空闲期。这打破了传统无服务器架构的运营模式，在非活动期间效率低下。AI 工作负载需要一个既能处理突发流量又能处理空闲期，且开销最小的计算模型。

### AI 云计算

AI 云的核心是 **弹性计算**，它针对这些工作负载进行了优化，同时消除了传统无服务器和服务器模式之间的权衡，例如冷启动、手动扩展、过度配置和低效并发。

弹性计算采用无服务器模型进行部署，同时在扩展以创建新资源之前智能地重用现有资源。通过 **动态 CPU 定价**，不仅资源消耗减少，而且您只需在代码主动执行时支付计算费用。

对于空闲时间较长的工作负载，例如等待外部响应的 AI 推理、智能体或 MCP 服务器，与传统无服务器相比，这种资源效率可将成本降低高达 90%。这种效率同样适用于 AI 智能体使用工具时。

### 工具执行

在识别意图并生成计划的推理阶段之后，智能体通常会执行工具。AI SDK 管理此过程：注册工具、将其暴露给模型并处理执行，所有这些都运行在弹性计算上。

工具可以根据任务顺序或并行执行。这些工具调用可能是在 **Vercel Functions** 中运行的简单函数，也可能被路由到 MCP 服务器（一种由 Anthropic 引入并得到 Vercel 支持的协议）。

### 简化的 MCP 服务器支持

MCP 服务器在许多方面类似于 API 路由：一个可以作为 Functions 运行的单一端点，但其内部就像一个量身定制的工具包，帮助 AI 完成特定任务。单个 MCP 服务器背后可能使用多个 API 和其他业务逻辑。

Vercel MCP 适配器简化了在 Vercel 上构建 MCP 服务器的过程。使用 `@vercel/mcp-adapter` 包，可以创建新的 API 端点，或将现有端点转换为服务 MCP，从而轻松为智能体提供对核心应用功能的访问。

### 使用Vercel沙盒实现安全执行

有时，动作涉及运行由Agent（智能体）生成的代码。由于此代码未经用户验证，因此被视为不可信代码。它可能完全无害，但此代码不应拥有对您的Vercel部署及其关联的环境变量、API密钥等的相同特权访问权限。

这时就需要**Vercel沙盒**。Agent（智能体）在采用动态CPU定价的Fluid计算上运行，为不可信代码启动临时的、隔离的服务器。虚拟机快速启动、安全运行、提供用户可访问的URL，然后干净地终止。

沙盒支持包括Node.js和Python在内的多种运行时环境，预装了常用软件包，并允许通过`sudo`权限安装其他软件包。
