## 摘要

React Server Components 中存在的多个高严重性漏洞已被负责任地披露。重要的是，这些漏洞不允许远程代码执行。

我们创建了新规则来解决这些漏洞，并将其部署到 Vercel WAF，以自动免费保护所有托管在 Vercel 上的项目。但是，请勿依赖 WAF 提供完全保护。必须立即升级到已修复的版本。

## 影响

### React CVE-2026-23864 (CVSS 7.5)

CVE-2026-23864 解决了 React Server Components 中的多个拒绝服务漏洞。这些漏洞通过向服务器函数端点发送特制的 HTTP 请求触发，可能导致服务器崩溃、内存不足异常或 CPU 使用率过高；具体影响取决于被利用的易受攻击代码路径、应用程序配置和应用程序代码。

这些漏洞存在于以下包的 19.0.x、19.1.x 和 19.2.x 版本中：

*   react-server-dom-parcel
*   react-server-dom-webpack
*   react-server-dom-turbopack

这些包包含在以下框架和打包器中：

*   Next.js: 13.x, 14.x, 15.x, 和 16.x。
*   其他嵌入或依赖 React Server Components 实现的框架和插件（例如，Vite, Parcel, React Router, RedwoodSDK, Waku）

## 解决方案

在创建了缓解措施来解决此漏洞后，我们将其部署到我们全球分布的平台，以保护我们的客户。我们仍然建议升级到最新的已修复版本。

React 和受影响的下游框架的更新版本包含了修复此问题的补丁。所有用户应尽快升级到已修复的版本。

## 修复版本

*   React: 19.0.4, 19.1.5, 19.2.4。
*   Next.js: 15.0.8, 15.1.12, 15.2.9, 15.3.9, 15.4.11, 15.5.10, 15.6.0-canary.61, 16.0.11, 16.1.5, 16.2.0-canary.9

使用上述包的框架和打包器应安装其各自维护者提供的最新版本。
