与Gemma 4对话，她会自行决定是否需要查看网络摄像头来回答你。这一切都在Jetson Orin Nano Super上本地运行。

```
你说话 → Parakeet STT → Gemma 4 → [如果需要则调用网络摄像头] → Kokoro TTS → 扬声器

```

按空格键开始录音，再按空格键停止。这是一个简单的视觉语言智能体（VLA）：模型根据你提问的上下文自行决定是否采取行动，没有关键词触发，没有硬编码逻辑。如果你的问题需要Gemma"睁开眼睛"，她会决定拍照、解读，并基于所见内容回答你。她不是在描述图片，而是在利用她看到的信息来回答你的实际问题。

## 步骤 4：运行 Gemma 4 服务

在启动演示之前，你需要一个运行着Gemma 4的 `llama-server`。我们将在Jetson上原生构建 `llama.cpp`，这能提供最佳性能和对VLA演示所需的视觉投影器的完全控制。

### 构建 llama.cpp

```bash
cd ~
git clone https://github.com/ggml-org/llama.cpp.git
cd llama.cpp
cmake -B build \
  -DGGML_CUDA=ON \
  -DCMAKE_CUDA_ARCHITECTURES="87" \
  -DGGML_NATIVE=ON \
  -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release -j4
```

### 下载模型和视觉投影器

```bash
mkdir -p ~/models && cd ~/models

wget -O gemma-4-E2B-it-Q4_K_M.gguf \
  https://huggingface.co/unsloth/gemma-4-E2B-it-GGUF/resolve/main/gemma-4-E2B-it-Q4_K_M.gguf

wget -O mmproj-gemma4-e2b-f16.gguf \
  https://huggingface.co/ggml-org/gemma-4-E2B-it-GGUF/resolve/main/mmproj-gemma4-e2b-f16.gguf
```

`mmproj` 文件是视觉投影器。没有它，Gemma 就看不见，所以不要跳过它。

### 启动服务器

```bash
~/llama.cpp/build/bin/llama-server \
  -m ~/models/gemma-4-E2B-it-Q4_K_M.gguf \
  --mmproj ~/models/mmproj-gemma4-e2b-f16.gguf \
  -c 2048 \
  --image-min-tokens 70 --image-max-tokens 70 \
  --ubatch-size 512 --batch-size 512 \
  --host 0.0.0.0 --port 8080 \
  -ngl 99 --flash-attn on \
  --no-mmproj-offload --jinja -np 1
```

有一个标志值得提及：`-ngl 99` 告诉 `llama-server` 将模型的所有层推送到GPU上（99只是表示"模型有多少层就推送多少层"）。如果你遇到内存问题，可以降低这个数字，将更少的层卸载到GPU，其余的留在CPU。不过对于这个设置，所有层都在GPU上应该可以正常工作。

### 验证服务已启动

从另一个终端：

```bash
curl -s http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"gemma4","messages":[{"role":"user","content":"Hi!"}],"max_tokens":32}' \
  | python3 -m json.tool
```

如果你收到JSON响应，那就没问题了。

## 工作原理

该脚本向Gemma 4暴露了一个工具：

```json
{
  "name": "look_and_answer",
  "description": "使用网络摄像头拍照并分析可见内容。"
}
```

当你提问时：

1.  你的语音被本地转录（Parakeet STT）
2.  Gemma 收到文本加上工具定义
3.  如果问题需要视觉，她会调用 `look_and_answer`，脚本抓取网络摄像头帧并发送回去
4.  Gemma 回答，然后Kokoro 大声说出来

没有关键词匹配。模型自行决定何时需要"看"。这就是VLA部分。

`llama-server` 上的 `--jinja` 标志实现了这一点，它激活了Gemma原生的工具调用支持。

## 故障排除

服务器内存不足，请再次执行步骤3的清理操作。关闭所有程序。该模型可在8GB内存上运行，但必须保持环境整洁。

无声音输出，请检查 `pactl list short sinks` 并确保 `SPK_DEVICE` 与实际声卡设备匹配。

麦克风录制无声，请先用 `arecord -l` 仔细检查设备，然后手动测试录音。

首次运行速度较慢属正常现象。系统正在下载模型并生成语音提示词。第二次运行会变快。

## 附加内容：只想在文本模式下试用 Gemma 4？

如果您不关心完整的视觉语言智能体演示，只想在 Jetson 设备上体验 Gemma 4 且无需编译任何内容，Jetson AI Lab 提供了一个开箱即用的 Docker 镜像，其中包含为 Orin 平台预编译的 `llama.cpp`：

```bash
sudo docker run -it --rm --pull always \
  --runtime=nvidia --network host \
  -v $HOME/.cache/huggingface:/root/.cache/huggingface \
  ghcr.io/nvidia-ai-iot/llama_cpp:latest-jetson-orin \
  llama-server -hf unsloth/gemma-4-E2B-it-GGUF:Q4_K_S
```

单行命令，无需编译，首次运行时 `-hf` 参数会从 Hugging Face 拉取 GGUF 格式模型。使用任何兼容 OpenAI 的客户端访问 `http://localhost:8080` 即可开始对话。

请注意：此 Docker 方案仅支持纯文本模式。它不会加载视觉投影器，因此无法与上文的视觉语言智能体演示配合使用。如需完整的摄像头交互体验，请继续使用步骤4中的原生构建方案。
