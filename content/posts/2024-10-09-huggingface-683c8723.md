# 欢迎使用 Gradio 5

过去几个月我们一直在努力工作，现在很高兴地宣布 Gradio 5 稳定版正式发布。

借助 Gradio 5，开发者只需几行 Python 代码，就能构建出性能卓越、可扩展、设计精美、易于访问且遵循最佳 Web 安全实践的生产级机器学习 Web 应用。

要体验 Gradio 5，只需在终端中输入：

```
pip install --upgrade gradio
```

然后开始构建您的第一个 Gradio 应用。

## Gradio 5：生产级机器学习应用

如果您之前使用过 Gradio，可能会好奇 Gradio 5 有哪些不同之处。

Gradio 5 的目标是倾听并解决 Gradio 开发者在构建生产级 Gradio 应用时最常遇到的痛点。例如，我们听到一些开发者反馈：

- "Gradio 应用加载太慢" → Gradio 5 带来了重大的性能改进，包括通过服务端渲染（SSR）提供 Gradio 应用的能力，使应用在浏览器中几乎瞬间加载完成。不再有加载转圈！🏎️💨

- "这个 Gradio 应用看起来太老派" → 许多核心 Gradio 组件，包括按钮、标签页、滑块以及高级聊天机器人界面，在 Gradio 5 中都采用了更现代的设计。我们还发布了一套新的内置主题，让您轻松创建外观新颖的 Gradio 应用 🎨

- "我无法在 Gradio 中构建实时应用" → 我们解锁了 Gradio 的低延迟流式传输功能！我们会在能提升速度的地方自动使用 base64 编码和 WebSocket，通过自定义组件支持 WebRTC，并且还增加了大量针对常见流式传输用例的文档和示例演示，例如基于摄像头的目标检测、视频流传输、实时语音转录与生成，以及对话式聊天机器人 🎤

- "LLM（大语言模型）不了解 Gradio" → Gradio 5 附带了一个实验性的 AI Playground，您可以使用 AI 生成或修改 Gradio 应用，并立即在浏览器中预览应用：https://www.gradio.app/playground

Gradio 5 在提供所有这些功能的同时，保持了 Gradio 简单直观的开发者 API。由于 Gradio 5 旨在成为适用于各类机器学习应用的生产级 Web 框架，我们还在 Web 安全方面进行了重大改进（包括对 Gradio 进行第三方审计）——更多内容将在后续文章中介绍！

## 破坏性变更

在 Gradio 4.x 中未触发任何弃用警告的 Gradio 应用，在 Gradio 5 中应能继续运行，但有少量例外情况。请在此处查看 Gradio 5 的破坏性变更列表。

## Gradio 的下一步计划？

我们在 Gradio 5 中做出的许多更改，旨在支持未来几周即将发布的新功能。敬请期待：

- 多页面 Gradio 应用，以及原生导航栏和侧边栏
- 支持通过 PWA 在移动设备上运行 Gradio 应用，并可能支持原生应用
- 更多媒体组件，以支持图像和视频领域的新兴模态
- 更丰富的 DataFrame 组件，支持常见的电子表格类型操作
- 与机器学习模型和 API 提供商的一键集成
- 进一步降低 Gradio 应用内存消耗的改进

以及更多功能！Gradio 5 为构建 Web 应用提供了坚实的基础，我们非常期待让开发者能够使用 Gradio 构建各种类型的机器学习应用。
