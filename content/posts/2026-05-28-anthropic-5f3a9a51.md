# 推出 Claude Opus 4.8

![推出 Claude Opus 4.8](/images/posts/5b855f1cba4a.jpg)

我们正在将 Claude Opus 升级至新版本：Claude Opus 4.8。它在 Opus 4.7 的基础上进行了多项基准测试改进，并且是更高效的协作者。即日起可用，价格不变。

Opus 4.8 与多项新功能同步推出。claude.ai 上的用户现在可以控制 Claude 在任务中投入的努力程度。Claude Code 新增了"动态工作流"功能，使其能够处理超大规模问题。而 Opus 4.8 的快速模式——模型能以 2.5 倍速度运行——现在比之前模型便宜三倍。

## 与 Opus 4.8 协作

早期测试者发现，Claude Opus 4.8 在执行 Agent（智能体）任务时判断更加可靠且更为敏锐。以下是这些测试者关于与 Opus 4.8 协作体验的引述：

Opus 4.8 最显著的改进之一是其诚实性。我们训练所有模型保持诚实——例如，避免做出无法支持的断言。但 AI 模型的一个普遍问题是，它们有时会仓促下结论，尽管证据不足却自信地声称工作取得了进展。早期测试者报告称，Opus 4.8 更倾向于标记其工作中的不确定性，且较少做出无依据的断言。这一点在我们的评估中得到了证实，评估显示 Opus 4.8 对其所编写代码中的缺陷不予标记的可能性比前代产品低约四倍。

与往常一样，我们在发布前对模型进行了详细的对齐评估。在积极特质方面，我们的对齐团队得出结论，Opus 4.8"在我们衡量亲社会特质（如支持用户自主性和以用户最佳利益行事）的指标上达到了新高。"评估还显示，Opus 4.8 的不对齐行为（如欺骗或与滥用行为合作）发生率显著低于 Opus 4.7，且与我们对齐最佳的模型 Claude Mythos Preview 相当。完整的对齐评估及一系列部署前安全测试已在 Claude Opus 4.8 系统卡中报告。

## 今日同步推出

除 Claude Opus 4.8 外，我们还进行了以下更新：

- **动态工作流。** 这项新功能以研究预览形式提供，允许 Claude 在 Claude Code 中承担更大的任务。Claude 可以规划工作，然后在单次会话中运行数百个并行子 Agent（智能体）（使用 Opus 4.8 时，Agent（智能体）可运行更长时间）。随后，它在向用户报告之前会验证其输出。例如，配备 Opus 4.8 的 Claude Code 现在可以从启动到合并，在现有测试套件作为基准的情况下，执行跨数十万行代码的代码库规模迁移。您可以在本文中了解更多关于动态工作流的信息——该功能适用于 Enterprise、Team 和 Max 计划的 Claude Code。
- **claude.ai 和 Cowork 中的努力控制。** 模型选择器旁新增了一个控件，允许用户选择 Claude 在回复中投入的努力程度。在较高努力设置下，Claude 会更频繁、更深入地思考以提供更好的回复。在较低努力设置下，Claude 会更快响应，并更慢地消耗用户的速率限制。用户现在拥有这一选择——努力控制适用于所有计划。
- **Messages API 现在接受消息数组内的系统条目。** 开发者可以在任务中途更新 Claude 的指令，而无需破坏提示词缓存或通过用户轮次路由更新。这可以在特定框架中用于在 Agent（智能体）运行时更新权限、Token 预算或环境上下文。

## 关于努力的说明

Opus 4.8 默认设置为高努力，我们认为这是质量和用户体验的最佳整体平衡。在编码任务上，此努力级别消耗的 Token 数量与 Opus 4.7 默认设置相似，但性能更优。用户可以选择"额外"（Claude Code 中为"xhigh"）或"最大"，模型将消耗更多 Token 以获得更好结果；我们建议对困难任务和长时间运行的异步工作流使用"额外"。我们已提高 Claude Code 中的速率限制，以适应更高努力级别的更高 Token 使用量；用户可以根据其特定项目选择最合适的设置。

## 下一步计划？

用户会发现 Opus 4.8 相比前代产品是适度但切实的改进。仍有更多工作要做：我们正在开发并发布能够以更低成本提供与 Opus 相同许多能力的模型。

不仅如此，我们计划发布比 Opus 智能水平更高的一类新模型。作为 Project Glasswing 的一部分，少数组织目前正在使用 Claude Mythos Preview 进行网络安全工作。这种能力水平的模型需要更强的网络安全保障才能公开发布。我们正在快速推进这些保障措施的开发，并预计在未来几周内能够将 Mythos 类模型带给所有客户。
