当 Databricks 在 2021 年率先推出 Delta Sharing 时，我们旨在解决每个数据团队都深有体会的问题：跨组织边界共享实时数据既缓慢又脆弱，且充满妥协。你要么复制数据——导致产生过时副本和合规性难题——要么只能局限于与同一平台上的合作伙伴共享，从而严重限制创新。

Delta Sharing 改变了这一切。一个单一的开放协议。无需复制数据。没有平台孤岛。在此后的五年里，它已成为最广泛采用的开放零拷贝数据共享协议——拥有超过 28,000 个数据接收方，33% 的共享通过开放连接器跨平台流动。SAP、Atlassian、梅赛德斯-奔驰、The Trade Desk、LSEG、S&P Global 等领先公司均已采用 Delta Sharing 来共享和协作处理数据。

但世界已经向前发展。Agentic AI 的兴起从根本上改变了企业需要共享的内容。今天，我们正在迈出下一步。

我们激动地宣布 OpenSharing——Delta Sharing 的下一代演进，也是业界首个为 Agentic 时代构建的开放协议。OpenSharing 将 Delta Sharing 推进为一个独立的开源项目，将其范围从数据共享扩展到完整的 AI 堆栈：模型、Agent（智能体）——跨越任何云、任何供应商和任何格式。

## 为什么共享协议需要为 AI 而演进

Delta Sharing 是为表和文件的世界构建的。但组织现在必须跨云、供应商和公司边界交换语义上下文、AI 技能、非结构化数据和自主 Agent（智能体）。如今的共享协议仍然锁定在供应商特定的格式中，无法处理 AI 逻辑，并且依赖于脆弱的网络连接，为每个新合作伙伴配置需要数周时间。

结果：协作变慢，数据孤岛持续存在，企业数据中被锁定的价值无法实现。

OpenSharing 解决了这个问题。它是一个单一的开放协议，可以跨任何格式、任何云和任何组织边界共享数据和 AI——原生支持 Delta Lake、Apache Iceberg 和 Parquet，使数据保留在其所在位置，并流向需要它的人。

## Databricks 上的 OpenSharing

OpenSharing 存在于两个层面。开源协议——现已由 Linux 基金会托管——是已发布的规范，任何供应商或社区成员都可以实施。Databricks OpenSharing 是该开放协议的企业级实现，构建在 Databricks 的其他功能之上，例如用于治理和审计日志记录的 Unity Catalog，以及用于可发现性的 Marketplace 等。

我们很高兴为 Databricks 上的 OpenSharing 推出一系列功能。

### Genie Agent 共享：共享受治理的 AI 体验，而不仅仅是数据

组织首次能够跨组织边界共享受治理的 AI 体验——而不仅仅是数据集。

Genie Agent 是 Databricks 基于 AI 的对话式分析环境。借助 OpenSharing，提供商现在可以与任何合作伙伴或客户共享 Genie Agent——包括其底层的语义上下文、业务指标和可重用的 AI 逻辑——并通过 Unity Catalog 实现端到端治理。可选地，提供商可以控制接收方访问数据的方式——包括隐藏专有的 Genie 指令、将数据访问限制为仅限 Genie Agent、设置每日提示词配额以及限制行导出上限。这些控制为数据提供商解锁了新的变现机会，例如基于使用量的定价，而不是完整的数据许可。

### SecureConnect 和 Global Distribution：更简单的多云网络，更低的出站成本

跨云数据共享一直存在两个不同的问题。Databricks 上的 OpenSharing 现在解决了这两个问题。

第一个是网络。当提供商存储位于私有网络之后——对于敏感数据交换或受监管行业来说几乎总是如此——接入新接收方可能需要数周的手动 IP 白名单、防火墙协调以及与云管理员的反复沟通。对于拥有数十或数百个接收方的提供商来说，这无法扩展。SecureConnect 解决了这个问题：一个由 Databricks 管理的代理，代表所有接收方路由存储访问。只需配置一次——无需为每个接收方更改防火墙设置。阅读公告博客。

![SecureConnect](/images/posts/e6aa2f1b56a2.gif)

第二个是出站成本。跨云查询会产生出站费用，并且会随着规模扩大而累积，成为一笔巨大且不可预测的成本，使得广泛的多云共享在经济上不可行。Global Distribution 通过自动跨区域和跨云复制解决了这个问题。接收方查询本地副本——速度快，且无出站费用。提供商获得可预测的成本结构。全球团队无论源数据位于何处，都能获得低延迟访问。

### 开放客户端互操作性与本地存储生态系统：在合作伙伴所在之处与他们协作

OpenSharing 建立在这样的信念之上：数据生态系统只有在真正开放时才能蓬勃发展——不仅是名义上的，而且是实践中的。这意味着支持您的合作伙伴已经在使用的格式、存储系统和客户端。

存储生态系统：治理一切，无论数据位于何处

并非所有企业数据都能——或应该——迁移到云端。监管要求、数据引力、边缘延迟以及纯粹的经济因素意味着一些世界上最有价值的数据将保留在本地。OpenSharing 可以触及这些数据。Databricks 存储生态系统将 Databricks 数据智能平台直接带到本地、私有云和边缘环境——由 OpenSharing 提供支持。存储合作伙伴实施 OpenSharing 服务器，将其数据资产连接到 Unity Catalog，而无需移动一个字节。无需迁移。无需复制。阅读公告。启动合作伙伴包括 MinIO（GA）、Everpure（Private Preview）、Qumulo（Private Preview Soon）和 VAST Data（Private Preview Soon）——Cohesity、Commvault、NetApp 和 Nutanix 将在年底前加入。这些合作伙伴共同管理着数百 EB 的企业数据。

Iceberg 互操作性 Delta Sharing 已在广泛的平台和连接器中得到支持，包括 Databricks、Tableau、Power BI、Apache Spark 和 Snowflake。OpenSharing 现已增加对 Apache Iceberg REST Catalog API 的支持——使得与任何兼容 Iceberg 的客户端共享数据成为可能。提供商还可以从外部目录（包括 AWS Glue、Hive Metastore 和 Snowflake Horizon）共享表——将外部数据引入受治理的 OpenSharing 生态系统，而无需复制。

## OpenSharing 的工作原理

基于使 Delta Sharing 成功的相同简洁性，OpenSharing 扩展了协议以支持完整的 AI 资产堆栈：

1.  数据提供商在 Unity Catalog 中创建共享——定义要共享哪些数据集、模型、Agent（智能体）或 Genie Agent，并设置细粒度的访问权限。
2.  接收方接收安全凭据，并直接从其现有工具、云或 Iceberg 客户端查询共享——无需在 Databricks 上。
3.  Unity Catalog 强制执行端到端治理——审计每次访问，实施行级和列级控制，并确保合规策略随每个共享资产一起传递。
4.  数据从不移动——接收方直接从提供商的云存储中查询实时数据，确保单一事实来源。

对于 Databricks 上的企业部署，SecureConnect 和 Global Distribution 在此流程之上分层——自动处理跨云网络和复制，无需更改提供商或接收方与其共享交互的方式。
