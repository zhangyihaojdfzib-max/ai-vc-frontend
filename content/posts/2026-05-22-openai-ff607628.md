# Rox 全面押注 OpenAI

通过将商业经验与深厚的 LLM 专业知识同 OpenAI 的模型相结合，Rox 让每一位销售人员都成为顶尖 1% 的销售精英。

Rox⁠(在新窗口中打开)正在重塑销售团队管理和增长营收的方式。其平台将分散的数据整合为统一的记录系统，并通过由 OpenAI 驱动的始终在线的 AI Agent 集群提供洞察。

Rox 背后的团队拥有深厚的 AI 与数据集成专业知识。联合创始人兼 AI 负责人 Avanika Narayan 将其博士研究中将 LLM 应用于数据整理和工作流自动化等企业任务的经验带到了 Rox。

她分享道：“我们看到了一个丰富的人类互动与协作生态系统存在于营收团队中，但意识到他们需要以数据为基础。”最终形成了一个三层愿景：记录系统、工作流系统以及 AI 驱动的辅助——这些通过 OpenAI 的 API 得以实现。

## 满足营收团队不断变化的需求

对于 Rox 来说，2024 年发布不仅仅是时机问题，更是机遇问题。基于使用量的营收模式的兴起以及向 AI 驱动的销售运营的转变，暴露了许多营收团队系统中的缺口。

OpenAI 的 API 为 Rox 提供了灵活性，使其能够处理从数据清洗到长工作流再到针对市场策略的精细化销售拓展等各种任务。“使用 OpenAI 的 API 让我们能够创造新颖的东西，并将创意注入 GTM 工作流中，”Rox 的创始 AI 工程师之一 Amol Singh 指出。

该平台的设计基于三个层次：

- **数据层**：Rox 使用 GPT‑4o mini 等较小模型，将来自不同数据仓库的分散数据统一起来，使其结构化且易于检索。
- **智能层**：中层模型执行复杂推理并确定行动优先级，与销售代表协作以增强其任务执行能力。
- **交互层**：GPT‑4o 和 OpenAI 的 Realtime API 等高级模型生成电子邮件、管理 LinkedIn 拓展，并准备支持语音的会议简报。

核心是“Rox Agent Swarm”，即分配给每个账户的一群始终在线的 AI Agent。

当销售代表离线时，Agent 监控账户；在工作时间，它们通过处理重复性任务和呈现可操作的洞察，使销售代表的效率提升 50%。

## 构建 Rox：由顶级销售专家启发的可配置营收平台

Rox 的发布之路受到早期实验和客户反馈的塑造。最初，团队探索了一个开放式聊天界面，但发现它不符合销售人员的需求；相反，“销售人员需要一个针对其特定工作流程量身定制的解决方案，输出更具创造性和独特性，”Rox 的创始 AI 工程师之一 Alex Derhacobian 表示。

团队转而构建一个端到端的可配置平台，每个 Agent 都能适应销售代表的具体账户和工作流程。

这种方法通过与 Ramp、Couchbase 和 Confluent 等公司的合作获得启发，并通过将顶级销售人员的实践最佳方案构建到 Rox 平台中，将他们的专业知识扩展到整个团队。

团队将其迭代方法——每天发布更新——归功于加速了开发进程。“传统上，这需要多个数据工程师和整个团队。我们只用两个人就做到了，这得益于 OpenAI 的模型以及构建难度的降低，”Singh 说。

## 销售接受管道翻倍

使用 Rox 的销售团队报告了显著的改进：

- **每周节省 8 小时以上**：销售代表重新获得了之前花在行政任务上的时间，使他们能够专注于更高价值的客户互动。
- **客户参与度提升 35%**：Agent 帮助销售代表及时响应关键账户变化，并在关键时刻进行互动。
- **销售管道 ROI 提升 2 倍**：Beta 客户的销售接受管道增加了 2 倍，促使许多客户在团队中扩大 Rox 的使用范围，并增加在平台上管理的账户数量。

Rox 在企业客户中迅速获得吸引力，在七个月内从零增长到 25 个账户。

## 与合适的合作伙伴共同构建

OpenAI 的模型是 Rox 方法的核心，使其能够在不同任务中提供灵活的 AI 解决方案。OpenAI“拥有一个完整的模型库，你可以从中选择，具有不同的能力和价格，”Narayan 说。

这种灵活性使 Rox 能够在数据处理中部署成本效益高的模型，同时将更高级的模型保留用于复杂推理任务。

Rox 的联合创始人分享了构建有效 AI 解决方案的三个关键经验：

1. **从强大的数据层开始**：“最重要的事情是建立好数据层并解决上下文管理问题，”Derhacobian 分享道。对半结构化和非结构化数据进行索引可确保洞察具有可操作性。
2. **专注于应用型 AI**：Rox 选择与 OpenAI 合作，而不是构建自己的模型。“我们不是为了研究或创建基础模型而来。我们与 OpenAI 合作，交付出色的应用程序，”Rox 联合创始人兼 CEO Ishan Mukherjee 表示。
3. **快速迭代**：现代应用型 AI 允许更快的节奏。“我们几乎每天发布更新，”Rox 的创始 AI 工程师之一 Damon Lin 说，将他们的成功归功于持续迭代。

团队计划扩展 Rox 的多模态能力和长周期任务辅助能力。由 OpenAI 的 Realtime API 驱动的新语音功能，已经帮助销售人员通过详细的实时简报为会议做准备。

借助 OpenAI，Rox 将继续赋能以人为核心的销售职能，最大化 GTM 营收潜力。
