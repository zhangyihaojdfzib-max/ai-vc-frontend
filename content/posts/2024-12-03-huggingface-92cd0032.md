# 投资于性能：借助LLM洞察微调小型模型——CFM案例研究

概述：本文深入探讨了资本基金管理公司（CFM）如何利用开源大语言模型（LLM）和Hugging Face（HF）生态系统来优化金融数据的命名实体识别（NER）。通过借助LLM辅助标注、HF推理端点以及使用Argilla优化数据，该团队将准确率提升了高达6.4%，并降低了运营成本，实现了比单独使用大型LLM成本低80倍的解决方案。

在本文中，您将了解到：

- 如何利用LLM进行高效的数据标注
- 借助LLM洞察微调紧凑模型的步骤
- 在Hugging Face推理端点上部署模型以实现可扩展的NER应用

这种结构化方法兼顾了准确性和成本效益，使其成为实际金融应用的理想选择。

资本基金管理公司（CFM）是一家总部位于巴黎的另类投资管理公司，同时在纽约和伦敦设有团队，目前管理资产总额达155亿美元。

CFM采用科学化的金融方法，利用定量和系统性手段来制定卓越的投资策略。

CFM一直与Hugging Face的专家支持团队合作，以紧跟机器学习领域的最新进展，并利用开源技术的力量支持其广泛的金融应用。此次合作的主要目标之一是探索CFM如何通过高效使用开源大语言模型（LLM）来增强其现有的机器学习应用场景。量化对冲基金依赖海量数据来决定是否买入或卖出特定金融产品。除了金融市场（如价格）的标准数据源外，对冲基金越来越多地从另类数据中提取洞察，例如新闻文章中的文本信息。将新闻纳入全自动交易策略的一大挑战是准确识别文章中提及的产品或实体（如公司、股票、货币）。虽然CFM的数据提供商会提供这些标签，但它们可能不完整，需要进一步验证。

CFM探索了多种改进金融实体识别的方法，包括使用LLM和较小模型的零样本NER、借助Hugging Face推理端点和Argilla进行LLM辅助数据标注，以及在精选数据集上微调较小模型。这些方法不仅利用了大型模型的通用性，还解决了实际金融应用中成本和可扩展性的挑战。

在开源模型中，Meta的Llama 3.1系列因其在各项基准测试中的强劲表现而脱颖而出，成为生成合成标注的首选。这些LLM在创建高质量标注数据集方面发挥了关键作用，将自动化与人类专业知识相结合，简化了标注流程，并提升了金融NER任务中的模型性能。

### 从端点获取预测

现在我们已经准备好了提示词和端点，下一步是使用数据集中的标题发送请求。为了高效完成此操作，我们将使用`huggingface_hub`库中的`AsyncInferenceClient`。这是`InferenceClient`的异步版本，基于`asyncio`和`aiohttp`构建。它允许我们向端点发送多个并发请求，从而更快、更高效地处理数据集。

```python

```

为了确保模型返回结构化输出，我们将使用一个特定的Pydantic模式`Companies`进行引导。

```python
from pydantic import BaseModel 
from typing import List, Dict, Any


class Companies(BaseModel):
    """
    表示预期LLM输出的Pydantic模型。
    
    属性：
        result (List[str])：来自LLM的公司名称或结果列表。
        normalized_result (List[str])：'标准化'后的公司名称列表，即经过处理/清理的名称。
    """
    result: List[str]
    normalized_result: List[str]

grammar: Dict[str, Any] = {
    "type": "json_object",
    "value": Companies.schema()  
    
}

```

我们还设置了生成参数：

```python
max_tokens: int = 512  
temperature: float = 0.1  

```

现在我们定义函数，用于向端点发送请求并解析输出：

```python
async def llm_engine(messages: List[Dict[str, str]]) -> str:
    """
    向LLM端点发送请求并获取响应的函数。

    参数：
        messages (List[Dict[str, str]])：传递给LLM的消息列表。

    返回：
        str：响应消息的内容，如果出错则返回'failed'。
    """
    try:
        
        response = await client.chat_completion(
            messages=messages,
            model="ENDPOINT",  
            temperature=temperature,
            response_format=grammar,
            max_tokens=max_tokens
        )
        
        answer: str = response.choices[0].message.content
        return answer
    except Exception as e:
        
        print(f"LLM引擎出错：{e}")
        return "failed"

def parse_llm_output(output_str: str) -> Dict[str, Any]:
    """
    将LLM输出的类JSON字符串解析为字典。

    参数：
        output_str (str)：LLM输出的字符串，预期为JSON格式。

    返回：
        Dict[str, Any]：从输入JSON字符串解析出的字典，包含'valid'标志。
    """
    try:
        
        result_dict: Dict[str, Any] = json.loads(output_str)
        result_dict["valid"] = True
        return result_dict
    except json.JSONDecodeError as e:
        
        print(f"JSON解码错误：{e}")
        return {
            "result": [output_str],
            "normalized_result": [output_str],
            "valid": False
        }

```

现在，我们创建一个`process_batch`函数，用于以可控批次处理请求，防止API端点过载或触发速率限制。这种分批处理方法使我们能够高效地并发处理多个请求，而不会使服务器饱和，从而降低超时、请求被拒或限流的风险。通过控制请求流量，我们确保稳定的性能、更快的响应时间以及更简便的错误处理，同时最大化吞吐量。

```python
import asyncio

```python
async def process_batch(batch):
    """ 
    获取一批样本的模型输出。
    
    该函数通过将一批样本发送至LLM（大语言模型）并并发收集结果来进行处理。
    
    参数:
        batch (List[Dict[str, str]]): 字典列表，每个字典包含单个样本的数据，
                                      其中包括"Article_title"。
    
    返回:
        List[str]: LLM（大语言模型）对该批次中每个样本的响应列表。
    """
    
    list_messages = []
    
    
    for sample in batch:
        messages = [
           {"role": "system", "content": SYSTEM_PROMPT}, 
           {"role": "user", "content": sample["Article_title"]} 
        ]
        list_messages.append(messages) 

    
    return await asyncio.gather(*[llm_engine(messages) for messages in list_messages])

```

我们将对整个数据集进行推理：

```python
from datasets import load_dataset, Dataset


dataset = load_dataset("Zihan1004/FNSPID", streaming=True)

iterable_dataset = iter(dataset["train"])


def get_sample_batch(iterable_dataset, batch_size):
    batch = []
    try:
        for _ in range(batch_size):
            batch.append(next(iterable_dataset))
    except StopIteration:
        pass
    return batch

```

然后我们创建主推理循环：

```python

batch_size = 128
i= 0
len_extracted = 0

while True:
    batch = get_sample_batch(iterable_dataset, batch_size)
    
    predictions = await process_batch(batch)
    parsed_predictions = [parse_llm_output(_pred) for _pred in predictions]
    try :
        parsed_dataset = [
            {"Article_title": sample["Article_title"],
             "predicted_companies": pred["result"],

             "valid": pred["valid"]} for sample, pred in zip(batch, parsed_predictions)
            ]
    except Exception as e :
        print(i,e)
        continue

    
    with open(os.path.join(CHECKPOINT_DATA, f"parsed_dataset_{i}.json"), 'w') as json_file:
        json.dump(parsed_dataset, json_file, indent=4)  
    
    len_extracted += len(parsed_dataset)

### 使用Argilla审查预测结果

利用LLM（大语言模型）生成的标注数据，下一步是整理一个高质量的子集，以确保对不同方法（包括零样本和微调方法）进行可靠评估。这个经过仔细审查的数据集也将作为微调较小模型的基础。LLM（大语言模型）标注的整个数据集也可用于弱监督框架下的微调。

数据集采样与划分

为了创建可管理的审查样本量，我们使用了LLM（大语言模型）标签，并通过rapidfuzz库采用模糊匹配对公司名称进行聚类。我们应用cdist函数计算公司之间的莱文斯坦距离，并以85为阈值进行聚类。为每个聚类选择一个代表性公司，并相应映射Llama的预测结果。最后，我们从每个公司相关聚类中抽取5%的新闻标题，以及从不含任何公司的标题中抽取5%，最终得到包含2714个样本的数据集。然后利用元数据，将采样数据集分为三部分：

- 训练集：2010年至2016年的新闻，用于训练小型机器学习模型 - 2405个样本
- 验证集：2017年至2018年的新闻，用于超参数调优 - 204个样本
- 测试集：2019年至2020年的新闻，用于评估模型在未见数据上的表现 - 105个样本

## 弱监督 vs. LLM辅助标注

在本实验中，我们探索了两种关键的NER数据标注方法：弱监督和LLM辅助标注。虽然弱监督能够在合成数据上进行可扩展的训练，但我们的研究结果表明，它无法达到与人工标注数据训练模型相同的准确率。对于1000个样本，人工标注耗时3小时，F1分数为0.915；而Llama 3.1-70b推理仅需2分钟，但F1分数略低，为0.895。速度与准确率之间的权衡取决于任务需求。
