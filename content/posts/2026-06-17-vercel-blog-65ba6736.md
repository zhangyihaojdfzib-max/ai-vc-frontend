## Link to headingAgentic 基础设施

Guillermo 在 Ship 大会上首先分享了他对 Vercel 的愿景：一个真正的全栈平台，你可以部署任何东西，包括能够思考的软件。

Agentic 基础设施包含三个部分：

- Vercel 是编码 Agent（智能体）部署软件的地方。当你询问 Claude Code 或 Codex 在哪里部署时，你会得到 Vercel，因为 Vercel 是为 Agent（智能体）的工作方式而构建的。
- Vercel 是你构建和部署自己 Agent（智能体）的地方。我们为你提供在生产环境中安全、大规模构建和运行应用及 Agent（智能体）所需的一切工具。
- Vercel 本身也由 Agent（智能体）实现自动化。Vercel 在生产环境中运行你的应用，处理流量、追踪、可观测性和异常情况。这些数据为我们的 Agent（智能体）提供了自主调查所需的上下文，然后提出拉取请求，而不仅仅是发出警报。

### Link to headingAgent Stack

无论工作负载如何，每个 Agent（智能体）都需要连接模型、跨多个步骤运行工作流，并连接使其有用的系统和人员。Vercel 为你提供了构建所有这三种能力所需的每个原语。

- AI SDK 为你提供一个 API 来调用任何模型，因此流式传输、工具调用和结构化输出在所有提供商中都能以相同方式工作
- AI Gateway 从一个端点将你的请求路由到数百个模型，并在提供商宕机时自动故障转移
- Workflow SDK 通过内置的自动重试、状态持久化和可观测性，使每次运行都持久可靠
- Vercel Sandbox 为每个 Agent（智能体）提供一个隔离的微虚拟机，用于在代码部署到生产环境之前运行和测试
- Chat SDK 让你能够从单一代码库将 Agent（智能体）部署到 Slack、Discord、GitHub 等平台

#### Link to headingVercel Connect

Vercel Connect 在伦敦的 Ship 2026 大会上发布，成为 Agent Stack 中最新的构建模块。

![](/images/posts/cc4ea96fc4d6.jpg)

Connect 为 Agent（智能体）提供对其使用的工具、数据和服务的安全访问，无需在环境变量中存储长期有效的提供商令牌。你的应用或 Agent（智能体）会请求一个临时凭证，其范围仅限于当前任务，因此没有长期存在的密钥可供泄露。

安全地连接你的 Agent（智能体）。

使用 Vercel Connect 保护你的提供商凭证安全，在你的 Agent（智能体）跨应用工作时对其进行身份验证和授权。

#### Link to headingeve

在 Vercel 构建了数百个 Agent（智能体）后，我们注意到底层架构具有相同的形态。eve 就是这种形态，作为一个开源框架，它是我们在 Vercel 构建、运行和扩展生产级 Agent（智能体）的方式。

eve Agent（智能体）位于单个目录中，指令用 Markdown 编写，工具用 TypeScript 编写。持久化执行、沙箱计算、审批、子 Agent（智能体）和评估都已内置。

在几分钟内交付一个生产级 Agent（智能体）。

每个 Agent（智能体）都是一个目录，你可以使用已有的命令来搭建、运行和部署。

## Link to headingVercel Agent

Malte 宣布了 Vercel Agent（内测阶段），这是 Vercel 平台上的智能层。它基于 eve 和 Agent Stack（智能体栈）构建，能够监控你的生产部署，自主调查告警和异常，并在 PR 中为你提供修复方案以供审核和批准。

![](/images/posts/a9587e1b19e1.jpg)

Vercel Agent 首创的权限模型将计划模式与权限授予相结合，使其对开发者和企业来说都安全可靠。Vercel Agent 不会要求你逐一批准操作，而是会规划完成某项任务所需的权限，然后在一个连贯的步骤中请求你批准。它以其独立的身份运行，默认只读，并在接触生产环境之前请求狭窄的、临时的权限。

**Arthur Viegers**（Cursor）：自主性应与风险挂钩，Agent（智能体）评估变更风险的能力越强，就越能允许其自主运行。Shopify 和 Amplitude 已经能自动审核并合并 60-70% 的低风险 PR，无需开发者介入；而涉及认证的两行代码变更仍需人工处理。

**Ryan den Rooijen**（Currys）：关于实体店消亡的预测从未成真，Currys 通过押注人类来引领其市场。对于 Agent（智能体）驱动的商业，悬而未决的问题是，客户希望 Agent（智能体）处理哪些购物旅程，而哪些他们仍然希望由真人来服务。

Nicolas Le Pallec（AKQA）：AI原生的客户体验用意图取代了导航。要实现这一目标需要三个层面：一种捕捉真实客户意图的方式，一个存储品牌和产品数据供AI系统检索的品牌大脑，以及能够动态生成页面而非提供静态页面的生成式UI。
