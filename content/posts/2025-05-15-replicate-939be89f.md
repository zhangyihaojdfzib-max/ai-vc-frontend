# 在 Hugging Face 上使用 Replicate 运行 30,000+ 个 LoRA

-   shridharathi

![一张精美的排版海报，文字为'在 Hugging Face 上使用 Replicate 运行 30,000+ 个 LoRA'，背景是色彩缤纷的 AI 生成图像](/images/posts/bae10424d7cf.webp)

LoRA 已成为训练图像模型以表达特定概念或风格的主流方法。想想 Studio Ghibli 的静帧风格，或者捕捉 80 年代赛博朋克的氛围。

Hugging Face 是分享和尝试 LoRA 的首选平台。艺术家、研究人员和爱好者们在那里上传他们的自定义风格，让任何人都能轻松使用。它已发展成为网络上最大的 LoRA 开源集合之一。

现在，LoRA 可以直接在 Hugging Face Hub 上使用 Replicate 进行推理。这得益于 Hugging Face 推理客户端的一个小更新，它在幕后连接到了 Replicate。结果是：您可以直接在 Hugging Face 界面上，使用您喜爱的 LoRA 进行快速、低成本的推理。

![一张精美的排版海报，文字为'在 Hugging Face 上使用 Replicate 运行 30,000+ 个 LoRA'，背景是色彩缤纷的 AI 生成图像](/images/posts/43bc7ec520da.webp)

### 工作原理

当您在 Hugging Face 上选择一个受支持的 LoRA 模型并选择 Replicate 作为推理提供商时，Hugging Face 会将请求路由到 Replicate 的 `black-forest-labs/flux-dev-lora` 模型。我们将请求的 LoRA 作为该模型的动态输入，通过 `lora_weights` 参数传入。

这意味着每个 LoRA 在 Replicate 上共享同一个后端模型，我们会即时动态切换正确的权重。例如：

```
const input = {
  prompt: "style of 80s cyberpunk, a portrait photo",
  lora_weights: "https://huggingface.co/fofr/flux-80s-cyberpunk",
};

const output = await replicate.run("black-forest-labs/flux-dev-lora", { input });
```

这使得我们能够支持 Hugging Face 的 Flux 库中的所有 LoRA，而无需为每个 LoRA 单独托管一个模型。
