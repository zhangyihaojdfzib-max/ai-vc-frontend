检索是现代人工智能系统的基石，而嵌入模型的质量决定了应用程序在企业数据中查找和推理的有效性。今天，我们在 Databricks 上推出 Qwen3-Embedding-0.6B，这是一款先进的嵌入模型，提供强大的检索性能、多语言覆盖和安全的无服务器部署。

结合 Agent Bricks 和向量搜索，该模型使团队能够直接在 Databricks 的企业数据上构建 AI Agent（智能体），检索相关上下文并在受治理的数据上进行推理，而无需将数据移出平台。

## 使用 Agent Bricks 构建检索驱动的 Agent（智能体）

先进的嵌入模型是现代人工智能系统的关键基础，使应用程序能够从海量企业数据集合中检索正确的上下文。现已在 Databricks 上可用的 Qwen3-Embedding-0.6B，为这类工作负载提供了强大的检索性能。

Qwen3-Embedding-0.6B 基于强大的 Qwen3 基础模型构建，来自广受采用的 GTE 系列背后的同一研究团队。该模型拥有 32k Token 的最大上下文窗口，为将文档切分成各种不同大小提供了极大的灵活性。此外，其指令感知设计让开发者能够通过简单的提示词，将模型定制到特定任务和语言，通常能将检索性能提升 1-5%。

在 Databricks 上，可以将其与 Agent Bricks 和向量搜索结合，直接在企业数据上构建检索驱动的 AI Agent（智能体）。团队可以使用向量搜索为文档建立索引，并在 Agent（智能体）执行期间检索相关上下文，使 Agent（智能体）基于存储在 Databricks 中的受治理数据进行推理。

## 此嵌入模型如何改进 Databricks 上的 AI Agent（智能体）

Qwen3-Embedding-0.6B 在其规模级别上提供了顶尖的质量。在 MTEB 多语言和英语 v2 排行榜上，它超越了大多数其他 0.6B 级别的模型，并超越了 OpenAI 和 Cohere 的旗舰嵌入模型，同时可与更大的 7B+ 模型相媲美。这意味着您无需承担超大模型的延迟和成本，即可实现顶级的检索性能。

该模型还通过 Matryoshka Representation Learning（MRL）提供了对成本和召回率的细粒度控制，该方法将最重要的信息集中在向量的前几个维度。这使得嵌入向量可以被安全地截断，以实现更便宜的存储和更快的搜索，同时保留大部分信号。使用 Qwen3-Embedding-0.6B，您可以在请求时选择从 **32 到 1024 维度** 的任何嵌入大小——使用较小的向量进行大规模召回索引，使用全尺寸向量进行更高精度的重排序。

要在使用 `databricks-qwen3-embedding-0-6b` 时使用此功能，请在您的 Embeddings REST API 请求中将可选的 `dimensions` 字段设置为所需的输出大小（32 到 1024 之间的 2 的幂）。详情请参阅 **Foundation Model REST API 文档**。

### 天生多语言

Qwen3-Embedding-0.6B 是 Databricks 托管的第一个多语言嵌入模型，从一开始就为全球工作负载设计。虽然许多嵌入模型以英语为主，多语言支持有限，但 Qwen3-Embedding-0.6B 继承了 Qwen3 基础模型的广泛语言覆盖能力，该基础模型在涵盖 100 多种语言的文本上进行了预训练。

这不仅使其在英语检索上表现出色，在多语言和跨语言任务上同样强大。应用程序可以用一种语言进行搜索，并以另一种语言检索结果，或者支持混合语言数据集以及跨多种编程语言的代码检索。

### 安全无服务器部署

与其他 Databricks 托管的基础模型一样，Qwen3-Embedding-0.6B 运行在 Databricks 平台内安全、完全托管的无服务器 GPU 上。

只需调用基础模型 API，Databricks 就会处理资源调配、自动扩缩和可靠性。由于该模型运行在具有地理感知、合规的基础设施上，您可以将嵌入向量靠近您的数据存储，满足数据驻留要求，并将检索直接与现有的 Databricks 工作负载集成。

### 立即试用 Qwen3-Embedding-0.6B！

无论您是在构建语义搜索、RAG（检索增强生成）管道、多语言检索还是文本分类系统，Qwen3-Embedding-0.6B 都提供了速度、效率和顶尖准确性的卓越组合。该模型在所有支持 **Foundation Model Serving** 的区域和云平台上均以 `databricks-qwen3-embedding-0-6b` 的名称提供，您可以在 Databricks Serving 页面试用此模型。它适用于所有模型服务模式：按 Token 付费、AI Functions（批量推理）和预置吞吐量。您也可以为向量搜索用例选择此模型。
