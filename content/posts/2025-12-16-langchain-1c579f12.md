Fastweb + Vodafone（隶属于瑞士电信集团）是欧洲领先的电信提供商之一，为数百万意大利客户提供服务。认识到传统的客户服务方法有待改进以满足其不断增长的需求，Fastweb + Vodafone 开启了一场人工智能转型，旨在彻底革新其客户支持服务方式。

电信行业的客户服务规模庞大，挑战复杂且相互关联。客户需要就账单查询、服务激活、漫游问题和技术支持等获得即时协助，通常期望在一次互动内解决问题。传统的 TOBi 系统难以处理那些需要上下文理解、访问多个系统并进行端到端解决的复杂请求。

对于 Fastweb + Vodafone 的呼叫中心顾问而言，挑战同样复杂。客服人员需要快速访问客户数据、了解服务历史并确定合适的解决方案，同时保持高服务标准。虽然手动查阅多个系统和知识库的过程是有效的，但若能获得智能支持，将能进一步提升客户体验的速度和一致性。因此，Fastweb + Vodafone 需要一个能够同时处理面向客户的互动并赋能呼叫中心客服人员持续提供卓越服务的人工智能解决方案。

**利用 LangChain 和 LangGraph 构建智能客户互动**

Fastweb + Vodafone 选择 LangChain 和 LangGraph 作为其人工智能转型的基础，因为他们的客户服务流程天然地映射到基于图的决策流程。他们的实施围绕两个旗舰项目展开：Super TOBi 和 Super Agent。

**Super TOBi：企业级人工智能客户服务**

Super TOBi 在 Fastweb + Vodafone 现有的聊天机器人（TOBi）基础上构建，将其转变为一个能够跨多个渠道处理复杂客户互动的智能体系统。该系统遵循结构化方法，模拟经验丰富的客户服务代表处理查询的方式。

该系统的架构围绕两个主要智能体组织，两者均实现为 LangGraph 图：监督者和用例。

监督者作为所有用户查询的中央入口点。其首要职责是应用护栏、过滤和塑造输入，以确保其有效且安全。此外，它还管理特殊场景，如对话结束、转接人工客服、简单互动（如问候）等。输入处理完毕后，监督者要么将查询直接路由到最合适的用例，要么在不确定的情况下提出澄清问题以确定正确的前进路径。通过这种方式，它编排整个对话流程，确保每个查询都能得到一致且高效的处理。

用例代表专门的智能体，每个都旨在解决特定类别的客户需求。它们能够访问明确定义的客户 API 子集，并遵循 LLM 编译器模式。这种方法使它们能够推理应调用哪些 API，在需要时协调多个步骤，并生成针对客户上下文定制的最终响应。重要的是，一些用例不仅限于自然语言响应：它们还可以发出结构化操作标签。这些标签使系统能够在对话中直接执行事务流程。例如，一个操作标签可能启动优惠激活、禁用正在进行的服务或触发支付方式更新。

当返回此类操作标签时，聊天机器人会在对话界面中自动执行它们。这种集成创造了对话与行动的无缝融合，使客户能够在同一次对话体验中，轻松地从提问过渡到完成交易。

Fastweb + Vodafone 在 LangGraph 中实现的 LLM 编译器模式使系统能够为每个客户请求生成全面的计划，无缝执行 API 调用、数据检索和多步骤问题解决。

目前，Super TOBi 通过客户伴侣应用和语音渠道为近 950 万客户提供服务，处理包括成本控制、有效优惠、漫游、销售和账单在内的用例，并取得了令人印象深刻的业务指标：90% 的正确率，82% 的解决率，以及 7 分制中 5.2 分的客户费力程度评分，这转化为更快的响应时间、更少的人工客服转接和更高的客户满意度。

**Super Agent：利用 LangGraph 赋能卓越呼叫中心**

Super Agent 是 Fastweb + Vodafone 面向内部的 AI 项目，通过即时诊断、合规指导和有来源支持的解释来增强每位顾问的能力。与面向消费者的机器人 Super TOBi 不同，Super Agent 从不直接与客户对话；相反，它为人工顾问提供确切的下一步指导。得益于这种方法，生成式 AI 的使用已将一次性解决率推高至 86% 以上。

该解决方案融合了 LangChain 的可组合工具与 LangGraph 的可控编排，并将所有运营知识作为动态图存储在 Neo4j 中。

**从业务模板到图：一站式自动化 ETL 流程**

业务专家使用包含定义步骤、条件和操作的结构化模板编写故障排除和信息程序。提交后，一个由 LangGraph 和特定任务 LLM Agent（包括 ReAct Agent）驱动的自动化 ETL 管道会将文档解析为 JSON，提取验证 API，执行一致性检查，并完善步骤定义。

内容被分解为节点和关系，并作为知识图谱的一部分存储在 Neo4j 中：步骤链接到条件、操作和相关 API 节点。随后，CI/CD 管道自动化构建、验证和部署流程，在数小时内将更新后的图谱推送到生产环境，且无需停机。

**意图路由器和执行流程**

来自顾问的传入请求，无论是关于用户问题的故障排除支持请求，还是一般知识库问题，都首先由 LangGraph 监督者处理，该监督者判断请求是匹配基于图的流程还是开放式问题。对于故障排除请求，系统会识别出由业务部门编写并作为知识图谱存储在 Neo4j 中的相应流程。在此阶段会自动注入 CRM 数据，以可靠地识别用户，确保每个请求都链接到正确的用户，并且响应是针对适当的客户上下文定制的。

**基于图的流程执行**

对于结构化的故障排除和故障隔离场景，监督者会激活一个流程子图执行器。系统使用 LangChain 和 LangGraph 从 Neo4j 中检索流程的第一个步骤节点及其关联的条件、操作和 API 节点。对于每个步骤，系统执行所需的 API 来验证步骤的条件。如果满足某个条件，流程结束：用户的问题被识别，并通过将相应的操作与从 API 收集的上下文信息相结合来生成响应。如果所有条件均未满足，则流程转到该流程中的下一个步骤节点，并迭代重复，直到找到问题及其解决方案。

**针对开放式问题的图 RAG**

关于公司知识库的通用或非结构化问题，则被路由到一个混合检索增强生成链，该链将向量存储与 Neo4j 知识图谱相结合。

向量数据库会找到一系列潜在相关的段落，随后知识图谱通过正确的上下文锚定答案，添加来源引用并确保其符合公司政策。
