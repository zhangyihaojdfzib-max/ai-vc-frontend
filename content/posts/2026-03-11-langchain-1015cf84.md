TL;DR：我们已在Deep Agents SDK（Python）和CLI中添加了一个工具，使模型能够在适当时机自行压缩其上下文窗口。

## 动机

**上下文压缩**是一种减少Agent工作记忆中信息的操作。较早的消息会被Agent进展的摘要或精简表示所取代，这些摘要或表示保留了与任务相关的内容。为适应有限的上下文窗口并减少**上下文腐化**，这一操作通常是必要的。

Agent框架通常通过固定的Token阈值来控制压缩（deepagents使用**模型配置文件**在任何给定模型上下文限制的85%处进行压缩）。这种设计并非最优，因为压缩存在合适的时机与不合适的时机：

- 在进行复杂的重构过程中进行压缩并不理想；
- 在开始新任务时，或者认为先前上下文将失去相关性时进行压缩则更为合适。

许多交互式编码工具都设有`/compact`命令或类似功能，允许用户在适当时机手动触发上下文压缩步骤。在deepagents的最新版本中，我们更进一步，向Agent公开了一个工具，使其能够自行触发上下文压缩。这实现了更具机会性的压缩，而无需让应用程序的用户意识到有限的上下文窗口或发出特定命令。

该工具目前已在**Deep Agents CLI**中启用，并在deepagents SDK中为可选功能。

我们普遍看好一个理念：框架应尽可能“让路”，并利用底层推理模型的改进。这是**苦涩教训**的一个实例：我们能否给予Agent更多对其自身上下文的控制权，以避免手动调整其框架？

## 何时应该压缩？

有多种情况可能需要进行上下文压缩操作。

在清晰的任务边界处：
- 用户表示将转向一个新任务，而较早的上下文可能已不相关
- Agent已完成一个可交付成果，且用户确认任务完成

从大量上下文中提取结果后：
- Agent通过消耗大量上下文（例如在研究任务中）获得了事实、结论、摘要或其他结果

在消耗大量新上下文之前：
- Agent即将生成长篇草稿
- Agent即将读取大量新上下文

在进入复杂的多步骤过程之前：
- Agent即将开始冗长的重构、迁移、多文件编辑或事件响应
- Agent已制定计划并即将开始执行步骤

已做出取代先前上下文的决定时：
- 新出现的需求使先前上下文失效
- 存在许多离题或死胡同，可以将其简化为摘要

枚举所有可能场景并不现实，但我们观察到，人类和LLM能够识别这些场景并在适当时机进行压缩，从而避免了在上下文窗口接近其限制时才进行压缩步骤的需要。您可以在其**系统提示词**中阅读我们为此工具向模型提供的指导。

## 调用该工具时会发生什么？

该工具的参数设置与现有的Deep Agents**摘要中间件**相同：我们保留最近的消息（可用上下文的10%），并对之前的内容进行摘要。最近的消息，包括对压缩工具的调用及相关响应，会保留在最近的上下文中。

## 使用方法

该工具作为独立的中间件实现，因此您可以通过将其添加到`create_deep_agent`的中间件列表中来启用它：

```python
from deepagents import create_deep_agent
from deepagents.backends import StateBackend
from deepagents.middleware.summarization import (
    create_summarization_tool_middleware,
)

backend = StateBackend  # if using default backend

model = "openai:gpt-5.4"
agent = create_deep_agent(
    model=model,
    middleware=[
        create_summarization_tool_middleware(model, backend),
    ],
)

```

更多详情请参阅SDK**文档**。

在CLI中，只需在准备精简上下文或转向新任务时调用`/compact`即可。

## 我们对此功能的体验

我们将此功能调整为保守型。Deep Agents在其虚拟文件系统中**保留所有对话历史**，允许在摘要后恢复上下文，但错误的上下文压缩步骤会带来干扰。我们测试了：

- 一个自定义评估套件，其中我们使用（我们自己的）**LangSmith追踪**向需要和不需要压缩的线程注入后续提示词；
- **Terminal-bench-2**，其中我们未观察到任何自主压缩的实例；
- 我们在**Deep Agents CLI**中自己的编码任务。

实际上，Agent在触发压缩方面是保守的，但当它们触发时，往往会选择那些能明显改善工作流程的时机。

自主上下文压缩是一个小功能，但它指出了Agent设计的一个更广泛方向：给予模型更多对其自身工作记忆的控制权，减少框架中僵化的、手动调整的规则。如果您正在构建长期运行或交互式的Agent，请在Deep Agents SDK或CLI中试用，并告诉我们您的反馈以及您希望它接下来处理哪些模式。
