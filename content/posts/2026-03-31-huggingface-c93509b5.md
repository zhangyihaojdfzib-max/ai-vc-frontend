# Granite 4.0 3B Vision：面向企业文档的紧凑型多模态智能

今天我们很高兴地宣布推出 Granite 4.0 3B Vision，这是一款专为企业文档理解设计的紧凑型视觉-语言模型。它专为从复杂文档、表格和结构化视觉内容中可靠地提取信息而构建。Granite 4.0 3B Vision 在以下能力方面表现出色：

- **表格提取**：从文档图像中准确解析复杂的表格结构（例如，多行、多列等）
- **图表理解**：将图表和图形转换为结构化的机器可读格式、摘要或可执行代码
- **语义键值对提取**：识别并定位不同文档布局中具有语义意义的键值字段对

该模型以 LoRA 适配器的形式发布，构建在我们的稠密语言模型 **Granite 4.0 Micro** 之上，保持了视觉与语言的模块化，以便在纯文本回退场景下使用，并能无缝集成到混合处理流程中。它继续支持视觉-语言任务，例如从图像生成详细的自然语言描述（例如，“详细描述此图像”）。该模型可以单独使用，也可以与 **Docling** 协同使用，通过深度视觉理解能力增强文档处理流程。

## Granite 4.0 3B Vision 的构建方式

Granite 4.0 3B Vision 的性能源于三项关键投入：通过新颖的代码引导数据增强方法构建的专用图表理解数据集、一种能够实现高细节视觉特征注入的 **DeepStack** 架构新变体，以及一种使模型对企业部署保持实用性的模块化设计。

### ChartNet：教会模型真正理解图表

图表对视觉-语言模型提出了挑战，因为理解它们需要同时对视觉模式、数值数据和自然语言进行联合推理，这是大多数 VLM 无法很好处理的组合，尤其是在空间精度至关重要时——例如从折线图中读取精确数值。为了弥补这一差距，我们开发了 **ChartNet**：一个专为图表解释和推理构建的百万级多模态数据集，详情将在我们即将发表的 CVPR 2026 论文中描述。

ChartNet 使用代码引导的合成流程，生成了 170 万个涵盖 24 种图表类型和 6 种绘图库的多样化图表样本 [见图 1]。其独特之处在于，每个样本由五个对齐的组件组成——绘图代码、渲染图像、数据表、自然语言摘要和问答对——为模型提供了关于图表含义（而不仅仅是外观）的深度跨模态视角。该数据集还包括人工标注和真实世界的子集，并经过视觉保真度、语义准确性和多样性的筛选。

其成果是一个训练资源，能够推动 VLM 从仅仅描述图表转变为真正理解它们所编码的结构化信息——在不同模型规模、架构和任务上均能获得一致的性能提升。

### DeepStack：更智能的视觉特征注入

大多数 VLM 在单一节点将视觉信息注入其语言模型，这迫使模型同时处理高级语义和细粒度空间细节。Granite 4.0 3B Vision 采用了不同的方法，即 **DeepStack 注入**：抽象视觉特征被路由到较早的层以进行语义理解，而高分辨率空间特征则被馈送到较后的层以保留细节。其结果是得到一个既能理解文档内容又能理解其位置的模型——这对于表格提取、图表理解和 KVP 解析等任务至关重要，在这些任务中，布局与内容同等重要。完整的技术细节，请参阅模型卡中的 **模型架构** 部分。

### 模块化：一个模型，两种模式

Granite 4.0 3B Vision 被打包为 **Granite 4.0 Micro** 之上的 LoRA 适配器，而非独立模型。在实践中，这意味着同一部署可以同时服务于多模态和纯文本工作负载，在不需要视觉功能时自动回退到基础模型。这使得企业集成变得直接明了，同时不牺牲性能。

## 如何使用

Granite 4.0 3B Vision 既可以作为独立的视觉信息提取引擎运行，也可以作为与 Docling 集成的全自动文档处理流程的一部分。该模型旨在支持跨不同文档类型和视觉格式的可扩展、准确提取。

1.  **独立的图像理解**
    Granite 4.0 3B Vision 可以直接在单个图像上运行，这使得该选项适用于那些已有工作流程、需要针对性视觉提取而无需修改上游系统的应用场景。这便于集成到现有的自动化工作流中，适用于轻量级、特定任务的工具（例如，表单解析器、图表分析器等）。

2.  **与 Docling 集成的文档理解流程**
    Granite 4.0 3B Vision 也可以与 Docling 无缝集成，以支持完整的端到端文档理解。此模式可以提供：
    - 大规模处理多页 PDF
    - 使用 Docling 自动检测、分割和裁剪图形、表格及其他视觉元素，并将干净的裁剪区域重定向到 Granite Vision 模型进行细粒度提取
    - 高效的工作流程，总体计算成本更低，吞吐量更快
    - 更高的准确性、更可靠的提取，以及在大规模文档集合中显著提升的效率

**示例用例**

- **表单处理**：利用键值对提取能力从发票、表格和收据中提取结构化字段，或通过图像转文本功能生成对图表的自然语言描述（例如“详细描述此图像”）。
- **财务报表分析**：使用Docling解析报告、检测图表并裁剪视觉元素。通过Granite Vision的chart2csv、chart2code功能处理图表，并利用tables_json能力处理表格，将其转换为结构化、机器可读的数据，从而获得可操作的洞察。
- **研究文档智能处理**：利用Docling处理密集学术PDF的OCR和版式解析，将提取的图表传递给chart2summary功能，表格裁剪区域传递给tables_html功能，实现在单一流程中使视觉内容与自由文本一同可检索。
