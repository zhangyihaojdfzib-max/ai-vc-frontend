您现在可以在AI Gateway中按成本、首Token时间（TTFT）或吞吐量（TPS）对模型背后的提供商进行排序。

默认的提供商顺序综合了提供商可靠性、模型输出质量、成本和响应速度。您现在可以使用`sort`来明确控制排序标准。

对于拥有多个提供商且成本或速度差异明显的模型，您可以使用`sort`在您选择的维度上进行优化。排序在请求时计算，因此新添加的提供商、价格变化以及观察到的延迟或吞吐量的变化都会自动生效，无需任何代码更改。

在`providerOptions.gateway`上设置`sort`为以下三个值之一：

| 值 | 描述 | 排序方向 | 使用场景 |
| :--- | :--- | :--- | :--- |
| `'cost'` | 按提供商列出的每百万Token输入价格排序 | 价格最低优先 | 高容量、对成本敏感的工作 |
| `'ttft'` | 按首Token时间中位数（毫秒）排序 | 延迟最低优先 | 对延迟敏感、响应速度至关重要的负载 |
| `'tps'` | 按每秒Token吞吐量中位数排序 | 吞吐量最高优先 | 总响应时间最重要的长输出生成 |

### 链接到标题基本用法

使用`sort`来确保针对您选择的指标进行优化。

在此示例中，AI Gateway有超过五个提供商提供`GPT OSS 120B`，且价格各不相同，因此对于希望路由到最低价格提供商的请求，按成本排序是一个有用的选项。

提供商按排序顺序尝试。仅当排名较高的提供商不可用时，才会回退到下一个提供商。

```
import { streamText } from 'ai';
const result = streamText({  model: 'openai/gpt-oss-120b',  prompt: 'Summarize this internal document.',  providerOptions: {    gateway: {      sort: 'cost',     },  },});
```

### 链接到标题与其他路由控制结合使用

`sort`与零数据保留（ZDR）等其他网关路由选项兼容。

下面的示例使用`deepseek/deepseek-v4-pro`进行一个对延迟和数据保留有要求的交互式请求：AI Gateway过滤出`Deepseek V4 Pro`中具有零数据保留的提供商，然后按首Token时间（TTFT）对剩余提供商进行排序。

```
import { streamText } from 'ai';
const result = streamText({  model: 'deepseek/deepseek-v4-pro',  prompt,  providerOptions: {    gateway: {      zeroDataRetention: true,      sort: 'ttft',     },  },});
```

DeepSeek V4 Pro的ZDR过滤和TTFT排序示例

`sort`还可以与`order`组合使用：在`order`中列出的提供商将被提升到最前面，其余提供商则遵循请求的排序标准。
