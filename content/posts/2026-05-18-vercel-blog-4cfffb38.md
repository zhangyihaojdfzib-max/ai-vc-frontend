# 合并提交状态现已在 GitHub 上可用

![](/images/posts/06eb496b2a10.jpg)

![](/images/posts/ae232276fe0f.jpg)

单体仓库现在可以选择在拉取请求上使用单一的合并提交状态，而不是每个项目单独一个提交状态。对于包含多个项目的仓库，团队可以一次性配置 GitHub 分支保护，然后在每个项目的设置中管理哪些 Vercel 项目是合并所必需的。

了解更多关于 GitHub 提交状态的信息，并从您的项目设置中启用合并提交状态。
