# Apple的Xcode现已支持Claude Agent SDK

Apple的Xcode是开发者构建、测试和分发适用于Apple平台（包括iPhone、iPad、Mac、Apple Watch、Apple Vision Pro和Apple TV）应用程序的集成开发环境。

今年九月，我们宣布开发者将能在Xcode 26中使用Claude Sonnet 4。当时Claude可用于编写代码、调试和生成文档——但其功能仅限于处理独立的、逐轮交互的请求。

现在，Xcode 26.3引入了对Claude Agent SDK的原生集成，该SDK正是驱动Claude Code的底层框架。开发者可直接在Xcode中获得Claude Code的全部能力——包括子智能体、后台任务和插件——所有这些都无需离开集成开发环境。

## 在Xcode中使用Claude执行长期自主任务

通过Claude Agent SDK，Claude现在能够在Xcode内部自主处理更复杂、更长期的编码任务。具体而言，此集成支持：

- **通过预览功能进行视觉验证**。借助新集成，Claude可以捕获Xcode预览界面，实时查看正在构建的界面效果，识别发现的问题并据此迭代调整。这在构建SwiftUI视图时尤为有用，因为视觉输出是最关键的考量因素。Claude能自主完成实现闭环，从而构建出更高质量的界面，首次尝试就能更贴近开发者的设计意图。
- **跨项目推理**。为Apple平台开发意味着需要处理多种框架和技术，如SwiftUI、UIKit、Swift Data等。Claude可以探索项目的完整文件结构，理解各组件间的关联，并在开始编写代码前确定需要修改的位置。当接收到任务时，它能基于对整个应用程序及其架构的理解开展工作——而不仅仅是当前打开的文件。
- **自主任务执行**。开发者可以向Claude下达目标，而非具体指令集。随后Claude会自行分解任务，决定需要修改哪些文件，实施更改，并在出现问题时进行迭代。当需要了解Apple API的工作原理或特定框架的使用方法时，Claude可以直接搜索Apple官方文档。它能根据需要更新项目并持续工作，直至任务完成或需要用户输入——这对经常独立工作或在小团队中工作的开发者而言，意味着显著的时间节省。
- **通过模型上下文协议交互**。除了在集成开发环境中直接访问Claude Agent外，Xcode 26.3还通过模型上下文协议提供其功能。使用Claude Code的开发者可以通过MCP与Xcode集成并捕获视觉预览，整个过程无需离开命令行界面。

## 可用性

Xcode 26.3现已作为候选发布版本，面向所有Apple开发者计划成员开放，即将在Apple App Store正式发布。更多详情请参阅Apple的公告。
