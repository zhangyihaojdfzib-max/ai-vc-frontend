许多创始人花费数年时间寻找产品市场契合。而对有些人来说，产品市场契合会主动找上门。

2016年，特里斯坦·汉迪怀揣着朴素的抱负创立了一家小型咨询公司：帮助初创企业巩固其数据系统。他和联合创始人迅速整合出一款内部产品——他们称之为数据构建工具，即dbt——以加速他们的咨询工作。

近十年后，这款工具已演变为一个价值42亿美元的云端分析工程平台，每周有超过5万个团队使用。如今，其背后的公司被称为dbt Labs（该公司近期ARR已突破1亿美元）。

汉迪从未想过自己会执掌一家价值数十亿美元的风险投资公司，甚至是一家咨询公司。尽管曾在多家初创企业工作，他从未怀有某天创立自己企业的梦想。他的创业之旅始于在分析技术初创公司RJMetrics工作期间——他爱上了最终促使他自立门户的云端技术。

他离职创立了Fishtown Analytics，公司以其家乡费城的社区命名。在运营咨询业务几周内，创始人们就开发了dbt供自用。他们将其作为开源仓库发布，心想或许会有少数人感兴趣，随后便继续服务客户。

事实证明，这款工具的受众远不止汉迪和GitHub上的一些数据极客。随后三年间，dbt通过口碑在线上传播，悄然积累了超过1000名用户——实现每年三倍增长。到2019年，汉迪勉强同意融资以扩大这款免费工具的自然增长规模，并于2021年将公司从Fishtown Analytics更名为dbt Labs，顺应了该工具突破性的成功。

"我本不想接受风险投资，但很明显，如果我们投资不足，就是对产品和已建立的社区不负责任，"汉迪说，"我们当时只有三名工程师支持着一千家公司。它被广泛使用，却完全资源不足。"

从咨询公司转型为SaaS企业并非寻常之举，但有其优势。当汉迪签署第一份投资条款书时，他已经与数十家初创企业数据团队建立了联系，并拥有dbt背后的社区引擎。

在此，汉迪坦诚分享了dbt Labs实现产品市场契合的非传统路径，揭示了公司从两人咨询工作室起步的所有曲折历程。这对于不确定是否适合走VC路线的有志创业者而言值得一读——它提醒我们，建立持久企业没有固定路径。

## 自举创立咨询公司

汉迪职业生涯的前十年在德勤和Squarespace领导分析职能，最终加入RJMetrics并担任数年营销副总裁。随后在2015年，云端数据的兴起颠覆了市场。

他接触了Amazon Redshift，仅通过两次查询就意识到这将彻底改变他的行业。"其性能比我之前使用的分析产品快了几个数量级，"他说，"在Redshift之前，只有大型企业才能获得这种性能。突然间，你只需刷卡支付每月160美元就能使用这类技术。"

汉迪看到技术格局在RJMetrics脚下发生变化。"我们永远无法预见到云端数据技术的崛起，"他说，"我们本应成为Looker那样的公司。云技术的到来颠覆了我们整个业务——它为Looker窃取我们原有的指数增长曲线创造了机会。"

汉迪被这项技术深深震撼，热切希望帮助初创企业利用云端数据平台。但他不想为此创立一家风险投资支持的公司。失败的风险显得过于巨大。

"当风险投资支持的公司一切顺利时，确实很棒。但往往情况并非如此，"他说，"领导一家未能获胜的组织会带来难以置信的个人压力。你从根本上感觉无法掌控进程，因为投资者对你抱有期望。"

因此汉迪认为咨询公司是风险较低的起点，但即使这个选择也令人畏惧。他难以想象自己成为创始人的样子。

"我以前从未创立过任何事业。我曾为三位其他创始人工作，总认为创始人是不同类型的人，而我不是那种人，"他说，"那是《社交网络》电影上映的时代，它让人们相信创始人应该是辍学大学生，会捶桌高喊'我们要改变世界！'"

其他疑虑也笼罩着他。"我害怕告诉所有亲友我尝试某件事却失败了，"他说。

于是他向曾共事的创始人们提出了咨询公司的想法，这最终帮助缓解了他的冒名顶替综合征。

回顾过去，汉迪认为对创业的犹豫是保持优先事项清晰的有效方式。"与其执着于建立公司的结果，我更执着于解决世界上某个问题的结果，"汉迪说。

在创始人朋友们的鼓励下，汉迪对创立Fishtown Analytics更加乐观。为降低个人风险，他从自己钱包中开出1万美元支票存入银行账户。"最坏情况，我可能损失几个月时间和1万美元。这不会是世界末日，"他说。

### 引入联合创始人

鉴于自身对创业的顾虑，汉迪起初不愿为新咨询公司寻找合作伙伴。用他的话说，他不想"多一张嘴吃饭"。

随后德鲁·巴宁重新进入汉迪的视野。巴宁曾在RJMetrics兼职软件工程师以支付大学费用，当时即将完成学位。他听说了汉迪的新事业——没有寻找毕业后的工作，而是主动提出加入。

"就在我起步时，德鲁把我拉到一边说：'嘿，我能帮忙吗？'"汉迪回忆道，"起初我相当抗拒。不是因为德鲁——他非常出色。而是想到在还没签下第一个客户时就要支付另一份薪水，压力很大。我只希望自己能解决这个问题。"

### 获得首批客户

现在涉及两份薪水，汉迪需要快速拓展业务。

他对SaaS和B2C公司需求的深刻理解形成了自然的客户画像。Fishtown Analytics的前两个客户是他在RJMetrics工作时结识的人。"这感觉很自然。我了解电商和SaaS，也知道有多少这类公司，因为我曾在RJMetrics做过市场分析，"他说。

以下是汉迪寻找客户的思路：

- **坚持极其具体的客户画像**，使新业务可复制。汉迪所有早期客户都符合相同标准：A轮、B轮或C轮的B2C或SaaS初创企业，特别是其CEO或营销副总裁，这些人需要理解他们的数据。"这是咨询公司建设者常犯的错误：他们接受任何找上门的工作。"
- **向客户清晰阐明**你将如何创造价值并节省成本。汉迪将Fishtown Analytics定位为初级数据分析师的精简替代。"我们说：'瞧，我们将比您内部招聘创造更多价值，'"他说，"我们的年费率约6万美元。我们知道能用不多的咨询费用快速交付难以置信的巨大价值。"

这种咨询工作的性质意味着 Handy 可以雇佣并培训像 Banin 这样的年轻人才加入团队。"如果你空降一位顾问并对他说'去和这位初创公司高管一起解决这些随机的数据问题'，那么他需要有 15 年的经验。但如果你说'嘿，这就是你要解决的具体问题集，这里有一些我们以前解决过类似问题的案例'，那么一个职业生涯刚起步的新人也能胜任。"

## 打造并开源 dbt 原型

当 Handy 和 Banin 创立 Fishtown Analytics 时，他们已经拼凑出了 dbt 的早期原型。他们将其视为提升自身咨询工作效率的工具——因此他们预计自己将是该工具仅有的两位用户。

在 2016 年那个时候，Handy 认为在被称为 ETL（提取、转换、加载数据以进行分析）的数据处理流程中，缺失了一个环节。许多软件公司处理提取和加载这两个环节，但很少有工具专门用于中间的转换环节——即清理数据以供使用。"为了做好咨询工作，我强烈认为我们需要一个工具，"他说。

dbt 的前提是这样的：将分析视为软件，由任何懂 SQL 的人编写，以融合分析和工程两个领域的最佳实践。

Handy 当时认为，像他这样兼具数据和技术复合技能、能够使用 dbt 的数据专业人士并不多。"它与之前存在的任何工具都截然不同，"他说，"2016 年之前的数据技术将数据分析师视为低层次、非技术人员。你可以在 Excel 或 Tableau 中看到这一点——你可以用这些产品做强大的事情，但界面更多是拖放式的。"

Handy 甚至向他的一些 VC 朋友提到了他正在构建的这个工具，他们也认为这可能没有很大的市场。

因此，Handy 和 Banin 构建了一个快速、粗糙但立即可用的产品，而没有投入大量的工程时间。Banin 是两人中编码能力更强的那个，所以他能够在晚上和周末快速构建出一个原型。

初始版本非常简陋，没有花哨的用户体验。"我们有一个版本，第一天大约投入了两周的工程时间，"他说，"当你构建命令行工具时，你不需要构建用户界面或登录系统，所以你可以用很小的努力获得很大的价值。"

因此，他们对客户完全隐藏了这个工具。"如果我们试图说服人们使用这个以前从未有人使用过的技术，可能效果不会好，"Handy 说。

但 Handy 和 Banin 自己用得很开心。"这是一个不科学的数字，但我想说，我们交付工作的速度至少比没有它时快 10 倍。它是让整个咨询模式运转起来的魔法银弹。"

dbt 从一开始就是开源的，当时只有 Handy 和 Banin 两个用户。

"当我们第一次构建 dbt 的初始版本时，我们把它放在 GitHub 的一个开放仓库中，并附上了 Apache 2.0 许可证，这样它就公开可用了，"Handy 说。"早期用户开始向他们的朋友介绍 dbt。人们会自己发现它。"

对于一个为 Handy 和 Banin 自身利益而开发的工具来说，这听起来可能是一个违反直觉的举动。Handy 表示，他决定开源软件源于对知识共享的承诺。"我们当时并不知道会有很多人使用 dbt，但我们认为这是个好主意，也许其他一些人会发现它的价值，"他说。

## 构建生态系统

一旦 Handy 签下了两个客户，他就不再需要过度依赖他在数据领域十多年积累的人脉。相反，他专注于营销和社区建设，以吸引潜在客户。

以下是 Handy 采取的一些举措，它们共同推动了 Fishtown Analytics 和 dbt 的口碑增长：

*   **从第一天起就押注内容营销。** 大多数创办咨询公司的人没有营销背景，但营销是 Handy 在初创公司七年的本职工作。因此，他没有忙于打陌生电话来寻找新的咨询客户，而是把时间花在写作上。"我通过撰写引起许多人共鸣的博客文章为我们建立了品牌。我有一份通讯，约有 6000 人阅读。"
*   **向客户揭开 dbt 的面纱。** 最终，Fishtown 开始与拥有更成熟数据团队的后期初创公司客户合作——这意味着他们雇佣了懂得如何使用 dbt 的技术数据人员。第一个直接使用 dbt 的 Fishtown 客户是床垫初创公司 Casper。"他们的数据团队在 2016 年雇佣我们来扩展他们的数据仓库。我们告诉他们，他们当时的做法不好，需要迁移到 dbt。所以他们要求做个演示。在接下来的一周里，我们重构了他们所有的现有管道，并将其迁移到 dbt。他们非常高兴，"他说。
*   **为 dbt 用户建立 Slack 频道。** 一旦 dbt 有了一些外部用户（包括像 Casper 这样的 Fishtown 客户和在 GitHub 上发现它的人），Handy 就为早期用户建立了一个 Slack 频道，以便交流经验。"到 2016 年底，大概有十几个人是早期采用者。我们都在试图弄清楚 dbt 如何让我们作为数据从业者以新的方式工作，"他说。

### dbt 独立潜力的初现

Handy 回忆起了他意识到 dbt 可能不仅仅是一个内部副业项目的时刻：一位知名的数据领导者在网上发现 dbt 后，出现在了 dbt 的 Slack 群组中。（他后来成为了 dbt 的长期用户之一，至今仍是。）

"一个陌生人在 Slack 上出现并说：'我听说过 dbt，我们一直在评估它供内部使用。'那个人是 SeatGeek 的数据科学副总裁。我们当时的反应是：'什么？'" Handy 说。

最终，Slack 社区开始代表 Handy 为 dbt 进行营销。"最奇怪的经历之一是，社区里那些管理数据团队的人会举办聚会，试图说服其他人使用 dbt——这样他们就可以雇佣 dbt 用户，"Handy 说。"我们没有为 dbt 设立专门的营销渠道。相反，人们会邀请我们参加关于我们正在构建的产品的聚会。"

## 转向软件业务并筹集资金

尽管 Handy 通过营销和社区建设为 Fishtown Analytics 赢得了稳定的客户群，但 dbt 的增长开始悄然超过咨询业务。到 2017 年，dbt——当时仍然是一个免费的开源工具——拥有了 100 个用户。

### 顿悟时刻：企业主动上门

起初，Handy 只是将 dbt 使用量的增加视为对咨询业务的利好。但鉴于他对数据趋势天生的好奇心，在 dbt 用户数超过 100 后，他和 Banin 在 dbt 内部安装了事件跟踪。"我们调出图表，意识到现在有 100 家公司使用 dbt，而我们只以咨询方式与其中大约 20 家合作过，"他说。

惊人的增长率没有逃过 Handy 的眼睛。"我们将那个时间序列数据放入 Google 表格，拟合了一条 S 曲线，它非常一致，每月增长 10%。当我在 Squarespace 时，我们也以同样的速度持续增长。所以我非常清楚地理解复利增长的价值。"

到 2019 年，Fishtown Analytics 由 Handy、Banin 和三名工程师组成。果然，增长曲线持续向右上方攀升。这家咨询公司现在已将 SeatGeek 和 Casper 列为客户，像 USAA 和可口可乐这样的大型企业也开始对使用 dbt 感兴趣——但 Fishtown 没有办法支持他们。"没有支持合同，他们无法单独使用开源软件，"他说。

很快，dbt 的用户数达到了 1000 家，并以每年 3 倍的速度增长。到 2019 年底，Handy 意识到，除了融资以维持可持续运营外，他别无选择。他说："我们当时只有三名工程师，却要支持一千家公司。dbt 通过其连接的数仓，实际产生了数亿美元的基础计算支出。我们不得不说服自己做出改变，不能再像以前那样运营公司了，尽管那样很有趣。"

![dbt 自 2016 年在 GitHub 上线至转型为软件业务期间的成长曲线](/images/posts/4c729cebb5a2.jpg)

### 意料之外的团队波折

然而，转型为一家风险投资支持的企业并非一帆风顺。Handy 表示，在接下来的一年里，Fishtown 流失了大约三分之一的现有团队成员。

到 2020 年初，Fishtown 已稳步发展成一个约 20 人的团队。同年 4 月，Handy 公开宣布了总额为 1290 万美元的种子轮和 A 轮合并融资，这是公司的首次融资。

他承认自己宣布消息的方式并非最佳。交易完成后，联合创始人立即在一次虚拟公司非现场会议上告诉了团队其他成员。"当我分享这个消息时，我以为每个人都会非常兴奋。但我意识到现场的气氛有些奇怪，"他说。

"对于任何企业战略的重大调整，这可能都是事实，但我低估了向团队解释新战略及其背后理由、以及不同成员如何适应新环境所需的工作量。相反，我只是扔下了一颗'炸弹'，然后说：'嘿，这难道不令人兴奋吗？'"

尽管经历了一些痛苦的团队过渡，Handy 仍对开始扩大 dbt 的增长感到兴奋。

## 将开源工具商业化

作为一家新获得风险投资支持的公司，Fishtown 的首要任务是为这款备受喜爱的开源工具打造一个商业版本。作为一家商业公司，接下来的篇章分为两个阶段：构建云服务产品，并说服企业为其付费。

### 增加云平台（以及构建它的工程师）

Handy 和团队在首次融资前就已经开始着手开发云平台，并于 2020 年初推出。

Fishtown 不得不将其有限的工程力量——当时只有大约五名工程师——投入到云平台功能的建设中。他们开发了一些标准的企业级功能，包括：

- 基于浏览器的 IDE
- 编排调度
- 基于角色的访问和控制
- 单点登录集成

一旦 Fishtown 获得了更多资金，他们便能够在 2020 年期间组建起一个两位数的工程师团队，以扩展云平台的开发。

### 赢得企业买家青睐

Handy 发现，dbt 作为一个主要由数据从业者和开发者使用的开源工具，已经自然而然地实现了产品市场契合。但在经营商业业务几年后，他意识到必须针对企业高层的数仓领导者重新构建增长曲线。

"尽管我们拥有难以置信的市场吸引力，但在最初商业化时，我们很难将这个开源命令行工具转变为企业愿意支付百万美元的产品，"Handy 说。

dbt 的产品驱动增长为云产品奠定了坚实的用户基础，这缩短了通常获取企业客户所需的时间。在完成 Fishtown 种子轮融资后的两个月内，公司就签下了两家财富 500 强公司。但这种产品驱动的势头无法长期维持企业在内部实现同样的增长曲线。"我们最初大约 1000 万美元的 ARR 是以产品驱动增长为导向的，所以感觉这种情况会持续下去。但在数据领域，这几乎从未成真，"他说。

"当你有足够的产品市场契合度时，有时你可以不必在产品和销售策略上做到极致。所以大约在 2022 年，我们从这种巨大的加速增长曲线中突然意识到，我们必须坐到'成人桌'旁，快速把事情想清楚，"Handy 说。

在产品驱动增长的火花开始减弱后，Handy 将注意力转向为云平台增加销售驱动的策略。"我们必须集中精力向资深数据领导者讲述连贯的故事。对于'为什么我应该使用商业产品而不是开源产品？'这个问题，我们必须有一个非常清晰、可解释的答案。而且这个答案必须能让那些头衔里带'C'的人理解，"他说。

Handy 的答案是：dbt Cloud 能够为各种规模的公司处理复杂数据。

"人们使用 dbt 的时间越长，他们的代码就越复杂，"他说。"这对于最资深的 dbt 用户来说是个问题，而这些用户通常来自大公司。因此，我们确实有机会通过 dbt Cloud 介入并为他们解决这个问题。"

为了向企业客户讲述这个故事，Handy 很自然地依赖于数据。"在一次用户大会上，我们展示了一张图表，显示了包含特定数量模型（超过 100 个、超过 1000 个等）的 dbt 项目数量，"他说。"我们看到这个数字不断攀升，作为用户我们自己也知道，'天哪，在一个包含 5000 个模型的 dbt 项目中工作是多么具有挑战性。' 因此，我们从这一量化数据点出发，询问我们社区中的人们对于这些庞大、复杂的 dbt 项目的体验，并验证了如果没有云平台，这确实是一个令人头疼的问题。"

## 延迟融资的隐性好处

Handy 在筹集任何风险资金之前，将 Fishtown Analytics 作为一家自举的咨询公司运营了近四年。他现在很庆幸自己选择了这条路。

"如果你走咨询路线，你会获得两种超能力，当然也要付出很多代价，"Handy 说。"我并不是说这是唯一正确的答案，或者适合所有人。"

在 Handy 看来，以咨询公司起步的优点如下：

- **你可以紧贴技术脉搏。** "直到我们筹集到风险资金的那一天，我每天都在专注于客户工作，"Handy 说。"我深入到了我们产品的细节及其所解决的问题中。我每天都像用户一样编写代码。"
- **你可以按照自己的时间表成长。** "要实现风险投资规模的成果，你需要一条指数增长曲线。而有时你无法凭空变出这样的曲线。"
