## 链接到标题 实时语音 Agent

实时功能让您的应用能够与用户进行对话。当用户说话时，模型会立即响应。由于模型是即时回复，而非等待完整的一轮对话，用户可以像与人交谈一样打断和覆盖模型的发言。它适用于语音助手、客户支持 Agent、免提工具，以及任何用户更愿意说话而非打字的场景。

它与将多个模型串联使用的区别在于，单个实时模型直接接收音频并直接生成音频，而非运行语音转文本、语言模型、文本转语音的流水线。

在浏览器中，`useRealtime` hook 管理 WebSocket 连接、麦克风捕获和音频播放。

连接通过您的 AI Gateway 凭证进行身份验证，因此您在服务器上生成一个短期令牌，仅将该令牌交给浏览器。您的 API 密钥永远不会到达客户端。添加一个生成令牌的路由：

```
1import { gateway } from '@ai-sdk/gateway';2
3export async function POST() {4  const { token, url } = await gateway.experimental_realtime.getToken({5    model: 'openai/gpt-realtime-2',6  });7  return Response.json({ token, url, tools: [] });8}
```

然后从客户端组件连接：

```
1'use client';2import { experimental_useRealtime as useRealtime } from '@ai-sdk/react';3import { gateway } from '@ai-sdk/gateway';4import { useMemo } from 'react';5
6export default function Page() {7  const model = useMemo(8    () => gateway.experimental_realtime('openai/gpt-realtime-2'),9    [],10  );11  const { status, connect, startAudioCapture } = useRealtime({12    model,13    api: { token: '/api/realtime/token' },14    sessionConfig: { voice: 'alloy', turnDetection: { type: 'server-vad' } },15  });16  17}
```

该 hook 捕获麦克风，通过 AI Gateway 将音频流式传输到模型，并播放语音回复。在浏览器之外，您可以通过 `getWebSocketConfig`、`serializeClientEvent` 和 `parseServerEvent` 自行通过 WebSocket 驱动会话。请参阅实时参考了解该路径。

### 链接到标题 实时会话内部

实时会话的工作方式与普通模型调用不同：

- **话轮转换与打断**：`turnDetection: { type: 'server-vad' }` 让服务器决定用户何时停止说话，并允许用户通过说话来打断模型、缩短回复（即“插入”），无需客户端静音计时器。
- **会话中的工具调用**：模型在回复过程中发出工具调用，您执行该调用并将结果作为客户端事件返回，模型会将答案融入后续回复中，而不会结束当前话轮。

话轮转换与打断：`turnDetection: { type: 'server-vad' }` 让服务器决定用户何时停止说话，并允许用户通过说话来打断模型、缩短回复（即“插入”），无需客户端静音计时器。

## 链接到标题 文本转语音

使用 `generateSpeech` 从文本生成语音音频。传入语音和输出格式，然后将结果写入文件：

```
1import { generateSpeech } from 'ai';2import { writeFile } from 'node:fs/promises';3
4const result = await generateSpeech({5  model: 'xai/grok-tts',6  text: 'Thanks for trying out AI Gateway.',7  voice: 'eve',8  outputFormat: 'mp3',9});10await writeFile('speech.mp3', result.audio.uint8Array);
```

## 链接到标题 语音转文本（转录）

使用 `transcribe` 将录音转录为文本。音频可以是缓冲区、base64 字符串或 URL：

```
1import { transcribe } from 'ai';2import { readFile } from 'node:fs/promises';3
4const result = await transcribe({5  model: 'openai/whisper-1',6  audio: await readFile('audio.mp3'),7});8console.log(result.text);
```

语音和转录是互补的，因此可以组合使用。您可以用一个模型生成音频，再用另一个模型将其读回，这是快速检查音频流水线两端的一种方式。
