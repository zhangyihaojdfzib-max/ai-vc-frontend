Vercel Sandbox 现已支持通过文件系统快照来捕获您的状态。您可以将 Sandbox 的完整文件系统状态捕获为快照，并使用 Sandbox API 从该快照启动新的 Sandbox。

这消除了在处理诸如依赖项安装、构建或夹具创建等耗时操作时的重复设置。只需创建一次环境，为其创建快照，即可在多个独立的运行中重复使用该精确的文件系统状态。

### 快照工作原理

快照会捕获正在运行的 Sandbox 的整个文件系统。新的 Sandbox 可以从该快照启动，从而立即访问预安装的依赖项和已配置的环境。

### 核心功能

- 使用 `sandbox.snapshot()` 从任何正在运行的 Sandbox 创建快照
- 通过 `source: { type: 'snapshot', snapshotId }` 从快照启动新的 Sandbox
- 将同一快照用于多个 Sandbox，以进行并行测试和实验
