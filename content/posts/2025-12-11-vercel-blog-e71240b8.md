### 摘要
在 React 服务器组件中发现了两个额外的漏洞：一个高严重性的拒绝服务漏洞（CVE-2025-55184）和一个中严重性的源代码暴露漏洞（CVE-2025-55183）。这些问题是安全研究人员检查原始 React2Shell 漏洞补丁时发现的。最初的修复并不完整，未能完全防止针对所有负载类型的拒绝服务攻击，从而导致了 CVE-2025-67779。
重要的是，这些新问题均不允许远程代码执行。
我们创建了新规则来解决这些漏洞，并将其部署到 Vercel WAF，以自动免费保护所有托管在 Vercel 上的项目。但是，不要依赖 WAF 提供完全保护。必须立即升级到已打补丁的版本。

### 影响

#### 拒绝服务（CVE-2025-55184）
可以构造并发送一个恶意的 HTTP 请求到任何 App Router 端点，该请求在反序列化时可能导致服务器进程挂起并消耗 CPU。

#### 源代码暴露（CVE-2025-55183）
可以构造并发送一个恶意的 HTTP 请求到任何 App Router 端点，该端点可能返回服务器操作的编译后源代码。这可能会泄露业务逻辑，但不会暴露机密信息，除非这些机密信息被直接硬编码在服务器操作的代码中。

这些漏洞存在于以下包的以下版本中：
`react-server-dom-parcel`
`react-server-dom-webpack`
`react-server-dom-turbopack`
版本：19.0.0、19.0.1、19.1.0、19.1.1、19.1.2、19.2.0 和 19.2.1。

这些包包含在以下框架和打包工具中：
*   Next.js：13.x、14.x、15.x 和 16.x。
*   嵌入或依赖 React 服务器组件实现的其他框架和插件（例如，Vite、Parcel、React Router、RedwoodSDK、Waku）。

### 解决方案
在创建了缓解措施以解决这些漏洞后，我们将其部署到我们全球分布的平台上以保护我们的客户。我们仍然建议升级到最新的已打补丁版本。
React 和受影响的下游框架的更新版本包含了防止这些问题的修复程序。所有用户应尽快升级到已打补丁的版本。

### 修复版本
*   React：19.0.2、19.1.3、19.2.2。
*   Next.js：14.2.35、15.0.7、15.1.11、15.2.8、15.3.8、15.4.10、15.5.9、15.6.0-canary.60、16.0.10、16.1.0-canary.19。

使用上述包的框架和打包工具应安装其各自维护者提供的最新版本。
