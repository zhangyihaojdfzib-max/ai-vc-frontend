### 使用 GPT-5 和 Vercel AI Cloud 构建开源氛围编码平台

氛围编码平台的前端使用 Next.js 和 AI SDK 构建。设置只需几行代码。

当用户提交消息时，会触发 `sendMessage` API。提示词和选定的模型（默认为 GPT-5）都会被发送到后端。部署到 Vercel 后，框架定义的基础设施会确定合适的资源，通过 Vercel CDN 快速提供前端服务，并使用 Vercel Functions 处理 API 调用。

后端是一个运行在 Fluid compute 上的简单函数，针对基于提示词的工作负载进行了优化。由于 AI 调用通常涉及模型推理时的空闲时间，Fluid compute 可以重新分配这些计算周期来服务其他请求，或避免对未使用的 CPU 收费。

```
1const result = streamText({2  model: "openai/gpt-5",3  system: prompt,4  messages,5  stopWhen: stepCountIs(20),6  tools7});
```

API 调用由 AI SDK 驱动。它接收完整的消息历史记录，使用 Vercel BotID 验证用户身份，并转发请求。将模型指定为字符串会自动使用 AI Gateway，从而无需管理单独的 API 密钥。

当响应开始从模型流式返回时，代码会在一个新配置的沙盒内执行。此环境是无状态的、隔离的，并在短暂超时后过期。它无法访问您的项目或数据，因此可以安全地运行任意代码。

沙盒将实时更新流式传输回前端。当命令在后端运行时，前端会即时反映其进度：

```
1{2  type: "data-run-command",3  data: {4    status: "done",5    sandboxId: "sbx_123",6    command: "npm install",7    commandId: "cmd_abc",8  }9}
```

这种方法使 UI 保持响应性和透明度，而无需等待整个任务链完成。

最后，对于潜在的高价值 API 调用，重要的是要通过 BotID 提供的复杂机器人防护和 Vercel Firewall 的速率限制来减轻滥用。
