基于 Node.js 的 Vercel Functions 现在可以检测到请求何时被取消，并在完成前停止执行。此功能可按路径进行配置，并涵盖诸如离开页面、关闭标签页或在 AI 聊天中点击停止等操作，以提前终止计算处理。

这减少了不必要的计算、Token 生成以及向用户发送其永远不会看到的数据。

要启用取消功能，请在您的 `vercel.json` 配置中添加 `"supportsCancellation": true`。您可以将其应用于特定路径或所有函数：

```
1{2  "functions": {3    "api/*": {4      "supportsCancellation": true5    }6  }7}
```

启用后，您可以使用 `Request.signal.aborted` 或 `abort` 事件来监听取消：

```
1export const GET = async (req: Request) => {2  const abortController = new AbortController();3
4  req.signal.addEventListener("abort", () => {5    console.log("request aborted");6    abortController.abort();7  });8
9  const res = await fetch("https://my-backend-service.example.com", {10    headers: {11      Authorization: `Bearer ${process.env.AUTH_TOKEN}`,12    },13    signal: abortController.signal,14  });15
16  return new Response(res.body, {17    status: res.status,18    headers: res.headers,19  });20};
```

如果您正在使用 AI SDK，请将 `abortSignal` 转发给您的流：

```
1import { openai } from '@ai-sdk/openai';2import { streamText } from 'ai';3
4export async function POST(req: Request) {5  const { prompt } = await req.json();6  const result = streamText({7    model: openai('gpt-4-turbo'),8    prompt,9    10    abortSignal: req.signal,11  });12  return result.toTextStreamResponse();13}
```
