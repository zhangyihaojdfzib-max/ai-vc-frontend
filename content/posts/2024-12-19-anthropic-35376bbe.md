# 扩展法律保护与API改进

我们正在推出全新简化的商业服务条款，其中包含扩展的版权赔偿保障，同时通过测试版Messages API提供更优的开发者体验。客户在使用Claude进行开发时将享有更强的保护与安心保障，以及更简化易用的API。

## 改进的服务条款

我们的商业服务条款（原服务协议）将使客户能够保留通过使用我们服务生成的任何输出的所有权，并保护其免受版权侵权索赔。根据更新后的条款，我们将为客户因授权使用我们的服务或其输出而面临的任何版权侵权索赔进行辩护，并支付由此产生的任何经批准的解决方案或判决费用。这些新条款将于2024年1月1日对Claude API客户生效，2024年1月2日对通过Amazon Bedrock使用Claude的客户生效。更多详情，请查阅我们更新的[商业服务条款](https://www.anthropic.com/legal/commercial-terms)，或我们的[Anthropic on Amazon Bedrock - 商业服务条款](https://www.anthropic.com/legal/bedrock-commercial-terms)。

## Messages API测试版

在使用现有API格式化提示词时很容易出现细微错误——尤其是当提示词由用户输入动态混合构建时。新的[Messages API](https://docs.anthropic.com/claude/reference/messages_post)将帮助您在开发早期发现错误，特别是在提示词构建方面，从而使您能够从Anthropic模型获得最佳输出。

改进前请求示例：

```
// POST https://api.anthropic.com/v1/complete
{
  "model": "claude-2.1",
  "max_tokens_to_sample": 1024,
  "prompt": "\n\nHuman: Hello, world\n\nAssistant: Hi, I'm Claude!\n\nHuman: Can you create a template for a quarterly executive brief?\n\nAssistant:"
}
```

改进后：

```
// POST https://api.anthropic.com/v1/messages
{
  "model": "claude-2.1",
  "max_tokens": 1024,
  "messages": [
    { "role": "user", "content": "Hello, world" },
    { "role": "assistant", "content": "Hi, I'm Claude!" },
    { "role": "user", "content": "Can you create a template for a quarterly executive brief?" }
  ]
}
```

我们计划通过更丰富、结构化的API推出许多新功能。此测试版功能是我们提供强大函数调用等服务的第一步，这些功能将很快加入Messages API。除了这些更新，我们还计划在未来几周内扩大Claude API的访问范围，以便开发者和企业能够使用我们可信赖的AI解决方案进行构建。
