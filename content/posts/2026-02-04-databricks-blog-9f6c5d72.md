众所周知，药物研发过程缓慢且成本高昂。平均研发周期长达10-15年，且有大量候选药物在临床试验阶段失败。一个主要的瓶颈在于研发早期难以准确识别合适的蛋白质靶点。

蛋白质是生物体的"功能分子"——它们催化反应、运输分子，并且是大多数现代药物的作用靶点。若能快速对蛋白质进行分类、理解其特性并识别出研究不足的候选靶点，将能极大加速药物发现进程（例如 Wozniak 等人，2024年，《自然化学生物学》）。

这正是数据工程、机器学习与生成式人工智能融合产生变革性影响之处。事实上，您可以在单一平台——Databricks Data Intelligence Platform 上构建整个流程。

## 我们构建的内容

我们的**AI驱动的药物发现解决方案加速器**通过四个关键流程展示了端到端的工作流：

1.  **数据摄取与处理**：从 UniProt 摄取并处理超过 50 万个蛋白质序列。
2.  **AI驱动的分类**：使用 Transformer 模型将这些蛋白质分类为水溶性或膜转运蛋白。
3.  **洞察生成**：利用 LLM 生成的研究洞察来丰富蛋白质数据。
4.  **自然语言探索**：所有经过处理和丰富的数据，均可通过支持自然语言查询的 AI 赋能仪表板和环境进行访问。

### 阶段 1：使用 Lakeflow 声明式管道进行数据工程

原始生物数据很少以干净、可直接分析的格式提供。我们的源数据是 **FASTA** 文件——一种表示蛋白质序列的标准格式，看起来像这样：

对于未经训练的眼睛来说，这些序列数据几乎无法解读——它是一串密集的单字母氨基酸代码。然而，在这个流程结束时，研究人员可以用自然语言查询这些相同的数据，提出诸如"**向我展示在人类中研究不足、具有高分类置信度的膜蛋白**"之类的问题，并获得可操作的洞察作为回应。

使用 Lakeflow 声明式管道，我们构建了一个**奖牌架构**，逐步精炼这些数据：

*   **青铜层**：使用 BioPython 原始摄取 FASTA 文件，提取 ID 和序列。
*   **白银层**：解析和结构化——我们使用正则表达式转换提取蛋白质名称、生物体信息、基因名称和其他元数据。
*   **黄金/丰富层**：经过整理、可供分析的数据，并丰富了分子量等衍生指标——已准备好用于仪表板、ML 模型和下游研究。这是分析师和科学家直接查询的可信层。

成果：Unity Catalog 中干净、受治理的蛋白质数据，已准备好用于下游 ML 和分析。至关重要的是，从此阶段延伸到其他阶段的数据血缘关系（如下文高亮所示）为科学可重复性提供了巨大价值。

### 阶段 2：使用 Transformer 模型进行蛋白质分类

在药物发现领域，并非所有蛋白质都同等重要。膜转运蛋白——那些嵌入细胞膜的蛋白质——是特别重要的药物靶点，因为它们控制着物质的细胞进出。

我们利用 **ProtBERT-BFD**，这是一个来自 Rostlab 的基于 BERT 的蛋白质语言模型，专门针对膜蛋白分类进行了微调。该模型将氨基酸序列视为语言，学习残基之间的上下文关系以预测蛋白质功能。

模型输出分类结果（膜蛋白或可溶性蛋白）以及置信度分数，我们将其写回 Unity Catalog，用于下游过滤和分析。

### 阶段 3：使用 GenAI 丰富数据

分类告诉我们蛋白质**是什么**。但研究人员需要知道它**为何重要**——最新的研究是什么？存在哪些空白？这是一个未被充分探索的药物靶点吗？

这就是我们引入 LLM 的地方。利用 Databricks 的基础模型 API 以及外部模型端点，我们创建了已注册的 AI 函数，用研究背景来丰富蛋白质记录。

### 阶段 4：自然语言探索

我们在启用了 Genie Space 的 AI/BI 仪表板中将所有内容整合在一起。

研究人员现在可以：

*   **按生物体、分类分数和蛋白质类型过滤**蛋白质
*   **探索分子量和分类置信度的分布**
*   用自然语言提问：**"向我展示在人类中研究不足、用于癌症治疗的高置信度膜蛋白"**

![图2. 已启用的Genie Space截图，显示自然语言查询输入以及作为结果呈现的过滤后蛋白质数据表格。](/images/posts/ceb5827a076e.png)

该仪表板查询 Unity Catalog 中相同的受治理表，AI 函数提供按需（或批处理）的丰富信息。
