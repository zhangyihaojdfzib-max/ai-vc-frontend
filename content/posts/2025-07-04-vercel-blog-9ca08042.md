您现在可以在 Vercel Sandbox 中使用 `sudo` 运行命令，从而获得对运行时环境设置的完全控制，就像在传统的 Linux 系统上一样。

这使得在执行代码之前，能够在运行时安装系统依赖项，例如 Go、Python 包或自定义二进制文件。

`sud`o 可通过 `runCommand` 方法使用：

```
1import { Sandbox } from "@vercel/sandbox";2
3const sandbox = await Sandbox.create();4await sandbox.runCommand({5  cmd: "dnf",6  args: ["install", "-y", "golang"],7  sudo: true,8});
```

沙盒的 `sudo` 配置设计得易于使用：

- `PATH` 环境变量得以保留
- `HOME` 环境变量被设置为 /root
- 自定义环境变量（如 `env: { FOO: "bar" }`）会被传递

借助 Sandbox 中的 `sudo`，可以更轻松地在具有适当权限的隔离环境中运行不受信任的代码，而无需任何变通方法。
