## 链接到标题为性能、持久性和效率而构建

## 链接到标题开箱即用的缓存

Vercel Blob 与 Vercel 缓存基础设施的深度集成，抽象了管理和配置，将数据传输作为标准请求生命周期的一部分进行处理。

当对 Blob 资产发起请求时：

1.  请求被路由到最近的 Vercel 区域，就像任何标准的 Vercel Edge 请求一样
2.  该区域将其识别为 Blob 请求并查询 Vercel 缓存
    - 缓存命中时，直接从缓存提供
    - 缓存未命中时，使用快速源传输从 Blob 存储获取资产并存储在缓存中
3.  响应返回给用户，产生 Blob 数据传输

## 链接到标题Blob 如何融入你的应用

Vercel 提供了一个 SDK 来简化文件上传。安装 @vercel/blob 后，上传文件到 Blob 只需一次函数调用。以下示例创建了一个接受文件并返回公共 URL 的 Vercel Function：

```
1import { put } from '@vercel/blob';2
3export async function PUT(request: Request) {4  const form = await request.formData();5  const file = form.get('file') as File;6  const blob = await put(file.name, file, { access: 'public', addRandomSuffix: true });7
8  return Response.json(blob);9}
```

Vercel Blob 支持高达 5TB 的大文件上传，并内置了对多部分上传、可恢复性、重试和并发的支持，这些功能由 Vercel Blob SDK 提供。你可以从客户端跟踪进度，无需变通方法即可处理大型媒体或用户生成的内容。

上传后，文件立即可公开访问，并在 Vercel 的交付网络中进行缓存。每次上传都可以追溯到其存储位置，因为 Blob 中的每个文件都使用结构化的路径格式。
