## 1. 引言 - Impulse：面向测量数据的时间序列分析

一次汽车测试活动会产生数十万个测量记录和数百 TB 的时间序列传感器数据。这些数据以 ASAM MDF4 等二进制格式存储，传统上使用 NI DIAdem 或 MATLAB 等桌面工具进行分析。领域工程师喜欢这些工具是有充分理由的。他们可以专注于实际分析，决定比较哪些信号以及哪些条件定义了关键事件，而无需成为大数据框架和分布式计算的专家。但这些工具无法扩展，基于孤立脚本的分析难以复现，而且数据脱离了现代企业其他部分所依赖的治理体系。

Impulse 是一个基于 Python 的分析库，作为 Databricks Labs 项目发布，在 Databricks 智能平台上弥合了这一差距。其核心（图 1）提供了三个关键要素：

1. 一种声明式时间序列分析语言（TSAL），让工程师能够用自然的 Python 表达信号运算、事件条件和聚合，无需掌握 Spark 专业知识。
2. 一个可插拔的查询引擎，将 TSAL 表达式编译为分布式 Spark 执行计划，跨存储在任何输入数据布局中的数千个记录执行查询。
3. 领域感知的抽象，直接映射到工程师思考数据的方式，包括测量容器、传感器通道、运行事件以及持续时间和距离加权的聚合。

在这篇博文中，我们展示了 Impulse 如何在 Databricks 上驱动 AVL 的测量数据湖仓。AVL 是一家世界领先的移动出行技术公司，专门从事车辆和能源系统的开发、仿真和测试。他们利用测量和仿真数据来验证设计、理解系统行为，并加速从虚拟模型到实际测试的数据驱动产品开发。我们将逐步介绍湖仓架构、三种互补的使用模式（服务于领域工程师、数据工程师和数据科学家），以及 AVL 在生产中看到的影响。Impulse 建立在与梅赛德斯-奔驰共同开发的、在我们之前的博文中描述的分层 Silver 层数据模型之上。

## 2. 架构 - 面向测量数据的湖仓

AVL 的平台遵循奖章架构，Unity Catalog 提供跨所有层的治理，Databricks Workflows 编排管道（见图 2）。

1. 源和摄取：原始测量文件（例如 ASAM MDF4 格式）使用 Databricks Solution Accelerator 摄取到 Bronze 层。AVL 扩展了该加速器，使其与 AVL Concerto（他们的测量数据管理系统，支持多种专有文件格式）配合使用。上下文元数据（车辆 ID、软件版本、项目标签等）与记录文件一起被摄取。

2. Silver 层：Bronze 数据被转换为面向测量数据的分层数据模型。该模型围绕容器（即单个文件）和通道（传感器信号）组织数据，每个都丰富了容器级和通道级的属性/标签和指标。Silver 层存储经过验证和质量保证的数据，为分析处理做好准备。数据质量保证规则使用 Databricks DQX 框架实现，并且完全可配置和可定制，以满足特定的下游分析需求。有关 Silver 层数据模型的更多详细信息，请参阅我们之前发布的博文。

3. + 4. 从 Silver 到 Gold：Silver 层为 Impulse 提供数据，Impulse 将声明式分析逻辑转换为分布式 Spark 执行。输出可以是用于报告的 Gold 层星型模式、用于探索的临时 DataFrame，或用于 ML 的特征矩阵（参见第 5 节）。

5. 服务和分析：Databricks Dashboards 或 Lakehouse Apps 等 BI 工具通过 SQL Warehouses 消费 Gold 层数据，实现无需接触计算管道的交互式探索。

## 4. 使用 Impulse 的三种方式——报告、临时分析和机器学习

Impulse 支持三种互补的使用模式（图 3），它们都基于相同的 TSAL 表达式语言和查询引擎构建。在结构化报告模式下，领域工程师定义事件和聚合，这些事件和聚合在所有匹配的记录上并行执行，并持久化到 Gold 层星型模式中，可直接用于 AI/BI 仪表板或 Lakehouse 应用。该管道可安排为 Databricks 工作流，以便在新测量数据到达时自动更新。在临时查询模式下，TSAL 表达式由查询引擎直接评估，并作为 Spark DataFrame 返回，用于笔记本中的交互式探索，而无需写入 Gold 层。在机器学习模式下，事件范围内的统计数据和直方图分布被提取为扁平特征矩阵，可直接传递给 MLflow、AutoML 或自定义训练管道。

AVL 如何在实际中使用 Impulse

在实际应用中，AVL 充分利用了 Impulse 框架的优势，主要使用其结构化报告模式来构建可配置、标准化的分析包（“工具箱”）。这些工具箱由领域工程师根据其具体的工程任务或分析重点，在传入的测量活动上执行。

生成的 Gold 层输出无缝集成到 Databricks 仪表板和 Lakehouse 应用中，工程师可以在其中交互式地探索结果，并创建直方图、热力图和其他统计可视化，以支持数据驱动的工程决策。

## 5. 成果与影响

借助 Impulse 框架和 Databricks 数据智能平台，AVL 构建了一个端到端的工程数据平台，以支持数据驱动的产品开发。该平台为汽车数据分析引入了新标准，并在多个方面带来了改进：

定量改进

- 显著缩短分析时间（与传统方法相比，从数天缩短至数分钟）
- 能够单次处理大量测量记录
- 与本地部署方案相比，节省基础设施成本

定性改进

- 通过自助分析赋能领域工程师
- 实现完全可复现且透明的分析
- 跨团队在单一、统一的数据平台上实现标准化
