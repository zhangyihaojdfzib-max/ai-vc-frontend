Vercel Sandbox 可观测性现在包含详细的资源指标，让您更深入地了解沙箱如何消耗计算和网络资源。

从仪表盘的“可观测性”标签页中，您现在可以监控：

- 活跃 CPU 及 CPU 使用率：衡量您的代码主动使用 CPU 的情况。活跃 CPU 以核心小时为单位计量，等待 I/O（如网络请求或模型调用）所花费的时间不计费
- 预置内存：分配给沙箱的内存乘以运行时间，以 GB-小时为单位计量
- 数据传输：进出沙箱的总数据量，包括包下载和 API 调用
- 运行中的沙箱和会话：任意时刻有多少沙箱在运行，以及会话何时停止或启动

每个指标都可以按沙箱名称和沙箱会话 ID 进行分组，因此您可以从总体使用情况向下钻取到负责的单个沙箱。

您还可以通过 Vercel CLI 查询和可视化指标：

```
vercel metrics schema vercel.sandbox
vercel metrics vercel.sandbox.cpu_usage --all
```

指标在团队和项目级别均可用，并且与沙箱使用情况的计费方式直接对齐，因此您可以将成本归因于特定工作负载，并及早发现意外使用情况。这对于跟踪大规模创建沙箱的 Agent（智能体）工作负载、根据实际利用率调整沙箱配置大小，以及识别数据传输量异常高的沙箱非常有用。

沙箱的可观测性功能包含在所有套餐中，Pro 和 Enterprise 套餐支持手动查询。了解更多信息，请参阅沙箱文档。
