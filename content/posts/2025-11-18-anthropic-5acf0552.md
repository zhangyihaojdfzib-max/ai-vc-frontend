# Claude 现已登陆 Microsoft Foundry 与 Microsoft 365 Copilot

今天我们宣布，微软与 Anthropic 正在拓展我们的合作伙伴关系。作为合作的一部分，Claude Sonnet 4.5、Haiku 4.5 和 Opus 4.1 模型现已在 Microsoft Foundry 中提供公开预览，Azure 客户可以在此构建生产级应用程序和企业级智能体。

这使得企业能够利用 Claude——全球在编码、智能体和办公任务方面表现最佳的模型——进行构建，同时继续使用他们现有的微软生态系统。开发者还可以在 Microsoft Foundry 中，通过我们的 AI 编码智能体 Claude Code 来使用 Claude 模型。

除了我们与 **Microsoft 365 Copilot** 现有的集成——Claude 在其中为复杂、多步骤研究的 Researcher 智能体提供支持，并支持在 Copilot Studio 中进行自定义智能体开发——之外，微软 **Excel 中的 Agent Mode** 现在也包含一个预览选项，可以直接在 Excel 中使用 Claude 来构建和编辑电子表格。您现在可以使用 Claude 来生成公式、分析数据、识别错误，并在 Excel 电子表格中迭代解决方案。

对于已经投资于 Microsoft Foundry 和 Microsoft 365 Copilot 的企业来说，采用新的人工智能能力通常意味着需要处理独立的供应商合同和计费系统——这会增加数周或数月的采购开销。这些集成消除了这些障碍。

## 在 Microsoft Foundry 中使用 Claude 进行构建

Claude 通过无服务器部署在 Microsoft Foundry 中可用，允许开发者在 Anthropic 管理基础设施的同时进行扩展。此集成使开发者能够：

- **立即开始构建**：通过 Foundry 的 API、工具和工作流部署 Claude
- **使用您现有的 Azure 协议**：Claude 符合 Microsoft Azure Consumption Commitment (MACC) 资格，并与当前的 Azure 协议和计费系统协同工作，无需单独的供应商审批
- **使用您偏好的语言构建**：使用 Python、TypeScript 和 C# SDK，配合 Microsoft Entra 身份验证来访问 Claude

Claude 在今日开始推出的 Global Standard 部署中可用，采用我们的标准 API 定价，US DataZone 即将推出。详情请访问我们的**定价页面**。

## 为您的用例选择合适的模型

微软客户可以直接在 Foundry 中访问 Claude 的前沿模型。

**Sonnet 4.5** 是全球最佳的编码模型，也是构建复杂智能体的最强模型。当您需要在复杂的推理、多步骤智能体工作流和自主编码任务上获得最先进的性能时，请使用 Sonnet 4.5。

**Haiku 4.5** 是我们最快的模型，以 Sonnet 三分之一的价格提供接近前沿的性能。在速度和成本效益至关重要的场景中部署 Haiku 4.5，例如子智能体、客户支持自动化、内容审核或实时编码辅助。

**Opus 4.1** 是用于专业推理任务的卓越模型。对于需要持续专注和严格关注细节的复杂、多步骤问题，请使用 Opus 4.1。

所有模型在 Foundry 中都支持多种 Claude 开发者平台功能，包括**代码执行工具**、**网络搜索和获取**、**引用**、**视觉**、**工具使用**、**提示词缓存**等。请探索我们的**文档**以了解其他支持的功能。

## 开始使用

Claude 现已通过 Microsoft Foundry 提供公开预览。请访问 **Microsoft Foundry 目录** 来部署 Claude Sonnet 4.5、Claude Haiku 4.5 或 Claude Opus 4.1，或探索我们的**文档**以了解更多信息。
