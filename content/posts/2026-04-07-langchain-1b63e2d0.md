今天，我们宣布与 Arcade.dev 建立新的合作伙伴关系，将其工具库引入 LangSmith Fleet。Arcade 是面向生产环境 Agent（智能体）的 MCP 运行时，提供安全的 Agent 授权、可靠的工具和治理能力。通过此项集成，您的 Agent 可以通过一个安全网关访问 Arcade 包含 7,500 多个专为 Agent 优化的工具集合。

[试用 Fleet] | [试用 Arcade]

LangSmith Fleet 使每个团队都能为日常工作创建、使用和共享 Agent。Fleet 中的 Agent 可以自主跨多个工具工作，例如从 Salesforce 提取数据、更新 Notion 中的页面以及在 Slack 中分享结果。但这意味着 Agent 需要可靠地访问团队所依赖的每一个工具。Arcade 的 MCP 网关通过一个端点，为 Agent 提供了连接到所有这些工具的安全通道。

### 集中管理所有工具的网关

网关是一种简化 Agent 连接到外部服务的有用模式。LLM（大语言模型）网关集中管理您对模型提供商的访问和凭证。同样的逻辑也适用于工具，管理单个连接的成本甚至更高。每个新工具都意味着其自身的身份验证流程、其独特的 API 特性以及其自身的持续维护。将这种情况乘以团队使用的所有工具，集成成本会迅速累积。

Arcade 的 MCP 网关为您的 Agent 提供了一个单一的访问点。在 Fleet 中连接您的 Arcade 账户，选择您的网关，您的 Agent 就能在几分钟内访问 Salesforce、Asana、Zendesk 以及其他数十个应用程序。

您可以为整个组织创建一个统一的网关，也可以为每个团队或用例创建定制的网关。用户使用自己的凭证进行连接，并获得与其工作相关的工具的访问权限，而无需增加工程团队的待办事项。

### 并非又一个 API 包装器

目前有很多可用的 MCP 服务器。其中许多是将现有的 REST API 包装在 MCP 协议中。这提供了标准化的工具发现功能，这很有用，但它并没有改变工具底层实际的工作方式。

当由 Agent 来调用这些工具时，这种区别就很重要。API 的设计假设是由人类程序员决定调用哪个端点以及如何构建请求。它们暴露了包含许多端点和参数组合的庞大接口。它们的模式描述的是数据结构，而非意图。它们期望结构化的输入，并在出错时返回原始的 HTTP 错误。一个基于自然语言上下文工作的 Agent 必须处理所有这些。当 Agent 出错时，您可能会得到幻觉参数、糟糕的工具选择，或者浪费 Token 在不相关的端点间循环。

Arcade 提供专为 Agent 设计的 MCP 工具。Arcade 工具的范围被缩小到 Agent 实际需要执行的操作，而非完整的 API 接口。每个工具都遵循一致的结构模式，并且工具描述是为语言模型如何选择和调用工具而编写的。更好的描述意味着更好的工具选择。

### 安全的工具身份验证与授权

LangSmith Fleet 和 Arcade 协同工作，为您的 Agent 管理工具的身份验证和授权。Arcade 处理基于每个用户、会话范围的授权。每个操作在运行时都强制执行最小权限原则，继承 Agent 所代理的特定用户的权限。这使得 Agent 工具能够在不同人员对不同系统拥有不同访问级别的环境中工作。

Fleet 是您配置凭证如何流入 Arcade 的地方。配置为"助手"的 Agent 在进行工具调用时会传递每个用户自己的凭证，因此操作会反映该用户在下游系统中的权限。配置为"爪牙"的 Agent 使用在所有用户间共享的一组固定凭证，这适用于 Agent 代表团队或服务而非个人行事的情况。
