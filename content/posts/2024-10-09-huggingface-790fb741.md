# 使用 Hugging Face + Dask 扩展基于 AI 的数据处理

Hugging Face 平台拥有众多数据集和预训练模型，使得使用和训练最先进的机器学习模型变得越来越便捷。然而，扩展 AI 任务可能很困难，因为 AI 数据集通常很大（数百 GB 到 TB 级别），并且使用 Hugging Face Transformer 进行模型推理有时计算成本很高。

Dask 是一个用于分布式计算的 Python 库，它通过将数据集分解为可管理的块来处理内存外计算（处理无法放入内存的数据）。这使得以下操作变得容易：

- 使用类似 pandas 的易用 API 高效加载和预处理 TB 级数据集
- 并行模型推理（可选择多节点 GPU 推理）

在本文中，我们展示了一个来自 FineWeb 数据集的数据处理示例，使用 FineWeb-Edu 分类器来识别具有高教育价值的网页。我们将展示：

- 如何使用 pandas 在本地处理 100 行数据
- 如何使用 Dask 在云端跨多个 GPU 扩展到 2.11 亿行数据

## 使用 Pandas 处理 100 行数据

FineWeb 数据集包含来自 Common Crawl 的 15 万亿 Token 英文网络数据，Common Crawl 是一个非营利组织，托管每月更新的公共网络爬取数据集。该数据集通常用于各种任务，如大语言模型训练、分类、内容过滤以及跨多个领域的信息检索。

在笔记本电脑上使用 pandas 下载和读取单个文件可能需要超过 1 分钟。

```python
import pandas as pd

df = pd.read_parquet(
    "hf://datasets/HuggingFaceFW/fineweb/data/CC-MAIN-2024-10/000_00000.parquet"
)
```

接下来，我们将使用 HFFineWeb-Edu 分类器来判断数据集中网页的教育价值。网页按 0 到 5 的等级进行评分，0 表示没有教育价值，5 表示教育价值很高。我们可以使用 pandas 对数据的 100 行子集执行此操作，在配备 GPU 的 M1 Mac 上大约需要 10 秒。

```python
from transformers import pipeline

def compute_scores(texts):
    import torch

    
    if torch.cuda.is_available():
        device = torch.device("cuda")
    elif torch.backends.mps.is_available():
        device = torch.device("mps")
    else:
        device = torch.device("cpu")

    pipe = pipeline(
        "text-classification",
        model="HuggingFaceFW/fineweb-edu-classifier",
        device=device
    )
    results = pipe(
        texts.to_list(),
        batch_size=25,                    
        padding="longest",
        truncation=True,
        function_to_apply="none"
    )
    
    return pd.Series([r["score"] for r in results])

df = df[:100]
min_edu_score = 3
df["edu-classifier-score"] = compute_scores(df.text)
df = df[df["edu-classifier-score"] >= min_edu_score]
```

请注意，我们在 `compute_scores` 函数中添加了一个检查可用硬件的步骤，因为在下一步使用 Dask 扩展时它将被分布式执行。这使得从在单台机器上本地测试（无论是在 CPU 上，还是在配备 Apple 芯片 GPU 的 MacBook 上）到跨多台机器（如 NVIDIA GPU）进行分布式部署变得容易。

## 使用 Dask 扩展到 2.11 亿行数据

整个 2024 年 2 月/3 月的爬取数据在磁盘上占用 432 GB，在内存中约为 715 GB，分布在 250 个 Parquet 文件中。即使在内存足够容纳整个数据集的机器上，串行处理也会慢得令人望而却步。

为了扩展，我们可以使用 Dask DataFrame，它通过并行化 pandas 来帮助处理大型表格数据。它与 pandas API 非常相似，使得从在单个数据集上测试到扩展到完整数据集变得容易。Dask 与 Parquet（Hugging Face 数据集上的默认格式）配合良好，支持丰富的数据类型、高效的列式过滤和压缩。

```python
import dask.dataframe as dd

df = dd.read_parquet(
    
    "hf://datasets/HuggingFaceFW/fineweb/data/CC-MAIN-2024-10/*.parquet" 
)
```

我们将使用 `map_partitions` 在 Dask DataFrame 上并行应用 `compute_scores` 函数进行文本分类，该函数在较大的 Dask DataFrame 中的每个 pandas DataFrame 上并行执行我们的函数。`meta` 参数是 Dask 特有的，用于指示输出的数据结构（列名和数据类型）。

```python
from transformers import pipeline

def compute_scores(texts):
    import torch

    
    if torch.cuda.is_available():
        device = torch.device("cuda")
    elif torch.backends.mps.is_available():
        device = torch.device("mps")
    else:
        device = torch.device("cpu")

    pipe = pipeline(
        "text-classification",
        model="HuggingFaceFW/fineweb-edu-classifier",
        device=device,
    )
    results = pipe(
        texts.to_list(),
        batch_size=768,
        padding="longest",
        truncation=True,
        function_to_apply="none",
    )

    return pd.Series([r["score"] for r in results])

min_edu_score = 3
df["edu-classifier-score"] = df.text.map_partitions(compute_scores, meta=pd.Series([0]))
df = df[df["edu-classifier-score"] >= min_edu_score]
```

请注意，我们选择的 `batch_size` 适用于此示例，但您可能需要根据工作流程中使用的硬件、数据和模型进行自定义（请参阅 HF 关于 pipeline 批处理的文档）。

现在我们已经确定了感兴趣的数据集行，可以保存结果以供其他下游分析使用。Dask DataFrame 自动支持分布式写入 Parquet。Hugging Face 使用提交来跟踪数据集更改，并允许并行写入 Dask DataFrame。

```python
repo_id = "<your-hf-user>/<your-dataset-name>"  
df.to_parquet(f"hf://datasets/{repo_id}")
```

由于这会导致每个文件创建一个提交，建议在上传后压缩历史记录：

```python

HfApi().super_squash_history(repo_id=repo_id, repo_type="dataset")
```

或者，您可以使用此自定义函数，该函数每次提交上传多个文件。

### 多 GPU 并行模型推理

有多种方法可以在各种硬件上部署 Dask。在这里，我们将使用 Coiled 在云端部署 Dask，这样我们可以根据需要启动虚拟机，然后在完成后清理它们。

```python
cluster = coiled.Cluster(
    region="us-east-1",                 
    n_workers=100,                      
    spot_policy="spot_with_fallback",   
    worker_vm_types="g5.xlarge",        
    worker_options={"nthreads": 1},
)
client = cluster.get_client()
```

在底层，Coiled 处理：

- 使用 GPU 硬件配置云虚拟机。在本例中，是 AWS 上的 `g5.xlarge` 实例。
- 设置适当的 NVIDIA 驱动程序、CUDA 运行时等。
- 通过包同步自动在云虚拟机上安装与本地相同的包。这包括工作目录中的 Python 文件。

该工作流程大约需要 5 小时完成，并且 GPU 硬件利用率良好。
