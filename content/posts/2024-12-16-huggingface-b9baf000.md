# 介绍合成数据生成器——用自然语言构建数据集

推出合成数据生成器，这是一款用户友好的应用程序，采用无代码方法，利用大语言模型（LLM）创建自定义数据集。最棒的是：简单的分步流程，让数据集创建变得轻松无技术门槛，任何人都可以在几分钟内无需任何代码即可创建数据集和模型。

合成数据是人工生成的、模拟真实世界数据的信息。它通过扩展或增强数据集来克服数据限制。

## 从提示词到数据集再到模型

合成数据生成器接收您所需数据的描述（您的自定义提示词），并通过合成数据管道返回适用于您用例的数据集。在后台，这由 distilabel 和免费的 Hugging Face 文本生成 API 驱动，但我们无需担心这些复杂性，可以专注于使用用户界面。

### 支持的任务

该工具目前支持文本分类和聊天数据集。这些任务将决定您生成的数据集类型：分类需要类别，而聊天数据需要对话。根据需求，我们将随时间添加评估和 RAG（检索增强生成）等任务。

#### 文本分类

文本分类通常用于对客户评论、社交媒体帖子或新闻文章等文本进行分类。生成分类数据集依赖于我们利用 LLM 处理的两个不同步骤。我们首先生成多样化的文本，然后为其添加标签。合成文本分类数据集的一个好例子是 argilla/synthetic-text-classification-news，它将合成新闻文章分类为 8 个不同的类别。

#### 聊天数据集

这种类型的数据集可用于监督微调（SFT），这是一种让 LLM 处理对话数据的技术，允许用户通过聊天界面与 LLM 交互。合成聊天数据集的一个好例子是 argilla/synthetic-sft-customer-support-single-turn，它展示了一个旨在处理客户支持的 LLM 示例。在此示例中，客户支持主题就是合成数据生成器本身。

通常，我们每分钟可以分别生成 50 个文本分类样本和 20 个聊天样本。所有这些都由免费的 Hugging Face API 驱动，但您可以通过使用自己的账户并选择自定义模型、API 提供商或生成配置来扩展规模。我们稍后会回到这一点，但让我们先了解基础知识。

### 让我们生成第一个数据集

我们将创建一个基本的聊天数据集。当您访问生成器时，您需要登录，以便工具能够访问您想要生成数据集的组织。这将允许工具上传生成的数据集。如果身份验证失败，您可以随时重置连接。

登录后，用户界面会引导您完成一个简单的三步流程：

#### 1. 描述您的数据集

首先提供您要创建的数据集的描述，包括示例用例，以帮助生成器理解您的需求。请尽可能详细地描述目标和助手类型。当您点击"创建"按钮时，将创建一个样本数据集，您可以继续执行步骤 2。

![](/images/posts/7e271d1130d1.png)

#### 2. 配置和优化

通过调整系统提示词（根据您的描述生成）以及调整任务特定设置来优化生成的样本数据集。这将帮助您获得所需的具体结果。您可以通过点击"保存"按钮并重新生成样本数据集来迭代这些配置。当您对配置满意后，继续执行步骤 3。

![](/images/posts/d1b08bef5319.png)

#### 3. 生成和推送

填写数据集名称和组织的常规信息。此外，您可以定义要生成的样本数量以及生成时使用的温度参数。该温度代表生成内容的创造性。让我们点击"生成"按钮开始完整生成。输出将直接保存到 Argilla 和 Hugging Face Hub。

![](/images/posts/31a90af0a08e.png)

现在我们可以点击"在 Argilla 中打开"按钮，直接进入我们生成的数据集。

### 审查数据集

即使处理合成数据，理解和查看您的数据也很重要，这就是为什么我们创建了与 Argilla 的直接集成，Argilla 是一个供 AI 工程师和领域专家协作构建高质量数据集的工具。这使您能够通过语义搜索和可组合过滤器等强大功能有效探索和评估合成数据集。您可以在此指南中了解更多信息。之后，我们可以将精选的数据集导出到 Hugging Face Hub，并继续使用它来微调模型。

![](/images/posts/f7ee85862401.png)

### 训练模型

不用担心；如今即使创建强大的 AI 模型也可以使用 AutoTrain 无需代码完成。要了解 AutoTrain，您可以查看其文档。在这里，我们将创建自己的 AutoTrain 部署，并像之前为合成数据生成器所做的那样登录。

还记得开头的 argilla/synthetic-text-classification-news 数据集吗？让我们训练一个能够正确分类这些示例的模型。我们需要选择"文本分类"任务并提供正确的"数据集来源"。然后，选择一个合适的项目名称并按下播放按钮！关于成本的弹出警告可以忽略，因为我们仍在免费的 Hugging Face CPU 硬件上工作，这对于这个文本分类示例来说已经足够了。

瞧，几分钟后，我们就拥有了自己的模型！剩下的就是将其部署为实时服务，或使用少量 Python 代码将其用作文本分类管道。

## 高级功能

即使您无需了解任何编码知识就能从提示词到专用模型，有些人可能仍希望使用一些更高级的技术功能来自定义和扩展其部署。

### 提高速度和准确性

您可以通过创建工具的自有部署并配置使用不同参数或模型来提高速度和准确性。首先，您需要复制合成数据生成器。请确保将其创建为私有 Space，以确保其他人无法访问。接下来，您可以更改某些环境变量的默认值。让我们来看一些场景：

1. 使用不同的免费 Hugging Face 模型。您可以通过将 MODEL 从默认值 meta-llama/Llama-3.1-8B-Instruct 更改为其他模型（如 meta-llama/Llama-3.1-70B-Instruct）来实现。
2. 使用 OpenAI 模型。您可以通过将 BASE_URL 设置为 https://api.openai.com/v1/ 并将 MODEL 设置为 gpt-4o 来实现。
3. 增加批量大小，这将每分钟生成更多样本。您可以通过将 BATCH_SIZE 从默认值 5 更改为更高的值（如 10）来实现。请注意，您的 API 提供商可能对每分钟请求数量有限制。
4. 私有 Argilla 实例。您可以通过将 ARGILLA_URL 和 ARGILLA_API_KEY 设置为您的免费 Argilla 实例的 URL 和 API 密钥来实现。

### 本地部署

除了在 Hugging Face Spaces 上托管该工具外，我们还将其作为开源工具以 Apache 2 许可证提供，这意味着您可以前往 GitHub 并根据需要自由使用、修改和适配它。您可以通过简单的 pip install synthetic-dataset-generator 将其作为 Python 包安装。创建时请确保配置正确的环境变量。

### 自定义管道

每个合成数据管道均基于 **distilabel** 构建，这是一个用于合成数据与 AI 反馈的框架。distilabel 是开源的；管道代码的亮点在于其可共享且可复现。例如，你可以在 Hub 的仓库中找到 **argilla/synthetic-text-classification-news** 数据集的管道。此外，你还可以找到许多其他 distilabel 数据集及其对应的管道。
