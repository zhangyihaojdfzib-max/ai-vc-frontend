# Reflection AI 发布 Asimov：突破性的代码理解智能体

Reflection AI 的首个产品里程碑。

想象一下，如果 Jeff Dean 能加入你的团队会怎样？卓越的技术实力、广阔的系统思维、深厚的团队内部知识、最优秀的领导者和导师，以及团队真正可靠的支柱。

今天，Reflection AI 激动地发布 Asimov：最优秀的代码理解研究智能体。可以将其理解为：用于代码理解的深度研究。它就像 Jeff Dean 在你团队中一样，拥有超群的智慧。

最具影响力的机遇不仅仅在于代码生成，更在于代码理解。工程师们花费较少的时间编写代码，而将大部分时间用于理解和设计代码。这些时间消耗在解析复杂的代码库、与同事进行白板讨论、在 Slack 和 Linear 中搜索线索、以及在 Cursor、Windsurf、ChatGPT 和 Stack Overflow 中深入探究问题。

工程团队真正的解锁点是什么？是深刻理解复杂的代码库及其背后的业务逻辑。

如果 Asimov 能够利用原本只存在于某位资深架构师脑海中的团队内部智慧，为团队中的任何工程师扫清障碍，会怎样？如果 Asimov 能够随着团队最全知、最实时的记忆而不断进化，又会怎样？

Asimov 已经是代码理解领域的顶尖智能体。在对一些最大型开源项目的维护者进行的盲测中，相对于 Cursor Ask 和 Claude Code（Sonnet 3.7 和 4），Asimov 的答案在大多数情况下更受青睐。

## 1. Asimov 为工程知识构建单一事实来源

Asimov 洞察一切。它摄取整个代码库、架构文档、GitHub 讨论、聊天记录等等。它构建关于你系统的持久记忆，记住关键决策，并充当工程组织可信赖的大脑。

![](/images/posts/80978ca0d0ae.webp)

## 2. Asimov 捕获全团队的内部知识

当其他产品通过规则或包含智能体指令的 README 文件专注于个体开发者偏好时，Asimov Memories 使工程团队能够捕获全团队的内部知识。

工程师可以更新 Asimov 的知识，例如“@asimov 记住 X 以 Y 方式工作。”这使得最资深的工程师能够将他们头脑中存储的上下文卸载给 Asimov，从而使整个团队受益。Memories 配备了基于角色的权限访问控制系统，允许组织控制谁可以编辑 Asimov 的知识。

![](/images/posts/28a23d5ead36.webp)

## 3. Asimov 设计用于摄取大量信息

Asimov 采用多智能体架构，在代码理解方面实现了最先进的性能。该架构由许多小型长上下文智能体（检索器）和一个大型短上下文推理智能体（组合器）组成。检索器从大型代码库中检索相关信息，组合器则将这些信息综合成对用户查询的连贯回答。这种设计理念与其他前沿的智能体研究架构相似，但应用于代码理解问题。Asimov 的当前版本由第三方模型驱动，但我们正在积极训练自己的模型以提升 Asimov 的性能。
