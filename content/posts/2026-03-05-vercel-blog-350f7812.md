Streamdown v2.4 为渲染 Markdown 的开发者引入了自定义钩子、无障碍功能以及用户体验改进。

团队现在可以使用多个新属性自定义其 Markdown 输出的外观。您可以通过向 `icons` 属性传递特定的组件映射来覆盖内置图标。

```
1import { Streamdown, type IconMap } from "streamdown"2
3<Streamdown4  icons={{ CheckIcon: MyCheckIcon }}5>6  {content}7</Streamdown>
```

`createCodePlugin` 现在接受用于浅色和深色 Shiki 主题的 `themes` 选项、用于自定义起始行号的 `startLine` 元选项，以及一个用于独立于代码块设置行内代码样式的 `inlineCode` 虚拟组件。

```
1import { createCodePlugin } from "streamdown"2
3const codePlugin = createCodePlugin({4  themes: {5    light: "github-light",6    dark: "github-dark",7  },8})
```

Streamdown 现在支持国际化和文本方向。`dir` 属性会根据第一个强 Unicode 字符自动应用从左到右或从右到左的格式，`translations` 属性支持自定义语言。

```
1<Streamdown2  dir="auto"3  translations={{ copy: "نسخ", copied: "تم النسخ" }}4>5  {content}6</Streamdown>
```

表格包含一个通过 `controls` 属性控制的全屏覆盖层，并配有滚动锁定和 Escape 键支持。开发者可以使用 `onAnimationStart` 和 `onAnimationEnd` 回调函数接入流式事件。

此版本修复了语法高亮块中空行折叠的问题，并防止有序列表在流式传输期间重新触发动画。

对于使用 Tailwind v4 的项目，新的 `prefix` 属性为工具类添加了命名空间，以避免冲突。

```
1<Streamdown prefix="sd-">{content}</Streamdown>
```

要开始使用，请[了解更多](https://streamdown.dev)。
