# 介绍 Precursor：通过持续客户端信号检测代理行为

机器人缓解是一场对抗游戏：攻击者不断适应，防御者持续回应，循环往复。在 Cloudflare，我们通过将全球网络的可见性与客户端环境信号相结合来保持领先。在网络层面，我们每天分析超过 1 万亿次请求，以了解占互联网 20% 以上的流量中的声誉、模式和异常。在客户端，我们通过 Cloudflare Turnstile 将检测推向了更深层次，该产品已从 CAPTCHA 替代方案演变为基于风险的管理挑战，可根据需要调整验证用户真实性的摩擦程度。

如今，Turnstile 每天在互联网上一些最敏感的端点上运行近 30 亿次，帮助在登录、注册和结账等关键时刻验证用户。这加强了对客户应用程序最重要区域的保护，但仍对应用程序的其余部分——人类和机器人在整个用户旅程中实际如何交互——留下了有限的可见性。

这就是我们今天通过推出 Precursor 来弥补的可见性差距。

### 介绍 Precursor

Precursor 是一个基于会话的客户端验证系统，以隐私为设计理念，使用动态注入的 JavaScript 在访问者与您的应用程序交互时持续收集行为信号。这些信号经过处理后实时整合到 Cloudflare 的机器人防护中，使我们能够持续区分人类流量与自动化或代理流量。

这将挑战提供的客户端检测扩展到您的整个 Web 应用程序。Precursor 是 Turnstile 的可选补充——两者都是我们的企业机器人管理功能的一部分。

这种基于用户旅程的检测之所以强大，是因为现代自动化越来越能够在短时间内表现得看似合法。机器人可以执行 JavaScript、使用真实浏览器环境，并通过单个 CAPTCHA 而不引起怀疑。但难以复制的是随时间推移的一致的人类行为。

Precursor 旨在捕捉这一交互层面，将行为本身转化为检测欺诈和滥用的可靠信号。通过评估整个会话期间的行为，Precursor 为每个决策增加了显著更多的信号。这提高了检测精度，使得无需依赖激进的挑战就能更容易地区分真实用户和自动化。对于合法用户，Precursor 意味着更少的不必要中断。对于机器人开发者，它通过要求模拟整个会话提高了运营自动化的成本。这构建起来更加困难，维护成本更高，且大规模运营的可靠性远低于前者。

### 犯错乃人类本性

当机器人开发者试图让鼠标移动看起来像人类时，他们通常会添加高斯噪声或均匀随机延迟。但人类的移动不仅仅是"有噪声"，它还受到物理规律的约束：

- 手腕转动：人类的鼠标移动通常呈弧形，受手腕活动范围和前臂旋转的限制。
- 认知负荷：从人类看到复选框到点击它之间存在可测量的延迟。
- 手部震颤：即使是最稳定的人类手部也会以生理性震颤频率振荡。

相比之下，机器人的行为往往会暴露它们自己。它们以线性插值或数学上理想的贝塞尔曲线移动。它们以人类永远无法复制的精度点击。即使它们确实设法模拟了人类错误，人类移动的节奏也只能通过检查整个会话才能看到。

鼠标移动只是 Precursor 评估的信号之一，但它清晰地说明了差异。下面是一个鼠标自动化库与网站交互的示例。您可以看到鼠标如何以完美的直线移动，总是返回原点，并以相同的速度做出反应。

现在，将其与人类浏览同一网站的情况进行对比：您会看到不规则的路径、小的修正和过冲，以及速度、时间和方向的变化。

单独来看，这些交互可能看起来合理。但在整个会话过程中，这些模式会以难以伪造的方式产生差异。Precursor 旨在捕捉和评估这些行为特征，随着它们在访问者与应用程序交互过程中逐渐形成。

### Precursor 的工作原理

为了随时间评估行为，Precursor 在客户端持续收集交互数据，并为该网站构建会话级别的活动视图。

1. 注入和收集层

当在您的应用程序上启用 Precursor 时，Cloudflare 会自动将一个轻量级脚本注入到通过我们网络的站点 HTML 响应中，无需额外配置、网络连接或第三方嵌入。注入的 Precursor 包体积小巧、经过混淆处理，并为每个响应动态组装。该包设计为不干扰托管 Web 应用程序的任何额外页面逻辑。

该脚本附加轻量级事件监听器，以捕获指针移动、键盘活动、焦点变化和可见性等交互信号。这些事件被序列化为紧凑格式并在内存中缓冲。按固定间隔，缓冲数据被发送到评估层进行分析。

2. 评估层

在边缘服务器上，传入的 Precursor 负载被反序列化为行为输入。一个调度器在输入数据上运行一系列评估器。每个评估器读取其关注的 Precursor 流，并可以向共享检测注册表发出信号。

评估器设计用于交叉引用数据。例如，它们确认指针活动与页面可见性持续时间相关，或者键盘事件仅在文本字段获得焦点时触发。然后，这些信息流被整合为单个信号，用于加权检测。

3. 会话集成

Precursor 数据是会话范围的，意味着它在整个会话期间累积。会话范围很重要，因为这意味着机器人无法通过刷新页面或重新开始新的挑战来重置其行为特征。该系统还会将会话元数据馈送到下游检测层，用于额外的影子模式启发式分析和会话分析、预测与实际完成情况以及会话违规启发式分析。这些边缘端的观察结果被记录下来，用于改进检测和调整会话的机器人评分。

4. 隐私设计

Precursor 旨在收集有助于区分人类模式与自动化和滥用模式的信号。

事件监听器捕获检测自动化和滥用所需的最少信息。例如，键盘活动以时间和节奏的形式捕获，而不是实际按下的键。此外，行为信号作为聚合模式而非单个行为进行评估，并由 Cloudflare 的机器人检测系统内部使用；它们不会暴露在客户仪表板中，也不会与用户账户、登录身份或持久配置文件关联。

综合来看，这使得 Precursor 能够保持对行为的持续演进评估，在最大化精度的同时最小化对良好用户的摩擦。
