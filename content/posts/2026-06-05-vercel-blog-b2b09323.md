Vercel Sandbox 现已在私有测试版中支持驱动器。驱动器是持久化、可附加的存储，其生命周期独立于任何沙箱。

创建一次驱动器，然后在启动沙箱时将其挂载到可配置的路径。当沙箱停止时，该驱动器仍可保留，以便后续附加到其他沙箱。

安装测试版 SDK（@vercel/sandbox@beta）或测试版 CLI（sandbox@beta），然后创建并挂载一个驱动器：

```
1import { Drive, Sandbox } from "@vercel/sandbox";2
3const drive = await Drive.getOrCreate({4  name: "agent-workspace",5});6
7const sandbox = await Sandbox.create({8  mounts: {9    "/workspace": {10      drive: drive.name,11      mode: "read-write",12    },13  },14});
```

Sandbox 驱动器适用于：

- 在一次性沙箱之间保留 Agent（智能体）工作空间
- 保留克隆的仓库、依赖项和构建输出
- 独立于沙箱生命周期管理数据

在一次性沙箱之间保留 Agent（智能体）工作空间

保留克隆的仓库、依赖项和构建输出

独立于沙箱生命周期管理数据

在私有测试版期间，一个驱动器每次只能被一个沙箱以读写模式挂载。在私有测试版阶段，Sandbox 驱动器不应用于生产数据。
