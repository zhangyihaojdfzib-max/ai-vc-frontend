AI Gateway 现已展示数百个模型的吞吐量和延迟指标，帮助您根据实时性能数据选择合适的模型。

指标在三个位置显示，每小时更新一次：

- 模型列表：每个模型的最佳性能（P50延迟和吞吐量）
- 模型详情页：提供商级别的性能细分
- REST API：滚动端点性能聚合数据（延迟和吞吐量，P50/P95）

模型列表：每个模型的最佳性能（P50延迟和吞吐量）

模型详情页：提供商级别的性能细分

REST API：滚动端点性能聚合数据（延迟和吞吐量，P50/P95）

### 模型列表

AI Gateway 的**模型列表**现已包含可排序的延迟和吞吐量列。每一行显示该模型在所有可用提供商中的最佳 P50 指标（最低延迟、最高吞吐量）。指标每小时更新，基于 AI Gateway 客户的实时请求数据。

![](/images/posts/2f157e948fdb.jpg)

按吞吐量排序可找到生成 Token 最快的模型，按延迟排序可找到首 Token 响应时间最快的模型。

### 模型详情页

在单个模型页面上，您可以查看每个有使用记录的提供商的 P50 延迟和吞吐量。这有助于您比较同一模型不同提供商的性能，并为您的用例选择最佳选项。

要访问这些页面，请点击**模型列表**中的任意模型，以获取 AI Gateway 中承载该模型的所有提供商的详细细分视图。指标每小时刷新，并且仅对拥有足够流量的提供商显示。

以下是 `openai/gpt-oss-120b` 的示例：

![](/images/posts/a96654cd28e8.jpg)

与整体模型列表类似，您可以在模型详情页上按延迟和吞吐量对提供商进行排序。

### REST API

这些指标也可以通过端点 REST API 以编程方式获取。使用时，请将 `[ai-gateway-string]` 替换为您感兴趣的模型的 `creator/model-name`。

```
1curl ai-gateway.vercel.sh/v1/models/[ai-gateway-string]/endpoints
```

这将返回指定模型按提供商划分的实时每小时 P50 和 P95 延迟（毫秒 TTFT）和吞吐量（T/s）。以下是 Cerebras 提供商针对 `zai/glm-4.7` 模型的端点输出示例。

```
1curl ai-gateway.vercel.sh/v1/models/zai/glm-4.7/endpoints
```

```
1      {2        "name": "cerebras | zai/glm-4.7",3        "latency_last_1h": {4          "p50": 456.5,5          "p95": 774.956        },7        "throughput_last_1h": {8          "p50": 354,9          "p95": 445.4510        },11      }
```

如果您想查询完整的模型列表，也可以结合使用模型指标端点与 `https://ai-gateway.vercel.sh/v1/models`。
