## 摘要

React 服务器组件中存在一个严重级别的漏洞（CVE-2025-55182），影响 React 19 及使用它的框架，包括 Next.js（CVE-2025-66478）。在某些条件下，特制的请求可能导致意外的远程代码执行。

我们创建了新规则来解决此漏洞，并快速部署到 Vercel WAF，以自动免费保护托管在 Vercel 上的所有项目。但是，不要依赖 WAF 提供完全保护。必须立即升级到已修复的版本。我们还与 React 团队合作，向最大的 WAF 和 CDN 提供商提供了建议。

无论您的托管提供商是谁，我们仍然强烈建议升级到已修复的版本。

## 影响

使用受影响版本 React 服务器组件实现的应用程序可能会以允许攻击者执行远程代码执行的方式处理不受信任的输入。该漏洞存在于以下包的 19.0、19.1.0、19.1.1 和 19.2.0 版本中：

- react-server-dom-parcel (19.0.0, 19.1.0, 19.1.1, and 19.2.0)
- react-server-dom-webpack (19.0.0, 19.1.0, 19.1.1, and 19.2.0)
- react-server-dom-turbopack (19.0.0, 19.1.0, 19.1.1, and 19.2.0)

react-server-dom-parcel (19.0.0, 19.1.0, 19.1.1, and 19.2.0)

react-server-dom-webpack (19.0.0, 19.1.0, 19.1.1, and 19.2.0)

react-server-dom-turbopack (19.0.0, 19.1.0, 19.1.1, and 19.2.0)

这些包包含在以下框架和打包工具中：

- Next.js 版本 ≥14.3.0-canary.77, ≥15 和 ≥16
- 嵌入或依赖 React 服务器组件实现的其他框架和插件（例如，Vite, Parcel, React Router, RedwoodSDK, Waku）

Next.js with versions ≥14.3.0-canary.77, ≥15 and ≥16

Other frameworks and plugins that embed or depend on React Server Components implementation (e.g., Vite, Parcel, React Router, RedwoodSDK, Waku)

## 解决方案

在创建缓解措施以解决此漏洞后，我们将其部署到我们的全球分布式平台，以快速保护我们的客户。我们仍然建议升级到最新的已修复版本。

React 和受影响的下游框架的更新版本包含了对用户输入的强化处理，以防止意外行为。所有用户应尽快升级到已修复的版本。如果您使用的是 Next.js 14.3.0-canary.77 或更高版本的 canary 版本，请降级到最新的稳定 14.x 版本。

### 已在以下版本中修复：

- React: 19.0.1, 19.1.2, 19.2.1
- Next.js: 15.0.5, 15.1.9, 15.2.6, 15.3.6, 15.4.8, 15.5.7, 15.6.0-canary.58, 16.0.7

React: 19.0.1, 19.1.2, 19.2.1

Next.js: 15.0.5, 15.1.9, 15.2.6, 15.3.6, 15.4.8, 15.5.7, 15.6.0-canary.58, 16.0.7

使用上述包的框架和打包工具应安装其各自维护者提供的最新版本。
