您现在可以使用 Vercel BotID 来保护任何 JavaScript 框架（如 SvelteKit 和 Nuxt）中最敏感的端点。

BotID 是我们针对高价值端点（如注册、结账和 AI 交互）的高级机器人防护方案。自推出以来，它已保护了近百万次 API 请求。

安装或升级到 `botid@1.4.3` 版本，通过新的 `initBotId({ protect: ... })` 函数，增加了对通用 JavaScript 环境的支持。

以下是在 SvelteKit 中设置 BotID 时使用 `initBotId` 的示例：

```
1import { initBotId } from 'botid/client/core';2 3export function init() {4  initBotId({5    protect: [6      {7        path: '/api/user',8        method: 'POST',9      },10    ],11  });12}
```

在 SvelteKit 项目中初始化 BotID

我们建议在 `Next.js v15.3+` 中使用 `initBotId()` 于 `instrumentation-client.ts` 文件以获得更好的性能。更早的版本可以继续使用 React `<BotIdClient>` 组件的方式。

请查阅更新后的文档，获取所有支持框架的设置说明。
