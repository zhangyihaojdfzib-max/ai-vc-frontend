更快的工作流程、动态内容以及更少的瓶颈：了解 MetaMask 迁移至 Vercel 的详情

自 2014 年以来，Consensys 凭借 Linea、Infura 和 MetaMask（全球使用最广泛的自主托管钱包，拥有数百万全球用户）等工具，塑造了 web3 运动。

随着区块链生态系统的迅速成熟，对能够与构建它的团队一样快速发展的网站的需求变得清晰。为满足这一需求，Consensys 将 MetaMask.io 迁移至 Next.js 和 Vercel，构建了一个专为扩展、速度和持续迭代而设计的架构。

## 链接到标题 迁移之路

在迁移之前，MetaMask.io 采用静态站点生成（SSG）构建，并托管在 GitHub Pages 上。虽然这种方法在早期发挥了作用，但随着产品的发展，它开始面临限制。部署速度变慢，像 API 事件这样的动态内容难以支持，CMS 也难以跟上内部开发的步伐。即使是小更新也常常需要手动变通。

Consensys 已通过其他 MetaMask 项目（如 MetaMask Learn）证明了 Vercel 的价值。扩展使用带有服务端渲染（SSR）的 Next.js，并在 Vercel 上实现标准化，是明确的下一步。

## 链接到标题 重建 MetaMask

此次迁移的重点是提高运营可靠性，确保网站能够安全地应对产品发布和快速增长。

Consensys 将 MetaMask.io 迁移至一个为扩展、速度和迭代而构建的现代技术栈：

- **Next.js** 为前端提供动态渲染支持，使团队能够完全控制性能和内容交付
- **Contentful** 提供了一个现代 CMS，使非开发人员能够更快地管理和更新内容，同时不牺牲灵活性
- **Vercel** 处理 CI/CD、托管、全球交付和可观测性，使团队能够以最小的开销在全球范围内部署、扩展和监控网站

通过这次迁移，团队提升了性能、可靠性和 SEO 的标准。他们进行了金丝雀发布，首先用 1% 的流量进行测试以验证系统稳定性，然后在四小时内扩展到 100%，无需任何回滚。

## 链接到标题 新架构的实际应用

如今，MetaMask.io 运行在 Vercel 的全球边缘网络上，以快速交付静态资产，同时 Vercel Functions 处理动态端点。借助增量静态再生（ISR）和智能缓存等功能，团队可以向最终用户提供更丰富的动态内容和实时数据（如代币价格），这在之前的 SSG 站点中是无法实现的。

此外，从工程到设计再到内容的跨职能团队可以实时协作。带有内联评论的预览 URL 和并发构建允许快速迭代并减少 QA 中的摩擦。

## 链接到标题 重要的成果

性能提升立竿见影。正常运行时间改善，部署加速，团队获得了更好的 SEO 效果。Speed Insights 确认了这些改进，内部反馈也很明确：产品、营销和设计等各个团队的运作速度更快，贡献也更有效。
