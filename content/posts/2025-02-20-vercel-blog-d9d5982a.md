TheFlags SDK——我们用于在 Next.js 和 SvelteKit 应用中使用功能标志的开源库——现已以新的包名 `flags` 发布。

```
1import { flag } from 'flags/next';2 3export const exampleFlag = flag({4  key: 'example-flag',5  decide() {6    7    return Math.random() > 0.5;8  },9});
```

新名称彰显了我们对开源以及该包独立于任何特定实体或平台的承诺。该 SDK 采用框架优先的方法，旨在简化使用、避免客户端标志评估，并通过消除布局偏移来改善用户体验。

我们正在与 Statsig、Optimizely 和 LaunchDarkly 等合作伙伴共同开发适配器，以确保与 Flags SDK 的无缝集成。

此前，每个提供商在 Next.js 等框架中使用功能标志时都采用各自的方法，这导致了行业内的重复劳动和实现上的差异。展望未来，Flags SDK 将帮助所有功能标志和实验提供商受益于其与框架的紧密集成，同时保留其独特能力。

如果你正在使用 `@vercel/flags`，请确保更新到版本 3.1.1，并将你的导入和 `package.json` 切换为 `flags`。

更多信息请参阅我们重新设计的文档和示例。
