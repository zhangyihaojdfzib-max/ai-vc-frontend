# 推出 Claude Sonnet 4.6

Claude Sonnet 4.6 是我们迄今为止能力最强的 Sonnet 模型。它在编码、计算机使用、长上下文推理、Agent（智能体）规划、知识工作和设计等所有技能上实现了全面升级。Sonnet 4.6 还具备一个处于测试阶段的 100 万 Token 上下文窗口。

对于我们免费版和 Pro 版的用户，Claude Sonnet 4.6 现已成为 claude.ai 和 Claude Cowork 中的默认模型。定价与 Sonnet 4.5 保持一致，每百万 Token 起价为 3 美元/15 美元。

Sonnet 4.6 为我们更多的用户带来了显著提升的编码能力。在一致性、指令遵循等方面的改进，使得获得早期访问权限的开发者们对 Sonnet 4.6 的偏好远超其前代模型。他们甚至常常更青睐它，而不是我们 2025 年 11 月发布的最智能模型 Claude Opus 4.5。

以往需要动用 Opus 级别模型才能获得的性能表现——包括在现实世界、具有经济价值的办公任务上——现在通过 Sonnet 4.6 即可实现。与之前的 Sonnet 模型相比，该模型在计算机使用技能方面也显示出重大改进。

与每一款新的 Claude 模型一样，我们对 Sonnet 4.6 进行了广泛的安全评估，总体显示其安全性与我们近期其他 Claude 模型相当或更优。我们的安全研究人员得出结论，Sonnet 4.6 具有“一种大体上热情、诚实、亲社会且有时幽默的性格，非常强的安全行为，并且没有迹象表明存在高风险形式的不对齐问题”。

## 计算机使用

几乎每个组织都有难以轻易自动化的软件：那些在现代接口（如 API）出现之前构建的专用系统和工具。要让 AI 使用此类软件，用户过去必须构建定制的连接器。但是，一个能够像人一样使用计算机的模型改变了这个等式。

2024 年 10 月，我们率先推出了一款通用计算机使用模型。当时我们写道，它“仍处于实验阶段——有时笨拙且容易出错”，但我们预期会有快速改进。OSWorld 是 AI 计算机使用的标准基准测试，它展示了我们的模型已经取得了多大的进步。它提供了数百个在模拟计算机上运行的真实软件（Chrome、LibreOffice、VS Code 等）的任务。没有特殊的 API 或专门构建的连接器；模型以与人非常相似的方式观察计算机并与之交互：点击（虚拟）鼠标和在（虚拟）键盘上打字。

在十六个月的时间里，我们的 Sonnet 模型在 OSWorld 上稳步取得进展。这些改进也体现在基准测试之外：早期的 Sonnet 4.6 用户在处理诸如浏览复杂电子表格或填写多步骤网页表单等任务时，看到了接近人类水平的能力，然后才在多个浏览器标签页中整合所有内容。

当然，在使用计算机方面，该模型仍然落后于最熟练的人类。但进步的速度依然显著。这意味着计算机使用在一系列工作任务中变得更加有用——并且能力更强的模型指日可待。

## 评估 Claude Sonnet 4.6

除了计算机使用，Claude Sonnet 4.6 在各项基准测试中均有提升。它以更实惠的价格点接近了 Opus 级别的智能水平，使其在更多任务中更具实用性。您可以在我们的系统卡片中找到关于 Sonnet 4.6 能力及其安全相关行为的完整讨论；以下是与近期其他模型的总结和比较。

![一张流行基准测试表格，展示了 Sonnet 4.6 相对于其他前沿模型的性能表现](/images/posts/6803816ffc26.jpg)

在 Claude Code 中，我们的早期测试发现，用户大约 70% 的时间更倾向于选择 Sonnet 4.6 而非 Sonnet 4.5。用户报告称，它在修改代码前能更有效地阅读上下文，并能整合共享逻辑而非重复它。这使得在长时间会话中使用它比早期模型更少令人沮丧。

用户甚至在 59% 的时间里更倾向于选择 Sonnet 4.6 而非我们 11 月发布的前沿模型 Opus 4.5。他们认为 Sonnet 4.6 明显更不容易过度设计和“懒惰”，并且在遵循指令方面有显著提升。他们报告称，虚假的成功声明更少，幻觉更少，在多步骤任务上的执行一致性更高。

Sonnet 4.6 的 100 万 Token 上下文窗口足以在一次请求中容纳整个代码库、冗长的合同或数十篇研究论文。更重要的是，Sonnet 4.6 能够有效地在整个上下文中进行推理。这可以使其在长远规划方面表现更佳。我们在 Vending-Bench Arena 评估中特别清楚地看到了这一点，该评估测试模型在一段时间内运营（模拟）业务的能力——并且包含竞争元素，不同的 AI 模型相互对抗以获取最大利润。

Sonnet 4.6 制定了一个有趣的新策略：在前十个模拟月中大力投资产能，支出远超其竞争对手，然后在最后阶段急剧转向专注于盈利能力。这一转向的时机帮助它以明显优势领先于竞争对手。

早期客户也报告了广泛的改进，其中前端代码和财务分析尤为突出。客户独立描述称，Sonnet 4.6 的视觉输出明显更加精致，与之前的模型相比，具有更好的布局、动画和设计感。客户达到生产质量结果所需的迭代轮次也更少。

## 产品更新

在 Claude 开发者平台上，Sonnet 4.6 同时支持自适应思维和扩展思维，以及处于测试阶段的上下文压缩功能，该功能会在对话接近限制时自动总结较早的上下文，从而增加有效上下文长度。

在我们的 API 上，Claude 的网页搜索和获取工具现在会自动编写和执行代码来过滤和处理搜索结果，仅保留相关内容在上下文中——既提高了响应质量，也提升了 Token 效率。此外，代码执行、记忆、编程式工具调用、工具搜索和工具使用示例现已全面开放。

Sonnet 4.6 在任何思维强度下都能提供强大的性能，即使关闭扩展思维也是如此。作为从 Sonnet 4.5 迁移的一部分，我们建议您探索各种设置，以根据您正在构建的内容找到速度与可靠性能之间的理想平衡。

我们发现，对于需要最深层次推理的任务，例如代码库重构、在工作流中协调多个 Agent（智能体），以及那些必须做到恰到好处至关重要的问题，Opus 4.6 仍然是最强的选择。

对于Excel用户中的Claude用户，我们的插件现已支持MCP连接器，让Claude能够与您日常使用的其他工具协同工作，例如标普全球、伦敦证券交易所集团、Daloopa、PitchBook、穆迪和FactSet。您可以要求Claude从电子表格外部获取上下文信息，而无需离开Excel。如果您已经在Claude.ai中设置了MCP连接器，这些相同的连接将在Excel中自动生效。此功能适用于Pro、Max、Team和企业版计划。

## 如何使用Claude Sonnet 4.6

Claude Sonnet 4.6现已登陆所有Claude计划、Claude Cowork、Claude Code、我们的API以及所有主要云平台。我们还将免费套餐默认升级为Sonnet 4.6——现在包含文件创建、连接器、技能和压缩功能。

如果您是开发者，可以通过Claude API使用`claude-sonnet-4-6`快速开始。
