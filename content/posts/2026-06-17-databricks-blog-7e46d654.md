以下是新功能：

## 从全页命令中心管理复杂工作

数据和机器学习开发很少在单次提示词中完成。用户可能需要检查现有逻辑、更新多个资产、运行代码、审查输出，并根据结果优化下一步。这项工作可能涉及笔记本、SQL、Lakeflow 管道、仪表板、作业、模型、服务端点和 Unity Catalog 资产。

我们重新设计了 Genie Code 体验，为团队提供了一个专门用于此类复杂数据和机器学习工作的命令中心。用户无需在较小的侧面板中管理较长的任务，而是可以使用全页体验来描述任务、跟踪进度、审查输出并持续迭代。

![Genie Code 全页用户体验](/images/posts/d3b91bdd67a5.png)

团队可以管理多个 Genie Code 线程，查看线程何时在执行或等待输入，并在新结果就绪时返回每个线程。他们可以重命名线程、搜索之前的对话，并在项目演进过程中保持方向感。

命令中心还使自定义功能更易于发现。指令、技能和连接器更加醒目，因此团队可以使用正确的标准、工具和工作区知识来指导 Genie Code。

要访问全页命令中心，请打开 Genie Code 侧面板，然后单击左上角的最大化按钮。

## 将基于 Agent（智能体）的开发引入机器学习工作流

在机器学习项目中，模型只是很小的一部分。大部分时间都花在围绕它的工程上：将原始数据转化为特征、运行实验、将候选模型投入生产，并在真实流量到来后保持其健康运行。这项工作缓慢且昂贵，这也是大多数团队运行的模型远少于其用例数量的原因。

用于机器学习的 Genie Code 正是针对这些工程问题。它是一组内置于 Genie Code 的能力和智能升级，因此您无需采用新工具。您已经在使用的同一个 Agent（智能体）将成为您现有 Databricks 机器学习栈中生产机器学习工程的专家。

![Genie Code 构建特征](/images/posts/b2aaa750c3a8.gif)

Genie Code 的专业知识来自两个方面。首先是 Databricks。我们与客户一起运行生产机器学习已有十多年，我们看到了模型在哪里失效、团队在哪里浪费时间，以及什么区分了一个有效的模型和一个看似正确却毫无预警失败的模型。Genie Code 在工作时应用这些经验教训，处理经验丰富的从业者会注意的细节，例如纠正类别不平衡和检查特征质量。

其次是您的团队。一个通用的编码 Agent（智能体）没有见过您过去的实验、您的业务指标、您的评估集，或者您如何权衡一个目标与另一个目标，因此它会猜测。Genie Ontology 弥补了这一差距。它学习您的团队如何构建特征、训练模型和评估候选模型，Genie Code 遵循这些模式，而不是退回到不相关的默认值。

凭借这两种知识，Genie Code 成为日常模型开发中更强大的合作伙伴。它按照您团队的模式编写特征，在涉及的多个文件之间进行协调编辑，运行和调试您的代码，并使用您自己的评估脚本比较候选模型。您始终参与其中并决定保留什么。

![Genie Code 评估模型](/images/posts/5c308b3b4dd3.gif)

Genie Code 现已原生集成到整个 Databricks 机器学习栈中。最新升级包括：

- MLflow。Genie Code 读取您的实验和可观测性数据：运行记录、工件、模型谱系、质量指标和系统指标。询问它“如何在训练期间提高 GPU 利用率？”或“我应该为此模型跟踪哪些其他指标？”，它会根据您自己的运行记录给出答案。
- 模型服务。Genie Code 检查端点健康状态和性能，诊断服务问题，并找到优化运行中端点的方法。
- 计算感知。当作业需要 GPU 进行训练时，Genie Code 会切换到 AI Runtime，并使用工作区环境功能来设置环境，从而让您跳过基础设施设置。

结果是一个 Agent（智能体），它完成现实世界数据科学任务的频率远高于通用编码 Agent（智能体）。

## 通过计划任务让 Genie Code 自主工作

到目前为止，Genie Code 主要是交互式的：您提问，它响应，您在工作进展过程中始终参与。计划任务改变了这一点。

即将推出的计划任务将让 Genie Code 在您不在笔记本电脑前时也能代表您工作，然后在您返回时将结果交给您审查。计划任务从一个提示词开始，并可选择关联一个相关资产，例如笔记本、工作流或仪表板。运行时，Genie Code 会创建一个包含结果的线程，用户可以审查、优化或继续交互。

例如，数据团队可以要求 Genie Code 检查隔夜作业结果、总结管道运行情况、解释仪表板指标的变化、准备每周分析，或在团队会议前审查模型性能。用户无需手动重新运行提示词，也无需在工作进行时保持活跃聊天状态。

计划任务将 Genie Code 从交互式辅助转向自主工作。它们帮助团队保持重要工作流的推进，同时结果保持可见、可审查，并基于 Databricks 上下文。

Genie ZeroOps 将这种方法扩展到生产运维中。它监控实时系统，调查问题，并准备修复方案供团队审查和批准。对于机器学习系统，这可以包括模型漂移、服务错误和上游管道问题。对于数据工程系统，它可以帮助团队从监控和诊断转向修复和优化。
