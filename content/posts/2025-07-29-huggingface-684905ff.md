# Trackio 简介：来自 Hugging Face 的轻量级实验追踪库

TL;DR：Trackio 是一个全新的、开源且免费的实验追踪 Python 库，它提供了一个本地仪表盘，并与 Hugging Face Spaces 无缝集成，便于分享和协作。由于 trackio 是 wandb 的即插即用替代品，你可以直接使用你已熟悉的语法开始！

## 背景

如果你训练过自己的机器学习模型，你就会知道在训练过程中追踪指标、参数和超参数，并在之后将其可视化以更好地理解训练过程是多么重要。

大多数机器学习研究人员使用特定的实验追踪库来完成这项工作。然而，这些库可能是付费的，需要复杂的设置，或者缺乏快速实验和分享所需的灵活性。

## 为什么我们转向 Trackio

在 Hugging Face，我们的科学团队已开始在研究项目中使用 Trackio，我们发现它相比其他追踪解决方案有几个关键优势：

**轻松分享与嵌入**：Trackio 使得与同事分享训练进度，或使用 iframe 将图表直接嵌入博客文章和文档变得极其简单。当你想要展示特定的训练曲线或指标，而不要求他人设置账户或浏览复杂的仪表盘时，这一点尤其有价值。

**标准化与透明度**：像 GPU 能耗这样的指标对于追踪并与社区分享非常重要，这样我们才能更好地了解模型训练的能源需求和环境影响。使用 Trackio（它直接从 `nvidia-smi` 命令获取信息）可以轻松量化和比较能耗，并将其添加到模型卡片中。

**数据可访问性**：与一些将你的数据锁定在专有 API 之后的追踪工具不同，Trackio 使得提取和分析正在记录的数据变得直接明了。这对于需要进行自定义分析或将训练指标整合到研究流程中的研究人员至关重要。

**实验灵活性**：Trackio 的轻量级设计使我们能够在训练运行期间轻松尝试新的追踪功能。例如，在训练期间记录张量时，我们可以决定何时将张量从 GPU 移动到 CPU，这在需要追踪模型/中间状态而不影响性能时，能显著提高训练吞吐量。

## 使用 Trackio

那么，Trackio 是什么，你该如何使用它呢？Trackio 是一个**开源 Python 库**，它允许你追踪任何指标，并使用一个本地的 **Gradio** 仪表盘将其可视化。你还可以将此仪表盘同步到 Hugging Face Spaces，这意味着你可以通过分享一个 URL 轻松地将仪表盘分享给其他用户。由于 Spaces 可以是私有的或公开的，这意味着你可以公开分享仪表盘，或者仅在 Hugging Face 组织成员内部分享。

## 用法

`trackio` 被设计为像 `wandb` 这样的实验追踪库的即插即用替代品。其 API 与 `wandb.init`、`wandb.log` 和 `wandb.finish` 兼容，因此你可以在代码中简单地将 `trackio` 导入为 `wandb`。

```diff
- import wandb
+ import trackio as wandb
```

这是一个示例：

```python
import trackio
import random
import time

runs = 3
epochs = 8

def simulate_multiple_runs():
    for run in range(runs):
        trackio.init(project="fake-training", config={
            "epochs": epochs,
            "learning_rate": 0.001,
            "batch_size": 64
        })
        for epoch in range(epochs):
            train_loss = random.uniform(0.2, 1.0)
            train_acc = random.uniform(0.6, 0.95)
            val_loss = train_loss - random.uniform(0.01, 0.1)
            val_acc = train_acc + random.uniform(0.01, 0.05)
            trackio.log({
                "epoch": epoch,
                "train_loss": train_loss,
                "train_accuracy": train_acc,
                "val_loss": val_loss,
                "val_accuracy": val_acc
            })
            time.sleep(0.2)
        trackio.finish()

simulate_multiple_runs()
```

### 与 🤗 Spaces 分享

要将本地仪表盘同步到 Hugging Face Spaces，只需向 `init` 传递一个 `space_id`：

```python
trackio.init(project="fake-training", space_id="org_name/space_name")
```

如果你在 Spaces 上托管仪表盘，你可以直接分享 URL 或使用 iframe 将其嵌入任何地方：

```html
<iframe src="https://org_name-space_name.hf.space/?project=fake-training&metrics=train_loss,train_accuracy&sidebar=hidden" width=600 height=600 frameBorder="0"></iframe>
```

由于 Spaces 可以是私有的或公开的，这意味着你可以公开分享仪表盘，或者仅在 Hugging Face 组织成员内部分享——这一切都是免费的！

当你将 Trackio 仪表盘同步到 Hugging Face Spaces 时，数据会被记录到 Spaces 上的一个临时 Sqlite 数据库中。因为如果你的 Space 重启，这个数据库会被重置，所以 Trackio 还会每 5 分钟将 Sqlite 数据库转换为 Parquet 数据集，并将其备份到一个 Hugging Face 数据集中。这意味着你可以随时轻松地在 Hugging Face 数据集中可视化你记录的指标：

提示：你可以通过向 `trackio.init()` 传入一个 `dataset_id` 来设置此数据集的名称。

### 与 🤗 Transformers 和 🤗 Accelerate 集成

Trackio 原生集成了 Hugging Face 库，如 `transformers` 和 `accelerate`，因此你可以用最少的设置记录指标。

使用 `transformers.Trainer`：

```python
import numpy as np
from datasets import Dataset
from transformers import Trainer, AutoModelForCausalLM, TrainingArguments


data = np.random.randint(0, 1000, (8192, 64)).tolist()
dataset = Dataset.from_dict({"input_ids": data, "labels": data})


trainer = Trainer(
    model=AutoModelForCausalLM.from_pretrained("Qwen/Qwen3-0.6B"),
    args=TrainingArguments(run_name="fake-training", report_to="trackio"),
    train_dataset=dataset,
)
trainer.train()
```

使用 `accelerate`：

```python
from accelerate import Accelerator

accelerator = Accelerator(log_with="trackio")
accelerator.init_trackers("fake-training")

...  

for step, batch in enumerate(dataloader):
    ...  
    accelerator.log({"training_loss": loss}, step=step)

accelerator.end_training()
```

无需额外设置——只需接入即可开始追踪。

## 设计原则

*   **API 兼容性**：与流行的实验追踪库 API 兼容，使得迁移到 Trackio 或从 Trackio 迁出都无缝衔接。
*   **本地优先**：日志和仪表盘默认在本地运行和持久化，同时可选择托管在 Hugging Face Spaces 上。
*   **轻量级且可扩展**：核心代码库少于 1000 行 Python 代码，易于理解和修改。
*   **免费且开源**：所有功能，包括在 Hugging Face 上托管，都是免费的。
*   **基于 🤗 Datasets 和 Spaces 构建**：用于稳健的数据处理和可视化。
