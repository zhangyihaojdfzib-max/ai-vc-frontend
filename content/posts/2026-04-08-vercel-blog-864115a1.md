## 摘要

React Server Components 中存在一个高严重性漏洞（CVSS 评分 7.5），可能导致拒绝服务攻击。

我们创建了新规则来解决这些漏洞，并将其部署到 Vercel WAF，以自动免费保护所有托管在 Vercel 上的项目。但是，请勿完全依赖 WAF 进行保护。必须立即升级到已修复的版本。

## 影响

可以向任何 App Router 服务器函数端点发送特制的 HTTP 请求，该请求在反序列化时可能触发过度的 CPU 使用。这可能导致未修复的环境中出现拒绝服务。

这些漏洞存在于 Next.js 13.x、14.x、15.x、16.x 版本以及使用 App Router 的受影响包中。该问题在上游被追踪为 CVE-2026-23869。

## 解决方案

在创建了缓解措施以解决此漏洞后，我们将其部署到我们全球分布的平台，以保护我们的客户。我们仍然建议升级到最新的已修复版本。

React 及受影响的下游框架的更新版本包含了防止此问题的修复程序。所有用户应尽快升级到已修复的版本。

## 修复版本

- = 15.0.0 将在 15.5.15 版本中修复
- = 16.0.0 将在 16.2.3 版本中修复
