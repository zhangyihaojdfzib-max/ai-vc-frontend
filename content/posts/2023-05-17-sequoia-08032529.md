# 开发者工具 2.0

Copilot 已成功捕获闪电。GitHub 这款为开发者打造的 AI 助手，能预测你接下来可能想写的代码，其成功是多方面的。它深受工程师喜爱，在约一年内吸引了超过百万用户，并显著提升了生产力。（即使你是顶尖工程师，它也可能帮你完成一半或更多的代码。）它也有望成为商业爆款。以每个席位每月 10-19 美元的价格，Copilot 有望在 GitHub 的 1 亿用户中创造 10 亿美元或更高的年收入。

Copilot 的成功引发了一场淘金热。创始人们竞相将大语言模型的能力引入其他各类行业，构建工具来帮助专业人士写作、编码、设计和创作媒体。出现了"律师的 Copilot"、"医生的 Copilot"、"设计师的 Copilot"，以及众多其他"X 的 Copilot"。

这些都是令人兴奋的方向。我们认为生成式 AI 将接连变革一个又一个行业，让各类专业人士在工作中更高效，并取悦一波又一波的消费者。

但对于开发者而言，还有更多事情可做。Copilot 利用了 OpenAI 的 Codex 模型，可能只是 AI 变革软件工程师工作方式的第一枪。Andrej Karpathy 在 2017 年预测神经网络将创造新一代软件——"软件 2.0"，而我们可能会看到帮助人们制作软件的工具经历同样的重塑——即"开发者工具 2.0"。

这里存在许多机会。一些创始人正在迭代 Copilot 普及的"编辑器内、编码时获取帮助"的体验，尝试不同的交互模式或不同的模型。例如 Replit 的 Ghostwriter、Sourcegraph 的 Cody、TabNine 等。

更多机会则存在于更广阔的领域。你可以瞄准工程师编写代码之外的工作，如调试和文档编写——或者工程团队的其他工作，如事件响应。你可以考虑除了"更快地编写代码"之外的价值主张，比如"编写性能更高或更安全的代码"。你可以摒弃插件形式，重建整个应用程序。你可以关注软件工程师之外的角色，比如需要提升笔记本编写效率的数据科学家（参见：Hex），或者辛苦编写 SQL 查询的数据分析师。这里有一片广阔的机会水域可供探索，有许多切入点可以融入开发者的工作流程。

![](/images/posts/5ced97a3bb2d.png)

要构建传奇公司来实现这一切并非易事。Copilot 因涉及软件盗版问题而面临法律审查。同时控制着 GitHub 和 VS Code 的现有巨头微软，享有显著的分发优势。许多创始人已经开始基于 LLM 进行构建，使得许多机会竞争激烈。GitHub 本身刚刚宣布计划通过基于 GPT-4 的全新版本 Copilot 提供更广泛的 AI 功能。

但我们认为，AI 变革软件工程存在巨大机遇，问题在于**谁**能成功，而非**是否**能成功。我们认为，决定胜负的关键因素包括：专注于开发者体验、提供全新的能力，以及在如何切入和扩展至开发者工作流程方面做出战略选择。在此领域的成功可能意味着重写工程实践的方式——并有机会打造一家划时代的公司。
