tl;dr：Anthropic 近日提出了智能体技能的概念。技能本质上是包含一个 SKILL.md 文件及相关文件（如文档或脚本）的文件夹，智能体能够发现并动态加载这些技能，从而在特定任务中表现更佳。我们已在 deepagents-CLI 中添加了技能支持。

**通用智能体的崛起**
像 Claude Code 和 Manus 这样的通用智能体已被广泛采用。虽然我们可能预期通用智能体会使用大量工具，但一个令人惊讶的趋势出现了：它们使用的工具数量极少。Claude Code 使用了大约十几个，而 Manus 使用的工具不到 20 个。

通用智能体如何能够仅用少量工具就胜任工作？关键在于赋予智能体访问计算机的能力。通过 bash 和文件系统工具，智能体可以像人类一样执行操作，而无需为每项任务配备专门的绑定工具。

我们已在 deepagents 中应用了这些原则。deepagents 是我们的开源智能体框架，支持文件系统操作和代码执行。请在此处查看我们的概述视频和相关幻灯片。

**具备技能的通用智能体**
工具数量很少的通用智能体如何执行多样化的操作？在我们的网络研讨会上，Manus 讨论了一种方法：将操作从工具卸载到文件系统。与其提供大量工具，不如为智能体配备一台计算机，其中包含用于广泛操作的脚本/指令。智能体只需使用其文件系统和 shell 工具，即可利用这些资源执行多种操作。

Anthropic 的技能正是遵循了相同的模式。技能只是一组文件夹，每个文件夹包含一个 SKILL.md 文件，该文件包含 YAML 前言和 Markdown 指令。下图来自 Anthropic 的博客文章，展示了 SKILL.md 文件的结构：

与传统的工具相比，技能具有两大优势：第一个好处是 Token 效率。技能是渐进式披露的。默认情况下只加载 YAML 前言；智能体仅在需要时才读取完整的 SKILL.md 文件。传统工具则要求将所有定义预先放入上下文，这可能会膨胀上下文窗口。您可以通过下图（来自 Anthropic 的博客文章）了解这一点：

第二个好处是减轻认知负荷。智能体调用一小部分原子工具，而不是在众多可能功能重叠的工具中导航——后者是导致上下文混乱的常见原因。下表比较了技能与工具，并说明了技能在编码操作方面的相关优势。

技能不仅能减少 Token 使用，还能实现强大的功能。正如 Anthropic 的 Barry Zhang 所提到的，技能是迈向持续学习的一步：智能体在遇到新任务时可以即时创建新技能。技能也易于在智能体之间共享，并在会话内组合，允许智能体根据需要引入多种技能。

**Deep Agents CLI + 技能**
我们的 deepagent-CLI 是一个开源编码助手，可以像 Claude Code 一样使用您的本地文件系统。我们已为 deepagent-CLI 添加了技能支持，使其能够使用庞大且不断增长的公共技能库。

只需为您的智能体创建一个技能文件夹，并从我们的代码库复制任何示例技能即可：
```
mkdir -p ~/.deepagents/agent/skills
cp -r examples/skills/web-research ~/.deepagents/agent/skills/
```
这些技能会在 deepagent CLI 启动时默认加载，您只需在终端中运行 `deepagents skills list` 即可查看所有可用技能：

当您向 deepagents 提出与任何技能相关的请求时，它将自动读取相关的 SKILL.md 文件并执行该技能。有关技能的完整概述，请在此处查看我们的视频并参阅 README。
