Chat SDK 现已支持将 PostgreSQL 作为状态后端，通过新的 PostgreSQL 适配器，它成为继 Redis 和 ioredis 之后又一个可用于生产环境的选择。

已经运行 PostgreSQL 的团队，无需在基础设施中额外引入 Redis，即可持久化订阅、分布式锁和键值缓存状态。

立即尝试 PostgreSQL 状态适配器：

```
1import { Chat } from "chat";2import { createPostgresState } from "@chat-adapter/state-pg";3
4const bot = new Chat({5  userName: "mybot",6  adapters: {7    8  },9  state: createPostgresState(),10});
```

该适配器使用 `pg`（node-postgres）库执行原始 SQL 查询，并在首次连接时自动创建所需的表。它支持基于 TTL 的缓存、跨多个实例的分布式锁，以及通过可配置的键前缀实现命名空间隔离的状态管理。

请阅读[文档](documentation)以开始使用，或浏览[适配器目录](adapters directory)。

特别感谢 @bai，他在 [PR #154](PR #154) 中的社区贡献为此适配器奠定了基础。
