# 自推测解码实现更快的文本生成

自推测解码是《LayerSkip：实现早期退出推理与自推测解码》一文中提出的一种新颖的文本生成方法。它结合了推测解码与大语言模型（LLM）早期退出的优势。该方法通过使用**同一模型**的早期层来起草Token，并使用后续层进行验证，从而实现高效生成。

这项技术不仅加快了文本生成速度，还实现了显著的内存节省并降低了计算延迟。为了获得端到端的加速，早期层的输出需要与最后一层的输出足够接近。这通过一种训练方法实现，如论文所述，该方法可在预训练期间以及针对特定领域进行微调时应用。自推测解码对于实际应用尤其高效，能够部署在较小的GPU上，并降低大规模推理所需的整体硬件占用。

## 推测解码与自推测解码

![LayerSkip演示GIF](/images/posts/145ca8854239.gif)

传统推测解码使用**两个模型**：一个较小的模型（草稿模型）生成一系列草稿Token，一个较大的模型（验证模型）验证草稿的准确性。较小模型执行生成的大部分工作，而较大模型则优化结果。这提高了文本生成速度，因为较大模型一次性验证完整的序列，而不是逐个生成草稿。

在自推测解码中，作者基于这一概念进行了扩展，但使用大模型的早期层来生成草稿Token，然后由模型的更深层进行验证。这种推测解码的"自"方面需要特定的训练，使得模型能够同时执行起草和验证。与传统推测解码相比，这反过来提高了速度并降低了计算成本。

## 与transformers库的使用

为了在🤗 transformers库中启用早期退出自推测解码，我们只需在`generate()`函数中添加`assistant_early_exit`参数。

以下是一个展示该功能的简单代码片段。

```sh
pip install transformers
```

```py
from transformers import AutoTokenizer, AutoModelForCausalLM

early_exit_layer = 4
prompt = "Alice and Bob"
checkpoint = "facebook/layerskip-llama2-7B"

tokenizer = AutoTokenizer.from_pretrained(checkpoint)
inputs = tokenizer(prompt, return_tensors="pt").to("cuda")

model = AutoModelForCausalLM.from_pretrained(checkpoint).to("cuda")
outputs = model.generate(**inputs, assistant_early_exit=early_exit_layer)
```

注意：虽然`assistant_early_exit`参数可能为任何仅解码器Transformer启用早期退出自推测解码，但除非模型经过专门训练，否则来自中间层的logits无法被解嵌入（通过LM Head解码的过程，将在博客后续部分描述）。您也**只会**在通过训练提高了早期层准确性的检查点上获得加速。《LayerSkip》论文提出了一种训练方法来实现这一点（即，应用早期退出损失，并逐步增加层丢弃率）。使用LayerSkip训练方法持续预训练的Llama2、Llama3和Code Llama检查点集合在此处提供。

## 早期退出与解嵌入

自推测解码中的一项关键技术是早期退出，即生成过程可以在预先指定的层停止。为了实现这一点，我们通过将这些层的logits投影到语言模型（LM）头上来**解嵌入**，以预测下一个Token。这使得模型能够跳过后续层并提高推理时间。

解嵌入可以在任何Transformer层执行，将早期退出转变为一种高效的Token预测机制。一个自然的问题出现了：当LM头最初仅训练用于处理最后一层时，如何使其适应于解嵌入早期层的logits？这就是训练修改发挥作用的地方。

## 训练修改：层丢弃与早期退出损失

在训练阶段，我们引入了**层丢弃**，它允许模型在训练期间跳过某些层。丢弃率在更深层中逐步增加，使模型减少对后续层的依赖，同时增强模型的泛化能力并加速训练。

除了层丢弃之外，还应用了**早期退出损失**，以确保LM头学会解嵌入不同层。使用早期退出训练模型的总损失函数由每个退出点（中间层）的归一化损失之和给出。该技术通过将学习任务分布到所有层来实现高效训练。

## 自起草与自验证

训练完成后，我们可以在推理期间应用自推测解码。该过程从**自起草**开始，即通过从某个中间层提前退出生成Token。推测Token的数量定义了在此阶段产生多少个草稿Token，而退出的层则决定了草稿阶段的大小和准确性。这两个参数都可以在推理时根据草稿阶段速度与准确性之间的**权衡**来指定。

下一阶段是自验证，即使用完整模型来验证草稿Token。验证模型会复用草稿模型的部分缓存。如果草稿Token与验证Token一致，它们就会被添加到最终输出中，从而更好地利用系统中的内存带宽，因为只要部分Token匹配，使用完整模型生成Token序列的成本远高于验证草稿。

在自验证阶段，仅需计算剩余层进行验证，因为早期层的计算结果已在草稿生成阶段缓存。

## 优化：共享权重、共享KV缓存与共享计算

自推测解码显著受益于缓存复用，尤其是KV缓存——它存储了草稿阶段计算出的键值对。该缓存使模型能够跳过冗余计算，因为草稿和验证阶段使用相同的早期层。此外，退出查询缓存存储了退出层的查询向量，使验证能够从草稿阶段无缝衔接。

与传统的双模型推测解码相比，早期退出自推测解码可受益于以下节省：

- **共享权重**：草稿和验证阶段复用前E层权重。
- **共享KV缓存**：草稿和验证阶段复用前E层的键值对。
- **共享计算**：通过使用仅保存退出层E-1查询向量的退出查询缓存，复用前E层的计算结果，使验证过程无需计算第0层至E-1层。

KV缓存与退出查询缓存的组合（称为KVQ缓存）可减少内存开销并提升推理延迟。

目前，🤗 transformers库已在此拉取请求中实现了第一项优化（共享权重）。随着使用该方法模型数量的增加，我们将考虑后续优化。如有兴趣，欢迎提交PR！

## 结论

LayerSkip利用早期退出、层丢弃和缓存复用之间的协同作用，创建了一个快速高效的文本生成流水线。通过训练模型从不同层解嵌入输出，并利用缓存优化验证过程，该方法在速度与准确性之间取得了平衡。因此，它在保持高质量输出的同时，显著提升了大语言模型的推理时间。由于使用单一模型同时作为草稿和验证模型，它相比传统推测解码技术还减少了内存占用。

自推测是一个令人兴奋的领域，同一LLM可以生成草稿Token并自我修正。其他自推测方法包括：

- **草稿与验证**：草稿阶段跳过预定的注意力层和前馈层。
- **MagicDec**：草稿阶段使用KV缓存的子集，适用于长上下文输入。
- **雅可比解码**与**前瞻解码**：草稿阶段由一系列“猜测Token”组成，这些Token可以是随机的，或从n-gram查找表中获取。
