Workflow SDK 5 测试版现已支持跨工作流和步骤边界的标准 `AbortController` 和 `AbortSignal` API。

在工作流内部创建一个控制器，将其信号传递给一个或多个步骤，并使用 `fetch` 已使用的相同 API 取消正在进行的操作。

```
1import { sleep } from "workflow";2
3export async function cancellableWorkflow() {4  "use workflow";5
6  const controller = new AbortController();7
8  const result = await Promise.race([9    fetchReport(controller.signal),10    sleep("30s").then(() => null),11  ]);12
13  if (result === null) {14    controller.abort();15    return { status: "timed-out" };16  }17
18  return result;19}20
21async function fetchReport(signal: AbortSignal) {22  "use step";23
24  const response = await fetch("https://api.example.com/report", {25    signal,26  });27
28  return response.json();29}
```

将信号传递到步骤中并取消正在进行的操作

该信号在挂起和确定性重放期间保持持久性。当某个步骤正在运行时，即使它位于单独的函数调用中，它也能感知到取消操作。取消也是协作式的；步骤必须检查信号或将其传递给支持 `AbortSignal` 的 API。

使用它可以在持久化超时赢得竞态时停止慢速步骤，在第一个成功响应后取消剩余的请求，通过多步骤管道传递单个信号，或在外部条件发生变化时取消并行工作。

请使用 `workflow@beta` 进行尝试，并阅读取消文档以了解更多信息。
