## 什么是视觉语言模型？

顾名思义，视觉语言模型是一种处理两种模态的多模态模型：视觉和文本。这些模型通常以图像和/或文本作为输入，并生成文本作为输出。

基于对图像和文本（输入）的理解来生成文本（输出）是一个强大的范式。它支持广泛的应用，从图像描述和对象检测到回答关于视觉内容的问题（如下表所示）。需要注意的是，nanoVLM 仅将视觉问答作为训练目标。

我们基于两种知名且广泛使用的架构来构建 nanoVLM。我们的视觉骨干网络（models/vision_transformer.py）是标准的视觉 Transformer，更具体地说是 Google 的 SigLIP 视觉编码器。我们的语言骨干网络遵循 Llama 3 架构。

视觉和文本模态通过一个模态投影模块进行对齐。该模块以视觉骨干网络产生的图像嵌入作为输入，并将其转换为与语言模型嵌入层的文本嵌入兼容的嵌入。然后这些嵌入被拼接起来并馈送到语言解码器中。模态投影模块由一个像素重排操作和一个线性层组成。

![模型架构示意图](/images/posts/61009ea8de80.png)

像素重排器减少了图像 Token 的数量，这有助于降低计算成本并加速训练，特别是对于对输入长度敏感的基于 Transformer 的语言解码器。下图演示了这一概念。

![像素重排示意图](/images/posts/1a9cb85d6b07.png)

所有文件都非常轻量且文档齐全。我们强烈建议你逐一查看它们，以更好地理解实现细节（models/xxx.py）。

在训练时，我们使用以下预训练的骨干网络权重：

1.  视觉骨干网络：`google/siglip-base-patch16-224`
2.  语言骨干网络：`HuggingFaceTB/SmolLM2-135M`

你也可以将骨干网络替换为 SigLIP/SigLIP 2（视觉骨干网络）和 SmolLM2（语言骨干网络）的其他变体。

## 训练你自己的 VLM

既然我们已经熟悉了架构，现在让我们换个话题，谈谈如何使用 `train.py` 训练你自己的视觉语言模型。

```bash
.
├── data
│   └── ...
├── models
│   └── ...
└── train.py     
```

你可以通过以下命令启动训练：

```bash
python train.py
```

这个脚本是你整个训练流水线的一站式解决方案，包括：

*   数据集加载和预处理
*   模型初始化
*   优化和日志记录

**配置**

首先，脚本从 `models/config.py` 加载两个配置类：

*   `TrainConfig`：用于训练的有用配置参数，如学习率、检查点路径等。
*   `VLMConfig`：用于初始化 VLM 的配置参数，如隐藏维度、注意力头数等。

**数据加载**

数据流水线的核心是 `get_dataloaders` 函数。它：

*   通过 Hugging Face 的 `load_dataset` API 加载数据集。
*   组合并打乱多个数据集（如果提供的话）。
*   通过索引应用训练/验证分割。
*   将它们包装在自定义数据集（`VQADataset`、`MMStarDataset`）和整理器（`VQACollator`、`MMStarCollator`）中。

这里一个有用的标志是 `data_cutoff_idx`，可用于在小数据子集上进行调试。

**模型初始化**

模型通过 `VisionLanguageModel` 类构建。如果你要从检查点恢复训练，只需：

```python
from models.vision_language_model import VisionLanguageModel

model = VisionLanguageModel.from_pretrained(model_path)
```

否则，你将获得一个全新初始化的模型，并可选择性地为视觉和语言骨干网络预加载权重。

**优化器设置：两个学习率**

由于模态投影器是全新初始化的，而骨干网络是预训练的，因此优化器被分成两个参数组，每个组有自己的学习率：

*   模态投影器使用较高的学习率
*   编码器/解码器堆栈使用较小的学习率

这种平衡确保了模态投影器快速学习，同时保留了视觉和语言骨干网络中的知识。

**训练循环**

这部分相当标准，但结构经过深思熟虑：

*   使用 `torch.autocast` 进行混合精度训练以提高性能。
*   通过 `get_lr` 实现带线性预热阶段的余弦学习率调度。
*   每批次记录 Token 吞吐量（tokens/sec）以进行性能监控。

每 250 步（可配置），模型会在验证集和 `MMStar` 测试数据集上进行评估。如果准确率提高，则保存模型检查点。

**日志记录与监控**

如果启用了 `log_wandb`，训练统计数据如 `batch_loss`、`val_loss`、`accuracy` 和 `tokens_per_second` 将被记录到 Weights & Biases 以进行实时跟踪。

运行名称使用元数据自动命名，如样本大小、批次大小、轮次数、学习率和日期，所有这些都由辅助函数 `get_run_name` 处理。

**推送到 Hub**

使用以下命令将训练好的模型推送到 Hub，供他人查找和测试：

```python
model.save_pretrained(save_path)
```

你可以轻松地使用以下命令推送它们：



这将使用默认参数，并对图像 `assets/image.png` 运行查询 "What is this?"。你可以像这样在自己的图像和提示词上使用此脚本：


如果你想了解脚本的核心部分，主要就是这几行代码：

```python
model = VisionLanguageModel.from_pretrained(source).to(device)
model.eval()

tokenizer = get_tokenizer(model.cfg.lm_tokenizer)
image_processor = get_image_processor(model.cfg.vit_img_size)

template = f"Question: {args.prompt} Answer:"
encoded = tokenizer.batch_encode_plus([template], return_tensors="pt")
tokens = encoded["input_ids"].to(device)

img = Image.open(args.image).convert("RGB")
img_t = image_processor(img).unsqueeze(0).to(device)

print("\nInput:\n ", args.prompt, "\n\nOutputs:")
for i in range(args.generations):
    gen = model.generate(tokens, img_t, max_new_tokens=args.max_new_tokens)
    out = tokenizer.batch_decode(gen, skip_special_tokens=True)[0]
    print(f"  >> Generation {i+1}: {out}")

```

我们创建模型并将其设置为 `eval` 模式。初始化用于对文本提示词进行分词的 Tokenizer，以及用于处理图像的图像处理器。下一步是处理输入并运行 `model.generate` 来生成输出文本。最后，使用 `batch_decode` 解码输出。
