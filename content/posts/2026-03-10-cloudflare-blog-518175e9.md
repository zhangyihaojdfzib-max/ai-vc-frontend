在网络安全领域，单一数据点很少能揭示全貌。现代攻击者不只是敲前门；他们会探测你的 API，用"噪音"淹没你的网络以分散团队注意力，并试图使用窃取的凭证潜入应用程序和服务器。

要阻止这些多向量攻击，你需要掌握全局。通过使用 Cloudflare 日志浏览器进行安全取证，你可以通过整合 14 个新数据集获得 360 度全方位可见性，覆盖 Cloudflare 应用服务和 Cloudflare One 产品组合的全部表面。通过关联来自应用层 HTTP 请求、网络层 DDoS 和防火墙日志以及零信任访问事件的遥测数据，安全分析师可以显著降低平均检测时间，并有效揭露复杂的、多层次的攻击。

继续阅读，了解更多关于日志浏览器如何为安全团队提供进行快速、深入取证的终极全景视图。

## 您整个技术栈的飞行记录仪

当代数字环境需要深度关联的遥测数据，以防御使用多种攻击向量的对手。原始日志充当应用程序的"飞行记录仪"，捕获每一次交互、攻击尝试和性能瓶颈。由于 Cloudflare 位于边缘，处于您的用户和服务器之间，所有这些事件甚至在请求到达您的基础设施之前就被记录下来。

Cloudflare 日志浏览器将这些日志集中到一个统一的界面中，以便快速调查。

### 支持的日志类型

#### 域名范围日志

重点：网站流量、安全事件和边缘性能。

HTTP 请求

作为最全面的数据集，它充当所有应用层流量的"主要记录"，能够重建会话活动、漏洞利用尝试和机器人模式。

防火墙事件

提供已阻止或已质询威胁的关键证据，使分析师能够识别拦截攻击的特定 WAF 规则、IP 信誉或自定义过滤器。

DNS 日志

通过跟踪在权威边缘解析的每个查询，识别缓存投毒尝试、域名劫持和基础设施层面的侦察。

NEL（网络错误日志）报告

通过跟踪客户端浏览器错误，区分有组织的第 7 层 DDoS 攻击和合法的网络连接问题。

Spectrum 事件

对于非 Web 应用程序，这些日志提供对 L4 流量（TCP/UDP）的可见性，有助于识别针对 SSH、RDP 或自定义游戏流量等协议的异常或暴力攻击。

Page Shield

跟踪和审计对您网站客户端环境（如 JavaScript、出站连接）的未经授权的更改。

Zaraz 事件

检查第三方工具和跟踪器如何与用户数据交互，这对于审计隐私合规性和检测未经授权的脚本行为至关重要。

#### 账户范围日志

重点：内部安全、零信任、管理变更和网络活动。

访问请求

跟踪基于身份的身份验证事件，以确定哪些用户访问了特定的内部应用程序以及这些尝试是否获得授权。

审计日志

提供 Cloudflare 仪表板内配置更改的痕迹，以识别未经授权的管理操作或修改。

CASB 发现

识别 SaaS 应用程序（如 Google Drive 或 Microsoft 365）内的安全配置错误和数据风险，以防止未经授权的数据暴露。

Magic Transit / IPSec 日志

帮助网络工程师执行网络层（L3）监控，例如审查隧道健康状况和查看 BGP 路由变更。

浏览器隔离日志

跟踪用户在隔离浏览器会话内的操作（例如，复制粘贴、打印或文件上传），以防止在不受信任的站点发生数据泄露。

设备状态结果

详细说明连接到您网络的设备的安全健康状况和合规状态，有助于识别受损或不合规的端点。

DEX 应用程序测试

从用户角度监控应用程序性能，有助于区分与安全相关的中断和标准的性能下降。

DEX 设备状态事件

提供用户设备物理状态的遥测数据，有助于将硬件或操作系统层面的异常与潜在的安全事件关联起来。

DNS 防火墙日志

跟踪通过 DNS 防火墙过滤的 DNS 查询，以识别与已知恶意域或命令与控制服务器的通信。

电子邮件安全警报

记录在网关上检测到的恶意电子邮件活动和网络钓鱼尝试，以追踪基于电子邮件的入口向量的来源。

网关 DNS

监控您网络上用户发出的每个 DNS 查询，以识别影子 IT、恶意软件回调或域名生成算法。

网关 HTTP

提供对加密和未加密 Web 流量的完全可见性，以检测隐藏的有效负载、恶意文件下载或未经授权的 SaaS 使用。

网关网络

跟踪 L3/L4 网络流量（非 HTTP），以识别未经授权的端口使用、协议异常或网络内的横向移动。

IPSec 日志

监控加密站点到站点隧道的状态和流量，以确保安全网络连接的完整性和可用性。

Magic IDS 检测

显示与入侵检测特征匹配的结果，向调查人员发出警报，提示已知的漏洞利用模式或恶意软件行为正在网络中传播。

网络分析日志

提供数据包级别数据的高层可见性，以识别针对特定基础设施的容量型 DDoS 攻击或异常流量峰值。

Sinkhole HTTP 日志

捕获定向到"黑洞" IP 地址的流量，以确认哪些内部设备正试图与已知的僵尸网络基础设施通信。

WARP 配置更改

跟踪终端用户设备上 WARP 客户端设置的修改，以确保安全代理未被篡改或禁用。

WARP 开关更改

专门记录用户启用或禁用其安全连接的时间，有助于识别设备可能处于未受保护状态的时段。

零信任网络会话日志

记录已认证用户会话的持续时间和状态，以描绘用户在受保护边界内访问的完整生命周期。

## 日志浏览器可在每个阶段识别恶意活动

通过 **HTTP 请求**、**防火墙事件** 和 **DNS 日志** 获得精细的应用层可见性，准确了解流量如何冲击您的面向公众的资产。使用 **访问请求**、**网关日志** 和 **审计日志** 跟踪内部移动。如果凭证被盗用，您将看到他们去了哪里。使用 **Magic IDS** 和 **网络分析日志** 来发现您私有网络内的容量型攻击和"东西向"横向移动。

### 识别侦察活动

攻击者使用扫描器和其他工具寻找入口点、隐藏目录或软件漏洞。要识别这一点，可以使用日志浏览器查询 `http_requests`，查找来自单个 IP 的任何 `EdgeResponseStatus` 代码为 401、403 或 404 的请求，或对敏感路径（例如 `/.env`、`/.git`、`/wp-admin`）的请求。

此外，`magic_ids_detections` 日志也可用于识别网络层的扫描活动。这些日志提供针对您网络的威胁的数据包级可见性。与标准 HTTP 日志不同，这些日志侧重于网络层和传输层（IP、TCP、UDP）的**基于特征的检测**。通过查询可以发现，在短时间内，单个 `SourceIP` 在广泛的 `DestinationPort` 值范围内触发了多个独特检测的情况。Magic IDS 特征可以专门标记诸如 Nmap 扫描或 SYN 隐形扫描等活动。

### 检查干扰活动

当攻击者进行侦察时，他们可能会尝试同时发起网络洪水攻击来伪装这一行为。请转向`network_analytics_logs`，以判断是否正将流量攻击用作烟幕弹。

### 识别攻击手法

一旦攻击者识别出潜在的漏洞，他们便开始制作攻击武器。攻击者会发送恶意负载（例如SQL注入或大型/损坏的文件上传）来确认漏洞。请审查`http_requests`和/或`fw_events`，以识别任何已触发的Cloudflare检测工具。Cloudflare在这些数据集中记录安全信号，通过`WAFAttackScore`、`WAFSQLiAttackScore`、`FraudAttack`、`ContentScanJobResults`等字段，可以轻松识别包含恶意负载的请求。请查阅我们的文档以全面了解这些字段。通过检查`action`、`source`和`ruleID`字段，`fw_events`日志可用于确定这些请求是否已突破Cloudflare的防御。Cloudflare的托管规则默认会阻止许多此类负载。请查看应用安全概览，以了解您的应用是否受到保护。

如果当前区域未启用托管规则，则显示在安全概览上的托管规则洞察信息

### 审计身份

那个可疑IP是否成功登录？使用`ClientIP`搜索`access_requests`。如果您在敏感的内部应用上看到"Decision: Allow"，那么您就知道有一个账户已遭入侵。

### 阻止泄露（数据外泄）

攻击者有时会使用DNS隧道来绕过防火墙，方法是将敏感数据（如密码或SSH密钥）编码到DNS查询中。日志中显示的将不再是像`google.com`这样的正常请求，而是冗长的编码字符串。通过检查以下字段，寻找对独特的、冗长的、高熵子域查询量异常高的情况：`QueryName`：查找类似`h3ldo293js92.example.com`的字符串；`QueryType`：通常使用`TXT`、`CNAME`或`NULL`记录来承载负载；`ClientIP`：识别是否有单个内部主机正在生成数千个此类独特请求。

此外，攻击者可能尝试通过将敏感数据隐藏在非标准协议中，或以异常方式使用常见协议（如DNS或ICMP）来绕过标准防火墙，从而泄露数据。通过查询`magic_ids_detections`日志，查找标记协议异常的签名（例如`SignatureMessage`中的"ICMP tunneling"或"DNS tunneling"检测）来发现此类行为。

无论您是在调查零日漏洞，还是追踪复杂的僵尸网络，您现在都可以轻松获取所需的数据。

## 跨数据集关联分析

通过在多个并发搜索之间切换，跨多个数据集调查恶意活动。借助日志浏览器，您现在可以使用新的标签页功能同时处理多个查询。在标签页之间切换以查询不同的数据集，或者使用查询结果进行筛选来调整查询和切换分析视角。

当您跨多个Cloudflare日志源关联数据时，可以检测到那些孤立来看似乎无害的复杂多阶段攻击。这种跨数据集分析使您能够看到从侦察到数据外泄的完整攻击链。

### 会话劫持（令牌窃取）

**场景：** 用户通过Cloudflare Access进行身份验证，但其后续的HTTP请求流量看起来像机器人。

**步骤1：** 在`http_requests`中识别高风险会话。

```SQL
SELECT RayID, ClientIP, ClientRequestUserAgent, BotScore
FROM http_requests
WHERE date = '2026-02-22' 
  AND BotScore < 20 
LIMIT 100
```

**步骤2：** 复制`RayID`并搜索`access_requests`，以查看哪个用户账户与该可疑的机器人活动相关联。

```SQL

SELECT Email, IPAddress, Allowed
FROM access_requests
WHERE date = '2026-02-22' 
  AND RayID = 'INSERT_RAY_ID_HERE'
```

### 钓鱼攻击后的C2信标通信

**场景：** 员工点击了钓鱼邮件中的链接，导致其工作站被入侵。该工作站向已知的恶意域名发送DNS查询，随后立即触发IDS警报。

**步骤1：** 通过检查`email_security_alerts`中的违规行为来发现钓鱼攻击。

```SQL
SELECT Timestamp, Threatcategories, To, Alertreason
FROM email_security_alerts
WHERE date = '2026-02-22' 
  AND Threatcategories LIKE 'phishing'
```

**步骤2：** 使用Access日志将用户的电子邮件（`To`）与其IP地址关联起来。

```SQL
SELECT Email, IPAddress
FROM access_requests
WHERE date = '2026-02-22' 
```

**步骤3：** 在`gateway_dns`日志中查找查询特定恶意域名的内部IP。

```SQL

SELECT SrcIP, QueryName, DstIP, 
FROM gateway_dns
WHERE date = '2026-02-22' 
  AND SrcIP = 'INSERT_IP_FROM_PREVIOUS_QUERY'
  AND QueryName LIKE '%malicious_domain_name%'
```

### 横向移动（Access → 网络探测）

**场景：** 用户通过Zero Trust登录，然后尝试扫描内部网络。

**步骤1：** 在`access_requests`中查找来自意外地点的成功登录。

```SQL
SELECT IPAddress, Email, Country
FROM access_requests
WHERE date = '2026-02-22' 
  AND Allowed = true 
  AND Country != 'US' -- 替换为您总部所在的国家
```

**步骤2：** 检查该`IPAddress`是否在`magic_ids_detections`中触发了网络层签名。

```SQL
SELECT SignatureMessage, DestinationIP, Protocol
FROM magic_ids_detections
WHERE date = '2026-02-22' 
  AND SourceIP = 'INSERT_IP_ADDRESS_HERE'
```

### 为更多数据敞开大门

从一开始，日志浏览器的设计就考虑了可扩展性。每个数据集模式都使用JSON Schema（一种广泛采用的描述JSON数据结构和类型的标准）进行定义。这一设计决策使我们能够轻松地从HTTP请求和防火墙事件扩展到Cloudflare全部遥测数据。为我们初始数据集提供支持的相同模式驱动方法，自然地扩展以适应Zero Trust日志、网络分析、电子邮件安全警报以及其间的一切。

更重要的是，这种标准化为摄取Cloudflare原生遥测之外的数据打开了大门。因为我们的摄取管道是模式驱动的，而非硬编码的，所以我们有能力接受任何可以以JSON格式表达的结构化数据。对于管理混合环境的安全团队而言，这意味着日志浏览器最终可以作为一个单一管理平台，将Cloudflare的边缘遥测数据与来自第三方的日志关联起来，所有这些都可以通过相同的SQL接口进行查询。虽然今天的发布重点是完成对Cloudflare产品组合的覆盖，但架构基础已经为未来客户能够使用自定义模式引入自己的数据源做好了准备。

### 更快的数据，更快的响应：架构升级

要有效调查多向量攻击，时机就是一切。日志可用性即使延迟几分钟，也可能意味着主动防御和被动损害控制之间的天壤之别。

这就是为什么我们优化了摄取过程，以获得更好的速度和弹性。通过增加摄取路径中某一部分的并发性，我们消除了可能导致"吵闹邻居"问题的瓶颈，确保一个客户的数据激增不会减慢另一个客户的可见性。这项架构工作使我们的P99摄取延迟降低了约55%，P50延迟降低了25%，从而缩短了边缘事件可供您进行SQL查询所需的时间。
