Vercel 现已支持xmcp，这是一个用于使用 TypeScript 构建和部署 MCP 服务器的框架，无需任何配置。

xmcp 使用基于文件的路由来为您的 MCP 服务器创建工具。

```
1my-project/2├── src/3│   ├── middleware.ts4│   └── tools/5│       ├── greet.ts6│       ├── search.ts7├── package.json8├── tsconfig.json9└── xmcp.config.ts
```

使用 xmcp 进行基于文件的路由

为您的工具创建文件后，您可以使用默认导出方式，这种方式对许多其他基于文件的路由框架来说都很熟悉。下面，我们创建一个“问候”工具。

```
12import { z } from "zod";3import { type InferSchema } from "xmcp";4
5export const schema = {6  name: z.string().describe("要问候的用户姓名"),7};8
910export const metadata = {11  name: "greet",12  description: "问候用户",13};14export default async function greet({ name }: InferSchema<typeof schema>) {15  const result = `Hello, ${name}!`;16
17  return {18    content: [{ type: "text", text: result }],19  };20}
```
