Fluid compute 是 Vercel 的下一代计算模型，旨在通过实时扩展、成本效益和最小开销来处理现代工作负载。传统的无服务器架构优化了快速执行，但在处理需要大量时间等待外部模型或 API 的请求时会遇到困难，导致计算资源浪费。

为了解决这些低效问题，Fluid compute 动态调整以适应流量需求，在配置新资源之前重用现有资源。Fluid 的核心是 Vercel Functions 路由器，它协调函数执行，以最大限度地减少冷启动、最大化并发性并优化资源使用。它会动态地将调用路由到预热的或活跃的实例，确保低延迟执行。

通过高效管理计算分配，路由器可以防止不必要的冷启动，并仅在需要时扩展容量。让我们看看它是如何智能管理函数执行的。

## 链接到标题协调函数执行

Vercel Functions 路由器应用 Fluid 原则来优化执行。它根据实时指标（如负载、可用性和类型）将请求动态分配给实例。

反过来，Fluid 实例处理多个并发请求，减少空闲计算周期并确保高效的工作负载分配。路由智能与灵活执行之间的这种相互作用使 Fluid compute 能够响应式扩展，保持低延迟性能，同时防止过度配置和性能下降。

这种优化是通过安全的基础设施设计实现的，其中实例通过持久 TCP 隧道仅与路由器通信。实例不直接接受流量，而是与函数路由器建立双向连接，使其能够接收传入的调用数据，同时将响应数据流式传回。

这种架构模式增强了安全性，并使 Vercel Functions 路由器能够精确控制工作负载分配和扩展决策。

### 链接到标题Vercel Functions 路由器：高效的请求管理

路由器实时优化每个请求——高效路由、选择最佳计算资源，并通过以下流程动态扩展：

1. 路由、安全和缓存用户按下回车键的那一刻，其请求通过 Anycast 路由到最近的 Vercel 存在点（PoP），以获得尽可能低的延迟。在继续之前：
   - Vercel 防火墙检查流量，阻止 DDoS 攻击和可疑模式等威胁
   - 如果在边缘缓存，则立即提供响应
   - 如果需要执行，请求将转到路由器

2. 计算选择与执行Vercel Functions 路由器根据邻近性、负载和可用性确定最佳执行区域。然后选择最佳计算实例：
   - 优先考虑正在运行的实例（活跃）以提高效率
   - 主动生成、预热的实例（当前空闲）可最大限度地减少冷启动，而字节码缓存可减少其启动时间

3. 处理与响应优化分配实例后，函数开始执行——运行逻辑、查询数据库或调用 API。如果启用了流式传输，则响应会在数据可用时立即开始流动，从而改善首字节时间（TTFB）。

4. 自适应扩展与资源效率为了保持性能并避免浪费计算资源，Fluid 持续：
   - 实时监控流量模式、实例健康状态和工作负载波动
   - 在流量激增期间主动扩展新实例
   - 优雅地缩减空闲实例以优化资源使用

从按键到响应，Fluid 动态优化执行，以速度和效率智能处理每个请求。
