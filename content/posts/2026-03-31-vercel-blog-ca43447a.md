axios npm 包在 2026 年 3 月 31 日被发现遭受了一次主动的供应链攻击。Vercel 调查了此问题并采取了补救措施以保护平台。Vercel 的系统未受影响。

npm 注册表已移除受感染的软件包版本，最新标签现在指向安全的 `axios@1.14.0` 版本。

*   我们已阻止从我们的构建基础设施到命令与控制主机名 `frclak.com` 的出站访问。
*   该软件包的恶意版本已被阻止并从 npm 下架。
*   Vercel 自身的基础设施和应用程序未受影响。我们建议您检查您的供应链是否存在暴露风险。

### 受影响版本

在构建环境中使用 `axios@1.14.1` 或 `axios@0.30.4` 的项目会受到此漏洞影响。

*   检查您的依赖项和锁定文件中是否存在：
    *   `axios@1.14.1`
    *   `axios@0.30.4`
    *   `plain-crypto-js@4.2.1`

### 解决方案

如果您的部署在构建环境中使用了上述恶意软件包版本，请采取以下措施：

*   搜索您的锁定文件和 `node_modules` 中的 `plain-crypto-js` 以识别受感染的安装。
*   重新部署您的项目，以确保构建使用的是干净的 `axios` 版本。
*   轮换 API 密钥、数据库凭据、Token 以及构建环境中存在的任何其他敏感值。
*   检查您的依赖树中是否存在对 `axios@1.14.1` 或 `axios@0.30.4` 的引用，并将其更新至 `axios@1.14.0`。
