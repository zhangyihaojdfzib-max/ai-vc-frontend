### 第一部分：Cosmos 3 有哪些新特性？

与之前的 Cosmos 版本相比，Cosmos 3 最大的变化在于它是一个全模态模型，基于混合 Transformer（MoT）架构构建。此前，开发者需要使用不同的模型来实现不同能力，如世界生成（Cosmos Predict）、可控生成（Cosmos Transfer）、场景理解（Cosmos Reason）和策略生成（Cosmos Policy）。Cosmos 3 通过单一模型即可实现所有这些功能，能够在一次统一的前向传播中推理并生成不同模态。

这意味着您现在可以通过一个模型完成以下所有任务：

- 根据文本、图像、视频或行动输入生成逼真且物理上合理的视频世界
- 推理运动、因果关系和空间关系等物理属性
- 基于当前状态预测未来的视频和行动序列

这对物理 AI 为何重要

Cosmos 3 有助于构建能够理解真实世界的物理 AI 系统。不仅仅是像素和 Token，还包括运动、因果关系、物理规律和行动。无论您是在训练机器人叠衣服、构建自动驾驶仿真，还是为仓库安全场景生成合成训练数据，Cosmos 3 正是为这些用例量身定制的基础模型。

架构

Cosmos 3 基于 MoT 主干构建，在单一统一架构中处理所有模态——文本、图像、视频、音频和行动。每种模态首先由专用编码器编码（视觉理解使用 ViT，视觉/音频生成使用 VAE，行动使用领域感知向量），然后投影到共享表示空间。

![cosmos3-architecture-diagram](/images/posts/bac6bde24484.png)

输入序列被分为两个子序列：一个自回归（AR）子序列，通过下一个 Token 预测处理推理和理解；一个扩散（DM）子序列，通过迭代去噪处理生成。AR 和 DM Token 在每个 Transformer 层中使用独立的参数集，但通过联合注意力机制进行交互——这使得单一模型能够在无需任何架构更改的情况下，无缝切换为 VLM、视频生成器、前向/逆向动力学模型或机器人策略。

模型版本

本次发布的 Cosmos 3 包含两种模型规模，针对不同部署场景进行了优化：

- Cosmos 3 Nano——这是 8B 参数模型（8B 推理器 + 8B 生成器），针对高效推理进行了优化。Cosmos 3 Nano 设计用于在 RTX PRO 6000 GPU 等工作站级计算设备上运行，可在 Hugging Face 的 nvidia/Cosmos3-Nano 获取。
- Cosmos 3 Super——这是 32B 参数模型（32B 推理器 + 32B 生成器），专为大规模合成数据生成（SDG）和研究设计，运行于 NVIDIA Hopper 和 Blackwell GPU 上。Cosmos 3 Super 可在 Hugging Face 的 nvidia/Cosmos3-Super 获取。

### 第三部分：使用 Diffusers 调用 Cosmos 3

Cosmos 3 已集成到 Hugging Face Diffusers 库中，只需几行代码即可轻松使用世界生成流程。您可以通过熟悉的 DiffusionPipeline 经由 Cosmos3OmniPipeline 运行 Cosmos 3。其目标是实现 Cosmos 3 的无缝采用以及与您现有流程的集成。

让我们看一个使用 Cosmos 3 Nano 模型进行单帧生成的文本到图像示例：

```python
import torch
from diffusers import Cosmos3OmniPipeline

pipe = Cosmos3OmniPipeline.from_pretrained(
    "nvidia/Cosmos3-Nano", torch_dtype=torch.bfloat16, device_map="cuda"
)

prompt = (
    "一个现代机器人研究实验室的中景镜头，白色墙壁和灰色地板。"
    "一只金属质感的机械臂安装在干净的白色工作台上，其夹爪位于"
    "一排彩色小物体上方。机器人旁边放着一台笔记本电脑和整齐排列的工具。"
    "后面墙上的大显示器显示着软件界面。场景被头顶的荧光灯"
    "明亮地照亮。"
)

result = pipe(prompt=prompt, num_frames=1, height=720, width=1280)
result.video[0].save("cosmos3_t2i.jpg", format="JPEG", quality=85)

```
