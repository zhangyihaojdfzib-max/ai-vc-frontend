AI 成本正变得越来越难以预测。随着团队更多地依赖编码 Agent（智能体）和其他高 Token 消耗的工作流，一个 API 密钥的消耗速度可能比任何人注意到的都要快：

- 可在无监督情况下循环或扇出的自主工作流
- 若被分享或发布可能遭遇意外流量的演示和原型
- 缺乏每模型成本意识而进行探索或实验的开发者

为任何 API 密钥设置支出上限后，一旦超出限额，AI Gateway 将拒绝该密钥的进一步请求，直到预算重置或您提高上限。该上限适用于通过该密钥运行的所有 AI Gateway 提供商和模型，从而更轻松地整合和管理 AI 成本。

### 链接到标题 Vercel 控制面板中的 API 密钥预算

在 AI Gateway API 密钥页面，点击“创建密钥”，启用“支出配额”选项，输入以美元为单位的限额，并选择刷新周期。

![](/images/posts/e5b75d1e783c.jpg)

![](/images/posts/0fa113837fed.jpg)

您也可以点击右侧的“...”菜单并选择“编辑密钥”，对现有密钥进行编辑，并添加、更改或移除预算。

### 链接到标题 Vercel CLI 中的 API 密钥预算

通过 Vercel CLI 以编程方式创建带预算的 API 密钥。格式如下：

```
vercel ai-gateway api-keys create --name <名称> --budget <美元金额> --refresh-period <周期>
```

为密钥搭配可选的刷新周期（daily、weekly、monthly 或 none），将限额限定在一个时间窗口内。每个周期在其 UTC 窗口开始时重置。

以编程方式创建的密钥也会显示在您团队的 AI Gateway API 密钥视图中，方便您在一个位置查看所有密钥。

阅读 API 密钥文档，了解更多关于设置和使用 API 密钥预算的信息。
