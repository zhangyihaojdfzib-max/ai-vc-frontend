## 异步子智能体

Deep Agents 现在可以将工作委派给在后台运行的远程智能体。与现有的内联子智能体（会阻塞主智能体直至其完成）不同，异步子智能体会立即返回一个任务 ID，并在远程服务器上独立执行。

更多详情请参阅 Deep Agents 文档。

### 动机

内联子智能体对于简短、专注的任务很有效，但它们运行时，会阻塞监督智能体的执行循环。对于需要数分钟而非数秒的工作——深度研究、大规模代码分析、多步骤数据管道——这就成了一个瓶颈。在子智能体返回之前，监督智能体无法响应用户或在其他工作上取得进展。当大多数智能体和子智能体都是短期运行时，这问题还不算大，但随着可处理任务长度的增长，阻塞已成为一个更严重的问题。

异步子智能体消除了这一限制。监督智能体可以并行启动多个子智能体，继续与用户对话，并在结果可用时收集它们。与内联子智能体不同，异步子智能体也是有状态的：它们在多次交互中维护自己的线程，因此监督智能体可以发送后续指令或在任务中途进行纠正。

这也为异构部署打开了大门，即一个轻量级的编排器可以将任务委派给运行在不同硬件上、使用不同模型或维护自己工具集的专用远程智能体。

### 使用方法

`subagents` 参数现已扩展，可以接受一个指向远程智能体的 `AsyncSubAgent` 规范：

```python
from deepagents import AsyncSubAgent, create_deep_agent

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    subagents=[
        AsyncSubAgent(
            name="researcher",
            description="Performs deep research on a topic.",
            url="<https://my-agent-server.dev>",
            graph_id="research_agent",
        ),
    ],
)
```

`AsyncSubAgent` 规范可以与现有的 `SubAgent` 和 `CompiledSubAgent` 规范自由混合使用——`create_deep_agent` 会根据每个规范的类型将其路由到相应的中间件。

交互模式是“发射后不管”：主智能体启动一个任务，继续工作或与用户交谈，并在需要时检查结果。多个异步子智能体可以并发运行。

### 服务器协议

任何实现了 Agent Protocol 的远程智能体都是异步子智能体的有效目标。这意味着你可以将 Deep Agent 指向任何符合 Agent Protocol 的服务器，包括任何使用 LangSmith 部署的智能体、使用服务器存根的自定义 FastAPI 服务，或任何其他实现。我们为 Deep Agents 添加了 Python 和 JS 的示例服务器实现。

如果省略 `url` 字段，Deep Agents 将使用 ASGI 传输与子智能体通信。这使得监督智能体和子智能体可以共同部署并在同一进程中通信。

完整的操作指南——包括部署配置、追踪、故障排除和参考实现——请参阅异步子智能体文档。

### 为何选择 Agent Protocol

异步子智能体需要一个标准的服务器协议，一个无论远程智能体如何部署或部署在何处都能实现的协议。在确定使用 Agent Protocol 之前，我们研究了其他可用的选项。

ACP（Agent Client Protocol）专为编辑器到智能体的通信而构建，在工具领域采用率正在增长。但对于我们的用例，ACP 有两个问题。首先，它围绕同步会话模型构建，客户端发送提示词并等待响应，这与异步子智能体无法清晰对应。其次，它目前仅支持 stdio 传输，这意味着远程智能体必须作为本地子进程运行。HTTP 支持已在规划中但尚未发布，因此 ACP 目前不适用于远程部署。

A2A（Agent-to-Agent Protocol）更接近需求，并且在技术上是兼容的。它拥有完整的 HTTP 支持和原生的异步任务模型。这是一个强大的协议，旨在解决行业内广泛的智能体互操作性挑战，涵盖推送/拉取订阅、智能体发现和能力协商等方面。然而，由于异步子智能体仍在发展中，我们优先选择了一个允许更快迭代的协议。未来版本可能会添加对 A2A 的支持。

模型对应清晰。Agent Protocol 围绕线程和运行构建。你创建一个线程来保存对话上下文，启动一个运行来开始工作，并在需要结果时检查它。这直接映射到异步子智能体的工作方式。这也意味着子智能体在多次交互中是有状态的。当你发送任务中途的更新时，远程智能体会在上下文中继续，因为线程历史被保留，而不是从头开始。

最终，我们得到了一个协议，它自然地适应异步任务模型，适用于托管和自托管部署，并且足够灵活，可以随着功能的发展而演进。

## 扩展的多模态支持

Deep Agents 之前支持从其虚拟文件系统读取图像。此版本将多模态支持扩展到 PDF、音频、视频和其他文件类型。智能体使用相同的 `read_file` 工具；无需更改 API。文件类型根据扩展名自动检测，内容以原生内容块的形式，附带适当的 MIME 类型传递给模型。详情请参阅 Deep Agents 文档。

重要的是，支持哪些模态取决于底层模型。你可以通过模型配置文件以编程方式检查支持的模态——每个 LangChain 聊天模型都会公开一个配置文件，声明其接受的输入类型。
