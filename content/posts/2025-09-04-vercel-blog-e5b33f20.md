## 什么是悬浮Promise？

当一个Promise的创建方式使其错误永远无法被处理或观察到时，它就是“悬浮”的。如果一个Promise被`await`、赋值给变量、从异步函数返回、与`void`运算符一起使用，或者有`.then(...).catch(...)`处理程序，则不被视为悬浮。

这条lint规则很重要，因为由悬浮Promise导致的控制流中断和未处理错误是出了名的难以捕获。许多工程师都有关于悬浮Promise导致生产环境崩溃的“篝火故事”。

考虑到这一点，让我们看看我们的工程师们想出的一些边缘情况。

## Promise数组

让我们从一个“巧干而非苦干”的绝佳例子开始。这个片段直接取自`typescript-eslint`关于其悬浮Promise规则的文档。

```
1[1, 2, 3].map(async (x) => x + 1)
```

它可能看起来无害，但很棘手，因为Promise数组从未被存储或等待，导致所有内部的Promise都处于悬浮状态。

## 类Promise对象

通常，TypeScript知道如果你返回一个`Promise`，当它悬浮时，linter可以捕获到：

```
1function normalPromise(): Promise<number> {2  return new Promise((_, reject) => reject(2))3}4
5normalPromise() 6

```

但如果你返回一个`PromiseLike`（`PromiseLike`是TypeScript内置类型），它只是结构上相似，并非真正的Promise：

```
1function promiseLike(): PromiseLike<number> {2  return new Promise((_, reject) => reject(2))3}4

```

由于`PromiseLike`只是结构匹配，调用`promiseLike()`并忽略其拒绝会导致一个未处理的异步结果悬浮在空间中。

`typescript-eslint`通过一个针对“Thenables”（`PromiseLike`所代表的另一种称呼）的特殊配置选项来处理这种情况。

## 条件类型别名

## 冻结的Promise对象

冻结一个Promise会改变其类型，使检测变得复杂。

```

```

这个表达式的类型是`Readonly<Promise<unknown>>`。这是因为`Object.freeze()`会阻止对你传递给它的任何对象进行写入操作。这很棘手，因为你用`Readonly`实用类型包装了类型，使得linter更难捕获。

## 短路运算符

逻辑运算符返回的是值，而非执行效果，因此它们可能导致 Promise 悬空。

```
1true && new Promise((_, reject) => reject(2)) 
```

经典的短路把戏：`true && <Promise>` 会计算右侧表达式，在纯表达式位置上产生一个 Promise 值。由于没有任何代码 `await` 或 `.catch` 它，这个拒绝状态就未被处理。这提醒我们 `&&` 并不会执行效果并等待，它实际上返回一个值——而这里恰好是一个不稳定的 Promise。

## 随机表达式

如果 Promise 只是有时会游离呢？当涉及随机性时，有时只有部分代码会返回 Promise。

```
1Math.random() > 0.52  ? new Promise((_, reject) => reject(2)) 3  : null
```

可能存在某些仅在运行时存在的值，可能来自用户输入或某些 API。在这种情况下，它仍然应该触发 lint 错误。此表达式的类型是 `Promise<unknown> | null`——这足以符合条件！

## 可选链回退

带回退的可选链可能产生未被处理的 Promise。与前两个例子类似，你可以用任意方式触发“短路”，本例使用了可选链。

```
1const optionalObject: Record<string, (() => unknown) | undefined> = {}2optionalObject?.nonExistentMethod?.() || new Promise((_, reject) => reject(2))
```

可选调用返回 `undefined`，因此 `||` 急切地计算右侧表达式，从而构造了一个拒绝状态的 Promise。由于该 Promise 仅存在于表达式位置（从未被赋值、等待或捕获），这个拒绝状态就悄无声息地隐藏在看似无害的回退逻辑背后。

## 立即调用函数

曾几何时，每个 JavaScript 开发者都必须了解**立即调用函数表达式**（IIFE，读作“if-y”）


用函数调用包裹并不会改变 Promise 未被处理的事实。

## 逗号运算符

最后，我们来看晦涩的**逗号运算符**。如果你曾查看过压缩后的 JavaScript 代码，很可能见过它的实际应用。
