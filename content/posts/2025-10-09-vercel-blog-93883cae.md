# 使用 GitHub Actions 阻止 Vercel 部署升级

## 作者

![](/images/posts/9f4a6b9a3584.jpg)

![](/images/posts/1e40f9e66538.jpg)

您现在可以阻止部署被升级到生产环境，直到选定的 GitHub Actions 成功完成。

在 Vercel 上，每次部署都始于预览环境，此功能确保只有通过测试或其他自动化检查的已验证构建才会发布到生产环境。

部署检查适用于所有连接到 GitHub 仓库的项目。

请在您的项目设置中配置此功能，或查阅文档了解更多信息。
