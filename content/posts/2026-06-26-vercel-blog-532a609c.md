您现在可以在 Vercel 仪表盘中查看 eve（开源 Agent 框架）的 Agent 运行记录。每个 eve 项目都会自动显示 Agent 运行标签页，一目了然地呈现每次会话的触发器、持续时间和 Token 用量。深入查看任何一次运行，即可检查对话中的每一轮交互、模型调用和工具调用。此前消失在函数日志中的运行时错误，现在会关联到出错的步骤。

两种视图读取相同的数据：

- 开发者模式：显示原始工具名称、输入和输出 JSON，以及工程师调试故障时所需的每一步 Token 计数。
- 业务模式：将工具名称人性化，隐藏 JSON，并生成通俗易懂的英文摘要，方便非技术审核人员审查 Agent 的行为。

开发者模式：显示原始工具名称、输入和输出 JSON，以及工程师调试故障时所需的每一步 Token 计数。

业务模式：将工具名称人性化，隐藏 JSON，并生成通俗易懂的英文摘要，方便非技术审核人员审查 Agent 的行为。

运行数据默认加密。数据保留时长随套餐变化：

- Hobby：12 小时
- Pro：1 天
- Enterprise：3 天

Hobby：12 小时

Pro：1 天

Enterprise：3 天

Pro 和 Enterprise 团队可通过 Observability Plus 将保留时长延长至 30 天。

使用自定义 OpenTelemetry 后端的团队，仍可从 `agent/instrumentation.ts` 将 AI SDK 跨度导出到任何目标，例如 Braintrust 或 Datadog。

打开项目中的 Agent 运行记录，并阅读文档以开始使用。
