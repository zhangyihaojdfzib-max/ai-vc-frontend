# Groq 正式成为 Hugging Face 推理服务提供商 🔥

我们非常高兴地宣布，**Groq** 现已成为 Hugging Face Hub 上支持的推理服务提供商！
Groq 加入了我们不断壮大的生态系统，直接在 Hub 的模型页面上增强了无服务器推理的广度和能力。推理服务提供商也已无缝集成到我们的客户端 SDK（包括 JS 和 Python 版本）中，让您能够轻松地使用首选提供商提供的各种模型。

**Groq** 支持多种文本和对话模型，包括最新的开源模型，例如 **Meta 的 Llama 4**、**Qwen 的 QWQ-32B** 等等。

Groq 技术的核心是语言处理单元（LPU™），这是一种新型的端到端处理单元系统，专为具有顺序计算组件、计算密集型的应用（如大语言模型）提供最快的推理速度。LPU 旨在克服 GPU 在推理方面的局限性，提供显著更低的延迟和更高的吞吐量。这使其成为实时 AI 应用的理想选择。

Groq 为公开可用的模型提供快速的 AI 推理服务。他们提供了一个 API，允许开发者轻松地将这些模型集成到自己的应用程序中。它采用按需、即用即付的模式，让用户能够访问广泛的公开可用 LLM。

现在，您可以在 Hugging Face 上将 Groq 的推理 API 用作推理服务提供商。我们非常期待看到您将利用这个新提供商构建出怎样的应用。

## 工作原理

### 在网站用户界面中

1.  在您的用户账户设置中，您可以：
    *   为您已注册的提供商设置您自己的 API 密钥。如果未设置自定义密钥，您的请求将通过 HF 路由。
    *   按偏好排序提供商。这适用于模型页面中的小部件和代码片段。

![推理服务提供商](/images/posts/ffb2ed6fa1d1.png)

2.  如前所述，调用推理服务提供商时有两种模式：
    *   自定义密钥（使用您自己的对应推理服务提供商的 API 密钥，请求直接发送给推理服务提供商）
    *   由 HF 路由（在这种情况下，您不需要提供商的令牌，费用将直接计入您的 HF 账户，而非提供商的账户）

![推理服务提供商](/images/posts/d2717fd3c654.png)

3.  模型页面会展示第三方推理服务提供商（与当前模型兼容的提供商，按用户偏好排序）

### 通过客户端 SDK

#### 使用 Python 和 huggingface_hub

以下示例展示了如何使用 Groq 作为推理服务提供商来调用 Meta 的 Llama 4。您可以使用 **Hugging Face 令牌**通过 Hugging Face 自动路由，或者如果您有的话，也可以使用您自己的 Groq API 密钥。

从源代码安装 `huggingface_hub`（参见**说明**）。官方支持将在即将发布的 v0.33.0 版本中提供。

```python
import os
from huggingface_hub import InferenceClient

client = InferenceClient(
    provider="groq",
    api_key=os.environ["HF_TOKEN"],
)

messages = [
    {
        "role": "user",
        "content": "What is the capital of France?"
    }
]

completion = client.chat.completions.create(
    model="meta-llama/Llama-4-Scout-17B-16E-Instruct",
    messages=messages,
)

print(completion.choices[0].message)

```

#### 使用 JS 和 @huggingface/inference

```js
import { InferenceClient } from "@huggingface/inference";

const client = new InferenceClient(process.env.HF_TOKEN);

const chatCompletion = await client.chatCompletion({
  model: "meta-llama/Llama-4-Scout-17B-16E-Instruct",
  messages: [
    {
      role: "user",
      content: "What is the capital of France?",
    },
  ],
  provider: "groq",
});

console.log(chatCompletion.choices[0].message);

```

## 计费

对于直接请求，即当您使用推理服务提供商的密钥时，您将由相应的提供商计费。例如，如果您使用 Groq API 密钥，费用将计入您的 Groq 账户。

对于路由请求，即当您通过 Hugging Face Hub 进行身份验证时，您只需支付标准的提供商 API 费率。我们不会加收额外费用，我们只是直接将提供商的成本转给您。（未来，我们可能会与我们的提供商合作伙伴建立收入分成协议。）
