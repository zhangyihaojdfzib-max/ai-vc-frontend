Sanity现已作为原生CMS集成在Vercel市场上线。团队现在可以直接从Vercel仪表板安装、配置和管理Sanity，无需再手动设置API Token和配置环境变量。

此集成将CMS设置保留在您现有的Vercel工作流程中，无需为资源调配和账户管理使用单独的仪表板。

### 开始使用集成

定义您的内容模式，设置客户端，并开始获取内容。模式通过代码定义您的内容结构，指定文档类型及其字段。

```
1import { defineField, defineType } from "sanity";2
3export const postType = defineType({4  name: "post",5  title: "Post",6  type: "document",7  fields: [8    defineField({ name: "title", type: "string" }),9    defineField({ name: "slug", type: "slug", options: { source: "title" } }),10    defineField({ name: "body", type: "array", of: [{ type: "block" }] }),11  ],12});
```

定义一个包含标题、slug和富文本正文字段的文章文档类型

在索引文件中注册您的模式类型，以便Sanity可以加载它们。

```
1import { postType } from "./postType";2
3export const schemaTypes = [postType];
```

导出所有模式类型供Sanity Studio使用

Sanity客户端将您的应用程序连接到您的内容。市场集成会自动将项目ID配置为环境变量。

```
1import { createClient } from "next-sanity";2
3export const client = createClient({4  projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID,5  dataset: "production",6  apiVersion: "2024-01-01",7  useCdn: false,8});
```

创建一个使用您项目环境变量配置的可复用客户端

配置好客户端后，您可以使用GROQ（图形关系对象查询）获取内容，这是Sanity的查询语言，用于精确请求您所需的字段。

```
1import { client } from "@/sanity/lib/client";2
3const POSTS_QUERY = `*[_type == "post"] | order(publishedAt desc)[0...12]{4  _id, title, slug, publishedAt5}`;6
7export default async function HomePage() {8  const posts = await client.fetch(POSTS_QUERY);9  return (10    <ul>11      {posts.map((post) => (12        <li key={post._id}>{post.title}</li>13      ))}14    </ul>15  );16}
```

获取最近的12篇文章并将其渲染为列表

这就是从安装到获取内容所需的全部步骤。从Vercel市场安装Sanity开始使用，或者部署Next.js + Sanity个人网站模板从一个可运行的示例开始。
