# 用 Gradio 的 gr.HTML 一键构建任何 Web 应用

Gradio 6 悄然发布了一项非常强大的功能：`gr.HTML` 现在支持自定义模板、作用域 CSS 和 JavaScript 交互。这意味着你可以构建几乎任何 Web 组件——而 Claude（或任何其他前沿 LLM）可以一次性生成整个组件：前端、后端和状态管理，全部在一个 Python 文件中。

我们通过构建不同类型的应用测试了这项功能。每个应用都是一个单独的 Python 文件，无需构建步骤，可在几秒钟内部署到 Hugging Face Spaces。

## 效率应用

**番茄钟计时器**：一个专注计时器，当你工作时，像素艺术树会随之生长。从种子开始，长出枝干，生出树叶。完成一个专注会话，这棵树就会加入你的森林。会话跟踪、主题切换、休息模式——全部可交互，全部在一个文件中。

仅树形动画通常就需要一个单独的 React 组件。在这里，它只是 `css_template` 中的 CSS 关键帧和 `js_on_load` 中的状态更新。

## 商业应用

**GitHub 贡献热力图**：点击任意单元格切换贡献记录。多种颜色主题。模式生成器（连续贡献、季节性、随机）。编辑时实时更新的统计数据。

**看板**：列与列之间完全支持拖放。内联编辑（双击任意卡片）。可实时过滤的搜索功能。可折叠的列。

拖放功能通常意味着引入一个库。在这里，它是通过 `js_on_load` 连接的原生 HTML5 拖放事件，状态通过 `trigger('change')` 同步回 Python。

## 创意应用

**幸运转盘**：流畅的 CSS 动画，跨重新渲染保持旋转状态。预设配置用于是/否决策、餐厅选择、团队挑选。你还可以动态添加自定义旋转扇区。

## 机器学习应用

对于机器学习工作，`gr.HTML` 在这里变得非常有趣：你可以构建能够处理你确切输出格式的专用组件，然后像使用任何内置的 Gradio 组件一样使用它们。

**检测结果查看器**：一个用于目标检测、实例分割和姿态估计结果的自定义查看器。渲染边界框、分割掩码、关键点和骨架连接——全部在一个可重用的 `gr.HTML` 子类中，可以直接插入你的模型流水线。

社区也用 gr.HTML 构建了一些创意组件：

**用于图像编辑的 3D 相机控制**：Gradio 应用内的完整 Three.js 视口！拖动手柄控制方位角、仰角和距离。你上传的图像出现在 3D 场景中，相机参数直接输入到 Qwen 最新的图像编辑模型中。这类交互式 3D 控件通常需要一个独立的前端——而在 Gradio 中，它只是一个 `gr.HTML` 子类 🔥

**实时语音转录**：使用 Mistral 的 Voxtral 模型进行实时转录。转录显示是一个自定义的 `gr.HTML` 组件，带有动画状态徽章、实时 WPM 计数器，以及在你说话时更新的样式化输出。无需使用 React 即可实现实时 UI 反馈！

## 工作原理

每个 `gr.HTML` 组件接受三个模板：

```python
gr.HTML(
    value={"count": 0},
    html_template="<button>Clicked ${value.count} times</button>",
    css_template="button { background: #667eea; color: white; }",
    js_on_load="""
        element.querySelector('button').onclick = () => {
            props.value = { count: props.value.count + 1 };
            trigger('change');
        };
    """
)
```

`${value}` 注入 Python 状态。`props.value` 从 JavaScript 更新它。`trigger('change')` 同步回 Python。这就是整个 API。

对于可重用组件，可以子类化 `gr.HTML`：

```python
class Heatmap(gr.HTML):
    def __init__(self, value=None, theme="green", **kwargs):
        super().__init__(
            value=value,
            theme=theme,
            html_template=TEMPLATE,
            css_template=STYLES,
            js_on_load=SCRIPT,
            **kwargs
        )
```

现在 `Heatmap()` 就像 `gr.Image()` 或 `gr.Slider()` 一样工作——在 Blocks 中使用它，连接事件处理程序，满足你的任何需求。

## 这对氛围编程为何重要

当你要求你的 LLM 构建一个自定义组件时，单文件输出至关重要。没有“现在创建样式文件”或“将其连接到你的构建配置”。只有一个可以立即运行的 Python 文件。

反馈循环变为：描述你想要什么 → 获取代码 → `gradio app.py` → 看到它运行 → 描述需要修复什么 → 重复。在 Gradio 的重载模式下，每个循环只需几秒钟。

使用 `gradio deploy` 部署到 Spaces，或使用 `demo.launch(share=True)` 分享临时链接。从想法到实时应用，只需几秒钟。

Gradio 自带 32 个交互式组件，但有时你完美的 AI Web 应用需要一些特殊的东西。这就是 `gr.HTML` 的用武之地。

如果你有想法，试试用 `gr.HTML` 构建它：向你的 LLM 描述你想要什么，生成代码，运行它。你可能会惊讶于在 5 分钟内能做出什么。
