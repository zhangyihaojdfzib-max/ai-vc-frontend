# 通过API微调FLUX.1

- zeke

你现在可以在Replicate上使用快速的FLUX训练器来微调模型。它速度快（不到2分钟），成本低（不到2美元），并且会提供一个可直接运行的温模型以及可供下载的LoRA权重。

FLUX.1是当下最热门的技术，这并非没有原因。它是一个快速、强大的图像生成模型，易于使用和微调，并能生成令人惊叹的图像。

上周，我们为您带来了一个使用人脸微调Flux的指南。该指南使用完全基于网络的工作流来创建微调后的Flux模型，无需编写任何代码。

我们听到一些用户反馈，他们希望通过API来微调Flux，因此本周我们带来了另一个教程，向您展示具体操作方法。

在本指南中，您将使用Replicate的HTTP API以编程方式创建并运行您自己微调后的Flux模型。

## 步骤0：先决条件

开始前，您需要准备以下内容：

*   Replicate账户
*   少量训练图像
*   用于训练成本的2-3美元小额预算
*   cURL，这款备受喜爱的命令行工具，自20世纪90年代以来一直用于发起HTTP请求

## 步骤1：收集训练图像

您需要准备几张自己的图像来开始。

您可以用少至两张训练图像来微调Flux，但为了获得最佳效果，建议至少使用10张或更多图像。理论上，随着训练数据中包含更多图像，您会持续获得更好的结果，但添加的图像越多，训练过程可能耗时越长。

收集训练图像时，请考虑以下几点：

*   支持WebP、JPG和PNG格式。
*   如果可能，请使用1024x1024或更高分辨率。
*   文件名无关紧要。可以随意命名文件。
*   图像可以有不同的宽高比；它们不必都是正方形、横向或纵向。
*   10张图像是一个不错的起始最小值。

收集好图像后，将它们放入一个zip文件中。假设您将它们全部放在一个名为`data`的文件夹中，运行以下命令来生成一个名为`data.zip`的文件：

```
zip -r data.zip data
```

## 步骤2：在环境中设置API令牌

您需要一个API令牌来向Replicate API发送请求。

访问`replicate.com/account/api-tokens`创建一个新的API令牌，然后将其复制到剪贴板。

大多数Replicate工具，如客户端库和Replicate CLI，都遵循一个惯例：在名为`REPLICATE_API_TOKEN`的环境变量中查找API令牌。

在终端中运行以下命令来设置`REPLICATE_API_TOKEN`环境变量：

```
export REPLICATE_API_TOKEN="r8_..."
```

提示：如果您打算在自己的计算机上使用cURL命令或代码发送大量API请求，您可能希望将`REPLICATE_API_TOKEN`环境变量设置在您的shell配置文件或点文件中，这样每次打开新的终端窗口时就不必重新输入。

## 步骤3：创建目标模型

接下来，您将在Replicate上为训练好的模型创建一个空模型。训练完成后，模型将作为一个新版本推送到此模型中。

您可以在您的个人账户下创建模型，或者如果您希望与团队或其他协作者共享访问权限，可以在组织中创建。

在Replicate上创建模型有几种方法，例如使用Replicate Web UI或Replicate CLI，但在本指南中，我们将通过向`models.create` API端点发送cURL请求来创建模型。

为您的模型选择一个描述性的名称，例如`flux-my-cool-finetune`或`flux-my-dog-fluffy`。Flux模型的一个流行惯例是在名称中包含`flux`，但这不是必需的。

在终端中运行以下命令来创建模型，将`your-username`和`your-model-name`替换为正确的值：

```
curl -s -X POST \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"owner": "your-username", "name": "your-model-name", "description": "An example model", "visibility": "public", "hardware": "gpu-a40-large"}' \
  https://api.replicate.com/v1/models
```

## 步骤4：上传训练数据

接下来，您需要将zip文件上传到互联网上可公开访问的某个地方，例如S3存储桶或GitHub Pages站点。

您也可以使用Replicate的Files API上传训练数据。以下是一个使用cURL的示例，假设您将训练数据文件命名为`data.zip`：

```
curl -s -X POST "https://api.replicate.com/v1/files" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "content=@data.zip;type=application/zip;filename=data.zip"
```

输出将是一个包含上传文件URL的JSON响应：

```
{"id":"MThjNTQwOTEtNDJmNS00Mjc2LWIzMTUtMzczMTNmNzYyYTEw","name":"data.zip","content_type":"application/zip","size":1431986,"etag":"9c5b5aa1178bd843722a8cce85ba778b","checksums":{"sha256":"9d0efe1c32d02fd8a0b01af67bea357d7279522aff8b4158a37529abe4713103","md5":"9c5b5aa1178bd843722a8cce85ba778b"},"metadata":{},"created_at":"2024-09-09T20:17:37.031Z","expires_at":"2024-09-10T20:17:37.031Z","urls":{"get":"https://api.replicate.com/v1/files/MThjNTQwOTEtNDJmNS00Mjc2LWIzMTUtMzczMTNmNzYyYTEw"}}
```

在该输出中找到以`https://api.replicate.com/v1...`开头的URL，并将其复制到剪贴板。您将在下一步中将其用作训练过程的输入。

提示：如果您安装了`jq`命令行JSON处理器，可以像这样一步上传文件并输出URL：

```
curl -s -X POST "https://api.replicate.com/v1/files" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "content=@data.zip;type=application/zip;filename=data.zip" | jq -r '.urls.get'
```

## 步骤5：启动训练过程

现在您已将训练数据上传到一个可公开访问的URL，下一步是使用API启动训练过程。

训练过程运行的时间将按秒计费。Flux模型的训练在Nvidia H100 GPU硬件上运行，撰写本文时成本为每秒0.001528美元。对于一次20分钟的训练（使用约20张训练图像和1000步时的典型情况），预计需要支付约1.85美元。模型训练完成后，您可以像使用任何其他Replicate模型一样通过API运行它，并且仅需为生成图像所需的计算时间付费。

训练过程有许多输入参数，如训练步数、LoRA秩、学习率等，但您只需要明确设置其中两个：

*   `input_images`：您之前上传的训练数据zip文件的URL。
*   `trigger_word`：一个唯一的字符串，例如`CYBRPNK3000`，它不是任何语言中的单词或短语。有关如何选择一个好的触发词的更多详细信息，请参阅我们上一篇关于人脸微调Flux的指南。

现在是用cURL发起训练请求的时候了，将`your-username`和`your-model-name`替换为正确的值，同时替换`input_images`和`trigger_word`输入：

```
curl -X POST \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
        "destination": "your-username/your-model-name",
        "input": {
            "input_images": "<your-training-data-url>",
            "trigger_word": "<some-unique-string>"
        }
    }' \
  https://api.replicate.com/v1/models/ostris/flux-dev-lora-trainer/versions/d995297071a44dcb72244e6c19462111649ec86a9646c32df56daa7f14801944/trainings
```

## 步骤 7：在网页上生成图像

训练过程完成后，您的模型就可以运行了。最简单的方法是直接在网页上运行。

您唯一需要输入的参数是**提示词**。开始时，其他参数可以保持默认。Flux 模型非常擅长遵循长提示词，因此提示词越详细、描述性越强，效果就越好。

请务必在提示词中包含您的**触发词**，以便在生成的图像中激活您新训练的概念。在提示词中加入触发词，相当于告诉模型：“嘿，你应该把输出重点放在我们训练过的内容上”。假设您有一个模型，可以生成**《我的世界》电影风格**的图像，触发词是 `MNCRFTMOV`。您的提示词可能是这样的：“a MNCRFTMOV film render of a blocky weird toad, minecraft style”。您可以尝试不同的提示词，但请始终包含触发词。

## 步骤 8：使用 API 生成图像

网页版交互界面是开始使用新模型的好地方，但每次点击生成一张图像很快就会变得乏味。幸运的是，您的模型也通过 API 托管在云端，因此您可以使用自己选择的编程语言，在自己的代码中运行它。

当您运行一个模型时，会看到针对不同语言（如 Node.js 和 Python）的标签页。这些标签页包含代码片段，展示了如何构建 API 调用来复现您刚刚在浏览器表单中输入的确切参数。

点击网页交互界面中的 `Node.js` 标签页查看 API 代码：

这将显示您自行运行模型所需的确切设置步骤和代码片段。以下是一个简化的 Node.js 代码示例（使用触发词 `ZIKI`）供您参考：

```
import Replicate from "replicate"
const replicate = new Replicate()
const model = "zeke/ziki-flux:dadc276a9062240e68f110ca06521752f334777a94f031feb0ae78ae3edca58e"
const prompt = "ZIKI, an adult man, standing atop Mount Everest at dawn..."
const output = await replicate.run(model, { input: { prompt } })
console.log(output)
```

## 步骤 9：使用语言模型来撰写更好的提示词

有时，从头构思一个好的提示词很困难，而使用像“ZIKI 穿着高领假日毛衣”这样非常简单的提示词，不会产生非常有趣的结果。

这时，语言模型就可以派上用场了。以下是一个语言模型提示词示例，可以帮助激发一些有趣的图像生成提示词创意：

为一个图像生成模型撰写十个提示词。这些提示词应描述一个名为 ZIKI 的虚构人物在各种场景中的情况。确保在每个提示词中都使用全大写的单词 ZIKI。使提示词高度详细且有趣，并在主题上多样化。确保提示词能生成包含清晰面部细节的图像。ZIKI 是一名 43 岁的成年男性。在提示词中包含一些对此的提及，以避免错误描述 ZIKI 的年龄或性别。不要暗示 ZIKI 的眼睛颜色。
