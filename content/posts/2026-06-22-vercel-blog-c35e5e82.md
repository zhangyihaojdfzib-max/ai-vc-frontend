您现在可以直接从 Vercel CLI 为 Vercel Blob 生成签名 URL。签名 URL 是一种具有设定过期时间的作用域 URL，允许您对特定对象执行单一操作。每个 URL 限定于一个操作（get、head、put 或 delete）、一个路径名，以及最长 7 天的自定义过期时间。请将 Vercel CLI 更新至 5.14.5 版本以开始使用。

### 链接到标题 通过一条命令生成预签名 URL

使用新的 `vercel blob presign` 命令为单一操作签名 URL，例如为私有文件提供短期访问权限，或允许您的用户上传特定大小和内容类型的图片。默认情况下，它会签发一个签名的 GET URL：

```
vercel blob presign media/photo.jpg --access private
```

将其限定为带有内容类型和大小限制的上传操作：

```
vercel blob presign uploads/image.jpg --access private --operation put --allowed-content-type image/* --maximum-size-in-bytes 10485760
```

通过 `--valid-for`（例如 15m、1h、7d）或绝对时间戳 `--valid-until` 控制 URL 的有效时长。传递 `--json` 可将结果、操作和过期时间以结构化输出形式获取，便于脚本处理。

### 链接到标题 签发独立的签名令牌

当您希望将签名令牌生成与 URL 生成分离时，使用 `vercel blob signed-token`，例如在您的 Agent（智能体）调查问题时限制对私有 blob 存储的写入访问。

```
vercel blob signed-token --pathname media/photo.jpg --operation get
vercel blob signed-token --operation put --allowed-content-type image/* --maximum-size-in-bytes 10485760
```

然后，您可以将生成的委托令牌和客户端签名令牌输入到 `vercel blob presign` 中：

```
vercel blob presign uploads/image.jpg --access private --operation put --delegation-token <delegationToken> --client-signing-token <clientSigningToken>
```

更新至最新版本的 Vercel CLI 并运行 `vercel blob presign --help` 查看所有可用选项。在文档中了解更多关于 Vercel Blob 签名 URL 的信息。
