运行时日志中的“包含类型”筛选器已被两个新的筛选器取代，以提高清晰度：

1. 资源：筛选 Vercel Edge 网络中用于处理请求的基础设施资源。示例包括无服务器函数、Edge 缓存和 Edge 中间件
2. 请求类型：筛选请求所使用的框架定义机制或渲染策略。示例包括 API 路由、增量静态再生（ISR）和定时任务

资源：筛选 Vercel Edge 网络中用于处理请求的基础设施资源。示例包括无服务器函数、Edge 缓存和 Edge 中间件

请求类型：筛选请求所使用的框架定义机制或渲染策略。示例包括 API 路由、增量静态再生（ISR）和定时任务

这些更新提供了关于请求处理方式的更细粒度洞察。从今天起，所有套餐均可使用这两个筛选器。
