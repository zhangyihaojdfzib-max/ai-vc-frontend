Node.js 运行时的中间件支持现已可用，通过熟悉的 API 为身份验证、个性化等功能提供完整的 Node.js 支持。

无论使用何种运行时，中间件将继续在 Vercel 上全球部署。我们首先在 Next.js 15.2 中发布了对 Node.js 中间件的支持。

此实验性功能需要 Next.js canary 频道。升级到 `next@canary` 并在配置中启用 `nodejsexperimental` 标志以使用它：

```
1import type { NextConfig } from "next";2
3const nextConfig: NextConfig = {4  experimental: {5    nodeMiddleware: true, 6  },7};8
9export default nextConfig;
```

你还必须在中间件文件中指定 Node.js 运行时：

```
1import type { NextRequest } from "next/server";2
3export function middleware(request: NextRequest) {4  5}6
7export const config = {8  runtime: "nodejs", 9};
```

立即使用 Next.js 15.2 部署。
