## 数据集SQL控制台介绍

在每个数据集上，您应该会看到一个新的"SQL控制台"徽章。只需点击一下，即可打开SQL控制台来查询该数据集。

所有工作都在浏览器中完成，控制台附带一些便捷功能：

- **100%本地**：SQL控制台由DuckDB WASM驱动，因此您无需任何依赖即可查询数据集。
- **完整DuckDB语法**：DuckDB支持完整的SQL语法，并内置了许多函数，用于正则表达式、列表、JSON、嵌入/向量等。您会发现DuckDB语法与PostgreSQL非常相似。
- **导出结果**：您可以将查询结果导出为parquet格式。
- **可分享**：您可以通过链接分享公共数据集的查询结果。

## 工作原理

### Parquet转换

Hugging Face上的大多数数据集以Parquet格式存储，这是一种针对性能和存储效率优化的列式数据格式。Hugging Face上的数据集查看器和SQL控制台直接从数据集的Parquet文件中加载数据。如果数据集是其他格式，前5GB会自动转换为Parquet。您可以在数据集查看器Parquet API文档中找到有关Parquet转换过程的更多信息。

SQL控制台使用Parquet文件，根据您的数据集拆分和配置创建可供查询的视图。

### DuckDB WASM 🦆

DuckDB WASM是驱动SQL控制台的引擎。它是一个在浏览器中基于Web Assembly运行的内进程数据库引擎，无需服务器或后端。

通过仅在浏览器中运行，它为用户提供了最大的灵活性，可以随心所欲地查询数据，无需任何依赖。同时，通过简单的链接分享可重现的结果也变得非常简单。

您可能会问："它适用于大型数据集吗？"答案是："是的！"

与任何技术一样，也存在一些限制。

- SQL控制台适用于许多查询。但是，内存限制约为3GB，因此可能会耗尽内存而无法处理查询（提示：尝试使用筛选器来减少查询的数据量，并配合使用LIMIT）。
- 虽然DuckDB WASM非常强大，但它与DuckDB的功能并不完全一致。例如，DuckDB WASM尚不支持用于查询数据集的`hf://`协议。

### 示例：将数据集从Alpaca格式转换为对话格式

现在我们已经介绍了SQL控制台，让我们探索一个实际示例。在微调大语言模型时，您通常需要处理不同的数据格式。一种特别流行的格式是对话格式，其中每一行代表用户和模型之间的多轮对话。SQL控制台可以帮助我们高效地将数据转换为这种格式。让我们看看如何使用SQL将Alpaca数据集转换为对话格式。

通常，开发人员会通过Python预处理步骤来处理此任务，但我们可以展示如何使用SQL控制台在不到30秒内完成相同的操作。

在上面的数据集中，点击"SQL控制台"徽章打开SQL控制台。您应该会看到以下查询自动填充。

准备好后，点击"运行查询"按钮执行查询。

```sql

WITH 
source_view AS (
  SELECT * FROM train  
)
SELECT 
  [
    struct_pack(
      "from" := 'user',
      "value" := CASE 
                   WHEN input IS NOT NULL AND input != '' 
                   THEN instruction || '\n\n' || input
                   ELSE instruction
                 END
    ),
    struct_pack(
      "from" := 'assistant',
      "value" := output
    )
  ] AS conversation
FROM source_view
WHERE instruction IS NOT NULL 
AND output IS NOT NULL;

```

在查询中，我们使用`struct_pack`函数为每个对话创建一个新的STRUCT行。

DuckDB有关于STRUCT数据类型和函数的优秀文档。您会发现许多数据集包含带有JSON数据的列。DuckDB提供了轻松解析和查询这些列的函数。
