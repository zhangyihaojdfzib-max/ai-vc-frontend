# 使用 Arize Phoenix 追踪与评估您的 Agent（智能体）

那么，您已经构建了自己的 Agent（智能体）。它能接收输入和工具，进行处理，并生成响应。也许它正在做决策、检索信息、自主执行任务，或者三者兼而有之。但现在关键问题来了——它的运行效率如何？更重要的是，您如何知道？

构建一个 Agent（智能体）是一回事；理解其行为是另一回事。这就是追踪和评估发挥作用的地方。追踪让您能够精确地看到您的 Agent（智能体）每一步在做什么——它接收了什么输入，如何处理信息，以及如何得出最终输出。可以把它想象成对 Agent（智能体）决策过程进行 X 光透视。同时，评估帮助您衡量性能，确保您的 Agent（智能体）不仅功能正常，而且确实有效。它是否产生了正确的答案？每一步的发现相关性如何？Agent（智能体）的响应构建得有多好？它是否符合您的目标？

Arize Phoenix 提供了一个集中式平台，让您能够实时追踪、评估和调试 Agent（智能体）的决策——全部集中在一个地方。我们将深入探讨如何实施这些功能来优化和改进您的 Agent（智能体）。因为构建只是开始——真正的智能来自于确切了解底层正在发生什么。

为此，让我们确保已经设置好了一个 Agent（智能体）！您可以按照以下步骤操作，或者使用您自己的 Agent（智能体）。

## 创建一个 Agent（智能体）

### 步骤 1：安装所需的库

```py
pip install -q smolagents

```

### 步骤 2：导入所有基本构建模块

现在让我们引入将要使用的类和工具：

```py
from smolagents import (
   CodeAgent,
   DuckDuckGoSearchTool,
   VisitWebpageTool,
   HfApiModel,
)

```

### 步骤 3：设置我们的基础模型

我们将创建一个由 Hugging Face Hub Serverless API 驱动的模型实例：

```py
hf_model = HfApiModel()

```

### 步骤 4：创建工具调用 Agent（智能体）

```py
agent = CodeAgent(
    tools=[DuckDuckGoSearchTool(), VisitWebpageTool()],
    model=hf_model,
    add_base_tools=True
)

```

### 步骤 5：运行 Agent（智能体）

现在是神奇的时刻——让我们看看我们的 Agent（智能体）实际运行。我们向 Agent（智能体）提出的问题是："获取谷歌从 2020 年到 2024 年的股价，并据此创建一张折线图？"

```py
agent.run("fetch the share price of google from 2020 to 2024, and create a line graph from it?")

```

您的 Agent（智能体）现在将：

1. 使用 DuckDuckGoSearchTool 搜索谷歌的历史股价。
2. 可能使用 VisitWebpageTool 访问网页以查找该数据。
3. 尝试收集信息并生成或描述如何创建折线图。

## 追踪您的 Agent（智能体）

一旦您的 Agent（智能体）开始运行，下一个挑战就是理解其内部工作流程。追踪帮助您跟踪 Agent（智能体）采取的每一步——从调用工具到处理输入和生成响应——使您能够调试问题、优化性能，并确保其按预期运行。

要启用追踪，我们将使用 Arize Phoenix 进行可视化，并使用 OpenTelemetry + OpenInference 进行仪表化。

从 smolagents 安装遥测模块：

```py
pip install -q 'smolagents[telemetry]'

```

您可以通过多种不同的方式运行 Phoenix。以下命令将在您的机器上运行一个本地 Phoenix 实例：

```py
python -m phoenix.server.main serve

```

关于 Phoenix 的其他托管选项，您可以创建一个免费的 Phoenix 在线实例，在本地自行托管应用程序，或在 Hugging Face Spaces 上托管应用程序。

启动后，我们注册一个追踪器提供程序，指向我们的 Phoenix 实例。

```py
from phoenix.otel import register
from openinference.instrumentation.smolagents import SmolagentsInstrumentor

tracer_provider = register(project_name="my-smolagents-app") 
SmolagentsInstrumentor().instrument(tracer_provider=tracer_provider) 

```

现在，任何对 smolagents 的调用都将被发送到我们的 Phoenix 实例。

现在追踪已启用，让我们用一个简单的查询来测试它：

```py
agent.run("What time is it in Tokyo right now?")

```

一旦 OpenInference 与 SmolAgents 设置完毕，每次 Agent（智能体）调用都会自动在 Phoenix 中被追踪。

## 评估您的 Agent（智能体）

一旦您的 Agent（智能体）启动并运行，并且其运行受到监控，下一步就是评估其性能。评估（evals）有助于确定您的 Agent（智能体）在检索、处理和呈现信息方面的表现如何。

有许多类型的评估可以运行，例如响应相关性、事实准确性、延迟等。请查阅 Phoenix 文档以深入了解不同的评估技术。

在这个例子中，我们将重点评估我们的 Agent（智能体）使用的 DuckDuckGo 搜索工具。我们将使用一个大语言模型（LLM）作为评判者来衡量其搜索结果的相关性——具体来说，是 OpenAI 的 GPT-4o。

### 步骤 1：安装 OpenAI

首先，安装必要的包：

```py
pip install -q openai

```

我们将使用 GPT-4o 来评估搜索工具的响应是否相关。这种方法被称为 LLM-as-a-judge（大语言模型作为评判者），利用语言模型对响应进行分类和评分。

### 步骤 2：检索工具执行跨度

为了评估 DuckDuckGo 检索信息的效果，我们首先需要提取调用该工具的执行轨迹。

```py
from phoenix.trace.dsl import SpanQuery
import phoenix as px
import json

query = SpanQuery().where(
    "name == 'DuckDuckGoSearchTool'",
).select(
    input="input.value", 
    reference="output.value", 
)


tool_spans = px.Client().query_spans(query, project_name="my-smolagents-app")

tool_spans["input"] = tool_spans["input"].apply(lambda x: json.loads(x).get("kwargs", {}).get("query", ""))
tool_spans.head()

```

### 步骤 3：导入提示词模板

接下来，我们加载 RAG 相关性提示词模板，它将帮助 LLM 分类搜索结果是否相关。

```py
from phoenix.evals import (
    RAG_RELEVANCY_PROMPT_RAILS_MAP,
    RAG_RELEVANCY_PROMPT_TEMPLATE,
    OpenAIModel,
    llm_classify
)
import nest_asyncio
nest_asyncio.apply()

print(RAG_RELEVANCY_PROMPT_TEMPLATE)

```

### 步骤 4：运行评估

现在，我们使用 GPT-4o 作为评判者来运行评估：

```py
from phoenix.evals import (
    llm_classify,
    OpenAIModel,
    RAG_RELEVANCY_PROMPT_TEMPLATE,
)

eval_model = OpenAIModel(model="gpt-4o")

eval_results = llm_classify(
    dataframe=tool_spans,
    model=eval_model,
    template=RAG_RELEVANCY_PROMPT_TEMPLATE,
    rails=["relevant", "unrelated"],
    concurrency=10,
    provide_explanation=True,
)
eval_results["score"] = eval_results["explanation"].apply(lambda x: 1 if "relevant" in x else 0)

```

这里发生了什么？

- 我们使用 GPT-4o 分析搜索查询（输入）和搜索结果（输出）。
- LLM 根据提示词对结果进行分类，判断是相关还是不相关。
- 我们分配一个二进制分数（1 = 相关，0 = 不相关）用于进一步分析。

要查看您的结果：

```py
eval_results.head()

```

### 步骤 5：将评估结果发送到 Phoenix

```py
from phoenix.trace import SpanEvaluations

px.Client().log_evaluations(SpanEvaluations(eval_name="DuckDuckGoSearchTool Relevancy", dataframe=eval_results))

```

通过这种设置，我们现在可以系统地评估我们 Agent（智能体）中 DuckDuckGo 搜索工具的有效性。使用 LLM-as-a-judge（大语言模型作为评判者），我们可以确保我们的 Agent（智能体）检索到准确且相关的信息，从而带来更好的性能。使用本教程，任何评估都很容易设置——只需将 RAG_RELEVANCY_PROMPT_TEMPLATE 替换为适合您需求的不同提示词模板即可。Phoenix 提供了各种预编写和预测试的评估模板，涵盖忠实度、响应连贯性、事实准确性等领域。请查阅 Phoenix 文档以探索完整列表，并为您的 Agent（智能体）找到最合适的模板！
