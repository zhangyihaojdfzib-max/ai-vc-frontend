# Hugging Face 与 Cloudflare 合作，通过 FastRTC 实现实时语音与视频无缝体验

我们很高兴宣布 Cloudflare 与 Hugging Face 达成全新合作，为 FastRTC 开发者提供使用 Hugging Face Token 即可即时接入企业级 WebRTC 基础设施的能力。

作为 FastRTC 与 Cloudflare 所能构建成果的预览，欢迎体验这款基于 Meta 全新 Llama 4 模型构建的语音聊天应用！

## 填补 AI 开发者工具箱中的空白

随着对话式 AI 成为工具、产品及服务的核心交互界面，实时通信基础设施对于支持自然的多模态交互愈发关键。
Hugging Face 构建了 FastRTC，通过抽象化 WebRTC（实时通信领域的黄金标准技术）的复杂性，让 AI 开发者仅需少量 Python 代码即可构建低延迟的 AI 驱动音频与视频流。

基于 WebRTC 的应用常因需要专用 TURN 服务器（用于在不同网络环境下建立可靠连接）而面临部署挑战。为解决此问题，Cloudflare 已构建覆盖全球超过 335 个地点的 TURN 服务器网络。

此次合作将 FastRTC 的简易开发方式与 Cloudflare 的全球 TURN 网络相结合，确保开发者能够创建具备全球连接能力的快速可靠 WebRTC 应用。

## 使用 Hugging Face 账户免费接入

持有有效 Hugging Face Access Token 的 FastRTC 开发者，每月可免费传输 10GB 数据，无需绑定信用卡。达到月度限额后，开发者可切换至自己的 Cloudflare 账户以获取更高容量（操作指南）。

## 这对 AI 开发者为何重要

此次合作对构建以下应用的 AI 开发者尤为有价值：

- 需要可靠低延迟音频流的语音助手
- 实时处理摄像头画面的视频分析应用
- 融合音频、视频与文本的多模态 AI 应用

通过本次合作，开发者可借助 FastRTC 专注于核心应用逻辑，同时无需构建和维护 TURN 基础设施。Cloudflare 的托管服务负责处理全球可扩展性与可靠性，使 AI 开发者无需承担基础设施维护开销即可交付卓越体验。

## 快速上手

该集成将在 FastRTC 0.0.20 及以上版本中可用。开始使用：

- 确保你拥有具备读取权限的 Hugging Face Token（在此获取）
- 安装或升级 FastRTC：`pip install --upgrade 'fastrtc[vad]'`
- 按如下方式配置你的 Stream 以使用 Cloudflare TURN 网络：

```python
from fastrtc import ReplyOnPause, Stream, get_cloudflare_turn_credentials
import os

os.environ["HF_TOKEN"] = "<your-hf-token>"

def echo(audio):
    yield audio

stream = Stream(ReplyOnPause(echo),
                rtc_config=get_cloudflare_turn_credentials)
stream.ui.launch()

```

使用 `python <你的脚本名称>.py` 运行脚本。

更多示例请参阅 Hugging Face 上的此合集以及 FastRTC Cookbook。
