## 链接到标题摘要

我们已发布 Next.js 的协同安全更新，解决了涉及拒绝服务、中间件与代理绕过、服务端请求伪造、缓存投毒以及跨站脚本攻击的 13 项安全公告。其中一项公告涉及上游 React 服务端组件漏洞，编号为 CVE-2026-23870。

### 链接到标题建议操作

React 和 Next.js 均已提供补丁版本，所有受影响用户应立即升级。

## 链接到标题影响

本次更新解决了以下安全公告：

### 链接到标题中间件与代理绕过

影响依赖 middleware.js 或 proxy.js 进行授权的应用程序。

- 高危：通过 App Router 分段预取 URL 绕过身份验证
- 高危：App Router 分段预取绕过（后续不完整修复）
- 高危：Pages Router i18n 默认语言路径绕过代理授权
- 高危：通过动态路由参数注入绕过
- 低危：中间件重定向可被缓存投毒

### 链接到标题拒绝服务

影响使用服务端函数、带缓存组件的部分预渲染或图片优化 API 的应用程序。

- 高危：React 服务端组件中的拒绝服务（上游跟踪为 CVE-2026-23870）
- 高危：使用缓存组件的应用程序因连接耗尽导致拒绝服务
- 中危：通过图片优化 API 导致的拒绝服务

### 链接到标题服务端请求伪造

影响处理 WebSocket 升级请求的应用程序。

- 高危：使用 WebSocket 升级的应用程序中的 SSRF

### 链接到标题缓存投毒

影响在 React 服务端组件响应前存在缓存层的应用程序。

- 中危：React 服务端组件响应中的缓存投毒
- 低危：通过 RSC 缓存破坏碰撞导致的缓存投毒

### 链接到标题跨站脚本攻击

影响在 App Router 中使用 CSP nonce 或使用消耗不受信任输入的 beforeInteractive 脚本的应用程序。

- 中危：使用 CSP nonce 的 App Router 应用程序中的 XSS
- 中危：使用不受信任输入的 beforeInteractive 脚本中的 XSS

## 链接到标题解决方案

这些漏洞已通过 React 和 Next.js 的补丁版本解决。打补丁是唯一完整的缓解措施，所有受影响用户应立即升级。

Vercel 未为此版本部署新的 WAF 规则；这些安全公告无法在 WAF 层可靠地拦截。

## 链接到标题受影响版本

| 软件包 | 受影响版本 | 升级至 |
|--------|------------|--------|
| Next.js 13.x, 14.x | 所有版本 | 15.5.18 或 16.2.6 |
| Next.js 15.x | <=15.5.17 | 15.5.18 |
| Next.js 16.x | <=16.2.5 | 16.2.6 |
| react-server-dom-* 19.0.x | <=19.0.5 | 19.0.6 |
| react-server-dom-* 19.1.x | <=19.1.6 | 19.1.7 |
| react-server-dom-* 19.2.x | <=19.2.5 | 19.2.6 |

## 链接到标题修复版本

- Next.js：15.5.18，16.2.6
- React：针对 react-server-dom-parcel、react-server-dom-webpack 和 react-server-dom-turbopack 软件包的 19.0.6、19.1.7、19.2.6

使用 react-server-dom-* 软件包的框架和打包工具应安装各自维护者提供的最新版本。
