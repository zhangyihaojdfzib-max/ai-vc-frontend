Vercel Flags 是 Vercel 平台内置的功能开关（Feature Flag）服务商。它允许您直接在 Vercel 仪表板中，通过目标规则、用户分群和环境控制来创建和管理功能开关。

Flags SDK 提供了一种与框架原生集成的方式，让您可以在 Next.js 和 SvelteKit 应用程序中定义和使用这些功能开关，并直接与您现有的代码库集成：

```
1import { vercelAdapter } from "@flags-sdk/vercel"2import { flag } from 'flags/next'; 3
4export const showNewFeature = flag({  5    key: 'show-new-feature',  6    decide: () => false,  7    description: 'Show the new dashboard redesign',8    adapter: vercelAdapter()9});
```

您可以在页面中像这样使用它们：

```
1import { showNewFeature } from '~/flags'; 2
3export default async function Page() {  4    const isEnabled = await showNewFeature();5
6    return isEnabled ? <NewDashboard /> : <OldDashboard />7;}
```

对于使用其他框架或自定义后端的团队，Vercel Flags 适配器支持 OpenFeature 标准，允许您整合跨不同系统的功能开关，并在功能开关管理方法上保持一致：

```
1import { OpenFeature } from '@openfeature/server-sdk';2import { VercelProvider } from '@vercel/flags-core/openfeature';3
45await OpenFeature.setProviderAndWait(new VercelProvider());6const client = OpenFeature.getClient();7
89const enabled = await client.getBooleanValue('show-new-feature');
```

Vercel Flags 的定价为每 100 万次功能开关请求 30 美元（每次事件 0.00003 美元）。功能开关请求是指任何读取底层功能开关配置的应用程序请求。一次请求即使评估了同一源项目的多个功能开关，仍计为一次功能开关请求。

Vercel Flags 目前处于测试阶段，所有计划的团队均可使用。
