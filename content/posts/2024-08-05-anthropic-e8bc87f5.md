# 规划人工智能问责制之路

![](/images/posts/7ef816994c34.jpg)

本周，Anthropic 向美国国家电信和信息管理局（NTIA）提交了一份关于《人工智能问责制意见征询》的回应。今天，我们希望分享我们的建议，因为它们体现了 Anthropic 在人工智能政策方面的部分核心主张。

目前，对于评估当今先进的人工智能系统，尚缺乏健全且全面的流程，更不用说评估未来能力更强的系统。我们的提交文件阐述了我们对确保人工智能问责制所需流程和基础设施的看法。我们的建议考虑了 NTIA 作为协调机构的潜在角色，该机构可与美国国家标准与技术研究院（NIST）等其他政府机构合作制定标准。

在我们的建议中，我们重点关注适用于高性能通用人工智能模型的问责机制。具体建议如下：

*   **资助研究以建立更好的评估**
    *   增加对人工智能模型评估研究的资助。开发严谨、标准化的评估是一项困难且耗时的工作，需要大量资源。增加资助，特别是来自政府机构的资助，有助于推动这一关键领域的进展。
    *   要求公司在近期披露评估方法和结果。部署人工智能系统的公司应被强制要求满足其评估方面的某些披露要求，但如果公开会损害知识产权或机密信息，则这些要求无需公开。这种透明度可以帮助研究人员和政策制定者更好地了解现有评估可能存在的不足。
    *   长期制定一套行业评估标准和最佳实践。像 NIST 这样的政府机构可以致力于建立评估人工智能模型能力、局限性和风险的标准和基准，供公司遵守。

*   **基于模型能力创建风险响应式评估**
    *   为人工智能系统开发标准能力评估。政府应资助并参与针对高级人工智能带来的关键风险（如欺骗和自主性）的严格能力和安全性评估的开发。这些评估可以为相称的、风险响应式的监管提供基于证据的基础。
    *   通过对安全性评估的更多研究和资助来确定风险阈值。一旦确定了风险阈值，我们就可以强制要求对所有模型进行针对该阈值的评估。
        *   如果模型风险低于此阈值，现有的安全标准可能已足够。验证合规性并部署。
        *   如果模型风险超过此阈值，且安全评估和缓解措施不足，则停止部署，显著加强监督，并通知监管机构。在允许部署前确定适当的保障措施。

*   **为大型人工智能训练运行建立预注册制度**
    *   建立人工智能开发者报告大型训练运行的流程，确保监管机构了解潜在风险。这涉及确定适当的接收方、所需信息以及适当的网络安全、保密性、知识产权和隐私保障措施。
    *   为进行大型训练运行的人工智能开发者建立一个保密注册处，在训练开始前向其本国的国家政府预先注册模型详细信息（例如，模型规格、模型类型、计算基础设施、预期训练完成日期和安全计划）。汇总的注册数据应受到最高可用标准和规格的保护。

*   **赋能第三方审计机构，使其具备以下特质……**
    *   **技术素养**——至少部分审计人员需要具备深厚的机器学习经验；
    *   **安全意识**——能够妥善保护宝贵的知识产权，这些信息如果被盗可能构成国家安全威胁；
    *   **灵活性**——能够进行稳健但轻量级的评估，既能发现威胁，又不会削弱美国的竞争力。

*   **在模型发布前强制进行外部红队测试**
    *   强制要求对人工智能系统进行外部红队测试，可以通过集中的第三方（例如 NIST）或去中心化的方式（例如通过研究人员 API 访问）进行，以标准化人工智能系统的对抗性测试。这应成为发布先进人工智能系统的开发者的先决条件。
    *   在外部红队测试成为模型发布前提条件之前，建立高质量的外部红队测试选项。这一点至关重要，因为目前红队测试人才几乎完全集中在私营人工智能实验室内部。

*   **推进可解释性研究**
    *   增加对可解释性研究的资助。为大学、非营利组织和公司的可解释性工作提供政府资助和激励措施。这将使得在较小模型上进行有意义的工作成为可能，从而推动前沿实验室之外的进展。
    *   认识到目前要求模型可解释的法规尚不可行，但随着研究进展，未来可能实现。

*   **通过明确反垄断规定促进行业在人工智能安全方面的协作**
    *   鉴于当前的反垄断法，监管机构应就允许的人工智能行业安全协调发布指导。阐明私营公司如何在符合公共利益的前提下合作而不违反反垄断法，将减少法律不确定性并推进共同目标。

- 建立AI开发者报告大型训练任务流程，确保监管机构了解潜在风险。这包括确定适当的接收方、所需信息，以及相应的网络安全、保密性、知识产权和隐私保护措施。
- 为进行大型训练任务的AI开发者建立保密注册机制，要求其在训练开始前向本国政府预先登记模型详细信息（例如模型规格、模型类型、计算基础设施、预计训练完成日期和安全计划）。汇总的注册数据应按照最高可用标准和规范进行保护。

- 具备技术素养——至少部分审计人员需要深厚的机器学习经验；
- 具备安全意识——能够妥善保护可能构成国家安全威胁的有价值知识产权；
- 保持灵活性——能够开展稳健而轻量化的评估，在捕捉威胁的同时不损害美国竞争力。

- 通过集中式第三方机构（如NIST）或去中心化方式（如通过研究者API访问），强制对AI系统进行外部红队测试，以标准化AI系统的对抗性测试。这应作为开发者发布先进AI系统的先决条件。
- 在红队测试成为模型发布前提之前，建立高质量的外部红队测试选项。这一点至关重要，因为目前红队测试人才几乎全部集中在私营AI实验室。

- 增加可解释性研究资金。为大学、非营利组织和企业的可解释性工作提供政府资助和激励措施。这将使针对较小模型开展有意义的研究成为可能，推动前沿实验室之外的进展。

- 鉴于现行反垄断法，监管机构应发布关于AI行业安全协作许可范围的指南。明确私营企业如何在符合公共利益的前提下开展合作而不违反反垄断法，将有助于减少法律不确定性并推进共同目标。

我们相信这套建议将使我们切实推进建立有效的AI问责框架。实现这一目标需要研究者、AI实验室、监管机构、审计师及其他利益相关方的通力合作。Anthropic致力于支持安全开发和部署AI系统的各项努力。评估、红队测试、标准制定、可解释性及其他安全研究、审计以及强有力的网络安全实践，都是在实现AI效益的同时降低其风险的有效途径。

我们相信AI可能在我们有生之年产生变革性影响，并希望确保这些影响是积极的。建立健全的AI问责与审计机制对实现这一目标至关重要。我们感谢有机会回应本次征求意见。
