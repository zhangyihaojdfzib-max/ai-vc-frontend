# 宣布推出 Cloudflare 账户滥用防护：防范来自机器人和人类的欺诈攻击

2026-03-12

- Jin-Hee Lee

![](/images/posts/0aee696e8370.png)

今天，Cloudflare 推出了一套新的欺诈防护功能，旨在在账户滥用开始之前就将其阻止。多年来，我们一直致力于赋能 Cloudflare 客户保护其应用程序免受自动化攻击，但威胁形势已经演变。混合自动化与人工滥用的产业化给网站所有者带来了复杂的安全挑战。例如，考虑一个在五分钟内从纽约、伦敦和旧金山访问的单一账户。这种情况下的核心问题不是“这是自动化的吗？”，而是“这是真实的吗？”

网站所有者需要工具来阻止其网站上的滥用行为，无论它来自谁。

在 2024 年的生日周期间，我们向所有客户（包括免费计划的所有用户）赠送了**泄露凭证检测**功能。自那时起，我们已将**账户接管检测 ID** 作为我们**机器人管理解决方案**的一部分添加进来，以帮助识别攻击您登录页面的机器人。

现在，我们将这些强大的工具与新的工具相结合。**一次性邮箱检查**和**邮箱风险**功能可帮助您为使用一次性邮箱地址注册的用户强制执行安全偏好设置（这是创建虚假账户和进行推广滥用的常见策略），或根据邮箱模式和基础设施被认为存在风险的邮箱。我们也很高兴地推出**哈希用户 ID**——通过对用户名进行加密哈希处理生成的每域标识符——它能让客户更好地洞察可疑的账户活动，并增强缓解潜在欺诈流量的能力，同时不损害最终用户的隐私。

我们今天宣布的新功能超越了自动化检测，能够识别人类用户**和**机器人中的滥用行为和风险身份。**账户滥用防护**现已提供早期访问，任何机器人管理企业客户都可以在有限时间内免费使用这些功能，直到今年晚些时候 Cloudflare 欺诈防护全面上市。如果您想了解更多关于此早期访问功能的信息，请**在此注册**。

### 泄露的凭证使登录变得极其脆弱

欺诈行为的准入门槛低得危险，特别是考虑到大规模数据集的可用性以及可大规模实施账户欺诈的自动化工具的获取。网站所有者不仅要应对单个黑客，还要应对产业化的欺诈。去年，我们强调了我们网络中**41% 的登录使用了泄露的凭证**。在暴露了一个包含**160 亿条记录**的数据库之后，这个数字只增不减，并且此后发生了多起引人注目的数据泄露事件。

更重要的是，用户会在多个平台上重复使用密码，这意味着多年前的一次泄露，今天仍然可能解锁一个高价值的零售账户甚至银行账户。我们的**泄露凭证检查**是一项免费功能，用于检查密码是否在互联网上其他服务或应用程序的已知数据泄露中遭到泄露。这是一项保护隐私的凭证检查服务，有助于保护我们的用户免受泄露凭证的侵害，这意味着 Cloudflare 在执行这些检查时不会访问或存储最终用户的明文密码。**密码会被哈希处理**——即使用加密算法将其转换为随机字符串——以便与泄露凭证数据库进行比较。如果您尚未开启我们的**泄露凭证检查**，请立即启用它，以保护您的账户免受简单攻击！

对于攻击者而言，访问庞大的泄露凭证数据库只有在他们能够快速在许多网站上循环使用这些凭证，以识别哪些账户因密码重用而仍然脆弱时才有用。在我们 2024 年的黑色星期五分析中，我们观察到我们网络中超过 **60% 的登录页面流量是自动化的**。这是大量试图入侵的机器人。

为了帮助客户保护其登录端点免受持续不断的攻击，我们添加了**专门针对账户接管（ATO）的检测**，以突出显示可疑的流量模式。这是我们近期关注**按客户检测**的一部分，我们为每个机器人管理客户提供独特的行为异常检测。如今，机器人管理客户可以直接在安全分析仪表板上查看并缓解其登录请求中尝试的 ATO 攻击。

在安全分析仪表板左侧的卡片中，您可以查看并处理尝试的账户接管攻击。

在过去一周，我们的 ATO 检测组合平均每天捕获了 **69 亿次可疑登录尝试**，遍布我们的网络。这些 ATO 检测，连同我们机器人管理解决方案中的许多其他检测机制，共同构成了针对 ATO 和其他恶意自动化攻击的**分层防御**。

### 从自动化到意图和身份

是检测自动化，还是检测意图和身份？这是个问题。我们的答案是：两者都是，因为两者都是稳健安全态势的关键层面。攻击者现在以前所未有的规模运作，这种规模以前是企业级服务才有的：他们利用大规模的凭证泄露，使用人工驱动的欺诈农场来伪造设备和位置，并创建合成身份来维持数千甚至数百万个虚假账户，用于推广和平台滥用。一个使用自动化工具的人可能正在盗取账户、滥用促销活动、实施支付欺诈，或者同时进行上述所有行为。

除此之外，自动化的获取变得前所未有的容易，特别是随着用户越来越熟悉使用 **AI Agent（智能体）**，甚至长期存在的“传统”浏览器也默认具备了智能体能力。无论是单个行为者使用 AI Agent，还是有组织的欺诈活动，威胁都不像单个脚本那么简单——它可能涉及人类意图，并通过自动化执行。

考虑以下我们从客户那里听到的场景：

- 我们这个月有 1000 个新用户，但其中一半以上是虚假身份，他们利用了免费试用，然后消失。
- 攻击者用正确的密码登录了，我怎么知道这不是真正的用户？
- 这个实体正以人类的速度行动，并且他们正在盗取账户。

这些问题**不能仅靠**评估自动化来解决；它们需要检查真实性和完整性。这正是我们专门的欺诈防护功能所要解决的差距。

### 评估可疑邮箱

让我们从评估潜在账户滥用的最早点开始：账户创建。虚假或批量账户创建是关于网站欺诈讨论的最大话题之一，因为它可以为攻击者打开访问应用程序——甚至整个商业模式的大门。

Cloudflare 为客户提供了两种从源头评估可疑账户创建的工具：

1.  一次性邮箱检测：检测用户是否使用常用于促销滥用和虚假账户创建的一次性或临时邮箱地址进行注册。这些一次性邮箱服务允许攻击者无需维护真实基础设施即可创建数千个"唯一"账户，尤其是那些无需创建账户即可即时访问或提供免费无限邮箱别名的未经验证的一次性邮箱。客户在构建规则以执行安全偏好时，可以使用此二进制字段，选择直接阻止所有一次性邮箱，或者对任何尝试使用一次性邮箱创建账户的用户发起**质询**。
2.  邮箱风险：Cloudflare 分析邮箱模式和基础设施，提供风险等级（低、中、高），客户可在安全规则中使用。我们知道并非所有邮箱地址都生而平等；格式为 `[email protected]` 的地址与 `[email protected]` 具有不同的风险特征。邮箱风险等级允许客户在账户创建环节表达其对风险和摩擦的容忍度。

一次性邮箱检测和邮箱风险功能现已可在安全分析和安全规则中使用，使网站所有者能够保护其账户创建流程。这些检测解决了一个根本问题：当账户开始实施滥用行为时，往往为时已晚。网站所有者已经支付了获客成本，欺诈用户已经消耗了促销积分，而补救措施则需要人工审核。在注册环节——这个最重要的时刻——缓解可疑邮箱意味着增加适当的摩擦。

### 推出哈希用户ID

理解滥用模式需要**可见性**：不仅是对网络，还包括账户活动。传统上，安全意味着通过 IP 地址和孤立的 HTTP 请求来识别自动化活动，但网站所有者不仅仅从网络信号的角度思考；他们也在考虑其用户和已知账户。这就是为什么我们正在扩展我们的缓解工具箱，以匹配应用程序的实际构建方式，专注于基于用户的欺诈活动检测。

攻击者可以毫不费力地轮换 IP 地址来隐藏其踪迹。但迫使他们反复生成新的、可信的账户会带来巨大的摩擦，尤其是在结合账户创建保护措施时。当我们超越网络层，将欺诈行为映射到特定的被盗用或滥用账户时，我们就可以发现与单个、持续行为者相关的针对性行为，并制止滥用。通过这种方式，我们将防御策略转移到账户层面，而不是与轮换的 IP 地址和住宅代理玩打地鼠游戏。这意味着**我们的客户可以根据其应用程序区分身份的方式来缓解滥用行为**。

为使网站所有者具备此能力，Cloudflare 正在发布**哈希用户ID**，客户可在**安全分析**、**安全规则**和**托管转换**中使用。用户ID是按域加密哈希处理的用户名字段值，每个用户ID都是为客户应用程序上的给定用户名生成的加密、唯一且稳定的标识符。**重要的是，实际用户名不会作为此服务的一部分被 Cloudflare 记录或存储。** 与识别登录流量然后加密凭据以进行比较的泄露凭据检查和账户接管检测一样，我们优先考虑最终用户的隐私，同时赋能我们的客户对欺诈行为采取行动。

通过访问哈希用户ID，网站所有者可以：

*   查看顶级用户：哪些账户的活动最多？
*   查看唯一用户何时从其通常不登录的国家/地区登录——或者一天内从多个国家/地区登录！
*   基于唯一用户缓解流量，例如阻止具有历史可疑活动的用户。
*   组合字段以查看账户何时成为泄露凭据攻击的目标。
*   查看哪些网络模式或信号与唯一用户相关联。

在安全分析仪表板中查看单个哈希用户ID的扩展视图，显示该唯一用户的活动详情，包括其登录位置和浏览器。

这种用户级别的可见性改变了网站所有者调查和缓解流量的方式。我们的客户不再孤立地检查单个请求，而是可以看到攻击者如何瞄准并隐藏在合法用户中的全貌。
