# SGLang 中的 Transformers 后端集成

Hugging Face Transformers 库是处理最先进模型的标准工具——从实验前沿研究到在自定义数据上进行微调。其简洁性、灵活性和庞大的模型库使其成为快速开发的强大工具。

但是，一旦您准备从笔记本环境转向生产环境，推理性能就变得至关重要。这正是 SGLang 的用武之地。

SGLang 专为高吞吐量、低延迟推理而设计，现在提供与 Transformers 作为后端的无缝集成。这意味着您可以将 Transformers 的灵活性与 SGLang 的原始性能结合起来。

让我们深入了解此集成能实现什么以及如何使用它。

# 摘要

SGLang 现在支持将 Hugging Face Transformers 作为后端，让您可以开箱即用地运行任何与 Transformers 兼容的模型，并获得高性能推理。

```python
import sglang as sgl

llm = sgl.Engine("meta-llama/Llama-3.2-1B-Instruct", impl="transformers")
print(llm.generate(["The capital of France is"], {"max_new_tokens": 20})[0])

```

无需原生支持——SGLang 会在需要时自动回退到 Transformers，或者您可以显式设置 `impl="transformers"`。

# Transformers 与 SGLang

让我们通过一个使用 `meta-llama/Llama-3.2-1B-Instruct` 的简单文本生成示例来比较两种方法。

## Transformers

Transformers 库非常适合实验、小规模任务和训练，但它并未针对高吞吐量或低延迟场景进行优化。

```python
from transformers import pipeline

pipe = pipeline("text-generation", model="meta-llama/Llama-3.2-1B-Instruct")
generate_kwargs = {
    "top_p": 0.95,
    "top_k": 20,
    "temperature": 0.8,
    "max_new_tokens": 256
}
result = pipe("The future of AI is", **generate_kwargs)
print(result[0]["generated_text"])

```

## SGLang

SGLang 采取了不同的路径，优先考虑效率，并具备 RadixAttention（一种内存高效的注意力机制）等特性。使用 SGLang 进行推理明显更快、更节省资源，尤其是在负载下。以下是使用离线引擎在 SGLang 中执行相同任务的代码：

```python
import sglang as sgl

if __name__ == '__main__':
    llm = sgl.Engine(model_path="meta-llama/Llama-3.2-1B-Instruct")
    prompts = ["The future of AI is"]
    sampling_params =  {
        "top_p": 0.95,
        "top_k": 20,
        "temperature": 0.8,
        "max_new_tokens": 256
    }
    outputs = llm.generate(prompts, sampling_params)
    print(outputs[0])

```

或者，您可以启动一个服务器并发送请求：

```bash
python3 -m sglang.launch_server \
  --model-path meta-llama/Llama-3.2-1B-Instruct \
  --host 0.0.0.0 \
  --port 30000

```

```python
response = requests.post(
    "http://localhost:30000/generate",
    json={
        "text": "The future of AI is",
        "sampling_params": {
            "top_p": 0.95,
            "top_k": 20,
            "temperature": 0.8,
            "max_new_tokens": 256
        },
    },
)
print(response.json())

```

请注意，SGLang 还提供了与 OpenAI 兼容的 API，使其可以轻松替代外部服务。

# SGLang 中的 Transformers 后端

通过新的 Transformers 后端集成，SGLang 现在可以自动回退使用其本身不原生支持的 Transformers 模型。这意味着在实践中：

- 即时访问添加到 Transformers 的新模型
- 支持来自 Hugging Face Hub 的自定义模型
- 减少工程开销

这解锁了更快的推理和优化的部署（例如启用 RadixAttention），同时又不牺牲 Transformers 生态系统的简单性和多功能性。

## 使用方法

```python
llm = sgl.Engine(model_path="meta-llama/Llama-3.2-1B-Instruct", impl="transformers")

```

请注意，指定 `impl` 参数是可选的。如果模型不被 SGLang 原生支持，它会自动切换到 Transformers 实现。

Hugging Face Hub 上任何与 Transformers 兼容（使用 `trust_remote_code=True` 并正确实现了注意力机制）的模型都可以与 SGLang 一起使用。您可以在官方文档中找到确切的要求。如果您的自定义模型满足这些条件，您只需要在加载时设置 `trust_remote_code=True`。

```python
llm = sgl.Engine(model_path="new-custom-transformers-model", impl="transformers", trust_remote_code=True)

```

## 示例

```bash
python3 -m sglang.launch_server \

  --port 30000

```

