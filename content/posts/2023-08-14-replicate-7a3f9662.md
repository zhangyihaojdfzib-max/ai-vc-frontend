# 语言模型的流式输出  

- zeke  

你知道在使用 ChatGPT 或 Vercel 的 AI 游乐场时，它会返回一个逐字渲染的动画响应吗？这不仅仅是为了制造戏剧性的视觉效果，让你觉得对话另一端有一个机器人在打字。这实际上是语言模型逐个生成 Token，并在运行过程中将它们流式传输回给你。  

Replicate 已经提供了通过轮询和 Webhook 在预测运行时接收增量更新的方式。但这些并不总是从运行中的模型获取更新最高效的方法。当你在构建像聊天应用这样的东西时，你真正需要的是一个实时更新的事件流。  

Replicate 的 API 现在支持语言模型的服务器发送事件流。这让你可以在模型运行时实时更新你的应用。在这篇文章中，我们将展示如何在 Replicate 上消费语言模型的流式响应。  

## 流式工作原理  

从高层次来看，在 Replicate 上消费事件流的工作原理如下：  

1. 你使用 `stream` 选项创建一个预测。  
2. Replicate 返回一个包含用于接收流式输出的 URL 的预测。  
3. 你在网页浏览器中连接到该 URL，并接收一个更新流。  

## 一个 Node.js 示例  

让我们通过一个使用 Replicate 的 Node.js 客户端的示例来逐步说明。  

首先，使用 `llama-2-70b-chat` 创建一个预测，并将 `stream` 选项设置为 `true`：  

```
import Replicate from "replicate";

const replicate = new Replicate({ auth: process.env.REPLICATE_API_TOKEN });

const prediction = await replicate.predictions.create({
  version: "2c1608e18606fad2812020dc541930f2d0495ce32eee50074220b87300bc16e1",
  input: { prompt: "Tell me a story" },
  stream: true,
});
```

注意预测响应中的 `stream` URL：  

```
console.log(prediction.urls.stream);
// https://streaming.api.replicate.com/v1/predictions/fuwwvjtbdmroc4xifxdcwqtdfq
```

要接收流式输出，在你的浏览器端 JavaScript 代码中使用预测中的流 URL 构建一个 `EventSource`：  

```
const source = new EventSource(prediction.urls.stream, {
  withCredentials: true,
});

source.addEventListener("output", (e) => {
  console.log("output", e.data);
});

source.addEventListener("error", (e) => {
  console.error("error", JSON.parse(e.data));
});

source.addEventListener("done", (e) => {
  source.close();
  console.log("done", JSON.parse(e.data));
});
```

## 使用 cURL 的命令行示例  

浏览器内置的 EventSource API 对于构建 Web 应用很有用，但响应是标准的 HTTP 事件流响应，因此你不必使用浏览器来消费它们。你也可以使用你选择的编程语言接收流式输出，或者使用像 cURL 和 jq 这样的命令行工具直接在终端中显示输出。  

在 shell 中复制并粘贴以下命令，以执行以下操作：  

1. 使用 `curl` 创建一个带有 `llama-2-70b-chat` 的预测  
2. 将预测响应通过管道传递给 `jq` 以提取流 URL 并打印出来  
3. 再次使用 `curl` 连接到流 URL 并接收更新流  

```
STREAM_URL=$(curl -s -X POST \
-d '{"version": "58d078176e02c219e11eb4da5a02a7830a283b14cf8f94537af893ccff5ee781", "input": {"prompt": "Tell me a story"}, "stream": true}' \
-H "Authorization: Bearer $REPLICATE_API_TOKEN" \
"https://api.replicate.com/v1/predictions" | jq -r .urls.stream)
echo $STREAM_URL
curl -H 'Accept: text/event-stream' $STREAM_URL
```

## 为你自己的模型添加流式支持  

在将你自己的公共或私有语言模型发布到 Replicate 时，你应该确保它们支持流式输出，以便你的模型用户能获得最佳体验。  

如果你正在微调一个现有的语言模型，那么你已经准备好了：你的微调模型将自动继承基础模型的流式支持。  

如果你正在使用 Cog 编写你自己的模型，关键是在生成 Token 时使用 `yield`，而不是从函数中返回最终结果。使用 `ConcatenateIterator` 来提示输出应拼接成一个字符串。以下是一个示例：  

```
from cog import BasePredictor, Path, ConcatenateIterator

class Predictor(BasePredictor):
    def predict(self) -> ConcatenateIterator[str]:
        tokens = ["The", "quick", "brown", "fox", "jumps", "over", "the", "lazy", "dog"]
        for token in tokens:
            yield token + " "
```

有关更多详细信息，请查看关于流式输出的 Cog 文档。
