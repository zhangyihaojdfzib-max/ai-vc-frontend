## 为智能体与自动化而生

Registrar API 旨在软件构建的任何地方都能良好工作：无论是在编辑器内部、部署流水线、后端服务，还是智能体驱动的工作流中。

其工作流设计得简洁且对机器友好。`搜索`返回候选名称。`检查`返回实时可用性和价格。`注册`接收一个最小化的请求，并返回一个符合工作流结构的响应，该响应可以立即完成，或者在耗时较长时通过轮询获取。这使得传统 API 客户端和代表用户行事的 AI 智能体都能直接使用。

实际上，这意味着智能体可以帮助完成整个流程：建议名称、确认哪个名称实际可注册、展示价格以供批准，然后完成购买，而无需强迫用户离开他们正在使用的工具。

### 步骤 1：搜索域名

使用 `search` 端点提交域名查询，可以带或不带域名后缀。

```JavaScript

    method: "GET",
    path: `/accounts/${accountId}/registrar/domain-search`,
    query: { q: "acme corp", limit: 3 },
  });
}
```

### 步骤 2：检查可用性和价格

搜索结果快速但非权威；它们基于缓存数据，热门名称的可用性可能在几秒钟内发生变化。`Check` 直接查询注册局。在注册前立即调用它，并将其价格响应作为最终依据。

```JavaScript
async () => {
  return cloudflare.request({
    method: "POST",
    path: `/accounts/${accountId}/registrar/domain-check`,
    body: { domains: ["acmecorp.dev"] },
  });
}
```

### 步骤 3：注册域名

唯一必需的字段是域名。WHOIS 隐私保护默认启用，不收取额外费用。如果您的账户有默认的注册人联系信息，API 会自动使用它；否则，您可以在请求中内联提供联系详情。您的默认支付方式将被自动使用。

```JavaScript
async () => {
  return cloudflare.request({
    method: "POST",
    path: `/accounts/${accountId}/registrar/registrations`,
    body: { domain_name: "acmecorp.dev" },
  });
}
```

注册通常在几秒钟内同步完成。如果耗时较长，API 会返回 202 Accepted 状态码，并附带一个用于轮询的工作流 URL。无论哪种情况，响应结构都是相同的，无需特殊处理。对于溢价域名，`Check` 响应会返回注册局设定的确切价格，而 `Register` 请求会将其作为明确的费用确认回显。

### 关于智能体与不可退款购买的说明

当智能体代表您注册域名时，它会向您的默认支付方式收费。域名注册一旦完成，不可退款。一个设计良好的智能体流程应该在调用注册端点之前，向用户确认域名和价格。`Check` 步骤的存在正是为了使这个确认步骤明确且无歧义。API 为您提供了正确构建此流程的工具；而实现它的责任在于您的智能体逻辑。

默认情况下，我们的 API 文档包含明确的面向智能体的指示，要求在注册 API 调用期间寻求用户许可。尽管如此，设计一个不会在未经您批准的情况下购买域名的智能体流程，仍然是人类的责任。

## 为什么 Cloudflare 能与众不同地做到这一点

Cloudflare 与当前众多新增域名工作流的开发者平台的不同之处在于，Cloudflare 自身运营着域名注册商。这意味着，项目的构建和部署平台同样可以用于搜索、注册和管理域名，而无需额外加价。

成本价定价是 Cloudflare 注册商模式的核心。无论您是通过控制台注册域名、直接调用 API，还是委托 Agent（智能体）代为操作，我们都只收取注册管理机构收取的精确费用。

## API 的下一步发展

此次 Beta 版本聚焦于域名生命周期的第一个关键环节：搜索、检查和注册。我们正积极扩展 API 功能，以覆盖更多核心注册商体验，从而让域名在购买后（而不仅仅是创建时）也能通过编程方式管理。这将包括转移、续费、联系人更新等生命周期管理功能。

该 API 是迈向更广泛的“注册商即服务”产品的第一步。该服务的开发正在进行中，我们计划在今年晚些时候推出。随着 API 的扩展，网站构建器、托管服务提供商、AI 产品和其他多租户应用程序等平台将能够把域名注册整合到其自身的用户体验中。用户可以在不离开其正在使用的服务或 Agent（智能体）驱动的工作流的情况下，搜索、购买并配置域名。
