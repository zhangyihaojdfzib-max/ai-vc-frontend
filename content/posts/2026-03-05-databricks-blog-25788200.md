Databricks 与全球孤儿（GO）项目建立了合作伙伴关系，该非营利组织致力于为家庭对接资源与社区支持，以预防儿童陷入系统性悲剧。

通过 Databricks for Good 计划——一项提供公益专业服务以创造社会影响力的倡议——Databricks 团队帮助 GO 项目夯实了数据基础，并加速了其使命推进与影响力拓展。

## 数据挑战与局限

2025年，GO 项目的本地合作伙伴在美国43个州服务了近12.2万名儿童，并在6个国家服务了5.3万名国际儿童。美国境内有超过1,600家活跃的合作机构提交需求，8,200个响应团队近乎实时地予以回应，数据分散存储于各个系统，导致报告生成面临挑战。因此，诸如“促成每项需求（即平台数据）的成本是多少（即财务数据）？”等关键问题，常常需要在自动化报告系统之外通过电子表格手动计算，这使得数据的可用性和一致性未能达到理想的高效水平。

GO 项目需要一个能够轻松从众多数据源提取数据、汇聚至一个可信数据层的工具，以驱动报告生成并提升整体数据的一致性和可用性。在将数据整合到统一数据平台的同时，还需确保数据治理、访问和权限的紧密集成，以便从内部员工到机构合作伙伴再到教会志愿者等所有类型的用户，都能根据其目的访问相应的数据子集。

为应对这些挑战，GO 项目选择了 Databricks，因其具备无服务器工作空间的便捷设置、与云平台的无缝集成、Unity Catalog 的治理能力，以及在同一平台上统一数据工程、分析和人工智能的能力。

## 使用 Databricks 实现数据架构现代化

在 Databricks for Good 合作期间，GO 项目与两名 Databricks 交付解决方案架构师和一名 Databricks 项目经理合作，历时三个月，设计并实施了如下所示的现代化数据架构。

![medallion architecture](/images/posts/532070b46664.png)

该解决方案围绕奖牌架构（青铜层、白银层、黄金层）设计，为分析和人工智能驱动的用例提供了可扩展且可信赖的基础。来自第三方 API 和 AWS RDS MySQL 的原始数据，通过开源和 Databricks 托管功能高效摄取，能够在数据量增长时保持管道的韧性，同时快速接入新的数据源。白银层的数据质量和可靠性通过管道预期（使用 Spark 声明式管道）得以保障，从而能够及早识别下游数据问题，并建立标准化的数据质量框架。

最后，数据被聚合到黄金层，作为下游消费的可信数据源。指标视图支持集中式仪表板，使不同团队都能便捷获取洞察，减少了对人工报告或专业技术支持的依赖。同时，这些精心管理的数据集支持生成个性化的 AI 通讯稿，而无需重新定义核心业务指标或创建并行的数据孤岛。

在此基础之上，Unity Catalog 作为跨所有数据和 AI 资产的统一治理层，使 GO 项目能够自信地扩展自助式分析和 AI 项目。

## 解决方案与成果

以下部分重点介绍了通过 Databricks for Good 合作交付的解决方案，以及其为 GO 项目带来的可衡量成果。

### 集中式 KPI 仪表板

GO 项目面临的一个主要挑战是，缺乏一个单一、可访问的视图来展示其合作伙伴网络中的组织绩效。关键指标分散存储在多个数据源中，需要团队手动收集和解读信息。这个过程既耗时又容易产生不一致。

通过 Databricks for Good 计划，Databricks 与 GO 项目合作，将这种分散的报告模式转变为基于数据湖仓架构构建的集中式、自动化 KPI 仪表板。

新数据被自动摄取和处理，取代了对静态导出或手动更新的依赖，使仪表板能够近乎实时地反映变化。这确保了领导层和一线团队始终基于最新的可用信息开展工作。最终成果是一个统一的单一事实来源，将整个组织的运营数据汇集到一个实时更新的仪表板中。

该仪表板利用了以下关键的 Databricks 功能：

- **指标视图**：用于标准化 KPI 定义，确保所有报告和仪表板中的计算一致。GO 项目不再需要四处发送 SQL 代码片段，并担心 WHERE 子句是否正确。
- **AI/BI 仪表板**：支持创建直观的、拖放式的可视化图表，满足 GO 项目的运营需求。
- **Databricks One**：允许业务用户安全地使用和交互 AI/BI 仪表板，而无需直接访问 Databricks 工作空间或底层数据集。

因此，GO 项目实现了以下业务和技术成果：

- **运营效率**：将报告周期从数天缩短至数分钟，使用户能够按需监控区域指标的健康状况，而无需等待电子邮件报告。
- **快速生产力与学习**：团队成员能够使用数据科学智能体快速编写和迭代复杂的分析查询，加速洞察生成，而无需深厚的 SQL 专业知识。
- **可扩展的报告基础**：通过在 Databricks 数据智能平台内直接建立标准化的报告基础，减少了对一次性、临时报告的依赖。

总体而言，整合的 KPI 仪表板为 GO 项目提供了及时、可操作的对外服务绩效洞察。凭借对关键指标的近乎实时可见性，该组织能够更快地响应、更有效地分配资源，并最终增强其防止更多儿童陷入系统性悲剧的能力。

### 个性化的 AI 生成捐赠者外联

通过集中式 KPI 仪表板建立了数据驱动的绩效视图后，GO 项目将重点转向通过更有效的利益相关者参与来激活这些洞察。GO 项目寻求大规模生成及时且个性化的内容，努力利用针对其当地社区个性化的数据来吸引捐赠者。

此前，GO 项目主要依赖手动流程。必须为每个利益相关者单独从 MySQL 数据库中提取数据，然后手动格式化并整合到通讯中，这使得频繁为所有潜在支持者定制信息变得困难。

通过 Databricks for Good 倡议，团队设计并实施了一个自动化系统，直接从 Databricks 数据智能平台上的管理数据集中生成个性化的、由 AI 驱动的通讯稿。通过将受治理的数据与内置的生成式 AI 功能相结合，该解决方案以最少的人工干预，将运营指标转化为适合利益相关者的叙述。

为实现此交付成果，使用了以下关键的 Databricks 功能：

- **Databricks Notebooks** 和 **Databricks AI 函数**（ai_query）：基于 Notebook 小组件（驱动细分逻辑）动态生成针对特定利益相关者的叙述摘要，避免了手动配置和自定义脚本。
- **基础模型 API**：直接从数据平台集成生成式 AI，实现在数据准备和转换的同时进行内容生成。
- **Unity Catalog Volumes**：将非结构化的、AI 生成的通讯稿输出以 PDF 格式安全存储在云存储中，简化了下游的分发和访问。

这为 GO 项目带来了以下业务和技术成果：

- **加速内容创作**：个性化的捐赠者市场概况草案生成现在只需**数秒**而非数日，使GO Project营销团队能够专注于内容优化和叙事构建，而非手动内容创作。
- **消除手动数据汇总**：团队不再需要从多个系统中提取月度影响数据。用户只需选择相关方，预配置的小部件便会自动筛选相关数据，并利用ai_query函数生成个性化内容。
- **可扩展的数据驱动捐赠营销**：通过Notebook、数据科学Agent（智能体）与可信数据管道的紧密集成，GO Project得以快速构建针对本地市场的捐赠营销产品。募资团队现在能以过去制作单份报告所需时间的一小部分，按需生成覆盖全美各地区的定制化数据支持报告。

这些能力共同推动GO Project超越静态报告，迈入个性化、AI驱动的叙事模式，从而在增强其使命可见度与影响力的同时，深化与相关方的关系。

在初始阶段，该解决方案依赖Databricks AI函数生成通讯内容。展望未来，GO Project计划利用**Agent Bricks**引入负责通讯不同板块的领域专属智能体。这一方法将进一步减少提示词调优开销，提升输出结果的一致性，并实现对底层大语言模型（LLM）更可扩展的优化。
