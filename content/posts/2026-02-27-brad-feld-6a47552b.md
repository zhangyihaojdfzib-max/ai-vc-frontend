## 骄傲的舅舅警报——Sabrina Feld

- #氛围编程
- #Claude Code
- #家庭

![Sabrina Feld 的主页，包含醒目的个人介绍、充满活力的艺术作品网格，以及她在斯克里普斯学院学习的简要说明。](/images/posts/6c7139c256f3.png)

上周三，我弟弟 Daniel 给全家人发了一封邮件，主题是“骄傲的爸爸警报！”。他的女儿 Sabrina 刚刚从零开始构建并发布了一个个人作品集网站。她没有使用 Squarespace 或 Wix。她构建了一个定制的 Next.js 网站，包含滚动触发动画、磨砂玻璃导航栏、带灯箱的自定义图片轮播和六个页面模板——全部自托管在 Netlify 上。

Sabrina 是斯克里普斯学院的大四学生，攻读科学技术与社会和美术双学位。她是一名产品设计师和美术家——不是软件开发人员。她使用 Claude Code 构建了整个网站。

她写了一篇关于这个过程的博客文章，描述了当你不知道 CSS 时如何指导 AI。她“不得不用其他方式变得精确”——使用设计词汇和视觉参考，而不是代码片段。当出现错误时，她描述症状并分享截图，而不是阅读堆栈跟踪。

让我印象深刻的一句话是：“模糊的提示词产生通用的设计。清晰的创作信念产生了感觉属于我自己的东西。”

这与我每天使用 Claude Code 构建时看到的情况相符。输出的质量直接与输入的具体程度相关。“让它看起来更好”会给你一些通用的东西。“我想要暖色调、编辑式布局，以及悬停状态使用毛茛黄作为强调色”则会给你看起来像是经过真正设计决策的东西。Sabrina 的做法是，在每次会话前都对她想要的东西有强烈的想法——收集设计参考、准备内容，并对美学有清晰的愿景。

在为期十一天的构建过程中，她进行了超过二十次反馈会话，大约有四到六小时的主动工作时间。AI 并没有消除迭代，而是让每一轮迭代变得更快。

去看看这个网站。她的艺术部分展示了单版画、粉彩、水彩和蓝晒作品。项目部分涵盖她在 StackHawk 的产品管理工作，包括她端到端负责的产品发布，以及为 AI 驱动的安全测试工具所做的研究。设计简洁，注重排版，带有深色页脚和她指定的毛茛黄强调色。

看到网站后，我做了任何一个书呆子舅舅都会做的事——我进行了一次安全审查。结果很扎实。TLS 配置正确，没有暴露敏感文件，生产环境中没有源映射，HTTP 正确重定向到 HTTPS。我发给 Sabrina 一份需要添加的安全头部列表和一些值得配置的 DNS 记录——大约十分钟的工作量就能解决发现的问题。

Sabrina 今年春天毕业后正在寻找产品设计和产品管理方面的职位。她的作品集在 sabrinafeld.com。
