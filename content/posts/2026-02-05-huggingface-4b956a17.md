# 欢迎使用 SyGra Studio

SyGra 2.0.0 引入了 Studio，这是一个交互式环境，它将合成数据生成转变为透明、可视化的创作过程。您无需再费力处理 YAML 文件和终端命令，而是可以直接在画布上编排流程，在提交前预览数据集，通过内联变量提示调整提示词，并实时观察执行流——所有这些都在一个面板中完成。其底层仍是同一个平台，因此您所有的可视化操作都会生成相应的、与 SyGra 兼容的图配置和任务执行器脚本。

### Studio 的功能

1.  通过引导式表单配置和验证模型（支持 OpenAI、Azure OpenAI、Ollama、Vertex、Bedrock、vLLM、自定义端点）。
2.  连接 Hugging Face、文件系统或 ServiceNow 数据源，并在执行前预览数据行。
3.  通过选择模型、编写提示词（支持自动建议变量）以及定义输出或结构化模式来配置节点。
4.  使用共享状态变量和基于 Pydantic 的映射来设计下游输出。
5.  端到端执行流程，并通过节点级进度即时查看生成的结果。
6.  使用内联日志、断点、基于 Monaco 的代码编辑器和自动保存的草稿进行调试。
7.  监控每次运行的 Token 成本、延迟和护栏结果，执行历史记录存储在 `.executions/` 目录中。

让我们一步步体验这个过程。

## 步骤 1：配置数据源

打开 Studio，点击“创建流程”，开始和结束节点会自动出现。在添加其他内容之前：

-   选择一个连接器（Hugging Face、磁盘或 ServiceNow）。
-   输入参数，如 `repo_id`、`split` 或文件路径，然后点击“预览”以获取样本行。
-   列名会立即变为状态变量（例如 `{prompt}`、`{genre}`），因此您可以确切知道在提示词和处理器中可以引用哪些内容。

一旦验证通过，Studio 会保持配置同步，并将这些变量贯穿整个流程——无需手动连接或猜测。

## 步骤 2：可视化构建流程

从面板中拖拽所需的模块。对于一个故事生成流程：

1.  放置一个名为“故事生成器”的 **LLM 节点**，选择一个已配置的模型（例如 `gpt-4o-mini`），编写提示词，并将结果存储在 `story_body` 中。
2.  添加第二个名为“故事摘要器”的 **LLM 节点**，在提示词中引用 `{story_body}`，并将输出到 `story_summary`。
3.  如果需要可重用的逻辑或分支行为，可以切换结构化输出、附加工具，或添加 Lambda/子图节点。

Studio 的详细信息面板将所有内容保持在上下文中——模型参数、提示词编辑器、工具配置、前后处理代码，甚至如果您需要并行生成，还包括多 LLM 设置。在提示词中输入 `{` 会立即显示所有可用的状态变量。

## 步骤 3：审查与运行

打开“代码面板”以检查 Studio 正在生成的确切 YAML/JSON。这与写入 `tasks/examples/` 目录的产物相同，因此您所见即所得。

当您准备好执行时：

-   点击“运行工作流”。
-   选择记录数量、批处理大小、重试行为等。
-   点击“运行”，然后在“执行”面板中实时观察节点状态、Token 使用量、延迟和成本的流式更新。详细的日志提供了可观测性，使调试变得轻松。所有执行记录都会写入 `.executions/runs/*.json`。

运行结束后，可以下载输出结果，与之前的执行进行比较，并获取延迟和使用情况的元数据详情。

## 运行现有工作流

### 运行 Glaive 代码助手工作流

SyGra Studio 也可以执行 `tasks` 目录中现有的工作流。例如，在 `tasks/examples/glaive_code_assistant/workflow` 中——它摄取 `glaiveai/glaive-code-assistant-v2` 数据集，起草答案，进行批判，并循环直到批判返回“NO MORE FEEDBACK”。

在 Studio 中，您会注意到：

1.  **画布布局** – 两个 LLM 节点（`generate_answer` 和 `critique_answer`）通过一个有条件的边连接，该边要么路由回节点进行更多修订，要么在批判满意时退出到 `END`。
2.  **可调输入** – 运行模态框允许您切换数据集分割、调整批处理大小、限制记录数量或调整温度，而无需接触 YAML 文件。
3.  **可观测的执行** – 实时观察两个节点依次点亮，检查中间批判，并监控状态。
4.  **生成的输出** – 生成合成数据，可用于模型训练、评估流程或标注工具。

## 开始使用

```bash
git clone https://github.com/ServiceNow/SyGra.git
cd SyGra && make studio

```
