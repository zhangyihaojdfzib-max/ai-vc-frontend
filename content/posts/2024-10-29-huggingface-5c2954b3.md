# 通用辅助生成：使用任意辅助模型实现更快的解码

TL;DR：许多LLM（大语言模型），如gemma-2-9b和Mixtral-8x22B-Instruct-v0.1，缺乏可用于辅助生成的更小版本。在这篇博文中，我们介绍通用辅助生成：一种由Intel Labs和Hugging Face开发的方法，它将辅助生成扩展为可与任意模型家族的小语言模型配合使用🤯。因此，现在可以将任何解码器或专家混合模型的推理速度提升1.5倍至2.0倍，且几乎零开销🔥🔥🔥。让我们深入了解！

## 引言

如今，最强的开源权重LLM通常拥有数十亿到数千亿的参数（你好，Llama-3.1-405B 👋），在生产环境中部署这些庞然大物带来了一系列工程挑战。其中一个挑战是，从这些大型模型生成文本速度很慢，这促使社区开发了多种技术来加速解码过程。辅助生成，也称为推测解码，是一种非常流行且实用的方法，可在不损失准确性的情况下加速LLM推理。在这篇博文中，我们探讨了辅助生成的工作原理，并分享了我们的研究成果，将其扩展到Hugging Face Hub上的14万个语言模型中的任意一个🚀！

## 辅助生成

辅助生成的核心思想是使用一对模型，称为目标模型和辅助模型。辅助模型是目标模型更小、更高效的版本，例如，您可以使用Llama-3.2-1B作为较大目标模型Llama-3.1-70B的辅助模型。
辅助生成是一个迭代过程。每个周期，辅助模型自回归地逐个生成一个Token序列。然后，目标模型在单次前向传播中验证序列中的所有辅助Token。速度提升是通过在目标模型的每次前向传播中确认多个Token实现的，而不是每次只生成一个Token。更详细的解释请参见原始博文。结合最近引入的动态推测策略，辅助生成可将文本生成速度提升1.5倍至3倍，具体取决于任务和所使用的模型。

辅助生成带来的显著加速有一个重大缺点：目标模型和辅助模型必须共享相同的分词器，这意味着它们必须来自同一模型家族。然而，许多广泛使用的模型缺乏既紧凑又足够精确以显著降低延迟的更小版本。根据我们的经验，当辅助模型比目标模型至少小50到100倍时，通常才能看到有意义的加速。例如，CodeLlama-13b缺乏更小的版本，而gemma-2-9b只有2b变体，其规模仍然不够小/速度不够快，无法实现显著的性能提升。

## 通用辅助生成

为了缓解这一痛点，Intel Labs与我们在Hugging Face的朋友共同开发了通用辅助生成（UAG）。UAG允许选择任意一对目标模型和辅助模型，无论它们的分词器是否相同。例如，gemma-2-9b可以作为目标模型，而微小的vicuna-68m可以作为辅助模型。

我们提出的方法背后的主要思想是双向分词器转换。一旦辅助模型完成一次生成迭代，辅助Token会被转换为文本，然后使用目标模型的分词器进行分词以生成目标Token。在验证步骤之后，目标Token同样被转换回辅助Token格式，并在下一次迭代开始前附加到辅助模型的上下文中。

由于辅助分词器和目标分词器使用不同的词汇表，因此需要处理它们之间的差异。为了准确重新编码新生成的辅助Token，必须预先添加一个由若干先前Token组成的上下文窗口。然后将整个序列重新编码为目标Token格式，并与最新的目标Token对齐，以精确定位新生成的Token应附加的位置。此过程如下方视频所示。

通用辅助生成已集成到🤗 Transformers的4.46.0版本中。

使用时，将tokenizer和assistant_tokenizer传递给generate()：

```python
>>> from transformers import AutoModelForCausalLM, AutoTokenizer

>>> prompt = "Alice and Bob"
>>> checkpoint = "google/gemma-2-9b"
>>> assistant_checkpoint = "double7/vicuna-68m"

>>> assistant_tokenizer = AutoTokenizer.from_pretrained(assistant_checkpoint)
>>> tokenizer = AutoTokenizer.from_pretrained(checkpoint)
>>> inputs = tokenizer(prompt, return_tensors="pt")

>>> model = AutoModelForCausalLM.from_pretrained(checkpoint)
>>> assistant_model = AutoModelForCausalLM.from_pretrained(assistant_checkpoint)
>>> outputs = model.generate(**inputs, assistant_model=assistant_model, tokenizer=tokenizer, assistant_tokenizer=assistant_tokenizer)
>>> tokenizer.batch_decode(outputs, skip_special_tokens=True)
['Alice and Bob are sitting in a bar. Alice is drinking a beer and Bob is drinking a']

```

## 未来方向

虽然使用do_sample=True的标准辅助生成采用推测采样算法（论文中的算法1），但UAG目前仅支持多项采样。在多项采样中，如果目标模型没有采样与辅助模型相同的Token，则该Token会被自动拒绝，而推测采样则不是这种情况。在实践中，这意味着使用do_sample=True的UAG的吞吐量将低于辅助模型具有相同分词器的情况。未来，我们计划为UAG增加对推测采样的支持。
此外，我们打算将UAG集成到🤗 Transformers的流水线中，以实现更简洁、更流畅的使用。
