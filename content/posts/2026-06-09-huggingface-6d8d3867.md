# 一个Agent如何通过串联两个Hugging Face Space构建3D巴黎画廊

我让一个编码Agent构建一个展示巴黎地标3D高斯泼溅效果的漂亮网站。我从未打开过图像生成器，也从未碰过3D重建工具。该Agent通过直接调用两个Hugging Face Space生成了所有资产（图像和3D泼溅效果），然后将它们整合到一个电影级查看器中。

## 积木式经济进入多媒体领域

Mitchell Hashimoto最近描述了一种他称之为"积木式经济"的转变：构建软件最有效的路径不再是打磨精良的单一整体，而是小型、文档完善的组件，供他人（尤其是Agent）进行组装。他的关键观察是：AI从零开始构建一切的能力尚可，但在将经过验证的组件粘合在一起方面表现出色。

这一论点此前主要围绕代码库展开。但同样的力量正在冲击多媒体AI。使用最先进的图像模型、视频模型、TTS模型或3D重建模型的难点从来不是模型本身，而是集成过程：SDK、权重、GPU、输入格式、轮询。如果每个模型都是一个有文档记录、可调用的模块，Agent就能像拼接npm包一样将它们粘合在一起。

这正是Hugging Face Space悄然演变而成的形态。

## 每个Space都是一个积木块，通过agents.md实现

Hub上托管着数千个最先进的模型（其中很大一部分是开源权重），并且大多数已部署为交互式Space。截至目前，每个Gradio Space都暴露了一个纯文本的agents.md文件，精确告知Agent如何调用它：

```bash
curl https://huggingface.co/spaces/VAST-AI/TripoSplat/agents.md
```

一次性返回所有必要信息：schema URL、调用和轮询模板、如何上传文件以及认证提示：

```
API schema:   GET  .../gradio_api/info
Call endpoint: POST .../gradio_api/call/v2/{endpoint} {"param_name": value, ...}
Poll result:  GET  .../gradio_api/call/{endpoint}/{event_id}
File inputs:  POST .../gradio_api/upload -F "files=@file.ext"
Auth:         Bearer $HF_TOKEN
```

无需客户端库，无需硬编码集成。Agent读取这些信息后就能端到端地驱动Space。设置好HF_TOKEN即可开始。

真正的突破在于串联：一个Space的输出成为下一个Space的输入。提示词 → 图像 → 3D。这就是这个画廊背后的完整流水线。

## 实际案例：巴黎地标 → 泼溅效果

该Agent串联了两个Space：

1. 图像：ideogram-ai/ideogram4将每个地标转化为干净、深色背景的"标本"照片（埃菲尔铁塔则变成了底座上的小立体模型）。输入提示词，输出图像。
2. 泼溅效果：VAST-AI/TripoSplat从每张单张图像重建出3D高斯泼溅效果（.ply文件）。输入图像，输出3D。

随后Agent还完成了"粘合"工作。它发现TripoSplat的输出是Y轴朝下的，便将其翻转直立，自动为每个地标取景，将.ply文件压缩为.ksplat（体积缩小约3倍，加载更快），构建了一个支持滚动切换和拖拽旋转的Three.js查看器，并将整个成果部署为静态Space。唯一的人工输入是审美层面的调整："拉远一点"，"用更适合泼溅效果的东西替换方尖碑"，"过渡动画时间太长"。

其中几个步骤是Agent对现实情况的反应。宽大的玻璃金字塔泼溅效果不佳，细长的方尖碑显得单调，单视角重建需要推断背面。这正是积木式经济所预言的"外包研发、快速迭代"循环，只不过这里的研发过程是一场对话。

## 两个提示词，一个全新画廊

积木块真正的考验在于复用的成本有多低。一旦这条流水线建立起来，生成全新的画廊只需一句话的成本。"创建一个类似的日本泼溅效果Space"，然后同样的指令给埃及，Agent便完成了其余工作：每个国家六张地标图像、六个泼溅效果、压缩、查看器以及部署好的Space。

- 🏛️ 埃及地标：大金字塔、狮身人面像、阿布辛贝神庙、图坦卡蒙面具、卡纳克神庙、门农巨像。
- ⛩️ 日本地标：东京塔、姬路城、金阁寺、大阪城、镰仓大佛、严岛神社鸟居。

同样的两个Space，同样的agents.md，只有提示词变了。这就是积木式经济的精髓：新多媒体应用的边际成本降至描述它的成本。

## 为什么这很重要

- 模型变得可组合。来自不同机构的最先进泼溅模型和图像模型，无需集成代码即可串联。Hub上的开源权重目录变成了一个可调用的多媒体原语库。
- Agent偏好有文档且可访问的内容。agents.md让Space变得极易访问，因此Agent会选择它，而不是需要手动设置的模型。这与Hashimoto对开源库的观察如出一辙。
- 集成曾是障碍，如今已基本消失。"将提示词转化为旋转的3D地标"曾经是一个项目，而在这里只是流水线中的一个步骤。
