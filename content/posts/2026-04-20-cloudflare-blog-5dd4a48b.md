# 构建智能体云：2026年智能体周期间我们发布的一切

2026-04-20

- 明路
- 安妮·王

![](/images/posts/76e16aaf07c8.png)

今天标志着我们首届“智能体周”的结束，这是一个完全致力于智能体时代的创新周。它的举办恰逢其时：在过去一年中，智能体迅速改变了人们的工作方式。编码智能体正在帮助开发者以前所未有的速度交付产品。支持智能体端到端地处理工单。研究智能体能在几分钟内验证跨越数百个来源的假设。人们不仅仅运行一个智能体：他们并行且全天候地运行多个智能体。

正如 Cloudflare 的首席技术官 Dane Knecht 和产品副总裁 Rita Kozlov 在我们欢迎来到智能体周的文章中指出的那样，智能体的潜在规模是惊人的：即使世界上只有一小部分知识工作者每人并行运行几个智能体，你也需要能够支持数千万个并发会话的计算能力。云基础设施所基于的“一个应用服务多个用户”的模式无法满足这种需求。但这正是开发者和企业想要做的：构建智能体，将其部署给用户，并进行大规模运行。

实现这一目标意味着要解决整个技术栈的问题。智能体需要能够从完整操作系统扩展到轻量级隔离环境的**计算**能力。它们需要内置于其运行方式中的**安全**和身份认证。它们需要一个**智能体工具箱**：合适的模型、工具和上下文来执行实际工作。智能体生成的所有代码都需要一条从下午的**原型**到生产**应用**的清晰路径。最后，随着智能体驱动着越来越多的互联网流量，网络本身也需要适应新兴的**智能体网络**。事实证明，我们八年前推出的无容器、无服务器计算平台 Workers 正是为此而生。自那时起，我们已经将其发展成为一个完整的平台，本周我们发布了专为智能体构建的下一波基础组件，正是围绕这些问题组织的。

我们致力于创建 Cloud 2.0 —— **智能体云**。这是为智能体成为主要工作负载的世界而设计的基础设施。

以下是我们本周发布的所有内容列表——我们不希望您错过任何一项。

## 计算

一切从计算开始。智能体需要运行的地方，以及存储和运行它们所编写代码的地方。并非所有智能体都需要相同的东西：有些需要完整的操作系统来安装软件包和运行终端命令，大多数则需要轻量级的环境，能够毫秒级启动并扩展到数百万个实例。本周我们发布了运行它们的环境，以及一个面向智能体的、兼容 Git 的新工作空间：
