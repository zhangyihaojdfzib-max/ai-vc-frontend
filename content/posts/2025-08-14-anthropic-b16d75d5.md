# Claude Opus 4.1

今天我们发布 Claude Opus 4.1，这是对 Claude Opus 4 在智能体任务、真实世界编码和推理能力方面的升级。我们计划在未来几周内发布对模型更大幅度的改进。

Opus 4.1 现已面向付费 Claude 用户和在 Claude Code 中提供。它也已在我们的 API、Amazon Bedrock 和 Google Cloud 的 Vertex AI 上可用。定价与 Opus 4 相同。

Opus 4.1 将我们在 SWE-bench Verified 上的最先进编码性能提升至 74.5%。它还提升了 Claude 的深度研究和数据分析技能，特别是在细节跟踪和智能体搜索方面。

![显示 Claude 在流行编码评估上进展的图表](/images/posts/e02d270d3dea.jpg)

GitHub 指出，Claude Opus 4.1 在大多数能力上相对于 Opus 4 都有所提升，在多文件代码重构方面的性能提升尤为显著。Rakuten Group 发现，Opus 4.1 擅长在大型代码库中精确定位需要修正的地方，而无需进行不必要的调整或引入错误，他们的团队在日常调试任务中更青睐这种精确性。Windsurf 报告称，在其初级开发者基准测试中，Opus 4.1 相比 Opus 4 带来了一个标准差的改进，显示出与从 Sonnet 3.7 升级到 Sonnet 4 大致相同的性能飞跃。

基准测试报告

Claude 模型是混合推理模型。本博文中报告的基准测试显示了使用或不使用扩展思考所获得的最高分数。我们在下面为每个结果注明了是否使用了扩展思考：

- 未使用扩展思考：SWE-bench Verified, Terminal-Bench
- 以下基准测试报告使用了扩展思考（最多 64K Token）：TAU-bench, GPQA Diamond, MMMLU, MMMU, AIME

TAU-bench 方法

通过在 Airline 和 Retail Agent Policy 的提示词附录中添加内容，指导 Claude 在使用扩展思考配合工具使用时更好地利用其推理能力，从而获得了这些分数。在多轮轨迹中，鼓励模型在解决问题时写下其想法（这有别于我们通常的思考模式），以最大限度地利用其推理能力。为了适应 Claude 因进行更多思考而产生的额外步骤，最大步骤数（按模型完成次数计算）从 30 步增加到 100 步（大多数轨迹在 30 步内完成，只有一个轨迹超过了 50 步）。

SWE-bench 方法

对于 Claude 4 系列模型，我们继续使用相同的简单框架，该框架仅为模型配备我们先前发布中描述的两个工具——一个 bash 工具和一个通过字符串替换操作的文件编辑工具。我们不再包含 Claude 3.7 Sonnet 使用的第三个“规划工具”。在所有 Claude 4 模型上，我们报告的是全部 500 个问题的分数。OpenAI 模型的分数报告基于一个 477 个问题的子集。
