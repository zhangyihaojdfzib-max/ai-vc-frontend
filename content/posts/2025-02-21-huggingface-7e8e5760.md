# SigLIP 2：更优秀的多语言视觉语言编码器

## 摘要

今天，谷歌发布了一系列新的、更优秀的多语言视觉语言编码器——SigLIP 2。作者在SigLIP（sigmoid损失函数）的训练目标基础上，扩展了额外的训练目标，以提升语义理解、定位能力和密集特征。

![新增训练目标](/images/posts/ed7be5bb0599.png)

SigLIP 2模型在所有模型规模上，在核心能力方面均优于旧版SigLIP，包括零样本分类、图像-文本检索，以及在为视觉语言模型（VLM）提取视觉表示时的迁移性能。

锦上添花的是动态分辨率（naflex）变体。这对于对宽高比和分辨率敏感的下游任务非常有用。

## 引言

视觉编码器很简单——它们接收图像，将其编码为一种表示，然后这种表示被用于下游任务，如分类、目标检测、图像分割以及更多视觉任务。研究人员一直在追求密集、具有局部感知能力且语义丰富的视觉表示。

CLIP和ALIGN是通过联合训练将图像编码器和文本编码器对齐的首批示例。这种方法为训练视觉模型开辟了新途径。SigLIP更进一步，用sigmoid损失函数取代了CLIP的对比损失函数，从而得到更优秀的编码器。

关键点是什么？通过更智能的训练目标，我们不断构建结构更清晰、粒度更细、能力更强的视觉编码器。SigLIP 2正是如此，它在SigLIP的训练目标之上应用了一系列非常有趣且智能的训练目标，以提供更好、更强的视觉语言编码器。

## 添加一个解码器（就这么简单）

让我们在组合中添加一个解码器。现在我们有了一个图像编码器、一个文本编码器和一个文本解码器。文本解码器将有三个目标：

1. 预测整体图像描述
2. 根据描述特定图像区域的描述预测边界框坐标
3. 根据边界框坐标预测特定区域的描述

解码器为视觉编码器提供了额外的信号，使其具有位置感知能力。这标志着SigLIP 2训练方案的第一项改进。

🤔 问题2：我们如何改进图像表示的细粒度局部语义？

## 使用全局-局部损失和掩码预测进行自蒸馏

为了改进图像表示中的细粒度局部语义，我们引入了两个关键的训练目标：全局-局部损失和掩码预测损失。从自监督学习文献中汲取灵感，我们使用自蒸馏。我们可以将一个模型作为教师，同一个模型作为学生。在每次迭代中，教师将是学生参数的移动平均值。

1. 全局-局部损失：学生网络获得训练图像的部分（局部）视图，并被训练以匹配从完整图像导出的教师表示。
2. 掩码预测损失：学生网络中50%的嵌入图像补丁被掩码标记遮盖。学生需要匹配教师在掩码位置的特征。

这些目标教会视觉编码器具有空间感知能力，并改进其局部语义。作者仅在80%的训练使用sigmoid损失和解码器损失完成后才添加此损失。这样做是为了节省计算资源（额外的损失相当昂贵）并且不会对编码器产生负面影响。

## 适应不同的分辨率

众所周知，图像模型可能对不同分辨率和宽高比非常敏感。在这里，我们可以利用两种不同的方法来使这些模型适应不同的分辨率和补丁大小。

1. 固定分辨率变体：从95%训练时的检查点开始，我们可以调整位置嵌入和补丁嵌入的大小，然后针对所需的（可能更大的）分辨率继续训练。
2. 动态分辨率变体：从FlexiViT（使用不同序列长度的输入）和NaViT（保持原始宽高比）中汲取灵感，我们可以创建NaFlex变体。这很有趣，因为我们可以使用单个模型进行OCR（宽高比失真小）和文档理解（适当的分辨率）。

带有-naflex后缀的模型是动态分辨率变体。虽然固定分辨率模型可以直接与现有的SiglipModel类一起使用，但您需要使用Siglip2Model来使用naflex变体。当您使用pipeline API时，我们会自动处理这个问题！

### 零样本分类

这里我们使用便捷的pipeline API来展示SigLIP 2的零样本分类能力。

```py
from transformers import pipeline

ckpt = "google/siglip2-so400m-patch14-384"
pipe = pipeline(model=ckpt, task="zero-shot-image-classification")

inputs = {
    "images": [
        "https://huggingface.co/datasets/merve/coco/resolve/main/val2017/000000000285.jpg", 
        "https://huggingface.co/datasets/merve/coco/resolve/main/val2017/000000000776.jpg", 
    ],
    "texts": [
        "bear looking into the camera",
        "bear looking away from the camera",
        "a bunch of teddy bears",
        "two teddy bears",
        "three teddy bears"
    ],
}

outputs = pipe(inputs["images"], candidate_labels=inputs["texts"])

```

### 为下游任务编码图像

您还可以使用以下代码编码图像：

```py
import torch
from transformers import AutoModel, AutoProcessor
from transformers.image_utils import load_image

ckpt = "google/siglip2-so400m-patch14-384"
model = AutoModel.from_pretrained(ckpt, device_map="auto").eval()
processor = AutoProcessor.from_pretrained(ckpt)

image = load_image("https://huggingface.co/datasets/merve/coco/resolve/main/val2017/000000000285.jpg")
inputs = processor(images=[image], return_tensors="pt").to(model.device)

with torch.no_grad():
    image_embeddings = model.get_image_features(**inputs)    

print(image_embeddings.shape) 

```

## 将编码器用于VLM
