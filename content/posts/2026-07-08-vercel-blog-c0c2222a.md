Flags SDK 和 Vercel Flags 现在可以批量评估多个功能开关，速度提升约 10 倍。

通过减少微任务队列开销和创建更少的 Promise，功能开关评估时间得到了改善。这种改进会随着功能开关数量的增加而扩展。

使用 `await evaluate([flagA, flagB])` 替代 `Promise.all([flagA(), flagB()])` 来受益于这些优化：

```
1import { evaluate } from 'flags/next';2import { flagA, flagB } from '../flags';3const [valueA, valueB] = await evaluate([flagA, flagB]);
```

使用数组解构批量评估多个功能开关。

你也可以传递一个对象来评估带有命名键的功能开关：

```
1import { evaluate } from 'flags/next';2import { flagA, flagB } from '../flags';3const { a, b } = await evaluate({ a: flagA, b: flagB });
```

使用对象解构批量评估多个功能开关。

`precompute()` 也会自动受益于这些改进。

升级 `flags` 和 `@flags-sdk/vercel` 到最新版本即可开始使用。更多信息请查阅文档。
