Streamdown 2.2 版本带来了逐词动画文本流式渲染、改进的自定义 HTML 处理能力，以及与 ReactMarkdown 的无缝兼容性，使得在现有项目中采用 Streamdown 变得更加容易。

## 动画流式渲染

通过导入 Streamdown 样式表并启用新的 `animated` 属性，流式内容将以平滑的逐词文本动画效果呈现。这为 AI 聊天界面和其他实时文本应用提供了精致的用户体验。

```
1import { Streamdown } from "streamdown";2import "streamdown/styles.css";3
4export default function Page() {5  return (6    <Streamdown animated isAnimating={status === "streaming"}>7      {markdown}8    </Streamdown>9  );10}
```

## 更优的自定义 HTML 支持

`components` 属性现在支持通过向 `allowedTags` 添加元素来接受自定义 HTML 属性。同时，Remend 引擎也得到了改进，能够在流式传输过程中剥离不完整的 HTML 标签，从而避免因部分标记导致的视觉错误。

```
1<Streamdown2  allowedTags={{3    source: ["id"],  4  }}5  components={{6    source: ({ id, children }) => (7      <button8        onClick={() => console.log(`Navigate to source: ${id}`)}9        className="text-blue-600 underline cursor-pointer"10      >11        {children}12      </button>13    ),14  }}15>16  {markdown}17</Streamdown>
```

## ReactMarkdown 兼容性

Streamdown 现已支持 ReactMarkdown 的其余属性，使得迁移工作只需一行代码的替换即可完成。使用 ReactMarkdown 的项目可以无需重构组件配置，直接切换到 Streamdown。

```
1<Streamdown2  allowedElements={["p", "h1", "h2"]}3  disallowedElements={["img"]}4  unwrapDisallowed={false}5  skipHtml={true}6>7  {markdown}8</Streamdown>
```

此版本还移除了 CommonJS 构建产物，为常见的 JavaScript、TypeScript 和 Shell 标签添加了捆绑语言别名，并修复了表格、代码块、LaTeX 和 Mermaid 图表中的大量渲染与安全问题。

了解更多信息，请查阅 Streamdown 文档。
