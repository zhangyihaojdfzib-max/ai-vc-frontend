# Python 微型智能体：约 70 行代码实现的 MCP 驱动智能体

**新功能**：tiny-agents 现已支持 `AGENTS.md` 标准。🥳

受《Tiny Agents in JS》启发，我们将此概念移植到了 Python 🐍，并扩展了 `huggingface_hub` 客户端 SDK，使其能作为 MCP 客户端运行，从而可以从 MCP 服务器拉取工具，并在推理过程中将其传递给 LLM（大语言模型）。

MCP（模型上下文协议）是一个开放协议，它标准化了 LLM（大语言模型）与外部工具和 API 的交互方式。本质上，它消除了为每个工具编写自定义集成的需要，使得为你的 LLM 接入新功能变得更加简单。

在这篇博客文章中，我们将向你展示如何开始使用一个连接到 MCP 服务器的 Python 微型智能体，以解锁强大的工具能力。你将看到启动你自己的智能体并开始构建是多么容易！

**剧透**：一个智能体本质上就是一个构建在 MCP 客户端之上的 while 循环！

## 我们的小型Python Agent：它（几乎）只是一个循环！

由于`MCPClient`处理了所有工具交互的工作，我们的`Agent`类变得异常简单。它继承自`MCPClient`并添加了对话管理逻辑。

Agent类非常小巧，专注于对话循环，代码可以在[这里](here)找到。

### 1. 初始化Agent

当创建一个Agent时，它接收一个Agent配置（模型、提供商、使用哪些MCP服务器、系统提示词），并使用系统提示词初始化对话历史。然后`load_tools()`方法遍历服务器配置（在`agent.json`中定义），并为每个配置调用父类`MCPClient`的`add_mcp_server`方法，从而填充Agent的工具箱。

```python


class Agent(MCPClient):
    def __init__(
        self,
        *,
        model: str,
        servers: Iterable[Dict], 
        provider: Optional[PROVIDER_OR_POLICY_T] = None,
        api_key: Optional[str] = None,
        prompt: Optional[str] = None, 
    ):
        
        super().__init__(model=model, provider=provider, api_key=api_key)
        
        self._servers_cfg = list(servers)
        
        self.messages: List[Union[Dict, ChatCompletionInputMessage]] = [
            {"role": "system", "content": prompt or DEFAULT_SYSTEM_PROMPT}
        ]

    async def load_tools(self) -> None:
        
        for cfg in self._servers_cfg:
            await self.add_mcp_server(**cfg)

```

在`run()`循环内部：

- 它首先将用户提示词添加到对话中。
- 然后调用`MCPClient.process_single_turn_with_tools(...)`来获取LLM的响应，并处理推理一步中的任何工具执行。
- 每个项目会立即被`yield`，从而实现向调用者的实时流式传输。
- 每一步之后，它会检查退出条件：是否使用了特殊的"退出循环"工具、是否达到最大轮次限制，或者LLM是否提供了看似针对当前请求的最终文本响应。
