## 该技能为您提供什么

Slack Agent 技能为您提供一个能够实现以下功能的 Agent：

- 跨消息和线程进行多轮对话
- 在执行敏感操作前暂停以获取人工批准
- 实时向 Slack 流式传输响应
- 自行读取频道和线程

您的 Agent 通过 **工具** 与 Slack 及您的系统交互，这些工具是它可以调用来执行操作或检索信息的函数。该模板内置了以下工具：

- 读取频道消息
- 获取线程上下文
- 加入频道（需人工批准）
- 按名称、主题或用途搜索频道

您还可以指示您的编程 Agent 添加连接到您自己系统的自定义工具。希望 Agent 查找客户记录、创建支持工单或查询数据库吗？这些都可以成为 Agent 知道何时以及如何调用的工具。


**Workflow DevKit** 是使 Agent 具备持久性的关键。Slack Agent 通常需要跨多条消息进行对话，或者等待数小时以获取某人对请求的批准。Workflow DevKit 允许 Agent 在对话中途暂停，等待外部输入，并在中断处精确恢复。工具调用在失败时会自动重试，并且响应会实时流式传输回 Slack。

**人在回路** 机制是内置的。当 Agent 需要执行敏感操作（如加入频道）时，它会发布一条带有“批准”和“拒绝”按钮的消息并暂停。您只需为 **活跃的 CPU 时间** 付费，因此等待不产生任何费用，即使审批需要数天。这种模式可扩展到任何需要批准的操作，从发送消息到修改数据，再到调用外部 API。

**AI Gateway** 通过单个 API 密钥为您的 Agent 提供访问数百个来自各大提供商的模型的能力。切换模型只需更改一行代码，如果某个提供商出现故障，AI Gateway 会自动路由到另一个，从而确保您的 Agent 保持运行。

```
1import { gateway } from "@ai-sdk/gateway";2
3const result = await generateText({4  model: gateway("anthropic/claude-sonnet-4.6"),5  prompt: userMessage,6});
```
