## 部署DeepSeek R1模型

### 在AWS上使用Hugging Face推理端点进行部署

Hugging Face推理端点提供了一种简单安全的方式，在AWS专用计算资源上部署机器学习模型用于生产环境。推理端点使开发者和数据科学家无需管理基础设施即可创建AI应用：将部署过程简化为几次点击，包括通过自动缩放处理大量请求、通过缩放到零降低基础设施成本，以及提供高级安全性。

通过推理端点，你可以部署DeepSeek-R1的6个蒸馏模型中的任何一个，以及由Unsloth制作的DeepSeek R1量化版本：https://huggingface.co/unsloth/DeepSeek-R1-GGUF。
在模型页面上，点击"部署"，然后点击"HF推理端点"。你将被重定向到推理端点页面，我们已为你选择了优化的推理容器和运行模型的推荐硬件。创建端点后，你可以以每小时8.3美元的价格在AWS上向DeepSeek R1发送查询🤯。

你可以在推理端点模型目录中找到DeepSeek R1和蒸馏模型，以及其他流行的开源LLM（大语言模型），它们已准备好以优化配置进行部署。

然后，实例化一个sagemaker_session，用于确定当前区域和执行角色。

```python
import boto3
```

```python
model_id = "deepseek-ai/DeepSeek-R1-Distill-Llama-70B"

    "SM_NUM_GPUS": json.dumps(8)
}
```

```python
image_uri = get_huggingface_llm_image_uri("huggingface-neuronx", version="0.0.25")
```

### 在EC2 Neuron上使用Hugging Face Neuron深度学习AMI进行部署

本指南将详细介绍如何在`inf2.48xlarge` AWS EC2实例上导出、部署和运行DeepSeek-R1-Distill-Llama-70B。

在开始之前，先确保满足几个前提条件。请确认你已在Marketplace上订阅了Hugging Face Neuron深度学习AMI。它提供了在Trainium和Inferentia上训练和部署Hugging Face模型所需的所有依赖项。然后，在EC2中使用该AMI启动一个`inf2.48xlarge`实例，并通过SSH连接。如果你从未操作过，可以查看我们的分步指南。

一旦通过实例连接，你可以使用以下命令在端点上部署模型：

```bash
docker run -p 8080:80 \
    -v $(pwd)/data:/data \
    --device=/dev/neuron0 \
    --device=/dev/neuron1 \
    --device=/dev/neuron2 \
    --device=/dev/neuron3 \
    --device=/dev/neuron4 \
    --device=/dev/neuron5 \
    --device=/dev/neuron6 \
    --device=/dev/neuron7 \
    --device=/dev/neuron8 \
    --device=/dev/neuron9 \
    --device=/dev/neuron10 \
    --device=/dev/neuron11 \
    -e HF_BATCH_SIZE=4 \
    -e HF_SEQUENCE_LENGTH=4096 \
    -e HF_AUTO_CAST_TYPE="bf16" \
    -e HF_NUM_CORES=24 \
    ghcr.io/huggingface/neuronx-tgi:latest \
    --model-id deepseek-ai/DeepSeek-R1-Distill-Llama-70B \
    --max-batch-size 4 \
    --max-total-tokens 4096

```

这将花费几分钟时间从Hugging Face缓存下载编译后的模型，并启动一个TGI端点。

然后，你可以测试该端点：

```bash
curl localhost:8080/generate \
    -X POST \
    -d '{"inputs":"为什么夜晚的天空是黑暗的？"}' \
    -H 'Content-Type: application/json'

```

测试完成后，请确保暂停EC2实例。
