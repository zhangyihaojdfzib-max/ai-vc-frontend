通过提供规范性和权威性的指导，组织可以将这一上下文层叠到其现有工具包中，从而更好地利用他们已在部署的安全产品。

## 什么是 Cloudflare One 堆栈？

Cloudflare One 堆栈是一套技能集合，可与任何 Agent（智能体）配合使用。与任何技能一样，您可以单独使用它们，叠加您自己的上下文，或在其之上构建工具。它是专门为帮助安全从业人员完成评估、部署和管理 Cloudflare One 的整个生命周期而构建的。

该堆栈通过综合员工手工整理的知识而构建，这些员工拥有数万小时与客户合作使用 Cloudflare One 产品的经验。它包含了在 Cloudflare 上规划、管理和实施用户及 Agent（智能体）安全基础设施的工具。它还包含了从 Zscaler 和 Palo Alto Networks 等传统供应商迁移的手工精选逻辑。

当与 Cloudflare 代码模式 MCP 服务器结合使用时，该堆栈为 Agent（智能体）提供了 Cloudflare API 的类型化接口。Agent（智能体）可以查询您的实时账户、检查配置，并通过一套精心策划的 Cloudflare 推荐工作流（而非临时 API 调用）进行更改。

## 堆栈中包含什么？

Cloudflare One 堆栈以两个轻量级技能文件形式提供：cloudflare-one 和 cloudflare-one-migration。它们共同涵盖了迁移至、构建实施方案、管理以及故障排除您的 Cloudflare One 部署：

- 通过 Cloudflare Access 实现远程访问和 VPN 替代
- 通过 Cloudflare Gateway 实现用户、网络、设备和数据安全
- 通过 Cloudflare Tunnel、Cloudflare Mesh 和 Cloudflare WAN 实现连接
- 迁移指导，包含从其他 SASE 供应商迁移的明确细节
- 网络拓扑图解读与生成，使您和团队能够以易于理解的方式可视化拟议的网络变更
- 供应商概念转换，映射 SASE 供应商之间的概念，以降低评估和切换供应商的门槛
- 故障排除与运维，包含数字体验监控（DEX）工具包和自动化规则建议

## 工作原理

该堆栈可在 Cloudflare Skills 仓库中获取。每个技能文件包含结构化知识、决策树和工具定义，当上下文匹配时，Agent（智能体）会自动加载这些内容。将其交给您的 Agent（智能体），让它帮助您设置、配置和管理零信任环境：

cloudflare-one 技能涵盖通用产品指导。例如，如果您询问 Agent（智能体）用 Cloudflare Tunnel 或 Cloudflare Mesh 替换 VPN 基础设施的最佳方式，该技能知道如何：

1. 盘点您现有的 VPN 应用程序，并识别每个应用程序所需的连接模型
2. 将每个应用程序映射到适当的 Cloudflare 原语——自托管 Access 应用程序、Tunnel 连接服务或 Mesh 连接网段
3. 生成推荐的部署顺序，以最大程度减少切换期间的干扰
4. 生成配置摘要，供您的团队在进行任何更改前审查

cloudflare-one-migration 技能涵盖供应商到供应商的转换。例如，如果您要求 Agent（智能体）将您的 Zscaler Private Access 应用程序迁移到 Cloudflare Access，该技能知道如何：

1. 将 Zscaler 应用程序定义映射到 Cloudflare Access 应用程序定义
2. 将 Zscaler 用户组和策略转换为 Cloudflare Access 策略
3. 使用 Cloudflare API 在您的账户中创建等效资源
4. 生成迁移摘要以及需要手动审查的内容摘要

堆栈中的迁移逻辑与 Cloudflare 的 Descaler 和 Deskope 程序中使用的逻辑相同。这些程序已使企业客户在数小时内（而非数月内）从 Zscaler 和 Netskope 迁移到 Cloudflare One。该堆栈使任何客户或合作伙伴随时都能获得这一能力，无需等待预定的项目安排。
