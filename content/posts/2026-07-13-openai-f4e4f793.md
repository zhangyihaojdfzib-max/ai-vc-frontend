# GPT‑5.6：与您雄心相匹配的前沿智能

每个Token带来更多智能，每美元带来更强性能，为您最艰巨的工作提供按需增强的能力。

在有限预览之后，我们正式全面发布GPT‑5.6模型系列：我们的新旗舰模型Sol，面向日常工作的均衡模型Terra，以及我们最具成本效益的模型Luna。

GPT‑5.6 Sol在智能和效率两方面树立了新标准，在编程、知识工作、网络安全和科学领域取得了最先进成果，同时以更少的Token和更低的预估成本超越了此前及竞品的前沿模型。结果是更强的每美元性能：相同投入下完成更多成功工作，或以更低总成本获得可比结果。我们还引入了一种加速最艰巨工作的新方式：ultra是我们最高能力的设置，通过协调多个Agent跨并行工作流，更快完成复杂任务。更强的计算机使用能力和设计判断力使GPT‑5.6 Sol成为我们迄今为止最精良的协作者，帮助其检查、优化并交付可直接使用的结果。

我们训练GPT‑5.6从每个Token中获取更多有用工作。在覆盖55个领域长期专业工作流的评估——Agent最后考试中，GPT‑5.6 Sol以53.6分创下新高，领先Claude Fable 5（自适应推理）13.1分。即使在中等推理水平下，它也比Fable 5高出11.4分，而预估成本仅为其四分之一左右。这种效率延伸至更小的模型，这些模型对于让智能更丰富、更经济至关重要：GPT‑5.6 Terra和GPT‑5.6 Luna以约十六分之一的成本超越了Fable 5。在人工智能分析智能指数上，这是一项涵盖Agent工作、编程、科学推理和通用能力的广泛智能衡量指标，GPT‑5.6 Sol在最大推理模式下与Fable 5的差距仅在一分以内，同时完成任务的时间缩短61%，预估成本约降低一半。

Agent最后考试：跨专业领域的长期Agent工作流。

GPT‑5.6发布时搭载了我们迄今为止最强大的安全防护措施，旨在抵御蓄意和适应性滥用，同时不广泛限制合法工作。在全面发布前，我们通过结合人工红队测试和大规模自动化测试，对模型和安全措施进行了迄今为止最广泛的评估。在预览期间，我们与专家组织及可信合作伙伴密切合作，在更广泛发布前对防御措施进行压力测试并加强安全防护。由此产生的系统将训练到模型中的防护层与实时检查、监控以及根据信任和风险校准的访问权限相结合。

## 默认高效，按需提供最大性能

GPT‑5.6 Sol是我们迄今为止最优秀的编程模型。在人工智能分析编程Agent指数上，GPT‑5.6 Sol在最大推理模式下以80分创下最新最先进水平，比Fable 5高出2.8分，同时输出Token减少一半以上，时间缩短一半以上，成本降低约三分之一。这一优势贯穿整个系列：Terra的表现略高于Fable 5，而Luna则超越了Opus 4.8；每个模型完成工作的时间约为三分之一，输出Token约为一半，预估成本约为四分之一。它还在Terminal‑Bench 2.1和DeepSWE上创下了最新最先进结果，这两项测试评估了真实代码库中的复杂命令行工作流和长期工程能力。

人工智能分析编程Agent指数：一项涵盖实现、终端使用和真实代码库的编程Agent性能独立指数。

GPT‑5.6可以编写和运行轻量级程序，协调工具、处理中间结果、监控进度，并在工作推进过程中选择下一步行动。这使得工具密集型任务能够以更少的Token、更少的模型往返和更少的指导来推进。响应API中的程序化工具调用无需开发者编写每一步脚本或将每个工具响应传回模型，即可过滤大量中间数据，仅保留关键信息，并沿途调整工作流。

对于需要更多时间和算力投入的问题，GPT‑5.6可以超越这种默认高效模式。max模式给予GPT‑5.6比xhigh更多的时间进行推理、探索替代方案、运行检查并修正方法。ultra模式则更进一步，默认协调四个并行Agent，以更高的Token使用换取更强结果和更快的任务完成时间。下图比较了ultra的默认四Agent设置与单Agent基线在BrowseComp、SEC-Bench Pro和Terminal-Bench 2.1上的表现；BrowseComp和SEC-Bench Pro还展示了16个Agent的配置。在所有三项评估中，增加并行Agent将得分-延迟前沿曲线向左上方移动，在更短时间内达到更强结果。在API中，开发者可以使用响应API中的多Agent测试版构建类似ultra的体验。

## 推动网络与科学前沿

GPT‑5.6 是我们迄今为止最强大的网络安全模型，以显著更少的 Token 实现了前沿性能。在衡量从接触易受攻击代码到实现任意代码执行进展的 ExploitBench2 上，它在可比输出 Token 预算下得分为 73.5%，而 GPT‑5.5 为 47.9%。在要求 Agent 将真实世界漏洞转化为有效利用的 ExploitGym3 上，它在两小时限制下的最高通过率几乎翻倍，从 GPT‑5.5 的 15.1% 提升至 24.9%；在六小时限制下，则达到 33.7%。在测试复杂软件概念验证生成的 SEC-Bench Pro 上，它以更低的延迟获得了 71.2% 的分数，而 GPT‑5.5 为 45.8%。

GPT‑5.6 支持重要的防御性任务，如安全代码审查、补丁修复、威胁建模和蓝队行动。OpenAI Daybreak 的“网络可信访问”计划中符合条件的个人和组织，可以通过对授权环境中经过验证的工作（包括漏洞分类与验证、恶意软件分析、检测工程和补丁验证）实施更精确的安全保障，来访问其更多的防御能力。

## GPT‑5.6 加速 OpenAI 发展

GPT‑5.6 是我们迄今为止加速 AI 研究的最强模型。在 OpenAI 内部，研究人员在开发循环中广泛使用它：诊断故障、优化训练系统、运行实验和解释结果。在 GPT‑5.6 的内部测试期间，我们已经看到了这种加速和更强的采用率，每位活跃研究人员的日均输出 Token 量是 GPT‑5.5 观察到的最高水平的两倍多。

这种工作方式正迅速成为标准。在过去六个月中，用于内部编码推理的研究计算份额增长了 100 倍，而内部 Agent Token 使用量增长了约 22 倍。这些采用指标本身并不能衡量研究进展，但它们显示了 AI 辅助在研究以及销售、市场营销、用户运营、财务等其他团队中的增长有多迅速。

为了直接衡量这种能力，我们基于真实的 AI 研究任务开发了一套内部评估套件，包括调试研究系统、优化内核和训练方案、运行机器学习实验以及改进另一个模型。

聚合 RSI 能力：在一套衡量递归自我改进进展的评估中，我们观察到 GPT‑5.6 Sol 相比 GPT‑5.5 提升了 16.2 个百分点，全面加速了内部研究。

## 随能力提升扩展安全与保障

随着模型能力的增强，我们强化了安全堆栈，以便先进智能能够保持广泛实用性，同时对最高风险的使用施加更严格的审查。对于 GPT‑5.6，我们构建了迄今为止最强大的安全系统，该系统根据每个模型的能力进行校准，并由比以往更多的算力驱动。

GPT‑5.6 模型在生物学和网络安全方面都比我们早期的模型能力更强，但在这两个类别中均未达到“临界”阈值。在网络安全方面，我们的测试表明，GPT‑5.6 更擅长发现和修复漏洞，而不是可靠地对加固目标进行自主、端到端的攻击——这为防御者在漏洞被利用之前加强系统提供了机会。在生物学方面，我们的测试表明，GPT‑5.6 可以支持合法研究，但不具备创造、设计或合成高度危险的新型威胁所需的端到端能力。

这两个领域本质上都具有双重用途。在网络安全领域，帮助攻击者利用漏洞的相同能力也可以帮助防御者发现、复现漏洞并构建可靠的修复方案。因此，过度封锁本身会带来安全风险。它可能阻止防御者测试系统和部署补丁，而恶意行为者则继续使用其他模型（包括能力日益增强的开源模型）以及成熟的工具。有效的安全措施会考虑请求的上下文和可能后果，在证据表明存在严重伤害风险时施加更强控制，同时保护合法的防御性工作。

GPT‑5.6 的安全措施是分层设计的，以提高准确性和冗余性，并旨在随着新攻击的出现而快速适应。训练到模型中的保护措施与实时检查、持续监控和账户级强制执行协同工作，即使在特定层级未能按预期工作时，也有助于系统保持安全。在许多系统中，仅靠分类器标志来决定阻止什么，依赖更难更改的低智能模型来防止伤害。我们的方法增加了一个推理监控器，用于审查对话以确定是否存在潜在伤害。这种设计旨在实现防御性工作的同时阻止严重滥用，最敏感的能力通过“可信访问”保留给经过验证的用户。由于某些保护措施使用了测试时推理，我们可以快速更新它们以弥补漏洞，而无需从头重新训练分类器。

我们在继续加强系统以抵御自适应攻击的同时，采取了更为保守的方法。与之前的模型相比，我们的 GPT‑5.6 Sol 网络安全措施阻止的潜在有害活动大约增加了十倍。由于这些措施可能会给良性使用带来摩擦，我们在 ChatGPT 和 Codex 中提供了一个选项，可以轻松地在能力较低的模型上重试提示词，我们将继续在保持高鲁棒性标准的同时，减少安全措施对良性使用的影响。这反映了我们的迭代部署方法：从保守开始，并根据从实际使用中学到的经验进行改进。

在全面可用之前，我们进行了迄今为止最密集的安全评估，包括广泛的红队测试、与外部专家进行的稳健能力和安全措施测试，以及大约 70 万 A100e GPU 小时的黑盒自动化红队测试。这使我们能够系统地探测可能的薄弱点，发现越狱漏洞，并帮助我们在发布前加强系统。

不存在绝对的安全，我们为日益强大的模型提供安全保障的工作仍在继续。新的弱点会被发现，新的越狱手段也会绕过现有的防护措施。每一代新模型都会带来新的攻击和滥用途径。我们通过分层防护、持续监控、快速修复以及防御社区的协作，来应对这一现实。针对 GPT‑5.6，我们将现有的安全⁠(在新窗口中打开)和生物学漏洞赏金计划与新的快速修复流程以及迄今为止最强大的监控工作相结合。来自研究人员、监控和真实世界滥用的发现将不断反馈到新的评估和更强的防护措施中。

在更新的 GPT‑5.6 系统卡⁠(在新窗口中打开)中了解更多关于我们防护措施的信息。

## 可用性与定价

GPT‑5.6 涵盖三个模型层级：Sol，我们的旗舰模型；Terra，一款性能与 GPT‑5.5 相当的低成本模型；以及 Luna，我们最快且最经济的模型。数字代表代际，而 Sol、Terra 和 Luna 是持久的能力层级，可以按照自己的节奏进行升级。

GPT‑5.6 自即日起在 ChatGPT、Codex 和 OpenAI API 上可用。全球范围内的发布现已开始，并将在未来 24 小时内逐步推进至全面可用。

- **聊天**：Plus、Pro、Business 和 Enterprise 用户可以通过中等和更高努力程度设置访问 GPT‑5.6 Sol。Pro 和 Enterprise 用户还可以选择 GPT‑5.6 Sol Pro，以在复杂任务上获得最高质量的结果。
- **ChatGPT Work 和 Codex**：Free 和 Go 用户访问 GPT‑5.6 Terra。Plus、Pro、Business 和 Enterprise 用户可以在 GPT‑5.6 Sol、Terra 和 Luna 之间进行选择，并为每个模型设置努力程度。maxi 对所有在 ChatGPT Work 和 Codex 中拥有 GPT‑5.6 访问权限的用户可用，并可在设置中开启。在 ChatGPT Work 中，ultra 对 Pro 和 Enterprise 用户可用。在 Codex 中，它对 Plus 及更高版本的用户可用。
- **API**：开发者可以通过 OpenAI API 访问 Sol、Terra 和 Luna。在 Responses API 中，程序化工具调用让 GPT‑5.6 可以在内存中编写和运行程序，以协调工具并处理中间结果，使其兼容零数据保留（ZDR）。多 Agent（智能体）功能（最初以 Beta 版提供）让 GPT‑5.6 能够运行并发子 Agent（智能体），并在单个请求中综合它们的工作成果。

GPT‑5.6 按每 100 万个 Token 定价，涵盖三个模型尺寸：Sol 为 $5 输入 / $30 输出；Terra 为 $2.50 输入 / $15 输出；Luna 为 $1 输入 / $6 输出。GPT‑5.6 还引入了更可预测的提示词缓存，包括对显式缓存断点⁠(在新窗口中打开)的支持以及至少 30 分钟的缓存生命周期。对于 GPT‑5.6 及后续模型，缓存写入按模型未缓存输入费率的 1.25 倍计费，而缓存读取则继续享受 90% 的缓存输入折扣。
