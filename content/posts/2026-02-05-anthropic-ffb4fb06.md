# 推出 Claude Opus 4.6

我们正在升级我们最智能的模型。

全新的 Claude Opus 4.6 在其前身的编码能力基础上进行了改进。它规划更周密，能更持久地执行智能体任务，在更大的代码库中运行更可靠，并拥有更好的代码审查和调试能力以发现自身错误。此外，作为我们 Opus 系列模型的首次尝试，Opus 4.6 在测试版中提供了 100 万 Token 的上下文窗口。

Opus 4.6 还能将其改进的能力应用于一系列日常工作任务：进行财务分析、开展研究、使用和创建文档、电子表格和演示文稿。在 Claude 可以自主多任务处理的 Cowork 中，Opus 4.6 可以代表您运用所有这些技能。

## 产品和API更新

我们对Claude、Claude Code和Claude开发者平台进行了大量更新，以让Opus 4.6发挥其最佳性能。

**Claude开发者平台**

在API方面，我们为开发者提供了更好的模型"努力程度"控制，并为长时间运行的Agent（智能体）提供了更大的灵活性。为此，我们引入了以下功能：

*   **自适应思考**。此前，开发者只能在启用或禁用扩展思考之间进行二元选择。现在，通过**自适应思考**，Claude可以自行判断何时进行更深层次的推理会有所帮助。在默认的努力程度（高）下，模型会在有用时使用扩展思考，但开发者可以调整努力程度，使其选择性更强或更弱。
*   **努力程度**。现在有四种**努力程度**可供选择：低、中、高（默认）和最大。我们鼓励开发者尝试不同的选项，以找到最适合的方案。
*   **上下文压缩（测试版）**。长时间运行的对话和Agent（智能体）任务经常会触及上下文窗口限制。**上下文压缩**会在对话接近可配置的阈值时，自动总结并替换较旧的上下文，使Claude能够执行更长的任务而不会触及限制。
*   **100万Token上下文（测试版）**。Opus 4.6是我们首个拥有100万Token上下文的Opus级别模型。对于超过20万Token的提示词，将适用溢价定价（每百万输入/输出Token分别为10美元/37.50美元）。
*   **12.8万输出Token**。Opus 4.6支持高达12.8万Token的输出，这使得Claude能够完成需要更大输出的任务，而无需将其拆分为多个请求。
*   **仅限美国的推理**。对于需要在美国运行的工作负载，**仅限美国的推理**功能可用，定价为Token价格的1.1倍。

**产品更新**

在Claude和Claude Code中，我们增加了新功能，让知识工作者和开发者能够使用更多他们日常使用的工具来处理更困难的任务。

我们已在Claude Code中引入**Agent（智能体）团队**作为研究预览。您现在可以启动多个并行工作的Agent（智能体）作为一个团队，并自主协调——这最适合拆分为独立的、读取密集型工作的任务，例如代码库审查。您可以使用Shift+上/下键或`tmux`直接接管任何子Agent（智能体）。

Claude现在也能更好地与您已使用的办公工具协同工作。Excel中的Claude以改进的性能处理长时间运行和更困难的任务，并且可以行动前先规划，无需指导即可处理非结构化数据并推断正确的结构，以及一次性处理多步骤更改。将其与PowerPoint中的Claude结合使用，您可以先在Excel中处理和构建数据，然后在PowerPoint中将其可视化呈现。Claude会读取您的布局、字体和幻灯片母版以保持品牌一致性，无论您是从模板构建还是根据描述生成完整的演示文稿。PowerPoint中的Claude现已作为研究预览版，向Max、Team和Enterprise计划用户开放。
