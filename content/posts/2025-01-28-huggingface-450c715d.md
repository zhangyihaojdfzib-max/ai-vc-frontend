# 欢迎使用 Hub 上的推理提供商 🔥

今天，我们正式在 Hub 的模型页面集成了四个出色的无服务器推理提供商——fal、Replicate、Sambanova、Together AI。它们也已无缝集成到我们的客户端 SDK（适用于 JS 和 Python），让您比以往更轻松地探索在您偏好的提供商上运行的各种模型的无服务器推理。

![推理提供商](/images/posts/e4f4a0e46ec4.png)

长期以来，我们一直在 Hub 上托管无服务器推理 API（我们于 2020 年夏季推出了 v1 版本——哇，时间过得真快 🤯）。虽然这实现了便捷的探索和原型开发，但我们已将核心价值主张优化为与社区协作、存储、版本控制以及大型数据集和模型的分发。与此同时，无服务器提供商蓬勃发展，Hugging Face 通过一系列优秀的提供商提供便捷统一的无服务器推理访问的时机已经成熟。

正如我们与 AWS、Nvidia 等优秀合作伙伴通过模型页面的“部署”按钮提供专用部署选项一样，与下一代无服务器推理提供商合作提供以模型为中心的无服务器推理也是自然而然的选择。

以下以 DeepSeek-ai/DeepSeek-R1 为例说明这一功能，该模型在过去几天内获得了主流关注 🔥：

Rodrigo Liang，SambaNova 联合创始人兼首席执行官：“我们很高兴能与 Hugging Face 合作，加速其推理 API。Hugging Face 开发者现在可以在大量优秀的开源模型上获得更快的推理速度。”

Zeke Sikelianos，Replicate 创始设计师：“Hugging Face 是开源模型权重的事实上的家园，并且一直是让 AI 更易于全球访问的关键参与者。我们在 Replicate 内部使用 Hugging Face 作为我们的权重注册表，我们很荣幸成为本次发布中首批亮相的推理提供商之一。”

这仅仅是个开始，我们将在未来几周内与社区在此基础上继续发展！

## 工作原理

### 在网站 UI 中

1. 在您的用户账户设置中，您可以：

- 为您已注册的提供商设置自己的 API 密钥。否则，您仍然可以使用它们——您的请求将通过 HF 路由。
- 按偏好排序提供商。这适用于模型页面中的小部件和代码片段。

![推理提供商](/images/posts/6e455540f31b.png)

1. 正如我们提到的，调用推理 API 时有两种模式：

- 自定义密钥（请求直接发送到推理提供商，使用您自己的相应推理提供商的 API 密钥）；或
- 由 HF 路由（在这种情况下，您不需要提供商的 Token，费用直接计入您的 HF 账户，而非提供商的账户）

![推理提供商](/images/posts/d2717fd3c654.png)

1. 模型页面展示第三方推理提供商（与当前模型兼容的提供商，按用户偏好排序）

### 从客户端 SDK

#### 使用 Python，通过 huggingface_hub

以下示例展示了如何使用 Together AI 作为推理提供商来使用 DeepSeek-R1。您可以使用 Hugging Face Token 通过 Hugging Face 自动路由，或者如果您有 Together AI API 密钥，也可以使用自己的密钥。

安装 huggingface_hub v0.28.0 或更高版本（发布说明）。

```python
from huggingface_hub import InferenceClient

client = InferenceClient(
    provider="together",
    api_key="xxxxxxxxxxxxxxxxxxxxxxxx"
)

messages = [
    {
        "role": "user",
        "content": "法国的首都是什么？"
    }
]

completion = client.chat.completions.create(
    model="deepseek-ai/DeepSeek-R1", 
    messages=messages, 
    max_tokens=500
)

print(completion.choices[0].message)

```

注意：您也可以使用 OpenAI 客户端库来调用推理提供商；请参阅此处关于 DeepSeek 模型的示例。

以下是如何使用在 fal.ai 上运行的 FLUX.1-dev 从文本提示词生成图像：

```python
from huggingface_hub import InferenceClient

client = InferenceClient(
    provider="fal-ai",
    api_key="xxxxxxxxxxxxxxxxxxxxxxxx"
)


image = client.text_to_image(
    "维米尔风格的拉布拉多犬",
    model="black-forest-labs/FLUX.1-dev"
)

```

要切换到不同的提供商，您只需更改提供商名称，其他一切保持不变：

```diff
from huggingface_hub import InferenceClient

client = InferenceClient(
-	provider="fal-ai",
+	provider="replicate",
    api_key="xxxxxxxxxxxxxxxxxxxxxxxx"
)

```

#### 使用 JS，通过 @huggingface/inference

```js
import { HfInference } from "@huggingface/inference";

const client = new HfInference("xxxxxxxxxxxxxxxxxxxxxxxx");

const chatCompletion = await client.chatCompletion({
    model: "deepseek-ai/DeepSeek-R1",
    messages: [
        {
            role: "user",
            content: "法国的首都是什么？"
        }
    ],
    provider: "together",
    max_tokens: 500
});

console.log(chatCompletion.choices[0].message);

```

### 从 HTTP 调用

我们直接在 huggingface.co 域名下公开路由代理，因此您可以直接调用它，这对于兼容 OpenAI 的 API 非常有用。您只需将 URL 替换为基础 URL：https://router.huggingface.co/{:provider}。

以下是通过 cURL 使用 Sambanova 作为推理提供商调用 Llama-3.3-70B-Instruct 的方法。

```bash
curl 'https://router.huggingface.co/sambanova/v1/chat/completions' \
-H 'Authorization: Bearer xxxxxxxxxxxxxxxxxxxxxxxx' \
-H 'Content-Type: application/json' \
--data '{
    "model": "Llama-3.3-70B-Instruct",
    "messages": [
        {
            "role": "user",
            "content": "法国的首都是什么？"
        }
    ],
    "max_tokens": 500,
    "stream": false
}'

```

## 计费

对于直接请求，即当您使用推理提供商的密钥时，由相应提供商向您收费。例如，如果您使用 Together AI 密钥，则在您的 Together AI 账户上计费。

对于路由请求，即当您通过 Hub 进行身份验证时，您只需支付标准的提供商 API 费率。我们不会额外加价，只是直接传递提供商成本。（未来，我们可能会与提供商合作伙伴建立收入分成协议。）

我们还为登录的免费用户提供少量配额的免费推理，但如果可以的话，请升级到 PRO！
