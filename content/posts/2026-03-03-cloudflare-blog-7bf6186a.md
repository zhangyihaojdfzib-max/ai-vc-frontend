# 演进 Cloudflare 威胁情报平台：可操作、可扩展且无需 ETL

多年来，网络安全行业一直饱受"数据引力"问题的困扰。安全团队被淹没在数十亿行的遥测数据之下，却仍然缺乏可操作的洞察。

威胁情报平台是一个集中式安全系统，用于收集、聚合和组织关于已知及新兴网络威胁的数据。它充当了原始遥测数据和主动防御之间至关重要的连接组织。

Cloudflare 威胁情报平台的底层架构使其有别于其他解决方案。我们通过采用基于分片、SQLite 支持的架构，演进我们的威胁情报平台，消除了对复杂 ETL（提取、转换、加载）流程的需求。通过在边缘直接运行 GraphQL，安全团队现在可以实时可视化和自动化威胁响应。我们不是使用一个庞大的数据库，而是将威胁事件分布在数千个逻辑分片上——这意味着即使在聚合全球数据集中的数百万个事件时，也能实现亚秒级的查询延迟。

通过将我们的全球遥测数据与分析人员执行的人工调查相结合，我们的情报平台创建了一个单一的事实来源，使安全团队能够从观察威胁转变为在整个 Cloudflare 网络中先发制人地阻止它。我们相信，您的情报平台不应该仅仅告诉您某件事是"坏的"；它应该告诉您为什么会发生，幕后黑手是谁，并自动防止它再次发生。

在这篇文章中，我们将探讨一些使 Cloudforce One 体验强大而有效的功能。

## 我们为何要推出威胁情报平台？

当我们在 2022 年宣布 Cloudforce One 团队时，我们很快意识到追踪对手基础设施需要尚不存在的工具。所以我们自己构建了。

最初作为一个内部项目，现已演变成一个为我们的用户设计的、云优先、具备 Agent（智能体）能力的威胁情报平台。我们已经从概念化跨不同数据集的"可观测"事件，发展到构建一个映射威胁整个生命周期的平台。如今，Cloudflare TIP 允许您将行为者与恶意软件关联起来，将案例与指标联系起来，并将所有内容存储在一个统一的生态系统中。

我们正在超越简单的数据访问，为您的 SOC 提供一个完全集成、可视化且自动化的指挥中心。我们构建此 TIP 的动机源于有效威胁情报的核心原则：相关性、准确性和可操作性。我们需要一个高度可扩展的系统，能够集成多个数据集，支持多租户，实现基于组和租户到租户的共享，并在边缘高效扩展。

通过使用 Cloudflare Workers，我们构建了一个确保快速创新的下一代开发者堆栈。我们现在可以将数百万个威胁事件合成为实时图表和示意图，并立即回答关键问题：发生了什么？这意味着什么？

因为我们的 GraphQL 端点构建在驱动威胁事件平台的同一个 Worker 中，所以您的数据始终是实时的，在数据摄取和可用性之间没有任何延迟。无论您是进行复杂分析还是深入探究特定事件，平台都能即时响应。随着 Workers 运行时的演进，我们的 TIP 会自动继承这些优化。例如，智能放置确保我们处理查询的 Worker 物理上靠近它们要分发到的 Durable Objects，从而最小化尾部延迟。而使用更大 CPU 限制和 Hyperdrive 的能力使我们能够在边缘直接维护更高性能的连接池，而不是将逻辑回传到单一数据中心。

### 超越 SIEM：历史背景与情报增强

虽然 SIEM（安全信息和事件管理）专为实时日志聚合和即时告警而设计，但它通常缺乏深度追踪对手所需的专用架构和长期保留。我们的 TIP 通过充当一个专用的情报层来填补这一空白，该层用历史行为者模式来丰富原始日志。我们平台的目标不是取代 SIEM，而是补充它。我们的 TIP 为威胁事件提供长期、结构化的存储——在边缘保留和索引——这是弥合技术遥测数据与高层洞察之间差距所必需的。

Cloudflare 托管防御和威胁情报平台旨在以共生循环的方式运行，为威胁检测和响应创造一个强大的力量倍增器。通过将 TIP 直接与 SOC 集成，分析人员可以为任何告警或事件获得即时、丰富的上下文。SOC 团队不仅能看到一个异常的 IP 地址或可疑的文件哈希，还能立即看到它的历史、它与已知威胁行为者的关联、它在更广泛活动中的角色，以及由 TIP 分析确定的风险评分。这种即时上下文消除了耗时的手动研究，实现了更快、更准确的决策。

相反，当情报分析团队调查事件并搜寻新威胁时，他们的发现成为新情报的重要来源。

新发现的入侵指标会被反馈到 TIP 中，为所有用户丰富平台并增强其自动防御能力。这种持续的反馈循环确保情报始终是最新的，并基于现实世界的观察，提供对威胁态势无与伦比的可见性，使安全团队能够从被动防御转向主动防御态势。

### 消除瓶颈的架构

为了确保 Cloudforce One 的每一份遥测数据都是可操作的，我们必须解决一个根本性的存储问题：如何在数十亿事件上提供低延迟、复杂查询，而无需传统集中式数据库的开销？

我们选择了一个基于 SQLite 支持的 Durable Objects 的分片架构。通过将威胁事件分布在这个高基数存储单元群中，我们确保在大容量数据摄取期间，没有单个数据库成为争用点。每个分片都是一个 Durable Object，为其自身的私有 SQLite 数据库提供一致的事务接口。

这种架构允许我们使用完整的 Cloudflare 开发者堆栈。我们使用 Cloudflare Queues 异步摄取和分发传入的遥测数据，确保大容量攻击峰值不会使我们的写入吞吐量饱和。一旦数据被摄取，就会存储在 R2 中以供长期保留，而"热"索引则保留在 Durable Object 的 SQLite 存储中，以便即时检索。

#### 边缘并行执行

这种方法在搜索时的真正威力得以显现。当用户查询我们的 GraphQL 端点（该端点也在 Worker 中运行）时，平台不会查询单个表。相反，它会并行地将请求分发到多个 Durable Objects。由于 Durable Objects 分布在我们全球网络中，我们可以以最小的延迟聚合结果。在验证用户权限并排除不包含我们事件的分片（按日期）之后，以下是 Worker 如何处理多分片分发的简化视图：

async function fetchFromShards(shards, query) {

  const results = await Promise.all(promises);

这种并行处理能力确保了流畅的体验，无论您是在审计单个数据集的一年历史记录，还是综合账户中所有数据集的一个月活动。通过将计算——即SQL执行——移至数据所在的位置，我们消除了单一庞大数据集的瓶颈。

#### 通过动态图表可视化威胁对手

电子表格上的数字无法讲述故事；模式才能。我们引入了动态可视化功能，帮助您"看清"威胁态势。

- **桑基图**：追踪攻击从源头到目标的流向，识别哪些区域受冲击最严重以及基础设施位于何处。
- **攻击的行业与数据集分布**：让用户能即时切换视角，查看特定攻击活动是否针对您所在的行业（例如金融或零售），抑或是一场广谱的通用攻击。

#### 通过属性映射关联遥测数据

单一指标（如IP地址）若缺乏历史和关联背景，其效用有限。我们将威胁洞察设计为枢纽点，使您能够将多个数据集中的不同威胁事件关联起来，整合为统一连贯的攻击活动或漏洞利用。

平台不再需要手动交叉比对，而是自动将我们内部的攻击者命名映射到公认的行业别名——例如将我们的内部追踪标识与"Fancy Bear"或"APT28"关联。这确保您本地环境的遥测数据能即时与更广泛的全球研究和威胁情报源互通。

#### 搜索、筛选与告警

保存的配置和实时通知功能，可在我们的遥测数据匹配您自定义筛选器的瞬间发出提醒，让您能以边缘速度做出响应。有效的威胁狩猎需要能够按特定技术属性筛选全球遥测数据。平台支持对整个数据集进行高基数搜索——包括IP地址、文件哈希、域名和JA3指纹——搜索结果通常在数秒内返回。

为超越手动搜索模式，您可将这些查询参数保存为固定配置。这些配置将作为实时通知引擎的触发器：当新输入的遥测数据匹配您定义的筛选条件时，平台会向您配置的终端推送告警。这种从拉取式搜索到推送式告警的转变，确保您的安全体系能在匹配数据被我们全球网络摄入的第一时间做出响应。

#### 自动化规则与STIX2导出

只有当情报能切实缩小攻击面时，才称得上"可执行"。我们构建的TIP平台能自动完成原始遥测数据与安全执行之间的转换。

对于使用第三方或内部SIEM/SOAR平台的组织，互操作性至关重要。然而，将不同的内部数据模式映射到STIX2（结构化威胁信息表达）标准传统上是高延迟的ETL任务。我们已将这种转换移至边缘。

当用户请求STIX2导出时，Worker会动态地将我们内部的SQLite记录映射到STIX2 JSON模式。这意味着我们首先将原始IP地址、文件哈希和域名转换为标准化的STIX网络可观测对象。接着，我们利用平台内部映射定义关系对象，将**指标**对象与**威胁行为者**或**恶意软件**对象关联，保留调查背景。最后，我们自动以UTC时间管理**修改时间**和**创建时间**戳，确保您的下游工具能追踪威胁演变过程。

#### 通过防火墙API实现即时防护

除了导出功能，平台还支持您在发现与防御之间形成闭环。当您在桑基图或特定攻击者活动中识别出恶意模式时，只需点击即可生成安全规则。

底层机制上，TIP直接与Cloudflare防火墙规则API交互。它会提取您调查中的筛选属性（例如特定JA3指纹结合已知恶意ASN列表），并将其编译成网络协议规则，该规则可在数秒内部署至我们的全球网络。

### 人机协同情报

虽然自动化处理了大部分遥测数据，但最复杂的威胁仍需人类直觉判断。我们直接将**信息请求门户**集成到平台中，使用户能够委托Cloudforce One分析师进行深度调查。

从技术角度看，RFI系统不仅是工单门户，更是数据增强管道。当订阅者使用若干"令牌"发起请求时，工作流将触发一系列事件：

- RFI Worker从分片SQLite存储中提取与查询相关的特定威胁事件ID，为分析师打包相关遥测数据
- Cloudforce One分析师使用内部版TIP进行逆向工程或在全球数据集中进行关联分析
- 调查完成后，结果（新的IOC、攻击者归因或活动记录）将写回我们的全球情报源

这确保了"人类洞察"不会仅仅停留在PDF报告中。相反，生成的元数据会在相关时作为威胁事件推回边缘，供您已配置的WAF或防火墙规则使用。我们已从静态的"报告"模式转向动态的"代码化情报"模式，人类分析能直接实时提升平台的自动检测逻辑。

## 从数据管理到主动狩猎

从管理ETL管道转向主动威胁狩猎，不仅是界面的更新，更是计算位置的变革。通过将存储、聚合和可视化层移至Cloudflare全球网络，我们消除了通常拖慢SOC效率的"数据引力"。防御者不再需要等待日志同步到中央存储库才能询问"这个IP是否与已知攻击活动相关？"答案现在边缘即可获取，就在流量被过滤的同一环境中。

为确保不同团队规模或特定需求都能获取这些情报，我们将Cloudforce One访问权限划分为三个功能层级：

- **Cloudforce One基础版**：允许客户访问威胁事件中的默认数据集，搜索威胁指标并进行威胁狩猎调查。
- **Cloudforce One进阶版**：允许客户通过信息请求获取我们的威胁情报分析师定制洞察。
- **Cloudforce One精英版**：完整套餐，包含品牌保护、大量信息请求额度以及所有威胁事件数据集的访问权限。

Cloudforce One Elite 作为完整解决方案，包含品牌保护、海量信息查询权限及全量威胁事件数据集访问权限。

互联网瞬息万变，而攻击者使用的基础设施演进更为迅猛。通过将遥测数据与响应逻辑集中至统一集成平台，您可告别重复构建数据管道的工作，转而专注于网络防御实践。
