AI Gateway 现已支持视频生成，您可以通过 AI SDK 6 创建具有照片级真实感画质、同步音频的电影级视频，并生成具有一致身份特征的个性化内容。

## 两种入门方式

视频生成功能处于测试阶段，目前可供 Pro 版、企业版计划用户以及付费的 AI Gateway 用户使用。

- **AI SDK 6**：通过您用于处理文本和图像的同一接口，以编程方式生成视频。一个 API、一套身份验证流程、一个覆盖整个 AI 流程的可观测性仪表板。

```
1import { experimental_generateVideo as generateVideo } from 'ai';2
3const { videos } = await generateVideo({4  model: 'xai/grok-imagine-video',5  prompt: 'A golden retriever catching a frisbee mid-air at the beach',6});
```

- **AI Gateway Playground**：在嵌入每个模型页面的可配置 AI Gateway Playground 中，无需编写代码即可体验视频模型。您可以比较不同提供商、调整提示词并下载结果，无需编写代码。要访问此功能，请在模型列表中点击任意视频生成模型。

## 首批四个视频模型；17 种变体

- **xAI 的 Grok Imagine** 速度极快，且擅长遵循指令。可在几秒钟内完成视频的创建和编辑，并支持风格迁移。
- **阿里巴巴的 Wan** 专长于基于参考的生成和多镜头叙事，具备跨场景保持身份特征的能力。
- **Kling** 擅长图像转视频和原生音频生成。其新的 3.0 模型支持带自动场景转换的多镜头视频。
- **谷歌的 Veo** 提供高视觉保真度和物理真实感。支持原生音频生成，并具备电影级光照和物理效果。

## 理解视频生成请求

视频模型需要的不仅仅是描述您想要的内容。与图像生成不同，视频提示词可以包含运动提示（摄像机移动、物体动作、时间安排），并可选择性地包含音频指示。每个提供商通过 `providerOptions` 暴露不同的能力，这些选项能解锁根本不同的生成模式。请参阅文档以获取特定于模型的选项。

## 文生视频

描述您想要的内容，即可获得视频。模型负责处理视觉效果、运动，并可选择性地处理音频。仅需一个简单的文本提示词，即可生成超写实、制作质量的素材。

**示例：规模化编程视频生成。** 为您的应用、平台或内容流程按需生成视频。无需许可费或制作过程，只需提示词和输出。

此示例使用 `klingai/kling-v2.6-t2v` 根据文本提示词生成指定宽高比和时长的视频。

```
1import { experimental_generateVideo as generateVideo } from 'ai';2
3const { videos } = await generateVideo({4  model: 'klingai/kling-v2.6-t2v',5  prompt: `Wide shot of a rocket lifting off from launch pad at dawn.6    Massive plume of orange fire and white smoke billows outward7    from the base. The rocket rises slowly at first, engines blazing,8    then accelerates upward. Pink and orange sunrise sky9    in the background. Ocean visible in the distance.`,10  aspectRatio: '16:9',11  duration: 5,12  providerOptions: {13    klingai: {14      mode: 'pro',15      sound: 'on',16    },17  },18});
```

**示例：创意内容生成。** 将简单的提示词转化为具有自然运动和电影级画质的精美视频片段，用于社交媒体、广告或故事讲述。

通过设置非常具体和描述性的提示词，`google/veo-3.1-generate-001` 可以生成细节丰富且运动完全符合期望的视频。

```
1import { experimental_generateVideo as generateVideo } from 'ai';2

```

## 图生视频

`klingai/kling-v2.6-i2v` 模型在您传入图像 URL 和提示词中的运动描述后，会为产品图像添加动画。

**示例：动画插图。** 通过微妙的运动让静态艺术作品变得生动。非常适合主题内容或规模化营销。

## 首尾帧

定义起始和结束状态，模型将生成它们之间的无缝过渡。

**示例：前后对比揭示。** 服装更换、产品比较、随时间的变化。上传两张图像，获得无缝过渡。

在此示例中，起始和结束状态由提示词和提供商选项中使用的两张图像定义。

![](/images/posts/57edb5ac47b8.jpg)

在此示例中，`klingai/kling-v3.0-i2v` 允许您在 `image` 中定义起始帧，在 `lastFrameImage` 中定义结束帧。模型将生成它们之间的过渡。

```
1import { experimental_generateVideo as generateVideo } from 'ai';2
3const { videos } = await generateVideo({4  model: 'klingai/kling-v3.0-i2v',5  prompt: {6    image: startFrameDataUrl, 7    text: `平滑的电影感转场：空荡的阁楼逐渐被家具填满。8      一张绿色天鹅绒沙发淡入视野，随后是一张木质咖啡桌。9      盆栽植物从地板上生长出来。一张有图案的地毯逐渐显现。10      带框的艺术品出现在墙上。后墙上是书架。11      柔和、无缝的转变。`,12  },13  duration: 5,14  providerOptions: {15    klingai: {16      lastFrameImage: endFrameDataUrl, 17      mode: 'std',18    },19  },20});
```

## 参考到视频

提供人物/角色的参考视频或图像，模型会提取其外观和声音，生成以其为主角、身份一致的新场景。

在此示例中，使用了2张狗的参考图像来生成最终视频。

```
1import { experimental_generateVideo as generateVideo } from 'ai';2

```
