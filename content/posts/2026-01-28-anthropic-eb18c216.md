# ServiceNow 选择 Claude 赋能客户应用并提升内部生产力

随着企业不再仅仅进行 AI 实验，而是开始将其投入到核心业务运营的生产环境中，规模和安全性与能力本身变得同等重要。

考虑到这一点，ServiceNow 选择了 Claude 作为其 ServiceNow Build Agent 的默认模型，并作为 ServiceNow AI 平台的首选模型。ServiceNow 帮助大型企业在单一平台上管理和自动化从 IT 支持、人力资源到客户服务的所有事务。Build Agent 帮助各种技能水平的开发者构建和使用 Claude 作为一个能够推理、决策并自主执行任务的“智能体”。

此外，ServiceNow 正在其全球超过 29,000 名员工中推广使用 Claude 和 Claude Code，以简化销售准备工作——将销售人员的准备时间缩短了 95%，并通过 Claude Code 提升工程生产力，从而缩短整个组织从想法到实施的时间。

企业每年使用 ServiceNow 平台处理超过 800 亿个工作流程——从解决安全事件、新员工入职到管理客户支持队列。现在，企业可以利用 Claude 的推理和编码能力来驱动这些工作流程，同时满足企业级应用所需的访问控制、使用监控和合规性要求。

“企业在应用 AI 时常犯的一个错误是将其视为一种偶尔使用的‘附加’工具。但要获得更好的结果，应将 AI 融入日常工作方式中，成为不可或缺的一部分——编织到员工每天处理的各类事务中。只有这样，才能真正开始看到这些系统的能力，这也是我们与 ServiceNow 合作正在做的事情。”Anthropic 的联合创始人兼首席执行官 Dario Amodei 表示。

“ServiceNow 与 Anthropic 的合作，正在通过面向全球最大企业的 AI 原生工作流程，将智能转化为行动。”ServiceNow 董事长兼首席执行官 Bill McDermott 表示。“这次合作旨在重新构想工作的完成方式。它将构建、部署和扩展关键任务应用的能力，交到每个行业、每个层级、每个人手中。我们共同证明，拥有开放生态的深度集成平台是构建未来的方式。”

## 面向 ServiceNow 客户的 Claude

赋能企业应用开发：Claude 是驱动 ServiceNow Build Agent 的默认模型，这是一个用于使用 AI 构建应用和自动化的企业级编码解决方案。Build Agent 早期已获得显著关注：ServiceNow 预计其使用量在未来 12 个月内将翻两番。通过将 Build Agent 与 Claude 集成，各种技能水平的开发者，包括专业程序员和公民开发者，都可以使用自然语言提示词来创建以往需要大量开发者支持的应用，或者设计、测试和使用智能体自动化。客户在 ServiceNow 中使用 Claude 来处理需要更高级 AI 的任务，同时仍能完全掌控和了解 AI 的行为。

加速产品采用和价值实现时间：ServiceNow 也正与我们合作，改进客户部署和采用 ServiceNow 产品的方式。借助 Claude，ServiceNow 的目标是将客户的实施时间缩短 50%——减少从初次销售沟通到自主部署的延迟。客户和合作伙伴将能够使用相同的 AI 驱动方法来加速他们自己的部署。

将创新的 Claude 驱动解决方案应用于行业：ServiceNow 正在使用 Claude 为特定行业（如医疗保健和生命科学）构建智能体应用。在这些环境中，Claude 将支持研究分析、理赔授权等任务——所有这些都在 ServiceNow 受监管的平台上运行。Claude 是这些任务的行业领先 AI 模型*，Claude Opus 4.5 在主要的医学基准测试和生命科学评估中处于领先地位。在 ServiceNow AI 平台的支持下，理赔授权时间可能从数天缩短到数小时，同时降低成本。ServiceNow 和 Anthropic 将共同将这些创新的行业解决方案推向市场。

## ServiceNow 内部将如何使用 Claude

ServiceNow 正在为其全球员工部署 Claude，在内部应用与提供给客户相同的 AI 能力：

变革销售生产力：ServiceNow 销售人员使用 Claude 驱动的辅导工具来准备客户会议。该工具将 Claude 连接到实时企业数据和网络搜索，使销售人员能够在一个地方综合潜在客户信息、账户背景和其他相关材料。早期测试结果显示，准备时间最多可减少 95%，帮助销售人员专注于战略对话而非手动研究。

通过 Claude Code 提升生产力：ServiceNow 还在公司内部的工程师、开发人员和技术团队中推广了 Claude Code，这是我们的 AI 编码助手。团队使用 Claude Code 来编写和审查代码、调试问题、自动化重复性开发任务并加速内部工具开发——从而缩短整个组织从想法到实施的时间。
